# P2 설계: 채널 신규 영상 무인 자동 수집 (`/collect-new`)

- **날짜**: 2026-07-01
- **상태**: 설계 승인 대기
- **범위**: 로드맵 P2. P1(자막 Plan A)은 완료·의존. P3(digest)·P4(주제 메타)는 별도 스펙.
- **의존**: 기존 study-note 파이프라인 + `/batch-notes` 디스패처 + P1 자막 경로를 **재사용**. 앞단에 "채널 열거 → 신규 감지 → 필터"만 얹는다.

---

## 1. 배경 · 결정

영상(MJiQFNp-k10)은 구독 채널의 전날 영상을 스케줄러가 무인 감지→요약→저장한다. 우리도 같은 무인 자동을 채택하되, 원천 아카이브(`notes/`, 원문 보존)는 그대로 두고 **수집 트리거만** 추가한다.

**브레인스토밍 확정 결정**:
- **동작 모델**: 완전 자동(무인) — 승인 게이트 없음.
- **채널 소스**: `notes/*/` 폴더명에서 자동 도출 (폴더명 = `@handle` 슬러그 = 관심 채널).
- **시간창**: 어제 하루 기본, 실행 시 오버라이드 가능.
- **하드캡**: 실행당 최대 20편, 초과 시 skip + 경고 로그.

**기술 타당성 (2026-07-01 검증)**:
- 폴더명 → `@handle` → `channel_id`: `yt-dlp --playlist-items 1 --print "%(channel_id)s" "https://www.youtube.com/@<handle>/videos"` → `UC...`
- 최근 영상 id+게시일: `curl -s "https://www.youtube.com/feeds/videos.xml?channel_id=UC..."` → `<entry>`별 `<yt:videoId>` + `<published>` (최근 ~15편). awk `RS="<entry>"`로 per-entry 파싱 검증됨.
- 어제 날짜: `date -d "yesterday" +%Y-%m-%d` (Git Bash GNU date) 검증됨.
- ID 중복: `notes/**/*.md`의 `영상 ID:` 줄 대조 (기존 스킬 Step 0.5 메커니즘).

## 2. ⚠ CLAUDE.md 가드레일 충돌 · 해결

**충돌**: 완전 자동은 `batch-notes.md` Negative Space "**사용자 승인 없이 Step 2 진입 금지 — 잘못 붙여진 URL이 많은 비용**" 및 CLAUDE.md §4 "승인 후 진행"과 충돌.

**해결 (제거 아닌 맥락별 분화)**: 그 가드레일의 목적은 *폭주 비용 방지*이지 *인간의 클릭 그 자체*가 아니다. 무인 경로에서 같은 목적을 **구조적 4중 상한**으로 지킨다:

1. **시간창**: 어제 하루(기본) → 자연히 소량
2. **하드캡**: 실행당 ≤20편, 초과 시 skip + 경고
3. **ID 중복 검사**: 이미 `notes/`에 있으면 제외
4. **동시성 캡**: 기존 배치 3~5

→ **인터랙티브 `/batch-notes`는 승인 게이트 유지**, **무인 `/collect-new`(및 그 스케줄)는 4중 상한으로 대체**. CLAUDE.md·batch-notes.md의 해당 문구를 이 분화로 개정한다. (사용자 승인받은 §9 절차.)

## 3. 목표 · 비목표

**목표**
1. `/collect-new [--since=어제|YYYY-MM-DD|Nd] [--cap=20] [--channels=a,b]` 커맨드: 추적 채널의 신규 영상을 감지해 무인으로 study-note화.
2. 채널 열거·신규 감지·필터를 앞단에 두고, 노트화·INDEX 갱신은 기존 파이프라인 재사용.
3. Claude Desktop 로컬 루틴으로 매일 무인 실행하는 셋업을 **문서로** 제공 (루틴 생성 자체는 사용자 몫).

**비목표**
- P3/P4.
- 원문 보존·정정·인용 규칙 변경.
- 채널 구독 자체 관리(유튜브 구독 동기화 등) — 소스는 `notes/` 폴더로 한정.
- 원격(remote) 스케줄 — 로컬 루틴만 (컴퓨터 켜져 있을 때 동작, 영상과 동일).

## 4. 설계

### 4.1 흐름

```
1. 채널 열거: notes/*/ 폴더명 목록 → 각 <handle>
   (--channels 지정 시 그 부분집합만)
2. channel_id 확보: 캐시(.claude/collect-state.json) 우선, 없으면 yt-dlp resolve 후 캐시
3. 신규 감지: RSS 피드 → [id|게시일] 파싱
4. 필터 (순서대로):
   a. 시간창: 게시일 ∈ [--since 범위] (기본 어제)
   b. ID 중복: notes/**/*.md 에 영상 ID 이미 있으면 제외
   c. 하드캡: 남은 게 --cap(기본 20) 초과면 오래된 것부터 자르고 경고 로그
5. 노트화: 남은 URL 목록을 batch 파이프라인 무인 모드로 처리
   (게시일은 RSS에서 이미 확보 → worker에 "게시일(확정)"으로 전달, P1 자막 prefetch 포함)
6. INDEX 갱신: 기존 batch-notes Step 4 재사용
7. 요약 로그: 채널별 발견/스킵/생성 수, 하드캡 절단 여부
```

### 4.2 채널 열거 · channel_id 캐시

- `notes/*/` 디렉터리명이 곧 `@handle`. `INDEX.md` 등 파일은 무시, 디렉터리만.
- `.claude/collect-state.json`: `{ "<handle>": { "channel_id": "UC..." } }`. 채널당 resolve 1회(≈2s)를 캐시로 절약. 파일 없으면 생성. resolve 실패(채널 리네임/삭제) 시 그 채널만 skip + 경고, 나머지 진행.

### 4.3 신규 감지 · 필터

- RSS awk 파싱: `RS="<entry>"`, `<yt:videoId>`·`<published>` 추출. **채널 레벨 `<published>`(피드 상단)를 entry로 오인하지 않도록** `NR>1`부터 처리 (검증된 방식).
- 시간창: `--since` 파싱 — `어제`(기본)=`date -d yesterday`, `YYYY-MM-DD`=그 날짜 하루, `Nd`=최근 N일. 게시일 문자열 비교(`YYYY-MM-DD`).
- ID 중복: 후보 각 id를 `notes/**/*.md`의 `영상 ID:` 줄과 대조 (Grep). 이미 있으면 제외.
- 하드캡: 남은 후보 > cap이면 게시일 오름차순 정렬 후 **오래된 것부터 절단**(최신 우선 유지), 절단 수를 경고 로그로 남긴다 (silent truncation 금지).
- **RSS 포화 경고**: RSS가 15편(최대)을 반환하고 그 15편이 모두 시간창 안이면 "더 오래된 신규가 잘렸을 수 있음" 경고 (극단 케이스).

### 4.4 노트화 (무인 batch)

- 남은 URL 목록을 `/batch-notes` 파이프라인으로 넘기되 **Step 2 승인 게이트를 건너뛴다**(4중 상한이 이미 적용됨).
- batch-notes.md에 "무인 호출(`/collect-new`발) 시 승인 게이트 대신 상한 검증에 의존" 분기를 명시.
- 게시일은 RSS에서 확보했으므로 worker에 `게시일(확정)`으로 전달 → yt-dlp 게시일 재추출 생략. 자막은 P1 prefetch 경로 그대로.
- INDEX 갱신은 기존 Step 4.

### 4.5 스케줄링 (문서)

- `.claude/commands/collect-new.md` 하단(또는 `docs/`)에 Claude Desktop **로컬 루틴** 셋업 가이드:
  - 루틴 폴더 = 이 저장소, 지침 = "어제 업로드된 영상을 수집해 노트화해줘" → `/collect-new` 발동.
  - 로컬 루틴은 컴퓨터 켜져 있을 때만 동작(꺼져 있으면 다음 시각). 영상과 동일한 한계 명시.
- 루틴 생성은 사용자가 UI에서 직접. 우리는 지침 문자열·주의점만 제공.

## 6. 엣지 케이스

- **어제 영상 없음**: 정상. 채널별 0건 → 전체 0건이면 "신규 없음" 로그 후 종료.
- **channel_id resolve 실패**: 그 채널 skip + 경고, 나머지 진행.
- **RSS 접근 실패(네트워크)**: 그 채널 skip + 경고.
- **하드캡 초과**: 최신 우선 유지, 절단 수 경고.
- **폴더는 있으나 실제 영상 전부 이미 노트화**: ID 중복으로 전부 제외 → 0건.
- **동일 영상이 두 채널 RSS에 (거의 없음)**: ID 중복 검사가 채널 무관 전역이라 한 번만.
- **부분 실패**: 한 채널·한 영상 실패가 나머지를 막지 않음 (기존 격리 원칙).

## 7. 검증 신호

- notes/ 폴더만으로 채널 목록이 도출됨 (별도 리스트 없이).
- 어제 영상이 있는 채널에서 그 영상만 신규로 잡히고, 이미 있는 영상은 제외됨.
- 하드캡 초과 시 절단 수가 로그에 남음 (silent 아님).
- `/collect-new`가 만든 노트가 인터랙티브 `/batch-notes` 산출물과 구조·품질 동일 (같은 파이프라인).
- 무인 실행에 사용자 승인 프롬프트가 없음. 대신 4중 상한이 적용됨.
- channel_id가 캐시되어 2회차 실행이 resolve를 생략함.

## 8. 로드맵
