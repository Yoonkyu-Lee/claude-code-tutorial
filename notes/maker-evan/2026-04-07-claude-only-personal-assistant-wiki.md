## 한 줄 요약
ChatGPT·Claude·Notion AI·Perplexity로 흩어진 AI 구독을 끊고, Claude 구독 하나 + Mac mini로 "나를 아는" 개인 비서(이메일·캘린더 알림)와 자동 업데이트되는 위키(Obsidian 메모 → Claude가 정리)를 코드 한 줄 없이 구성하는 방법.

## 메타
- **영상 유형**: 혼합형 (개념 설명 + 비개발자용 셋업 가이드)
- **타임스탬프**: 제공됨 (`get_timed_transcript`)
- **대상**: 비개발자 포함. 코드 작성 불필요, 설정 파일 몇 개만 수정
- **준비물**: Claude 구독(Max 플랜 권장), Mac mini(또는 기존 Mac / VPS), iPhone(iCloud 동기화용)

## 핵심 개념

### 1. 문제 정의 — 흩어진 AI 구독과 "나를 모르는" AI
화자는 ChatGPT[^1], Claude, Notion AI[^2], Perplexity[^3] 등 여러 AI 구독을 각각 따로 쓰는 상황을 문제로 짚는다.

> "각각 따로 써야 하잖아요. 이건 여기서 저건 저기서 내 정보가 여기저기 흩어져 있고 AI는 내가 언제 뭘 했는지도 몰라요." [00:24]

[^1]: 자막 "채피pt / 채치T / 채피" → ChatGPT.
[^2]: 자막 "노션 AI" → Notion AI.
[^3]: 자막 "펄플렉시티" → Perplexity.

### 2. Andrej Karpathy의 "AI 비서는 나를 아는 시스템" 주장
화자는 테슬라 前 AI 총괄 Andrej Karpathy[^4]가 최근 쓴 글을 인용한다.

> "AI 비서는 결국 나를 아는 시스템이어야 한다. 내 메모, 내 일정, 내 이메일 그리고 내 위키, 내 정보들을 다 알고 있어야 진짜 비서다." [00:37]

[^4]: 자막 "안드레스 카파 / 안드레드 카파" → Andrej Karpathy (영상 내 "테슬라 AI 총괄"이라는 컨텍스트로 확정). 영상은 Karpathy 발언을 화자가 옮긴 것이므로 정확한 원문 인용이 아닌 화자의 요약일 수 있음.

`[의견]` 화자는 현재 ChatGPT가 이 "나를 아는" 역할을 못 한다고 본다:

> "근데 지금 채치T가 그걸 해 주나요? 못 해요. 매번 처음부터 설명해야 돼요." [00:46]

### 3. 시스템의 세 가지 구성요소
화자는 시스템을 딱 세 가지로 정리한다 [00:58]:

1. **내 두뇌 = 위키**: iPhone에서 Obsidian[^5] 앱으로 메모 → 자동 싱크 → Claude가 언제든 꺼내 봄
2. **내 비서 = Claude Code Desktop**: Mac에서 백그라운드로 돌며 이메일 확인·캘린더 확인 → 중요한 게 있으면 연락
3. **연락 창고 = Telegram**: Claude가 사용자에게 연락할 때 쓰는 채널

[^5]: 자막 "옵시디언" → Obsidian.

> "에반님 30분 뒤에 미팅 있어요. 안 읽은 중요한 메일 세 개 있어요. 이런 식으로요. 이게 전부예요. 복잡한 개발 없어요. 코드 한 줄 안 써도 돼요." [01:18]

### 4. LLM Wiki — 지식이 쌓일수록 똑똑해지는 위키
화자는 이 시스템의 가장 핵심을 Karpathy가 공개한 "LLM 위키"[^6] 개념으로 설명한다.

> "내가 새 문서를 저장하면 AI가 그걸 읽고 기존 위키 페이지들을 알아서 업데이트해요. 연관된 내용끼리 링크도 연결하고 모순되는 내용도 정리해 줘요. 그러니까 지식이 쌓일수록 위[키]가 똑똑해지는 거예요. 내가 정리 안 해도." [03:55]

[^6]: 자막 "lm 위키" → LLM wiki (Karpathy의 개념. 외부 보강 섹션 참조).

`[의견]` 화자가 보는 이 개념의 의의:

> "지금까지 지식 관리에 가장 큰 문제가 뭐였냐면 정리하는 게 너무 귀찮다는 거였거든요. ... 근데 AI가 그 귀찮은 일을 대신해 주면 어떨까요? 그게 바로 이 시스템이에요." [04:07]

### 5. Dispatch — 폰에서 던진 질문이 Mac에서 실행
Claude 모바일 앱의 Dispatch[^7] 기능으로, iPhone에서 질문하면 그 질문이 Mac에서 실행되고 위키를 참고한 답이 돌아온다.

> "이동 중에 문득 생각이 났어요. 아 그 고객사 담당자 이름이 뭐였지? 디스패치에서 물어보면 내 노트에서 찾아줘요. 다음 주 캘린[더] 어때? 미팅 많아? 이것도 돼요. 비서한테 문자 보내는 느낌이에요." [05:03]

[^7]: 자막 "디스패치" → Dispatch (Claude 모바일 앱 기능. 외부 보강 섹션 참조).

## 셋업 워크플로 (영상 순서대로)

영상은 비개발자 대상이라 명령어 디테일보다 단계 흐름 위주다. 정확한 설정 파일 키·전체 명령어는 영상에 나오지 않음 (`[불명확]`은 해당 부분에 표기).

### Step A. 준비물 결정 — Claude 구독 + 머신
- **Claude 플랜**: Max 권장. Pro는 사용량 제한 때문에 자동화를 돌리면 금방 막힘.
  > "프로는 사용량 제한이 있어서 자동화 작업 돌리다 보면 금방 막혀요. 맥스는 그 제한이 훨씬 넉넉해서 하루 종일 돌려도 끝[이] 없어요." [01:33]
- **머신 옵션**:
  - Mac mini[^8] 신규 구매 — 가장 저렴한 모델 기준 약 90만 원. "한 번 사면 끝이고 전기세도 별로 안 나와요. ... 24시간 알아서 돌아가요." [02:03]
  - 기존 Mac 사용 가능. 단 "자[는] 동안에도 [켜는] 게 싫으신 분들은 맥[mini] 추천"
  - Mac 없는 사람 → VPS. `[의견]` "솔직히 비개발자분들한테는 좀 까다로워요." [02:17]

[^8]: 자막 "맥민이 / 맥니 / 맨 / 맥 이미" 등 → Mac mini로 정정 (가격·24시간 구동 컨텍스트로 확정). 단 [02:09] "맥 이미 있으신 분들"은 "Mac 이미 있으신 분들"로 일반 Mac을 가리킴.

`[의견]` 오픈소스/터미널 기반 대안(OpenClaw[^9])과의 비교:
> "오픈 클로[로]도 비슷한 시스템 만들 수 있어요. 근데 오픈 클로는 터미널 열고 명령어 치고 환경 설정하고 비개발자한테는 [진]입 장벽이 꽤 있어요. 클로드[ ]코드 데스크탑 앱은 그것보다는 훨씬 낫거든요. GUI로 되어 있어서 ... 수월하게 시작할 수 있어요." [01:43]

[^9]: 자막 "오픈 클로 / 오픈 클로스" → OpenClaw (정정 사전 G/B 항목 기준). 단 사전에 "OpenClaw 또는 OmniClaw"로 표기되어 있어 정확한 제품명은 `[불명확]` 여지 있음.

### Step B. Telegram 봇 연결 (약 5분)
> "텔레그램 설정이 제일 쉽다고 생각됩니다." [02:25]

1. Telegram에서 BotFather[^10] 검색
2. 채팅에 `/newbot`[^11] 입력 → 봇 이름 지정 → 토큰 발급
3. 그 토큰을 Claude Code에 연결

[^10]: 자막 "보파더" → BotFather (Telegram 공식 봇 생성 봇).
[^11]: 자막 "뉴봇 보" → `/newbot` (BotFather 명령어).

> "그럼 이제 클로드가 저한테 문자를 보낼 수 있게 됩니다. ... AI가 저한테 먼저 연락을 하는 거잖아요." [02:37]

### Step C. Claude Code Desktop 앱 설치 + 스케줄 태스크
1. Claude 공식 사이트에서 Claude Code Desktop[^12] 앱 다운로드·설치
2. Telegram 플러그인 켜기 — "설정 파일 하나만 건드리면 돼요." [02:49] (`[불명확]` 어떤 설정 파일·어떤 키인지는 영상에 없음)
3. **스케줄 태스크 설정** — 화자가 "이게 핵심"이라 강조 [02:53]. Claude에게 자연어로 지시:
   > "매 시간마다 내 [Gmail][^13] 확인해서 안 읽은 메일 있으면 텔레그램으로 알려 줘. 30분 뒤 미팅 있으면 미리 알려 줘. 매일 아침 9시에 오늘 일정 요약해 줘." [02:57]

[^12]: 자막 "클로드 코드 데스크탑" → Claude Code Desktop.
[^13]: 자막 "GMIL" → Gmail.

> "이게 자동으로 돌아가요. 제가 뭔가를 하지 않아도요. 대신 PC는 켜져 있어야 합니다." [03:07]

**실제 동작 예시** [03:13]:
- 아침: "오늘 일정입니다. 오전 10시 팀 미팅, 오후 2시 클라이언트 콜. 오늘 안 읽은 중요 메일 두 개 ..."
- 미팅 30분 전: "30분 뒤 팀 미팅 시작합니다."

### Step D. 위키 — Obsidian + iCloud 동기화
1. iPhone에서 Obsidian으로 메모
2. Obsidian 보관함을 **iCloud**[^14]에 올림 → Mac으로 자동 싱크
   > "동기[화]는 아이클라우드로 해요. 아이폰에서 메모하면 맥[에] 자동으로 싱크돼요. 따로 설정할 것도 없어요. 이미 아이폰 쓰고 계시면 바로 됩니다." [04:45]
3. Claude가 싱크된 노트를 참고해 위키로 정리·질의응답
   > "내 [불명확: 원문 'VRL'] 프로젝트 [요]약해 줘. 저[장]된 메모에[ ]든 마케팅 아이디어 뭐 있었지? 이렇게 물어볼 수 있어요. 클로드가 내 노트에서 찾아서 알려 줘요." [04:38]

[^14]: 자막 "아이클라우드" → iCloud.

### Step E. (선택) Dispatch로 모바일에서 사용
Step C·D가 돌아가면, Claude 모바일 앱의 Dispatch로 외출 중에도 같은 위키·캘린더에 질의 가능 (위 핵심 개념 5 참조).

### Andrej Karpathy의 LLM Wiki (2026-04, GitHub gist)
- Karpathy(OpenAI 공동창업자·前 테슬라 AI 총괄)가 2026년 4월 공개한 "idea file" 형태의 개념. 새 앱·라이브러리·SaaS가 아니라 Claude Code나 Codex 같은 에이전트에 붙여 쓰는 **개념 패턴**.
- RAG와의 차이: RAG는 질의할 때마다 문서 청크를 새로 검색·재발견함(누적 없음). LLM Wiki는 새 소스를 넣으면 LLM이 읽고 **기존 위키 페이지를 점진적으로 갱신** — 엔티티 페이지 업데이트, 토픽 요약 수정, 모순 표시.
- 슬래시 명령 3종: **Query**(인덱스 읽고 관련 페이지로 답변+인용), **Ingest**(소스를 `wiki/raw/`에 복사 후 영향받는 페이지 갱신·신규 페이지 생성), **Lint**(모순·낡은 내용·고아 페이지·누락 상호참조·인덱스 드리프트 감사).
- 출처: [karpathy/llm-wiki gist](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f), [Analytics Vidhya 해설](https://www.analyticsvidhya.com/blog/2026/04/llm-wiki-by-andrej-karpathy/), [nashsu/llm_wiki (구현체)](https://github.com/nashsu/llm_wiki)

### Claude Dispatch (Cowork 기능)
- 모바일 기기와 데스크톱 Claude를 QR 코드로 페어링하는 **원격 제어** 기능. 폰에서 작업을 던지면 데스크톱(macOS/Windows)에서 실행, 완료 시 알림.
- 클라우드 컴퓨팅이 아니라 원격 제어 — 데스크톱을 끄거나 앱을 닫으면 세션도 멈춤(영상의 "PC는 켜져 있어야" 발언과 일치).
- 요구사항: 최신 Claude Desktop 앱 실행 중 + 컴퓨터가 깨어 있어야 함. Pro·Max 플랜 베타로 Claude Cowork에서 제공, Desktop 앱 + 모바일 앱 둘 다 필요.
- 출처: [Claude Code Docs — Remote Control](https://code.claude.com/docs/en/remote-control), [Claude Help Center — Assign tasks from anywhere](https://support.claude.com/en/articles/13947068-assign-tasks-to-claude-from-anywhere-in-cowork), [MacRumors (2026-03-24)](https://www.macrumors.com/2026/03/24/claude-use-mac-remotely-iphone/)

## 미해결 / 검수 포인트
- **`[불명확]` 마커**: 위키 질의 예시의 "VRL 프로젝트" [04:38] — `VRL`이 무엇인지 컨텍스트로 확정 불가. 사용자 본인 프로젝트 약어 추정이나 단정 안 함.
- **OpenClaw vs OmniClaw**: 정정 사전상 음차 "오픈 클로"가 둘 다 가능. 정확한 제품명 미확정 (`[불명확]` 여지).
- **설정 파일 디테일 부재**: Telegram 플러그인을 켜는 "설정 파일 하나" [02:49]의 정확한 경로·키, Claude Code의 스케줄 태스크(cron/스케줄러) 설정 방법이 영상에 안 나옴. 비개발자용 개요 영상이라 실제 구축 시 별도 문서 필요.
- **Karpathy 인용의 정확성**: [00:37] 인용은 화자의 요약일 수 있어 원문 그대로가 아닐 가능성. LLM Wiki gist 원문과 대조 권장.
