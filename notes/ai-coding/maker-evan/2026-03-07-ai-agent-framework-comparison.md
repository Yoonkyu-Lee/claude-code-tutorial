### 1. LangChain — 화자 비추천

- **노출도**: 검색하면 제일 많이 나오고 튜토리얼도 제일 많음. AI 에이전트 만들기 영상은 거의 다 여기서 시작 [00:43]
- **화자 평가** `[의견]`:

> "근데 저 이거 비춥니다. 명확하게 말할게요." [00:48]

- 화자가 든 실사용 마찰점 [01:02]:
  - 일부만 바꾸려 해도 알 수 없는 용어를 계속 파야 함

### 3. Google ADK — 화자 비추천 (버그 + 신뢰 문제)
- **태생**: 2025년 구글이 공개. 여러 AI를 만들어 팀처럼 일하게 한다는 아이디어 [02:23]
- **잘되는 부분** [02:33]:
  - 각 AI에 역할 부여, 순차/동시 실행 지정 가능
  - AI가 작업 전에 필요한 정보를 미리 가져오거나 데이터를 저장·로드하는 부분이 생각보다 쉽게 붙음
  - 참고 예제 코드가 꽤 많음
- **막히는 부분** [02:43]:
  - 정해진 방식 그대로 쓰면 되지만, 그 틀에서 조금만 벗어나려 하면 갑자기 안 됨
  - 우회하려면 **몽키패치**(monkey patch)를 여러 군데서 해야 함 — "원래 안 되는 걸 임시방편으로 꼬아서 억지로 되게 만드는 것" [02:51]
  - 버그가 많음 (기본 기능이 안 되거나, 어제 되던 게 오늘 안 됨)
- **더 근본적인 문제** `[의견]`:

> "구글이 만들었다는 거 자체가 저는 좀 신뢰가 안 가요. 구글은 오픈소스 도구 만들어 놓고 나중에 조용히 손 떼버린 전적이 엄청 많아요." [03:08]

> "ADK도 지금은 열심히 업데이트 되지만 1, 2년 뒤에 어떻게 될지 솔직히 장담을 못 하겠어요." [03:24] `[의견]`

### 4. Pydantic AI — 화자가 가장 기대 (출력 일관성)
- **푸는 문제**: AI 응답을 정해진 포맷(structured output)으로 받게 함 [03:43]

> "이게 솔직히 제가 제일 기대하고 있는 거예요. 만든 팀이 개발자들 사이에서 꽤 신뢰받는 팀이에요. 좋은 도구 만드는 사람들로 알려진 곳이거든요." [03:37] `[의견]`

> "이 형식으로만 대답해라고 딱 정해 놓으면 항상 그 형식으로만 나와요. 이상하게 대답하면 알아서 다시 해요." [03:53]

- **단점** [04:02]:
  - 최근 도구라 큰 서비스 적용 사례가 많지 않음
  - 막히면 검색해도 답이 잘 안 나오는 경향
  - 다만 docs가 매우 잘 되어 있어, AI에게 docs를 주고 맡기면 잘 처리한다는 게 화자 의견 `[의견]` [04:09]

### 5. Claude Agent SDK — 화자 추천 (파일·코드 자동화)
- **다른 점**: 다른 도구는 "어떻게 AI를 만들까"를 다루는데, 이건 "AI가 뭘 할 수 있게 할까"를 다룸 [04:17]

- 폴더에서 문제 찾아 보고서 만들기 같은 작업을 AI가 직접 파일을 열고 읽고 수행. 기능을 따로 만들 필요 없이 내장 [04:33]
- 여러 AI 역할 분담 협업, 중간에 멈췄다가 이어서 작업하기 가능 [04:42]
- **단점** [04:46]:
  - Claude만 씀 → Anthropic 유료 계정 없으면 못 씀, 다른 AI로 교체 불가
  - 직접 만든 기능 붙이기가 다소 번거로움
  - 다른 도구보다 시작이 좀 어려움
- **언제 쓰면 좋나**: 파일 작업·코드 관련 자동화. "클로드 코드 경험이 그대로 들어온다" [04:59]

## 외부 보강
영상은 각 프레임워크를 화자의 경험·의견 중심으로 다루므로, 객관적 정체성만 공식 출처로 보강한다. (영상 내용과 분리)

- **Pydantic AI**: Pydantic(파이썬 데이터 검증 라이브러리, FastAPI의 기반)을 만든 팀이 내놓은 에이전트 프레임워크. 핵심 기능은 LLM 출력을 Pydantic 모델로 강제하는 structured/validated output. 출처: [Pydantic AI 공식 문서](https://ai.pydantic.dev/) `[불명확: 영상 시점 기준 정확한 안정성·버전 상태는 미확인]`
- **Google ADK (Agent Development Kit)**: 구글이 공개한 멀티 에이전트 오케스트레이션 프레임워크. sequential/parallel agent 구성, 툴 연동 지원. 출처: [Google ADK 문서](https://google.github.io/adk-docs/)
- **Claude Agent SDK**: Anthropic이 Claude Code의 에이전트 능력(파일 읽기/쓰기, 도구 사용, 멀티턴 작업)을 프로그래밍 인터페이스로 노출한 SDK. 출처: [Anthropic Claude Agent SDK 문서](https://docs.anthropic.com/en/api/agent-sdk/overview)
- **TensorFlow vs PyTorch 비유**: 화자가 든 역사적 전환(연구·산업이 TensorFlow에서 PyTorch로 이동)은 딥러닝 커뮤니티에서 널리 회자되는 일반론. 영상은 이를 "선발 주자가 비직관적이면 결국 뒤집힌다"는 프레임으로만 사용.
