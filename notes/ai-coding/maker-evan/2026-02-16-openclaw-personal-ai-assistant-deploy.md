# OpenClaw로 24시간 개인 AI 비서 배포하기

## 출처
- 영상: [오픈클로로 생산성 10배 향상 시키기](https://www.youtube.com/watch?v=JHZeaQj2Q74)
- 영상 ID: `JHZeaQj2Q74`
- 채널: 메이커 에반
- 게시: 2026-02-16
- 게시일 출처: 디스패처 확정값 (메인이 yt-dlp로 확보)

> 자막 정정 주의: 한국어 자동자막이 `오픈 클로`/`오픈클로`/`오픈클론`으로 잡은 것은 모두 **OpenClaw**(프로젝트명)로 보정했다. `레일이`/`레일웨이`는 클라우드 호스팅 서비스 **Railway**, `파워스 AI`/`파이워스`/`파이어스`/`5월스 AI`는 모델 프로바이더 **Fireworks AI**(컨텍스트로 추정, [불명확] 항목 참조), `클루트허브`/`클로드하`는 플러그인 마켓 **ClawHub**, `채비pt`/`채치T`/`최치PT`는 **ChatGPT**로 보정했다.

## 한 줄 요약
오픈소스 AI 비서 게이트웨이 **OpenClaw**를 Railway 클라우드에 배포해 텔레그램 등 메신저에서 24시간 돌아가는 개인 AI 비서를 만드는 방법과, 반드시 알아야 할 보안 주의 사항을 정리한 실습 가이드.

## 메타
- 영상 유형: 튜토리얼/실습형 (배포 워크플로 + 보안 체크리스트)
- 길이: 약 6분
- 자막: 한국어 자동 생성 (타임스탬프 제공)
- 핵심 도구: OpenClaw, Railway, Fireworks AI (GLM5), 텔레그램

## 핵심 개념

### OpenClaw란
- 화자 설명: 오픈소스 개인 AI 비서 게이트웨이.

> "오픈 클로는 오픈소스 개인 AI 비서저 게이트웨이에요. 쉽게 말하자면 여러분이 쓰는 와츠앱이나 텔레그램, 슬랙, 디스코드 이런 메신저에 AI를 직접 연결해 주는 거예요." [00:28]

- 지원 채널: 12개 이상 [00:37]
- 핵심 차별점 — 자체 서버에서 직접 운영:

> "채치T 앱처럼 남의 서버에 의존하는게 아니라 내가 원하는 서버에서 직접 돌리는 거예요. 어떤 AI 모델을 쓸지, 내 데이터를 어디로 보낼지 전부 내가 컨트롤할 수 있다는게 핵심이에요." [00:41]

### 프로젝트 배경 (화자 서술)
- 제작자: 오스트리아 개발자. 이전에 회사를 크게 매각, 이후 3년간 43개 프로젝트를 실험했고 그 중 하나가 "터졌다(크게 성공했다)". 최근 OpenAI에 합류. [00:50] *(외부 보강: 제작자는 Peter Steinberger — 외부 보강 섹션 참조)*

> "이전에 회사를 크게 매각했고 [맥에서] 3년간서 43개 프로젝트를 실험했는데 그 중 하나가 터졌습니다. 최근에는 오픈 AI에 합류했어요." [00:51]

- 이름 변천사 — 일주일 만에 세 번 변경:

> "원래 이름이 클로드봇이었거든요. 클로드에서 따온 이름인데 앤스로픽이 상표권 문제로 이름 바꿔 달라 했어요. 그래서 몰트보로 바꿨는데 발음이 영 별로라 3일 만에 또 바꿔서 오픈 클로가 된 거예요. 일주일 만에 이름 세 번이 바뀐 거죠." [01:01]

  - ClawdBot → Moltbot(상표권 이슈) → OpenClaw(발음 문제)
- 인기 지표: GitHub 스타 145,000개 이상, 포크 20,000개 이상. [01:13]

## 배포 워크플로 (시간순)

화자가 제시한 전체 과정은 3단계.

> "하나, 오픈 클로를 설치하고 둘, AI 모델을 연결하고 셋 레일이에 배포하면 끝이에요." [01:27]

### 1단계: 설치
- 터미널에서 명령 몇 줄로 끝남. [02:27]
- 상세 절차는 영상 설명란의 가이드 문서 링크 참조 (영상이 직접 명령을 보여주지 않음).

### 2단계: AI 모델 연결 — 중요한 갈림길

화자가 강조한 분기점. 용도에 따라 연결 방식이 달라진다.

**(A) 내 컴퓨터에서만 쓸 때 → ChatGPT 구독 방식**
- ChatGPT Plus 월 20달러, Pro 월 200달러로 추가 비용 없이 GPT-5 사용 가능. [01:38]
- **제약**: 내 컴퓨터에서만 동작. 서버 배포 시에는 사용 불가. [01:44]
- **위험 경고** (화자 강조):

> "오픈 AI의 인증 방식을 우회하는 거라 가지고 자칫 잘못하다가는 밴을 먹을 수가 있습니다." [01:51]

**(B) 서버 배포(24시간 운영) → API 키 방식**
- 추천: **Fireworks AI**의 **GLM5** 모델. [02:02]

> "100만 토큰당 [0.8달러 ... ] 8달러밖에 안 안 하는 가성 위가 미친 모델이에요." [02:06]
> [불명확: 자막이 "0.8달러밖에 8달러밖에"로 가격을 두 번 다르게 말함 — 100만 토큰당 정확한 단가가 불명확. 원문 "100만 토큰당 0.8달러밖에 8달러밖에 안 안 하는"]

- 대안: OpenAI나 Anthropic 키도 가능하나 매우 비쌈. [의견] "돈이 많으시다면 이거를 추천드리는데 그게 아니라면 저는 [Fireworks]를 추천드립니다." [02:14]

**정리** [02:18]:
- 내 컴퓨터 전용 → ChatGPT 구독
- 서버 24시간 운영 → Fireworks AI API 키

### 3단계: Railway에 배포
- Railway 선택 이유 (화자):

> "기터브에 코드 올려 놓으면 자동 배포를 해 주고요. 서버가 죽으면 알아서 다시 살려주고 가격도 굉장히 저렴하거든요." [02:47]

- 절차 [02:54]:
  1. Railway 가입 후 GitHub 프로젝트 업로드 → Railway 대시보드에서 자동 배포
  2. **저장 공간(storage) 추가**
  3. **환경 변수 3개만 입력** (나머지 설정은 프로젝트 안에 포함되어 건드릴 것 없음)
  4. 배포 완료 시 나오는 주소로 접속 → 설정 화면에서 텔레그램 등 메신저 연결

### 비용 (화자 추정)
- Railway 서버비: 월 약 5달러 [03:16]
- Fireworks AI 사용량: 월 5~25달러 추정 (사용량에 따라 변동) [03:20]
- 합계: 월 30~40달러 [의견] "개인 AI 비서치고는 나쁘지 않[은] 금액이라고 생각이 됩니다." [03:27]

## 보안 주의 사항 (화자가 가장 강조한 부분)

> "그전에 반드시 알아야 할게 있어요. 오픈 클로는 보안 이슈가 굉장히 많거든요." [03:33]

### 알려진 취약점 (화자 서술)
- 2026년 1월 말 보안 감사에서 **총 512개 취약점** 발견, 그중 **8개가 크리티컬 등급**. [03:37]
- 가장 심각한 것 — 원클릭 장악:

> "악기적인 링크를 한 클릭하면 공격자가 내 AI 비서를 장악할 수 있습니다." [03:46]

- 플러그인 마켓 **ClawHub**: 5,700개 넘는 플러그인 중 약 341개가 악성. 가짜 암호화폐 지갑·유틸리티로 위장해 악성 코드 설치. [03:53]
  - [불명확: 악성 개수를 자막이 "341G"로 표기 — "341개"로 추정. 외부 보강의 ClawHub 수치와 차이 있음, 외부 보강 섹션 참조]
- **프롬프트 인젝션**: 이메일·웹페이지에 숨겨진 악의적 명령을 AI가 읽고 실행할 수 있음.

> "이건 AI 에이전트 전반의 근본적인 한계긴 합니다." [04:12]
> *(자막 "푸른푸트 인젝션" → prompt injection 보정)*

### 화자가 실제로 지키는 5가지 안전 수칙 [04:23]
1. **강력한 비밀번호 설정** — 글자 하나로도 설정되니 절대 그러지 말 것. [04:25]
2. **항상 최신 버전 유지** — 2~3일마다 보안 업데이트, 한 번에 40개 넘는 취약점이 패치되기도. [04:31]
3. **ClawHub 플러그인 함부로 설치 금지** — 특히 새로 만들어진 계정이 올린 것은 의심. [04:38]
4. **중국 AI 모델 경유 주의** — GLM5·딥시크 등 중국 모델은 반드시 **Fireworks나 Together AI**를 경유. 같은 모델이라도 미국 서버에서 돌아 데이터가 중국으로 가지 않음. [04:44]

> "중국 회사에 직접 연결하면 내 데이터가 전부 중국 서버로 가요. 절대 직접 연결하면 안 됩니다." [04:56]

5. **민감 정보·금융 데이터는 넣지 말 것** — 일정 관리·메모 정리 같은 가벼운 작업 위주로 맡기는 게 안전. [05:01]

### 화자의 균형 잡힌 입장
> "근데요. 이런 보안 이슈가 있다고 못 쓰는게 아니에요. 제대로 알고 대비하면 충분히 안전하게 쓸 수 있거든요." [04:16]

### 보안 감사 수치 (영상 주장 검증)
- **512개 취약점 / 8개 크리티컬**: Argus Security Platform의 첫 공식 감사(2026-01-25)에서 발견된 수치와 일치. 영상 주장이 정확.
- 대표 크리티컬: **CVE-2026-25253** (CVSS 8.8) — cross-site WebSocket hijacking을 통한 원클릭 원격 코드 실행. 영상의 "악의적 링크 한 클릭 → 비서 장악" 설명과 부합.
- **ClawHub 악성 플러그인**: 외부 자료는 "1,400개 이상의 악성 skill이 ClawHub에서 활동(ClawHavoc 캠페인, macOS AMOS 인포스틸러를 생산성 도구로 위장)"이라고 보고. 영상의 "341개" 수치는 감사 시점·집계 기준 차이로 보이며, **악성 플러그인이 다수 존재한다는 방향성은 일치**.
  - 출처: [betterclaw.io — OpenClaw Security 2026](https://www.betterclaw.io/blog/openclaw-security-2026), [Kaspersky 블로그](https://www.kaspersky.com/blog/openclaw-vulnerabilities-exposed/55263/)
- 2026년 4월 기준 누적 138개 이상 CVE 공개 (CVSS 9.9 등급 포함). 영상 게시(2026-02-16) 이후에도 취약점이 계속 누적된 정황. (참고용, 영상 시점 밖)
