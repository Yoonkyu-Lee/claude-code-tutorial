# Claude Cowork Business Automation

Claude Cowork의 Schedule 기능과 Playwright MCP·Claude for Chrome 확장만으로 인스타그램 모니터링·마케팅 트렌드 리서치·X→Threads 자동 포스팅 등 루틴 업무를 자연어 프롬프트로 자동화하는 사례 시연.

## 출처
- 영상: [클로드 코워크로 업무 자동화 끝냈습니다](https://www.youtube.com/watch?v=ErsFDGqAyQc)
- 영상 ID: `ErsFDGqAyQc`
- 채널: 메이커 에반 (@maker-evan)
- 게시: 2026-06-13 (호출 측 확정값)

## 한 줄 요약
Claude Cowork의 Schedule 기능에 Playwright MCP(카카오톡 "나와의 채팅" 전송용)와 Claude for Chrome 확장(브라우저 컨트롤)만 연결하면, 코딩 지식 없이 자연어 프롬프트 몇 줄로 인스타그램 신규 게시물 알림·일일 마케팅 트렌드 요약·X→Threads 자동 포스팅 같은 루틴 업무를 매일 아침 자동 실행시킬 수 있다는 점을 세 가지 사례로 시연.

- 자막: 한국어 자동 자막 (타임스탬프 포함)

## 핵심 개념

### Claude Cowork의 위치
화자의 정의:

> "클로드 코드나 혹은 챗보다 훨씬 자유도가 높은데 훨씬 편리한 그런 기능들이 있다고 보시면 될 거 같고요." [00:19]

> "이 클로드 코드 코드가 아니라 코워크를 이용을 해 가지고 훨씬 다양한 업무와 그리고 스케줄된 업무들, 루틴한 업무를 이제 자연어를 통해서 이제 생성을 할 수 있다고 보시면 될 것 같습니다." [00:27]

### Schedule 기능의 의미
> "이 스케줄가 뭐 하는 친구냐면 말 그대로 어떤 루틴한 업무를 매일매일 실행을 해 주는 그런 친구라고 보시면 될 거 같습니다." [02:45]

### "묻지 않고 실행" 모드
스케줄 작업이 사용자 개입 없이 돌게 하려면 작업 편집 화면에서 **묻지 않고 실행** 옵션을 켜고 저장해야 한다.

> "먼저 여기서 해 주셔야 될게 질문 대신에 묻지 않고 실행 기능을 실행을시켜 주시면 됩니다." [03:02]

> "주의하실 점이 뭐냐면 편집에 들어가셔 가지고 어 이거를 묻지 않고 실행 버튼을 하신 다음에 저장을 하시는 걸 추천드립니다." [05:20]

## 환경 셋업

영상에서 시연한 셋업 순서를 그대로 옮긴다.

### 1) Playwright MCP 연결 (카카오톡 "나와의 채팅" 전송용)
- Claude 데스크톱 앱 좌하단 **맥스 버튼** → **설정** → **커넥터** → **맞춤 설정**
- 웹에서 **커넥터 불러오기** → **Playwright MCP** 선택 → 연결 [00:54]
- 연결 후 표시되는 커넥터 URL을 실제 웹페이지에서 열어 **카카오톡 로그인** [01:15]
- 로그인 후 **카카오톡 나챗방** 친구를 클릭 → **도구함에 추가** [01:26]
- Claude 앱 재시작 시 Playwright MCP의 "쓰기/삭제 도구 — 카카오톡 나챗방"이 노출 → **항상 허용** 설정 [01:39]

[불명확: 화자는 "플레이 MCP"라고 발음하지만 컨텍스트(카카오톡 채널 통합·MCP 커넥터 형태)상 Playwright MCP 또는 Pley/Play 계열의 별도 MCP일 가능성이 있음. 자막만으로는 정확한 제품명을 100% 확정하지 못함.]

### 2) Claude for Chrome 확장 설치 (브라우저 컨트롤용)
- 브라우저(Chrome)에서 확장 프로그램 검색 → **Claude for Chrome** 설치 [01:56]
- 또는 맞춤 설정에서 동일 확장 설치 가능 [02:10]
- 화자 표현:

> "그러면 이제 클로드 코드가 이 창을 컨트롤할 수 있는 권한이 생긴다고 보시면 될 거 같아요." [02:22]

### 3) Cowork → Schedule 진입
- Claude 앱 왼쪽 상단의 **채 / 코워크 / 코드** 세 탭 중 **코워크** 선택 [02:37]
- **스케줄** 버튼 → **새 작업** → **클로드와 함께 창작하기** [02:42]

## 카탈로그 (시연된 자동화 사례 3종)

### 사례 1: 인스타그램 특정 계정 신규 게시물 알림 (매일 오전 9시)
화자의 시나리오:

> "저는 어떤 병원 마케팅을 하는 마케터라고 한번 가정을 해 볼게요. 제 임무는 뭐냐면 매일 아침마다 인스타그램에서 특정 SNS 계정 … SNS 마케팅 포스트가 올라왔는지 안 왔는지를 어 체크를 해 가지고 저한테 알림을 보내는 거라고 보시면 될 거 같습니다." [03:24]

대상 계정 예시: **바이미 성형외과** 인스타그램 계정 [03:41]

입력한 프롬프트(영상 내 받아쓰기):

> "나는 어 병원 마케터인데 매일 아침 9시에 여기 사이트에 새로운 게시물이 올라왔는지 어 알림을 주는 기능을 만들고 싶어. 새로운 게시물이 올라왔으면 네가 클로 어 해당 개시물 링크와 함께 요약을 해서 플레이 MCP를 사용해서 나와의 채팅방에 해당 게시물의 내용과 링크를 보내 줘." [04:00]

실행 흐름:
- Schedule 생성 → "인스타그램 바이미 데일리 체크"라는 이름의 작업이 만들어짐 [05:07]
- **지금 실행** 버튼으로 즉시 테스트 가능
- Claude for Chrome MCP가 인스타그램을 직접 크롤링 (브라우저 컨트롤) [05:54]
- 글을 하나씩 열어 정보 수집 → Playwright MCP로 카카오톡 나와의 채팅에 전송
- 결과: 카카오톡에 **"바이미 PS 새 개시물"** 형태로 링크·요약 도착 [06:43]

화자 평:
> "제가 건드리지 않았는데도 불구하고 저는 이제 버튼만 눌렀습니다. 어 이 친구가 알아서 이제 크롤링을 하고 있습니다." [06:07]

### 사례 2: 일일 마케팅 트렌드 리서치 (매일 오전 9시)
입력한 프롬프트(영상 내 받아쓰기):

> "오늘 하루 동의 마케팅 트렌드 인스타 틱톡 SNS에서 활발했던 마케팅 트렌드를 조사해 줘. 그리고 이걸 알려 줘. 이걸 알려서 카카오톡 나와의 채팅방에 플레이 MCP를 사용해서 트렌드 요약을 보내 줘." [07:35]

실행 흐름:
- Schedule 이름: "데일리 마케팅 트렌드 카카오" [08:46]
- 웹 검색 기반으로 트렌드 수집 → 카카오톡 전송
- 실제 결과물에 포함된 항목 예: 언필터드 콘텐츠 부상, 챌린지 부활, 올리비아 로드리고 신보, 피파 월드컵 관련 트렌드 [09:48]

화자의 보강 팁:
> "이 부분은 프롬프트를 좀 어 세세화시켜 가지고 예를 들어 가지고 한국의 어 인스타그램 마케팅 트렌드 이런 식으로 검색을 하셔 가지고 하시는 것도 추천을 드리고 있습니다." [10:15]

### 사례 3: X(트위터) 글 수집 → 번역 → Threads 자동 포스팅
> "XAI에 있는 최신 글들을 긁은 다음에 번역을 해 가지고 어 쓰레드 데일리에다가 올리는 그런 친구라고 보시면 될 거 같습니다." [10:30]

[불명확: 화자는 "XAI"라고 발음했으나 컨텍스트(X 사이트에서 글을 긁어 번역)는 X(구 트위터) 플랫폼 자체를 가리키는 것으로 보임. xAI 회사 계정 한정인지는 자막만으로 확정 불가.]

실행 흐름:
- Claude for Chrome이 X 사이트 접속 → 글 수집 [11:00]
- 적절한 글 선별 → 한글 번역
- Chrome 창을 Threads로 전환 → 번역문 붙여넣기 → 포스트 버튼 클릭 [11:37]
- 자신의 Threads 프로필에 게시 완료 확인 [12:05]

화자 평:
> "이런 쓰레드 자동화 뭐 X 자동화 이런 것들이 다 가능하다라고 보시면 될 거 같아요. 이런 작업들이 어렵지 않고 굉장히 쉽게 쉽게 쉽게 이제 활용이 가능하다라고 보시면 될 것 같습니다." [12:14]

## 팁

### 구독 플랜 선택
> "[의견] 소닉 모델. 그러니까 프로 구독만 해도 사실 코어크를 쓰는데 크게 문제는 없거든요. 근데 물론 많은 작업을 할 때는 무조건적으로 추천드리는 거는 맥스 100불짜리를 좀 추천을 드리고 있습니다. 뭐 여러 가지 작업을 할 때 어 토큰양이 부족한 경우가 좀 종종 발생한 거 같더라고요." [09:00]

[불명확: 화자가 "소닉"으로 발음했지만 Claude 모델 라인업에서 "소넷(Sonnet)"을 가리킬 가능성이 매우 높음. 자막 정정 사전에 따라 Sonnet으로 해석 가능하나, 화자가 Pro 플랜과 함께 거론한 맥락이라 정확한 모델명 매핑까지 확정하지는 않음.]

### 코딩 지식 없이 사용 가능
> "제가 진짜 어 사용한 거는 플레이 MCP하고 이 클로드인 크롬이 두 개만 사용해 가지고 정말 자연어만 이용해서 어떠한 코딩 지식 없이 저는 단지 로그인 하고 이 상태에서 처리를 한 거를 확인을 할 수가 있었습니다." [13:05]

### 응용 범위 제안 ([의견])
> "[의견] 어떤 특정 카페에 들어가 가지고 뭔가 리뷰를 긁는다든지 뭐 이커머스에 예시라면 자산홀에 있는 리뷰를 뭐 에이블리에 올린다 뭐 혹은 에이블리에 올라온 리뷰를 자산홀에 올린다 이런 작업까지 전부 가능하다고 보시면 될 거 같습니다." [12:40]

[불명확: "자산홀"은 컨텍스트(이커머스·에이블리와 병치)상 한국 패션 커머스 플랫폼 명을 가리키는 것으로 추정되나, 자막만으로 정확한 표기를 확정하지 못함.]

## 화자의 메시지

### 격차론 ([의견])
