## 핵심 개념

### Claude Cowork란

> "쉽게 말하면 클로드가 직접 선발이 돼서 일을 처리해 주는 작업장이에요." [01:47]

> "코워크의 핵심은 자동화예요. 우리가 옆에 붙어 있을 필요가 없어요. 일 던지고 다른 일 하다가 결과받으면 끝. 이게 되니까 비로써 진짜 동료처럼 느껴지기 시작하는 거예요." [02:11]

- 영문명: **agentic workspace** (자막 발음 그대로는 "에이전틱 워크스페이스")
- 단어 자체의 풀이: "Cowork = 같이 일하기 / 협업" [00:02]
- 기존 채팅과의 차이: 기존은 "캐치볼처럼 주고받는 구조", Cowork는 **결과물을 말해 주면 클로드가 그걸 만들어 내는 구조** [01:54~02:01]

### 기존 AI 사용의 3대 불편 (도입부 문제 정의)

화자가 코워크를 도입하기 전 정리한 기존 채팅형 AI의 한계:

1. **일이 한 단계로 안 끝남** [00:38~00:55]
   > "마치 인턴 옆에서 한 마디 한 마디 가르치는 사장님 같은 거예요. 그러다 보면 내가 하는게 더 빠르다는 생각이 들죠." [00:53]

2. **AI가 매번 기억을 잃음** [00:59~01:16]
   > "회사의 신입 직원이 매일 아침 출근할 때마다 어제 일을 다 잊어버리고 오는 거랑 똑같아요." [01:09]

3. **AI가 내 컴퓨터 파일을 못 봄** [01:16~01:30]
   - 파일 → 복사 → 채팅 → 답변 → 다시 파일 복사, 하루 종일 반복하면 지침
   > "어떤 사람들은 이런 불편함이 너무 심해서 AI 안 쓰고 말겠다는 분도 계세요." [01:30]

### Before / After 비교

화자가 직접 제시한 일상 사례:

> "비포는 클로드 채팅창에 질문 던지고 답변 복사에서 엑셀 붙여 놓고 다시 질문하고 또 답 받고요. 30분짜리 일을 한시간 동안 붙어서 해요. 애프터는 코워크에 이번 분기 매출 데이터 정리해서 엑셀로 만들어 줘. 한 마디 던지고요. 점심 먹고 오면 엑셀 파일이 폴더에 떡한이 만들어져 있어요." [03:00~03:18]

> "[의견] 하루에 한 시간씩만 줄어도 한 달에 20시간이거든요. 그게 1년이면 240시간이에요. 일주일치 일하는 시간이 통째로 절략되는 거죠." [03:25~03:31]

## 카탈로그 — 4대 핵심 기능

화자는 명시적으로 "핵심 4가지"라고 묶었다.

### 1. 로컬 파일 관리 (Local File Management)

화자가 "**이게 사실 가장 중요해요**"라고 강조 [03:37].

- 클로드가 **컴퓨터 안 파일을 직접 읽고 씀**. 업로드/다운로드 불필요 [03:46~03:50]
- 사용자가 할 일: **폴더만 알려주는 것** [03:56]

비유:
> "그동안 클로드는 회사밖에서 일하는 외주 직원이었어요. 우리가 파일을 출력해서 우편으로 보내주고… 근데 코워크는요. 클로드가 회사 안으로 들어온 거예요. 우리 옆자리 직원처럼요." [03:59~04:15]

왜 중요한가:
> "진짜 일은 다 파일에 있거든요. 보고서, 데이터, 코드, 디자인 시장 다 폴더에 있잖아요. 채팅창에 복사 붙여 놓기로는 한 개가 있어요. 파일이 100개면 100번 복사해야 되니까요." [04:19~04:27]

보안 안전장치:
- 폴더 단위로 허용 안 하면 클로드는 못 봄 [04:34]
- 중요한 작업 시 매번 확인을 받음 ("이 파일 지어도 돼요" 식) [04:37~04:43]

### 2. 크롬 연동 (Chrome Integration)

> "이건 클로드인 크롬(Claude for Chrome)이라는 거랑 짝꿍이에요. 한마디로 클로드가 우리 대신 웹브라우저를 조작할 수 있게 된 거예요." [04:45~04:50]

예시:
> "이 회사 채용 공고 다 정리해 줘. 한 마디 하면요. 클로드가 알아서 회사 채용 페이지 열고 공고 하나씩 클릭해서 내용 읽고 정리해서 표로 만들어요." [04:51~05:00]

화자가 강조한 결합 효과:
> "이게 결합되면 진짜 강력해져요. 로컬 파일 플러스 크롬이 두 개가 만나면요. 웹에서 자료 가져와서 내 컴퓨터 파일로 정리하는게 완전 자동화돼요. 예를 들어 매주 월요일에 경쟁사 가격 추척해서 엑셀 업데이트 하는 일… 30분씩 잡아먹던 일들 코워크가 알아서 해 줘요." [05:13~05:26]

주의/안전장치:
- 잘못 클릭으로 결제 진행 위험 [05:31]
- 민감 작업 시 확인 요청
- **10분 안에 답 안 하면 자동으로 안 하고 넘어감** [05:38~05:41]

### 3. 병렬 작업 (Parallel Work)

> "복잡한 일이 들어오면 클로드가 알아서 작은 일로 쪼개요. 그리고 그 작은 일들을 동시에 여러 개 처리해요." [06:01~06:07]

비유: 코스 요리에서 셰프 한 명이 전체를 만드는 옛 방식 → 여러 셰프가 동시에 만드는 방식 [06:18~06:26]

가능한 이유 (화자 분석):
> "큰 일을 작은 일로 쪼개는 능력. 그리고 쪼갠 일들을 따로따로 진행시키는 능력. 이 두 가지가 잘 만들어져 있어서 그래요. 우리가 따로 설정 안 해도 알아서 잘합니다." [06:28~06:35]

### 4. 전문 산출물 생성 (Excel · PowerPoint · Word)

화자: **"이게 사실 사람들이 제일 좋아하는 기능이에요."** [06:39]

> "클로드가 단순히 텍스트만 뱉어 주는게 아니라 진짜 업무에 쓰이는 문서들 이런 것들을 만들어 줍니다." [06:46]

엑셀 차별점:
- 그냥 표만 아니라 **함수까지 작동** (SUM, VLOOKUP, 피벗 테이블) [06:52~06:58]
- 셀 함수 + 차트 자동 생성 [07:03~07:07]

파워포인트:
- 표지 / 목차 / 각 슬라이드 내용 / 디자인까지 일정 수준 [07:11~07:17]

화자의 위치 전환 비유:
> "그동안 클로드는 글 쓰는 비서였어요. 근데 이제 진짜 업무를 아는 비서로 바뀌었습니다." [07:19~07:23]

> "[의견] 코워크가 80% 정도 만들어 주면 우리는 20%만 손 보면 되니까 시간이 엄청 절약돼요." [07:32~07:34]

## 카탈로그 — 추가 기능 & 확장

### 디스패치 (Dispatch) — 핸드폰에서 집 컴퓨터 원격 조작

> "핸드폰으로 집 컴퓨터한테 일시키는 기능이에요. 원격 조작입니다." [07:42~07:44]

시나리오:
- 집에 데스크탑 + Claude 데스크톱 앱 ON + 인터넷 연결 상태로 외출
- 핸드폰 모바일 앱에서 일 하나 던지면 → 작업은 집 데스크탑에서 돌고 → 결과는 핸드폰으로 확인 [07:46~08:06]

데스크탑에서 돌리는 이유 (2가지):
1. 컴퓨터 성능 그대로 사용 (핸드폰보다 빠르고 안정적) [08:17]
2. 데스크탑에 미리 등록해 놓은 **폴더와 도구**를 다 쓸 수 있음 [08:20~08:24]

> "데스크탑이 진짜 일터고 핸드폰은 원격 리모컨인 셈이에요." [08:28~08:31]

조건:
- Pro 또는 Max 구독
- 맥 / 윈도우 최신 Claude 데스크톱 앱 [08:34~08:38]

안전장치 (크롬 연동과 동일):
- 민감 작업 시 알림
- **10분 안에 답 없으면 자동으로 안 하고 넘어감** [08:46~08:50]

### 프로젝트 (Projects) — 영역별 전용 워크스페이스

> "특정 영역의 작업을 위한 연구 워크스페이스예요." [09:02~09:06]
> [불명확: "연구"는 자막 표기이지만 문맥상 "전용"의 오인식 가능성. 원문 그대로 보존]

비유: 회사 부서별 사무실 (마케팅 / 개발 / 영업) [09:06~09:18]

#### 프로젝트 5요소

| 요소 | 역할 |
|---|---|
| **설명 (Description)** | 프로젝트가 뭐 하는 데인지. **Dispatch가 작업 라우팅을 정할 때 이 설명을 본다** [09:30~09:43] |
| **폴더 (Folders)** | 클로드가 읽고 쓸 로컬 디렉토리. **파일 하나당 최대 50MB**, 여러 폴더 등록 가능 [09:43~09:50] |
| **지시 사항 (Instructions)** | 세션마다 자동 적용되는 기본 안내. 예: "답변 한국어", "코드는 TypeScript", "보고서는 인템플릿 따라" [09:52~10:05] |
| **링크 (Links)** | 참고 URL [10:19] |
| **메모리 (Memory)** | 세션 끝나도 사라지지 않는 프로젝트 단위 저장소. 클로드가 배운 것/발견한 것을 계속 저장 [10:21~10:30] |

지시 사항 비유:
> "신입 직원한테 매일 아침 '오늘도 한국어로 일하셔야 돼요' 안 가르쳐도 되는 거예요. 처음에 회사 매뉴얼만 잘 만들어 놓으면 직원이 알아서 잘 따르잖아요." [10:07~10:13]

#### 세션 시작 시 자동 동작

- 프로젝트 선택 → 폴더 자동 마운트
- 지시 사항 자동 적용
- 만든 파일은 프로젝트 폴더에 저장
- 작업하며 쌓인 지식은 메모리에 자동 저장 [10:32~10:47]

> "마케팅 프로젝트에 들어가면 마케팅 모드, 개발 프로젝트에 들어가면 개발 모드 한 번 세팅해 두면 매번 새로 설명 따로 안 해도 됩니다." [10:50~10:55]

#### 헷갈리지 말 것: Cowork 프로젝트 ≠ Claude AI 프로젝트

화자가 명시적으로 짚은 구분점 [10:55~11:26]:

| 구분 | 위치 | 로컬 폴더 접근 | 다른 기기 |
|---|---|---|---|
| **Claude AI 프로젝트** | 클라우드 (계정) | 못 봄 | 어디서든 접속, 팀 공유 가능 (유료) |
| **Cowork 프로젝트** | 우리 컴퓨터 로컬 | 가능 | 다른 컴퓨터에서는 못 봄 |

> "이 둘은 합쳐지지 않아요. 다만 클로드 AI 프로젝트를 코워크에 링크에서 같이 쓸 수는 있습니다." [11:20~11:26]

#### 프로젝트 활용 팁 (화자 제시 4가지) [11:28~12:07]

1. **처음 한 번 세팅에 시간 투자** — 인스트럭션 꼼꼼히 적을수록 매 세션이 편해짐
2. **디스패치와 같이 사용** — 백그라운드 작업을 특정 프로젝트로 보내고 원격 조작
3. **메모리 활용** — 클로드가 뭘 저장했는지 가끔 확인, 잘못된 정보는 수정
4. **아카이브 활용** — 프로젝트 아카이브해도 메타데이터만 사라지고 로컬 파일은 유지, 나중에 다시 꺼낼 수 있음

### 커넥터 (Connectors) — MCP 기반 외부 도구 연결

> "MCP라는 표준을 이용해서 클로드를 외브 도그나 데이터 소스에 연결하는 거예요." [12:10~12:16]

MCP 설명 (화자):
> "모델 콘텍스트 프로토콜의 줄임말인데요. 너무 어려워하지 마세요. 그냥 AI랑 외브 서비스를 연결할 때 쓰는 공통 약속 정도로 이해하면 돼요." [12:16~12:23]

비유 — **콘센트 규격**:
> "한국 어디 가도 220V 콘센트가 똑같이 생겼잖아요… MCP가 AI 세 개의 콘센트 규격이에요. 한번 정해 놓으니까 어떤 도그도 다 꽂아서 쓸 수 있어요." [12:25~12:43]

연결 예시 [12:45~12:54]:
- **Notion** → 페이지 읽기/쓰기
- **Slack** → 메시지 송수신
- **GitHub** → 코드 저장소 조작

실용 예시:
> "어제 회의록 (Notion) 보고 액션 아이템 (Linear의) 티켓으로 만들어 줘. 이런 식으로 할 수 있습니다." [13:05~13:11]

### 스킬 (Skills) — 작업 매뉴얼

> "커넥터스가 외브랑 연결하는 통로를 하면 스킬즈는 특정 작업을 어떻게 처리해야 할지 알려주는 매뉴얼이라고 보시면 될 거 같습니다." [13:17~13:25]

구조:
- `SKILL.md` 파일 + 보조 스크립트 + 리소스 파일들이 들어있는 폴더 [13:29~13:33]
- 폴더 안에 "이 작업이 들어오면 이렇게 처리해 줘"라고 적어둠 [13:33~13:36]

#### 점진적 공개 (Progressive Disclosure)

화자가 특이점이라고 명시:

> "스킬즈는 항상 활성화돼 있는게 아니에요. 클로드가 처음 시작할 때 메타데이터만 읽어요. 그러다가 우리가 시킨 일이 어떤 스킬에 해당되면 그때서야 그 스킬 내용 전체를 불러요. 이거를 점진적 공개라고 불립니다." [13:36~13:46]

이유:
> "모든 매뉴얼을 한 번에 다 읽는 거보다 메타데이터만 읽고 그거를 필요한 순간에만 부르는 것이 훨씬 효율적이기 때문이죠." [13:49~13:56]

### 플러그인 (Plugins) — 모든 것을 묶은 패키지

> "이건 커넥터, 스킬, 그리고 다른 거 다 합쳐서 패키지로 만든 거예요." [13:58~14:02]

4. **훅 (Hooks)** — "특정 시점에 자동으로 실행되는 스크립트"

## 환경 셋업 — 6단계 도입 가이드

화자가 "실전에서 어떻게 시작하면 좋을지 단계별로 정리"라고 명시한 순서 [15:02~16:01]:

### Step 1. Claude 데스크톱 앱 설치 [15:10~15:21]
- 맥 또는 윈도우 **최신 버전**
- **Cowork는 데스크톱 앱에서만 작동, 웹에서는 안 됨**
- 모든 기능 사용을 위해 **Pro 또는 Max 구독** 필요

### Step 2. 첫 프로젝트 만들기 [15:21~15:38]
- 거창하게 시작하지 말 것. 평소 자주 하는 일 하나 고를 것
- 예: 매주 작성하는 주간 보고서
- 폴더 만들고 → Cowork 프로젝트 등록 → Instructions에 형식 적기 → Memory에 지난주 핵심 이슈 적기

### Step 3. 디스패치로 원격 조작 시도 [15:38~15:40]

### Step 4. 기본 스킬 활용 [15:40~15:45]
- 엑셀 / 파워포인트 만들어 달라고 해 보기

### Step 5. 커넥터 추가 [15:45~15:54]
- 매일 쓰는 도구 (예: Google Drive, GitHub Issues [불명확]) 연결해서 흐름 확인

### Step 6. 플러그인 마켓플레이스 둘러보기 [15:54~16:01]
- 이미 만들어진 좋은 플러그인 활용

## FAQ (화자가 직접 정리한 5문)

[17:15~17:56]

2. **데이터가 외부로 나가나?** → **로컬 파일 처리는 내 컴퓨터 안에서만** 일어남. 단, **AI 처리 자체는 Anthropic 서버를 거치므로 완전한 로컬 처리는 아님**

4. **맥/윈도우 둘 다 되나?** → 둘 다 됨. **단 맥이 좀 더 안정적인 것 같음** [의견]
5. **ChatGPT 등 다른 AI와 같이 써도 되나?** → **물론**. 도구는 도구일 뿐
