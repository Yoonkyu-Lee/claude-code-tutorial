## 0. "Claude Code 정도면 시작되나?" — 된다 (유튜브와의 결정적 차이 하나)

유튜브가 공짜로 주는 건 둘이다: ① 무료 호스팅 ② **무료 유통**(알고리즘이 관객을 데려옴).
이 업계는 —

| | 유튜브 | 이 업계(기술자 버전) |
|---|---|---|
| 호스팅 | 공짜 | **공짜** (Cloudflare/GitHub Pages, 정적사이트) |
| 시작 속도 | 하루 | 사이트는 **몇 시간** |
| 유통(관객) | 알고리즘이 즉시 노출 | ❌ **검색이 데려오는데 몇 주~몇 달(SEO)** |
| 필수 비용 | 0 | **도메인 ~1.5만원/년** 하나(실험 단계엔 생략 가능) |

→ **Claude Code + 무료 스택 = 사실상 0원 시작.** 이 판을 "글쓰기 대결"이 아니라
**데이터·자동화 엔지니어링 문제**로 바꾸면 Claude Code가 불공정한 무기가 된다.
**돈으로도 Claude로도 못 사는 유일한 것 = 구글이 색인·순위를 잡아주는 시간.** 이건 인내로만 낸다.

## 1. 큰 그림 — 전체 워크플로 (Phase 0~6)

원칙: "AI로 글 양산"이 아니라 **`[검색 패턴] × [남이 못 구하는 데이터] × [의도 충족 템플릿]`**.
각 Phase 끝에 **kill 조건**(시간·돈 상한)을 달아 생존편향을 차단한다.

| Phase | 이름 | 한 줄 |
|---|---|---|
| **0** | 정찰 & 가설 | 짓기 전에 "이길 판인가"를 데이터로 검증 (아래 상세) |
| **1** | 데이터 해자 확보 | 가설에 맞는 구조화 데이터 실제 확보(공공 API·허용 크롤·자체수집)→DB. *데이터 없으면 프로젝트 없음* |
| **2** | 파이프라인 & 사이트 구축 | Claude Code로 수집→정제→템플릿 주입→정적사이트 생성→사이트맵 (수백~수천 페이지) |
| **3** | 시드 발행 & 색인 | 100~500페이지 발행→서치콘솔 등록→색인 + sticky 요소(계산기·지도·실데이터) + 샘플 사람 검수 |
| **4** | 계측 & 스케일 | 서치콘솔 선행지표(노출·평균순위·색인률) 관찰→뜨는 패턴만 확장, 죽은 패턴 폐기 |
| **5** | 수익화 | 트래픽 붙으면 광고 사다리(AdSense→Ezoic→Mediavine) + 제휴, 또는 무료도구/제품 전환 |
| **6** | 해자 & 방어 | 데이터 우위 심화 + 소유 오디언스(이메일) + 구글 외 유입 다각화 + 알고리즘 모니터링 |

## 2. Phase 0 완필 — "정찰 & 가설" (뭐부터 하나)

**핵심 원칙: 첫날 블로그 글을 쓰지 않는다.** 알파남 방식의 치명적 실수가 "일단 글부터 양산"이다.
Phase 0은 글 한 편 안 쓰고 **3~5일 안에 끝나는 리서치 스프린트**다.

### 0-1. 수익 방정식 먼저 (방향 결정)
`수익 = 트래픽 × 단가`. 두 갈래:
- **광고형** — 단가 낮음, **대량 트래픽** 필요 → 프로그래매틱과 궁합 좋음
- **제휴/도구형** — 트래픽 적어도 **의도 높으면** 수익, 페이지 적어도 됨
→ 권장 경로: **광고형 프로그래매틱으로 시작 → 도구형으로 진화**.

### 0-2. 가설 공식
> **"[반복되는 검색 패턴]을, [남이 쉽게 못 구하는 구조화 데이터]로 채운, [검색 의도를 충족하는 템플릿] 페이지."**

예시(감 잡기용):
- `"[전기차 모델] 보조금 [지역]"` × 지자체 보조금 공공데이터 × 계산기
- `"[개발도구 A] vs [도구 B]"` × 도구 스펙/요금 DB × 비교표 (Zapier 패턴)
- `"[도시] 한 달 살기 비용/비자"` × 물가·비자 데이터 (Nomad List 패턴)
- `"[영양제 성분] 효능/상호작용"` × 성분 DB (건강=YMYL, 난이도↑)

### 0-3. 검증 3종 (Claude Code로 자동화) ← "뭐부터"의 답
가설 후보 2~3개를 뽑아 각각 확인:
1. **수요 있나?** — 사람들이 이 패턴을 검색하나? → 구글 자동완성·연관검색 수집, Google Trends, 무료 키워드 툴
2. **데이터 구할 수 있나?** — 페이지마다 다르게 만들 데이터셋을 합법적으로? → 공공데이터포털·정부 API·robots.txt 허용 크롤. **여기서 막히면 그 가설 폐기.**
3. **이길 수 있나 + 돈 되나?** — 지금 1페이지가 약한가(얇은 페이지) 강한가(위키·대형 브랜드)? SERP에 광고·제휴가 붙나? → 상위 10개 SERP 수집·분석

### 0-4. 한 장짜리 가설 + kill 조건
후보 점수화 → **하나만** 선택. 문서 한 장:
- 패턴 / 데이터 소스 / 예상 페이지 수 / 수익 방향
- **kill 조건**: "3개월 안에 색인률 X% 또는 노출 Y 미달 → 폐기·전환"

### 0-5. 최소 인프라 예약 (30분)
- GitHub 레포 + 정적사이트 뼈대(Astro/Next) 스캐폴드
- (선택) 도메인 등록 — 가설 서면. 실험만이면 무료 서브도메인
- 구글 서치콘솔 계정 준비

## 3. "당장 오늘 할 첫 행동"
Claude Code에게: *"이 세 니치 가설에 대해 ① 구글 자동완성·연관검색 수집 ② 각 가설의 공공데이터 소스 존재·접근성 조사 ③ 상위 10개 SERP 경쟁 강도 분석 → 점수표"*.
반나절이면 **"이 판은 이길 만하다/아니다"**가 데이터로 나온다. 글은 그다음.

> 알파남: "블로그 개설 → 글 양산"으로 시작.
> 우리: **"가설 검증 → 데이터 확보 → 파이프라인"**으로 시작.
> 같은 하루를 써도, 그는 밴당하면 0 되는 글더미를, 우리는 복제 불가능한 데이터 자산을 쌓는다.
