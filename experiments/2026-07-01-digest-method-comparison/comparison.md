- **공정성**: 두 digest는 같은 포맷(`_shared/digest-format.md`)을 목표로, **소스·프로세스만** 다르게 했다. 두 digest 에이전트는 서로의 결과를 못 보는 격리 환경에서 병렬 생성.

## 결과 요약

| 항목 | A (노트기반) | B (자막기반) |
|---|---|---|
| 제목 | ⚠ 자체 편집("60억 쓸어담은 알파남") — 포맷의 "과장 톤 금지" 소폭 위반 | 영상 원제 그대로("…75억…") — 원제가 클릭베이트지만 사실은 원제 |
| 타임라인 구간 | 10개 (`[00:51]`~`[32:28]`) | 15개 (`[00:00]`~`[34:00]`) |
| 커버리지 | 노트가 인용한 구간에 **묶임** — 오프닝 훅[00:00]·마무리 과시[34:00] 누락 | 자막 처음~끝 통독 → **오프닝 훅·개인 서사·세일즈 클로징·부 과시 엔딩까지** 포착 |
| 회의적 라벨링 | 노트의 "화자 주장(미검증)"·`[의견]`을 **그대로 상속** (공짜로 일관) | 독립적으로 라벨링, 잘 처리했으나 재작업 |
| 세일즈 구조 포착 | 있음(별도 PDF·단톡방 유도 언급) | **더 날카로움** (강의비 1.5억 서사, 4.7만명, "담당자한테 혼나서 못 알려준다" 클로징까지) |
| 음차 처리 | 노트가 이미 복원한 표기 재사용(안정) | 자막에서 재추정(Taboola 등) — 재작업·오추정 여지 |
| 압축률 | 노트 대비 ~5% (약 40줄) | 자막 대비 ~3.3% (약 44줄) |
| 소스 토큰 | 노트 240줄(~41k) | 자막 1344줄(~70k) |
| **전체 비용** | **노트 생성(~66k) + digest(~41k)** = 높음 | digest 단독(~70k) = 중간, **노트 불필요** |

## 관찰

1. **커버리지는 B 우세**: B는 자막을 처음~끝 읽어 흐름 전체(훅→서사→시연→클로징→과시)를 잡았다. A는 **노트가 인용한 타임스탬프에만 접근**하므로, 노트가 안 담은 오프닝/엔딩을 원천적으로 못 넣는다. → A의 품질 상한은 **소스 노트의 커버리지**에 종속된다.

2. **일관성·회의성은 A 우세(공짜)**: A는 노트가 이미 "화자 주장/미검증"으로 정제한 라벨을 그대로 물려받아, 추가 판단 없이 검증 규율을 유지했다. B는 같은 판단을 **매번 새로** 해야 한다.

3. **A의 제목 일탈**: A는 원제 대신 "알파남 60억" 같은 편집 제목을 지어냈다(포맷의 과장 톤 금지 위반). 노트기반이라고 자동으로 절제되는 건 아니며, 스킬에 "제목은 노트 출처 제목을 따른다"는 못을 박아야 한다(개선점).

4. **비용 구조가 결정적**: A는 값이 싸 보이지만 **원문 노트가 선행돼야** 한다. 노트 생성까지 합치면 A의 총비용이 B보다 크다. B는 노트 없이 단독으로 돈다.

## 판단 (제안 — 채택은 사용자)

두 방식은 **경쟁이 아니라 다른 시나리오**에 맞다:

- **B(자막기반) → 대량 트리에이지용**: `/collect-new`처럼 노트가 아직 없는 다수 영상을 훑을 때. 노트를 안 만들어도 독립적으로, 흐름 전체를 커버. 영상(MJiQFNp-k10)의 원래 용도와 정확히 일치.
- **A(노트기반) → 이미 정리한 영상의 곁가지**: 우리가 원문 충실 노트를 **어차피 만드는** 영상에 한해, 그 위에 싼값으로 얹는 "한 눈 요약". 검증 라벨을 공짜로 상속.

→ **`digests/<topic>/<channel>/` 정식 레이어의 기본 생성기로는 B**(노트 유무와 무관하게 돌아가고 커버리지가 안정적)를 채택하고, **A는 노트가 있는 영상에 대한 선택적 보강**으로 남기는 구성을 제안. 단 A/B 공통으로 "제목은 원제 유지" 규칙을 스킬에 추가.

## 개선점 (실험이 드러낸 것)

- 두 digest 스킬 공통: **제목은 영상 원제를 따른다**(자체 편집 금지)를 `_shared/digest-format.md`에 명문화.
- A 스킬: 노트가 인용 안 한 오프닝/엔딩을 놓치는 한계를 룰북에 명시(소스 노트 커버리지에 종속).
- (이미 반영) 자막 HTML 엔티티(`&gt;` 등) 복원 — 인터뷰 화자 마커에서 발견돼 `vtt-to-text.awk` 수정 완료.
