## 한 줄 요약
Claude Code를 "치매에 걸린 아인슈타인"(천재인데 기억을 못 함)에 비유하면서, 처음 쓰는 사람이 흔히 도구부터 쌓다 포기하는 패턴을 막기 위해 "기본 상태 → 실전 프로젝트 → 스킬 → MCP → MCP 토큰 최적화 → 컨텍스트 엔지니어링 → 토큰 절약 → 소모량 파악 → 한계까지 써보기 → 하니스 찾기"의 10단계를 순서대로 밟으라고 권하는 개념설명 + 단계별 가이드 혼합형 영상.

## 메타
- 영상 유형: 혼합형 (비유 중심 개념설명 + 10단계 순차 로드맵 가이드)
- 대상 청중: Claude Code를 켜 봤지만 "유튜브 사람들처럼 잘 못 쓰겠다"고 느끼는 초·중급자
- 핵심 프레임: **순서**가 전부다 — 10단계를 1단계 자리에서 하면 흔들린다

## 핵심 개념

### 비유 1: 클로드 코드 = "치매에 걸린 아인슈타인"
화자가 영상 전체를 관통하는 비유. 코드·아키텍처·디버깅은 천재적으로 잘하지만 기억을 못 한다는 것.

> "클로드 코드는요. 치에 걸린 아인슈타인이에요. 복잡한 코드 짜고 아키텍트 설계하고 에러 디버깅하고 엄청 잘해요. 근데 기억을 못 해요. 오늘 대화한 내용을 내일 대화 시작하면 완전히 잊어버려요." [01:23][^1]

세션 간 망각의 구체적 증상:

> "어제 이 방식은 절대 쓰지 마라고 했는데 셋[새] 세션 켜면 그 지시가 없던 일이 돼요. 어제 함께 고친 버그를 오늘 또 만들어요. 내 코딩 스타일, 내 프로젝트 규칙, 지금까지 쌓[은] 모든 맥락 새 대화 여는 순간 다 사라지거든요." [01:35]

대화 도중에도 망각이 온다는 예고:

> "근데 이게 끝이 아니에요. 대화 중간에도 치매가 오는 경우가 있어요." [01:48]

→ 이 "대화 중간의 치매"가 6단계에서 다루는 **컨텍스트 컴팩션(compaction)**의 복선이다.

### 비유 2: 도구부터 쌓는 실수 = "수영 못 하는데 올림픽 장비"
커뮤니티에서 가장 흔한 실수: 처음부터 플러그인·MCP·프레임워크를 다 깔고 시작한다.

> "처음부터 플러그인 깔고 MCP 다섯 개 연결하고 프레임워크 설치하고 시작해요. 뭔가 세팅이 빵빵해 보이잖아요. 근데 막상 쓰다 보면 이상한 게 생겨요. [에]러가 나는데 어디서 나는지 모르고 클로드가 이상하게 행동하는데 왜 그런지 모르고 그냥 AI가 멍청한가 보다 하고 포기하게 되거든요." [00:35]

> "[의견] 사실은요. AI가 멍청한 게 아닙니다. 이해 없이 도구만 쌓아올린 거예요." [00:51]

> "수영 못 하는 사람한테 올림픽 선수용 장비를 다 사줬어요. 최신 수경 전신 수영복. 근데 정작 물에서 팔을 어떻게 움직이는지, 숨은 언제 쉬는지 모르면요. 그 장비가 무슨 소용이에요?" [00:57]

### 핵심 주장: 문제는 AI가 아니라 "순서"
> "[의견] 이거요. AI 탓이 아닙니다. 순서 문제예요. 오늘 그 순서 정확하게 알려 드릴게요. 10단계 로드맵으로요." [00:17]

마무리에서 한 번 더 강조:

> "핵심은 하나예요. 순서대로 해야 한다는 것. 10단계를 1단계에서 하면 안 돼요. 성급하게 올라가면 기초가 없어서 흔들려요." [11:33]

## 10단계 로드맵 (시간순 워크플로)

### 1단계 — 플러그인 없이 기본 상태로 시작
설치 직후의 까만 터미널 그대로, 일주일만 참고 관찰하라.

> "딱 일주일만 참으세요. 이 기간에 봐야 하는 게 있어요. 이 치매 아인슈타인이 어떻게 일하는지 직접 눈으로 봐야 해요. 파일을 어떤 순서[로] 읽는지, 복잡한 작업을 어떻게 쪼개서 하는지, 오류[가] 나면 어떻게 혼자 복구하려고 하는지, 그리고 언제 이상하게 행동하는지" [02:14]

기본 상태를 안 보면 플러그인이 뭘 보완해 주는지도 모른다 → "그냥 블랙박스". 자연히 "왜 이건 고치고 저건 못 고치지? 왜 대화가 길어지면 이상해지지?" 같은 질문이 생기는 것이 성장이다. [02:39]

### 2단계 — 튜토리얼 지옥 탈출, 실전 프로젝트 시작
영상·블로그·강의만 5시간 보고 정작 만든 건 없는 상태를 경계.

> "클로드 코드는 이론으로 [배]우는 도구가 아닙니다. 자전거 타는 법을 책으로 100시간 공부해도 실제로 타 보지 않으면 못 하잖아요. 넘어져 봐야 [배]워요." [02:56]

> "[의견] 완성도 50%짜리 앱을 만들면서 배우는 게 완벽한 튜토리얼 [열] 개 보는 것보다 훨씬 빠르게 늘어요." [03:13]

→ 가게부 앱이든 일정 정리 도구든 "내가 쓸 거 하나" 정해서 같이 만들어 보라.

### 3단계 — 스킬 써보고 만들기
반복 작업이 보이기 시작하는 시점. 배포·코드리뷰·커밋 메시지마다 같은 설명을 반복하는 것이 비효율.

> "반복되는 작업 순서를 레시피처럼 저장해 두는 거예요. 볶음밥 만드는 법을 매번 구글링하는 대신 내 레시피 노트에 적어 두고 꺼냈으면 되잖아요. 스킬도 그거예요." [03:39]

스킬의 토큰 이점:

> "스킬의 또 다른 장점 토큰을 거의 안 먹어요. 호출할 때만 [로]드 되거든요. 스킬 열 개 설치해도 기본 소모가 거의 없어요." [03:56]

판단 기준:

> "내가 두 번 이상 반복한 작업이 있으면 그걸 스킬로 만드세요." [04:04]

### 4단계 — MCP 연결하기
> "MCP는 모델 콘텍스트 프로토콜[Model Context Protocol]의 약자예요. 어렵게 들리는데 쉽게 말하면 클로드 코드에 [팔]다리를 붙여 주는 거예요." [04:10]

MCP가 붙여주는 "팔다리" 예시:
- GitHub MCP[^2]: 클로드가 직접 PR 만들고 이슈 달고 코드 리뷰
- 브라우저 MCP: 웹페이지 열고 클릭하고 스크린샷
- 데이터베이스 MCP: DB에 직접 쿼리[^3] 날리기

> "즉 팔다리를 달아줘서 도구를 사용할 수 있게 하는 거죠." [04:35]

### 5단계 — MCP 대신 스킬·CLI 사용하는 테크닉 (MCP 토큰 최적화)
MCP를 켜는 순간 대화 시작 전부터 비용이 발생한다는 함정.

> "토큰이라는 게 있어요. 클로드 코드를 쓸 때 쓰이는 [일종]의 연료예요. 이 치매 아인슈타인의 기억 용량이라고 생각하면 돼요. 이 용량이 꽉 [차]면 오래된 기억부터 지워지기 시작해요." [04:45]

> "근데 MCP 서버들은 세션 시작할 때 '나 이런 기능들이 있어요'라는 목록을 통째로 클로드 머릿속에 밀어 넣거든요. 여러분이 대화 한 글자도 쓰기 전에 이미 소비가 되고 있는 거예요." [04:57]

구체적 수치(커뮤니티 측정 인용):

> "실제로 커뮤니티에서 측정한 수치가 있어요. MCP 서버 다섯 개 켜 놓으면 전체 용량의 41%가 사라진[다]." [05:05]

해결책 두 가지: ① 자주 안 쓰는 기능은 MCP 대신 스킬로(호출 시에만 로드), ② 안 쓰는 MCP는 끈다. [05:12]

> "자, 1단계에서 5단계까지 했으면 기본 세팅은 다 됐습니다. 튜토리얼 레벨이 끝난 거죠. (...) 여기까지가 기초입니다. 이제부터가 진짜 고수의 영역이에요." [05:20]

### 6단계 — 컨텍스트 엔지니어링 (화자가 꼽은 가장 중요한 단계)
> "[의견] 사실 전체 10단계 중에서 제일 중요한 단계라고 생각해요." [05:41]

> "컨텍스트 엔지니어링은 클로드한테 상황에 딱 맞는 정보만 골라서 주는 기술이에요. 이 치매 아인슈타인한테 뭘 기억하게 할지 뭘 잊게 할지 설계하는 거예요." [05:45]

**비유 3: 신입 직원 인수인계 문서 = CLAUDE.md**

> "두 번째 방법은 인수인계 문서 한 장을 잘 써두고 출근할 때마다 그걸 읽게 하는 거예요. 클로드 코드에서 그 인수인계 문서가 클로드.md[CLAUDE.md] 파일이에요. 이 파일을 프로젝트 폴더에 만들어 두면 클로드가 대화 시작할 때 자동으로 읽어요." [06:05]

CLAUDE.md에 담을 것: 프로젝트 구조, 코딩 규칙, 자주 쓰는 명령어, 절대 하면 안 되는 것들. [06:18]

**CLAUDE.md 작성 시 흔한 실수 — 너무 길게 쓰기:**

> "클로드[.]MD를 너무 길게 쓰는 거예요. 일단 다 써[놓]아서 1,000줄, 2[000줄] 쓰는 분들 있거든요. 그러면 안 돼요. 길면 길[은]만큼 토큰을 먹어요. 그리고 너무 많은 규칙은 클로드도 다 못 따라와요. 사람도 규칙이 100개면 다 기억 못 하잖아요." [06:25]

**중요한 작성 팁 — `/init` 자동생성보다 "AI가 잘 인지 못 하는 부분"을 직접:**

> "[의견] 이게 굉장히 중요한 건데 슬래시 이닛[/init]을 통해 가지고 생성된 클로드.m[d]보다는 AI 즉 클로드가 잘 인지를 못하는 부분에 대해서 클로드.md 파일에다가 기입을 해 둬야지 효과가 좋습니다." [06:39]

**컨텍스트 컴팩션(compaction) — "대화 중간의 치매":**

> "[클]로드 코드는 컨텍스트가 일정량 쌓[이]면 자동으로 압축을 해 버려요. 지금까지 나눈 대화를 강제로 요약해 버리는 거예요. 치매 발[작]이 오는 거예요." [06:57]

컴팩션 발동 시 증상: 100% 따르던 지시를 위반, 처음부터 다시 설명, 이미 고친 버그 재발, 읽던 파일을 잊고 다시 읽기 시작. [07:05]

> "그래서 [컴]팩션은 최대한 피해야 합니다." [07:19]

컴팩션 회피 3가지:
1. CLAUDE.md를 짧고 핵심적으로 — 컴팩션 후에도 자동으로 다시 읽히므로 사실상 장기 메모리.
   > "결국 클[로]드.MD가 [치]매 아인슈타인[의] 장기 메모리예요." [07:29]
2. `/clear` 명령어 적극 사용 — "관련 없는 [세]업[작업] 시작할 때는 무조건 슬래시 클리어를 사용하세요. 대화를 쌓지 않고 깨끗하게 시작하면 [컴]팩션이 발동될 일이 잘 없습니다." [07:37]
3. 작업을 잘게 쪼개기 — "한 세션에 하나의 작업만 하는 습관이에요." [07:51]

### 7단계 — 토큰 절약 방법론
1. **필요 없는 파일은 주지 마라**: 로그인 기능 고치는데 전체 프로젝트를 다 읽힐 필요 없다. 관련 파일만. "클로드가 읽는 파일 수를 줄이면 줄일수록 토큰이 절약되고 [컴]팩션도 늦게 와요." [08:07]
2. **모델 선택**: "복잡한 설계는 [Opus]가 [낫]지만 어 코드 수정 (...) 이나 간단한 작업은 [Sonnet]이 훨씬 빠르고 저렴해요. 작업 난이도에 맞는 모델을 골라 쓰는 게 토큰도 아끼고 속도도 빨라요." [08:13][^4]
3. **MCP 대신 CLI 도구·스킬**: "파일 검색은 [grep] 명령으로 되고 간단한 배포 작업은 스킬로 만들 수 있어요. MCP는 진짜 필요할 때만." [08:23][^5]

### 8단계 — 한 세션에 얼마나 쓰는지 직접 확인 (소모량 파악)
6~7단계의 최적화 기법을 실제로 적용하면서 자기 소비 패턴을 파악하는 단계. 복잡한 작업 하나를 처음부터 끝까지 완주해 보라.

> "요리 예산이랑 똑같아요. 한 달에 식비를 얼마나 쓰는지 모르면 어떻게 절약해요? 먼저 내가 얼마나 쓰는지 알아야 줄일 수 있잖아요." [08:52]

→ 감각이 생기면 플랜 선택도 현명하게 할 수 있다 (소모량 모르면 너무 많이/적게 사게 됨). [09:03]

### 9단계 — (Max 플랜 등으로) 한계까지 써보기
> "기본 프로[Pro]플랜 위에 두 가지가 있어요. 다섯 배짜리가 월 100달러, 20배짜리가 월 200달러예요." [09:14][^6]

> "차를 처음 샀을 때 주차장에서만 타다가 고속도로를 한 번도 안 타 본 상태로 '이 차[다]' 하는 분 없잖아요. 실제로 밟아봐야 이 차가 어디까지 가는지 알 수 있어요." [09:23]

핵심은 능력의 상·하한을 둘 다 본다는 것:

> "그리고 '아, 이건 클로드 코드로 하면 안 되는 거구나'를 알게 되는 것도 중요해요. 모든 걸 클로드 코드로 하려는 집착에서 벗어나게 되거든요. 뭘 클로드한테 맡기고 뭘 내가 해야 하는지 명확해져요." [09:41]

### 10단계 — 하니스 찾기
**비유 4: 하니스 = 등산 안전벨트**

## 팁 (영상에서 강조한 행동 지침)
- 1단계: 설치 직후 **딱 일주일은 플러그인 없이** 기본 상태로만 관찰
- 3단계: **두 번 이상 반복한 작업**은 무조건 스킬로
- 5단계: 안 쓰는 MCP는 끈다 (MCP 5개 = 컨텍스트 41% 소모)
- 6단계: CLAUDE.md는 **짧고 핵심적으로**, `/init` 자동생성보다 "AI가 잘 못 알아채는 부분"을 직접 적기
- 6단계: 관련 없는 작업 시작 시 **무조건 `/clear`**, 한 세션에 한 작업
- 7단계: 작업 난이도에 맞는 모델(Opus/Sonnet) 골라 쓰기, 관련 파일만 주기
- 10단계: 하니스는 1~9단계 경험을 쌓은 **뒤에** 도입

## 외부 보강

> 아래는 영상 본문이 아니라 노트 작성자가 공식 문서·커뮤니티로 보강한 부분이다. 영상 내용과 구분된다.

- **컨텍스트 컴팩션(auto-compact)**: Anthropic 공식 문서상 Claude Code는 컨텍스트 윈도가 일정 비율(기본 약 95%)에 도달하면 자동으로 이전 대화를 요약(auto-compact)한다. `/compact`로 수동 트리거하거나 설정에서 끌 수도 있다. 영상의 "강제 요약" 설명과 일치. (출처: Anthropic Docs — Claude Code, "Manage Claude's memory / costs")
- **`/init`**: Claude Code의 슬래시 명령으로, 현재 프로젝트를 스캔해 CLAUDE.md 초안을 자동 생성한다. 영상은 이 자동 생성물을 그대로 두기보다 "AI가 잘 인지 못 하는 부분"을 사람이 직접 보완하라고 권한다. (출처: Anthropic Docs — Claude Code memory)
- **MCP 토큰 41% 수치**: 영상은 "커뮤니티에서 측정한 수치"로 인용. 정확한 출처는 영상에 명시되지 않음. MCP 서버는 연결 시 tool 정의·schema를 시스템 프롬프트에 로드하므로 켜진 서버가 많을수록 초기 컨텍스트 점유가 커지는 것은 사실. 정확한 41%는 환경·서버 구성에 따라 달라질 수 있어 **단일 출처로 일반화하긴 어려움**. (참고: 이 저장소 `2026-05-13-daily-claude-code-tools-six.md` 등에서도 MCP 토큰 점유 이슈가 반복 언급됨)
- **Max 플랜 가격(영상 시점)**: 영상은 5x = 월 $100, 20x = 월 $200로 설명. 가격·배수는 변동 가능하므로 도입 시점에 공식 가격 페이지로 재확인 권장. (출처: claude.com 요금 페이지 — 영상 게시 시점 기준 값일 수 있음)
