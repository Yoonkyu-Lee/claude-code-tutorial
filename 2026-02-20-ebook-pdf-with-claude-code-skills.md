# 누워서 딸깍 한 번에 전자책 완성 — AI로 전자책(PDF) 만들기 4단계

## 출처
- 영상: [누워서 딸깍 한번에 전자책 완성](https://www.youtube.com/watch?v=sMWoMdl9r_g)
- 영상 ID: `sMWoMdl9r_g`
- 채널: 메이커 에반
- 게시: 2026-02-20 (게시일 출처: 디스패처 확정값, yt-dlp)

## 한 줄 요약
웹 리서치 에이전트로 자료 조사 → 초안 검수 → Pinterest 레퍼런스 + PDF 스킬로 디자인 → 깨진 부분 정밀 수정, 이 4단계로 개발자가 아니어도 Claude Code만으로 고퀄리티 전자책(PDF)을 만든다.

## 메타
- 영상 길이: 약 4분 24초
- 영상 유형: 튜토리얼/실습형 (시간순 워크플로 중심)
- 타임스탬프: 제공됨
- 대상: 개발자가 아니어도 따라 할 수 있음을 명시한 입문자용 워크플로

## 핵심 개념

### 스킬(Skill) = AI에게 전문가가 되는 방법을 알려주는 것
> "스킬이란 AI한테 전문가가 될 수 있도록 방법을 알려 주는 거예요. 예를 들어서 AI가 그냥 [일반 의사]였다면 안과 스킬을 주면 안과 의사, 치과 스킬을 주면 치과 의사 이런 식으로 진화하는 거예요." [01:23][^1]

- 전자책 제작에는 **PDF 스킬**을 준다.
> "이제 클로드 코드한테 PDF 스킬로 전자책 만들어 달라고 하면 돼요." [01:36][^2]

### 도구와 역할을 분리해서 부여
> "AI한테 도구와 역할을 줘야 돼요. 저는 여러 가지 스킬과 에이전트를 사용하고 있어요. 요리사한테 청소시키면 안 되잖아요. 마찬가지로 PDF 만들 땐 PDF 스킬, 자료 조사할 땐 웹 리서치 에이전트를 써야 돼요." [02:37]

- PDF 작업 → PDF 스킬
- 자료 조사 → 웹 리서치 에이전트

## 워크플로 (전자책 만들기 4단계)

화자가 명시한 4단계. [00:48]에서 "딱 네 단계"로 선언하고, [03:37]~[03:53]에서 다시 요약한다.

### 1단계: 자료 조사를 AI에 맡기기
> "자료 조사는 AI한테 맡겨요. 저는 제가 만든 웹 리서치 에이전트를 켜서 사용해요. 이 상태로 이 주제로 자료 조사해 줘라고 던지면 AI가 알아서 인터넷 뒤지고 정리해서 문서 파일로 뽑아 줘요." [00:51]

- 대안: 이미 가진 코드·자료를 AI에 읽히고 그것을 바탕으로 책을 쓰게 할 수도 있다.
> "혹은 내가 이미 가진 코드나 자료들을 AI한테 읽게 하고 그걸 바탕으로 책을 쓰게 할 수도 있어요." [01:03]

### 2단계: AI가 쓴 초안을 내가 정밀 검수
> "AI가 써 준 초안을 내가 검수해요. AI가 초안 써졌다고 그냥 쓰면 절대 안 됩니다. 내 언어로, 내 관점으로 다듬어야 돼요." [01:10][^3]

- 이 단계는 사람이 할 수밖에 없는 부분이지만, 시간은 크게 줄어든다.
> "이 부분은 어쩔 수 없이 내가 해야 하는 거긴 한데 시간이 확 줄어요. 처음부터 쓰는 거랑 다듬는 거랑 완전 다르거든요." [01:16]

### 3단계: Pinterest 레퍼런스 + PDF 스킬로 디자인 생성
> "여기서 진짜 꿀팁 하나 드릴게요. 이게 없으면 디자인이 좀 밋밋하게 나오거든요. 핀터레스트 가세요. 검색창에 PDF 디자인 템플릿 치세요. 딱 마음에 드는 거 하나 골라요." [01:41][^4]

- 화자는 미니멀한 스타일(흰 배경 + 깔끔한 글씨체)을 골랐다.
- 고른 이미지를 저장해 두고 Claude Code에 함께 던진다.
> "그 이미지 저장해 두고 클로드 코드한테 같이 던지는 거예요. 이 내용으로 전자책 만들어 줘. 디자인은 이 레퍼런스 참고해서 미니멀하게. 그럼 AI가 그 스타일대로 PDF를 뽑아 줘요." [01:54]

### 4단계: 깨진 부분을 정확히 지시해 하나씩 수정
처음 뽑은 PDF는 완벽하지 않다 — 이걸 정상으로 받아들이라고 강조한다.
> "처음 뽑은 PDF 완벽하게 나오는 경우는 없습니다. 글씨 크기[가 이상하]거나 줄 간격 망가지거나 페이지 넘어갈 때쯤 내용 잘리거나 이런 거 생기면 당황하지 마세요. 원래 그렇습니다." [02:03]

- **하면 안 되는 것**: "전체 다시 해 줘"라고 통째로 요청하기.
> "이때 하면 안 되는 게 전체 다시 해 줘. 이런 거예요. 그러면 또 답[이 안 나와요]. 정확하게 지시해야 돼요." [02:13]

- **올바른 방식**: 깨진 부분을 콕 집어 구체적으로 지시.
> "3페이지 챕터 제목 글자 크기 너무 커. 좀 줄여줘. 표가 페이지 밖으로 넘어가. 안에 딱 맞게 해 줘. 이렇게 하나씩 딱 집어서 고치는 거예요. 이 반복을 몇 번 하면 진짜 예쁜 전자책이 나와요." [02:21]

## 팁 (3가지 주의사항)

화자가 [02:32]에서 "중요한 주의 사항"으로 묶어 제시한 3가지.

### 주의 1: 적절한 스킬·에이전트(도구·역할)를 부여
- (위 "핵심 개념 — 도구와 역할 분리" 참조)
> "요리사한테 청소시키면 안 되잖아요." [02:43]

### 주의 2: 작업 하나 끝나면 새 대화창 시작
> "작업 하나 끝나면 대화창 새로 시작하세요. 클로드 코드는 대화 기록이 쌓이면 쌓일수록 느려지고 실수도 [잦아져요]. 마치 머릿속이 꽉 찬 것처럼요." [02:51][^5]

- 자료 조사 끝 → 새 대화. PDF 작업 끝 → 또 새 대화. **작업 단위로 끊는 게 안정적.**
> "작업 단위로 끊어 주는 게 훨씬 안정적이에요." [03:02]

### 주의 3: 50페이지를 넘기지 마라 (강조)
> "50페이지 넘기지 마세요. 이건 진짜 중요한 거예요. 전자책 욕심 내서 100페이지 만들려고 하면 AI가 중간에 내용을 잊어버려요. 뒤죽박죽되거나. 50페이지 이하로 딱 핵심만 담는 게 훨씬 완성도 있게 나와요." [03:04][^6]

### 디자인 팁: 무조건 미니멀하게
> "핀터레스트에서 레퍼런스 고를 때 이것만 기억하세요. 웬만하면 미니멀하게 가세요. 화려한 거 고르면 AI가 구현 못 하는 부분이 많이 생깁니다. 그라데이션 막 들어가고 복잡한 레이아웃이면 깨질 가능성이 높아요. 흰 배경, 깔끔한 포인트, 여백만 [활용한] 디자인 이런 미니멀 스타일이 오히려 결과물이 예쁘게 나와요." [03:19]

## 의견 / 후기

- [의견] 화자도 처음엔 회의적이었으나 직접 해 보니 됐다고 말한다.
> "솔직히 저도 처음엔 설마 이게 되겠어? 했어요. 근데 직접 해 보니까 되더라고요. 이것도 생각보다 훨씬 완성도 있게. 전자책 한 권 이제 더 이상 며칠짜리 프로젝트 아닙니다." [04:03]

- [의견] 기존 방식의 병목(자료 정리 이틀, 디자인 막막함, PDF 저장 시 글자 깨짐·이미지 날아감)을 직접 겪었다고 설명한다. [00:29]~[00:43]

## 미해결 / 불명확
- 영상에서 사용한 "PDF 스킬"의 실제 구현·출처는 명시되지 않음. (Anthropic 공식 pdf 스킬인지, 화자 자작 스킬인지 불명)
- "웹 리서치 에이전트"는 화자 자작이라고만 언급. 구성은 미공개.
- [02:13]의 "그러면 또 답[이 안 나와요]" 부분은 자동자막 끊김으로 정확한 어휘 불명확 — 문맥상 "원하는 결과가 안 나온다"는 취지로 보임.
