# digest-from-transcript (Layer 1 — paraphrase / 시간순 심층 워크스루)

자막을 **빠짐없이 읽을 수 있는 산문**으로 재서술한다. 요약이 아니다 — "영상 말을 끝까지 안 들어도 내용을 거의 다 파악하는 정리글"이 목표. 이 산출물이 Layer 2(summary)의 소스가 된다.

## 입력
- YouTube URL 또는 이미 확보된 자막 clean text 경로
- 저장 경로 (호출 측 지정)

## 워크플로
1. **자막 확보**: study-note 스킬 Step 3 경로 재사용 (yt-dlp Plan A → MCP 폴백). clean text 파일이 주어지면 그걸 읽는다.
2. **메타 확보**: 제목·채널·게시일·조회수·길이.
3. **1패스 통독**: 자막 전체를 읽어 흐름과 구간 전환점 파악.
4. **paraphrase 작성** (아래 포맷). 영상 처음~끝을 흐름 전환마다 구간으로 나눠 각 구간을 2~4문장으로 **빠짐없이** 서술.
5. **저장 + 보고**.

## 포맷

```markdown
# [영상 원제 — 편집 금지, 그대로]

> 채널 · 게시일 · 조회수 · 길이 · [영상 링크](URL)
> 정리 방식: 시간순 심층 워크스루 (paraphrase)
> [검증 배너 — 수익/홍보 영상이면: "수치는 화자 주장, 미검증"]

## 3줄 오리엔테이션
- 읽기 전 방향 3줄 (영상이 결국 뭘 말하는지)

## 전문 — 시간순 워크스루
### [mm:ss–mm:ss] 구간 소제목
그 구간 내용을 빠짐없이 2~4문장으로 서술.
- 화자 주장·수치는 "(주장/미검증)"로 표시, 객관 사실과 섞지 않음
- [의견] 태그로 화자 주관 구분
- 인상적/핵심 발언은 > "원문" [mm:ss] 인용
### [다음 구간] …
(처음~끝, 15~30구간까지 무방)
```

## 규칙
- **오인식 자막 교정**: 자동자막의 명백한 음차·오인식은 문맥으로 알아서 교정해 자연스러운 표기로 쓴다 (예: 음차된 브랜드명·인명). 문맥으로도 불확실하면 `[불명확: 원문 "..."]`로 남기고 단정하지 않는다.
- **완전 충실**: 실질 내용 누락 금지. 버리는 것 = 순수 반복·필러·광고 삽입구·반복 CTA. 단 "유료자료·단톡방 유입 구조" 같은 세일즈 장치는 **한 번은 기록**한다(정보로서).
- **분량 하드캡 없음**. 목표는 자막의 대략 20~40% (물만 빼고 내용은 다 담기). 억지 압축 금지.
- **정확성 규율 완전 유지**: 화자 주장/미검증 표시, `[의견]` 분리, `[불명확]`, `[mm:ss]` 인용. 마케팅 톤 옮기지 않기.
- **제목은 영상 원제 그대로** (자체 편집·클릭베이트 순화 금지).
- **타임스탬프 날조 금지** — 자막의 실제 값만.

## Negative Space
- study-note를 만들지 않는다 (별개 워크플로). 주제별 재구성도 하지 않는다 (그건 Layer 2).
- 요약하지 않는다 — 이건 "충실한 재서술" 레이어. 압축은 summarize 스킬 몫.
