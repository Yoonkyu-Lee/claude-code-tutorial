# digest 공유 포맷 (Method A·B 공통 목표 산출물)

두 digest 방식(노트기반 A / 자막기반 B)은 **소스·프로세스만 다르고 출력 포맷은 동일**하다 — 공정한 A/B 비교를 위해서다. 영상(MJiQFNp-k10)이 안내한 요약 포맷을 기반으로 한다.

```markdown
# [영상 제목]

> 채널 · 게시일 · 조회수 · 길이 · [영상 링크](URL)
> digest 방식: note-based(A) | transcript-based(B)   ← 실험 식별용

## 핵심 요약
- 3~5개 불릿. 영상의 큰 그림을 압축. 각 불릿은 한 문장.

## 파트별 타임라인
- [mm:ss] **소제목** — 한 줄 내용
- [mm:ss] **소제목** — 한 줄 내용
  (영상 흐름을 5~10개 구간으로. 타임스탬프는 소스에서 가져옴.)

## 한 줄 인사이트
> 이 영상에서 가져갈 단 하나의 문장.
```

## 규칙 (두 방식 공통)
- **압축이 목적**: study-note와 달리 원문 보존이 아니라 핵심만 뽑는다. "훑고 넘기기"용.
- 마케팅·과장 톤 금지. 사실과 화자 주장이 섞이면 화자 주장임을 드러낸다.
- 타임라인 타임스탬프는 소스(노트 인용 또는 자막 라인)에서 실제 값을 쓴다 — 지어내지 않는다.
- 분량 목표: 화면 한 눈(대략 원문 자막의 5% 이하).
