# Harness Study Note Skill

CLAUDE.md의 작업 우선순위·노트 작성 규칙·negative space를 따른다. 이 파일은 그 규칙을 **어떻게** 실행하는지 정의한다.

## 원칙 (충돌 시 우선순위)

이 스킬의 단계·룰은 **일관성**과 **사용자 helpfulness** 두 축을 만족하기 위한 도구다.
둘이 충돌할 때:

1. **충실성·정직성 룰**(원문 보존, 의견/사실 분리, 불명확 표기, 출처 명시)은 **양보 불가**
2. 그 외 영역에서 일관성 vs helpfulness가 충돌하면 **helpfulness 우선**
3. 명세에 없지만 사용자에게 유용한 부수 정보가 보이면 **자발적으로 보강**하라 —
   단, 추가한 곳에 무엇을 왜 추가했는지 Step 8 보고에서 명시

## 입력
- YouTube URL (필수)
- 사용자가 게시일을 직접 알려주면 그걸 우선 사용 (필수 fallback 줄임)

## 워크플로

### Step 0.5: 중복 검사 (자막 가져오기 전에 먼저)
URL에서 **영상 ID**를 추출한다. (`v=` 파라미터 값, 또는 `youtu.be/` 뒤의 11자)
영상 ID는 의역 파일명과 달리 불변이므로 중복 감지의 신뢰할 수 있는 키다.

프로젝트 폴더의 모든 `.md`를 훑어 `영상 ID: ` + 추출한 ID가 들어있는 줄이 있는지 검사한다
(Glob으로 `*.md` 수집 → Grep 또는 Read).

이미 같은 ID의 노트가 존재하면:
- **단독 실행**(/study-note 또는 직접 호출): 사용자에게 "이미 `<파일명>`으로 정리됨. 다시 만들까요?"라고 묻고, 답을 받기 전엔 진행하지 않는다.
- **일괄 실행**(study-note-worker로 호출됨): 자막을 가져오지 말고 **즉시 종료**한다. 출력 형식의 상태를 `중복(skipped)`으로, 파일 필드에 기존 파일명을 담아 호출 측에 보고한다.

중복이 아니면 Step 1로 진행.

이 단계를 자막 가져오기보다 **먼저** 두는 이유: 중복이면 자막 호출·토큰을 통째로 절약한다.

### Step 1: 메타데이터 확보
`youtube-transcript` MCP의 `get_video_info`를 호출한다. 반환 필드에서:

- 게시일 (publish date / upload date)
- 영상 제목 (title)
- 영상 길이 (length, 메타 섹션에 기록용)
- 채널명 (메타 섹션에 기록용)

호출 실패 또는 **게시일 필드 누락 시 다음 fallback을 순서대로 시도**한다:

1. **yt-dlp** (1차 fallback, 가장 신뢰): Bash/PowerShell로 다음 명령 실행
   ```
   yt-dlp --skip-download --print "%(upload_date)s|%(title)s|%(channel)s|%(duration)s" <URL>
   ```
   출력 형식: `YYYYMMDD|제목|채널|초`. 게시일은 `YYYYMMDD` → `YYYY-MM-DD`로 변환.
   `yt-dlp` 미설치면 다음 단계로.

2. **WebSearch** (2차 fallback): "<영상 제목> youtube"로 검색해 결과에서 추출.

3. **사용자 입력** (최종 fallback): 위 모든 단계가 실패하면 사용자에게 게시일과 짧은 영문 제목을 물어보고 멈춤.

**`WebFetch`는 게시일 용도로 쓰지 않는다** — YouTube SPA 구조에서 신뢰할 수 없는 결과를 반환하기 때문 (`.claude/known-issues.md` 참조).

각 fallback이 동작한 경우 노트 메타 섹션에 출처를 짧게 명시한다 (예: `게시일 출처: yt-dlp`).

### Step 2: 파일명 생성
형식: `YYYY-MM-DD-kebab-case-english-title.md`

- 영문 제목은 3~6단어, 모두 소문자, 띄어쓰기는 `-`로
- 한국어 영상이면 핵심 의미를 영문으로 옮긴다 (음역 X, 의역 O)
  - 예: "하니스 완전 공개" → `harness-explained` 또는 `complete-harness-guide`
- 관사·채널명 제외
- 같은 이름 파일이 이미 있으면 **덮어쓰기 전 사용자 확인** (1차 중복 방어는 Step 0.5의 ID 검사. 이건 ID가 다른데 우연히 파일명만 겹치는 드문 경우용 2차 방어)

### Step 3: 자막 가져오기
1차 시도: `get_timed_transcript` (타임스탬프 포함)
실패 시 fallback: `get_transcript` (타임스탬프 없음, 노트 메타 섹션에 "타임스탬프 미제공" 명시)

`get_available_languages`는 호출하지 않는다 — 한국어 자동 자막이 없는 영상이 나오면 그때 fallback으로 사용 검토. (대부분의 메이커 에반 영상은 한국어 자동 자막 존재)

### Step 4: 자막 품질 정정
자동 자막의 흔한 오인식을 **컨텍스트로 복원**한다.

- **명백한 오인식**(기술 용어·고유명사 등 컨텍스트로 95% 이상 확실): 정정 후 본문에 사용. 첫 등장 시 한 번 보정 사실을 각주로 노출
- **모호한 발화**(컨텍스트로도 의도가 불확실): `[불명확: 원문 "..."]`로 표시하고 추측 금지

**정정 사전**: `references/caption-corrections.md`에 자주 등장하는 영문 고유명사·기술 용어의 음차 → 정확 표기 매핑이 있다. Step 4를 시작할 때 이 파일을 한 번 읽어 머릿속에 띄워두고 자막을 훑는다. 새 패턴을 발견하면 사용자가 명시 요청한 경우에만 사전을 보강한다 (자동 갱신 금지). 사전이 존재하는 배경은 `.claude/known-issues.md` 참조.

흔한 패턴 (사전의 일부):
- 기술 용어: `패스트 APA` → `FastAPI`, `넥트 JS` → `Next.js`, `멀이드` → `Mermaid`
- 영문 약어 한글 표기: `M C P` → `MCP`, `에이 피 아이` → `API`
- 채널 고유 용어: `한스` → `하니스`, `콜드` → `Code` (Claude Code 문맥)
- Claude 모델: `오퍼스`/`소넷`/`하이쿠` → `Opus`/`Sonnet`/`Haiku`

### Step 5: 영상 유형 판별 + 섹션 매핑
자막 전체를 한 번 훑은 뒤, 영상 성격을 분류:

- **튜토리얼/실습형**: 단계별 명령·설정 위주 → 시간순 워크플로 섹션 강조
- **개념 설명형**: 비유·정의·구조 위주 → 핵심 개념 섹션 강조
- **사례 소개형**: 사용 예시·결과물 위주 → 카탈로그 섹션 강조
- **혼합형**: 위 조합

CLAUDE.md의 default 섹션 구조(출처 → 한 줄 요약 → 메타 → 핵심 개념 → 카탈로그 → 팁 → 환경 셋업 → 외부 보강 → 다음 단계 → 미해결)를 **기본 틀**로 사용하되:

- 해당 영상에 내용이 **없는 섹션은 생략** (억지로 채우지 않음)
- 영상에 default로 못 담는 substance가 있으면 **새 섹션 추가**
- 추가/생략 결정은 Step 8 보고에 명시

### Step 6: 본문 채우기
CLAUDE.md 3번 규칙 엄수:
- 화자 발언 원문은 `> "..."` + `[mm:ss]`
- `[의견]`과 객관 사실 분리
- 불명확은 `[불명확]`
- 외부 보강 시 출처 명시 (URL 또는 문서명)

**"출처" 섹션에는 반드시 영상 ID 줄을 포함한다** (Step 0.5의 중복 검사가 이 줄을 읽는다):

```
## 출처
- 영상: [제목](URL)
- 영상 ID: `<11자 영상 ID>`
- 채널: ...
- 게시: YYYY-MM-DD
```

누락하면 안 되는 항목:
- **화자가 강조한 발언** ("이건 중요", "꼭", "한 번 해두면", "처음에는 ~지만" 같은
  대조 구문). 자막에 별표가 없으니 의미 단서로 잡아야 함.
- "다음 단계" 섹션은 두 칸으로 분리:
  - "다음 단계 (영상이 언급한 후속)" — 화자가 예고한 것
  - "다음 단계 (이 노트의 후속 작업)" — 사용자가 할 후속 — 빈 채로 두기 (사용자 채움)

### Step 7: 외부 보강 (선택)
영상 정보만으로 부족한 부분이 있을 때만:
- `WebSearch`로 공식 문서·블로그 검색
- `WebFetch`로 본문 가져와 인용
- 보강은 "## 외부 보강" 섹션 안에 모아두고 영상 내용과 섞지 않는다

### Step 8: 저장 + 보고
파일을 프로젝트 루트(CLAUDE.md와 같은 위치)에 UTF-8로 저장.

사용자에게 보고:
- 저장 경로
- 사용한 섹션 / 생략한 섹션 (+ 이유 한 줄)
- 추가한 섹션이 있다면 그것 (+ 이유)
- 자발적 보강이 있다면 무엇을 왜 추가했는지 (원칙 3번)
- 외부 보강 출처 목록
- `[불명확]` 마커가 N개 있다면 보고 (검수 유도)

영상 내용 자체를 보고 안에서 요약하지 않음 — 파일을 열어보면 됨.

### Step 8.5: 영상 재훑기
저장 직전 자막 전체를 한 번 더 빠르게 훑으면서, 본문에 인용 안 된 발언 중
"의외로 중요해 보이는" 게 없는지 확인. 있으면 추가. 없으면 그대로 저장.

이 단계는 첫 패스에서 빠진 메시지를 잡기 위한 안전망.

### Step 9: INDEX.md 색인 항목 산출

프로젝트 루트의 `INDEX.md`는 모든 study-note의 색인이다. 한 노트가 새로 생기면 색인에도 항목 하나가 들어가야 한다.

**색인 메타 5종**을 산출한다 (이 단계가 끝나야 노트가 진짜로 완료된 것이다):

1. **링크 줄**: `## YYYY-MM-DD — [영문 제목](파일명.md)` (영문 제목은 H1 헤딩의 제목 부분)
2. **한 줄 요약**: 노트의 `## 한 줄 요약` 섹션 본문 그대로
3. **태그**: 노트가 실제로 다루는 주제 키워드 5~8개 (해시태그 형태, `#ClaudeCode` 등). 노트에 없는 주제를 만들지 않는다. 표기 규칙은 `references/index-tag-conventions.md`가 있으면 참조, 없으면 영문은 공백 없이(PascalCase 또는 그대로), 한국어는 그대로.
4. **도구·스킬**: 노트가 다루는 핵심 도구/스킬을 쉼표로 나열
5. **영상 유형**: 개념설명 / 튜토리얼 / 카탈로그 / 혼합 중 하나 (Step 5에서 판별한 결과 그대로)

#### 단독 실행과 일괄 실행의 분기

- **단독 실행**(사용자가 직접 호출 또는 `/study-note`): worker가 직접 `INDEX.md`를 편집해 항목을 게시일 순서에 끼워 넣는다. 같은 영상 ID의 기존 항목이 있으면 새 항목으로 덮어쓴다.
- **일괄 실행**(`study-note-worker`로 dispatch됨): worker는 `INDEX.md`를 **직접 편집하지 않는다** (병렬 worker 사이 race condition 위험). 대신 Step 9의 5종 메타를 종료 보고에 그대로 실어 메인에게 넘긴다. INDEX 갱신은 `/batch-notes` 메인이 일괄로 처리한다.

#### 종료 보고에 포함할 형식

일괄 실행 시 worker의 종료 보고는 본문 출력 없이 다음 블록을 포함한다:

```
INDEX_BLOCK_BEGIN
## YYYY-MM-DD — [영문 제목](파일명.md)

**한 줄 요약**: ...

**태그**: #태그1 #태그2 ...

**도구·스킬**: ..., ...

**영상 유형**: ...
INDEX_BLOCK_END
```

이 블록을 그대로 메인이 `INDEX.md`에 붙일 수 있게 한다.

## Negative Space (CLAUDE.md 6번 보강)
- 이 SKILL.md를 작업 중 수정하지 않는다
- 기존 노트 파일을 수정·삭제하지 않는다 (사용자가 명시 요청 시에만)
- 영상이 default 패턴과 매우 안 맞으면 **강제로 끼우지 말고 사용자에게 보고**한 뒤 결정 (CLAUDE.md 7번)

## 검증 신호 (스킬이 잘 굴러갔다는 지표)
- 같은 영상을 두 번 돌렸을 때 파일명·섹션 구조가 동일
- 인용된 모든 발언에 `[mm:ss]` 있음
- `[의견]`이 한 번이라도 등장 (영상에 의견이 있는 경우)
- 외부 보강이 본 내용과 시각적으로 분리되어 있음
- Step 8 보고에 자발적 보강이 명시되어 있음 (있는 경우)
