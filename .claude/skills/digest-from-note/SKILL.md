# digest-from-note (Method A — 노트기반, 독자 방식)

이미 저장된 study-note(`notes/<주제>/<채널>/*.md`)를 **소스로** digest를 만든다. 자막을 다시 읽지 않는다 — 우리가 이미 검증·정정·구조화한 텍스트를 재사용해 정확성·일관성을 높인다.

산출 포맷은 `.claude/skills/_shared/digest-format.md`를 **그대로** 따른다 (Method B와 동일 포맷 → 공정 비교).

## 입력
- 소스 노트 경로 (`notes/<주제>/<채널>/YYYY-MM-DD-title.md`)
- 저장 경로 (호출 측이 지정; 실험 시 `experiments/.../digest-A-note.md`)

## 워크플로
1. **노트 통독**: 소스 노트를 읽는다. 활용할 섹션:
   - `## 출처` → 메타(제목·채널·게시일·링크). 조회수는 노트에 없으면 호출 측/`yt-dlp`로 보강.
   - `## 한 줄 요약` → digest의 한 줄 인사이트 씨앗.
   - `## 핵심 개념` / `## 카탈로그` → 핵심 요약 불릿의 재료.
   - 본문의 `[mm:ss]` 인용 → 파트별 타임라인의 타임스탬프.
2. **압축**: `_shared/digest-format.md` 포맷으로 채운다.
   - 핵심 요약 3~5불릿: 노트의 핵심 개념·카탈로그를 상위 개념으로 압축.
   - 파트별 타임라인: 노트에 이미 있는 `[mm:ss]` 값을 재사용해 5~10구간.
   - 한 줄 인사이트: 노트 `한 줄 요약`을 다듬어 단 한 문장으로.
3. **저장 + 보고**: 지정 경로에 저장. 어떤 노트 섹션을 썼는지 보고.

## 원칙
- 노트가 이미 지킨 규율(의견/사실 분리, `[불명확]`)을 digest에서도 **유지**한다 — 노트가 `[불명확]`으로 둔 걸 digest에서 단정하지 않는다.
- 노트에 없는 사실을 새로 만들지 않는다 (자막 재확인은 이 방식의 범위 밖 — 필요하면 Method B).
- 압축이 목적. 노트의 긴 인용을 그대로 옮기지 않는다.

## Negative Space
- 자막을 새로 가져오지 않는다 (노트가 유일 소스). 노트가 없으면 먼저 study-note를 만들어야 함.
- 원문 노트를 수정하지 않는다 (읽기 전용 소스).
