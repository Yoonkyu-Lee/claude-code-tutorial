# CLAUDE.md

이 파일은 Claude Code가 모든 세션에서 자동 로드합니다. 프로젝트의 정체성·톤·규칙을 정의해요.

## 1. 프로젝트 정체성

이 저장소는 **유튜브 영상 기반 다주제 연구 노트 모음**입니다.  
여러 유튜브 채널 영상과 공식 문서를 교차 검증해서 마크다운 노트로 누적하고, 그 노트들을 발판으로 사용자에게 유용한 판단을 내립니다. 시작은 Claude Code·AI 코딩(`ai-coding` 주제)이었고, 앞으로 비즈니스 등 다양한 주제로 확장합니다.

산출물은 두 종류:
- `notes/<주제>/<youtube-handle>/YYYY-MM-DD-간단영문제목.md` — 영상 1편 = 노트 1편 (주제 → 채널 → 노트, 영상 게시일 기반)
- `.claude/` 하위의 스킬·에이전트·커맨드 — 이 노트들에서 추출한 패턴을 도구화한 것

### 주제·채널 폴더 규약
- 계층: `notes/<주제>/<채널-handle>/`. 주제 슬러그는 소문자 kebab (예: `ai-coding`, `business`). 채널 폴더명 = 유튜브 `@핸들` 그대로.
- INDEX 3계층: 루트 `index.md`=주제 카탈로그, `notes/<주제>/INDEX.md`=채널 카탈로그, `notes/<주제>/<채널>/INDEX.md`=노트 색인.
- 채널은 한 주제에 고정. 이미 있는 채널이면 그 주제를 그대로 쓰고, **새 채널/새 주제는 사용자에게 확인** 후 폴더·INDEX 생성 + 상위 카탈로그에 행 추가.

## 2. 사용자 컨텍스트 (한 번만 적고 끝)

- 한국어 모국어, 영어 능숙 (UIUC ECE 졸업)
- 학부 수강: ECE 385, 391, 408, 411, 444, 479, 481
- 비전공자도 알아들을 비유 환영
- 환경: ASUS ROG Zephyrus G16 (Windows 네이티브), RTX 4080 Laptop GPU, WSL2도 있음.

## 3. 노트 작성 규칙 (모든 노트 공통)

- **언어**: 본문 한국어. 명령어·파일경로·코드는 영어 원문 그대로
- **톤**: 학술 정리 + 실용 가이드. 마케팅 톤·과장 금지
- **포맷**: `notes/<주제>/<채널핸들>/YYYY-MM-DD-간단영문제목.md`. 영문 제목은 띄어쓰기를 대시로 대체, 전부 소문자로 통일.  
  섹션 구조 기본 템플릿: 출처(영상 링크, 제목, **영상 ID**, 채널, 게시일) → 한 줄 요약 → 메타 → 핵심 개념 → 카탈로그 → 팁 → 환경 셋업 → 외부 보강 → 다음 단계 → 미해결
- **인용**: 화자 발언 원문은 `> "..."` 인용 블록 + `[mm:ss]` 타임스탬프
- **의견 분리**: 화자 주관(`[의견]`)과 객관 사실 절대 혼합 금지
- **불명확**: 추측 금지, `[불명확]` 명시
- **외부 보강**: 영상 정보만으로 부족한 부분은 공식 문서로 보강하되 출처 명시

## 4. 작업 우선순위

새 노트를 만들 때 Claude는 다음 순서로 일합니다:

1. `harness-study-note` 스킬 로드 (있으면)
2. URL에서 채널 핸들 추출 → **주제 + 채널** 결정, 대상 폴더 `notes/<주제>/<핸들>/` (기존 채널이면 그 주제 사용; 새 채널/주제면 사용자에게 확인 후 생성)
3. 영상 ID로 중복 검사 — 이미 정리된 영상이면 건너뛰거나 사용자에게 확인 (검사 범위: `notes/**/*.md`)
4. 게시일 기반 파일명 생성
5. 영상 자막 가져오기 (yt-dlp 1차 / transcript MCP 폴백 — 상세는 SKILL.md Step 3)
6. 자막을 스킬 룰북에 따라 구조화
7. 외부 보강 (필요 시 web_search)
8. 파일을 `notes/<주제>/<핸들>/`에 저장 후 사용자에게 경로 보고

여러 영상을 한꺼번에 처리할 때는 `/batch-notes` 커맨드가 각 영상을 `study-note-worker` 서브에이전트로 격리·병렬 처리한다.

채널을 정해 URL을 붙여넣는 대신, 추적 채널(`notes/<주제>/<채널>/`)의 어제 신규 영상을 **무인으로** 감지·정리하려면 `/collect-new` 커맨드를 쓴다(스케줄러 연동 가능). 인터랙티브 `/batch-notes`는 "승인 후 진입" 가드레일을 유지하고, 무인 `/collect-new`는 그 가드레일을 구조적 4중 상한(시간창·하드캡·ID중복·동시성)으로 대체한다.

## 5. 파일 명명 규약

- 노트: `notes/<주제>/<youtube-handle>/YYYY-MM-DD-kebab-case-english-title.md` (핸들은 유튜브 `@핸들`에서 `@` 제거, 주제는 소문자 kebab)
- 채널 색인: `notes/<주제>/<youtube-handle>/INDEX.md`
- 주제 색인: `notes/<주제>/INDEX.md` (채널 카탈로그)
- 루트 색인: `index.md` (주제 카탈로그)
- 스킬: `.claude/skills/<kebab-case-name>/SKILL.md`
- 에이전트: `.claude/agents/<kebab-case-name>.md`
- 커맨드: `.claude/commands/<kebab-case-name>.md`

## 6. 안 하는 것 (negative space)

- 영상 내용을 **요약**하지 않음 (원문 보존이 원칙)
- 화자가 안 한 말을 채워 넣지 않음
- 영상 정리 작업 중 `CLAUDE.md`·`SKILL.md` 자동 수정 금지 (사용자가 명시 요청 시에만)
- 노트 파일을 **삭제**하지 않음 (수정만, 삭제는 사용자 확인 후)

## 7. 막혔을 때

- 자막 못 가져옴 → 사용자에게 보고하고 멈춤 (수동 transcript 입력 옵션 제안)
- 영상이 길어 토큰 한계 → 챕터 단위로 분할 처리 후 머지
- 영상 기조가 기존 패턴과 안 맞음 → 강제로 끼워 맞추지 말고 사용자에게 보고

## 8. Git 습관
- 의미 있는 작업 단위가 끝나면 Claude Code가 알아서 `git add` + 적절한 메시지로 커밋한다.
- 푸시는 사용자 확인 후 (또는 사용자가 "push까지 해줘"라고 하면 바로).

## 9. 자기 개선 루프 (발견은 자율, 수정은 승인)

Claude Code는 작업 중 반복 재발하는 문제를 발견하면 다음을 따른다:

1. **발견·기록 (자율)**: 같은 유형의 문제가 2회 이상 재발하면 인지하고,
   작업 종료 보고에 "재발 패턴 발견" 항목으로 보고한다.

2. **제안 (자율)**: 해결법 후보를 2~3개 제시한다. 각 후보의 트레이드오프
   (외부 의존, 복잡도, 신뢰성)를 한 줄씩. 추천안과 그 근거도.

3. **수정 (승인 필수)**: `.claude/` 하위 문서(CLAUDE.md, SKILL.md, 에이전트,
   커맨드, 사전 등)를 고치기 전에 **반드시 사용자 확인을 받는다.**
   - 사용자가 명시적으로 "고쳐"라고 한 경우에만 수정
   - 수정 시, 사용자가 과거에 내린 결정과 충돌하지 않는지 먼저 점검한다.
     충돌하면 수정 전에 그 충돌을 보고한다. (예: 사용자가 "X 쓰지 마"라고
     했는데 해결법이 X를 쓰면, 임의로 진행하지 말고 보고)
   - 수정 후에는 무엇을 왜 바꿨는지 요약하고 커밋 메시지에 명시한다.

4. **금지**: 사용자 결정을 자기 판단으로 뒤집지 않는다. 더 나은 방법이
   보여도 제안에 그치고, 채택은 사용자 몫이다.
