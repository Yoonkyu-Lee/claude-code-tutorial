# CLAUDE.md

이 파일은 Claude Code가 모든 세션에서 자동 로드합니다. 프로젝트의 정체성·톤·규칙을 정의해요.

## 1. 프로젝트 정체성

이 저장소는 **Claude Code 활용법 연구 노트 모음**입니다.  
유튜브 영상(주로 메이커 에반)과 공식 문서를 교차 검증해서 마크다운 노트로 누적하고, 그 노트들을 발판으로 사용자의 실제 환경을 구축하는데 유용한 판단을 내립니다.

산출물은 두 종류:
- `claude-code-harness-study-note-NN.md` — 영상 1편 = 노트 1편 (번호 누적)
- `.claude/` 하위의 스킬·커맨드 — 이 노트들에서 추출한 패턴을 도구화한 것

## 2. 사용자 컨텍스트 (한 번만 적고 끝)

- 한국어 모국어, 영어 능숙 (UIUC ECE 졸업)
- 학부 수강: ECE 385, 391, 408, 411, 444, 479, 481
- 비전공자도 알아들을 비유 환영
- 환경: ASUS ROG Zephyrus G16 (Windows 네이티브), RTX 4080 Laptop GPU, WSL2도 있음.

## 3. 노트 작성 규칙 (모든 노트 공통)

- **언어**: 본문 한국어. 명령어·파일경로·코드는 영어 원문 그대로
- **톤**: 학술 정리 + 실용 가이드. 마케팅 톤·과장 금지
- **포맷**: 영상게시날짜로인덱싱-간단영문제목.md의 섹션 구조를 기본 템플릿으로 따름. 띄어쓰기는 대시로 대체, 전부 소문자로 통일.  
  (출처(영상 링크, 제목 등) → 한 줄 요약 → 메타 → 핵심 개념 → 카탈로그 → 팁 → 환경 셋업 → 외부 보강 → 다음 단계 → 미해결)
- **인용**: 화자 발언 원문은 `> "..."` 인용 블록 + `[mm:ss]` 타임스탬프
- **의견 분리**: 화자 주관(`[의견]`)과 객관 사실 절대 혼합 금지
- **불명확**: 추측 금지, `[불명확]` 명시
- **외부 보강**: 영상 정보만으로 부족한 부분은 공식 문서로 보강하되 출처 명시

## 4. 작업 우선순위

새 노트를 만들 때 Claude는 다음 순서로 일합니다:

1. `harness-study-note` 스킬 로드 (있으면)
2. 다음 노트 번호 자동 산정 (`ls *.md`로 최신 번호 +1)
3. 영상 자막 가져오기 (transcript MCP 사용)
4. 자막을 스킬 룰북에 따라 구조화
5. 외부 보강 (필요 시 web_search)
6. 파일 저장 후 사용자에게 경로 보고

## 5. 파일 명명 규약

- 스킬: `.claude/skills/<kebab-case-name>/SKILL.md`
- 커맨드: `.claude/commands/<kebab-case-name>.md`

## 6. 안 하는 것 (negative space)

- 영상 내용을 **요약**하지 않음 (원문 보존이 원칙)
- 화자가 안 한 말을 채워 넣지 않음
- 영상 정리 작업 중 `CLAUDE.md`·`SKILL.md` 자동 수정 금지 (사용자가 명시 요청 시에만)
- 노트 파일을 **삭제**하지 않음 (수정만, 삭제는 사용자 확인 후)

## 7. 막혔을 때

- 자막 못 가져옴 → 사용자에게 보고하고 멈춤 (수동 transcript 입력 옵션 제안)
- 영상이 길어 토큰 한계 → 챕터 단위로 분할 처리 후 머지
- 영상 기조가 기존 패턴과 안 맞음 → 강제로 끼워 맞추지 말고 사용자에게 보고
