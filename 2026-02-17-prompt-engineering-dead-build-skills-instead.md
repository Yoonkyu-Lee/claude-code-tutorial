# 프롬프트 엔지니어링은 죽었다 — 대신 스킬을 만들어라

## 한 줄 요약
모델이 바뀔 때마다 최적 프롬프트가 달라지고 최신 추론 모델은 기존 프롬프트 기법(Chain-of-Thought 등)을 오히려 쓰지 말라고 하므로, 사람이 모델별 민감도를 추적·최적화하는 "프롬프트 엔지니어링"은 의미를 잃었고, 대신 재사용 가능한 스킬·컨텍스트·파이프라인 구조를 만드는 쪽으로 옮겨가야 한다는 주장.

## 핵심 개념

### 1. 모델이 바뀌면 최적 프롬프트도 바뀐다
화자는 OpenAI가 모델을 낼 때마다 새 프롬프팅 가이드를 발행한다는 점을 근거로 든다.

> "오픈 AI가 모델 하나 낼 때마다 프롬프팅 가이드를 새로 쓰고 있어요. 이게 무슨 뜻이냐면요. 모델이 바뀌면 프롬프트도 바뀌어야 한다는 거예요. 오픈 AI가 직접 인정하고 있는 거거든요." [00:15]

GPT-4.1 가이드를 직접 인용한다:

> "다른 모델의 최적화된 프롬프트는 이 모델에서 [증시/즉시] 작동하지 않을 수 있습니다. 공식 문서입니다." [00:27]

[불명확: 원문 "증시 작동하지 않을 수 있습니다" — '즉시(immediately)'의 오인식으로 보이나 단정하지 않음. 실제 GPT-4.1 공식 가이드 문구는 "외부 보강" 섹션 참조.]

### 2. 최신 추론 모델은 기존 기법을 쓰지 말라고 한다
같은 OpenAI 안에서도 모델 세대에 따라 권고가 정반대로 뒤집힌다는 대조 구문이 이 영상의 핵심이다.

> "GPT 4대는 모델한테 생각할 시간을 줘라. Chain-of-Thought를 적극 권장했어요. 근데 [GPT-5] 나오니까요. 같은 오픈 AI가 think 스텝 바이 스텝 쓰지 마세요. 불필요합니다. 오히려 성능이 떨어진다고 공식 문서에 써 놨어요." [00:34]

[보정: 자막 "5원" → 문맥상 "GPT-5" 계열, "체인 오브 더우트/또우트" → Chain-of-Thought (CoT), "출론 모델" → 추론(reasoning) 모델. 이하 동일.]

> "[GPT-5.1] 가이드에서는 더 노골적이에요. [추론] 모델에게 더 생각하라고 요청하면 오히려 성능이 나빠집[니다]." [00:46]

화자의 정리:

> "같은 회사, 같은 오픈 AI인데 모델마다 [프롬프팅] 규칙이 완전히 다른 거예요." [00:52]

이 현상이 OpenAI만의 문제가 아니라는 점도 덧붙인다.

> "근데 이게 오픈 AI만 그런 게 아닙니다. [Anthropic] [Claude]도 비슷한 논조로 이야기를 계속하고 있어요." [00:57]

### 3. 사람이 모델별 민감도를 다 추적하는 건 불가능하다 [의견]
> "프롬프트 엔지니어링을 제대로 하려면요. 각 모델별로 민감도를 내가 직접 파악해야 돼요. 근데 모델이 업데이트 되면 또 다 바뀌어요. 오픈 AI가 마이그레이션 가이드를 따로 써야 할 정도로요." [01:23]

> "[의견] 이걸 사람이 다 추적하려 하면서 최적화하는 게 가능할까요? 저는 연구원이 아닌 이상은 불가능하다고 생각합니다." [01:36]

### 4. 미래는 스킬(Skills)이다
"프롬프트 엔지니어링이 죽었다"가 "AI 잘 쓰는 법이 사라졌다"는 뜻은 아니라는 전환. 화자는 Anthropic 개발자들의 발표를 근거로 든다.

> "최근에 [Anthropic] 개발자들이 AI 엔지니어 [컨퍼런스/서비스]에서 발표를 하나 했어요. 이 발표에서 [Anthropic] 개발자들이 뭐라고 했냐면요. 스킬이 미래다라고 했어요." [01:51]

[불명확: 원문 "AI 엔지니어 서비스" — 'AI Engineer 컨퍼런스(Summit/Conf)'의 오인식 가능성이 있으나 영상만으로는 행사명을 단정할 수 없음.]

스킬의 이점을 화자가 정리한 방식:

> "스킬은 만들어 놓으면 매번 긴 프롬프트를 안 써도 됩니다. 모델이 바뀌어도 워크플로우는 그대로고요. 계속 재사용할 수 있어요." [01:59]

> "프롬프트 한 줄로 AI를 조종하려는 시대는 끝났어요. 대신 AI한테 어떤 맥락을 줄 것인가, 어떤 워크플로우로 돌릴 것인가? 스킬을 만들고 컨텍스트를 구조[화]하고 특정 작업에 맞는 파이프라인을 짜는 거예요." [02:05]

## 최종 정리 (화자의 4줄 요약)
화자가 영상 끝에서 직접 정리한 네 가지 [02:17–02:32]:

1. 하나, 제너럴하게(범용으로) 먹히는 프롬프트는 없다.
2. 둘, 최신 모델들은 기존 프롬프트 기법을 쓰지 말라고 한다.
3. 셋, 모든 모델 민감도를 전부 다 파악하는 거 아니면 의미가 없다. [보정: 자막 "일이 없다" → 문맥상 "의미가 없다"로 해석]
4. 넷, 미래는 스킬(Skills)이다 — 프롬프트 한 줄이 아니라 AI가 일하는 구조를 만드는 것.

> "프롬프트 엔지니어링 공부하지 마세요. 대신 AI한테 일을 시키는 구조를 공부하세요." [02:31]

## 팁
- [의견] 모델 세대 교체가 잦은 환경에서는, 특정 모델에 과적합된 프롬프트보다 모델이 바뀌어도 살아남는 워크플로우(스킬·컨텍스트·파이프라인)에 투자하는 것이 비용 대비 효과가 크다는 것이 화자의 입장이다.
- 추론(reasoning) 모델에 "단계별로 생각하라(think step by step)" 같은 CoT 지시를 기계적으로 붙이는 것은 최신 가이드 기준 권장되지 않으며, 오히려 지연·성능 저하를 부를 수 있다 (근거는 외부 보강 참조).

## 외부 보강

영상의 인용·수치 주장을 공식 문서·논문으로 교차 확인했다. 영상 발화와 섞지 않기 위해 여기 모았다.

### GPT-4.1 프롬프팅 가이드 (영상 [00:27] 인용 대조)
- OpenAI 공식 GPT-4.1 Prompting Guide의 실제 문구: *"existing prompts optimized for other models may not immediately work with this model, because existing instructions are followed more closely and implicit rules are no longer being as strongly inferred."*
- 즉 영상의 "다른 모델의 최적화된 프롬프트는 이 모델에서 즉시(immediately) 작동하지 않을 수 있습니다"는 공식 문구와 일치한다. (출처: [GPT-4.1 Prompting Guide, OpenAI Cookbook](https://cookbook.openai.com/examples/gpt4-1_prompting_guide))
- OpenAI는 별도 [Prompt Migration Guide](https://developers.openai.com/cookbook/examples/prompt_migration_guide)도 제공한다 — 영상 [01:32]의 "마이그레이션 가이드를 따로 써야 할 정도" 주장과 부합.

### Chain-of-Thought가 성능을 떨어뜨릴 수 있다는 연구 (영상 [00:34], [01:05] 대조)
- Princeton 연구진(컴퓨터과학·심리학과)의 논문 "Mind Your Step (by Step): Chain-of-Thought Can Reduce Performance on Tasks Where Thinking Makes Humans Worse": 인간이 과하게 생각할수록 성능이 나빠지는 유형의 작업에서 CoT가 LLM/LMM 성능을 크게 떨어뜨린다는 결과. (출처: [Princeton CoCoSci 논문 PDF](https://cocosci.princeton.edu/papers/liu2025chain.pdf), [arXiv HTML](https://arxiv.org/html/2410.21333v1))
- 추론 모델의 "overthinking"(사소한 질의에도 과도한 추론 토큰 생성 → 지연·성능 저하) 문제는 다수 후속 연구에서 다뤄지고 있다.
- [불명확] 영상이 든 구체 수치 — "프린스턴 논문에서 [GPT-5 프리뷰]에 CoT 적용 시 성능 36포인트 하락" [01:05], "Wharton 경영대학 리포트에서 추론 모델 CoT 사용 시 시간만 80% 더 걸리고 향상은 거의 없음" [01:12] — 은 이번 검색으로 정확한 원 수치·출처를 특정하지 못했다. 방향성(CoT가 추론 모델에서 비용 대비 이득이 적거나 해가 될 수 있음)은 위 연구들과 일치하나, 36/80% 수치는 영상의 주장으로만 기록하고 검증 보류.

### 스킬(Skills) — "미래는 스킬이다" (영상 [01:51] 대조)
- Anthropic의 Agent Skills는 Claude가 특정 작업을 수행하도록 지침·스크립트·리소스를 패키징한 폴더 단위 구조다. 모델 세대가 바뀌어도 같은 스킬을 재사용할 수 있다는 점이 영상의 주장과 부합한다.
- [불명확] 영상이 인용한 "AI 엔지니어 서비스에서의 Anthropic 발표"의 정확한 행사명·연사·발표명은 영상만으로 특정 불가. 'AI Engineer' 컨퍼런스 계열로 추정되나 확증 자료를 찾지 못함.
