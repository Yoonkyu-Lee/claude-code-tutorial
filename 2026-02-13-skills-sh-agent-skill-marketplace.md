# skills.sh — AI 에이전트용 스킬 마켓플레이스 입문

## 출처
- 영상: [skills.sh 하나면 AI 코딩 레벨이 달라집니다](https://www.youtube.com/watch?v=jae2bVCCokc)
- 영상 ID: `jae2bVCCokc`
- 채널: 메이커 에반
- 게시: 2026-02-13 (게시일 출처: 디스패처 확정값 — yt-dlp)

## 한 줄 요약
skills.sh는 AI 에이전트(Claude Code·Cursor 등 18종 이상)에 `SKILL.md` 형태의 절차적 지식을 패키지로 설치·공유하는 마켓플레이스로, `npx skills add` 한 줄이면 깔리며 — 다만 커뮤니티 저품질·악성 스킬을 거르기 위해 공식 벤더 우선·설치수 확인·코드 점검의 3원칙을 지켜야 한다.

## 메타
- 영상 길이: 약 5분 14초
- 영상 유형: 혼합형 (개념 설명 + 설치 튜토리얼 + 스킬 카탈로그)
- 타임스탬프: 제공됨 (자동자막 기반)
- 자막 정정: 한국어 자동자막의 영문 고유명사 음차를 컨텍스트로 복원함 (아래 각주 참조)

> 자막 정정 각주: "스킬스.sh / 스스.SH" → `skills.sh`, "버셀" → `Vercel`, "MPX / MPM" → `npx / npm`, "리액트" → `React`, "쉐신 앱 라우터" → `Next.js App Router`(컨텍스트상 추정), "샤드 CN" → `shadcn/ui`, "리모션" → `Remotion`, "파이드 스킬" → `find-skills`, "PRES / 프랙티스" → `best practices`, "앤트로픽" → `Anthropic`, "코팔러" → `Copilot`, "레디" → `Reddit`, "기타" → `GitHub`(컨텍스트상 추정). 첫 등장 시 한 번만 보정 사실을 노출하고 이후 정정 표기를 사용한다.

## 핵심 개념

### skills.sh의 정체
> "MPM이 코드 라이브러리를 위한 패키지 매니저잖아요. skills.sh는 AI 에이전트의 지식을 위한 패키지 매니저예요." [00:59]

화자는 skills.sh를 **AI 에이전트 지식의 패키지 매니저**로 정의한다. npm이 코드 라이브러리를 다루듯, skills.sh는 에이전트가 따를 지식·능력을 다룬다.

> "쉽게 말해서요. AI 에이전트한테 새로운 능력을 주는 거예요. 마치 게임에서 캐릭터한테 스킬북을 먹이는 것처럼요." [01:13]

### 스킬의 실체는 마크다운 파일
> "스킬의 정체는요. 사실 별거 아니에요. 스킬.닷 MD라는 마크다운 파일입니다. 거기에 이런 상황에서는 이렇게 해라라고 써 놓으면 AI가 그걸 읽고 그대로 따르는 거예요." [01:35]

스킬은 `SKILL.md` 마크다운 파일이며, "이런 상황에서는 이렇게 해라"는 지침을 적어두면 AI가 읽고 따른다.

### 스킬을 쓰는 사람 vs 안 쓰는 사람
> "이걸 쓰는 사람은 AI한테 React 최신 best practices도 알려주고 웹 디자인 가이드라인도 심어지고 심지어 Remotion으로 영상 만드는 법까지 가르칠 수 있어요. 안 쓰는 사람은 기본 Claude에 기본 지식으로 그냥 쓰는 거죠. 같은 도구 쓰는데 결과물이 천지 차이인 거죠." [00:37]

화자의 핵심 대조: 같은 Claude·Cursor를 써도 스킬 적용 여부에 따라 결과물 품질이 갈린다는 것. (영상 도입부의 "아직도 기본 상태 그대로 쓰고 계신 거 아니에요?" [00:02]와 같은 메시지)

### 작동 예시 (React)
> "기본 Claude한테 컴포넌트 만들어 줘라고 하면 그냥 아는만큼 만들어 주잖아요. 근데 Vercel에서 만든 React 프랙티스 스킬을 설치하면 Claude가 서버 컴포넌트의 패턴도 알고 [Next.js] App Router 규칙도 알고 Vercel이 권장하는 방식 그대로 코드를 짜줘요." [01:27]

## 설치 방법

### 전체 공식 스킬 설치
> "터미널 열고 아래 명령어만 입력하시면 됩니다. 끝이에요. 진짜로 이 한 줄이면 Vercel에서 만든 공식 스킬들이 쫙 깔려요." [01:47]

영상 요약 파트의 발화 기준 명령은 `npx skills add` 한 줄이다. [04:32]

```
npx skills add
```

> 주의: 영상 자막에서 명령의 정확한 인자(전체 설치 vs 특정 패키지)가 화면 캡처 의존이라 음성만으로는 일부 모호하다. `[불명확: 자막 "이 한 줄이면 Vercel에서 만든 공식 스킬들이 쫙 깔려요" — 정확한 패키지명/플래그는 음성에 없음]`. 정확한 사용법은 아래 "외부 보강" 참조.

### 특정 스킬만 설치
> "특정 스킬 하나만 깔고 싶다. 그러면 뒤에 골뱅이 붙이고 이름을 적어 주시면 됩니다." [01:56]

`@` 뒤에 스킬 이름을 붙여 개별 설치한다. ("골뱅이" = `@`)

### 에이전트 자동 감지
> "여기서 진짜 편한게 뭐냐면요. CLI가 여러분이 어떤 에이전트를 쓰는지 감지를 하고 그거에 알맞게 설치를 해 줍니다." [02:00]

CLI가 사용 중인 에이전트를 감지해 그에 맞게 설치한다. 영상은 "Claude Code·Cursor·[Copilot] 등 18개 이상의 에이전트를 지원" [04:34]한다고 명시.

### 사이트에서 직접 받기
> "스킬.SH 사이트에 들어가 보세요. 그러면 트렌딩 스킬이랑 인기 스킬이 쫙 나와요. 카테고리별로 볼 수도 있고 검색에서 원하는 스킬도 찾을 수 있어요. 스킬 클릭하면 설명이랑 설치 명령어가 나오니까 복사에서 터미널에 붙여 넣기만 하면 됩니다." [03:16]

## 팁 — 안전하게 스킬 고르는 3원칙

화자는 무분별한 설치 위험을 강하게 경고한다:

> "skills.sh에 올라온 스킬을 아무거나 다 까시면은 안 됩니다. 왜냐하면 커뮤니티 제출 스킬 중에 품질이 좀 떨어지는 것들이 좀 많습니다. Reddit에서도 80%가 AI가 대충 만든 저품질이다. 이런 말이 나올 정도거든요. 심지어 보안 이슈도 있었어요. 일부 스킬에서 숨겨진 악성 명령어가 발견된 적이 있거든요." [03:46]

> 주의: "Reddit에서 80%가 저품질" [03:53]은 화자가 인용한 커뮤니티 발언으로, 출처·정량 근거가 영상에 제시되지 않았다. `[불명확: 수치 근거 미제시]`로 받아들이는 게 안전하다.

화자가 제시한 **3원칙** [04:05]:
1. **공식 벤더 스킬 우선** — "Vercel·Anthropic 프레임워크에서 만든 공식 스킬부터 까세요."
2. **설치수 확인** — "많이 깔린게 그만큼 검증된 것이라고 보시면 될 거 같고요." [의견]
3. **GitHub에서 코드 확인** — "SKILL.md 내용을 직접 읽어보고 수상한게 없는지 체크해요." [04:19]

## 화자가 정리한 요약 (영상 클로징)
> "스킬.sh는 여러분의 AI를 범용 도구에서 전문가로 업그레이드하는 가장 쉬운 방법이에요." [04:46]

영상 클로징에서 화자가 직접 정리한 4가지 [04:25]:
1. skills.sh는 AI 에이전트에 새로운 능력을 주는 스킬 마켓플레이스. Vercel이 만들었고 완전 무료 오픈소스.
2. 설치는 `npx skills add` 한 줄.
3. Claude Code·Cursor·Copilot 등 18개 이상 에이전트 지원.
4. 보안에 신경 쓸 것 — 공식 벤더 스킬 위주, 설치 전 코드 확인.

## 외부 보강

영상의 설치 명령·출시 시점·설치수 주장을 공식 출처로 교차 확인한다. (영상 내용과 분리)

- **공식 패키지·명령**: skills CLI는 `vercel-labs/skills`로 공개돼 있으며, 설치 명령은 `npx skills add <package>` 형태다. 스킬은 npm 패키지가 아니라 **git 저장소**에서 설치된다 (영상의 "골뱅이 + 이름" 발화는 이 패키지 지정 인자에 해당). — [vercel-labs/skills (GitHub)](https://github.com/vercel-labs/skills)
- **출시·정체**: Vercel이 "skills, the open agent skills ecosystem"을 공식 발표. skills.sh는 스킬 패키지의 디렉터리·리더보드 역할. — [Introducing skills (Vercel Changelog)](https://vercel.com/changelog/introducing-skills-the-open-agent-skills-ecosystem)
- **지원 에이전트**: amp, antigravity, claude-code, codex, cursor, gemini-cli, github-copilot, goose, opencode, roo, trae, windsurf 등 18종 이상 — 영상의 "18개 이상" 주장과 일치. — [Vercel Agent Skills 문서](https://vercel.com/docs/agent-resources/skills)
- **보안 측면**: Vercel이 Snyk와 함께 스킬 공급망 보안을 다루고 있다는 별도 발표가 있다 — 영상의 "악성 명령어 발견" 우려와 맥락이 닿는다. — [Securing the Agent Skill Ecosystem (Snyk)](https://snyk.io/blog/snyk-vercel-securing-agent-skill-ecosystem/)
- **find-skills 스킬**: 영상이 "무조건 깔라"고 한 1번 스킬의 실제 소스. — [find-skills SKILL.md](https://github.com/vercel-labs/skills/blob/main/skills/find-skills/SKILL.md)

> 보강 주석: 영상은 "버셀이 올해 1월에 공식 출시" [00:51] 및 "최상위 스킬 19만 건 이상 설치" [00:56]를 언급했으나, 정확한 출시 날짜와 19만이라는 특정 설치수는 공식 출처에서 시점·수치가 변동하므로 영상 발화로만 기록하고 단정하지 않는다. `[불명확: 19만 설치 수치 — 변동 가능, 영상 발화 기준]`.
