## 요약 (주제별)

### 시리즈 콘셉트 (1인기업 도전기 EP1)
- 화자는 이전 영상에서 "평범한 사람이 AI로 무장하면 혼자서도 회사를 만들 수 있다"고 약속했고, 말로만 하면 "강의팔이"와 다를 게 없으니 직접 AI만으로 기업을 만드는 과정을 이번 화부터 시작한다고 선언 [00:00].
- 세 가지 제약 조건: (1) 기획·개발·디자인 등 **모든 작업을 AI 도구로만**, (2) **컴퓨터 한 대만** 사용, (3) 서비스 오픈까지 **전 과정을 공유** [00:25].
- [의견] 완성된 성공담이 아니라 "맨바닥에서 삽 뜨는 과정"을 함께 보는 것이 콘텐츠의 진짜 재미라는 입장. 잘 될지는 화자 본인도 모른다고 밝힘 [00:40].
- 화자 주장(미검증): 자신의 "10년간 사업 노하우"로 AI를 잘 다뤄 멋진 서비스를 만들겠다 [00:56].

### 방법론 — "고통 지점(pain point)" 찾기
- 사업의 핵심은 "10에 가까운 고통을 명확히 해결해 주는 것"이라는 이전 영상의 프레임을 재사용. 고통 지수(1~10) 개념은 우측 상단 카드로 다시 볼 수 있다고 안내 [01:04].
- 고통을 찾는 가장 좋은 방법은 **리뷰**를 보는 것 — 사람들이 불편을 리뷰로 남기므로, 팔려는 상품과 비슷한 상품의 리뷰가 고통 지점을 가장 빨리 찾는 길이라는 주장 [01:32].
- 문제: AI 서비스는 리뷰를 찾기 어려워, 대신 **AI를 다루는 유튜브 영상들의 댓글**을 분석하기로 함. 다만 AI 분야는 변화가 빨라 수많은 영상 댓글을 일일이 찾는 게 막막했다고 함 [01:45].

### 도구 — 유튜브 댓글 자동 수집·고통 분석기
- 화자는 이 막막함을 해결할 **도구를 AI로 직접 제작**. 기능: AI 관련 영상 댓글을 자동 수집 → 고통 느끼는 지점 탐지 → 고통 지수 1~10으로 분류 → 그중 6~10 고통을 분석해 해결책을 제안 [02:07].
- 이 도구는 영상 말미에서 사용법과 함께 무료로 공유한다고 안내 (세일즈/유입 장치) [02:27].
- 화자 주장(미검증): 이 도구로 "수천 개 댓글"을 분석해 결과를 얻었다 [02:34].

### 분석 결과 — 사람들이 AI에서 느끼는 고통 (화자 주장)
- 많은 사람이 "AI를 어떻게 사용하는지" 막막해함, 유용한 AI가 매번 뒤바뀌어 어려움, 남들보다 뒤처지는 것에 대한 막막함 [02:40].
- 세부 고통: 어떤 프롬프트를 써야 할지·어떤 AI 도구를 어떻게 조합할지 몰라 헤맴, 똑같이 따라 해도 자기 결과물만 형편없이 나오고 그동안 토큰·시간만 소모됨, "남들은 잘하는데 나는 왜 안 되지" 하는 질문 반복 [02:55].
- 화자 판단: 이 답답함이 납득할 만한 6~10 사이의 꽤 큰 고통이라고 평가 [03:14].

### 타깃 그룹 도출 + AI 교차검증
- 수익화로 이어지려면 6~7 수준이 아니라 **고통을 10으로 가장 강하게 느끼는 그룹**을 찾아야 한다는 논리 [03:21].
- 이 질문을 그대로 클로드(Claude)에 물어 4개 정도의 그룹을 제안받음 — 생각지 못한 그룹도, 공감되는 그룹도 있어 설득력을 느꼈다고 함 [03:37].
- 검증을 위해 ChatGPT("채피티")·Gemini("재미나이")에도 동일 질문 → **겹치는 그룹**이 있었음. 이어 세 AI에 해결책도 동일하게 물어 비슷한 답을 받음 [03:49].
- [의견] AI는 그럴싸한 기획안을 내주지만 상상력 부분은 아직 사람을 못 따라오므로, 해결책을 AI가 명확히 짚어주진 못한다 [04:07].

### 결정된 서비스 — "AI 레시피" 공유·판매 사이트
- 통계와 예시 해결책에 자기 생각을 더해 결정한 서비스: **AI 레시피를 공유하고 판매하는 사이트** ("딱 한 줄 설명") [04:23].
- 단순 프롬프트 공유가 아니라, 여러 프롬프트를 사용해 만들어 낸 **결과물에 대한 레시피**를 공유·판매하는 것. 이 기획은 제작 과정에서 보완·수정될 수 있다고 단서 [04:35].
- 이 내용을 기반으로 클로드 AI에 사업 계획서와 사업 일정을 요청해 받았다고 함 [04:47].

### 실전 절차 — 도구 사용법 (댓글 수집 + Google API + 클로드 분석)
- 댓글에 공유한 링크 클릭 → "댓글 수집" 파일의 예시 보기 클릭 → 댓글 수집 페이지 표시. 수집엔 **구글 API 키**가 필요 [05:15].
- 구글 클라우드 콘솔 접속 → 구글 아이디 로그인 → "API 및 서비스" → 좌측 "라이브러리" → 하단 **YouTube Data API** 카드 선택(기존 사용자는 "관리", 신규는 "사용") → "사용자 인증 정보" → "사용자 인증 정보 만들기 → API 키" → API 제한 사항에서 맨 하단 **YouTube Data API만** 체크 → 확인 → 만들기 → 발급된 키 복사 [05:27].
- 발급 키를 도구에 입력 + 사업 주제 키워드 입력 → 댓글 수집됨 → "모두 복사하기" [06:26].
- 두 번째 파일을 클로드 AI에 업로드 → 프롬프트 기재 → 미리보기 창에 복사한 댓글 붙여넣기 → "분석하기" → 고통 지수별 분류 + 클로드 해설 산출. 시청자도 이 흐름으로 사업 계획서까지 따라 만들라고 권유 [06:39].

### 다음 단계 — 디자인 & 웹사이트 (Figma Make)
- 다음은 디자인 단계. 시청자 다수가 디자인 비전공자일 수 있으므로, 화자의 **업무 매뉴얼을 디자인 AI에 건네 디자인을 통째로 맡기는 방법**을 알려줄 예정 [07:05].
- 색·폰트·버튼 느낌을 고민할 필요 없이 AI가 트렌드에 맞게 디자인해 주며, 마음에 안 들면 참고 스타일을 넣어주면 된다고 함 [07:18].
- [의견/근거 주장] AI의 강점은 학습 데이터에서 나오는데, 디자인에서 방대한 데이터를 가진 곳이 **피그마(Figma)**이므로 **피그마 메이커(Figma Make)** AI로 초기 사이트를 만들 것 [07:40].
- 다음 영상에서 피그마 메이커로 웹사이트 밑그림을 함께 그리는 작업을 진행하겠다고 예고 [08:04].

### 세일즈·유입 구조
- 무료 도구(댓글 수집기 + 고통 분석기)를 댓글 링크로 배포 → 시청자가 자기 분야에 직접 적용하도록 유도 [02:27] [05:11].
- 시리즈 연속 시청 유도: 구독·좋아요·알림 설정을 누르면 다음 편을 가장 빠르게 볼 수 있다는 표준 CTA [08:29].

## 전문 — 시간순 워크스루

### [00:00–01:02] 이전 약속과 세 가지 제약 조건
화자는 이전 영상에서 "평범한 사람이 AI로 무장하면 혼자서도 회사를 만들 수 있다"고 약속했음을 상기시키며, 말로만 하면 "강의팔이"와 다를 게 없으니 오늘부터 직접 AI만으로 기업을 만들어 보겠다고 선언한다. 지금 당장은 아무 계획이 없고, 모든 건 AI에게 물어 사람들이 진짜 답답해하는 고통 하나를 골라 해결하는 서비스를 만들겠다고 한다. 진행 조건은 세 가지 — 기획·개발·디자인 등 모든 작업을 AI 도구만으로, 컴퓨터 한 대만 사용, 서비스 오픈까지 전 과정 공유.
> "평범한 사람이 AI로 무장해 혼자서도 회사를 만들 수 있다고요. 그런데 말로만 하면 그건 그냥 강의팔이나 다를게 없겠죠." [00:04]

### [00:40–01:02] 콘텐츠의 취지 (완성담이 아닌 과정 공유)
화자는 자기도 떨리고 잘 될지 안 될지 모른다고 인정하면서, 완성된 성공담이 아니라 맨바닥에서 삽 뜨는 과정을 시청자와 함께 공유하는 것이 콘텐츠의 진짜 재미라고 말한다([의견]). 다만 자신의 "10년간 사업 노하우"로 AI를 최대한 잘 다뤄 진짜 멋진 서비스를 만들겠다고 덧붙인다(화자 주장/미검증).

### [01:04–01:45] 1단계: 사람들의 "고통 지점" 찾기 — 리뷰 활용
계획의 첫 번째는 사람들의 고통 지점을 찾는 것. 지난 시간에 "사업은 10에 가까운 고통을 명확하게 해결해 주는 것"이라고 말했음을 재확인하고, 고통 지수 관련 내용은 우측 상단 카드로 다시 볼 수 있다고 안내한다. 자신은 AI 관련 서비스를 만들 것이므로 사람들이 AI를 쓰며 어디서 제일 답답해하는지를 직접 찾아봤다고 한다. 고통을 찾는 가장 좋은 방법은 리뷰 — 사람들이 불편을 리뷰로 남기기에, 상품을 팔려면 비슷한 상품의 리뷰를 보는 게 고통 지점을 찾는 가장 빠른 방법이라는 것.

### [01:45–02:34] 도구 제작: 유튜브 댓글 자동 수집·고통 분석기
그런데 AI 서비스는 리뷰를 찾기가 쉽지 않아, 대신 AI를 다루는 유튜브 영상들의 댓글을 분석하기로 한다. 다만 AI는 매번 새로운 게 나오고 빠르게 변해 수많은 영상 댓글을 하나하나 찾아 분석하기가 막막했고, 그래서 이를 해결할 도구를 하나 만들기로 했다. 도구는 AI 관련 영상 댓글을 자동 수집하고, 고통을 느끼는 지점을 찾아 고통 지수 1~10으로 분류하며, 그중 6~10 정도 고통을 분석해 필요한 해결책을 제안한다. 이 도구 역시 AI로 만들었고, 모두가 쓸 수 있게 영상 말미에 사용법과 함께 공유하겠다고 한다(무료 배포/유입 장치).

### [02:34–03:19] 분석 결과: 사람들이 AI에서 느끼는 고통 (화자 주장/미검증)
이 도구로 수천 개 댓글을 분석했더니, 많은 사람이 AI 사용법에 막막함을 느끼고, 유용한 AI가 매번 엎치락뒤치락 바뀌는 어려움을 겪으며, 남들보다 뒤처지는 것에 대한 막막함을 느낀다는 분석이 나왔다고 한다. 구체적으로는 어떤 프롬프트를 써야 할지, 어떤 AI 도구를 어떻게 조합할지 몰라 헤매고, 똑같이 따라 했는데 자기 결과물만 형편없이 나오며 그동안 토큰과 시간만 줄줄 새어 나간다. "남들은 잘 만드는데 나는 왜 안 되지"라는 질문을 계속 던진다는 것.
> "똑같이 따라했는데 자기 결과물만 형편없이 나오고 그러는 동안 토큰과 시간만 줄줄 세나갑니다." [03:04]

화자는 이 답답함이 자기가 보기에도 납득할 만한 6~10 사이의 꽤 큰 고통이라고 평가한다.

### [03:19–04:07] 타깃 그룹 도출과 세 AI 교차검증
이제 이 고통을 10 정도로 강하게 느끼는 그룹이 누구인지 찾는 것이 중요하다고 한다 — 그래야 수익화로 연결되므로 6~7 그룹이 아니라 10으로 가장 힘들게 느끼는 그룹을 찾아야 한다는 논리. 이 질문을 그대로 클로드 AI에 물었더니 네 개 정도의 그룹을 제안했고, 생각지도 못한 그룹과 공감되는 그룹이 섞여 설득력을 느꼈다고 한다. 하지만 검증이 필요해 ChatGPT·Gemini에도 동일 질문을 했더니 겹치는 그룹들이 있었다. 그룹을 찾았으니 다음은 해결책 — 역시 세 AI에 동일하게 물어 비슷한 답변을 받았다.
> "AI는 그럴사한 기회관을 내주지만 상상력의 부분은 아직 사람을 따라오기 힘드니까요." [04:12]

여기서 화자는 해결책을 AI가 명확히 짚어줄 수는 없다고 선을 긋는다([의견]).

### [04:07–05:00] 결정된 서비스와 사업 계획서
통계와 예시 해결책에 자기 생각을 더해, 명확한 해결책을 제시하는 서비스를 만들기로 결정한다 — 바로 "AI 레시피를 공유하고 판매하는 사이트"가 서비스의 한 줄 설명이다. 이 기획은 추후 제작 과정에서 보완·수정될 수 있다고 단서를 단다. 단순히 프롬프트를 공유하는 것이 아니라, 여러 프롬프트를 사용해 만들어 낸 결과물에 대한 레시피를 공유·판매하는 것이라고 설명한다. 이 내용을 기반으로 클로드 AI에 간단한 사업 계획서와 사업 일정을 요청해 "멋진 사업 계획서"를 받았다고 한다.
> "바로 AI 레시피를 공유하고 판매하는 사이트입니다. 이게 제 서비스에 딱 한 줄 설명입니다." [04:25]

### [05:00–06:26] 실전 절차 ①: 도구 공유와 Google YouTube Data API 키 발급
이제 레시피를 담을 웹사이트를 만들어야 하는데(디자인·화면·결제 붙이기), 그 전에 시청자도 자기 서비스에 맞는 고통 지수와 타깃 그룹을 찾아야 하니 자신이 쓴 도구와 사용법을 공유한다. 먼저 댓글에 공유한 링크를 클릭해 "댓글 수집" 파일의 예시 보기를 클릭하면 댓글 수집 페이지가 뜬다. 수집을 위해선 구글 API가 필요하다. 절차: 웹에서 "구글 클라우드 콘솔" 검색·접속 → 구글 아이디 로그인 → "API 및 서비스" 클릭 → 좌측 메뉴 "라이브러리" → 하단 "YouTube 데이터 API" 카드 클릭(화자는 이미 사용 중이라 "관리"로 보이지만 시청자는 "사용" 버튼) → "사용자 인증 정보" → 상단 "사용자 인증 정보 만들기 → API 키" → 다른 건 두고 "API 제한 사항" 맨 하단의 "YouTube 데이터 API"만 체크 → 확인 → 만들기 → 발급된 API 키 복사.

### [06:26–07:05] 실전 절차 ②: 댓글 수집 → 클로드 분석
발급받은 API를 도구에 넣고, 진행하려는 사업과 관련된 주제의 키워드를 입력하면 댓글이 수집된다. 이 댓글을 "모두 복사하기"로 복사한 뒤, 두 번째 파일을 클로드 AI에 올리고 프롬프트를 기재한다. 그러면 미리보기 화면에 창이 뜨는데, 복사한 댓글을 모두 붙여넣고 "분석하기"를 누르면 클로드 AI가 고통 지수별로 분류·분석하고 관련 해설을 전달해 준다. 이렇게 상세한 분석이 나오면 시청자도 이를 기반으로 사업 계획서까지 따라 만들어 보라고 권한다.

### [07:05–08:04] 다음 단계: 디자인과 Figma Make
사업 계획서를 만들었다면 다음은 디자인 단계. 이 영상을 보는 사람들이 디자인 비전공자일 수 있으므로, 디자인을 도와주는 AI에 자신의 업무 매뉴얼을 건네 디자인을 모두 맡기는 방법을 알려주겠다고 한다. 어떤 색·폰트를 쓰고 버튼을 어떤 느낌으로 할지 전혀 고민할 필요가 없으며, AI가 트렌드에 맞게 디자인해 주고, 마음에 안 들거나 원하는 스타일이 있으면 참고할 스타일을 넣어주기만 하면 된다는 것. AI마다 분야별 강점이 있고 그 강점은 사이트가 가진 학습 데이터에서 나오는데, 디자인에서 방대한 데이터를 가진 곳이 피그마이므로 디자인을 제일 잘 뽑아 줄 피그마 메이커라는 AI로 초기 사이트를 만들겠다고 한다([의견/근거 주장]).
