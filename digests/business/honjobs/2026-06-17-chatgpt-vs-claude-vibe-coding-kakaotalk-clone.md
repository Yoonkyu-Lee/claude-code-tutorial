# 챗GPT vs 클로드 ‘코딩없이’ 1시간만에 카카오톡 똑같이 만들기

> **검증 배너 (미검증 · 화자 주장)**
> 이 영상은 AI 바이브 코딩을 소개하는 홍보성/유입 성격의 콘텐츠다. "코딩 지식 없이 1시간 만에 카카오톡을 만들었다", "10분 만에 코딩이 완료됐다", "40분밖에 안 지났다" 같은 **소요 시간·완성도 주장은 모두 화자 본인의 시연 주장이며 제3자 검증이 없다.** 편집된 영상이라 실제 시행착오·재시도 횟수가 생략됐을 수 있다. 클로드 코드 vs 코덱스 우열 판단도 화자의 주관적 인상(`[의견]`)이다. 아래 요약·전문에서 사실(무엇을 했는가)과 의견(어느 쪽이 낫다)을 분리해 표기했다.

## 주제별 상세 요약

### 1. 이 영상이 하는 것 (실험 설계)
- 목표: **코딩 지식 없이**, 챗GPT(코덱스)와 클로드(클로드 코드) **두 도구로 각각 카카오톡 메신저 클론을 만들어** 비교한다. 각 도구에 **1시간 타이머**를 부여한다. (사실)
- 마지막에는 더 잘 만든 쪽을 골라, 실제로 대화가 오가는 메신저까지 완성한다. (사실, 화자 계획)

### 2. 공통 준비 — 프롬프트와 참고 이미지
- 클로드에게 "카카오톡을 만들 수 있는 프롬프트"를 먼저 **요청해서 프롬프트 자체를 생성**시킨다. (사실)
- 대기 중 카카오톡의 주요 화면을 **캡처해 보관**한다. 이 캡처 이미지를 나중에 코드 생성 시 함께 첨부한다. (사실)
- 생성된 프롬프트를 **코덱스(챗GPT 제공)와 클로드 코드(클로드 제공)에 각각 동일하게 투입**한다. → 프롬프트를 통제 변수로 두고 두 코딩 도구를 비교하는 구조.

### 3. 클로드 코드로 만든 결과
- 프롬프트 + 카카오톡 캡처 이미지를 클로드 코드에 넣어 요청. **약 10분 뒤 코딩 완료**(화자 주장). (미검증 수치)
- 화자는 클로드 코드를 **로컬 PC에 설치**해 사용 중이라, 우측에 미리보기 화면이 바로 송출됨. (사실, 환경)
- 초기 결과 평가 (사실 기반 관찰):
  - 겉보기: "정말 잘 만들어졌다". (의견)
  - 문제점: 프로필이 안 눌림, 파일 업로드·이모티콘 전송 등 미구현 기능이 아직 많음, 채팅이 **반복 전송되는 버그**. (사실 — 화자가 관찰한 오류)
  - 오류 수정을 요청하니 "빠르게 보완"됨. (사실/의견 혼재)

### 4. AI 친구와 대화 기능 붙이기 (Claude API)
- 단순 클론은 재미없으니 **AI 친구를 추가해 실제로 채팅**되도록 세팅. (사실)
- 친구 추가 시 **AI의 성향(성격)을 설정**해 실제 친구처럼 대화하도록 프롬프트 작성. (사실)
- AI 기능에는 API가 필요 → 절차 (사실, 재현 가능한 정보):
  1. 구글에 "클로드 API 키" 검색 → 클로드 API 독스 진입
  2. API 키 버튼 클릭 → 키 생성 버튼 → 이름 설정 → API 키 발급
  3. 발급된 키를 복사해, 생성된 메신저 안의 입력란에 붙여넣기
- API를 넣자 **로컬 PC에서 AI 친구와 바로 톡**이 가능해짐. (사실)
- 예시: "머스크" 캐릭터와 대화 → 설정한 까칠한 성격대로 응답. (사실, 시연)
- 여기까지 **약 40분 경과** 주장. (미검증 수치)

### 5. 코덱스(챗GPT)로 만든 결과
- 타이머를 다시 1시간으로, **같은 프롬프트 그대로** 코덱스에 투입. (사실, 통제된 비교)
- 관찰:
  - "체계적으로 잘 구현", 겉보기 훌륭, 기능도 대체로 구현됨. (의견)
  - 클로드는 이미지 구현이 약해 프로필을 안 만들었는데, **GPT는 이미지에 강해 프로필 사진을 자동으로 넣어줌**. (사실 — 화자 관찰)
  - 버튼이 안 눌리거나, 채팅 목록↔친구 목록 이동이 즉시 안 되는 **잔 오류가 다소 많았음**. (사실)

### 6. 비교 결론 (대부분 [의견])
- 전체 성능: **클로드 코드가 조금 더 나았다** — 단, "큰 차이는 없다". (의견)
- 소요 시간: 두 도구 비슷. (사실/관찰)
- 이미지: **GPT가 강함**. (사실 기반 의견)
- 소통·작업 스무스함: **클로드 코드가 개인적으로 더 좋음**. (의견, "개인적으로" 명시)
- 디테일: 클로드 코드가 더 좋음 → 이후 작업은 **클로드 코드 결과물로 진행**. (의견 → 결정)

### 7. 완성 단계 — 이모티콘 + 앱화
- **이모티콘 직접 제작 파이프라인** (여러 AI 도구 조합, 사실):
  1. 이미지 창작은 챗GPT가 가장 좋아 → 챗GPT에 귀여운 캐릭터 그리기 요청
  2. 그 캐릭터의 연속(응용) 동작은 **제미나이(Gemini)**에 요청
  3. **포토스케이프(PhotoScape)**로 움직이는 GIF 제작
- 클로드 코드에 "이모티콘 저장 기능 + 대화에서 사용하는 기능" 구현 요청 → 대화창에서 이모티콘 전송 가능해짐. (사실)
- **데스크톱 앱화**: 앱 이름을 "코코아톡(코코아)"으로 정하고, 클로드 코드에 "데스크탑에 앱처럼 만들어 클릭하면 실제 사용 가능하게" 요청 → 바탕화면에 앱 아이콘 생성, 클릭 시 바로 실행. (사실, 시연)
- 결론: 코딩 지식 없이 1시간 만에 시작해 나만의 메신저 "코코아톡" 완성. (화자 주장) 다음 영상도 바이브 코딩 예고 + 구독/좋아요/알림 유도. (사실 — 채널 홍보)

### 8. 실전에 옮길 때 참고 (노트 관점, 검증된 사실만)
- **도구 역할 분담 패턴**: 코드 생성(클로드 코드/코덱스) · 이미지 창작(챗GPT) · 연속 동작(제미나이) · GIF 편집(포토스케이프) · 대화 지능(Claude API). 하나로 다 하지 않고 강점별로 나눔.
- **재현 가능한 절차**로 유효한 것: (a) 참고 앱 화면 캡처를 이미지로 첨부, (b) 클로드에 프롬프트 자체를 생성시켜 재사용, (c) Claude API 키 발급·삽입 흐름.
- **주의**: 영상은 편집본이며 "1시간/10분/40분" 시간 주장과 완성도는 미검증. 실제로는 반복 오류 수정·재프롬프트가 더 필요할 수 있다(영상에서도 반복 전송 버그·미눌림 버튼 등 오류가 나왔음).
