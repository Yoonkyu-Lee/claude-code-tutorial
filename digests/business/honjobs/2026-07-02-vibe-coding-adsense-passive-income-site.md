## 요약 (주제별)

### 1. 핵심 주장 — "방치형 자동 수익 사이트"
- 화자는 바이브 코딩으로 만든 사이트 하나로 **매달 평균 10만원 이상**을 벌었다고 주장(미검증). 10만원은 "한 달 교통비 정도"로 큰 돈은 아니라고 스스로 인정.
- [의견/주장] 핵심은 "세팅 한 번만 해두면 관리 없이도 돈이 들어온다"는 것. 이런 사이트를 **10개** 만들면 손대지 않고 굴러가는 수익 구조가 된다고 주장(미검증).
- 실제 수익 증빙(정산 화면·기간·트래픽)은 영상에 없음.

### 2. 서비스 구조 비유 (식당)
- 인터넷 서비스를 식당에 비유: **도메인=가게 자리**, **프론트엔드(화면)=인테리어**, **서버=창고(재료 보관)**, **백엔드=요리사(버튼·로그인 처리)**. 이 네 가지가 인터넷 서비스의 핵심.

### 3. 무료 도구 스택
- 원래 네 요소 모두 비용이 들지만, 화자는 전부 **무료**로 구성한다고 제시:
  - **서버(창고) = Supabase(슈퍼베이스)**
  - **프론트엔드+백엔드(인테리어+요리사) = Claude Code(클로드 코드)**
  - **운영/배포 = Vercel(버셀)**
- 세팅 팁: Claude Code 데스크톱 버전 다운로드 → 실행 → 코드 영역에 `claude.md`(CLAUDE.md) 파일을 넣으면 클로드가 더 똑똑해짐. 화자가 댓글에 올린 파일을 받아 넣으면 세팅 끝이라고 안내.

### 4. 무엇을 만들지 — 아이템 선정 원칙
- [의견] "**지금 내가 불편한 걸 만든다**"가 핵심 원칙. 내가 불편하면 똑같이 불편한 사람이 반드시 존재. 처음부터 대단한 걸 만들지 말고 작게 시작해 발전시켜라.
- 화자 사례: 혼자 일하며 **영수증 증빙 정리**가 가장 골치였는데, 검색해도 회계 프로그램은 기능이 많고 무거웠음. 영수증 증빙 처리만 깔끔히 되는 도구가 없어서 직접 제작.

### 5. 프롬프트 레시피 도구
- 대부분 "프롬프트를 어떻게 쓸지, 디자인을 어떻게 할지" 막히는데, 화자는 **프롬프트 레시피 생성 사이트**를 사용. 도메인은 자막상 `bbggggl.com` [불명확: 자동자막 음차라 실제 도메인 확인 불가].
- "말로 설명하기"에서 필요 기능을 나열: 영수증 이미지 업로드 / 업로드한 영수증이 A4에 정렬 / 금액 자동 합산 / 정리된 A4 자료 다운로드. 확인하면 기본 뼈대가 잡히고, 디자인 느낌·버튼·폰트·추가 기능(유지보수 최소화를 위해 검색 기능 정도만)을 고른 뒤 레시피 생성.
- [주장] 이 레시피는 기획을 반영하고 기본 뼈대까지 포함해 **토큰을 아끼고 시행착오를 줄여준다**.

### 6. 제작 실전 절차
- 생성된 레시피를 Claude Code에 그대로 붙여넣고 **10~15분** 대기하면 사이트가 나옴. 기능 확인 후 부족한 부분은 사람에게 말하듯 요청하며 다듬기.
- 중요: 클로드가 아무리 잘 만들어도 **직접 눌러보고 써봐야** 한다.

### 7. 애드센스(광고 수익) 준비
- 광고를 붙여 수익화하므로 세 가지 필수: ① 기능이 잘 작동 ② 빈 페이지가 없어야 ③ 필수 정보가 담겨야. 애드센스 검수는 사람이 하므로 전부 AI에 맡기지 말고 최소한 직접 검수.
- Claude Code에 "구글 애드센스 심사 준비가 됐는지 검토하고, 사이트맵을 추가해 SEO도 최적화해줘"라고 요청.
- 애드센스 통과 꿀팁: 유용한 기능 / 사용법·FAQ·팁 등 안내서 / 개인정보 처리방침 같은 필수 정보 / 사이트맵이 있으면 더 좋음.

### 8. 홍보·유입 (SEO·커뮤니티·SNS·백링크)
- 애드센스 가입만으론 아무도 사이트를 모름. SEO를 해도 저절로 검색에 반영되지 않음.
- 필수: **구글 서치 콘솔 + 네이버 서치 어드바이저에 사이트 등록**.
- 기능이 필요한 사람이 모인 커뮤니티(예: 세무 처리하는 직장인·개인사업자)에 알리되, 대놓고 광고하면 커뮤니티 정지 위험 → 실제 사용 경험 풀듯 조심스럽게.
- **백링크**: 링크가 여기저기 퍼질수록 구글이 사이트를 신뢰해 노출↑. 결국 "기능을 계속 좋게 만드는 게 최고의 홍보".
- [의견] 가장 효율적인 건 **SNS**. 타깃이 좋아할 콘텐츠를 계속 올림. 콘텐츠당 100명만 봐도 1~2명 유입되면 수익으로 이어질 수 있다(주장).

### 9. 마무리 4단계 요약 + 세일즈 훅
- 전체를 4단계로 정리: ① 내가 불편한 것 찾기 ② 해결 기능을 레시피로 만들어 Claude Code에 넘기기 ③ 광고를 붙여 수익 세팅 ④ 사람들에게 알리기.
- [의견] 돈이 안 돼도 "내가 불편해서 만든 거니 나라도 쓰면 되지"라는 가벼운 마음으로 도전 권유.
- **세일즈 훅**: 화자는 이런 사이트를 여러 개 만들다가 지금은 **1인 기업 만드는 프로젝트**를 진행 중이며, "제대로 된 회사를 만들어 보고 싶다면 내 1인 기업 프로젝트 영상도 보라"고 유도. 구독·좋아요 요청.

## 3줄 오리엔테이션
- 코딩 몰라도 Claude Code + 무료 도구(Supabase·Vercel)로 "내가 불편한 문제"를 푸는 작은 사이트를 만들어 애드센스 광고를 붙이는 전 과정을 보여준다.
- 화자는 영수증 증빙 정리 사이트를 예시로, 아이템 선정 → 프롬프트 레시피 → 제작 → 애드센스 준비 → SEO·커뮤니티·SNS 홍보까지 순서대로 시연한다.
- 결론은 "이런 걸 여러 개 만들어 자동 수익을 쌓아라"이며, 끝에 자신의 1인 기업 프로젝트 영상으로 유도하는 홍보성 콘텐츠다.

## 전문 — 시간순 워크스루

### [00:00–00:38] 훅 — 사이트 하나로 매달 10만원 (주장)
바이브 코딩으로 돈을 벌 수 있냐고 물으며, 화자는 바이브 코딩으로 만든 사이트 하나로 **매달 평균 10만원 이상**을 벌었다고 주장(미검증)한다. 10만원이면 한 달 교통비 정도라 큰 돈이 아닐 수 있지만, 이런 사이트가 여러 개면 얘기가 달라진다고 한다.
> "중요한 점은 이 돈이 잠을 잘 때에도 들어온다는 점인데요." [00:17]

[의견/주장] 처음 세팅 한 번만 잘해두면 그 뒤로는 관리를 안 해도 돈이 되는 방법이며, 이런 사이트를 **10개** 만들면 손대지 않고 굴러가는 수익 구조가 만들어진다(미검증). 오늘은 이 사이트를 어떻게 만들었고 어떻게 돈이 들어왔는지 전 과정을 처음부터 끝까지 보여주겠다고 예고.

### [00:38–00:55] 약속 — 하루 만에, 코딩 없이
끝까지 보면 시청자도 **오늘 단 하루 만에** 돈이 될 사이트 하나를 직접 만들 수 있다고 한다. 코딩을 한 줄 몰라도 되니 따라 해보라고 권유. 본격 시작 전에 구도가 어떻게 돌아가는지 먼저 설명하겠다고 함.

### [00:55–01:41] 식당 비유로 서비스 구조 설명
인터넷 서비스를 식당 차리기에 비유한다. 가게 자리를 구하고(=**도메인**, 손님이 이 주소로 찾아옴), 인테리어를 하고(=화면, **프론트엔드**), 손님이 버튼을 누르거나 로그인하면 처리하는 요리사가 있고(=**백엔드**), 요리사가 재료를 꺼내오는 창고(=**서버**)가 있다. 정리하면 도메인·프론트엔드·서버·백엔드 네 가지가 인터넷 서비스가 돌아가는 핵심이라고 함.

### [01:41–02:09] 네 요소를 전부 무료로 — 도구 스택
식당 차리는 데 돈이 들듯 이 네 가지도 원래 전부 비용이 들지만, 오늘 만들 건 "방치해도 돈 버는 사이트"니 전부 무료로 하는 방법을 알려주겠다고 한다. **창고(서버)는 Supabase(슈퍼베이스)**, **인테리어·요리사(프론트+백엔드)는 Claude Code(클로드 코드)**, **가게 운영(배포)은 Vercel(버셀)**이 맡는다. 이 셋으로 0원 제작을 한다고 함. 지금은 "이런 것들이 있구나" 정도만 알면 된다고 안내.

### [02:09–02:15] 실제로는 클로드 코드가 시키는 대로만
실제 제작 때는 Claude Code가 시키는 대로만 따라 하면 되니 부담 갖지 말라고 안심시킴.

### [02:15–02:56] 무엇을 만들지 — "내가 불편한 것"
어떤 가게를 차릴지 정해야 하는데, 화자는 혼자 일하다 보니 잡다한 업무를 다 처리했고 그중 **영수증 정리**가 제일 골치였다고 한다. 검색해도 죄다 회계 프로그램이라 기능이 많고 무거웠는데, 자신은 그냥 영수증 증빙 처리 하나만 깔끔하게 되길 원했지만 그런 사이트를 찾기 어려워 직접 만들기로 했다.
> "지금 내가 불편한 걸 만든다. 내가 불편하면 나랑 똑같이 불편한 사람이 분명히 존재합니다." [02:44]

[의견] 처음부터 대단한 걸 만들려 하지 말고 작은 것부터 시작해 점점 좋은 서비스로 발전시키라고 조언.

### [02:56–03:25] Claude Code 설치·세팅
제작 과정을 그대로 시연한다. 구글에 "클로드 코드"를 검색해 데스크톱용 버전을 다운로드·실행. 창이 뜨면 코드 부분을 클릭하고 거기에 `claude.md`(CLAUDE.md) 파일을 하나 넣는데, 이 파일이 클로드를 훨씬 똑똑하게 만들어준다고 함. 화자가 댓글에 올려둔 파일을 받아 넣으면 Claude Code 세팅은 끝이라고 안내(세일즈/유입 장치: 댓글 배포 파일).

### [03:25–03:53] 프롬프트 레시피 생성 사이트
이제 만들고 싶은 사이트를 프롬프트로 설명하면 되는데, 대부분 여기서 막힌다고 한다 — 프롬프트를 어떻게 쓸지도 막막하고 디자인이 머릿속에 없어 AI가 어떤 형식으로 만들지 감이 안 오기 때문. 그래서 화자는 **프롬프트 레시피를 짜주는 사이트**를 쓴다. 구글에 `bbggggl.com`을 검색해 가입하면 레시피를 짜주는 사이트가 나온다고 함. [불명확: 도메인 `bbggggl.com`은 자동자막 음차로 보이며 실제 주소 확인 불가]

### [03:53–04:16] 필요 기능 나열
사이트에서 "말로 설명하기"를 누르고 내 사이트를 말하듯 설명하며 필요한 기능을 나열한다. 화자는 ① 영수증 이미지를 업로드하는 기능 ② 업로드한 영수증들이 A4 용지에 정렬되는 기능 ③ 금액을 자동 합산하는 기능 ④ 정리된 A4 자료를 다운로드하는 기능이 필요하다고 기재했다.

### [04:16–04:34] 뼈대·디자인 선택
확인을 누르면 이게 기본 뼈대로 잡히고, 그 아래에서 디자인 느낌·버튼·폰트까지 고를 수 있다. 기능도 더 넣을 수 있지만, 유지보수가 거의 안 드는 사이트를 만들어야 하니 **검색 기능 정도만** 넣으라고 권함. 다 골랐으면 "레시피 만들기" 버튼을 누른다.

### [04:34–04:50] 레시피를 Claude Code에 투입
[주장] 이렇게 나온 레시피는 기획을 바로 반영할 뿐 아니라 기본 뼈대까지 들어 있어 토큰도 아끼고 시행착오도 확 줄여준다. 이 레시피를 앞서 설치한 Claude Code에 그대로 붙여넣고 **10~15분**만 기다리면 원하던 사이트가 나온다고 함.

### [04:50–05:07] 다듬기 + 직접 써보기
기능이 잘 도는지 확인하고 부족한 부분은 "이건 이렇게 바꿔줘" 하며 사람에게 말하듯 요청해 다듬는다. 여기서 중요한 점 하나 — 클로드가 아무리 잘 만들어도 결국 **직접 눌러보고 써봐야** 한다고 강조.

### [05:07–05:32] 애드센스용 세 가지 체크
이 사이트에 광고를 붙여 돈을 벌 것이라 세 가지를 꼭 챙겨야 한다: ① 기능이 당연히 잘 작동해야 하고 ② 비어 있는 페이지가 있으면 안 되며 ③ 필수적인 정보가 담겨 있어야 한다. 애드센스를 신청할 건데 애드센스 검수도 결국 사람이 하므로, 모든 걸 AI에 맡기지 말고 최소한 직접 검수해야 한다고 함.

### [05:32–05:52] 애드센스 심사 준비 프롬프트
어느 정도 만족스럽게 다듬어졌으면 Claude Code에 이렇게 요청한다.
> "구글 에드센스 심사 준비가 잘 됐는지 검토를 해주고, 사이트 맵을 추가해서 SEO도 최적화하게 해줘" [05:38]

그러면 클로드 코드가 제출할 수 있게 사이트를 마지막으로 점검해준다.

### [05:52–06:14] 애드센스 통과 꿀팁
검수 통과 꿀팁 세 가지: 첫째, 앞서 말한 대로 **유용한 기능**이 있어야 한다. 둘째, 사용 방법·자주 묻는 질문·관련 팁처럼 사용자가 웹사이트를 쓰기 위한 **안내서**가 잘 갖춰져 있으면 좋다. 셋째, **개인정보 처리방침** 같은 필수 정보가 잘 갖춰져 있어야 하고 사이트맵이 있으면 더 좋다.

### [06:14–06:41] 검색 등록 (구글·네이버)
검수가 잘되고 애드센스 가입이 완료됐다면 마지막으로 꼭 해야 할 게 있다. 애드센스에 가입됐어도 아무도 내 사이트를 모르고, SEO 최적화를 했어도 저절로 검색에 반영되지 않는다. 더 빨리 알리는 방법: 먼저 **구글 서치 콘솔과 네이버 서치 어드바이저에 내 사이트를 등록**한다 — 이건 필수이며, 그래야 검색에 잘 노출된다.

### [06:41–07:05] 커뮤니티 홍보 (조심스럽게)
그다음은 이 기능이 필요한 사람들이 모인 곳에 알리는 것. 화자는 세무 처리를 하는 직장인이나 개인사업자들이 모인 커뮤니티에 올릴 거라고 함. 단 대놓고 광고하면 커뮤니티 정지를 당할 수 있으니, 실제로 써본 경험을 풀듯 조심스럽게 알려야 한다. 가장 이상적인 건 기능이 좋아서 사람들이 알아서 링크를 퍼뜨려 주는 것.

### [07:05–07:19] 백링크 원리
내 링크가 여기저기 사이트에 퍼지는 것을 **백링크**라 하는데, 이게 쌓일수록 구글이 사이트를 신뢰해 검색에 더 잘 노출시켜 준다. [의견] 그러니 결국 기능을 계속 좋게 만드는 게 최고의 홍보라고 함.

### [07:19–07:41] SNS가 가장 효율적
[의견] 가장 효율적인 것은 SNS. 내 타깃이 좋아할 콘텐츠를 계속 SNS에 올리는 것이다. 많이 볼 필요도 없이 콘텐츠 하나당 100명만 봐도 그중 한두 명만 들어와도 수익으로 이어질 수 있다(주장). 초반 노력으로 검색 노출이 시작되면 가만히 놔둬도 자동 수익이 만들어지니 초반에는 이 부분들을 꼭 시도해볼 필요가 있다고 함.

### [07:41–08:04] 전체 4단계 정리
오늘 진행한 것을 네 가지로 정리: ① 내가 불편한 것을 찾고 ② 그걸 해결할 기능을 레시피로 만들어 Claude Code에 넘겨주고 ③ 거기에 광고를 붙여 수익을 만들 세팅을 하고 ④ 마지막으로 그걸 사람들에게 알리는 것. 이게 전부라고 함. [의견] 설령 돈이 안 되더라도 내가 불편해서 만든 거니 나라도 쓰면 되지 하는 가벼운 마음으로 도전하라고 권유.

### [08:04–08:35] 세일즈 훅 — 1인 기업 프로젝트
작은 사이트 하나로 수익이 나기 시작하면 그때부터 AI 다루는 게 정말 재미있어진다고 한다. 화자 본인도 이런 사이트를 여러 개 만들다가 지금은 한 발 더 나아가 **1인 기업을 만드는 프로젝트**를 진행 중이라고 밝힘. 아무리 대단한 서비스도 방법은 오늘과 완전히 동일하니, 오늘 작은 프로젝트를 실행해보고 제대로 된 회사를 만들고 싶어졌다면 자신의 1인 기업 프로젝트 영상도 시청하라고 유도. 도움이 됐다면 구독과 좋아요를 부탁하며 마무리.
