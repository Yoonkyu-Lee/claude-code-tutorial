# 애플 발표 보자마자 3000만원 지른 이유

> ColorScale · 2026-03-06 · 약 16분 · [영상 링크](https://www.youtube.com/watch?v=CwlrDb2SA-w)
> 정리 방식: 시간순 심층 워크스루 (paraphrase) + 주제별 상세 요약
> 검증 배너: 이 영상의 성능 수치·구조 분석 상당수는 **발표 전/직후 시점의 화자 예측·추정·"들리는 소문"**이며 실측이 아니다. 벤치마크 점수, 코어 구조 추정, 가격 정책 전망은 모두 (주장/미검증)으로 읽어야 한다.

## 요약 (주제별)

### 1. M5 Max CPU 코어 재편 — 개명과 신설 코어
- 구성 변화: M4 Max 최대 16코어(성능 12 + 효율 4) → **M5 Max 18코어(슈퍼 코어 6 + 성능 코어 12)**.
- **"슈퍼 코어"는 기존 성능 코어의 개명**이다. M4 성능 코어 대비 성능 차이는 10% 이내(주장)로, 개명할 만큼 드라마틱하진 않다[의견].
- 최상위 코어만 놓고 보면 개별 성능 +10%에 개수는 절반(12→6)이라 **묶음 성능은 오히려 하락**했다.
- 진짜 변수는 신설된 12개의 **성능 코어**. "들리는 소문"(미검증)으로 슈퍼 코어의 **약 70% 성능**. 효율 코어(슈퍼 코어의 절반 이하)와는 확실히 다른 등급이며, 절대 성능으로는 M2~M3 성능 코어 사이 수준으로 추정.
- 설계 의도(추정): ① 스위트 스팟 클럭대의 전성비 극대화 ② 코어당 트랜지스터 절감 → 전 코어 가동 시 전력 효율 + **같은 면적에 코어 수를 늘리기 유리**.

### 2. 성능 예측치 (모두 화자 추정, 미검증)
- Geekbench 6 싱글 **약 4,300점**, 멀티 **약 30,000점**.
- Cinebench 2026 멀티·Blender CPU에서 인텔 285K, AMD 9950X급 데스크톱 CPU와 대등할 것.
- 결론: 대부분의 프로그램에서 **노트북 CPU가 데스크톱 CPU를 절대 성능으로 앞선다**.
- GPU: 깡성능 **+20%**, 레이트레이싱 포함 **+30%**, 3D 렌더링은 M4 Max 대비 **+40%**(애플 발표 기반). 렌더링 기준 **랩탑용 RTX 5090과 비빌 수준**(추정).

### 3. 뉴럴 액셀러레이터와 통합 메모리(UMA)의 재평가
- 엔비디아 텐서 코어에 대응하는 **뉴럴 액셀러레이터가 GPU에 탑재**, AI 연산 M4 Max 대비 **4배**(주장).
- UMA의 가치: 렌더링 씬이나 LLM이 VRAM을 넘으면 GPU 성능이 무의미해지는데, UMA는 **VRAM 상당 용량을 매우 싸게** 확보. 다만 그동안은 **절대 연산 성능이 엔비디아 대비 너무 낮아** 장점을 못 살렸다.
- 실사용 근거: M3 Ultra에 **약 257GB LLM**을 올려 로컬 운용[불명확: 자막 "오픈 클로"] 중인데, 입력 토큰을 읽어 해석하는 단계(대역폭보다 **연산 성능**이 지배적)가 느려 **응답 시작 지연**이 답답하다.
- M5 Ultra 가정(추정): M3 Ultra 대비 연산 약 **5배** → 응답 시작 시간 1/5, 메모리 대역폭 1.5배 → 토큰 생성 1.5배. **소규모 회사 내부 사용에는 쓸 만한 수준**이 될 것[의견].

### 4. 퓨전 아키텍처 — M5 Pro/Max는 단일 칩이 아니다
- 구조: ① CPU + 시스템 요소 파트, ② GPU + 라스트 레벨 캐시 + 메모리 컨트롤러 파트를 **퓨전 아키텍처로 연결**. UltraFusion과 개념은 비슷하나 구현은 다른 차원(상세는 이전 영상).
- TSMC 패키징으로 **나뉜 두 칩이 하나처럼 동작** → 개별 칩 사이즈 축소 → **수율 상승**.
- 조합식 제품 구성: 같은 CPU 파트에 Max용 GPU를 붙이면 M5 Max, Pro용 GPU를 붙이면 M5 Pro. GPU는 Max 설계에서 아래 절반을 빼면 Pro가 되는 구조로 추정.

### 5. 수율 정치학 — Pro가 Max에 하극상할 수 없었던 이유 ("제피셜")
- M4 Max: 기본 10P+4E, 업그레이드 12P+4E / M4 Pro: 기본 8P+4E, 업그레이드 10P+4E.
- M4 Pro는 M4 Max 설계에서 GPU 절반을 날린 것으로 추정되므로 **양품이면 12P도 가능**했을 것. 그런데 10개로 캡이 씌워진 이유는 ① 거대 칩에서 필연적으로 나오는 **CPU 결함 다이를 살려야 했고** ② **Pro가 Max를 하극상하면 안 되기** 때문 — 화자 개인 추측("제피셜", 미검증).
- 분리 생산의 소비자 편익: CPU 파트 불량 자체가 급감 → **Max는 기본 구성부터 CPU 코어 풀 탑재**, **Pro도 업그레이드 시 Max와 동일한 CPU 구성** 가능(추정).
- 확장 여지: 단가 하락분으로 CPU/GPU 코어를 **1.5배**씩 늘릴 수도. 단일 칩이면 비용이 폭증하지만 두 칩 분할이면 적당한 상승으로 가능. 신설 성능 코어 특성과 결합하면 인텔 **Sierra Forest**나 **Zen 5c 기반 EPYC**처럼 코어 수를 크게 늘리는 길도 열린다(전망).

### 6. 발목을 잡는 것 — 5년 된 폼팩터의 열·전력 한계
- M5 Max는 위의 코어 증설 방식을 쓰지 **않았다**. 범인은 **M1 Pro/Max와 함께 나온 현행 MacBook Pro 폼팩터**(햇수로 5년째)[의견].
- 근거 수치: M1 Max 최대 부하 **약 100W** → M4 Max는 CPU+GPU 풀로드 시 **200W 초과**(주장). **MagSafe 140W 연결 상태에서도 배터리가 빠지고**, 팬 소음이 극심하며 칩 온도는 100도 근처.
- 즉 퓨전 아키텍처로 코어를 더 넣을 길이 열려도 **지금 폼팩터로는 추가 발열을 식힐 수 없다.**

### 7. M6 세대 전망
- **빠르면 올해 하반기, 늦어도 내년 상반기**에 완전히 새로운 폼팩터의 MacBook Pro(디자인·방열·전력 공급 개선) 등장 예상.
- 새 폼팩터 + 퓨전 아키텍처 + 신설 성능 코어 + 공정 개선[불명확: 자막 "이노 검정"]이 결합되는 **M6에서 또 한번 큰 폭의 성능 향상** 가능성.
- 루머: 터치스크린, 셀룰러. 확정적 변화로는 **탠덤 OLED 디스플레이** — 아이패드 프로 사례로 볼 때 휘도도 미니LED LCD에 밀리지 않아 **화질상 장점만 있는 업그레이드**[의견]. "화면 사이즈만 빼면 몇 천만 원짜리 레퍼런스 모니터 급"이 된다.

### 8. Studio Display XDR — 스펙과 "가성비" 논리
- Pro Display XDR 후속. **27인치 5K**(해상도 정체는 아쉬움[의견]), **로컬 디밍 존 576 → 2,304개**, **최대 휘도 2,000니트**, **썬더볼트 5 / 140W 충전 / 데이지 체인**.
- 색영역: P3 커버 → **Adobe RGB + P3**. 둘은 상하위 관계가 아니라 빨강은 P3, 초록은 Adobe RGB가 더 순수하므로 둘 다 덮으려면 더 넓어야 한다. 효과는 ① Adobe RGB까지 컬러 매니지먼트로 정확 표시 ② **Rec.2020 커버리지 약 72% → 약 80%**(화자 단순 계산, 미검증).
- 가격 **520만 원**, Pro Display XDR과 달리 **스탠드 포함**. 비교군은 ASUS ProArt 정도이고, 국내 동가격대보다 스펙 우위. 진짜 상위 제품(8K, 디밍 존 4,000개)은 **국내 미판매 + 2천만 원**이라 Studio Display XDR은 **스위트 스팟 제품**[의견].

### 9. 현업 HDR 워크플로 관점 (구매 정당화의 실무적 근거)
- 촬영 팀 및 영화·OTT 납품 컬러 그레이딩(DI) 업체와 협업 중인데, 이런 곳도 전원에게 수천만 원 레퍼런스 모니터를 줄 수 없어 **Pro Display XDR을 여러 대 운용**한다.
- 기존 화자 환경: Studio Display에 메인 작업 공간, 옆의 MacBook Pro/아이패드 프로에 **클린피드**를 띄워 컬러 확인.
- Pro Display XDR(디밍 존 576개)조차 **전체 화면 프리뷰는 괜찮지만 작업 창 안의 작은 HDR 프리뷰로 컬러를 만지기는 어렵다** → 결국 보조 화면 병용이 필요. 디밍 존 4배 증가는 **메인 작업 공간에서 바로 HDR 프리뷰**를 가능케 하는 변화[의견].

### 10. "3천만 원 지른" 이유 — 가격·공급망 판단
- 구매 내역: **MacBook Pro 옵션 2대 + Studio Display XDR 2대**.
- 이유 ①: 현 워크스테이션이 M3 Max 2대 + M4 Max 1대인데, M6를 기다렸다간 **세 대를 한꺼번에 교체**하게 되어 부담이 폭증. 게다가 3D 렌더링을 본격 시작해 당장 성능이 부족.
- 이유 ②: AI로 **램·낸드 가격 폭등**, 가려졌지만 **파운드리 비용도 급등**. 다만 애플은 낸드·D램 대량 소비자이자 **TSMC 2위 우수 고객**이라 싼 장기 계약 물량이 남아 있고 신규 협상도 상대적으로 유리(추정).
- 이유 ③: **아이폰 18 시리즈까지는 치킨 게임에 가까운, 애플답지 않은 가격 정책**이 유지되겠지만 그 이후 싼 물량이 소진되고, 경쟁사는 이미 가격을 올린 상태에서 OLED·새 폼팩터·M6가 겹치면 **가격 퀀텀 점프**(아이패드 프로 OLED 전례) 예상.
- 이유 ④: 디스플레이 같은 제품은 **라이프 사이클이 길고 시간이 지나도 가격이 잘 안 떨어져** 어차피 살 거면 지금이 가장 싸다[의견].
- 국내 1차 출시 대상이며, **다음 주 수요일 실물 수령 후 후속 영상** 예고.

## 3줄 오리엔테이션
- 애플 신제품 중 화자가 주목한 건 딱 둘: **M5 Max MacBook Pro**와 **Studio Display XDR**(Pro Display XDR 후속).
- M5 세대의 핵심은 ① 코어 이름 재편(슈퍼 코어 + 신설 성능 코어) ② 뉴럴 액셀러레이터로 AI 연산 4배 ③ CPU/GPU를 분리해 붙이는 **퓨전 아키텍처** — 셋이 합쳐져 수율·원가·확장성이 달라졌다는 분석.
- 그런데도 화자는 "지금 폼팩터가 한계"라 M6가 진짜배기일 거라 보면서도, **메모리·파운드리 가격 폭등 → M6 때 가격 퀀텀 점프**를 예상해 지금 질렀다고 설명한다.

## 전문 — 시간순 워크스루

### [00:00–00:52] 신제품 중 딱 두 개만 본다
애플이 신제품을 쏟아냈지만 화자는 MacBook Pro와 Studio Display XDR 두 가지만 다룬다. M5 MacBook Pro 나올 때 "큰 거 온다"고 예고했었는데 그게 왔다는 것.

- M4 Max도 **싱글코어 성능은 이미 AMD 9950X, 인텔 285K 같은 데스크톱 CPU보다 높았다**(주장). 다만 전 코어를 끝까지 짜내는 렌더링류에서는 체급 차이가 남아 있었는데, M5 Max는 그 시나리오에서도 최상위 데스크톱 CPU에 명함을 내밀 수준이 될 것(주장/미검증).
- GPU도 크게 좋아져 영상 편집·3D 렌더링에서는 **랩탑용 RTX 5090과 비벼볼 수준**일 것(주장/예측).
- Pro Display XDR 후속인 Studio Display XDR은 "가성비가 미쳤다"[의견]. 발표를 보고 MacBook Pro 옵션 2대 + Studio Display XDR 2대를 바로 구매했다고 밝힌다.

### [00:52–01:20] M5 Max CPU — 코어 이름이 바뀌었다
M4 Max가 최대 16코어였던 것과 달리 M5 Max는 18코어. 구성도 "성능 12 + 효율 4"에서 **"슈퍼 코어 6 + 성능 코어 12"**로 바뀌었다.

겉보기엔 고성능 코어가 추가된 것 같지만 실제로는 **기존에 성능 코어라 부르던 것이 슈퍼 코어로 개명**된 것이고, 새로 들어간 12개의 "성능 코어"가 지금까지 없던 신규 CPU 코어라는 게 핵심.

### [01:20–01:55] 개명할 만큼의 향상인가 — 오히려 최상위 묶음 성능은 감소
M4의 성능 코어 대비 M5 Max의 슈퍼 코어 성능 차이는 **10% 이내**(주장)로, 이름을 바꿔 부를 만큼 드라마틱하다고 보기는 어렵다[의견]. 절대 성능만 보면 이미 데스크톱 CPU 단일코어를 넘던 것에서 10% 더 올라간 것이니 대단하긴 하다.

> "기존에 12개 있던 코어의 개별 코어 성능을 10% 올리고 개수는 절반으로 줄였다. 최고 성능 코어들을 쫙 다 묶었을 때의 성능은 오히려 떨어진 거예요." [01:47]

### [01:55–02:30] 신설 성능 코어의 정체 — 슈퍼 코어의 70%
그래서 새로 추가된 12개의 성능 코어가 중요해진다. **들리는 소문 인용**(미검증)으로는 이 성능 코어가 슈퍼 코어의 **약 70% 성능**을 내는 조금 더 작은 코어다.

기존 효율 코어는 단일코어 성능이 슈퍼 코어의 절반에도 한참 못 미쳤으므로 효율 코어와는 확실히 다른 등급이다. 슈퍼 코어 자체가 워낙 빠르니 그 70%라 해도 **M2~M3 성능 코어 사이 정도**는 나올 것이고, 최신 데스크톱 CPU 최고 성능 코어와 비교해도 크게 뒤처지지 않는다(추정).

### [02:30–02:52] 설계 의도 — 스위트 스팟 전성비 + 트랜지스터 절감
새 성능 코어는 두 가지 목적으로 설계된 것으로 보인다[의견]: ① 흔히 **스위트 스팟**이라 부르는 적당한 클럭 대역에서 전성비 극대화, ② 코어당 트랜지스터 수 절감. 결과적으로 모든 코어가 동시에 도는 상황의 전력 대비 성능을 높이고, **같은 칩 면적에 코어 수를 늘리기 훨씬 유리한 구조**가 된다.

### [02:52–03:19] 성능 예측치
이를 바탕으로 한 화자 예측(미검증):
- Geekbench 6: 싱글 약 **4,300점**, 멀티 약 **30,000점**
- Cinebench 2026 멀티, Blender CPU 벤치마크에서도 285K·9950X급 데스크톱 CPU와 비빌 결과

> "노트북에 들어가는 CPU가 우리가 사용하는 대부분의 프로그램들에서 데스크톱 CPU를 절대 성능 기준으로 앞설 거라는 얘기입니다." [03:09]

### [03:19–03:49] GPU — 깡성능 20%, 레이트레이싱 30%, 렌더링 40%
M5 Max GPU는 CPU보다 향상폭이 크다. 일반 그래픽(깡성능) **약 20%**, 레이트레이싱 포함 시 **30%**, 게임이 아닌 3D 렌더링 작업은 M4 Max 대비 **40% 가까이** 향상(애플 발표 기반 주장). 이를 근거로 3D 렌더링에서는 RTX 5090에 명함을 내밀 수준이 될 것으로 예상(추정).

### [03:49–04:11] 뉴럴 액셀러레이터 — AI 연산 4배
드디어 **엔비디아 텐서 코어에 대응하는 뉴럴 액셀러레이터가 GPU에 탑재**됐고, 덕분에 AI 연산 성능이 M4 Max 대비 **4배**로 빨라졌다(주장). 3D 렌더링·AI 성능 향상이 특히 중요한 이유는, 애플 실리콘의 구조적 강점인 **통합 메모리 아키텍처(UMA)**가 빛을 발하는 지점이 정확히 그 분야이기 때문.

### [04:11–04:34] UMA의 잠재력과 그동안의 한계
GPU 렌더링은 CPU보다 빠르지만 씬 용량이 커서 VRAM에 안 들어가면 무용지물이고, LLM도 모델 크기가 GPU VRAM을 넘으면 GPU 성능을 살리기 어렵다. UMA는 **VRAM에 해당하는 용량을 말도 안 되게 싼 가격으로 확보**할 수 있게 해주므로 이런 상황에서 잠재력이 크다. 다만 지금까지는 **그 상황에서의 절대 연산 성능이 엔비디아 GPU 대비 너무 낮아** 장점을 살리지 못했다.

### [04:34–05:17] M3 Ultra 실사용 경험 — 프리필이 답답하다
화자는 실제로 M3 Ultra에 약 **257GB 규모의 LLM**을 올려 [불명확: 자막 "오픈 클로" — 로컬 LLM 운용 도구로 추정] 세팅을 해뒀는데 속도가 아쉽다고 한다.

AI가 답을 시작하려면 입력 데이터를 먼저 쭉 읽고 해석하는 단계가 필요한데, 이 단계는 처리할 토큰이 이미 다 깔려 있어 **메모리 대역폭보다 순수 AI 연산 성능이 중요**하다. M3 Ultra는 그 연산 성능이 낮아, 대량 토큰과 함께 들어간 명령에 대해 **실제 응답/행동을 시작하기까지의 지연**이 답답하다는 것.

### [05:17–05:55] M5 Ultra 가정 — 응답 시작 5배, 생성 1.5배
M5 세대가 M4 대비 AI 연산 4배라면, **M5 Ultra가 나올 경우 M3 Ultra 대비 약 5배**로 볼 수 있다(추정). 즉 응답 시작까지 걸리는 시간이 약 1/5로 줄고, 메모리 대역폭도 1.5배 늘었으니 응답 시작 후 토큰 생성 속도도 1.5배가 된다(추정).

> "이 정도 스피드가 되면 진짜 작은 규모의 회사에서 내부적으로 쓰기에는 괜찮은 성능이 될 것 같습니다." [05:39]

정리하면 M5 세대 GPU 업그레이드로 **UMA 구조가 진짜 빛을 볼 수 있는 환경이 조성됐다**는 결론[의견].

### [05:55–06:31] 퓨전 아키텍처 — M5 Pro/Max는 단일 칩이 아니다
M5 Pro와 M5 Max는 M4 Pro/Max와 달리 **하나의 칩이 아니다**. ① CPU + 시스템 요소가 들어간 파트, ② GPU + 라스트 레벨 캐시 + 메모리 컨트롤러가 들어간 파트, 이 둘이 **퓨전 아키텍처**로 연결된 구조.

Max 칩 두 개를 붙여 Ultra를 만들던 UltraFusion과 개념은 비슷하지만 구현은 완전히 다른 차원이라고 한다(상세는 이전 영상 참조). 쉽게 말해 TSMC의 패키징 기술로 **나뉜 두 칩이 마치 하나의 칩처럼 동작**하는 것.

### [06:31–06:43] 1차 효과 — 칩이 작아지고 수율이 오른다
모든 요소를 한 칩에 때려박던 기존 방식 대비 실제 생산하는 칩 사이즈가 훨씬 작아지고, 칩이 작아지면 **수율이 오른다**.

### [06:43–07:41] Pro가 Max에 하극상하면 안 됐던 사정 (화자 추정)
"애플한테만 좋은 얘기 아니냐"에 대한 반박으로 M4/M5의 CPU 구성을 비교한다.

- M4 Max: 기본 성능 10 + 효율 4, 업그레이드 시 성능 12 + 효율 4
- M4 Pro: 기본 성능 8 + 효율 4, 업그레이드 시 성능 10 + 효율 4

M4 Pro는 M4 Max와 같은 설계에서 아래쪽 GPU 절반을 날린 구성으로 생산했을 것으로 추정되는데, 그렇다면 **불량 없는 양품은 성능 코어 12개 구성도 가능**했을 것이다. 그런데 왜 10개로 캡이 씌워졌나 — 면적 큰 CPU 코어가 12개나 들어간 거대 칩을 만들다 보면 **CPU 쪽에 결함이 있는 물건이 반드시 나오고**, 그 칩들까지 살려야 하니 Pro의 10코어 구성이 필요했으며, 동시에 Pro가 Max를 하극상하면 안 되니 캡이 유지됐다는 것 — 화자 표현으로 **"제피셜"**(개인 추측, 미검증).

### [07:41–08:06] 2차 효과 — 분리 생산이 바꾸는 등급 구성
이제 CPU 파트와 GPU 파트를 나눠 만드니 애초에 **CPU 파트의 불량 자체가 확 줄고**, "Max용으로 만들었는데 CPU에 빵꾸가 났다"는 케이스가 사라진다. 코어가 다 살아 있는 칩만 골라 Max로 만들면 되므로 **Max는 기본 구성에서도 CPU 코어가 전부 살아 있게** 됐고, **Pro도 업그레이드하면 CPU 구성을 Max와 동일하게** 맞출 수 있게 된 것 아니겠냐는 추정.

### [08:06–08:24] 조합식 제품 구성
GPU 파트는 기본적으로 Max용 설계를 만들고 그 아래쪽 절반을 빼고 생산하면 Pro가 되는 구조일 것으로 추정. 같은 CPU 파트에 **Max용 GPU를 붙이면 M5 Max, Pro용 GPU를 붙이면 M5 Pro**가 되는 형태.

### [08:24–09:10] 3차 효과 — 코어 수를 늘릴 수 있는 길
칩이 작아져 수율이 오르고 생산 단가가 떨어지면, 그만큼 CPU/GPU 파트를 키워 최대 성능을 올릴 여지가 생긴다. 단순하게 **CPU 코어 1.5배, GPU 코어 1.5배**를 넣는 식도 가능. 이 구성을 단일 칩으로 찍으면 비용이 말도 안 되게 뛰지만, 두 칩으로 나눠 찍으면 적당한 비용 상승으로 구현 가능하다.

여기에 같은 면적에 더 많이 넣을 수 있는 신설 성능 코어 특성이 합쳐지면, 인텔 **Sierra Forest**나 **Zen 5c 코어 기반 EPYC**처럼 코어 개수를 확 늘리는 것도 가능할 것(전망).

### [09:10–09:32] 그런데 M5 Max는 그렇게 하지 않았다 — 범인은 폼팩터
M5 Max는 방금 말한 방식으로 성능을 올리지 않았다. 화자가 지목한 범인은 **M1 Pro/M1 Max와 함께 공개된 현행 MacBook Pro 폼팩터**[의견]. 햇수로 5년째라 상당히 오래됐고, 처음엔 훌륭한 폼팩터였기에 아직도 어느 정도 역할을 하지만 이제는 한계가 보인다.

### [09:32–10:07] 전력·발열 한계의 구체적 근거
- 이 폼팩터와 함께 나온 **M1 Max는 최대 부하 전력 소모가 약 100W**였는데, **M4 Max는 CPU+GPU를 최대 부하로 갈면 200W를 넘는다**(주장).
- **MagSafe 140W 전원을 연결한 상태에서도 배터리가 빠진다.**
- 같은 "Max" 딱지지만 M1 Max와 M4 Max는 체급이 다른 칩. 풀로드 시 팬 소음이 미친 듯이 나고 칩 온도도 100도 가까이 올라간다.

> "퓨전 아키텍처 때문에 코어 개수를 더 넣을 수 있는 길이 열렸다고 해도 지금 폼팩터에서는 여기서 추가로 생기는 열을 식힐 수 없는 거예요." [09:57]

### [10:07–10:19] 새 폼팩터 시점 전망
애플이 이 사실을 가장 잘 알 것이므로, **빠르면 올해 하반기, 늦어도 내년 상반기에 완전히 새로운 폼팩터의 MacBook Pro**가 나올 것(예측). 디자인은 당연히 바뀌고 방열 설계·전력 공급 개선도 있을 수 있다.

### [10:19–10:52] M6 세대가 진짜 — 특히 탠덤 OLED
새 폼팩터가 나오면 퓨전 아키텍처 + 새 성능 코어 특성을 결합해 CPU/GPU 코어를 더 넣어 성능을 올릴 가능성이 열린다. 여기에 [불명확: 자막 "이노 검정" — 공정 세대 개선으로 추정]과 개선된 아키텍처까지 결합될 **M6 세대에서 또 한번 큰 폭의 성능 향상** 가능성이 높다(예측).

루머로는 터치스크린·셀룰러 추가도 있고, 무엇보다 **디스플레이가 탠덤 OLED로 바뀐다**. 아이패드 프로에서 이미 봤듯 휘도 면에서도 미니LED LCD에 밀리지 않으므로 **화질 관점에서는 장점만 있고 단점은 없는 확실한 업그레이드**[의견].

> "진짜 이제 화면 사이즈만 빼면 몇 천만 원짜리 레퍼런스 모니터 급의 디스플레이가 되는 거예요." [10:54]

### [10:58–11:26] 그런데 왜 지금 M5 Max를 두 대나 샀나 — 이유 ①
"디스플레이랑 실리콘에 미친 놈"이 [불명확: 자막 "토사고" — 곧 구형이 될 마지막 세대라는 취지로 추정] MacBook Pro가 될지도 모르는 M5 Max를 한 대도 아니고 두 대 산 이유.

- 현재 워크스테이션이 **M3 Max 2대 + M4 Max 1대**인데, M6 MacBook Pro가 너무 좋아 **세 대를 한꺼번에 교체하려다 가랑이 찢어지는 그림**이 떠올랐다[의견].
- 최근 3D 렌더링을 본격적으로 시작했더니 당장 성능이 모자란다.

### [11:26–12:00] 이유 ② — 램·낸드·파운드리 가격 폭등과 애플의 예외적 위치
AI 때문에 **램과 낸드 가격이 미쳐 돌아가는 것**은 다 아는 사실이고, 그 광풍에 가려졌지만 **파운드리 비용도 엄청 올랐다**. 다만 애플은 단일 회사 기준 꾸준히 엄청난 양의 낸드·D램을 소비하고, 엔비디아에 밀리긴 하지만 **TSMC의 두 번째 우수 고객**이다. 그래서 싼 가격에 장기 계약한 메모리 물량이 아직 남아 있는 것 같고, 신규 협상에서도 다른 회사보다 덜 비싸게 계약하는 것 같다(추정).

### [12:00–12:32] 이유 ③ — M6 때 가격 퀀텀 점프 예상
지금 발표되는 제품들과 **아이폰 18 시리즈까지는 치킨 게임에 가까운, 애플답지 않은 가격 정책이 유지**될 것으로 본다(예측). 하지만 아이폰 18까지 뽑고 나면 싸게 계약한 램·낸드 물량이 끝날 것이고, 그때쯤이면 다른 컴퓨터 제조사들은 이미 가격을 확 올렸을 것이다.

거기에 OLED 탑재, 폼팩터 변경, M6 시리즈가 겹쳐 상품성이 확 오르면 — 아이패드 프로가 OLED 넣으면서 가격 올랐던 전례처럼 — **가격이 퀀텀 점프할 것 같다**(예측).

> "짜증이 나지만 지를 수밖에 없었습니다." [12:30]

### [12:32–12:47] Studio Display XDR 스펙
Pro Display XDR의 후속 모델.
- 27인치 5K 해상도에 머문 건 아쉽다[의견]
- **로컬 디밍 존 576개 → 2,304개**로 대폭 증가
- **최대 휘도 2,000니트**

### [12:47–13:30] 색영역 — P3에서 Adobe RGB + P3로
기존에는 P3를 거의 완벽히 커버하는 색영역이었는데, 이번엔 **Adobe RGB + P3**로 바뀌었다. 둘은 상하위 관계가 아니라 **빨강 쪽은 P3가 더 순수하고 초록 쪽은 Adobe RGB가 더 순수**하므로, 둘 다 커버하려면 색영역이 그보다 더 넓어야 한다는 뜻이다.

이는 ① Adobe RGB 색까지 컬러 매니지먼트해 한 번에 정확히 보여줄 수 있고, ② HDR이 정의하는 **Rec.2020 색영역에서도 더 다양한 색**을 보여줄 수 있다는 의미. 단순 계산으로 Rec.2020 커버리지가 **기존 약 72% → 약 80%**로 늘 것으로 추정(화자 계산, 미검증).

### [13:30–14:11] 가격 520만 원이 왜 가성비인가
가격은 **520만 원**이며, Pro Display XDR과 달리 **스탠드(발)가 포함된 금액**이다. 가성비라는 말에 물음표가 붙을 수 있지만, 이건 **HDR 콘텐츠 제작자가 쓰는 장비**로 봐야 한다는 것이 화자의 관점.

비교 대상은 ASUS ProArt 계열 정도인데, 국내 비슷한 가격대 제품보다 Studio Display XDR의 스펙이 훨씬 좋다. 딱 비교할 만한 제품(8K 해상도, 디밍 존 4,000개)은 확실히 더 좋지만 **국내 미판매에 가격이 2천만 원**이다. 그래서 Studio Display XDR은 **스위트 스팟을 노린 제품**이라는 평가[의견].

### [14:11–14:31] 현업 맥락 — 모두에게 레퍼런스 모니터를 줄 순 없다
화자는 요즘 촬영 팀들, 그리고 영화·유명 OTT 납품작의 컬러 그레이딩을 하는 DI 업체와 일하고 있는데, 이런 곳에서도 모든 기사에게 몇 천만 원짜리 레퍼런스 모니터를 줄 수는 없어 실제로 **Pro Display XDR을 여러 대 활용**하고 있다. 이런 상황에서 Studio Display XDR은 상당히 매력적인 옵션이 된다.

### [14:31–15:16] 기존 HDR 작업 환경의 불편 — 클린피드 이원화
기존 화자의 HDR 작업 환경은 Studio Display에 메인 작업 공간을 띄우고, 옆에 MacBook Pro 디스플레이나 아이패드 프로를 두고 **클린피드를 띄워** 컬러를 볼 때는 그쪽을 봐야 하는 방식이었다.

[불명확: 자막 "KD 님"] 같은 경우 Pro Display XDR을 쓰는데, 로컬 디밍 존이 576개 수준이라 **전체 화면을 꽉 채워 프리뷰하면 괜찮지만, 작업 공간 안 작은 창으로 뜨는 HDR 프리뷰를 보고 컬러를 만지기는 어렵다**. 그래서 이 경우에도 MacBook Pro나 아이패드 프로에 클린피드를 띄워 함께 봐야 한다. MacBook Pro/아이패드 프로 병용 구성이 가성비가 워낙 좋아 그렇게 쓰고 있긴 하지만, **메인 작업 공간에서 바로 HDR 프리뷰가 되면 훨씬 낫다**[의견].
