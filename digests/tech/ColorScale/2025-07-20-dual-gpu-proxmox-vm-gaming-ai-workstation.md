# 1323만원짜리 PC를 만든 이유

> ColorScale · 2025-07-20 · 길이 약 8분 45초 · [영상 링크](https://www.youtube.com/watch?v=CYsJ7udTcPI)
> 정리 방식: 시간순 심층 워크스루 (paraphrase) + 주제별 상세 요약
> 검증 배너: 부품 가격·성능 소감은 화자 본인 구성 기준의 주장이며 별도 검증되지 않음. 영상 중반 MSI 파워서플라이 구간에는 제조사 제공 제품에 대한 광고 멘트가 포함된다.

## 요약 (주제별)

### 1. 프로젝트 목표 — 게이밍 PC와 AI 서버의 물리적 통합
- 게임용 PC와 인공지능 연구용 서버를 **한 대의 물리 머신**으로 합치는 올인원 프로젝트. 핵심 요구는 "게임이 돌아가는 중에도 AI 학습이 병행"되는 것.
- 총 부품 비용 **1,323만 7,410원**(화자 주장, 미검증).
- 부수적으로, 이 PC를 잠시 임시 NAS로 쓰면서 이전에 만든 "슈퍼 NAS"의 미완료 벤치마크를 마무리해 해당 시리즈를 완결하겠다는 계획도 언급.

### 2. 가상화 아키텍처 — Type-1 하이퍼바이저 + VM 2개
- VMware·Parallels 같은 Type-2는 호스트 OS(Windows/macOS) 위에서 돌고 VM은 다시 그 위에서 돌아 오버헤드가 크며 **특히 GPU 쪽 제한**이 많다는 것이 Type-1 선택 이유.
- 채택 스택: **Proxmox**(커널 통합 Type-1) 위에 Windows VM(게임·업무)과 Linux VM(AI 학습)을 병렬 구동. 실질적으로 한 머신에서 **OS 3개**가 동시에 돈다.
- 필요 시 Windows VM을 꺼서 전 자원을 Linux VM에 몰아줄 수 있는 유연성.

### 3. 부품 선택 논리 (왜 이 부품이어야 했는가)
- **GPU 2장**: RTX 5090(게임) + 기존 보유 RTX 4090(AI). 합산 목적이 아니라 VM별 배정 목적.
- **CPU 9950X3D**: VM 두 개 동시 구동에는 코어 수가 필요해 16코어 외 선택지가 사실상 없음. 3D 캐시가 붙은 CCD 코어를 Windows에 몰아주면 Windows 입장에선 9800X3D처럼 동작하고, 나머지 코어가 하이퍼바이저 + Linux VM 담당.
- **메인보드 MSI 갓라이크(MEG Godlike)**: 단자 측면 배출, 보드 내장 디스플레이, GPU 2장 장착, 10G/5G 이더넷 + Wi-Fi 7. **국내 미출시**라 Computex 취재차 대만에서 직접 구매.
- **메모리**: 64GB 모듈 2개, 총 **128GB** 튜닝램, 6400MHz.
- **스토리지**: 2TB NVMe SSD **4개**.
- **쿨러·케이스**: 화면 달린 쿨러 + 곡면 유리 케이스. 후면 배선(Project Zero/BTF) 방식을 원했으나 **X870E 칩셋에는 해당 보드가 잘 나오지 않아** 포기.

### 4. 전력·발열 — 두 장의 하이엔드 GPU를 감당하는 법
- **RTX 5090은 수랭(Suprim Liquid) 버전** 선택. 3슬롯 이상을 차지하는 4090 위에 또 대형 카드가 붙으면 카드 사이 공간이 부족해 쿨링 효율이 떨어질 것이 명백했기 때문.
- **파워: MSI 1600W, 80 PLUS 티타늄, ATX 3.1.** 과거 발화 논란이 있었던 12VHPWR의 업그레이드 커넥터를 채택하고, **커넥터가 처음부터 2개** 있어 GPU 2장 구성에서도 변환 케이블이 불필요한 점을 최대 장점으로 꼽음.

### 5. 성능 결과 — 가상화 손실은 실제로 얼마나 되는가
- **CPU**: Windows VM에 X3D 쪽 물리 코어 8개 + SMT 논리 코어 8개를 고정 할당. Geekbench 6, Cinebench 2024에서 **네이티브 9800X3D 대비 크게 빠지지 않는다**는 측정 주장(구체 수치는 화면 자료로만 제시).
- **GPU**: **PCIe 패스스루**로 장치 자체를 VM에 넘겨 이론상 성능 손실이 거의 없음.
- **동시 부하 데모**: Linux VM에서 AI 학습 벤치, Windows VM에서 사이버펑크 2077 벤치를 동시에 구동 — 소음은 커지지만 양쪽 모두 정상 동작.
- **실사용 호환성**: 보안 프로그램이 붙는 은행 업무·정부 사이트도 화자 사용 범위 내에서 문제없이 접속(화자 주장).
- [의견] "가상 머신은 성능 손해가 크다"는 통념은 그간의 가상화 기술 발전으로 옛말이며, 설정만 잘 잡으면 손실을 최소화할 수 있다는 것이 화자의 결론.

### 6. 기술 해설 — 메인보드 PCB 층수와 신호 무결성 ("오늘의 컴퓨터 지식")
- 메모리 성능은 CPU의 메모리 컨트롤러와 메모리 자체뿐 아니라, **둘 사이 신호를 얼마나 왜곡 없이 전달하느냐**(메인보드 역할)에 달려 있음.
- PCB 층수가 적으면 한 층에 배선이 몰려 신호선 길이를 성능 최적 형태로 설계하기 어렵고, 선들이 밀집해 상호 노이즈가 발생 → 같은 CPU·같은 메모리라도 고속 통신이 어려워질 수 있음.
- 따라서 8층·10층 PCB를 쓰는 플래그십 보드가 이 지점에서 메리트를 가질 수 있다.

### 7. 램 안정화 일화 (경험 기반, [의견] 포함)
- 직전 시스템(7800X3D + X670)에서는 호환 리스트에 있는 튜닝램조차 부팅이 매우 느렸고, 화자는 이를 CPU 수율("뽑기") 문제로 해석.
- 이번에는 128GB에서 6400MHz 동작을 보장하는 제품이 보드 호환 리스트에 없어 기대 없이 프로파일을 적용했는데 한 번에 안정 진입, 메모리 컨트롤러 1:1 매칭도 유지, 메모리 테스터 부하에서도 문제 없음.
- [의견] CPU 수율 영향도 있겠지만 **고급 보드의 영향이 컸을 것**이라는 "킹리적 갓심".

### 8. 광고·세일즈 구조
- MSI 1600W 파워는 **국내 미출시 상태에서 제조사가 제공한 제품**이며, 화자가 "제품 받았으니 광고 멘트 한 마디 넣는다"고 명시적으로 고지한 뒤 "동급 스펙 대비 가격이 괜찮고 불빛도 난다"고 짧게 언급. [광고 고지됨]
- Windows VM 세팅에 필요한 상세 설정 항목은 **영상 설명란(더보기) 링크**로 안내 — 심화 정보를 외부 링크로 유도하는 구조.
- 마무리에 구독·좋아요·알림 CTA, 슈퍼땡스 후원자 및 멤버십 가입자 감사 인사.

## 3줄 오리엔테이션
- 게이밍 PC와 AI 학습 서버를 **한 대의 물리 머신**으로 합치기 위해, Type-1 하이퍼바이저(Proxmox) 위에 Windows VM과 Linux VM을 동시에 올린 1,323만 원대 워크스테이션 제작기다.
- GPU 두 장(RTX 5090 + RTX 4090), 9950X3D, 128GB 튜닝램, 1600W 티타늄 파워 등 "두 개의 컴퓨터 몫"을 감당하기 위한 부품 선택 논리가 핵심이다.
- 가상화 오버헤드 문제는 CPU 코어 고정 할당 + GPU 패스스루로 해결했고, 게임 벤치와 AI 학습 벤치를 동시에 돌려 실사용 가능함을 보여준다.

## 전문 — 시간순 워크스루

### [00:00–00:14] 프로젝트 개요와 총 부품 비용
지난 영상에서 예고했던 "게이밍 PC + 인공지능 연구용 서버를 올인원으로 만드는 프로젝트"의 준비가 끝났다고 시작한다. 부품 가격만 **1,323만 7,410원**(화자 주장)이라는 숫자를 먼저 제시한다.

> "부품 가격만 1,323만 7,410원." [00:08]

### [00:14–00:37] GPU 두 장 구성 — RTX 5090 + RTX 4090
게임과 AI 연구 모두에서 가장 중요한 부품은 그래픽카드라고 전제하고, 게임용 최강 GPU로 RTX 5090을 준비했다고 밝힌다. 여기서 끝이 아니라 **게임이 돌아가는 중에도 AI 학습이 병행되도록** RTX 4090을 한 장 더 넣어 총 2장 구성을 만들었다. 즉 GPU 두 장은 SLI 같은 합산 목적이 아니라, 두 개의 가상 머신에 각각 배정할 목적이다.

### [00:37–01:17] 가상 머신·하이퍼바이저 기초 설명, Type-1을 쓰는 이유
보통 물리 컴퓨터 하나에 OS 하나가 도는 것이 일반적이지만, 하이퍼바이저라는 소프트웨어가 CPU·램·저장장치 같은 자원을 쪼개 가상의 컴퓨터를 만들고 그 위에서 또 다른 OS를 돌릴 수 있다고 설명한다. VMware나 Parallels 같은 제품은 Windows·macOS 같은 **호스트 OS 위에서 도는** 소프트웨어(Type-2)이고, 가상 머신은 다시 그 위에서 돌기 때문에 오버헤드가 크고 특히 GPU 쪽 제한이 많다고 지적한다. 그래서 이번 구성은 하이퍼바이저 자체가 커널에 통합된 **Type-1 하이퍼바이저**를 쓴다.

> "하이퍼바이저 자체가 커널에 통합돼 있는 타입 1 하이퍼바이저를 쓸 겁니다." [01:13]

### [01:17–01:35] VM 역할 분담 설계
Type-1 하이퍼바이저 위에 Windows VM과 Linux VM을 올리고, Windows VM은 게임·업무용, Linux VM은 AI 학습 처리용으로 세팅할 계획이다. 필요하면 Windows VM을 꺼버리고 모든 자원을 Linux VM에 몰아줄 수도 있다는 점을 유연성으로 언급한다.

### [01:35–01:55] 이전 NAS 프로젝트 언급과 임시 NAS 활용
예전에 만든 "슈퍼 NAS"는 만들자마자 바로 실사용에 들어가는 바람에 아직 제대로 벤치마크를 못 했다고 밝힌다. 이번 PC에는 2TB NVMe SSD가 **네 개** 들어가 있어서, 잠깐 이 신규 PC를 임시 NAS로 쓰면서 그 사이에 기존 NAS의 벤치마크를 돌려 해당 시리즈를 완결하겠다는 계획을 덧붙인다.

### [01:55–02:26] CPU 선택 — 9950X3D여야만 했던 이유
1차 목표가 강력한 게이밍 PC이므로 CPU는 라이젠 X3D 라인에서 골라야 하는데, 가상 머신 두 개를 동시에 돌리려면 코어 수가 16개인 **9950X3D** 외에 사실상 선택지가 없다고 말한다. 3D 캐시가 붙어 있는 쪽 CCD의 코어를 Windows VM에 몰아주면 Windows 입장에서는 마치 9800X3D를 쓰는 것처럼 동작하고, 나머지 코어들이 하이퍼바이저와 Linux VM 구동을 담당하는 방식으로 구성한다. (자막에는 "990X3D"로 오인식되어 있으나 문맥상 9950X3D)

### [02:26–02:42] 케이스·쿨러 선택과 후면 배선 보드의 벽
쿨러는 채널 성격에 맞게 **화면이 달린 제품**을 골랐고, 그 화면이 밖에서 잘 보이도록 곡면 유리 케이스에 마음이 갔다고 한다. 안쪽이 다 보이는 만큼 케이블을 뒤로 빼는 후면 커넥터 방식(MSI Project Zero / ASUS BTF 계열)을 쓰고 싶었지만, **X870E 칩셋에서는 그런 보드가 잘 나오지 않더라**는 제약에 부딪힌다.

### [02:42–03:10] 메인보드 — MSI 갓라이크, 대만 직구
차선책으로 고른 것이 MSI 갓라이크(MEG Godlike) 보드다. 단자들을 옆으로 뺄 수 있고, 보드 자체에 화면이 달려 있으며, GPU 두 장 장착이 가능하고, 10G·5G 이더넷과 Wi-Fi 7까지 지원한다는 점을 이유로 든다. 플래그십 보드라 품질도 좋을 것이라 기대했는데, 문제는 **국내 미출시**여서 Computex 취재차 MSI 본진인 대만에 간 김에 직접 사 왔다고 밝힌다. 받아보니 확실히 고급형답게 때깔이 좋다는 소감.

### [03:10–03:31] GPU 쿨링 — 5090은 수랭 버전으로
기존부터 쓰던 RTX 4090은 슬롯 세 개를 다 차지하고 아래로도 더 튀어나오는 대형 카드다. 이런 카드가 하나 더 위에 붙으면 사이 공간이 너무 좁아 쿨링 효율이 떨어질 것이 뻔하다는 판단에서, RTX 5090은 **라디에이터가 달린 수랭 제품(Suprim Liquid)**으로 준비했다. GPU 열을 라디에이터로 빼서 그쪽에서 식히므로 밀집 구성에서도 쿨링이 유리하다는 논리다.

### [03:31–04:15] 파워서플라이 — MSI 1600W 티타늄, ATX 3.1
RTX 4090과 5090은 각각 하나만 써도 전력을 많이 먹는 제품이라, 두 장을 감당하려면 그만큼 강력한 파워가 필요하다. GPU부터 보드까지 MSI로 통일하는 김에 MSI 제품군을 찾아보니 **1600W, 80 PLUS 티타늄, ATX 3.1** 규격의 딱 맞는 제품이 있었다고 한다. 과거 발열·발화 문제가 있었던 12VHPWR 커넥터의 업그레이드 버전(12V-2x6)이 들어가 있고, 무엇보다 **이 커넥터가 애초에 두 개** 달려 있어 GPU 두 장을 쓰면서도 변환 케이블을 쓸 필요가 없다는 점을 핵심 장점으로 꼽는다.

### [04:15–04:36] 광고 고지
이 파워 역시 국내 미출시 상태였는데, 출시 시점이 궁금해 MSI에 문의했더니 곧 출시한다며 미리 써보라고 제품을 제공받았다고 밝힌다. 영상 시청 시점에는 아마 출시됐을 것이라며, 제품을 받았으니 광고 멘트를 한 마디 넣는다고 명시적으로 고지한 뒤 "같은 스펙 대비 가격이 괜찮고 불빛도 난다"고 짧게 언급한다. [의견 / 광고]

### [04:36–04:55] 완성, 그리고 이전 PC의 램 트라우마
모든 부품을 합쳐 완성된 결과물을 보여준다. 직전까지 쓰던 컴퓨터는 7800X3D + X670 보드 조합이었는데, 분명히 호환 리스트에 있는 튜닝램을 썼는데도 부팅이 엄청 오래 걸렸다며 그것을 **CPU 뽑기(수율) 문제**로 해석한 경험을 꺼낸다.

### [04:55–05:34] 128GB 튜닝램이 6400MHz로 한 번에 잡힌 사연
이번 PC의 메모리도 튜닝램이며, 모듈 하나당 64GB짜리 두 개로 **총 128GB** 구성이다. 마더보드 호환 리스트에는 이 용량대에서 6400MHz 동작을 보장하는 제품이 안 보여 "밑져야 본전"이라는 심정으로 기대 없이 프로파일을 적용했는데, 너무 스무스하게 들어갔다고 한다. 메모리 컨트롤러 클럭과 1:1 매칭이 깨진 것 아닌가 확인했지만 그것도 아니었고, 메모리 테스터로 빡세게 돌려도 문제없이 안정화됐다. CPU 뽑기의 영향도 있겠지만 **고급 보드의 영향이 컸을 것**이라고 "킹리적 갓심"(합리적 의심)을 한다고 표현한다. [의견]

### [05:34–06:16] 오늘의 컴퓨터 지식 — 메인보드 PCB 층수와 신호 무결성
CPU의 메모리 컨트롤러와 메모리 자체의 능력도 중요하지만, **그 둘 사이에서 신호를 얼마나 왜곡 없이 깨끗하게 전달하느냐**가 메인보드의 역할이라고 설명한다. 비싼 보드에서 "PCB 8층, 10층" 같은 이야기를 하는 이유는, 층수가 적으면 한 층에 더 많은 배선을 욱여넣어야 하고 신호선 길이를 성능에 최적화된 형태로 설계하기 어려워지기 때문이다. 또 선들이 다닥다닥 붙어 있으면 서로 노이즈를 주고받아, 같은 CPU·같은 메모리를 써도 고속 통신이 어려울 수 있다. 결론적으로 플래그십 보드가 이런 부분에서 메리트를 가질 수 있다는 정리다.

> "PCB의 층수가 적으면 그만큼 한 층에 더 많은 배선이 들어가야 된다는 얘기고요." [05:50]

### [06:16–06:36] 완성 시스템 시연 — OS 세 개가 동시에
화면상으로는 그냥 Windows가 깔린 컴퓨터처럼 보이지만, 실제로는 **OS가 총 세 개** 돌고 있는 상태라고 밝힌다. 하이퍼바이저 역할을 하는 **Proxmox**에 접속해 있고, 한쪽 터미널은 그 위에서 도는 Linux VM에 붙어 있으며, 모니터에 떠 있는 것이 Windows VM이다.

### [06:36–06:54] 동시 부하 데모
Linux VM에서 AI 학습 벤치마크를 돌리는 동시에 Windows VM에서 사이버펑크 2077 내장 벤치마크를 구동하는 장면을 보여준다. 소리는 시끄러워졌지만 양쪽 모두 굉장히 잘 돌아간다고 설명한다 — 이 프로젝트의 목표였던 "게임 중 AI 학습"이 실제로 성립함을 보이는 대목이다.

### [06:54–07:33] 가상화 성능 손실은 얼마나 되나 — CPU 쪽
가상 머신이라고 하면 성능 손해가 클 것이라 생각하는 사람이 많지만, 그동안 가상화 기술 발전이 상당해서 설정만 잘 만지면 손실을 최소화할 수 있다고 말한다. 구체적으로 Windows VM에 **X3D 캐시가 있는 쪽 물리 코어 8개와 거기 딸린 SMT 논리 코어 8개**를 전부 할당했고, 그 결과 9800X3D를 쓰는 것과 거의 비슷한 상태가 된다. Geekbench 6, Cinebench 2024 결과가 **네이티브 9800X3D 측정치와 비교해도 크게 빠지지 않는다**는 것이 화자의 측정 주장이다. (구체 점수는 영상 화면 자료로만 제시, 자막에는 수치 없음)

### [07:33–07:48] GPU 패스스루와 실사용 호환성
GPU는 **패스스루** 방식으로 PCI Express 장치 자체를 VM에 넘겨줬기 때문에 이론적으로 성능 손실이 거의 없다고 설명한다. 실사용 측면에서도 본인이 쓰는 범위 내에서는 은행 업무나 정부 사이트 접속(보안 프로그램이 붙는 영역)이 문제없이 된다고 밝힌다.

### [07:48–08:12] 설정 공개와 피드백 요청
다만 이 정도로 Windows VM을 세팅하려면 손봐야 할 설정이 많다며, 어떤 항목들을 건드려 이 시스템을 만들었는지는 **더보기(설명란) 링크**에 자세히 적어두겠으니 관심 있는 사람은 참고하라고 안내한다. 시스템에 대한 의견이나 놓친 부분이 있다고 생각하면 댓글로 남겨달라고 요청한다.

### [08:12–08:32] 마무리 — 구독 요청과 후원자 언급
구독·좋아요·알림 설정을 부탁하고, 슈퍼땡스를 보낸 시청자들(레이지브, 김개민, 스크루진, H진 등)과 멤버십 가입자들에게 감사를 전하며 영상을 마친다.
