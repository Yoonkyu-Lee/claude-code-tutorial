# 무료로 풀린 초대박 게임, 호그와트 레거시 최신 맥북 프로, 맥미니 테스트 | 윈도우 게임을 이 정도로? | GPTK 3.0 | Rosetta 2

> 검증 배너: IT/테크 실측·해설 영상. 화자(에러)의 분석·추정과 확정 사실을 구분해 표기. 프레임 수치 비교는 하지 않았다고 화자가 명시.

## 요약 (주제별)

### 무엇을 했나 (개요)
- 애플이 공개한 **GPTK(Game Porting Toolkit, 게이밍 포팅 툴킷) 3.0**를 이용해, 윈도우 전용 게임 **호그와트 레거시**(해리 포터 세계관)를 맥에서 구동한 실측 리뷰.
- 촬영 기준 5월 4일까지 에픽 게임즈 스토어에서 해리 포터 25주년 기념으로 79,800원짜리 게임이 **무료 배포** 중이었다(확정 사실).
- 테스트 기기: **M5 Pro 맥북 프로**, **M4 Pro 맥미니** 두 대.
- 핵심 소감 [의견]: "윈도우용으로 만든 게임인데 이 작은 맥미니에서 이렇게까지 돌아간다"는 점이 놀랍고, GPTK가 강력해졌다.

### 실행 절차·도구
- **Heroic Games Launcher(히로익)** 를 게임 관리자(리모컨 역할)로 사용해 GPTK를 연결·실행.
- 히로익에서 에픽 게임즈 계정으로 로그인(화자는 "에러 언리얼텍" 계정 사용) → 에픽 스토어에서 무료 게임 다운로드.
- 히로익 세팅에서 **GPTK 3.0-3**(당시 최신 버전) 선택·설치.
- 스팀에는 이 게임의 맥 전용 앱이 따로 없고, 애플 GPTK를 통해 연결하는 방식.

### 그래픽 옵션별 실측 결과
- 기본(권장) 설정, QHD 디스플레이 기준으로 테스트. 시네마틱과 초반 스토리 구간에서 프레임이 잘 나오고 렌더링 양호(주장/실측).
- 업스케일링은 **AMD FSR3**(퀄리티/고성능), 프레임 생성은 **AMD 프레임 제너레이션** 사용.
- 모든 옵션을 **울트라 + 레이트레이싱 ON**: 시네마틱은 깔끔하나 캐릭터 **머리카락·파티클이 "찌글찌글"**(지글거림/셰이더 노이즈)하게 표현됨. 안티에일리어싱 미적용 또는 프레임 생성 과정의 과한 어프록시메이션으로 추정(추정).
- 레이트레이싱 OFF: 머리카락 지글거림이 확실히 사라짐. → 애플이 미지원해서가 아니라, **게임이 DirectX 기반 윈도우용으로 개발**돼 맞는 설정을 찾는 과정의 문제로 화자는 해석 [의견].
- 프레임 제너레이션 끄고 진행 시 화질 개선, 약간의 지글거림은 있으나 플레이에 지장 없음.
- 업스케일링을 아예 끄면 정지 화면은 매우 깨끗하나 움직일 때 파티클이 약간 발생. 옵션을 추가할수록 지글거림이 늘어나는 경향.
- 레이트레이싱 전부 고성능으로 켜면 프레임 수가 크게 떨어져 플레이가 힘든 구조.
- M5 Pro 맥북 프로에서도 실행됨. AMD 그래픽 카드 호환 경고창이 뜨지만 무시하고 실행 가능. 해상도는 다소 낮음.

### 번역 스택 구조 (GPTK / Wine / Rosetta 2 / Heroic)
- **Heroic**: 실행을 관리하는 게임 관리자(리모컨).
- **Wine(와인)**: 맥 시스템을 윈도우처럼 착각하게 만드는 **호환 계층(compatibility layer)**. 가상 머신이나 OS 에뮬레이션이 아니라 API·커맨드를 호환시키는 계층. → 성능이 확 깎이는 로제타식 박스 가상화와는 다르다고 강조.
- **GPTK / D3DMetal**: 윈도우의 **DirectX 명령을 애플의 Metal로 변환**해 GPU 렌더링을 담당. 이것이 게임 구동의 핵심.
- **Rosetta 2(로제타 2)**: **CPU 명령어 번역** 담당. 인텔 x86으로 돌던 것을 애플 실리콘용 ARM64로 변환.
- 정리: Wine(윈도우 API 호환) + GPTK/D3DMetal(그래픽=GPU 번역) + Rosetta 2(CPU 번역)가 각각 동시에 동작해 윈도우 게임을 맥에서 구동.

### 왜 어려운가 — 아키텍처 차이
- 윈도우 x86 하드웨어용 게임과 ARM 기반 소프트웨어는 완전히 다름. 비유: 안드로이드 카카오톡과 iOS 카카오톡이 겉은 같아도 하드웨어·OS가 달라 프로그램 체계가 다름.
- **메모리 구조 차이**: 윈도우는 CPU 할당 메모리 / GPU 할당 메모리(V램·DDR)가 **분리**돼 커맨드가 각각 날아감. 애플 실리콘은 CPU·GPU·NPU가 **유니파이드(통합) 메모리**를 공유. → 메모리 사용 방식부터 다름.
- **API 차이**: 윈도우는 DirectX, macOS는 독자 규격 드라이버 **Metal**. 그래픽 파이프라인이 복잡한 트리플 A는 DirectX로 섬세하게 만들어져 있어, GPU에 일을 시키는 방식 자체가 달라 "똑똑한 번역"이 필요.
- 화자 비유 [의견]: 한국어 "누리끼리한/구리구리한" 표현을 영어로 정확히 옮기기 어렵듯, DirectX 명령을 Metal로 정확히 바꾸기 어렵다. 단순 치환이 아니라 게임 엔진이 의도한 렌더링 흐름을 Metal에서 처음부터 재구성해야 함.

### 병목·한계·불안정 요소
- **그래픽 번역(GPTK)이 병목**. GPTK가 똑똑해질수록 렌더링 품질이 개선될 여지.
- 최초 실행 시 **셰이더 준비(shader cache) 구성**에 시간이 오래 걸리고, 처음 보는 그래픽 효과 등장 시 순간 끊김(셰이더 스터터링) 발생. 캐시가 쌓인 이후 재실행 시엔 문제 없음(추정 원인: 셰이더 캐시 구성).
- 울트라 + 업스케일링 + 레이트레이싱 조합에서 실행이 안 되는 경우가 있음. **M4 Pro는 괜찮은데 M5 Pro에서 실행 실패** 사례가 있어 충돌 가능성 조사 중.
- 그래픽 카드 호환 문제 표시(커널단 하드웨어 이슈)가 가끔 뜨지만 무시하면 실행됨.
- 텍스처 처리 불안정(머리카락 뭉개짐), 메모리 할당 변환 등이 더 살펴봐야 할 부분.

### M5 Pro GPU / 뉴럴 액셀러레이터 (관찰)
- M5 Pro의 GPU에는 **뉴럴 액셀러레이터가 GPU 내부에 별도 탑재**(뉴럴 엔진과는 다름). AI용 매트릭스 연산 용도.
- 이로 인해 기본 GPU 성능에서 게이밍 성능 개선이 있을 것으로 화자는 봄(추정). 단, 이 영상은 일반 게임 테스터처럼 프레임 수치를 비교한 것이 아님을 명시.

### 애플의 게이밍 전략 [의견/분석]
- 애플은 **하드웨어를 하나도 바꾸지 않고** 포팅 툴킷만으로 맥에서 게임 구동 환경을 구축. 이것이 확장되면 "맥이 게임기가 되는" 흐름 가능.
- 하이엔드 게이머는 여전히 윈도우+엔비디아/AMD를 쓰겠지만, 북미에서는 애플 맥 비중이 높아 **티핑 포인트**에서 애플이 게이밍을 크게 가져갈 수 있다는 예측.
- GPTK는 **우회(브릿지)** 역할이고, 동시에 게임사들과 협력해 **Metal 네이티브 포팅**(예: 예전 테스트한 사이버펑크는 앱스토어에서 네이티브 지원)이 나오면 성능이 우수. 애플 실리콘의 강력한 컴퓨팅·전성비로 트리플 A도 충분히 가능하다는 기대.
- 20년간 축적된 윈도우·DirectX 생태계를 무너뜨리기 위해, 애플은 자사 생태계 강점 위에 GPTK라는 브릿지를 확장해 나갈 것으로 예측. 다음 달 행사(맥락상 WWDC급 이벤트로 추정)에서 게임 포팅 관련 개선점이 더 나올 것으로 기대.
- 현재 수준에선 특정 게임은 되고 특정 게임은 안 될 수 있음(최적화 편차). 호그와트 레거시는 잘 돌아가는 편.

## 전문 — 시간순 워크스루

### [00:00–00:13] 오프닝 훅
작은 맥미니에서 이렇게 돌아간다는 것 자체가 신기하고, 게이밍 포팅 툴킷이 정말 강력해졌다는 소감으로 시작한다. "아예 윈도우 용으로 만들어진 게임인데 이렇게까지 한다고?" 하는 느낌이라고 말한다 [의견].
> "이 작은 맥미니에서 이렇게 돌아가는 자체가 전 좀 신기해요. 그리고 게이밍 포팅 툴킷이 정말로 강력해졌구나." [00:00]

### [00:13–00:30] 인사·테스트 개요
화자(에러)가 인사하고, 촬영 기준 5월 4일까지 무료로 풀린 호그와트 레거시(해리 포터 세계관 게임)를 소개한다. 무료 배포 소식을 듣고 **M5 Pro 맥북 프로**와 **M4 Pro 맥미니**에서 각각 돌려봤다고 밝힌다.

### [00:30–00:44] 스팀엔 맥 전용 앱이 없다 → GPTK로 연결
스팀 게임에는 맥 전용으로 나온 앱이 따로 있는 게 아니라, 애플이 공개한 **GPTK(게이밍 포팅 툴킷)** 를 이용해 연결한다고 설명한다. 실제로 해보니 자연스럽게 잘 돼서 관련 내용을 살펴보겠다고 예고한다.

### [00:44–01:07] Heroic 런처 + 에픽 로그인, 무료 배포 확인
플레이를 위해 **히로익(Heroic) 게임 런처**로 GPTK를 연결해 실행하려 한다. 히로익에서 에픽 게임즈 계정으로 로그인할 수 있고, 화자는 "에러 언리얼텍" 계정으로 로그인했다. 왼쪽 스토어 → 에픽 스토어에 들어가면 해리 포터 25주년 기념으로 **79,800원짜리 게임이 무료**로 풀린 것을 확인할 수 있고, 여기서 다운로드하면 된다.

### [01:07–01:17] GPTK 3.0-3 설치
세팅에 들어가면 **게임 포팅 툴킷 3.0-3**이 당시 최신 버전이며, 이걸 선택해 설치한다. 와인이 무엇인지 등 기술적 내용은 뒤에서 설명하겠다고 한다.

### [01:17–01:41] M4 Pro 맥미니 실측 — 시네마틱·프레임 양호
M4 Pro 맥미니에서 캐릭터를 세팅하고 초반 스토리 시네마틱을 실행하는데 프레임이 잘 나온다(주장/실측). 권장 사양으로 돌려 프레임이 잘 나오는 형태로 렌더링된다고 설명한다. 핵심은 **로제타 같은 에뮬레이터로만 돌리는 게 아니라 실제로 게임이 아주 잘 돌아간다**는 점이며, 트리플 A급인데도 그렇다. 단, 현재는 레이트레이싱(빛 추적)을 적용하지 않은 버전이라고 명시한다.

### [01:41–01:58] 하이엔드 대비 한계 인정
정말 빡센 외장 그래픽 카드로 하는 것 대비하면 하이엔드 게이머에겐 부족할 수 있지만, "이만한 맥미니에서 돌렸는데도 이 정도"면 충분하다는 취지. 게임플레이 중 가끔 멈추는 경우와 그래픽이 약간 어색한 부분이 있으나, 일부러 빡세게 해봐도 플레이엔 크게 무리가 없었다.

### [01:58–02:26] 옵션 세팅 — QHD + FSR3 + 프레임 제너레이션, 전부 울트라
디스플레이가 QHD여서 QHD에서 테스트 중이다. **AMD FSR3**로 업스케일링 품질을 고성능으로 올리고 프레임 생성도 **AMD 프레임 제너레이션**으로 설정. 그래픽 설정을 원래 권장에서 **울트라**로 전부 바꾸고, 심지어 **레이트레이싱까지 켜서** 다시 시작해 테스트했다.

### [02:26–02:59] 울트라+레이트레이싱의 문제 — 머리카락 지글거림
게임플레이를 해보니 화면이 약간 "찌글찌글"하게 생긴다. 전부 울트라라서 안티에일리어싱이 제대로 안 된 건지, 프레임 생성 과정에서 과한 어프록시메이션을 한 건지(추정) 시네마틱은 깔끔한데 **캐릭터의 머리카락 같은 피사체가 지글거린다**. 파티클·머리카락 표현이 지저분해지는 부분으로, 업스케일링 효과가 제대로 적용 안 돼서 그런 것으로 보인다(추정).

### [02:59–03:16] 레이트레이싱 OFF → 지글거림 해소, 원인 해석
업스케일링을 낮추고 FSR 품질도 낮추고 레이트레이싱도 껐다. 레이트레이싱을 아예 안 하니 머리 질거림이 확실히 없어졌다. 화자는 이를 "애플이 그것도 지원 안 하네"가 아니라, **게임 자체가 윈도우용 다이렉트X 기반으로 개발돼** 거기에 맞는 설정을 찾는 과정이 힘든 것으로 해석한다 [의견].

### [03:16–03:38] 프레임 제너레이션 끄고 뽑기 — 화질 개선
프레임 제너레이션을 끄고 레이트레이싱으로 쭉 뽑으니 화질이 더 좋아진다. 약간 지글거리는 게 신경 쓰이긴 하지만 플레이엔 전혀 문제 없는 타입이라고 정리한다.

### [03:38–03:56] 업스케일 OFF 및 최고 레이트레이싱 — 프레임 급락
업스케일이 문제일까 봐 아예 꺼보니 정지 화면은 매우 깨끗한데 움직일 때 파티클이 약간 있다. 옵션을 추가할 때마다 지글거리는 게 생기는 것 같다. 고성능으로 레이트레이싱을 전부 빡세게 켜니 **프레임 수가 많이 떨어져** 플레이하기 힘든 구조가 된다.

### [03:56–04:14] 다시 소감 — 맥미니 구동에 감탄
작은 맥미니에서 이렇게 돌아가는 것 자체가 신기하고 GPTK가 강력해졌다는 소감을 반복한다 [의견]. 게이머가 이걸 게임하겠다고 맥미니를 사진 않겠지만, **이미 맥미니를 가진 수많은 유저는 이걸로 그냥 하면 된다**는 점을 강조한다.

### [04:14–04:28] M5 Pro 맥북 프로 실측 — 경고창 뜨지만 실행됨
M5 Pro 맥북 프로에서도 실행. **AMD 관련 그래픽 카드 호환 경고창**이 뜨지만 실행은 된다. 똑같이 만들어 테스트했고 해상도는 다소 낮은 급이며, 권장 사용으로 하면 깨끗하게 나온다.

### [04:28–04:49] M5 Pro GPU의 뉴럴 액셀러레이터
이번 M5 Pro의 GPU는 **GPU 안에 뉴럴 액셀러레이터가 따로 들어갔고**, 이는 뉴럴 엔진과는 다르다고 이전에 별도 설명했다고 언급한다. AI용 매트릭스 연산 용도로 들어갔기 때문에 기본 GPU 성능에서 게이밍 성능 개선이 있을 것으로 봄(추정). 다만 일반 게임 테스터처럼 프레임을 막 비교하는 것은 아니라고 명시한다.

### [04:49–05:11] 윈도우 게임 ≠ 맥 게임
이게 어떻게 되는지 이해하고 애플의 게임 전략을 생각해봐야 한다는 취지로 넘어간다. 채널을 꾸준히 본 사람은 알겠지만, **윈도우 게임을 만드는 것과 맥에서 돌아가는 게임을 만드는 것은 완전히 다른 형태**라고 짚는다.

### [05:11–05:38] x86 vs ARM, 카카오톡 비유
윈도우 안에서도 x86 기반 하드웨어용 게임과 ARM 기반 소프트웨어는 완전히 다르다(게임도 소프트웨어). 비유: 안드로이드에서 도는 카카오톡과 iOS에서 도는 카카오톡이 겉은 같아도 프로그램 체계가 다른데, **하드웨어가 다르고 그 위 OS가 다르고 명령어 자체가 다르기 때문**이다.

### [05:38–05:49] GPTK = 그 둘을 잇는 것, 로제타식 박스와 다름
그 다른 것들을 연결해 주는 형태가 애플의 게임 포팅 툴킷이다. 윈도우 가상 환경을 박스로 만들어 성능이 확 깎이는 **로제타(박스 가상화) 환경과는 다르다**고 강조한다.

### [05:49–06:07] 왜 쓸 만하고 왜 완벽하진 않은가 — 이해 전제
왜 쓸 만하게 되는지, 왜 완벽하게는 안 되는지 이해하려면 하드웨어와 OS 스케줄러 관련 얘기를 알아야 한다. 그럼에도 애플이 GPTK를 꾸준히 3.0까지 올린 이유, 그리고 사람들이 이걸 많이 쓰기 시작했다는 사실을 이해할 필요가 있다고 한다.

### [06:07–06:32] 호그와트 레거시의 태생 — DirectX/x86/분리 메모리
3년 전 나온 호그와트 레거시는 애초에 애플의 언어로 만들어지지 않았다. **윈도우 기반 그래픽 처리인 다이렉트X 기반**이고, 하드웨어 배경은 **x86 기반 CPU**, 메모리도 **GPU의 V램 + DDR 같은 레거시 D램** 기반이지, 애플 M시리즈의 **유니파이드(통합) 메모리 구조**가 아니다. 유니파이드 메모리는 CPU·GPU·NPU가 모두 통합 메모리에 접근한다.

### [06:32–06:53] 메모리 커맨드 방식의 근본 차이
윈도우는 "내 CPU에 할당된 메모리로 갔다, 내 GPU에 할당된 메모리로 갔다"는 식으로 **분리된 구조로 커맨드가 날아간다**. 반면 맥은 애초에 메모리가 합쳐져 있어 **메모리 쓰는 방법부터 다르다**. GPTK가 이런 것들을 연결해 준다.

### [06:53–07:11] DirectX vs Metal, ARM M시리즈
윈도우 게임은 윈도우 API·다이렉트X를 토대로 하는데, macOS는 다이렉트X가 아니라 **메탈(Metal)이라는 독자 규격 드라이버**를 돌리고, ARM 기반 자체 설계 **M시리즈**에 통합 메모리를 갖고 있어 서로 다르다. 그래서 윈도우 게임을 그대로 가져와선 당연히 안 되고 번역기가 필요하다.

### [07:11–07:33] 실행 스택 — Heroic / Wine / GPTK·D3DMetal
그 번역기가 게임 포팅이다. 호그와트 레거시를 돌릴 때 화자는 **히로익(실행 관리 프로그램)** 으로 실행했고, **와인(Wine)** 을 통해 윈도우 API와 호환되게 하며, 실제 구동은 **다이렉트X를 메탈로 바꾸는 GPTK, 그리고 D3D→메탈** 연결로 이뤄진다. 그렇게 M시리즈 하드웨어를 메탈로 접근한다.

### [07:33–07:59] Rosetta는 CPU만, GPTK는 GPU 드라이버 번역
호그와트를 **로제타 같은 걸로 CPU에서 돌린다는 건 단순히 x86을 ARM64로 바꾸는 역할**이다. 즉 게임 포팅 툴킷(GPTK)은 **GPU로 돌리는 드라이버를 게임 구동용으로 바꿔 주는 역할**이고, 로제타는 **CPU 명령어만 바꾸는 역할**이다. 와인은 윈도우 게임이 맥이라는 시스템을 윈도우처럼 착각하게 만드는 **호환 계층**.

### [07:59–08:19] Wine의 정체 — 가상화 아닌 API 호환 계층
와인 자체는 가상 머신이나 OS 에뮬레이션이 아니라, 부르는 커맨드·API를 호환하게 만드는 계층이다. 결국 다이렉트X로 돌아가는 것을 **GPTK/D3D→메탈**로 바꿔 주는 역할을 애플이 적극적으로 밀고 있다.

### [08:19–08:35] GPTK가 핵심 병목
이게 없던 시절엔 여기 자체가 병목이며, **이 번역이 얼마나 잘 만들어지냐에 따라** 원래 저쪽(윈도우)에서 돌던 GPU 그래픽 렌더링을 예쁘게 바꿔 줄 수 있다. 다이렉트X와 메탈은 구조가 아예 다르고(메탈은 통합 메모리, 다이렉트X는 분리 메모리) 셰이더 실행 방식도 다르다.

### [08:35–08:58] 단순 치환이 아니다 — 트리플 A의 섬세한 파이프라인
"이 명령어는 이걸로 바꿔"라는 식인데, 트리플 A는 그래픽 파이프라인이 매우 복잡하고 다이렉트X로 섬세하게 만들어져 있다. GPU에 일을 시키는 방식이 아예 달라서 이를 똑똑하게 바꿔 주는 역할이 필요하고, 그게 GPTK의 역할이다.

### [08:58–09:19] 셰이더 캐시 — 초반 스터터링
실제로 셰이더를 처음 실행할 때 셰이더 준비에 굉장히 오래 걸린다(맥북 프로·맥미니 모두). 이런 구성을 하는 것으로 추정된다. 처음 보는 그래픽 효과가 등장할 때(셰이더 스터터링) 순간적으로 끊기다가, **셰이더 캐시가 쌓인 이후 재실행 시엔 문제가 없다**. 초반 적응 단계가 필요하다.

### [09:19–09:37] 고옵션 실행 실패 — M5 Pro 이슈 조사 중
높음/울트라, 업스케일링, 레이트레이싱을 적용하면 실행이 안 되는 경우가 있는데, 특히 **M4 Pro는 괜찮은데 M5 Pro에서 실행이 안 되는 경우**가 있어 충돌 여부를 살펴보고 있다.

### [09:37–09:51] 호환 경고창 무시 → 실행됨
그래픽 카드 호환 문제가 발생한 것처럼 표시되는 경우가 있지만 무시하고 실행하면 된다. 커널단 하드웨어 얘기라 이런 충돌이 가끔 있어도, **실행되게끔 번역 레이어가 역할**을 한다.

### [09:51–10:13] Rosetta 2로 CPU 별도 번역 — GPU·CPU 동시 번역
CPU 언어 번역은 **로제타 2**로 따로 한다. 인텔 CPU용으로 돌던 것을 애플 실리콘이 처리할 수 있게 ARM64로 번역하면서, GPU와 CPU 번역을 동시에 진행한다. 즉 히로익(리모컨) + 와인 + GPTK + 로제타가 각각 번역되는 구조로 하면 윈도우 게임마저 맥에서 잘 돌아간다.

### [10:13–10:32] 그래픽 번역이 병목 재강조
좀 더 자세히 보면 결국 **그래픽 번역 부분이 병목**이다. GPTK가 얼마나 똑똑해지냐에 따라 이런 부분들이 점점 개선(확정)될 수 있다.

### [10:32–10:53] 번역의 본질 — 렌더링 흐름 재구성
비유 [의견]: 한국어 "누리끼리한/구리구리한" 같은 표현을 영어로 정확히 옮기기 어렵듯, 다이렉트X 명령을 메탈로 정확히 바꾸기가 쉽지 않다. 단순 번역이 아니라 **게임 엔진이 의도한 렌더링 흐름을 메탈에서 처음부터 재구성**해야 한다. 그래서 비슷한 결과가 나오는 포인트들이 있다.

### [10:53–11:03] 그럼에도 GPTK 3세대의 성과
애초에 완벽한 번역은 안 될 수 있지만, AI가 발전하고 프로그래밍 플로우가 발전하면 개선될 것이다. **벌써 GPTK 3세대만에 이 정도로 잘 나오기** 때문에, 맥 하드웨어가 워낙 뛰어나서 맥 전용 게임이 아니어도 준수한 게이밍 성능을 뽑아낼 수 있다는 기대 포인트가 있다 [의견].

### [11:03–11:34] 유니파이드 메모리 소감 — 공유의 양날
일반 CPU는 CPU·GPU가 분리되고 각자 램이 분리돼 있다(요즘 통합 분위기도 있음). 애플 실리콘은 CPU·GPU 모두 유니파이드 메모리를 통합 기대한다. 이 구조라 오히려 메모리를 공유하다 보니, 윈도우는 분리를 전제로 게임을 짜는데 통합 메모리라서 **약간 충돌이나 비율적 문제**가 생길 수도 있다(추정). 이를 GPTK에서 더 잘 만들어 가면 좋겠다는 바람.

### [11:34–11:51] 다음 달 행사 기대 — 지속 개선 전망
다음 달 행사(맥락상 애플 이벤트/WWDC급으로 추정)에서 게임 포팅 관련 내용·개선점이 하나씩 나올 것으로 본다. 애플 하드웨어는 계속 발전하고, CEO도 하드웨어 기반으로 [불명확: 원문 "존토너스가 수정이 돼서 나가고 있기"] AI 시대에 기대되는 부분이 있다.

### [11:51–12:15] 오류의 대부분은 텍스처 처리
제작하며 발생한 오류의 대부분은 **텍스처 처리가 불안정한 부분**이다. 앞서 머리카락이 뭉개져 나온다거나, 메모리 할당 변환 등을 더 살펴봐야 한다. 결국 GPTK는 아직 부족한데, 애초에 윈도우용 게임이니 당연히 부족할 수 있다.

### [12:15–12:36] 엔지니어 관점 — "이걸 어떻게 이렇게까지"
엔지니어 관점에서 보면 "OS도 다르고 하드웨어도 다른데 이걸 어떻게 이렇게까지 했지"라는 부분이 있다 [의견]. 그만큼 GPTK가 스마트하게 잘 번역해 주는 수준까지 왔다는 것 자체가 의미가 있다. 게다가 애플은 **하드웨어를 하나도 안 바꿨다**.

### [12:36–13:02] 포팅 툴킷만으로 맥=게임기 흐름
포팅 툴킷 자체만으로 게임을 지원하게 되는 순간부터 맥에서 게임을 돌리는 환경이 구축되고, 이게 확장되면 "맥이 게임기가 돼 버린다"는 느낌으로 퍼질 수 있다 [의견]. 하이엔드 게이머는 윈도우·엔비디아·AMD를 사서 돌리겠지만, **북미에서는 애플 맥 비중이 중심**이라 어떤 티핑 포인트에서 애플이 게이밍을 확 먹어버릴 수 있다는 예측. 현재는 특정 게임은 되고 특정 게임은 안 될 수 있으며(최적화 편차), 호그와트 레거시는 잘 돌아가는 편이었다.

### [13:02–13:34] 브릿지 + 네이티브 포팅 — 사이버펑크 예
GPTK가 다리를 놓아준 역할이고, 예전에 테스트한 **사이버펑크**는 앱스토어에서 다운받아 네이티브하게 지원돼 성능이 우수했다. 최근 게임들은 애플 네이티브로 나온 것이 많고 게임사들과 협력이 활발하다. GPTK가 **우회로로 동작하는 동시에, 메탈로 직접 최적화한 네이티브 포팅** 게임이 나오면 성능이 매우 우수하다. 애플 실리콘의 강력한 컴퓨팅과 전성비를 볼 때 트리플 A도 충분히 가능하다.

### [13:34–13:57] 생태계 전략 예측
결국 생태계 얘기다 [의견]. 윈도우와 다이렉트X가 20년간 갖춰온 빡센 생태계를 무너뜨리기 위해, 애플은 자사 생태계의 강력함에서 출발해 **브릿지(게임 포팅 툴킷)를 확장**해 나갈 것으로 예측한다. 여러 요소 중 하나만 불안해도 지원 안 되는 게임이 있을 수 있지만, 이것도 이제 시작이다.
