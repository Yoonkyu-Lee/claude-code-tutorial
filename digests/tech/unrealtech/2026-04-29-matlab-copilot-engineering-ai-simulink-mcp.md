## 요약 (주제별)

### MATLAB Copilot의 정체성
- MathWorks가 MATLAB에 새로 붙인 AI 어시스턴트가 "코파일럿(Copilot)"이다. 화자는 이를 단순 코드 생성기(ChatGPT류 LLM)와 구분한다.
- 핵심 차별점: 코드만 뱉는 게 아니라 시뮬레이션·데이터 분석·그래픽 등 **MATLAB 고유 생태계 전체를 이해한 상태**에서 MATLAB의 기능들을 직접 사용하며 돕는 "조수" 역할이라는 것 [의견].
- 채팅 방식(우측 패널)과 라이브 에디터(.mlx) 안에서의 **인라인 프롬프팅**(Windows: Ctrl+Shift+P, Mac: Cmd+Shift+P) 두 가지로 호출 가능.

### 데모 1 — 반도체 웨이퍼 수율/불량 분석
- MATLAB 기본 라이브러리 예제로 진행. copy command로 예제를 바로 열고, WM-811K 계열로 보이는 웨이퍼 디펙트 맵 데이터를 다운로드해 테스트 [불명확: 자막 "WM 파릴 K"].
- 웨이퍼 맵은 0/1/2 값의 unit8 행렬(0=바깥, 1=정상, 2=결함). 원본 상태로는 눈대중 파악만 가능해 분석 처리가 필요하다는 점을 지적.
- Copilot에게 데이터 구조를 설명하고 "보기 좋은 웨이퍼 맵을 그리는 MATLAB 함수를 작성해줘"라고 요청 → 함수부(예: plotWaferMap)와 실행부로 나뉘어 생성, 코드로 옮겨 figure로 시각화.
- 추가 요청 예: "배경 0을 제외하고 다이 수를 분모로 결함율·수율 계산, 약 200개 웨이퍼 데이터 사용, 주석 없이" → 코드 즉시 생성. 이어 "3차원 그래프로 시각화" 요청 시 3D 플롯 생성 + 스케일 조절 재생. 주석도 Copilot이 자동 생성.

### 데모 2 — Simulink 자율주행 긴급 제동(AEB)
- Simulink는 MATLAB의 시스템 설계 도구로, 화자는 블록을 연결해 시스템을 구현하는 방식을 두고 "바이브 코딩의 아버지(20년 전부터)"라 표현 [의견].
- 예제: "Autonomous Emergency Braking with Sensor Fusion"을 command로 열어 AEB 테스트 벤치를 띄움. 센서 신호 → 디시전 로직 → 전방 차량이 가까우면 자동 브레이크.
- 기존 AEB 브레이크 판단 로직을 Copilot으로 대체하는 시나리오. "my AEB logic" 블록을 추가하고, 기존 블록의 입력/출력에 맞춰 MATLAB Function 블록용 코드를 요청(예: 전방 장애물 거리 < 20이면 brake signal -5, 아니면 0).
- 생성된 로직 블록을 기존 자리에 연결(TTC=time-to-collision, PB1 등 입출력 매칭). 데이터 타입 불일치는 command로 안내되어 재세팅. 세팅 테스트 통과 후 시뮬레이션 탭에서 실행 → 시간에 따른 매트릭·조감도(bird's-eye) 스코프로 동작 확인.
- 화자가 꼽는 핵심: Simulink 모델의 움직임을 Copilot이 전부 이해하고 있다는 점 [의견].

### 데모 3 — MATLAB MCP 연동 (Claude 등 외부 에이전트)
- MATLAB에 최근 MCP(Model Context Protocol) 기반 연동이 추가됨. MCP는 Claude Code, VS Code의 GitHub Copilot 같은 외부 에이전틱 AI 도구가 표준 방식으로 외부 도구를 직접 호출하게 해주는 프로토콜.
- 의미: 기존 LLM이 "코드를 만들어 주는" 역할에 가까웠다면, MCP가 붙은 MATLAB에서는 AI 에이전트가 **실제 MATLAB 워크플로우를 실행하고 그 결과를 받아 다음 추론까지 이어감**. AI가 MATLAB을 "도와주는 대상"이 아니라 "직접 사용하는 계산 엔진"으로 다루게 됨 [의견].
- 설치: MathWorks 사이트 안내 → GitHub의 latest release에서 MATLAB MCP core server 설치 → Claude에 연결. 일반 MCP 설치처럼 허용 클릭 시 MATLAB 자동 연결.
- 연결 후 Claude가 MATLAB + 툴박스 9개를 전부 인식. Claude에서 MATLAB 명령 실행 가능(권한 확인 후). 예: 예제 데이터 로드·기본 시각화, 가장 큰 피크 5개를 표로 정리·계산. 시연 중 태양 흑점(sunspot) 데이터 자동 로딩·분석, 결과 PNG 저장까지 확인.

### 화자의 결론·경계
- MATLAB Copilot은 단순 코드 생성 AI가 아니라 반도체 데이터 분석~자율주행 설계까지 MATLAB 엔지니어링 생태계 전반의 스마트 어시스턴트라는 평가 [의견].
- 경계: Copilot 결과를 그대로 쓰는 게 아니라 **엔지니어의 전문 지식으로 검토·발전시키는 과정이 반드시 필요**하다고 명시.
- 큰 그림: AI가 기존 개발 툴 전반으로 퍼지면서 개발뿐 아니라 연구·검증 분야에서도 변화가 일고 있다는 관찰 [의견].

> "시뮬레이션이라든가 데이터 분석 그래픽 같은 매틀랩 고유의 생태계 안에서 전체를 이해한 상태로 매틀랩의 다양한 기능들을 사용하면서 도움을 받을 수 있는 (…) 코파일러 조수 같은 존재" [01:06]
첫 시연으로 MATLAB이 제공하는 웨이퍼 데이터를 Copilot에게 물어 웨이퍼 맵을 그리는 것을 예고한다.

### [01:26–01:56] 웨이퍼 수율 예제 로드
MATLAB Map 기본 라이브러리로 시현. 웨이퍼 맵에 딥러닝으로 디펙트를 클래시파이(분류)하는 예제를 가져온다. copy command에서 예제를 바로 열 수 있고, 해당 예제는 웨이퍼 디펙트 맵 데이터를 다운로드해 테스트하도록 되어 있다 [불명확: 자막 "다운로드 WM 파릴 K 웨이퍼디펙트 맵"; WM-811K 계열 데이터로 추정되나 단정 안 함]. 명령을 넣어 웨이퍼 파일을 로드한다.

### [01:56–02:33] 웨이퍼 맵 데이터 구조와 분석 필요성
로드된 웨이퍼 맵은 전체가 1·2·0으로 구성되며 디펙트 부분이 2로 표기된다. 이 상태로는 눈대중 확인만 가능하고 실질적 데이터 파악이 어려워 분석가 처리가 필요하다고 지적한다. 그래서 MATLAB Copilot 우측 창에 데이터를 설명한다 — unit8 행렬이고 0은 바깥, 1은 정상, 2는 결함이라고 알려주면서 "보기 좋은 웨이퍼 맵으로 그리는 MATLAB 함수를 작성해줘"라고 입력한다.

### [02:33–03:17] 함수 생성과 시각화
플랫폼 안에서 MATLAB이 함수를 만들어 주는 것을 보여준다. 함수로 제공해 달라고 했기 때문에 (plotWaferMap 류의) 함수부와 실제 실행부 두 부분으로 나뉘어 나온다. 이를 코드로 옮겨 실행하면, 앞서 예제에서 불러온 웨이퍼 데이터 1번의 웨이퍼 맵이 figure로 바로 시각화된다. 화자는 이 모든 게 MATLAB 생태계 안에서 만들어진다는 점을 강조한다.

### [03:17–03:48] 인라인 프롬프팅
Copilot은 우측 채팅뿐 아니라 라이브 편집 .mlx 파일 안에서 바로 쓰는 인라인 프롬프팅도 된다. Mac은 Cmd+Shift+P, Windows는 Ctrl+Shift+P. "앞서 읽어온 웨이퍼 맵에 대해 알려줘"라고 치면 소스 코드가 바로 나오고, 실행해 볼 수 있다.

### [03:48–04:34] 수율 계산과 3D 플롯 자동화
예전에는 다 직접 신경 써서 짰어야 했지만 그럴 필요가 없다고 한다. 예로 "웨이퍼 맵에서 배경 0을 제외하고 다이 수를 분모로 결함율과 수율을 계산해줘, 약 200개 웨이퍼 데이터 사용, 주석은 달지 말고"라고 하면 코드를 바로 생성한다. 이어 "3차원 그래프로 시각화해서 잘 보이게 해줘"라고 추가하면 그에 맞게 3D 플롯을 만든다. 화자는 논문 쓸 때 플로팅에 머리 싸매던 걸 회상하며 자동 생성됨을 언급한다. 재생하면 스케일 조절도 되고, 주석 생성 기능을 누르면 코드에 주석도 자동으로 달아준다.
> "아, 진짜 너무 편합니다." [04:34]

### [04:34–05:17] Simulink 소개와 AEB 예제 예고
이번엔 Simulink. MATLAB에서 쓰는 시스템 설계 도구다. 화자는 [의견] Simulink를 "바이브 코딩의 아버지 같은 존재"라 부르며, 20년 전부터 블록을 연결만 하면 전체 시스템이 돌아가는 모델링을 할 수 있었다고 설명한다. 이번 시뮬레이션 대상은 자율주행 차량의 긴급 제동 시스템(AEB로 지칭). 센서에서 시그널이 들어오면 특정 디시전 로직에 따라 브레이크를 자동으로 걸며, 전방 차량이 너무 가까워지면 자동 제동한다. 이 제동 판단 로직을 MATLAB Copilot으로 만들어 본다.

### [05:17–06:03] AEB 테스트 벤치 열기
MATLAB의 "Autonomous Emergency Braking with Sensor Fusion" 예제를 command로 열어 테스트한다. 프로젝트에 직접 안 들어가도 실행 방법이 안내되어 있어 그대로 실행하면 AEB 테스트 벤치가 바로 뜬다. 화자는 이걸 다시 "바이브 코딩의 시초"라 표현한다 [의견]. AEB 컨트롤러가 어떻게 동작하고 브레이킹 입출력을 어떻게 내는지 실험할 수 있다. Copilot에게 요청: 전방 장애물까지 거리(distance to obstacle) 수치가 20보다 작으면 브레이크 시그널 -5를 주고, 그렇지 않으면 0을 준다.

### [06:03–06:50] 기존 로직을 대체할 블록 추가
AEB 테스트 벤치 전체 구성과 그 안의 AEB 컨트롤러를 보여준다. 화면의 브레이킹 컨트롤러(브레이크를 동작시키는 컨트롤러) 안에 기존 AEB 브레이크 판단 로직이 있고, 이 부분을 Copilot으로 대체하려는 것이다. 블록을 하나 추가해 "my AEB logic"이라 이름 붙인다. 기존 블록들의 입력이 쭉 있고 출력이 있으니, 그 입력·출력을 맞춰 원하는 대로 만들어 달라고 하면 된다.

### [06:50–07:20] 로직 코드 요청·생성
Copilot에 이렇게 입력한다 — "Simulink의 MATLAB Function 블록에 넣을 간단한 AEB 로직 MATLAB 코드를 작성해줘." 그러면서 원래 있던 입력들과 출력을 적어주고, 로직이 어떻게 되어야 하는지 조건들을 다 명시한다. 실행하면 그에 맞는 로직을 세팅해 준다.

### [07:20–07:57] 블록 연결(대체)
생성된 로직이 나오면 기존 것과 연결·대체해야 한다. 해당 부분들을 지우고 연결을 싹 해준 뒤 제대로 연결됐는지 확인한다. TTC(time-to-collision)에 해당하는 부분, PB1도 원하는 대로 한 다음 블록을 연결한다. 이렇게 하면 원하는 형태의 실험이나 모델을 바로 적용하기 좋다. 화자가 꼽는 핵심 포인트: Simulink와 Copilot이 이 모델의 움직임을 다 이해하고 있다는 것 [의견].

### [07:57–08:38] 데이터 타입 점검과 시뮬레이션 실행
블록 크기를 크게 조절할 수도 있고, 제대로 세팅됐는지 체크한다. 데이터 타입이 안 맞으면 command로 안내가 뜨고, 그에 맞게 다시 세팅이 나온다. 이를 반영·실행해 설정을 따르고 세팅 테스트를 다시 하면 "별 이상 없네"가 된다. 테스트 시나리오를 실행 — 시뮬레이션 탭에서 전체 시스템을 돌리면 매트릭이 시간에 따라 운영되는 걸 볼 수 있다. 버드아이(bird's-eye) 조감도 스코프를 켜고 실행하면 전방 차량이 근처에 있을 때 어떻게 되는지 시뮬레이션되는 걸 확인할 수 있다.

### [08:38–09:28] MATLAB MCP 연동의 의미
한 단계 더 가서, 최근 MATLAB에 MCP(Model Context Protocol) 기반 연동이 추가됐다. MCP는 Claude Code나 VS Code의 GitHub Copilot 같은 외부 에이전틱 AI 도구가 표준 방식으로 외부 도구를 직접 호출할 수 있게 해주는 프로토콜이다 [불명확: 자막 "안대공화 채널"은 채널명 오인식으로 추정]. 그동안 LLM이 코드를 만들어 주는 역할에 가까웠다면, MCP가 붙은 MATLAB에서는 AI 에이전트가 실제 MATLAB 워크플로우를 실행하고 그 결과를 받아 다음 추론까지 이어갈 수 있다.
> "이게 왜 중요하냐면 매틀랩이 단순 코드 편집기가 아니라 수치 계산이라든가 신호 처리, 모델 피팅, 검증 체계까지 갖춘 엔지니어링 실행 플랫폼이기 때문" [09:11]
그래서 AI가 MATLAB을 도와주는 대상이 아니라 직접 사용하는 계산 엔진으로 다룰 수 있게 됐다는 것 [의견].

### [09:28–10:12] MCP 설치 절차
MathWorks 사이트에 MATLAB MCP 관련 안내가 있고, GitHub에 연결해 설치할 수 있게 되어 있다. GitHub 다운로드로 가서 latest release 부분에서 가장 최근 MATLAB MCP 코어 서버를 눌러 설치하면 Claude에 연결되어 MATLAB을 설치할 수 있게 나온다. MATLAB이 MCP로 액세스 가능하도록 세팅되는 것. "클로드에서 직접 매트랩 연결해봐" 식으로 왔다 갔다 하며, 보통 MCP 설치하듯 진행한다. MATLAB MCP 코어 서버 연결 창이 뜨고 허용을 누르면 자동으로 MATLAB이 연결돼 뜬다.

### [10:12–11:08] Claude에서 MATLAB 실행
Claude에서 보면 MATLAB과 툴박스 아홉 개가 전부 인식된다. 이제 Claude에서 MATLAB을 향한 명령을 다 실행할 수 있다. 예로 "예제 데이터를 불러와서 기본 시각화"(MATLAB이 가진 데이터 활용), "가장 큰 피크 다섯 개를 구해 표로 정리하고 MATLAB에서 계산·실행하도록 해라"라고 하면 중간에 권한 확인 후 실행된다. 왼쪽에서 선스팟(sunspot, 태양 흑점) 데이터가 자동 로딩·분석되고, 여러 AI 파운데이션 모델을 가져와 연결할 수도 있다. 데이터는 요약해 주지만 왼쪽에는 실제 결과가 다 생성돼 있어 MATLAB에서 바로 운용 가능하다. 선스팟 픽스(figure) PNG를 저장했다고 하니 열어 보면 MATLAB에 있는 것들을 만들어 준 것을 확인할 수 있다.

### [11:08–11:47] 마무리 — 조수이되 검토는 필수
정리하면 MATLAB Copilot은 단순히 코드만 짜주는 AI가 아니라 반도체 데이터 분석부터 자율주행 시스템 설계까지, MATLAB이라는 거대한 엔지니어링 생태계 전반에서 활용되는 스마트 어시스턴트 조수라고 본다 [의견]. 다만 Copilot이 낸 결과를 그대로 쓰는 게 아니라 엔지니어의 전문 지식으로 검토하고 발전시키는 과정이 반드시 필요하다고 못 박는다. 그럼에도 복잡한 아이디어를 현실로 만드는 과정의 장벽을 허물고 빠르게 하면서 더 중요한 본질에 집중하도록 돕는다는 것, 그리고 AI가 기존 개발 툴 전반으로 퍼지며 개발·연구·검증 분야에서도 변화가 일고 있다는 것을 알 수 있는 포인트라며 영상을 맺는다.
