> 정리 방식: 시간순 심층 워크스루 (paraphrase) + 주제별 상세 요약

## 요약 (주제별)

### Replit Agent 4의 핵심 컨셉 — 코딩과 디자인의 결합
- Agent 4는 "코드 몇 줄 더 잘 써 주는 수준"이 아니라, 디자인 탐색 → 결과물 생성 → 기능 확장이라는 흐름 자체를 하나로 묶으려는 방향이라고 화자는 설명한다. [의견] 코딩과 디자인이 완전히 따로 놀지 않고 결합된 형태라는 점을 반복 강조.
- 프롬프트를 던져 단일 결과 하나만 받는 데서 그치지 않고, 그 결과를 바탕으로 바로 디자인 방향을 더 탐색할 수 있다는 것이 가장 큰 차이라고 본다.

### Infinite Canvas (인피닛 캔버스) — 여러 시안을 한눈에
- Agent 4에는 "패널 에이전트"를 쓰거나 "인피닛 캔버스에서 디자인한다"는 개념이 있다. 캔버스는 도화지가 무한대로 펼쳐진 형태로, 같은 주제(예: 가족 여행 앱)에 대해 여러 디자인 시안을 나란히 만들어 한눈에 비교할 수 있다.
- 캔버스 안에는 웹페이지, PPT, 이전에 만든 결과물 등 여러 아티팩트가 함께 놓인다. 특정 시안을 클릭해 거기에 코멘트를 날리거나, 그 지점에서부터 이어서 작업을 시작할 수 있다. 드로우(그리기)도 가능해 협업 도구로도 쓸 수 있다.
- [의견] Agent 4의 "가장 핵심 기능"은 결국 이 부분 — 하나의 정답만 받는 게 아니라 여러 시안을 빠르게 비교하며 방향을 고르는 것.

### 서브 에이전트 병렬 실행
- 히어로 섹션을 4가지 버전으로 제안해 달라고 하면, 4개의 서브 에이전트가 동시에 시작해 각각을 병렬로 빌드한다. 예로 "가족 진화형 히어로", "미니멀형 히어로" 등 각기 다른 방향을 동시에 출력해 비교하게 해 준다.
- 슬라이드 작업에서도 순차(sequential) 처리와 서브 에이전트 병렬 처리를 상황에 맞게 알아서 섞어 돌리는 모습이 보인다.

### 데모 1 — AI 여행 일정 앱 랜딩 페이지
- 프롬프트: "가족 여행 계획을 쉽게 짜주는 AI 여행 일정 앱의 랜딩 페이지를 만들어 줘 — 히어로 섹션, 주요 기능 3개, 사용 예시, CTA 포함, 모바일에서도 깔끔하게."
- 에이전트가 React와 Vite로 빌드하겠다고 하며 스스로 디자인 베리에이션을 만들어 캔버스에 배치. 결과물은 "트리메이트 AI"라는 브랜드로, "온 가족이 함께 더 특별한 여행", "무료로 시작하기" 등의 카피가 들어간 랜딩 페이지가 빠르게 생성됨.
- 히어로 4개 버전 중 4번을 골라 랜딩 페이지에 반영하고 전체 톤을 더 세련되게 다듬어 달라고 요청 → 에이전트가 각 파일을 업데이트하며 CSS 색상 변수를 교체, 폰트까지 맞춰 실시간으로 반영되는 모습을 확인. 데스크탑/모바일(아이폰 17 프로 맥스, 갤럭시 S25 울트라 등) 각 화면에서 어떻게 보이는지 바로 확인 가능.

### 데모 2 — 같은 프로젝트 안에서 슬라이드(PPT) 생성
- "프로젝트를 소개하는 다섯 장짜리 슬라이드를 만들어 줘 — 일반 사용자가 이해하기 쉬운 소개 자료 느낌으로" 요청.
- Replit이 강조하는 건 "앱만 만드는 게 아니라 프로젝트라는 개념" — 하나의 프로젝트 울타리 안에서 랜딩 페이지, 슬라이드 등 여러 결과물을 만든다. 크리에이티브 방향(브랜드 컬러 등)을 앞서 만든 랜딩 페이지와 동일하게 유지해 같은 톤으로 슬라이드를 생성.
- 캔버스에서 랜딩 페이지 결과물, 히어로 4개 시안, 그리고 지금 만들고 있는 슬라이드까지 함께 확인 가능. 프리뷰로 크게 볼 수도 있음.

### 데모 3 — 기능 단위 분할 확장 (로그인/온보딩, 메인 화면)
- 기능을 하나로 몰지 않고 작업을 나눠 확장. 먼저 로그인 + 온보딩 플로우 요청 → 카카오/구글 소셜 로그인 연동, 4단계 온보딩 형태로 알아서 구성.
- 로그인 페이지를 구현만 하는 게 아니라, 에이전트가 가상 머신 안에서 전체 플로우를 직접 테스트 — 온보딩 시 예산 입력 등에 문제가 없는지 스스로 체크. 화자는 이것 자체가 생산성이라고 평가.
- 이어 "여행 일정 생성 메인 화면"을 별도 요청 → 에이전트가 현재 앱 구조를 파악하고 온보딩에서 입력된 정보를 자동 반영해 일정을 만들어 준다. "인증 가드가 정상 작동 중"임을 확인하고 플래너 접근, AI 일정 생성 버튼으로 실제 일정이 생성되는 것까지 확인. 결과물의 자체 검증도 에이전트가 수행.
- [의견] 기능을 단위로 나누면 나중에 task처럼 관리하거나 더 복잡한 프로젝트로 확장할 때 캔버스로 흐름을 잡기 편하다.

### 화자의 종합 평가
- [의견] Agent 4의 핵심은 "결과물 하나를 빨리 만든다"가 아니라, 캔버스에 여러 결과물을 펼쳐 디자인 방향을 탐색하고 → 같은 프로젝트 안에서 다른 결과물까지 만들고 → 기능을 나눠 점점 확장하는 흐름이 예전보다 훨씬 자연스러워진 점. 그래서 지금 호평이 많다고 본다.
- [의견] 단순 "코딩 보조 툴"로 보기엔 AI가 에이전틱하게 처리하는 범위가 넓어져, 창의적 작업이 실제 결과물로 이어지는 시간을 줄여 주는 작업 환경으로 볼 만하다며 사용을 권함.

## 3줄 오리엔테이션
- Replit이 새로 출시한 Agent 4를 화자가 직접 써 보며 시연하는 영상 (Replit 협업).
- 랜딩 페이지 → 슬라이드 → 기능 분할 확장의 3단계로, "인피닛 캔버스"에서 여러 디자인 시안을 병렬로 만들고 비교·반영하는 흐름을 보여준다.
- 핵심 메시지: 코딩과 디자인이 결합되고, 하나의 프로젝트 안에서 여러 결과물을 만들며, 창작→결과물 시간을 줄여 주는 작업 환경.

## 전문 — 시간순 워크스루

### [00:00–00:14] 오프닝 훅 — 캔버스에서 디자인 탐색
초반 티저로, 처음 프롬프트를 던져 단일 결과를 받는 데서 더 나아가 그 결과를 바탕으로 바로 디자인 방향을 더 탐색할 수 있다는 점을 강조한다.
> "코딩하는 것과 디자인이 결합된 형태라는 겁니다." [00:10]

### [00:15–00:30] 인사와 Replit 소개
화자(에러/안될공학)가 인사하며 Replit을 소개한다. 원래는 브라우저 안에서 바로 개발할 수 있는 툴로 알려졌고, 최근에는 AI 에이전트 쪽으로 밀고 있었는데, 이번에 Agent 4를 출시했다고 전한다. Agent 4는 코드 몇 줄 더 잘 써 주는 수준이 아니라, 디자인을 탐색하고 서치해 결과물을 만들고 기능을 확장하는 흐름 자체를 훨씬 자연스럽게 묶으려는 게 보인다고 평한다. [의견]

### [00:30–01:01] 오늘 다룰 3가지
직접 테스트하며 짧게 세 가지를 볼 것이라고 예고한다. (1) 웹페이지(랜딩 페이지)를 간단히 만들고, (2) 같은 프로젝트 안에서 슬라이드까지 만든 뒤, (3) 기능을 나눠 확장을 해 본다. 시청자도 바로 따라 쓸 수 있다고 안내.

### [01:01–01:31] Replit 진입 화면과 첫 프롬프트
로그인한 Replit 화면에는 웹사이트, 모바일, 디자인, 슬라이드 애니메이션, 3D 게임 등을 선택해 하나의 플로우 안에서 통합 처리할 수 있게 돼 있다. 예로 "가족 여행 계획을 쉽게 짜주는 AI 여행 일정 앱의 랜딩 페이지를 만들어 줘 — 히어로 섹션, 주요 기능 3개, 사용 예시, CTA까지, 모바일에서도 깔끔하게"라는 프롬프트를 넣고 실행한다.

### [01:31–01:52] React + Vite 자동 빌드, 캔버스 개념 등장
AI 워크 플랫폼이 에이전틱하게 알아서 처리하다 보니, "React랑 Vite로 빌드하겠습니다" 하면서 스스로 디자인 베리에이션을 만들어 캔버스에서 웹페이지를 만든다. Agent 4에서는 패널 에이전트를 쓰거나 "인피닛 캔버스에서 디자인한다"는 개념이 있다고 설명.

### [01:52–02:31] 인피닛 캔버스가 뭔가
보통은 프로젝트에서 하나하나 따로 작업하지만, 인피닛 캔버스는 도화지가 무한대로 펼쳐진 것이다. 같은 "A의 가족 여행"이라도 여러 디자인 시안을 만들어 한눈에 볼 수 있다. 지금 보이는 게 그 캔버스로, 웹페이지도 있고 PPT를 만든 것도 있으며, 아래로 내려가면 이전에 만든 여러 시안이 놓여 있다. 캔버스 안에서 여러 테스트를 해 볼 수 있다는 것이 요지.

### [02:31–03:02] 코딩+디자인 결합, 첫 결과물
다시 핵심을 강조 — 처음 프롬프트로 단위 결과를 딱 받는 데서 더 나아가 이를 바탕으로 디자인 방향을 탐색할 수 있는, 코딩과 디자인이 결합된 형태라는 것. 결과물로 "트리메이트 AI"라는 브랜드가 나오고 "온 가족이 함께 더 특별한 여행", "무료로 시작하기" 같은 카피가 들어간 랜딩 페이지가 예쁘게 빠르게 생성된다. [의견] "너무나도 예쁘게 만들어 주지 않습니까?" [03:02]

### [03:02–03:31] 히어로 4버전 제안 요청
"히어로 섹션 디자인을 네 가지 버전으로 제안해 달라"고 하며 여러 형태를 만들어 캔버스에 담아 비교. Agent 4의 가장 핵심 기능이 결국 이 부분이라고 정리한다. [의견] 하나의 정답만 받는 게 아니라 여러 시안을 빠르게 비교하며 방향을 고르는 것 — 최종본만 찍어내는 게 아니라 캔버스 안에서 디자인 방향을 탐색하며 맞춰가는 과정.

### [03:31–04:08] 서브 에이전트 병렬 빌드
캔버스를 눌러 보면 기존 것뿐 아니라 새로운 것들이 만들어지고 있다. 방금 요청한 히어로 4가지를 동시 다발적으로 빌딩 중 — 서브 에이전트가 동시에 시작해 각각(가족 진화형 히어로, 미니멀형 히어로 등)을 병렬로 출력하고 비교하게 해 준다. 다 되면 4개 서브 에이전트 결과가 나란히 나와 바로 비교 가능.

### [04:08–04:31] 4번 버전 선택·반영 요청
4번째 버전을 골라 "히어로 섹션 4번 버전을 랜딩 페이지에 반영하고 전체 분위기를 더 세련되게 다듬어 달라"고 요청. 시안을 보고 끝나는 게 아니라 마음에 드는 방향을 골라 실제 앱 결과물로 이어갈 수 있다는 점 — 디자인 탐색과 웹페이지 구현이 완전히 따로 놀지 않는다는 개념.

### [04:31–04:48] 캔버스 위 협업·코멘트
캔버스에 결과물이 계속 퍼지므로, 특정 지점에서 시작하거나 클릭해 해당 요소에 코멘트를 날릴 수 있다. 드로우도 가능해, 실제 협업 시에도 작업 공간 기준으로 무언가를 할 수 있다.

### [04:48–05:19] 4번 반영 실시간 확인
4번 캐주얼 디자인을 랜딩 페이지에 반영하기 위해 각 파일을 업데이트 — 비슷한 톤으로 테마를 맞춰 CSS 색상 변수를 교체·변형하겠다는 내용이 확인된다. 이미 랜딩 페이지에 실시간 반영되는 모습이 보이고, 폰트까지 알아서 업데이트해 준다.

### [05:19–05:50] 데스크탑·모바일 반응형 확인
모바일에 맞춰서도 잘 나오도록 코멘트했기 때문에, 웹페이지로 만든 뒤 나중에 모바일로 만들어 달라고 할 수도 있음을 언급. "데스크탑·모바일 모두 완벽하게 나오겠습니다" 하며 풀 사이즈, 아이폰 17 프로 맥스, 갤럭시 S25 울트라 등 각 화면에서 어떻게 보이는지 바로 확인한다. [의견] "너무 좋지 않습니까?" [05:50]

### [05:50–06:20] 슬라이드(PPT) 생성 요청
랜딩 페이지를 만들어 놓고, 그 토대와 느낌 그대로 이어 PPT 슬라이드를 만든다. "프로젝트를 소개하는 다섯 장짜리 슬라이드를 만들어 줘 — 각 슬라이드에 어떤 내용을 담을지, 일반 사용자가 이해하기 쉬운 소개 자료 느낌으로"라고 요청. Replit이 말하는 건 앱만 만드는 게 아니라 "프로젝트"라는 개념으로, 하나의 울타리 안에서 원하는 여러 결과물을 만들어 간다는 것.

### [06:20–07:03] 같은 문맥·같은 브랜드로 슬라이드 빌드
보통은 앱 따로, 각 형태를 따로 테스트하며 가지만, 여기서는 같은 문맥 안에서 다 처리한다. 크리에이티브 방향도 알아서 동일한 브랜드 컬러를 유지 — 앞서 만든 랜딩 페이지처럼 슬라이드 아티팩트를 서서히 만들며 이어간다. 프리뷰 말고 캔버스로 보면 랜딩 페이지, 히어로 4가지 시안, 그리고 지금 만드는 슬라이드 부분까지 함께 확인 가능. 프리뷰로 크게 볼 수도 있음.

### [07:03–07:21] 순차 + 병렬 혼합, 슬라이드 완성
각각을 시퀀셜하게 진행하는 것도 있지만, 적절히 알아서 서브 에이전트를 여러 개 굴리며 처리하는 모습도 보인다. 슬라이드가 나오자 화자는 한 번에 해결해 주니 만족스럽다고 하며, 같은 톤으로 계속 맞춰 간다고 정리.

### [07:21–07:48] 기능 분할 — 로그인·온보딩 플로우
이제부터는 기능을 하나로 몰지 않고 작업을 나눠 확장. 먼저 로그인·온보딩 플로우를 만드는데, 카카오나 구글 소셜 로그인을 연동하고 4단계 온보딩 형태로 알아서 연결해 만든다. 랜딩 페이지에서 왔다 갔다 계산하며 세부 내용이 업데이트되는 모습이 보인다.

### [07:48–08:15] 로그인 페이지와 가상 머신 테스트
프리뷰에서 로그인 부분이 예쁘게 만들어진 것을 확인 — 카카오·구글 시작 버튼, 홈으로 돌아가기 등. "무료로 시작하기"를 누르면 바로 로그인으로 이어진다. 중요한 건 로그인 페이지를 구현만 하는 게 아니라, 에이전트가 가상 머신 안에서 전체 플로우를 실제로 테스트한다는 점.

### [08:15–08:31] 온보딩 자체 검증 = 생산성
온보딩에서 "얼마를 쓰니, 예산이 얼마니" 같은 입력을 했을 때 문제가 없는지 에이전트가 직접 체크하는 모습이 보인다. 화자는 이것 자체가 생산성이라고 평가. 이어 "여행 일정 생성 메인 화면"을 별도로 요청.

### [08:31–09:16] 메인 화면 — 온보딩 정보 자동 반영
기능을 단위로 나누면 나중에 task처럼 관리하거나 복잡한 프로젝트로 확장할 때 캔버스로 흐름을 잡기 편하다. [의견] "여행 일정 생성 메인 화면을 만들어 줘"라고 하니 현재 앱 구조를 파악하고 온보딩에서 입력한 정보를 자동 반영해 일정을 만들어 준다. 대가족 여부 등 정보를 넣으면 그 토대로 구성. "첫 여행 일정 만들러 가기"를 누르면 일정이 나오며 웹페이지를 만들어 간다 — 고퀄의 웹페이지를 빠르고 예쁘게 원하는 대로 만들 수 있다고 평가.

### [09:16–09:51] 인증 가드·일정 생성 확인
"인증 가드가 정상 작동 중"임을 확인하고 플래너에 접근 → 만들어진 것이 실시간으로 보인다. 여행 인원, "AI 일정 생성하기"를 누르면 알아서 일정을 생성. [의견] "이거 내가 만든 건지 얘가 만든 건지 너무 잘 만들었다." AI가 자기들이 알아서 검증도 해 준다. 이런 걸 실제로 바이브 코딩하면 재미있을 것 같다고 소감.

### [09:51–10:33] 마무리 정리와 평가
정리 — Agent 4에서 중요한 건 결과물 하나를 빨리 만든다가 아니라, 캔버스에 여러 결과물을 펼쳐 디자인 방향을 탐색하고, 같은 프로젝트 안에서 다른 결과물까지 만들고, 기능을 나눠 점점 확장해 나가는 흐름이 예전보다 훨씬 자연스러워진 점이라고 본다. [의견] 그래서 지금 호평이 많다. 단순 코딩 보조 툴로 보기엔 AI가 워낙 빠르게 에이전틱하게 발전하고 있어, 창의적 작업이 실제 결과물로 이어지는 시간을 줄여 주는 작업 환경으로 보며 사용을 권한다. "지금까지 에러였습니다."
