# 모든 AI 병목에 NVIDIA가 있다 | 젠슨황 키노트 발표에 AI 투자의 흐름이 보인다

> unrealtech · 2026-06-04 · [영상 링크](https://youtu.be/zOUUFrMGwEg)
> 정리 방식: 시간순 심층 워크스루 (paraphrase) + 주제별 상세 요약
> 검증 배너: NVIDIA GTC 타이페이 키노트 해설 영상. 성능 배수·처리량 수치는 대부분 엔비디아/케이던스 등 벤더 발표를 화자가 옮긴 것이며 독립 검증되지 않았다 (주장/미검증). 화자의 해석·전망은 [의견]으로 분리.

## 요약 (주제별)

### 이번 키노트의 한 줄 메시지 — "compute is revenue"
- 화자 해석으로는, GTC 타이페이 키노트의 핵심은 "지표(성능)가 더 빨라졌다"가 아니라 **AI가 이제 돈을 쓰는 기술이 아니라 돈을 벌어오는 인프라가 되었다**는 것. [의견]
- 키노트 첫 장면이 "AI가 돈을 벌기 시작했다"는 그래프였고, 이것이 발표 전체의 톤을 결정했다는 게 화자의 진단. [의견]
- 예전에는 서버를 사면 비용(돈이 나감)이었지만, AI 서비스에서는 서버가 돌아갈수록 토큰이 나오고 → 사용량이 늘고 → 매출이 생긴다는 논리. 컴퓨터가 "비싼 계산기"에서 "매출을 찍어내는 기계"가 되었다는 것. (화자가 정리한 키노트 논지)
- 결론적으로 AI 경쟁이 "모델 성능표" 밖으로 나왔고, 싸움은 **누가 더 싸게·더 많이·더 안정적으로 AI가 돈을 벌게 만드느냐**로 넘어가고 있다는 해석. [의견]

### AI 팩토리 개념 — 데이터 센터와 무엇이 다른가
- 젠슨황이 계속 쓰는 "AI 팩토리"는 데이터를 저장하는 창고(기존 데이터 센터: 검색 처리·영상 스트림·클라우드)가 아니라, **토큰과 결과물을 찍어내는 공장**에 가깝다는 개념. (키노트 논지)
- 질문 → AI가 답변 생성·코드 작성·자료 검색·시뮬레이션·문서 분석으로 업무를 대신 끝냄 → 이 결과물이 서비스 매출로 이어짐. 그래서 GPU는 "부품 하나"가 아니라 공장의 핵심 생산 설비.
- 다만 공장은 기계 하나로 안 돌아간다: 전기·물류·작업자·운영 시스템이 필요하듯, AI 팩토리도 CPU가 받쳐주고 네트워크가 안 막히고 스토리지·보안·소프트웨어가 전체를 관리해야 함. 키노트의 핵심 메시지는 **이 "돈 버는 공장" 전체를 엔비디아 방식으로 다 설계했다**는 것. (화자 정리)

### 에이전트의 정의 — LLM + 하네스(harness)
- 젠슨황은 에이전트를 "챗봇의 다음 버전" 정도로 보면 안 되고, **LLM과 하네스가 합쳐진 것**으로 정의. 화자는 이게 키노트 전체를 이해하는 핵심 키라고 봄. [의견]
- LLM = 두뇌(말 이해·추론·코드 생성·계획 수립). 하네스 = 주변 장치(메모리, 툴 호출, 권한 관리, 코드 실행 환경/샌드박스 관리, 검색, 검증 루프). 이 모든 게 붙어야 단순 답변하는 챗봇이 아니라 "전체 일을 돌리는 에이전트"가 된다.
- 엔비디아 설명에 따르면 에이전틱 AI의 하나의 프롬프트는 "prompt → retriever → tool use → response generation"으로 이어지는 **수천 단계의 여정**이 될 수 있다. 그래서 병목이 GPU 하나에만 있지 않게 됐다는 게 논지의 전환점.

### 케이던스(Cadence) 칩 설계 에이전트 사례
- 케이던스가 **칩스택 AI 슈퍼 에이전트**를 발표. 단순히 칩 설계를 돕는 AI가 아니라, 오픈 셸(open shell — [불명확: 원문 "오픈 셰"])로 에이전트를 보호하고 칩 설계와 검증 워크플로우를 **자율적으로 실행**한다고. (엔비디아 발표 인용)
- 엔비디아가 이 칩스택의 **첫 고객**으로서 자사 칩 설계 검증에 사용한다고 밝힘. 보도에 따르면 수백만 개의 자동화 테스트를 돌리며, **칩 검증 시간을 5주 → 하루 미만**으로 줄일 수 있다고 주장. (주장/미검증)
- 화자 해석: AI 에이전트가 "이메일 써주는 수준"을 넘어 **반도체 설계 검증 같은 고부가 엔지니어링 업무**로 들어가고 있음을 보여주는 핵심 장면. 다만 실제 산업 현장에서 어느 범위까지 적용될지는 두고 봐야 함. [의견]

### 병목 이동 (1) CPU — 베라(Vera) CPU
- AI가 답변만 만들 땐 GPU가 핵심 같지만, AI가 "일"을 하기 시작하면 GPU 밖에서 큰 일이 벌어짐: 파이썬 런타임 구동, 샌드박스 생성, DB 조회, 외부 툴 호출, 에이전트 간 프로세스 조율 — 상당수를 CPU가 처리. GPU가 아무리 빨라도 CPU가 데이터를 준비 못 하면 GPU가 기다리게 되고, 이는 공장 기계가 멈춘 것과 같음.
- 그래서 엔비디아는 **AI 에이전트를 위해 설계된 첫 CPU "베라"**를 내놓음. 스펙(화자가 나열): 88개 코어(spatial multithreading 언급 — [불명확: 자동자막 음차]), 최대 **1.2 TB/s LPDDR5X 메모리 대역폭**, NVLink C2C 기반 CPU-GPU 코히어런트 대역폭. (주장/미검증)
- 성능 발표: 베라 CPU는 **X86 CPU 대비 에이전틱 워크로드에서 1.8배 빠른 태스크 컴플리션**. 베라 루빈은 그레이스 블랙웰 대비 **에이전트 처리량(throughput) 10배**. (주장/미검증)
- 핵심 취지: 수많은 에이전트가 동시에 코드 실행·툴 호출·데이터 이동·작업 조율을 할 때 **CPU가 GPU의 발목을 잡지 않게** 하겠다는 것. 특히 에이전트 오케스트레이션 프레임워크/런타임은 파이썬 기반이 많은데, 인터프리터 방식이라 한 줄씩 처리하는 오버헤드·컨텍스트 전환·멀티스레딩 제약·메모리 접근 병목이 생김. 베라가 파이썬의 구조적 한계를 다 없앤 건 아니지만, 더 많은 작업을 병렬 처리하고 메모리 병목을 줄여 GPU에 데이터를 끊김 없이 공급하도록 설계됐다는 점이 포인트. (화자 정리)
- 화자 해석: 베라는 "엔비디아가 CPU 시장도 먹겠다"가 아니라 **GPU가 돈을 벌도록 주변 병목을 제거하는 CPU**로 접근한 것. GPU를 대체하려는 게 아니라 GPU가 더 오래·더 바쁘게·더 수익성 있게 일하게 만드는 장치. [의견]

### 상징적 사례 — 뉴욕 증권 거래소(NYSE)
- 엔비디아 발표에 따르면 NYSE가 레드햇(Red Hat — [불명확: 원문 "레드 판다"])·HP와 함께 베라 CPU를 활용해 고성능·저지연 "AI-ready 마켓 인프라"를 확장하려 한다고. (엔비디아 발표 인용)
- NYSE가 **하루 1.1조 개 이상의 메시지**를 처리한다고 언급. (주장/미검증)
- 화자 해석: 단순히 "금융권 고객 확보"가 아니라, 초저지연 인프라(데이터 밀리면 안 되고 레이턴시 커지면 안 되는 곳)에서 시스템이 안정적으로 도는 핵심이 베라 CPU라는 것 — 베라가 GPU 옆 조연이 아니라 AI 팩토리의 데이터 흐름을 붙잡는 조율자 역할을 노린다는 해석. [의견]

### 병목 이동 (2) 네트워크 — 스펙트럼 X 이더넷 포토닉스 / CPO
- AI 팩토리가 수십만~수백만 GPU 규모로 커지면 GPU끼리 대화해야 함(학습·추론·스토리지 접근 시 데이터가 계속 오감). 기존 방식은 전력·발열·신호 감쇠·지연 시간 같은 물리 법칙 한계에 부딪힘.
- 해법으로 **스펙트럼 X 이더넷 포토닉스**와 **CPO(co-packaged optics, 코패키지 옵틱스)**가 다시 주목. CPO는 광 모듈을 스위치 칩에 훨씬 가깝게 붙이는 방식 — 전기 신호가 멀리 이동할수록 전력을 더 먹고 열도 나고 신호도 약해지므로, 전기 신호 이동 거리를 줄이고 더 빨리 빛(광신호)으로 바꿔버림.
- 엔비디아는 베라 루빈 플랫폼에서 스펙트럼 X 이더넷 포토닉스를 도입한다고 발표. 기존 트랜시버 기반 네트워크 대비 **전력 효율 5배, AI 업타임 5배, 디플로이먼트 타임 1.3배** 개선을 언급. (주장/미검증)
- 화자 해석: 단순 네트워크 장비 발표가 아니라, AI 팩토리가 커질수록 네트워크가 "주변 장치"가 아니라 공장 내부의 빠르고 섬세한 신경망이 된다는 것. [의견]

### 병목 이동 (3) 보안 — 블루필드(BlueField)
- 보안은 대중적으로 덜 화려하지만 엔터프라이즈 기준 매우 중요. 에이전트는 챗봇과 달리 회사 시스템 안에 들어가 파일 열고, DB 조회하고, 컨피덴셜 정보 보고, API 호출하고, 코드 만들어 실행하고, 권한 있는 작업까지 수행함 → 그 순간 권한 탈취·민감 데이터 유출 위험 발생. "에이전트가 사람 대신 일할 수 있다 = 사람 대신 사고도 칠 수 있다." (화자 표현)
- 그래서 **블루필드 4**(BlueField-4, [불명확: 원문 "블루필드 4 STX"]) 같은 장치가 중요. 블루필드 계열은 단순 데이터 전송 가속 장치가 아니라, 데이터가 오가는 길목에서 보안·격리를 담당. 메인 CPU/OS가 다 감시하는 방식으로는 부족할 수 있어, AI 에이전트가 파일·네트워크를 오가는 관문에서 하드웨어적으로 감시·통제할 필요가 생긴다는 논지.

### 전체 운영 시스템 — DGX(?) 와 스택 구성
- 화자 정리: GPU는 두뇌를 돌리고, 베라 CPU는 일을 조율하고, 스펙트럼 X는 신경망을 깔고, 블루필드는 데이터 관문을 지키며, 이 전체 공장을 운영하는 시스템이 **DGX**([불명확: 원문 "DSX"])다.
- 이렇게 보면 엔비디아가 왜 GPU 말고 이런 것까지 만드는지 이해된다 — AI가 돈을 벌기 시작하면 그 돈 버는 흐름의 **모든 병목을 잡고 싶어 한다**는 것. [의견]

### 확장 (1) PC — RTX 스파크(DGX Spark)
- 젠슨황은 흐름을 클라우드/데이터 센터에서 PC로도 확장. **RTX 스파크**([불명확: 원문 "RTX 스파크", DGX Spark 계열로 추정])는 AI 에이전트를 개인 PC로 끌어내리려는 방향. 모든 에이전트가 클라우드에서만 돌 필요는 없고, 개인 데이터·로컬 파일·프라이버시가 중요한 작업은 PC 안에서 도는 게 유리할 수 있음.
- 키노트 인용: "Nvidia is reinventing all of PC ... announcing a whole new line. Three revolutionary Windows machines covering desktop, laptop and workstations. All 100% Windows compatible, 100% Nvidia AI runs on these ..." (원문 인용, 자동자막 일부 불완전)
- 로이터 보도 인용: RTX 스파크는 **마이크로소프트 협력·미디어텍(MediaTek) 개발**로 나왔고, 델·HP·레노보·에이수스·마이크로소프트 등 PC 생태계와 연결될 예정. (보도 인용)
- 화자 해석: 기존 PC 선택 기준(CPU 성능·그래픽·배터리)에서 앞으로는 "내 PC가 얼마나 강력한 로컬 에이전트를 돌릴 수 있느냐"가 중요한 기준으로 나올 수 있음. [의견]

### 확장 (2) 로봇 — 아이작 그루트 / 유니트리 H2 플러스
- 엔비디아가 **아이작 그루트(Isaac GR00T) 레퍼런스 휴머노이드 로봇**을 발표. 유니트리 H2 플러스 휴머노이드 바디 + 샤파(Sharpa — [불명확: 원문 "샤파"])의 파이브핑거드 핸드 + 제슨 토르(Jetson Thor) 온보드 컴퓨터 + 아이작 그루트 소프트웨어 스택을 결합한 **오픈 레퍼런스 디자인**.
- 의미: 엔비디아가 로봇 완제품 회사를 하겠다는 게 아니라 레퍼런스라는 것. 몸은 유니트리 같은 회사, 손은 샤파 같은 회사가 만들고, **엔비디아는 그 위에 두뇌와 신경계를 잡음**(제슨 토르=온보드 컴퓨터, 아이작 그루트=로봇이 보고·이해하고·행동하게 만드는 소프트웨어/모델 스택).
- 발표 직후 유니트리는 IPO를 한 것으로 알려졌고, 휴머노이드 로봇 시장에서 빠르게 성장하는 회사로 주목받는 상황. (화자 언급)
- 화자 주의: 레퍼런스 디자인 = 대량 상용화 성공은 아님. 그러나 엔비디아가 로봇의 몸을 직접 만들기보다 **로봇 산업의 두뇌·신경계 개발 플랫폼**을 대령하고 있다는 포인트는 중요. [의견]

### 결론과 앞으로 볼 포인트
- 키노트 종합 해석: 엔비디아가 "GPU를 버리고 CPU 회사가 된다"가 아니라, **GPU가 너무 중요해졌기 때문에 GPU 주변의 모든 병목을 직접 다 설계하겠다**는 이야기. "AI 팩토리"는 단순 마케팅 용어를 넘어, 토큰 코스트를 극단으로 줄이고 전력당 토큰 성능을 올리며 에이전트 처리량·타임투퍼스트프로덕션 같은 지표를 등장시켜 **AI 인프라를 공장처럼 설계·측정하겠다**는 뜻. [의견]
- 로봇 발표도 "내년에 휴머노이드가 거리·공장에 쏟아진다"는 뜻은 아니지만, 로봇을 움직이는 AI 스택을 누가 장악하느냐의 싸움은 이미 시작. 스마트폰 시대 iOS/안드로이드처럼, 로봇 시대엔 피지컬 AI 플랫폼이 중요해지고 엔비디아가 그 자리를 선점하려 함. [의견]
- 앞으로 볼 포인트(화자 제시): ① 베라 루빈의 실제 양산·공급(칩 하나가 아니라 서버·냉각·전력·네트워크·스토리지·제조가 다 붙어야 함 → GTC 타이페이에서 대만 서플라이 체인이 강조된 이유), ② 베라 CPU/CPO가 NYSE 같은 상징 뉴스 외에 실제 어느 규모로 배치되고 클라우드·엔터프라이즈에서 얼마나 빠르게 채택될지, ③ 케이던스 같은 에이전트 사례를 기업들이 얼마나 신뢰하고 쓸지, ④ CPO·스펙트럼 X 이더넷 포토닉스가 실제 배치에 잘 적용될지.
- 화자의 개인적 결론: "AI가 돈을 벌기 시작하자 컴퓨팅의 의미가 달라졌다. 모델 잘 만드는 회사보다 인프라를 제대로 만드는 게 중요해졌다." 엔비디아는 더 이상 "더 빠른 칩 파는 회사"로 자신을 설명하지 않고, "compute is revenue" 시대에 그 돈이 만들어지는 생산 라인 전체를 설계하는 회사로 포지셔닝. GPU 전쟁으로 시작된 흐름이 이제 "팩토리 전쟁"이 될 수 있고, 그 공장을 누가 설계하느냐가 다음 AI 산업의 질서를 바꿀 수 있다는 전망. [의견]

## 3줄 오리엔테이션
- NVIDIA GTC 타이페이 키노트 해설. 핵심은 "AI가 돈을 벌기 시작했다(compute is revenue)"이며, 데이터 센터를 "AI 팩토리(토큰 찍어내는 공장)"로 재정의한다.
- 젠슨황은 에이전트를 "LLM + 하네스"로 정의하고, 에이전트 워크로드가 GPU 밖으로 병목을 퍼뜨린다고 본다 — 그래서 엔비디아가 CPU(베라)·네트워크(스펙트럼 X/CPO)·보안(블루필드)·운영(DGX)·PC(RTX 스파크)·로봇(아이작 그루트)까지 전부 다시 설계했다.
- 성능 배수·처리량 수치는 대부분 벤더 발표(주장/미검증)이고, 화자의 해석·전망은 [의견]으로 분리해 읽어야 한다.

## 전문 — 시간순 워크스루

### [00:00–01:09] 키노트의 첫 그래프 — "AI가 돈을 벌기 시작했다"
영상은 젠슨황 발언 클립("The computer industry has been completely changed")으로 시작한다. 화자는 "AI가 돈을 벌기 시작했다"는 그래프가 사실상 키노트의 첫 장면을 결정했다고 말한다. 이번 GTC 타이페이 키노트에서 엔비디아가 던진 메시지는 "지표가 더 빨라졌습니다"가 아니라 오히려 그 반대로, **AI는 이제 돈을 쓰는 기술이 아니라 돈을 벌어오는 인프라가 되었다**는 것이었다고 정리한다. 그 목표를 위해 엔비디아는 GPU뿐 아니라 CPU·네트워크·스토리지·보안 소프트웨어·로봇까지 전부 다시 짜는 모습을 숫자로 보여줬다. 나열된 발표: 베라 루빈은 그레이스 블랙웰 대비 에이전트 처리량 10배(주장/미검증), 베라 CPU는 X86 CPU 대비 에이전틱 워크로드에서 1.8배 빠른 태스크 컴플리션(주장/미검증), NYSE는 하루 1.1조 개 이상 메시지 처리 인프라에 베라 CPU 활용 언급, 케이던스는 칩 검증 시간을 5주→하루 미만으로 줄이는 AI 슈퍼 에이전트 사례.

### [01:09–01:40] 두 시간 키노트를 한 문장으로 — 경쟁이 성능표 밖으로
화자는 현장에서 두 시간에 가까운 청사진을 듣고 나니 발표들이 하나의 문장으로 연결됐다고 한다. [의견] 바로 "AI 경쟁은 이제 모델 성능표 밖으로 빠져나왔다"는 것.

> "이제 싸움은 누가 더 싸게 더 많이 더 안정적으로 AI가 돈을 벌게 만들 수 있느냐로 넘어가고 있습니다." [01:34]

### [01:40–02:26] AI 팩토리란 무엇인가 — 창고가 아니라 공장
왜 엔비디아가 계속 "AI 팩토리"라고 말하는지부터 짚는다. 보통 데이터 센터라 하면 서버 가득한 건물에서 검색 처리·영상 스트림·클라우드 서비스를 돌리는 걸 떠올린다. 그런데 젠슨황의 AI 팩토리는 데이터를 저장하는 창고 개념이 아니라, 말 그대로 **토큰과 결과물을 찍어내는 공장**에 가깝다. 질문을 하면 AI가 답변을 만들고 코드를 짜고 자료를 검색한 뒤 시뮬레이션을 돌리고 문서를 분석해 업무를 대신 끝낸다. 이 모든 결과물이 결국 서비스 매출로 이어진다는 것이 이번 키노트의 메시지 — 바로 "compute is revenue".

### [02:26–03:12] compute is revenue — 비용이 아니라 매출
젠슨황 클립 뒤, 화자가 풀이한다. 컴퓨팅이 비용이 아니라 매출이라는 말. 예전에는 서버를 사면 돈이 나갔는데, AI 서비스에서는 서버가 돌아갈수록 토큰이 나오고, 토큰이 나올수록 사용량이 늘고, 사용량이 늘수록 매출이 생긴다는 논리다. 컴퓨터가 "비싼 계산기"에서 매출을 만들어내는 기계로 바뀐 것 — 전기를 넣으면 24시간 쉬지 않고 결과물을 찍어내는 기계. 이 관점에서 GPU는 부품 하나가 아니라 공장의 핵심 생산 설비다.

### [03:12–03:49] 공장은 기계 하나로 안 돈다 — 전체를 엔비디아 방식으로
그런데 공장은 기계 하나로 돌아가지 않는다. 기계가 있어도 전기·물류·작업자·운영 시스템 전반이 필요하다. AI 팩토리에서도 CPU가 받쳐지고, 네트워크가 막히면 안 되고, 스토리지·보안·소프트웨어가 전체 공장을 관리해야 한다. 그래서 이번 키노트의 핵심 메시지는 **돈을 버는 공장 전체를 엔비디아 방식으로 다 설계했다**는 것에 가깝다. 그리고 이 공장을 가장 바쁘게 만들 워크로드로 젠슨황은 "에이전트"를 잡았다.

### [03:49–04:56] 에이전트의 정의 — LLM + 하네스
화자는 에이전트를 "챗봇의 다음 버전" 정도로 보면 안 된다고 강조한다. 젠슨황은 에이전트를 **LLM과 하네스, 두 개가 합쳐진 것**으로 정의했고, 화자는 이 말이 키노트 전체를 이해하는 핵심 키라고 본다. [의견] LLM은 두뇌 — 말을 이해하고 추론하고 코드를 만들고 계획을 세우는 능력. 하지만 두뇌만으론 일을 못 한다. 하네스가 바로 그 주변 장치로, 메모리·툴 호출·권한 관리·코드 실행 환경(샌드박스) 관리·검색·검증 루프 등 이 모든 게 잘 붙어야 단순히 답만 하는 챗봇에서 전체 일을 돌리는 에이전트가 된다.

### [04:56–05:35] 케이던스 칩스택 AI 슈퍼 에이전트
이 구조가 실제로 드러난 곳으로 화자는 케이던스 사례를 핵심으로 꼽는다. [의견] 케이던스는 칩스택 AI 슈퍼 에이전트를 발표했는데, 단순히 칩 설계를 도와주는 AI 정도가 아니다. 엔비디아 키노트에 따르면 케이던스는 오픈 셸([불명확])을 사용해 이 에이전트를 보호하고, 칩 설계와 검증 워크플로우를 자율적으로 실행한다. 심지어 엔비디아가 이 칩스택의 첫 고객으로서 자사 칩 설계 검증에 사용한다고 밝혔다. 보도에 따르면 이 시스템은 수백만 개의 자동화 테스트를 돌리며, 칩 검증 시간을 5주에서 하루 미만으로 줄일 수 있다고 한다. (주장/미검증)

### [05:35–06:09] 에이전트가 고부가 엔지니어링으로
화자는 이게 실제 산업 현장에서 어느 범위까지 적용될지는 두고 봐야 한다고 단서를 단다. 다만 AI 에이전트가 이메일 써주는 수준을 넘어 반도체 설계 검증 같은 고부가 엔지니어링 업무로 들어가고 있음을 보여주는 핵심 장면이라고 평가한다. [의견]

### [06:09–07:03] 병목이 GPU 밖으로 — 수천 단계의 여정
여기서 중요한 변화가 생긴다. 엔비디아는 에이전틱 AI의 하나의 프롬프트가 "prompt → retriever → tool use → response generation"으로 이어지는 수천 단계의 여정이 될 수 있다고 설명한다. 이 말이 중요한 이유는, 이제 병목이 GPU 하나에만 있지 않게 됐다는 것. AI가 답변만 만들 때는 GPU가 핵심 같았지만, AI가 일을 하기 시작하면 GPU 밖에서 엄청난 일이 벌어진다 — 파이썬 런타임이 돌고, 샌드박스가 생기고, 데이터베이스를 계속 조회하고, 외부 툴을 호출하며 에이전트끼리 프로세스를 조율해야 한다. 이 상당수는 CPU가 처리해야 한다. GPU가 아무리 빨라도 CPU가 데이터를 준비 못 하면 GPU가 기다리게 되는데, AI 팩토리에서 GPU가 기다린다는 건 공장 기계가 멈춘 것과 같다. 그래서 이 대목에서 베라 CPU가 나온다.

### [07:03–08:28] 베라 CPU — 에이전트를 위해 설계된 첫 CPU
엔비디아가 AI 에이전트를 위해 설계한 첫 CPU "베라"를 소개한다. 화자가 나열한 스펙(주장/미검증): 88개 코어와 spatial multithreading 방식의 병렬 처리 능력([불명확: 자동자막 음차]), 최대 1.2 TB/s LPDDR5X 메모리 대역폭, NVLink C2C 기반 CPU-GPU 코히어런트 대역폭. 화자는 스펙을 늘어놓으면 복잡하지만 핵심은 하나라고 한다 — 수많은 에이전트가 동시에 코드를 실행하고 툴을 호출하고 데이터를 옮기고 작업을 조율할 때 **CPU가 GPU의 발목을 잡지 않게** 하겠다는 것. 특히 에이전트를 오케스트레이션하는 프레임워크·런타임은 파이썬 기반이 많은데, 파이썬은 생산성은 좋지만 인터프리터 방식이라 한 줄씩 처리하는 오버헤드, 컨텍스트 전환·멀티스레딩 제약, 메모리 접근 병목 같은 문제가 계속 생긴다. 베라가 파이썬의 모든 구조적 한계를 없앤 건 아니지만, 수많은 에이전트 태스크가 동시에 쪼개져 실행될 때 더 많은 작업을 병렬 처리하고, 메모리 병목을 줄이고, GPU에 데이터를 끊김 없이 공급하도록 설계됐다는 점이 주목 포인트. 그래서 화자는 베라를 "엔비디아가 CPU 시장도 먹겠다"는 발표라기보다 **AI 팩토리에서 GPU가 돈을 벌도록 주변 병목을 제거하는 CPU**로 접근한 것이라 해석한다. [의견]

### [08:28–09:32] NYSE 사례 — 초저지연 인프라의 조율자
베라 CPU 발표에서 상징적이었던 사례가 하나 더 있다 — 뉴욕 증권 거래소(NYSE). 엔비디아 발표에 따르면 NYSE가 레드햇([불명확: 원문 "레드 판다"])·HP와 함께 베라 CPU를 활용해 고성능·저지연 AI-ready 마켓 인프라를 확장하려 한다. NYSE가 하루 1.1조 개 이상의 메시지를 처리한다고 언급했는데(주장/미검증), 화자는 이걸 단순히 "금융권 고객이 생겼다" 정도로 보면 안 된다고 한다. 증권 거래소처럼 데이터가 밀리면 안 되고 레이턴시가 커지면 안 되는 초저지연 인프라에서 시스템이 안정적으로 도는 핵심이 베라 CPU라는 것. 그래서 베라 CPU는 GPU 옆의 조연이 아니라 AI 팩토리의 데이터 흐름을 붙잡는 조율자 역할을 노린다고 해석해야 한다. [의견]

### [09:32–10:52] 네트워크 병목 — 스펙트럼 X 이더넷 포토닉스와 CPO
병목은 CPU에서 끝나지 않는다. AI 팩토리가 수십만~수백만 GPU 규모로 커지면 네트워크가 다음 문제가 된다. GPU 수가 늘면 지표(칩)들이 서로 대화해야 하고, 거대한 AI 모델을 학습할 때도 추론할 때도 스토리지 접근 시에도 데이터가 계속 오간다. 기존 방식은 전력·발열·신호 감쇠·지연 시간 같은 물리 법칙 한계가 있다. 그래서 그 물리를 바꿨다. 이번에 스펙트럼 X 이더넷 포토닉스와 CPO(코패키지 옵틱스)가 다시 주목받은 이유가 이것. CPO는 광 모듈을 스위치 칩에 훨씬 가깝게 붙인 방식으로, 전기 신호(전자)가 멀리 이동할수록 전력을 더 먹고 열이 나고 신호가 약해지므로, 전기 신호가 움직이는 거리를 줄이고 더 빨리 빛(광신호)으로 바꿔버린다. 엔비디아는 베라 루빈 플랫폼에서 스펙트럼 X 이더넷 포토닉스를 도입한다고 발표했고, 기존 트랜시버 기반 네트워크 대비 전력 효율 5배, AI 업타임 5배, 디플로이먼트 타임 1.3배를 언급했다(주장/미검증). 화자는 이게 단순 네트워크 장비 발표가 아니라, AI 팩토리가 커질수록 네트워크가 주변 장치가 아니라 공장 내부의 빠르고 섬세한 신경망이 된다는 뜻이라고 본다. [의견]

### [10:52–12:00] 보안 병목 — 블루필드
그다음은 보안(시큐리티). 대중적으로 덜 화려해 보일 수 있지만 엔터프라이즈 기업 기준으로는 매우 중요하다. 에이전트는 챗봇과 달리 회사 안 시스템에 들어가 휘젓고 다니기 때문이다 — 파일을 열고, DB를 조회하고, 컨피덴셜 정보를 보고, API를 호출하고, 코드를 만들어 실행하고, 권한이 있는 작업까지 실행해야 한다. 그 순간 권한 탈취·민감 데이터 유출이라는 무서운 문제가 발생한다. 화자의 표현으로 "에이전트가 사람 대신 일할 수 있다는 건 사람 대신 사고도 칠 수 있다는 말"이다. 그래서 블루필드 4([불명확: 원문 "블루필드 4 STX"]) 같은 장치가 중요해진다. 블루필드 계열은 단순히 데이터 전송을 빠르게 하는 장치가 아니라, 데이터가 오가는 길목에서 보안과 격리를 맡는다. 메인 CPU나 OS가 모든 걸 감시하는 방식으로는 부족할 수 있어, AI 에이전트가 파일과 네트워크를 오가는 순간 그 관문에서 하드웨어적으로 감시·통제할 필요가 생긴다.

### [12:00–12:24] 스택 종합과 DGX — 왜 GPU 말고 이런 것까지
화자가 스택을 종합한다. GPU는 두뇌를 돌리고, 베라 CPU는 일을 조율하고, 스펙트럼 X는 신경망을 깔고, 블루필드는 데이터 관문을 지킨다. 그리고 이 전체 공장을 운영하는 시스템이 DGX([불명확: 원문 "DSX"])다. 이렇게 보면 엔비디아가 왜 GPU 말고 이런 것까지 만드는지 이해된다 — AI가 돈을 벌기 시작하면 그 돈 버는 흐름의 모든 병목을 잡고 싶어 한다는 것. [의견]

### [12:24–13:29] PC로 확장 — RTX 스파크와 세 종의 윈도우 머신
여기까지가 클라우드·데이터 센터 이야기라면, 젠슨황은 이 흐름을 PC와 로봇으로도 확장했다. RTX 스파크 발표는 AI 에이전트를 개인 PC로 끌어내리려는 방향이다. 모든 에이전트가 클라우드에서만 돌 필요는 없고, 개인 데이터·로컬 파일·프라이버시가 중요한 작업은 PC 안에서 도는 게 유리할 수 있다.

> "Nvidia is reinventing all of PC and today we're announcing a whole new line. Three revolutionary Windows machines covering desktop, laptop and workstations. All 100% Windows compatible, 100% Nvidia AI runs on these ..." [12:38]

로이터 보도에 따르면 RTX 스파크는 마이크로소프트와의 협력, 미디어텍과의 개발을 통해 나왔고, 델·HP·레노보·에이수스·마이크로소프트 등 PC 생태계와 연결될 예정이다. 화자는 이 흐름이 기존 PC 기준도 바꿀 수 있다고 본다. [의견] 예전엔 PC를 고를 때 CPU 성능·그래픽 성능·배터리를 봤지만, 앞으로는 "내 PC가 얼마나 강력한 로컬 에이전트를 돌릴 수 있느냐"가 중요한 기준으로 나올 수 있다는 것.

### [13:29–14:22] 로봇으로 확장 — 아이작 그루트 레퍼런스 휴머노이드
로봇 쪽에서는 유니트리 H2 플러스를 레퍼런스 바디로 쓴 발표가 눈에 띄었다. 엔비디아는 아이작 그루트 레퍼런스 휴머노이드 로봇을 발표하며, 유니트리 H2 플러스 휴머노이드 바디 + 샤파([불명확])의 파이브핑거드 핸드 + 제슨 토르(온보드 컴퓨터) + 아이작 그루트 소프트웨어 스택을 결합한 오픈 레퍼런스 디자인을 제시했다. 이게 의미하는 건 엔비디아가 로봇 완제품 회사를 하겠다는 게 확실히 아니라는 것 — 레퍼런스다. 로봇의 몸은 유니트리 같은 회사가, 손은 샤파 같은 회사가 만들고, 엔비디아는 그 위에 두뇌와 신경계를 잡았다. 제슨 토르가 온보드 컴퓨터라면, 아이작 그루트는 로봇이 보고·이해하고·행동하도록 만드는 소프트웨어·모델 스택에 가깝다.

### [14:22–14:53] 유니트리 IPO와 레퍼런스의 의미
이 발표 직후 유니트리는 IPO를 한 것으로 알려졌고, 휴머노이드 로봇 시장에서 빠르게 성장하는 회사로 주목받는 상황이다. 화자는 여기서도 과장하면 안 된다고 단서를 단다 — 레퍼런스 디자인이 곧 대량 상용화 성공을 의미하는 건 아니다. 그렇지만 엔비디아가 로봇의 몸을 직접 만들기보다 로봇 산업의 두뇌·신경계 개발 플랫폼을 대령하고 있다는 포인트는 여전히 중요하다. [의견]

### [14:53–15:38] 키노트 종합 — 병목을 다 설계하겠다는 선언
화자는 이번 키노트를 "엔비디아가 GPU를 버리고 CPU 회사가 된다"는 얘기도 아니고, "GPU가 너무 중요해졌기 때문에 GPU 주변의 모든 병목을 엔비디아가 직접 다 설계하겠다"는 이야기로 해석해야 한다고 정리한다. 베라 CPU도 GPU를 대체하려는 게 아니라 GPU가 더 오래·더 바쁘게·더 수익성 있게 일하도록 만드는 장치에 가깝다. 또 "AI 팩토리"라는 말이 그냥 데이터 센터를 멋있게 부르는 마케팅 용어라기보다(물론 마케팅 표현도 있지만) 더 구체화됐다는 것 — 토큰 코스트를 극단으로 줄이고 전력당 토큰 성능을 올리며, 에이전트 처리량·타임투퍼스트프로덕션 같은 지표가 등장한다는 것 자체가 AI 인프라를 공장처럼 설계하고 측정하겠다는 뜻이 된다. [의견]

### [15:38–16:43] 로봇 플랫폼 선점과 앞으로 볼 포인트
마지막으로 로봇 발표도 당장 내년에 휴머노이드가 거리·공장에 쏟아진다는 뜻은 아니지만, 로봇을 움직이는 AI 스택을 누가 장악하느냐는 이미 시작된 싸움이라는 것. 스마트폰 시대 iOS/안드로이드 같은 운영 체제가 중요했듯, 로봇 시대엔 피지컬 AI 플랫폼이 중요해지는 상황에서 엔비디아가 그 자리를 선점하겠다는 것. [의견] 앞으로 볼 포인트로 화자가 제시한 것: ① 베라 루빈의 실제 양산·공급(AI 팩토리는 칩 하나가 아니라 서버·냉각·전력·네트워크·스토리지·제조가 다 붙어야 함 — GTC 타이페이에서 대만 서플라이 체인이 강조된 이유), ② 베라 CPU/CPO가 NYSE 같은 상징 뉴스 외에 실제 어느 규모로 배치되고 클라우드·엔터프라이즈에서 얼마나 빠르게 채택될지, ③ 케이던스 같은 에이전트 사례를 기업들이 얼마나 신뢰하고 쓸 수 있느냐, ④ CPO·스펙트럼 X 이더넷 포토닉스가 실제 배치에 얼마나 잘 적용되는지.

### [16:43–17:30] 화자의 개인적 결론 — GPU 전쟁에서 팩토리 전쟁으로
화자는 GTC 타이페이에서 본 것을 이렇게 정리한다. [의견] AI가 돈을 벌기 시작하자 컴퓨팅의 의미가 달라졌다. 모델을 잘 만드는 회사보다 인프라를 제대로 만드는 게 중요해졌다. 그래서 엔비디아는 이제 더 빠른 칩을 파는 회사로 스스로를 설명하지 않고, "compute is revenue" 시대에 그 돈이 만들어지는 생산 라인 전체를 설계하는 회사로 스스로를 포지셔닝하고 있다.

> "GPU 전쟁으로 시작된 이 거대한 흐름, 이제 팩토리 전쟁이 될 수 있습니다. 그리고 그 팩토리 공장을 누가 설계하느냐가 다음 AI 산업의 질서를 바꿀 수도 있다는 생각이 듭니다." [17:21]

화자는 이번 키노트를 단순 기술 발표가 아니라 산업의 병목이 어디로 이동하고 있는지를 보여준 사건에 가깝다고 개인적으로 생각한다고 마무리한다. "지금까지 타이페이에서 에러였습니다."
