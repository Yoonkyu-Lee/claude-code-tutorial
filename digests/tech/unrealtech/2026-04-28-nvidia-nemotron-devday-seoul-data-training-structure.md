## 요약 (주제별)

### 1. 행사 성격 — Nemotron DevDay Seoul이 무엇이었나
- 엔비디아가 서울 마포(공덕)에서 주최한 개발자 초대 행사. 화자(에러)는 초대받아 현장 참관.
- 단순히 새 AI 모델을 소개하는 자리가 아니라, Nemotron이 "무엇을 하려는가"를 개발자에게 이해시키고 함께 경험하는 자리였다는 게 화자 정리.
- 우리나라에서 처음 열린 Nemotron DevDay라 의미가 있다는 게 화자 평가. 해커톤도 함께 열려 개발자들이 직접 개발하는 세션도 있었음.

### 2. Nemotron의 정의 — "모델"이 아니라 "오픈 패밀리"
- Nemotron은 계속 오픈 형태로 공개되어 온 시리즈. 단일 AI 모델이 아니라 모델 + 데이터셋 + 라이브러리 + 리서치가 엮인 "오픈 패밀리"로 정의된다(발표 내용).
- 보통 오픈 공개가 가중치(weight)만 푸는 데 그치는 것과 달리, Nemotron은 데이터셋까지 다 제공하는 오픈소스 형태라는 점을 강조.
- 키노트는 브라이언 카탄자로 부사장이 열었고, 화자는 그를 단독 인터뷰로 소개한 적이 있다고 언급. 프리트레이닝 팀 크리스, 데이터 세션 발표자 등이 등장.

### 3. 핵심 메시지 — "엔비디아는 GPU 회사가 아니라 AI 팩토리 OS 회사"
- [의견/화자 정리] 엔비디아는 GPU로 상징되지만 실질적으로는 AI 팩토리용 전체 생산 라인(OS)을 제공하는 회사다.
- Nemotron을 통해 AI 팩토리(AI 생산 체계) 전체 — 모델, 학습 데이터셋, 테크닉/라이브러리, 인프라, 강화학습 인프라, 합성 데이터 공정까지 — 를 보여준 자리였다는 정리.
- 젠슨 황이 GTC에서 말해온 "AI 팩토리" 개념과 맞닿음. AI 팩토리를 만들려면 AI 자체·AI 설계법·향후 개발 트라젝토리까지 알아야 하므로 에코시스템 전체를 제공한다는 논리.

### 4. 데이터가 진짜 병목 — "좋은 모델보다 좋은 데이터가 희소"
- 행사에서 가장 먼저 깔린 문제의식: 좋은 모델보다 좋은 데이터가 희소해지고 있다.
- 인터넷에는 데이터가 많아 보여도 프론티어 모델에 바로 쓸 고품질 데이터는 제한적. 중복·노이즈·저작권·개인정보가 많고, 검색을 잘하는 모델·툴을 정확히 쓰는 모델·장기 컨텍스트 코딩 에이전트 같은 "행동 데이터"는 인터넷에 자연스럽게 존재하지 않음.
- 과거엔 모델 키우고 데이터 많이 넣고 컴퓨팅 많이 하면 성능이 올랐지만, 그 "단순한 시대"가 끝나가는 것 아니냐는 얘기가 나왔다는 게 화자 전언. 벤치마크 트레이드오프(쇼트/롱 컨텍스트, 퀄리티/다양성)가 많아 어떤 데이터·레시피·목표로 학습하느냐가 더 중요해짐.

### 5. 합성(synthetic) 데이터 = "검증 가능한 학습 재료 설계 기술"
- 엔비디아는 합성 데이터를 많이 써서 학습. 단 합성 데이터는 "가짜 데이터를 빨리 만드는 기술"이 아니라, 생성 → 검증(밸리데이션 데이터) → 판단(저지) → 인간 개입 휴리스틱 → 벤치마크 에러 분석의 반복 공정으로 만들어짐. 이 점이 인상 깊었다는 게 화자.
- [의견] 합성 데이터를 "검증 가능한 학습 재료로 설계하는 기술"로 보면, 그 자체가 압도적 차별성/격차가 될 수 있다.
- 데이터 생성이 프롬프트 노가다가 아니라 버전 관리가 되는 엔지니어링 작업에 가깝다는 관점.

### 6. 엔비디아의 데이터 도구 — Nemo Data Designer / Curator
- Nemo Data Designer: 파이썬 코드로 데이터셋 스키마를 먼저 정의 → 언제나 재생산 가능·누구나 생성 가능, 다양성 분포 결정과 검증까지 설정하는 도구.
- Nemo Curator: 대규모 데이터셋을 정제하는 도구. 클라우드/인터넷 데이터 소스 + LLM/합성 데이터를 섞어 정제·분류·중복 제거해야 진짜 학습 가능한 데이터가 만들어진다는 설명.
- 화자 비유: 자율주행 데이터를 현실에서 전부 녹화해 오는 게 아니라 합성으로 구성하듯(엔비디아 Omniverse/Alpamayo류 언급), 세상 사건들을 합성적으로 구성하는 것과 연결됨. [불명확: 음차 "알파마이요" — 자율주행 합성 데이터 관련 프로젝트로 추정]

### 7. 학습 구조 — 프리트레이닝 커리큘럼 & 에이전틱/리즈닝 중심
- 프리트레이닝은 한 번에 꺼내는 학습이 아니라 커리큘럼이 있음. Nemotron 3 Super는 초반에 다양성 중심으로 약 20조(20T) 토큰, 이후 퀄리티 중심으로 약 5조(5T) 토큰을 학습했다고 함(발표 인용, 세부 수치는 발표 전언).
- 모델이 배우는 능력은 "무엇을 학습했나"보다 데이터 분포(무엇을 얼마나 썼나)에 의해 정해진다는 관점.
- 지도학습 단계에서 에이전트/챗/롱 컨텍스트/리즈닝을 잘 블렌딩. 데이터 출처가 소프트웨어 엔지니어링·제너럴 툴 유즈 등이라, 목표가 예쁜 챗봇이 아니라 에이전틱 behavior와 리즈닝 중심임을 알 수 있다는 정리. 슬로건 "Asia is for general"(엔비디아가 지원하는 AI는 매우 넓은 범위) 인용. [불명확: 음차, 슬로건 정확 문구]

### 8. 강화학습 — 멀티 인바이런먼트 RL
- 단일 환경 강화학습은 리워드 해킹에 취약 → 여러 환경에서 나온 궤적(trajectory)을 잘 섞어 만든 RL이 필요. 이 궤적도 합성해서 구성.
- 분산 시스템, 실행 환경 검증기, 시뮬레이션 유저를 포함한 전체 운영을 한눈에 보는 구성. 여러 환경 인터랙션에서 시행착오 → 에러 분석 → 학습 재반영. 이렇게 만든 모델이 OpenClaw/Nemo Claw 등에 적용됨.
- 화자 비유: 알파고의 37수처럼, 실제 작업 안으로 들어가 유의미한 데이터를 찾아가는 과정이 멀티 인바이런먼트 RL 배치.

### 9. 온프레미스/온디바이스와 Nemo Guardrails(Nemo Claw)
- 1층 "빌더 클라우드" 세션에서 OpenClaw와 함께 엔비디아가 미는 Nemo Claw류를 체험. Nemo Claw는 엔비디아만의 가드레일을 선정해 기업/개인/연구소가 안정적으로 활용하게 하는 시스템으로 소개.
- 정구영 SA(GTC 발표 경력)가 직접 이메일 초안 작성 데모를 시연: 덜 에이전틱한 작업을 시키면 온도가 40도→상승. 내 소유 GPU 위에 오픈 모델을 공짜로 받아 무한 반복해도 전기값만 내면 되므로, 클라우드 1년 비용보다 저렴할 수 있다는 주장(주장/미검증).
- Q&A: Nemo Claw는 온디바이스만이 아니라 클라우드·온디바이스 둘 다 타깃(OpenClaw처럼). 모델 종류는 상관하지 않고 보안만 상관 — Anthropic 모델을 쓰되 온프레미스에서 에이전트 행동을 막는 역할 정도. 오픈클로에서 Nemotron 작은 모델이 "작은데 쓸 만하다"는 커뮤니티 평가도 있었다는 언급.

### 10. 한국 특화 — Nemotron Personas Korea
- LLM이 생각하는 "한국 사람"이 편향돼 있다는 문제 제기: 외국인이 한국 하면 김치·김연아·BTS를 떠올리듯, 특정 모델이 한국인 직업을 "유자 재배 농민"·"요양보호사" 등으로 편중되게 인식(발표 예시 인용, 모델명은 음차라 [불명확]).
- 한국어를 잘하는 것과 한국 사회의 상식·분포를 이해하는 건 다른 문제 → 서구권 분포에 맞춰 답하면 할루시네이션 원인이 됨. 주거 점유 형태·도시 분포·음식 분포(샐러드 비중이 과하게 높다 등) 같은 것도 왜곡.
- Nemotron Personas Korea: 통계청·대법원 등 한국 기관 데이터를 활용해 한국 현실(이름·문화 포함)을 반영한 데이터셋. GitHub에서 실제 사용 가능. 추론 문제 생성·툴 사용·안전 문제 등에 활용. 핵심은 "쓸 수 있다"보다 이걸 발판으로 다양성을 더 확장해 나갈 수 있다는 점.

### 11. 화자의 종합 해석 & 전망
- [의견] 오픈 모델 경쟁의 핵심이 "베이스 체크포인트 공개"를 넘어, 합성 데이터 파이프라인 구성 / 데이터 믹스 / 강화학습 멀티 인바이런먼트 도입 / 국가별 특화로 이동하고 있다.
- [의견] 좋은 AI가 큰 모델 하나로 제시되는 게 아니라 데이터 품질과 생성 공정에서 나온다는 걸 이해할 수 있는 발표였다.
- [의견] 합성 데이터가 만능은 아니고, 로컬 데이터도 현실을 완벽히 반영하진 못함. 그래서 작은 스코프로 쪼개 다시 엔지니어링하는 과정이 계속됨. 엔비디아의 대표 성공 사례인 CUDA 에코시스템처럼, Nemotron을 방점으로 한 에코시스템도 확장을 노린다는 게 화자 관측.

### [00:27–00:48] 행사 성격 — 모델 소개가 아니라 "이해시키는 자리"
[의견] 이 행사는 단순히 엔비디아의 새 모델을 소개하는 자리로 볼 수도 있지만 사실은 아니라는 게 화자 정리다. Nemotron 시리즈는 계속 오픈 형태로 공개돼 왔는데, "Nemotron이 과연 무엇을 할 것인가"를 개발자들과 함께 제대로 이해하고 경험해 보는 행사였다고 본다.

### [00:48–01:32] "AI 팩토리" 전체를 보여준 자리
Nemotron을 통해 엔비디아는 AI 팩토리(AI 생산 체계) 전체를 보여줬다는 정리다. 모델만 얘기하는 게 아니라 학습 데이터셋, 테크닉·라이브러리·리서치와 관련된 오픈 패밀리로 정의된다. AI 모델을 직접 설계함으로써 AI 인프라를 어떻게 설계할지, 전체 에코시스템을 어떻게 잘 만들지와 연관된다. 하드웨어·소프트웨어·심지어 테틱(합성) 데이터, 강화학습 인프라까지 다 활용하면 "여러분만의 AI를 잘 만들 수 있다"는 게 핵심이라고 요약한다.

### [01:32–01:48] "엔비디아는 GPU 회사가 아니다"
> "그러니까 이제 GPU 회사가 아닌 거예요. 엔비디아를 GPU로 많이 상징이 되지만 실질적으로는 AI 팩토리용의 이 OS 전체 생산 라인을 다 제공하는 회사다." [01:32]

[의견] 그런 측면에서 Nemotron이 굉장히 중요하고, 우리나라에서 Nemotron DevDay가 열린 것이라는 정리.

Nemotron은 오픈소스 AI 컨트리뷰션이 많이 올라가고 있고, 데이터셋까지 다 제공하는 그야말로 오픈소스 형태. 보통은 가중치만 공개하는데 다 풀어 놓고 "너희 이거 해 봐라"고 제시한다. 엔비디아는 Nemotron뿐 아니라 로보틱스의 GR00T, 바이오메디컬의 Clara, 피지컬 AI의 Cosmos 등으로 AI 전반의 외연을 확장하고 있다는 정리.

### [05:43–06:33] "큰 모델이면 된다"는 오해 — 단순한 시대의 끝
[의견] 사람들이 가장 오해하는 지점이 "AI는 그냥 큰 모델이면 된다"는 것. 과거엔 모델이 커지고 데이터·컴퓨팅을 많이 하면 성능이 오르는 게 맞았지만, 이번 행사에서 그 단순한 시대가 끝나가는 것 아니냐는 얘기가 나왔다. Nemotron 프리트레이닝 팀 크리스의 발표: 벤치마크에 따라 제작하다 보니 쇼트 컨텍스트/롱 컨텍스트 트레이드오프, 퀄리티/다양성 트레이드오프가 많고, 어떤 데이터·레시피·목표로 학습하느냐가 더 중요해진다. 엔비디아는 합성 데이터를 많이 써서, "합성 데이터를 LLM으로 많이 뽑으면 쉽게 되겠지"라고 생각할 수도 있다는 대목.

### [06:33–07:04] 합성 데이터 = 검증 가능한 학습 재료 설계
키노트 이후 첫 데이터 세션에서는, 합성 데이터는 그냥 생성만 하면 끝이 아니라 검증하는 밸리데이션 데이터, 판단하는 저지, 인간이 개입하는 휴리스틱, 벤치마크 에러 분석 같은 반복 공정을 통해 생성된다는 점이 인상 깊었다는 정리. [의견] 합성 데이터를 "가짜 데이터를 빨리 만드는 기술"이 아니라 "검증 가능한 학습 재료로 설계하는 기술"로 보면, 그 자체가 압도적 차별성/격차가 될 수 있다.

### [07:04–07:36] 예고 — Nemotron Personas Korea
잠시 뒤 다룰 우리나라 페르소나 데이터셋 얘기를 미리 꺼낸다. 외국인이 생각하는 한국인 이미지가 있듯, LLM이 생각하는 한국인 직업도 편중돼 있다 — 어떤 모델은 "유자 재배 농민", 다른 모델은 "요양보호사"로 생각한다는 예시(모델명은 음차라 [불명확]). 한국어를 잘하는 것과 한국 사회의 상식·분포를 이해하는 건 다른 문제라, Nemotron Personas Korea 세션에서 이를 이해할 수 있었던 재밌는 장면들이 있었다.

### [07:36–08:13] 데이터 병목 — "좋은 데이터가 희소해지고 있다"
행사 전반을 정리하며 가장 먼저 봐야 할 부분은 데이터라고 본다. 첫 데이터 세션에서 "AI 데이터 바틀넥(병목)"을 다뤘다. 인터넷에는 데이터가 많아 보이지만 프론티어 모델에 바로 쓸 고품질 데이터는 제한적 — 그냥 다 긁어 쓰면 성능이 향상되지 않는다. 중복·노이즈·저작권·개인정보가 많고, 검색을 잘하는 모델·툴을 정확히 쓰는 모델·장기 컨텍스트 코딩 에이전트 같은 "희귀한 고급 능력의 데이터"가 부족하다.

### [08:13–08:56] 합성 데이터는 "설계 대상" — 정제·평가까지가 능력
그런 행동 데이터는 인터넷에 자연스럽게 존재하지 않으므로, 합성 데이터(테틱 데이터)는 사용자가 남긴 현실 데이터를 모으는 게 아니라 시스템이 설계하는 대상이 된다. 현실 데이터는 비용도 들고 커버리지도 약하고 편향·제어 문제가 있다. 그래서 그냥 생산하고 끝이 아니라 평가도 하고, 실제 트레이닝 하며 어떤 에러가 발생하는지 잘해서 정제된 양질의 데이터를 생성하는 능력 자체가 설계 능력·기술이라는 정리.

### [08:56–09:32] Nemo Data Designer & Nemo Curator
엔비디아는 Nemo Data Designer를 갖고 있는데, 파이썬 코드로 데이터셋 스키마를 먼저 정해 놓으면 언제나 재생산 가능하고 누구나 만들 수 있다. 다양성 있는 데이터 분포 결정과 검증도 설정한다 — 즉 코딩/엔지니어링에 가깝고, 프롬프트 노가다가 아니라 버전 관리가 되는 엔지니어링 작업. Nemo Curator도 대규모 데이터셋을 정제하는 도구로, 클라우드/인터넷 데이터 소스가 들어오면 LLM·신데틱 데이터와 섞어 데이터를 생산해 나간다.

### [09:32–10:00] 비유 — 자율주행 합성 데이터
자율주행 데이터를 현실에서 다 녹화해 오는 게 아니라, 엔비디아가 (알파마이요류로) 합성적으로 세상의 사건들을 구성하듯, 정제·분류·중복 제거를 해야 진짜 학습 가능한 데이터가 만들어진다는 비유. [불명확: 음차 "알파마이요"]

### [10:00–10:36] 케이스 스터디 — 서치 툴 콜링
검색 능력은 그냥 API를 연결한다고 생기는 게 아니다. 모델이 "언제 검색해야 하지? 어떤 질문을 만들어야 하지? 결과가 나오면 어떻게 사고·추론해야 하지?"를 배워야 하는데, 이런 건 인터넷에 안 나와 있으니 설계를 제대로 해야 한다. 젠슨 황 → 오리건 스테이트 유니버시티로 연결해 가듯, 결국 어떤 행동을 할 것이냐 하는 정책(policy)을 학습하는 문제라 여기에 집중한다.

### [10:36–11:19] 데이터 믹스처 & 프리트레이닝 커리큘럼
두 번째 발표(데이터 믹스처)에서 Nemotron 3 Super 학습 데이터를 소개. 모델이 배우는 능력은 "무엇을 학습했나"보다 데이터 분포(무엇을 얼마나 썼나)로 정해진다. 프리트레이닝은 한 번에 꺼내는 학습이 아니라 커리큘럼이 있음 — Nemotron 3 Super는 초반에 다양성 중심으로 20조(20T) 토큰, 이후 퀄리티 중심으로 5조(5T) 토큰을 학습했다고 함(발표 전언).

### [11:19–11:43] Nemotron 계보 — Megatron부터 3 Super/Ultra까지
실제 Nemotron 개발 계보: 예전 Megatron 시절부터 시작해 Nemotron Instruct, Nano V2, 엔비디아 3 Nemotron Nano, 지금 3 Super·Ultra까지 나온 상황. [의견] 합성 데이터를 토대로 3 Super가 이렇게까지 성능이 올라온 걸 보면 엔비디아가 이 정도 수준이라는 게 대단하다는 평. [불명확: 계보 모델명 일부 음차]

### [11:43–12:21] 지도학습 — 에이전틱 behavior & 리즈닝이 중심
전체 파이프라인: 데이터 준비 → 프리트레이닝 → 지도학습. 지도학습 때도 에이전트·챗·롱 컨텍스트·리즈닝을 잘 블렌딩. 핵심은 예쁜 챗봇을 만드는 게 아니라, 에이전틱 behavior와 리즈닝이 이미 중심에 들어와 큰 비중을 차지한다는 것. 데이터 출처가 소프트웨어 엔지니어링·제너럴 툴 유즈 등이라, AI가 에이전트 AI로 진입했을 때 그것들을 얼마나 잘 쓸 수 있는지를 중심으로 만들어졌음을 알 수 있다는 정리.

### [12:21–13:11] 멀티 인바이런먼트 강화학습 — "Asia is for general"
내부적으로 강화학습을 수행할 때 분산 시스템, 실행 환경 검증기, 시뮬레이션 유저를 포함한 전체 운영을 한눈에 본다. 엔비디아는 "(G)Asia is for general"이라 표현 — 지원해야 하는 AI가 굉장히 넓은 세대라 세상에 존재하는 거의 모든 AI 얘기를 하는 것. AI에 바라는 능력이 너무 많으니, 단일 환경(싱글 인바이런먼트) 강화학습은 리워드 해킹에 취약해서, 여러 환경에서 나온 궤적을 잘 섞어 만든 강화학습이 필요하다고 얘기하며, 그 궤적(트레토리)도 합성해서 구성한다고 언급. [불명확: 슬로건 음차]

### [13:11–13:41] 시행착오 → 에러 분석 → 재반영 = 알파고 37수 비유
여러 환경의 인터랙션이 있어야 시행착오가 생기고, 그걸로 에러를 분석해 학습에 다시 반영하고, 그렇게 OpenClaw/Nemo Claw에 적용되며 모델이 제대로 만들어진다. 실제 작업 안에 들어가야 하니, 알파고의 37수 같이 유의미한 데이터를 찾아가는 과정으로서 멀티 인바이런먼트 RL 배치를 생각하면 된다는 비유.

### [13:41–14:20] 마지막 세션 — 한국 페르소나의 편향 문제
마지막 세션은 최초의 우리나라 페르소나 데이터셋. 외국인이 한국인 하면 김치·김연아·BTS만 생각하듯, LLM이 생각하는 한국인도 유자 재배 농민 등으로 편향돼 있다. AI가 한국형 데이터를 취사선택해 학습한 게 아니라 서구권 분포에 맞춰 학습했기 때문에, 한국의 뉘앙스·문화를 이해 못 한 채 답할 수 있고 이것이 할루시네이션의 원인이 될 수 있다.

### [16:00–16:58] 종합 — 오픈 모델 경쟁의 축 이동
