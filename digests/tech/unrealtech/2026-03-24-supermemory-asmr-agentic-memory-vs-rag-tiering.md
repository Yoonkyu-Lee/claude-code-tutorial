## 요약 (주제별)

### Supermemory와 ASMR 방식이 무엇인가
- Supermemory는 스타트업으로, 기존 RAG(레그)를 쓰지 않고도 장기 기억 벤치마크에서 훨씬 높은 점수를 냈다고 주장한다.
- 이 실험적 방식의 이름이 **ASMR** = Agentic Search and Memory Retriever(에이전틱 서치 앤 메모리 리트리버). 여러 에이전트가 역할을 나눠 기억을 읽고 찾는 구조.
- 화자는 이것이 아직 Supermemory의 주력 엔진이 **아니라**고 명시하며(Supermemory 본인들도 "아직 우리의 주력 슈퍼 메모리 엔진이 아니다"라고 밝힘), 메모리·추론의 한계가 어디까지 가는지 실험한 결과라고 정리한다.

### "99%" 주장의 정확한 의미
- 99%는 "무슨 얘기를 해도 다 기억한다"는 뜻이 **아니라**, **LongMemEval(롱 메모리 이밸루에이션)** 벤치마크에서 나온 점수다. (주장/미검증)
- 세부: 8-variant(앙상블 계열)에서 98.6%, 12-variant(디시전 계열)에서 97.2% → "거의 99%에 달했다"고 표현. (주장/미검증, 자막 음차로 variant 세부 명칭은 [불명확])
- LongMemEval은 단순히 긴 문서를 넣고 얼마나 기억하는지 보는 시험이 아니라, 약 500가지 질문으로 정보 추출·멀티턴 추론·시간 추론·지식 업데이트 등을 종합 평가하는 벤치마크. 관련 논문에서 상용 LRM(제미나이·GPT 등)은 장기 기억 과제에서 성능 하락을 보인다고 설명. 기존 모델은 대체로 90% 수준이었고, 이를 99%까지 올린 점이 의미 있다는 것. (수치는 화자 인용/미검증)

### 기존 RAG의 한계
- RAG: LLM이 학습한 것만 답하면 최신 정보를 못 따라가므로, 별도 벡터 데이터베이스를 구축해 프롬프트와 유사도 검색을 한 뒤 유사한 청크를 LLM에 함께 넣어 정확도를 높이는 방식.
- 핵심 문제는 "기억을 못 한다"가 아니라 **저장은 다 돼 있는데 상황에 맞게 꺼내고·정렬하고·최신화하는 능력이 약하다**는 것.
- 색깔 예시: 처음 "파란색 좋아" → 이후 "빨간색 좋아" → 마지막 "검은색 최고"라고 했을 때, 옛 시스템은 셋 다 저장돼 있어 무엇이 진짜 최신·유효한지 구분을 못 한다. 여러 세션에 나뉘어 있으면 연결도 어렵다.

### Supermemory가 바꾼 것 — 저장 방식과 관계 추적
- **저장 방식부터 변경**: 대화 세션을 통째로 저장하는 대신, 의미 단위로 블록을 나눠 **아토믹하게** 메모리를 구성. 청크를 그냥 자르는 게 아니라 memory generation(메모리 생성)으로 애매한 표현을 풀어쓴 정보 조각으로 정제해 저장. 화자는 이를 팔란티어의 **온톨로지** 개념에 비유. [의견]
- **세 가지 관계 추적**: Update(A→B 사실 수정), Extend(C 의미에 D 디테일 확장), Derive(여러 조각에서 논리적으로 새 결론 추론). 사람이 생각하는 구조와 비슷하다는 것. (음차로 세 용어 표기는 update/extend/derive로 교정)
- **두 개의 시간축**: 대화가 말해진 타이밍이 아니라, 의미적으로 어떤 시점의 사건인지를 **이벤트 시점**과 **다큐먼트(문서) 시점**으로 나눠 저장. 이 덕에 템퍼럴 리즈닝(시간 추론)·널리지 업데이트(지식 갱신) 점수가 올랐다는 설명. (주장/미검증)

### 검색 방식의 변화
- 노이즈 섞인 raw 청크를 바로 찾는 대신, **메모리 단위로 시멘틱 서치(의미론적 검색)**를 먼저 수행 → 의미적으로 유사한 메모리를 찾으면 그와 연결된 원본 소스 청크를 결과에 다시 넣어 준다.
- 앞 단계에서 이미 의미론적으로 추출·정제해 뒀기 때문에, 검색도 벡터 임베딩 유사도만 쓰는 게 아니라 의미론을 유지한 채 진행.

### ASMR 아키텍처 (에이전트 분업)
- 여러 에이전트가 역할 분담: **리더(Reader) 에이전트**는 세션 전체(사용자↔AI 대화)를 쭉 읽고, **서치(Search) 에이전트**는 의미를 따라 능동적으로 찾고, 마지막 **앙상블(ensemble, n=3 계열)** 단계에서 여러 추론 경로를 모아 답을 낸다.
- 병렬로 여러 에이전트를 돌리므로 **토큰·리소스를 더 많이 쓴다**는 점은 화자가 명시하는 트레이드오프.

### KV캐시와의 구분 (중요 포인트)
- 장기 기억 하면 흔히 트랜스포머의 **KV캐시**를 떠올리지만, Supermemory와는 층위가 다르다.
- KV캐시: 프리필(prefill) 단계에서 입력 토큰 간 어텐션 관계를 계산해 만든 키·밸류 값을 캐시로 저장, 디코드(decode) 단계에서 토큰을 하나씩 생성할 때 재사용. 요즘은 파라미터 저장 용량보다 KV캐시 용량이 더 버거워지는 상황.
- KV캐시는 **현재 세션 상태의 장기 기억**(내가 넣은 문서·PDF·웹사이트 등 배경 지식)이고, Supermemory가 다루는 것은 **다른 세션·실시간·여러 타임라인의 정보** 등 그보다 상위 개념. 이런 것을 전부 KV캐시에 토큰 수준으로 담아 대응하기는 어렵다는 것.

### 하드웨어 계층 — 메모리 티어링 (엔비디아·그로크)
- KV캐시 수요는 그대로 남고, 그 위에 **별도의 익스터널 메모리 계층**이 붙는 방향. 결국 메모리가 더 많이 필요해진다.
- 엔비디아 루빈 플랫폼의 **LPX**(그로크 LPU 계열)를 디코드의 어텐션이 아닌 **FFN** 부분에 적용, SRAM으로 저지연·고반응성 대응. 에이전트 루프(멀티턴)로 바로바로 답해야 하는 필요 때문.
- **CMX**(구 ICMS=Inference Context Memory Storage에서 Inference를 빼고 Context Memory Storage로 개명): DRAM 대신 SSD(낸드 플래시)를 DRAM처럼 써서 HBM에 못 담는 KV캐시를 SSD까지 내려 활용. 블루필드 4에 탑재. (음차 교정: CMX/ICMS)
- 메모리 티어링: HBM → LPDDR → GDDR → 낸드 플래시/SSD. GDDR은 루빈엔 있으나 다음 로드맵(파인만 계열로 추정, [불명확])에선 빠질 수도 있다는 표현.

### 전체 구조 정리 (계층 분리)
- **의미론적 메모리 층**(어떤 기억을 고를까) = Supermemory(시멘틱 메모리) 담당.
- **오케스트레이션 층**(어디서 가져올까) = 다이나모(NVIDIA Dynamo 계열, 음차 [불명확]) 같은 소프트웨어가 담당.
- **컨텍스트 저장·디코드 가속 층** = CMX(SSD에 컨텍스트 메모리 저장) + LPX/LPU(디코드 FFN 저지연 대응).
- 핵심 포인트는 소프트웨어(기억 선택)와 하드웨어(메모리 티어링) 양쪽에서 **층위가 나뉘고 있다**는 것.

### 화자의 전망 [의견]
- RAG로 갈 것이라 모두가 기대했던 적도 있지만, 새 기술이 나오면서 접근법이 계속 바뀐다. 상황이 바뀌면 시스템도 바뀔 것.
- 멀티턴으로 세션을 유지해야 하는 상황에서 Supermemory 같은 방식은 별도 익스터널 스토리지(느리지만 대용량인 SSD·하드)를 상정하므로 그 방향으로 확장될 수 있다.
- 소프트웨어의 기억 선택이 에이전틱·의미론적으로 분화되고, 하드웨어도 분화되면서 에이전틱 AI가 "생각보다 훨씬 빨리" 완성형으로 다가오고 있다고 평가.
- ASMR은 **일주일 뒤 오픈소스로 공개** 예정이며 화자도 직접 테스트해 볼 예정.

## 3줄 오리엔테이션
- Supermemory라는 스타트업이 기존 RAG 대신 **ASMR**(여러 에이전트가 의미 단위로 기억을 저장·검색하는 방식)로 장기 기억 벤치마크에서 거의 99%(주장/미검증)를 냈다는 소식을 해설한다.
- 핵심은 "AI가 기억을 더 많이 한다"가 아니라, **무엇을 어떻게 기억하고 꺼낼지(소프트웨어 알고리즘)** 층과, 그걸 뒷받침하는 **메모리 티어링(하드웨어)** 층이 각각 분화되고 있다는 것.
- 엔비디아 루빈의 LPX·CMX(메모리 티어링)와 Supermemory(기억 선택)는 서로 다른 층위이며, 이 모든 흐름이 에이전틱 AI를 앞당기고 있다는 게 화자의 결론.

### [00:00–00:20] 오프닝 훅
ASMR이라는 것은 여러 에이전트가 역할을 나눠 기억을 읽고 찾는 구조로 돌아간다는 한 문장으로 영상이 시작된다. 이어 화자(러)가 인사하며, AI가 오랫동안 대화해도 이전에 했던 얘기를 잘 기억하고 제대로 답하는 방법이 계속 나오고 있다고 운을 뗀다.

### [00:20–01:03] Supermemory의 등장과 99% 주장, 에이전틱 AI 맥락
Supermemory라는 스타트업이 자기들 방식으로 하니 기존 RAG(데이터베이스를 따로 구축해 유사도 검색으로 최신 정보를 찾아 LLM에 넣는 방식)보다 훨씬 높은, 거의 99%에 달하는 새 벤치마크 점수를 냈다고 주장한다. (주장/미검증) 화자는 단순히 "기억하는 정도가 늘었다"로 보면 단편적이라며, 지금 주목받는 **에이전틱 AI**(나를 대신해 멀티로 여러 번 대화하며 왔다 갔다 하는 방식)에서 대화가 몇 번만 오가도 이전 내용을 제대로 기억 못 하거나, 중복된 얘기 중 무엇이 더 중요·최신인지 구분 못 하는 문제가 생긴다고 짚는다.

### [01:03–01:44] 컨텍스트 길이 한계와 하드웨어 vs 소프트웨어
이런 문제에 대응하려면 컨텍스트 길이가 길어져야 하는데, 그 길이에 한계가 있다면 다른 대처가 필요하고 그 실마리를 찾은 것이라고 설명. 엔비디아가 GTC에서 루빈 플랫폼을 소개하며 ICMS(현재 CMX로 개명)나 그로크의 LPX 같은 시스템으로 분할 접근하는 것은 **하드웨어 쪽**에서 일어난 일이고, Supermemory가 제시한 방식은 "이걸 어떻게 읽어낼 것이냐"라는 **알고리즘적 소프트웨어 관점**으로 이해하면 된다며, 둘을 통합해 정리하겠다고 예고한다.

### [01:44–02:35] "99%"의 정확한 의미와 ASMR 명명
Supermemory의 99% 주장은 "무슨 얘기를 해도 다 기억한다"가 아니라 **LongMemEval**이라는 벤치마크 결과다. 8-variant 앙상블 계열에서 98.6%, 12-variant 디시전 계열에서 97.2%로, 거의 99%에 달했다는 것. (주장/미검증, variant 세부 명칭은 음차라 [불명확]) 화자는 이것이 공식 주력 모델은 아니라고 지적하며, Supermemory 본인들도 밝혔다고 인용한다.

> "먼저 분명히 말씀드리면은 이것은 아직 저희의 주력 슈퍼 메모리 엔진이 아닙니다" [02:14]

특정 제약과 무관하게 메모리·추론의 한계가 어디까지 갈 수 있는지 실험해 봤더니 이런 결과가 나왔고, 에이전트 측면에서 개선되어 이를 **ASMR = 에이전틱 서치 앤 메모리 리트리버**라고 이름 붙였다는 것.

### [02:35–03:04] 띄우기가 아니라 흐름 읽기 [의견]
화자는 Supermemory를 홍보하려는 게 아니라, 이런 연구·실험·개발 결과가 계속 나오는 걸 보니 **주류인 RAG 시스템에서 점차 이쪽으로 많이 바뀌겠다**, 그러면 이를 지원하는 반도체·하드웨어 전체 시스템도 같이 바뀔 수 있다는 포인트를 보려는 것이라고 밝힌다. [의견]

### [03:04–04:05] LongMemEval 벤치마크의 성격
LongMemEval은 그냥 긴 문서를 넣고 얼마나 기억하는지 보는 시험이 아니라, 약 500가지 질문으로 정보 추출·멀티턴 응답·시간 추론·지식 업데이트 등 전반을 추론하는 벤치마크다. 이를 제시한 논문(자막 음차 "롱")에 따르면 제미나이·GPT 등 상용 모델을 LRM으로 돌렸을 때 장기 기억 과제에서 성능 하락을 보인다. 즉 "예전에 뭐라고 했더라"를 떠올리는 수준이 아니라, 과거 정보가 뒤섞였을 때 지금 시점에서 진짜 유효한 것을 골라낼 수 있느냐까지 보는 구조. 이전 모델들은 대체로 90% 수준이었는데 이를 99%까지 올린 건 상당히 의미가 있다는 것. (수치는 인용/미검증)

### [04:05–05:04] RAG란 무엇인가
RAG를 다시 설명한다. 보통 질문하면 LLM이 학습한 것만으로 바로 답하는데, 그러면 최신 정보를 못 따라간다. 그래서 최신 정보를 담는 별도의 데이터베이스(정보 저장 공간)를 구축하고, 프롬프트를 **벡터 DB**와 맞춰 유사도 있는 데이터를 찾아 LLM에 함께 넣어 실시간 정보까지 녹여 답하게 하는 방식이다. 그런데 이런 RAG 시스템엔 기본적 문제가 있다 — 데이터베이스 안에서 시간이나 중요도에 따라, 여러 대화가 있어도 무엇이 진짜 정확한 정보인지가 명확히 구분돼 있지 않다는 것.

### [05:04–06:24] 기억의 진짜 문제 — 저장이 아니라 꺼내기 (색깔 예시)
화자는 이제 AI가 **무엇을 기억해야 하는지를 고르는 방식**이 바뀌고 있다고 정리한다. 단순히 벡터 DB로 구축하는 게 아니라, 기억을 어느 메모리 층위에 둘지, 어떤 의미를 어떤 컨텍스트에 저장하고 어떻게 꺼낼지를 다루는 새 시스템이라는 것. 색깔 예시: 처음 "파란색 좋아" → "빨간색 좋아" → 마지막 "검은색 최고"라고 하면, 옛 시스템은 셋 다 좋아하는 것으로 저장해 무엇이 제일 좋은지 모른다. 핵심은 RAG의 문제가 "기억을 못 한다"가 아니라 **기억은 다 저장돼 있는데, 상황에 맞게 꺼내고 정렬해 최신화하는 능력이 약하다**는 것.

> "기억은 다 저장되어 있는데 그거를 상황에 맞게 꺼내고 정렬해서 최신화하는 능력이 약하다" [06:16]

### [06:24–07:44] Supermemory의 저장 방식 — 아토믹 메모리와 온톨로지
Supermemory는 결국 AI를 병렬로 돌리므로 토큰·리소스를 더 많이 쓴다(화자가 명시). 옛날 방식은 대화를 그냥 잘라 임베딩해 질문이 오면 비슷한 조각을 벡터에서 꺼내는 식이라, 그게 최신인지·나중 정보로 덮였는지·여러 세션에 나뉜 걸 어떻게 연결할지 답을 못 냈다. Supermemory는 **저장 방식부터** 바꿔서, 세션을 통째 저장하는 대신 **의미 단위 블록으로 나눠 아토믹하게** 메모리를 구성한다. 블록을 넣을 때 단순 청크가 아니라 **memory generation**으로 애매한 표현을 풀어쓴 정보 조각으로 변형·정제해 저장한다. 화자는 이를 팔란티어의 **온톨로지** 개념에 비유한다. [의견]

### [07:44–09:02] 세 가지 관계 추적과 두 개의 시간축
조각을 낸 다음 세 가지 대표 과정으로 정보 조각들의 관계를 추적한다. **Update**: 예전 사실 A가 업데이트되어 B가 됐음을 찾아냄. **Extend**: 원래 C 의미밖에 없던 것이 D 디테일까지 확장됐음. **Derive**: 여러 정보 조각으로부터 논리적으로 새 결론을 추론 — 사람이 생각하는 구조와 비슷. 기존 RAG는 데이터베이스가 서로의 관계성을 무시한 채 청크별로만 들어가 있었는데, Supermemory는 update/extend/derive 세 형태로 관계를 추적할 수 있게 하고, 저장량을 무작정 늘리는 게 아니라 **시간 정보를 따로 잡아 각 메모리에 두 종류의 시간축**(이벤트 시점, 다큐먼트 시점)을 붙였다고 설명. 대화가 말해진 타이밍이 아니라 의미적으로 어떤 시점의 사건인지를 나눠 정리하기에 템퍼럴 리즈닝·널리지 업데이트 점수가 올랐다는 것. (주장/미검증)

### [09:02–09:45] 검색 방식의 변화 — 메모리 단위 시멘틱 서치
검색 방식도 바뀌었다. 노이즈 섞인 raw 청크를 바로 찾는 대신, **메모리 단위로 시멘틱 서치(의미론적 검색)**를 먼저 진행한다. 의미적으로 비슷한 메모리가 나오면 그와 연결된 원본 소스 청크를 결과에 다시 넣어 준다. 앞에서 이미 의미론적으로 추출해 놨으니 검색도 그 의미론을 유지하며 처리한다 — 단순 벡터 임베딩 유사도 검색과 다른 지점.

### [09:45–10:22] ASMR 아키텍처 — 에이전트 분업
ASMR(에이전틱 서치 앤 메모리 리트리버)은 여러 에이전트를 두고 역할을 나눠 기억을 읽고 찾는 구조다. 에이전트가 여러 개 돌아 토큰도 많이 쓰이지만, **리더 에이전트**는 세션 전체(사용자↔AI 대화)를 쭉 읽고, **서치 에이전트**는 의미를 따라 능동적으로 찾는다. 마지막 **n=3 앙상블**에서 여러 추론 경로를 모아 답을 낸다.

### [10:22–13:05] KV캐시 개념과 Supermemory와의 구분
장기 기억 하면 흔히 트랜스포머의 **KV캐시**를 얘기하는데, 이와 구분해서 봐야 한다. 프롬프트를 처음 칠 때 각 단어 간 어텐션 관계를 모두 읽는 **프리필** 단계에서 키·밸류(KV) 값이 생성되고, 이를 캐시로 넣어 계속 쓰는 것이 KV캐시다. 요즘은 파라미터 저장 용량보다 KV캐시가 여러 턴 돌며 장기 기억을 담느라 용량이 버거워진다. KV캐시도 LLM 안에서 메모리 역할을 하는 건 맞지만, 그것은 내가 이미 넣어 놓은 문서(프롬프트·PDF·웹사이트)에 대한 **배경 장기 기억**이며, 바로 쓰는 건 HBM에서, 잠깐은 LPDDR/DRAM에, 오래 안 쓸 콜드 KV캐시는 느린 SSD나 바깥에 내려 둔다. 반면 Supermemory가 다루는 것은 실시간·다른 세션의 정보, 프로젝트 마감 수정 같은 여러 타임라인의 병렬 세션 정보로, 이를 KV캐시에 토큰 수준으로 다 담아 대응하긴 힘든 **상위 개념**이다. 즉 KV캐시(HBM·DRAM·스토리지 수요는 그대로)에 더해 그 위에 별도의 익스터널 메모리 계층이 붙어, 오랫동안 에이전트로 왔다 갔다 하려면 별도 메모리가 또 필요해진다 — 결국 메모리가 더 많이 필요하다는 것.

### [13:05–14:44] 프리필/디코드와 LPU(그로크), 어텐션 vs FFN
프리필/디코드를 다시 예시로 설명한다. "the quick brown fox"를 LLM에 넣으면 단어 간 어텐션 관계를 계산하며 KV(키·밸류) 값을 만드는 과정이 **프리필**이고, 그다음 단어를 한 개씩 생성할 때마다 앞서 계산한 KV 값을 넣어 하나 생성하고 또 캐시를 모아 넣어 생성하는 반복이 **디코드**다. 서로 다른 세션이나 에이전틱으로 길어지는 것을 모두 KV캐시로 담긴 힘들다. 그래서 엔비디아가 그로크의 **LPU**를 디코드에 적용했는데, 정확히는 디코드 전체가 아니라 디코드 안의 어텐션(여러 토큰 관계 계산으로 여전히 빡센 부분) 말고, **짧게 엄청 반복하는 루프(FFN) 부분**에 SRAM을 넣어 대응한다. 에이전트 시스템은 한 번 말하고 끝나는 게 아니라 질문받고 기억 조회 후 컨텍스트를 붙여 또 답하는 멀티턴 루프라, KV캐시가 많이 쓰이는 동시에 바로바로 답할 필요가 생겨, SRAM으로 밴드위스는 크고 레이턴시는 적은 고반응성 방식으로 루빈 플랫폼의 LPX가 LPU로 탑재됐다는 것.

### [14:44–16:03] CMX(구 ICMS)와 Supermemory의 층위 차이
Supermemory로 가기 전에 엔비디아가 하려는 것은 블루필드 4 같은 **CMX**다. 원래 ICMS(Inference Context Memory Storage)였는데 Inference를 빼고 Context Memory Storage로 고쳐 CMX라 부른다. CMX는 결국 DRAM을 안 쓰고 SSD를 쓰겠다는 개념 — HBM에 다 저장 못 하는 KV캐시를 SSD까지 내려 쓰겠다는 것. 비휘발성이라 느리고 용량 큰 낸드 플래시를 마치 DRAM처럼 휘발성처럼 쓴다. 화자는 이것이 앞의 Supermemory와는 **다른 개념**임을 강조한다.

> "슈퍼 메모리는 어떤 기억을 골라야 될까 이거를 고민하는 층이고 여기는 이미 골라져 가지고 나온 콘텍스트를 디코드 하기 전에 KV캐시를 어떻게 잘 준비를 해 둘까 요걸 고민한 층" [15:27]

즉 Supermemory는 애초에 어떤 정보를 고르냐, 엔비디아 KV캐시·LPX·CPX는 이미 고른 현재 세션의 기억·배경 지식을 토큰 낼 때마다 빨리 내주기 위한 디코드(FFN) 대응이라는 것.

### [16:03–17:38] 전체 계층 정리 — 메모리 티어링
계층을 나누면: 의미론적으로 메모리를 갖는 부분(어떤 기억을 쓸까라는 알고리즘적 설계)은 **시멘틱 메모리 = Supermemory**가 담당. 그다음 "이걸 어디서 가져올까"를 고민하는 **오케스트레이션 레이어**는 다이나모(NVIDIA Dynamo 계열, [불명확]) 같은 소프트웨어가 담당. 이렇게 가져온 뒤 **CMX**의 SSD에 컨텍스트 메모리를 담고, 실제 대답할 때 **LPX/LPU**로 디코드의 FFN을 저지연·로우 레이턴시로 바로바로 처리한다. 하드웨어적으로는 확실히 **메모리 티어링**이 진행 중 — HBM → LPDDR → GDDR → 낸드 플래시/SSD. GDDR은 루빈엔 있으나 다음 로드맵(파인만 계열로 추정, [불명확])에선 빠질 수도 있다. SSD가 메모리 계층으로서 컴퓨팅에도 쓸 수 있는 영역이 됐고, 이것이 블루필드 4에 들어간다. 결국 반도체 업체들은 서로 다른 메모리에서 KV캐시를 얼마나 잘 가져올지의 관점에서 인프라를 고민하며 역할을 분리하고 있다는 것.

### [17:38–19:00] 종합 — 층위 분리가 핵심 [의견]
정리하면 결국 **층위가 나뉜다**는 게 가장 중요한 포인트라고 화자는 강조한다. Supermemory는 실험적 테스트지만, 에이전틱하게 장기적으로 대화할 때 KV캐시 한계를 벗어나 여러 에이전트로 각각을 의미론적으로 저장하면 훨씬 정확도 높게 답한다는 걸 보였다는 점에서 의미가 있고, KV캐시를 최대한 저장해 층위를 나눠 메모리 티어링하는 것은 반도체 업체들이 할 일. 이 모든 게 혁명적으로 다 일어난다고 단정하는 건 아니지만, 이런 결과가 계속 나오고 있다는 것. RAG로 갈 것이라 모두가 기대했던 적도 있지만 새 기술이 나오며 접근법이 바뀐다. 상황이 바뀌면 시스템도 바뀔 것이고, 멀티턴으로 세션을 유지해야 하는 상황에선 Supermemory 같은 방식이 별도 익스터널 스토리지(느리지만 대용량인 SSD·하드로 시멘틱 정보를 담는)로 확장될 수 있다고 본다. [의견]

### [19:00–19:46] 마무리 — 에이전틱 AI의 가속 [의견]
소프트웨어에서 어떤 기억을 고를지 하는 메모리 자체도 에이전틱·의미론적으로 추론 가능한 형태로 분화되고 있고, 하드웨어에서도 분화가 이뤄지듯 각각 어떤 기억을 꺼낼지의 고민, 꺼낸 뒤 얼마나 빨리 대응할지, 대답할 때 LPU·SRAM으로 빠르게 내는 것들이 종합되고 있다. 그래서 실시간으로 계속 하고 있는 에이전틱 AI가 점차 완성형으로 가고 있는 것 아닌가, 이 결과가 의미가 깊다고 평가. ASMR은 **일주일 뒤 오픈소스로 공개** 예정이라 화자도 직접 테스트해 볼 계획이며, 하드웨어·소프트웨어·오케스트레이션 알고리즘이 모두 에이전트에 타게팅해 만들어지다 보니 에이전트 AI가 생각보다 훨씬 빨리 다가오고 있는 것 아닌가 하며 마무리한다. [의견]
