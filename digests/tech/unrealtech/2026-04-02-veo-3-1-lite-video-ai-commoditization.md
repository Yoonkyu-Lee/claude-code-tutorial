## 요약 (주제별)

### 1. 이번 발표의 핵심 — Veo 3.1 Lite Preview
- 구글이 3월 31일 Veo 3.1의 라이트(Lite) 버전 프리뷰를 출시. 구글 공식 릴리즈 노트는 이를 "가장 비용 효율적인 비디오 생성 모델"로, "rapid iteration / high-volume application"용으로 소개(화자 인용).
- 같은 날 구글은 기존 Gemini 2.5 Flash-lite Preview 09-2025를 종료하고 Gemini 3.1 Flash-lite Preview로 전환. 즉 영상만의 이벤트가 아니라 구글 전체 API 포트폴리오를 라이트/플래시-라이트(대량 생성) 중심으로 재편한 것이라는 게 화자 해석 [의견].
- Veo는 이미 Flow 플로우에서 사용 가능. 프롬프트·시작/종료 프레임 지정, Quality/Fast/Lite 중 선택, 소모 크레딧·생성 개수·가로세로 비율 선택 가능.

### 2. 실사용 비교 (Lite vs Fast vs Quality)
- 화자가 Nano Banana 2로 "무술 포즈의 귀여운 강아지와 고양이" "밤새 춤추는 사람들의 타임랩스" 등을 Quality/Fast/Lite 세 버전으로 각각 생성해 비교.
- 화자 체감 [의견]: 세 버전 간 품질 차이가 "거의 없다". Fast는 훨씬 빠르고, Lite도 음악·배경 인물이 자연스러워 "오히려 Lite가 더 사실적으로 보이기도" 함.
- 단, Lite는 중간에 서사(스토리 개연성)를 넣기보다 배경을 통째로 전환(밤→낮)하는 식 — 아직 완벽한 구성은 아니지만 대량 클립 생성용으로는 충분하다는 평 [의견].

### 3. 가격 구조 (화자 인용 수치)
- 기존 Standard/Quality: 720p·1080p 기준 초당 $0.4(약 600~700원), 4K Standard 초당 $0.6(약 900원) → 4K로 10초면 약 1만 원.
- 기존 Fast: 1080p 초당 $0.15(약 200~300원), Fast 4K 초당 $0.35(약 500원) → 10초 약 5,000원.
- 신규 Lite: 4K 미지원, 720p/1080p만. 각각 초당 $0.05·$0.08. 10초 기준 약 $0.5(약 700원). 기존 대비 약 1/10~1/12 수준으로 급락.
- (주장/미검증) 위 단가는 화자가 릴리즈/가격표를 보고 전한 값. Lite는 오디오는 지원하되 4K·익스텐션(extension)은 미지원.

### 4. Lite의 진짜 의미 — 영상 AI의 산업재화/양산화
- 화자 핵심 논지 [의견]: "최고 고품질 모델이 나왔다"보다 "라이트 버전이 등장했다"가 훨씬 중요. 영상 생성이 프리미엄 데모 경쟁에서 벗어나 대량 API 호출(양산)로 소비되는 "산업재(commodity)"로 전환되는 신호.
- 텍스트 LLM이 토큰 단가로 승부하듯, 영상 AI도 토큰/API로 대량 소비되는 구조로 이동. 다만 영상은 해상도·초당 과금이라 재생성·편집 시 비용이 급격히 커짐 → 그래서 저단가 라인업이 수요를 끌어당김.
- Lite는 단순 "싸게 판다"가 아니라, 고해상도·고부가 기능을 아예 제외하고 대량 생성용으로 스큐를 짠 별도 상품이라는 추정. 창작자용 단발 도구가 아니라 반복·자동화 애플리케이션(마케팅 자동화, 에이전트 파이프라인, B-roll/배경) 타깃.

### 5. TPU와 컴퓨팅 경제학 (화자 추정)
- 네이티브 오디오까지 붙은 영상 생성은 무거운 멀티모달리티라 추론에 막대한 컴퓨팅 소모. 구글이 TPU를 일찍 도입한 효율 이점이 여기서 드러난다는 추정 [의견].
- Lite는 4K·익스텐션 같은 가장 비싼 추론 경로를 덜어내고 전성비(전력 효율) 쪽으로 돌린 버전으로 추정.
- 다만 화자는 어떤 칩으로 Veo Lite를 돌렸는지는 공식 문서에 없다고 명시 [불명확]. Trillium·Ironwood 언급(처리량 3배, 효율 67% 향상, Ironwood는 추론 특화 — 화자 인용). 올해 Google I/O 기대.
- 화자 균형 관점: TPU가 모든 걸 커버하진 못하며, "엔비디아 아성을 무너뜨린다"에는 동의하지 않음. TPU는 그 목적이 아니라, 구글이 원하는 특정 애플리케이션(Lite 등)을 자체 설계 칩으로 아주 싸게 만드는 강점의 근거.

### 6. 소라(Sora) 공백과 구글의 포지셔닝
- 확정 사실(화자 인용): OpenAI Sora의 웹/앱 서비스는 4월 26일 종료, API 호출은 올해 9월 24일 종료 예정.
- 소라 종료 이유는 재무 이슈/IPO 대비 등 추정이 많다는 소개 [의견]. 화자 해석: 단순 제품 실패라기보다 자본 시장·기업용 수익 모델(슈퍼앱)에 집중하려는 선택.
- 소라가 비운 자리를 Veo 3.1(및 Lite)이 흡수. 이미 소라 대신 Veo 3.1을 쓰는 경우가 많다는 관찰 [의견]. Lite는 낮은 단가로 영상 AI 시장 점유율을 "잡아먹으려는" 전략 상품이라는 해석.

### 7. capex·인프라 가동률 (화자 인용 + 해석)
- 구글 2026 가이드: capex 약 $1,750억~$1,850억(2025 대비 거의 2배) — 데이터센터·칩 투자 (화자 인용).
- 로이터 인용: 구글 클라우드 매출 48% 성장. 수요가 이미 크다는 신호.
- 화자 논지 [의견]: 데이터센터 투자는 이미 들어간 고정비 → 가동률(utilization)이 핵심 지표. 칩 효율 개선 → 가격 하락 → 저가 라인업 → 호출 증가 → 데이터센터 가동↑ → 고정비 흡수, 이 순환이 매끄러워야 투자 회수·수익 확산 가능. 올해 진행을 지켜봐야 함.

### 8. OpenAI vs 구글 — 갈라진 행보
- 화자 대비 [의견]: OpenAI는 연산비 큰 영상 전선을 접고 코딩·기업용 슈퍼앱에 집중(수익성 불확실한 소라에서 발을 뺌). 계속 투자금을 받아야 하는 구조.
- 구글은 클라우드·광고·검색·유튜브로 이미 수익을 내고, 깔아둔 인프라를 수익화하는 입장이라 영상을 오히려 확장. 풀스택으로 준비돼 소라 공백을 적극 흡수 가능.

### 9. 에이전틱 수요라는 배경 근거
- 최근 AI 수요는 harness 엔지니어링, 작은 모델 재검증, 여러 모델 오케스트레이션/오토메이션 등 에이전틱 형태로 이동. 너무 무거운 영상 생성으로 토큰을 소모하면 안 쓰게 되므로, Lite가 시대 요구에 맞는다는 논지 [의견].
- 근거로 인용한 수치: Anthropic(Claude Code 기준) 매출 25억 달러 초과, 활성 사용자가 2026-01-01 이후 2배, 기업용·퍼블릭 커밋 증가. OpenAI는 작년 11월 기준 비즈니스 고객 100만 명 초과, 월 매출 약 20억 달러 (모두 화자 인용, 미검증). [불명확: 자막상 "2020년 2월"로 들리나 문맥상 2026년 2월로 추정]

### [02:42–03:09] 품질 비교 1 — 거의 차이 없음
Quality 버전은 중간 모먼트와 뒤 사람들이 사라지는 모습 등이 있고, Fast 버전은 "거의 차이가 없이" 훨씬 빠르게 생성됨. Lite 버전도 괜찮고 음악도 제대로 나오며, 화자는 오히려 지금 생성 결과만 보면 Lite가 훨씬 사실적으로 보이고 뒤 사람들도 자연스럽게 나온다고 평가 [의견].

### [03:09–04:00] 품질 비교 2 — 밤새 춤추는 타임랩스
"밤새도록 춤추는 사람들의 모습을 타임랩스로 표현"하는 프롬프트로, 마지막 장면이 다른 무대로 바뀌는 스토리 개연성을 확인. Quality는 음악이 잘 나오고 사람들이 자연스럽게 춤추며 뒤로 배도 자연스럽게 지나감. Fast도 무대가 채워지는 모습이 괜찮음. Lite는 노래가 더 신나지만, 중간에 서사를 넣기보다 뒤 배경을 통째로 전환해 밤→낮으로 바꾸는 식.

Lite는 아직 완벽한 구성은 아니지만 볼륨(대량 생성)용이며, 토큰 자체를 싸게 만들어 많은 사람이 쓰게 하려는 컨셉. 가격 인용:
- Standard/Quality: 720p·1080p 초당 $0.4(약 600~700원), 4K Standard 초당 $0.6(약 900원) → 4K 10초면 약 1만 원.

- 신규 Lite: 4K 미지원, 720p·1080p만, 각각 초당 $0.05·$0.08 → 10초 약 $0.5(약 700원). 기존 대비 약 1/10~1/12로 급락 (주장/미검증).

Lite는 단순히 싸게 파는 개념이 아니라, 고해상도·고부가 기능을 아예 제외하고 대량 생성용으로 스큐를 짠 별도 상품으로 추정 [의견] — 그래서 용어를 따로 만들었을 것. 오디오는 지원하되 4K·익스텐션은 미지원. "과금 많이 나오는 건 사람들이 안 쓸 테니, 너네 많이 써서 많이 테스트해라" 쪽으로 라인업/포트폴리오를 다시 짠 것으로 봄. 자주 만드는 사람, 낮은 레이턴시, 자동화 중심 API 수요를 끌어당기려는 것이며, 빠르게 이것저것 만드는 애플리케이션에 유용.

### [05:54–06:41] 핵심 프레임 — 영상 생성의 산업재화
화자는 "최고 고품질 모델이 나왔다"보다 "이런 라이트 버전이 등장했다"가 훨씬 중요하다고 강조 [의견]. 영상 생성이 이제 재료가 "산업재화"되기 시작한다는 것 — 에이전틱 AI가 LLM을 토큰으로 막 쓰듯 영상 AI도 토큰/API로 많이 쓰게 지원 가능. 텍스트 모델도 단가가 핵심이라 조금만 쓰면 토큰이 차는데, 영상은 초당 과금이라 제시 횟수·후속 편집 등 비용이 급격히 커짐. 그래서 라이트 버전을 넣은 것은 가끔 만드는 창작자용 도구가 아니라 반복 생성 애플리케이션을 사격수(타깃)로 한 것.

### [06:41–07:16] 기술적 배경 — 멀티모달리티와 TPU
영상 생성 비용은 품질 하나로 결정되지 않음. 네이티브 오디오까지 완전히 붙은 비디오 생성은 LLM의 텍스트 생성보다 훨씬 무거운 멀티모달리티라 추론 과정에 엄청난 컴퓨팅을 씀. 구글이 TPU를 빨리 도입해 경량화한 부분이 여기서 드러나는 것이 아닌가 하는 추정 [의견]. 메모리·대역폭이 동시에 커지는데, Lite는 4K·익스텐션 같은 컴퓨팅을 줄이고 전성비 효율 쪽으로 돌려 가장 비싼 추론 경로를 덜어낸 버전으로 추정.

### [07:16–08:21] 소라 공백과 타깃 — 마케팅 자동화·에이전트
Lite의 핵심 타깃은 영화 같은 제대로 된 한 편이 아니라, 대량 생성으로 마케팅 자동화나 빠르게 불려가는 에이전트 AI 등 빠른 수요 자동화가 필요한 상황. 영상 생성 AI가 많지 않은 가운데 딱 빈 포지션이 소라의 서비스 중지. 확정 사실: 소라 웹/앱 서비스는 4월 26일 종료, API 호출은 올해 9월 24일 종료 예정 (화자 인용). 종료 이유로 재무 이슈/IPO 대비 추정이 많다는 소개 [의견]. 외부 정보를 종합하면 단순 제품 실패라기보다 자본 시장과 기업용 수익 모델(슈퍼앱)에 더 집중하려는 것으로 해석. 소라가 멈춘 지금 그 수요를 바로 당길 수 있는데 마침 라이트 버전이 나왔다는 것.

### [08:21–09:18] capex·클라우드 매출 — 가동률 논리
구글 2026 가이드로 투자 금액(capex)이 거의 $1,750억~$1,850억까지 간다는 얘기(2025 대비 거의 2배)로, 이미 데이터센터·칩(TPU/GPU)에 투자가 들어가고 있음 (화자 인용). TPU가 효율적이고 전성비가 좋다면 놀리지 말고 계속 돌려야 함 — 최고 모델을 만드는 것뿐 아니라 인프라를 얼마나 많이 돌리고 추론에 쓰느냐가 수익 확보 전략일 수 있음 [의견]. 로이터 소식으로 구글 클라우드 매출이 48% 성장(화자 인용), 수요가 이미 많음. 깔아둔 AI 인프라로 서비스를 확보해야 하는데 마침 소라가 없어지고 Veo 3을 많이 쓰니, 그걸 가져와 자리를 선점하겠다는 의지가 보인다는 해석.

### [09:18–09:54] 다시 강조 — 전략 상품으로서의 Lite
라이트는 단순 가격 인하로 염가에 하겠다는 느낌이 아니라는 재강조 [의견]. 데이터센터 투자는 이미 고정비가 들어간 것이라 반드시 돌려야 하고, 가동률 자체가 핵심 지표. 수요가 약하면 네트워크·전력이 전부 돈으로 흘러 버림. 소라의 빈자리를 차지하면서 Veo 3.1이 워낙 좋아 이미 소라 대신 Veo 3.1을 쓰는 경우도 많음. 저가 라인업을 단순 추가한 게 아니라 낮은 단가로 영상 AI 시장 점유율을 아예 잡아먹으려는 전략 상품으로 이해할 수 있음.

### [09:54–10:29] OpenAI vs 구글 — 갈라진 행보
OpenAI와 구글의 행보가 다름 [의견]. OpenAI는 연산비가 엄청 큰 영상 전선을 접고 코딩·기업용 슈퍼앱으로 집중해 포지션을 잡음 — 모든 걸 다 할 수 없고 계속 투자금을 받아야 하기 때문. 반면 구글은 이미 클라우드 매출 성장, 광고 수익, 검색, 유튜브 등으로 엄청난 수익을 내고 있어, 이미 많이 깔아놓은 인프라를 수익화하는 입장. 그래서 같은 시장에서도 한쪽(OpenAI)은 영상을 버렸고 구글은 더 확장하며 이런 모델을 내놓았다는 해석.

### [10:29–11:03] 에이전틱 시대 요구에 맞는 Lite
최근 AI 수요는 harness 엔지니어링, 작은 모델 재검증, 여러 모델을 호출해 오토메이션/오케스트레이션하는 에이전틱 형태로 이동 중. 너무 무거운 동영상 생성 AI로 토큰을 소비해 버리면 사람들이 안 쓸 것이므로, 시대가 요구하는 시점에 라이트 버전으로 성능을 어느 정도 만족시키면서도 호출 횟수가 많아도 개발자에게 부담스럽지 않은 포지션을 가져간 것으로 예상 [의견].

### [11:03–12:04] 근거 — 에이전틱 API의 수익 증명 (수치 인용)
그 근거로, Anthropic이 (Claude Code 기준) 매출이 25억 달러를 넘었다고 보도됐다고 인용 [불명확: 자막상 "2020년 2월"로 들리나 문맥상 2026년 2월 추정]. 주간 활성 사용자가 2026-01-01 이후 두 배 늘고, 기업용도 크게 늘고, 퍼블릭 커밋도 많이 늘었다는 것 — 에이전트형 API 호출 자체가 모델 제작사와 서빙 기업의 확실한 수익 포지션임을 보여줬다는 논지. OpenAI도 작년 11월 기준 비즈니스 고객이 100만 명을 넘고 월 매출도 약 20억 달러라는 얘기가 나옴 (모두 화자 인용, 미검증). 소비자 채팅을 넘어 에이전틱으로 넘어가며 사용자·컴퓨팅·기업·개발자가 API를 활용하는 형태로 가면, Veo 3.1 Lite도 B-roll·배경 같은 워크로드를 상당히 흡수할 수 있다는 전망.

### [12:04–12:47] TPU 세대·효율 (인용) + 라이트의 기반
어떤 칩으로 Veo Lite를 돌렸는지는 공식 문서에 안 나와 있다고 명시 [불명확]. 다만 Trillium, Ironwood 같은 칩의 효율이 계속 어마어마하게 늘고 있으며, 처리량이 3배 늘었는데 효율은 67% 향상, Ironwood부터는 본격적으로 추론에 초점을 둔다는 얘기가 있었다고 인용. 올해 Google I/O도 이런 부분에서 기대됨. 영상 생성은 대표적으로 컴퓨팅을 빡세게 요구하는 고비용 추론 워크로드라 효율적인 프로세서가 필요한데, 구글은 자체 설계 가능한 TPU가 있어 자신들에게 맞는 형태를 만들 수 있음. TPU가 모든 걸 커버하진 못해도 이런 라이트 버전을 만들 수 있는 기반이 된다는 것.

### [12:47–13:27] 구글의 경쟁력 + 주의할 순환 구조
사람들이 구글이 경쟁력을 가진다고 말하는 이유는 이런 칩이 진화할수록 API 단가가 인하될 수 있기 때문 [의견]. 주의할 점: 한쪽에서 초대형 capex를 집행해 계속 투자하고, 다른 쪽에서 라이트 같은 볼륨 상품을 내는 것은 하나의 구조. 칩 효율 개선 → 가격 하락 → 저가 라인업 → 호출 증가 → 데이터센터 가동 증가 → 고정비 흡수. 이 순환이 매끄럽게 만들어져야 현재 투자 규모를 잘 회수하며 수익을 확산할 수 있으니, 올해 이게 어떻게 진행되는지를 잘 봐야 함.

### [13:27–14:14] 풀스택 구글의 흡수 능력 + 양산화
OpenAI가 수익성이 불확실한 소라에서 발을 뺀 반면, 구글은 풀스택으로 다 준비하고 돈도 벌고 있어 그 수요를 적극적으로 흡수할 여건이 마련됨 — 그래서 적극적으로 API 호출용으로 내놓은 것으로 예상 [의견]. 결국 AI 기반 영상 생성 시장은 새롭게 나타난 것이고, 에이전트가 동작하는 자동화 파이프라인에 자연스럽게 통합되면서도 토큰을 너무 많이 소모하지 않는 형태로 갈 수 있다는 기대. 즉 대량 생산("양산")이 가능할 만큼 가격이 떨어졌다고 볼 수 있음.

### [14:14–14:58] 정리 — 인프라 전쟁이 핵심
"더 좋은 모델 만들었어요"라는 게임은 지나갔고, 이제 더 효율적인 인프라를 누가 가지느냐가 관건 [의견]. 구글 TPU 도입 때 "엔비디아 아성을 무너뜨린다"는 얘기가 나왔지만, 화자는 이에 동의하지 않음 — TPU 자체가 엔비디아 아성을 무너뜨리려 나온 게 아니기 때문. 구글이 TPU를 갖고 있다는 것 자체가 엄청난 자산이며, 라이트 버전 같은 자신들이 원하는 애플리케이션을 TPU로 만들면 엄청 싸게 제공할 수 있다는 강점. 앞으로 더 많은 API 호출을 처리할 인프라를 어떻게 구축하느냐, 즉 "인프라 전쟁"이 핵심이며 이 경쟁을 지켜보면 재밌을 것이라며 마무리.
