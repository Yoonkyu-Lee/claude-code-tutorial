## 요약 (주제별)

### 1. 행사 개요와 화자의 관전 포인트
- 화자(안될공학)가 OpenAI Founder Day(Seoul) 현장에 참석해 리포트한 영상.
- 화자가 보고 싶었던 두 가지: (1) OpenAI가 한국 스타트업들과 실제로 어떤 일을 하는지, (2) "AI에 돈을 쓰는데 성과가 잘 안 보인다"는 업계 화두에 대해 OpenAI가 어떤 방향으로 움직이는지.

### 2. 토큰 맥싱 vs 아웃컴 맥싱 (행사의 핵심 논쟁)
- 청중 질문: 비싼 모델로 토큰을 많이 태우는 시대는 지났고, 저렴한 모델을 조합해 결과만 잘 내면 되는 시대 아니냐. OpenRouter처럼 싼 모델을 묶어 프론티어 모델을 이기는 연구도 나온다.
- [의견/화자] 이 질문은 최근 팔란티어 CEO 알렉스 카프가 OpenAI를 겨냥해 던진 "토큰 맥싱" 비판(AI 사용량 증가가 곧 생산성은 아니다)과 정확히 같은 맥락이라고 화자는 해석.
- OpenAI 최고연구책임자(CRO로 지칭, 이름 마크 챈으로 전언 — [불명확: 자막 음차 "마크체이"]) 답변 요지: 모든 서비스가 제일 크고 똑똑한 모델을 필요로 하진 않는다. 중요한 건 토큰을 얼마나 썼느냐가 아니라 어떤 결과를 냈느냐이며, 토큰은 지능을 가늠하는 대리 지표일 뿐이다.
- 단, 그는 "가장 똑똑한 모델에 대한 수요는 앞으로도 늘 있다 — 문제가 정말 복잡해지면 결국 최고 지능을 원하게 된다"고 덧붙임.
- [의견/화자] OpenAI 최고연구책임자가 팔란티어 비판의 핵심을 상당 부분 인정한 셈이며, 그러면서도 최고 모델 수요는 사라지지 않는다고 방어한 점이 흥미롭다고 평가.

### 3. 모델 성능과 기업 생산성 사이의 간격
- [의견/화자] 모델 성능이 좋아진다고 기업 생산성이 자동으로 오르지 않는다. 개발자가 코드를 빨리 쓰는 것과 회사 제품 출시 주기가 짧아지는 것은 별개.
- 생산성이 실제로 오르려면 모델 앞뒤로 필요한 것들: 어떤 업무를 AI에 맡길지 결정, 회사 데이터의 안전한 연결, 결과 오류 검증 과정, 누가 어떤 권한으로 AI를 쓰는지 관리, 비용 폭발 방지를 위한 요청별 모델 배치, 기존 직원 업무 방식 변화.

### 4. 한국 스타트업의 구조적 비용 부담
- 청중 질문: 샌프란시스코와 한국의 임금 차이는 약 2배인데 AI API 비용은 달러로 고정. 한국 스타트업의 AI 비용 부담이 미국보다 구조적으로 크지 않냐.
- [의견/화자] 이는 단순 환율 문제가 아니라 ROI 계산 구조의 문제. AI가 줄여주는 인건비는 한국 임금 기준, API 비용은 달러 기준이라, 같은 작업을 자동화해도 미국은 절감 인건비가 커 ROI가 쉽게 나오지만 한국은 같은 API 비용으로 ROI 증명이 구조적으로 더 어렵다.
- OpenAI 답변(구체적): 지역별 가격 민감도를 인지하고 한국 같은 시장에 맞춘 가성비 오퍼링 제공, 창업자용 대량 무료 API 크레딧 지원, 최상위 모델이 아니어도 가성비 좋은 모델 조합으로 프론티어급에 가까운 앱을 저렴하게 구축 가능, 같은 수준 모델을 매년 10배 이상 저렴하게 제공 중이라고 강조. (수치는 OpenAI 측 주장/미검증)
- [의견/화자] 그래도 가격 인하·크레딧과, 한국 기업이 달러 기준 비용으로 실제 ROI를 증명하는 것은 여전히 별개 문제라고 지적.

### 5. 스타트업 사례 (1) 플리토 — 글로벌 데이터 루프
- 언어 데이터 기반으로 글로벌 커뮤니케이션 문제를 해결하는 상장사. 표면적으로는 AI 번역 품질 개선 사례.
- [의견/화자] 더 중요한 것은 번역 속도가 아니라, 여러 국가 고객의 목소리를 실시간 이해해 제품 개선에 다시 반영하는 속도. AI가 이 흐름에 들어가면 번역이 비용 절감 도구를 넘어 글로벌 고객 데이터를 하나의 제품 개발 루프로 묶는 인프라가 된다.
- OpenAI 입장에서 플리토는 범용 언어 모델 능력을 지역별 데이터·산업 경험에 연결해 줄 파트너로서 중요.

### 6. 스타트업 사례 (2) 프로토파이 — 복잡도 기반 모델 라우팅
- 디자이너가 코딩 없이 고품질 인터랙션 프로토타입을 만드는 툴. 전 세계 100만 명 이상 사용, OpenAI·구글·마이크로소프트·폭스바겐 등에서도 사용된다고 발표 (주장).
- 다루는 문제: AI로 만드는 속도는 빨라졌지만 그 결과물이 실제 제품 수준 품질을 갖추는지 어떻게 보장하느냐.
- OpenAI로부터 단순 API 사용을 넘어 직접 기술 지원(생성 품질 개선, 비용 최적화, 모델 업데이트 때마다 성능이 흔들리지 않게 하는 방법)을 받고 있다고 밝힘.
- 발표 핵심: 요청 복잡도를 로우/미디엄/하이로 나눠 그에 맞는 다른 에이전트·다른 모델을 호출 → 최소 비용으로 최대 결과. [의견/화자] 이것이 토큰 맥싱이 아니라 아웃컴 맥싱의 실제 구현.
- [의견/화자] 동시에 든 생각: 100만 명이 쓰는 정교한 회사조차 어떤 요청에 어떤 모델이 효율적인지를 OpenAI의 직접 개입 없이는 스스로 최적화하기 어려웠다는 뜻. 팔란티어가 비판한 구조가 바로 이것(고객이 직접 알기 어렵고 공급자가 개입해야 함)이며, 아웃컴 맥싱 실현 경로가 결국 OpenAI 의존을 키우는 방식이라는 점은 긍정·구조적 문제 양면으로 볼 수 있다.

### 7. AWS 파트너십 — 기업 인프라 진입과 락인
- 마지막 세션은 AWS 파트너십. 6월 1일부로 GPT 5.5 / 5.4 코덱스 모델이 AWS Bedrock 위에 올라옴 ([불명확: 모델명 자막 음차 "GPT 5.5 5.4 코덱스"]). Bedrock은 여러 AI 모델을 하나의 플랫폼에서 제공하는 AWS 통합 서비스.
- [의견/화자] 기업 AI 도입을 막는 장벽은 기술보다 조직 쪽: 보안팀은 데이터가 외부망 타는 걸 꺼리고, IT팀은 AI용 권한 체계 신설을 부담스러워하고, 재무팀은 고정 지출 요금제를 싫어한다. Bedrock은 이 셋을 동시에 건드림 — 내부망 프라이빗 연결, 기존 권한 관리 체계 그대로 적용, 실제 쓴 만큼만 과금.
- [의견/화자] OpenAI 혼자 낮추기 어려웠던 장벽을 AWS가 기업들과 쌓아온 신뢰 위에서 푸는 구조. 그러나 편리함과 의존성은 방향은 달라도 결과가 같을 수 있어, 모델이 AWS 인프라·기업 업무 프로세스에 깊이 들어갈수록 다른 모델로 갈아타는 비용(락인)이 올라간다.

### 8. 화자의 종합 해석 — 두 흐름의 동시 진행
- [의견/화자] OpenRouter식 "싼 모델 여러 개로 프론티어를 이긴다"가 가능하면 기업은 특정 회사 모델에 매일 이유가 없어진다(복잡도 따라 더 싼 모델로 갈아타면 됨). 하지만 프로토파이의 복잡도 라우팅은 그 갈아타기를 OpenAI가 직접 설계해 주는 방식이었고, AWS 파트너십도 OpenAI 모델을 기업 내부에 깊이 심는 구조.
- [의견/화자] 즉 "모델 갈아탈 자유를 주는 흐름"과 "모델을 더 깊이 심어 갈아타기 어렵게 만드는 흐름"이 같은 무대에서 동시에 벌어짐. 이 파트너십을 단순한 생태계 지원으로만 읽으면 이면의 락인을 놓치게 된다.
- [의견/화자] 세 장면(플리토의 글로벌 데이터 루프, 프로토파이의 복잡도 기반 라우팅, AWS를 통한 기업 인프라 진입)은 모두 한 방향 — 토큰을 더 태우게 하는 게 아니라 모델이 실제 업무에서 결과를 내게 하는 것 — 을 가리킨다. 마크 챈이 무대에서 인정한 말이 빈말이 아니라는 신호로 해석.

### 9. 남는 질문 (화자가 던진 관전 포인트)
- [의견/화자] 아웃컴 맥싱 실현 경로가 결국 OpenAI 의존 구조라면, 팔란티어가 지적한 문제가 해결된 건지 형태만 바뀐 건지 생각해 볼 필요.
- 한국 기업 비용 부담도 가격·크레딧으로 일부 덜어준다 했지만 달러 기준 ROI 증명 구조 자체가 사라지는 건 아님.
- 앞으로 볼 것은 데모가 아니라 결과: 프로토파이 라우팅이 실제 고객 이탈을 줄이는지, 플리토 데이터 루프가 제품 개선 속도를 높이는지, 아웃컴이 중요하다고 인정한 OpenAI가 토큰 기반 수익 구조를 실제로 어떻게 바꾸는지.

## 3줄 오리엔테이션
- OpenAI Founder Day Seoul 현장 리포트: 행사의 숨은 주제는 "토큰 맥싱(비싼 모델로 토큰 많이 태우기) vs 아웃컴 맥싱(결과 중심)" 논쟁이었고, OpenAI 최고연구책임자가 팔란티어의 토큰 맥싱 비판을 상당 부분 인정했다.
- 화자는 플리토(글로벌 데이터 루프)·프로토파이(복잡도 기반 모델 라우팅)·AWS Bedrock 파트너십 세 장면을 통해, OpenAI가 "모델 밖"의 실제 업무 결과와 기업 인프라로 경쟁 축을 옮기고 있다고 읽는다.
- 다만 그 아웃컴 맥싱 실현 경로가 OpenAI 의존·락인을 키우는 구조라는 양면성, 그리고 한국 기업의 달러 기준 ROI 부담이 남는 질문으로 제시된다.

### [00:00–00:11] 오프닝 — 현장 도착
화자(안될공학)가 인사하며, 오늘 OpenAI Founder Day에 참석하러 왔고 어떤 흥미로운 이야기가 나오는지 잘 살펴 전달하겠다고 밝힌다.

### [00:12–00:29] 관전 포인트 두 가지
화자가 이번 행사에서 보고 싶었던 것은 크게 두 가지라고 정리한다. 하나는 OpenAI가 한국 스타트업들과 실제로 어떤 일을 하고 있는지, 다른 하나는 요즘 AI 업계에서 계속 나오는 "AI에 돈을 쓰는데 성과가 잘 안 보인다"는 얘기에 대해 OpenAI가 어떤 방향으로 움직이는지였다.

### [00:29–00:47] 청중의 토큰 맥싱 질문
행사 중간 청중 한 명이 마이크를 잡고 마침 그 질문을 먼저 꺼냈다.
> "이제 무조건 비싼 모델로 토큰을 많이 태우는 시대는 갔고, 저렴한 모델을 조합해서 결과만 잘 내면 되는 시대 아닌가요?" [00:35]
실제로 OpenRouter 같은 데선 싼 모델들을 묶어 강력한 프론티어 모델을 이기는 연구도 나오고 있다는 부연이 붙는다.

### [00:47–01:04] 팔란티어의 "토큰 맥싱" 비판과의 연결
[의견/화자] 이 질문은 최근 팔란티어 CEO 알렉스 카프가 OpenAI를 직접 겨냥해 꺼낸 "토큰 맥싱" 비판과 정확히 맞닿아 있다고 화자는 해석한다. AI 사용량이 늘어나는 게 곧 생산성은 아닐 수 있다는 문제의식으로, 그 내용은 지난 영상에서 다뤘고 행사장에서 마침 같은 맥락의 질문이 나온 것이라고 설명한다.

### [01:04–01:24] OpenAI 최고연구책임자의 답변
이 질문을 받은 사람은 OpenAI의 최고 연구자(CRO로 지칭) 마크 챈 [불명확: 자막 음차 "마크체이"]이었다. 답변 요지: 솔직히 모든 서비스가 제일 크고 똑똑한 모델을 필요로 하진 않는다. 중요한 건 토큰을 얼마나 썼느냐가 아니라 어떤 결과를 냈느냐이며, 토큰은 지능을 가늠하는 대리 지표일 뿐이다.

### [01:24–01:45] "그래도 최고 모델 수요는 남는다"
다만 그는 이런 말도 덧붙였다.
> "가장 똑똑한 모델에 대한 수요는 앞으로도 늘 있을 겁니다. 문제가 정말 복잡해지면 결국 최고 지능을 원하게 되거든요." [01:25]
[의견/화자] 팔란티어가 OpenAI를 향해 던진 비판의 핵심을 최고연구책임자가 상당 부분 인정한 셈이며, 그러면서도 최고 모델 수요는 사라지지 않는다고 덧붙인 점이 흥미로웠다고 평한다.

### [01:45–02:08] 모델 성능 ≠ 기업 생산성
[의견/화자] 그렇다면 OpenAI는 실제로 어떻게 움직이는가. 화자는 모델 성능이 좋아진다고 기업 생산성이 자동으로 오르는 건 절대 단순한 문제가 아니라고 짚는다. 개발자가 코드를 조금 빨리 쓰는 것과 회사 제품 출시 주기가 실제로 짧아지는 것은 다른 얘기이며, 모델 성능과 기업 프로덕티비티 사이에는 생각보다 큰 간격이 있다.

### [02:08–02:31] 그 간격을 메우는 데 필요한 것들
[의견/화자] 기업 생산성이 실제로 오르려면 모델 앞뒤로 훨씬 많은 것이 필요하다: 어떤 업무를 AI에게 맡길지 정하기, 회사 데이터를 안전하게 연결하기, 결과가 틀렸을 때 검증하는 과정, 누가 어떤 권한으로 AI를 쓰는지 관리, 비용이 폭발하지 않도록 요청마다 모델을 다르게 배치, 기존 직원들의 업무 방식 변화.

### [02:31–02:51] 한국 기업에 더 날카로운 간격 — 임금·API 비용 질문
[의견/화자] 이 간격은 한국 기업에게 특히 더 날카롭게 작용한다. 마크 챈에게 던져진 질문 중 하나:
> "샌프란시스코와 한국의 임금 차이가 대략 두 배 정도 됩니다. 그런데 AI API 비용은 달러로 고정이에요. 그렇다면 한국 스타트업이 느끼는 AI 비용 부담은 미국보다 구조적으로 훨씬 클 수밖에 없지 않나요?" [02:37]

### [02:51–03:11] ROI 계산의 구조적 불리함
[의견/화자] 이건 단순 환율 얘기가 아니다. AI 도입의 ROI를 계산할 때 AI가 줄여주는 인건비는 한국 임금 기준이고 API 비용은 달러 기준이다. 같은 작업을 자동화해도 미국에서는 절감되는 인건비가 커서 ROI가 쉽게 나오는데, 한국에서는 같은 API 비용을 내고도 ROI를 증명하기가 구조적으로 더 어렵다.

### [03:11–03:41] OpenAI의 구체적 답변 (가격·크레딧·모델 조합)
이 질문에 대한 OpenAI 답변은 꽤 구체적이었다: 지역별 가격 민감도를 인지하고 있고 한국 같은 시장에 맞춘 가성비 좋은 맞춤형 오퍼링을 제공하겠다는 것, 창업자들이 비용 부담 없이 모델을 테스트하도록 대량의 무료 API 크레딧을 지원하고 있다는 것, 가장 비싼 최상위 모델이 아니어도 가성비 좋은 모델들을 조합하면 프론티어급에 가까운 애플리케이션을 저렴하게 만들 수 있다는 것. 또 같은 수준의 모델을 매년 10배 이상 저렴하게 제공하고 있다는 점도 강조했다. (수치·주장은 OpenAI 측 발언/미검증)

### [03:41–03:53] 방향은 분명하나 ROI 증명은 별개
[의견/화자] 방향은 분명하다. 다만 가격을 낮추고 크레딧을 주는 것과, 한국 기업이 달러 기준 비용으로 실제 ROI를 증명하는 것은 여전히 다른 문제라고 못 박고, 이 부분은 뒤에서 다시 짚겠다고 예고한다.

### [03:53–04:01] 국내 스타트업 발표 — 두 사례 예고
행사에서는 OpenAI 기술을 실제 제품에 적용한 국내 스타트업들의 발표도 있었고, 화자가 주목한 두 사례를 소개한다.

### [04:01–04:13] 사례 (1) 플리토 소개
첫 번째는 플리토. 언어 데이터 기반으로 글로벌 커뮤니케이션 문제를 해결하는 상장사다. 표면적으로는 AI 번역 품질 개선 사례처럼 보일 수 있으나, 화자가 보기에 더 중요한 게 따로 있었다.

### [04:13–04:33] 플리토의 진짜 문제 — 고객 목소리의 실시간 반영
[의견/화자] 글로벌 기업의 진짜 문제는 번역 한 문장을 만드는 속도가 아니라, 여러 국가에서 쏟아지는 고객의 목소리를 실시간으로 이해하고 그 데이터를 제품 개선에 다시 반영하는 속도다. 언어 장벽 때문에 해외 고객 피드백이 본사까지 제대로 들어오지 않으면 기업은 시장별 문제를 항상 늦게 알게 된다.

### [04:33–04:49] AI가 만드는 글로벌 데이터 루프
[의견/화자] AI가 이 흐름 안에 들어가면 번역은 비용 절감 도구를 넘어 글로벌 고객 데이터를 하나의 제품 개발 루프로 묶는 인프라가 될 수 있다. OpenAI 입장에서도 플리토 같은 파트너가 중요한 이유는, 범용 언어 모델의 능력을 실제 지역별 데이터·산업 경험에 연결해 줄 수 있는 파트너이기 때문이다.

### [04:49–05:09] 사례 (2) 프로토파이 — 품질 보장 문제
두 번째는 화자가 가장 주목한 장면, 프로토파이라는 회사다. 플리토가 AI를 언어·데이터의 접점에 넣었다면, 프로토파이는 다른 차원의 문제 — AI로 뭔가를 만드는 속도는 빨라졌는데 그 결과물이 실제 제품 수준의 품질을 갖추는지 어떻게 보장하느냐 — 를 다룬다.

### [05:09–05:20] 프로토파이 규모와 사용처
프로토파이는 디자이너가 코딩 없이 고품질 인터랙션 프로토타입을 만들 수 있는 툴이다. 전 세계 100만 명 이상이 사용하고, OpenAI·구글·마이크로소프트·폭스바겐 같은 곳에서도 쓰인다고 한다. (발표 주장)

### [05:20–05:37] OpenAI의 직접 기술 지원
이 회사는 AI 기능을 제품에 넣는 과정에서 OpenAI로부터 직접 기술 지원을 받는다고 밝혔다. 단순히 API를 가져다 쓰는 게 아니라 생성 품질 개선, 비용 최적화, 모델이 업데이트될 때마다 성능이 흔들리지 않게 하는 방법까지 함께 작업하고 있다고 했다.

### [05:37–06:00] 복잡도 기반 라우팅 = 아웃컴 맥싱의 구현
발표 중, 요청의 복잡도에 따라 로우·미디엄·하이로 나눠 그에 맞는 다른 에이전트와 다른 모델을 호출한다는 부분이 있었다. [의견/화자] 맨 처음 청중 질문(토큰을 최대한 많이 쓰는 게 좋은 에이전트가 아닐 수 있다)을 상기시키며, 복잡도를 먼저 판단하고 그에 맞는 모델을 선택해 최소 비용으로 최대 결과를 뽑는 구조가 바로 토큰 맥싱이 아니라 "아웃컴 맥싱"의 실제 구현이라고 짚는다.

### [06:00–06:18] 그러나 — 스스로 최적화 못 하는 구조
[의견/화자] 동시에 화자는 다른 생각도 들었다. 프로토파이는 전 세계 100만 명이 쓰는 정교한 소프트웨어 회사인데, 그런 회사조차 어떤 요청에 어떤 모델을 써야 효율적인지를 OpenAI의 직접 개입 없이는 스스로 최적화하기 어려웠다는 뜻이기도 하다.

### [06:18–06:34] 팔란티어 비판이 정확히 이 구조
[의견/화자] 팔란티어가 비판한 게 정확히 이 구조였다 — 토큰을 효율적으로 쓰는 방법을 고객이 직접 알아내기 어렵고 공급자가 손잡고 개입해 줘야 한다는 것. 아웃컴 맥싱을 실현하는 경로가 결국 OpenAI에게 더 의존하는 방식이라는 점은 긍정적으로 볼 수도, 구조적 문제로 볼 수도 있다.

### [06:34–06:48] AWS 파트너십 — Bedrock에 올라온 OpenAI 모델
마지막 세션은 AWS 파트너십이었다. 6월 1일부로 GPT 5.5 / 5.4 코덱스 모델이 AWS Bedrock 위에 올라왔다 [불명확: 모델명 자막 음차 "GPT 5.5 5.4 코덱스"]. Bedrock은 AWS가 여러 AI 모델을 하나의 플랫폼에서 제공하는 통합 서비스다.

### [06:48–07:06] 기업 도입의 진짜 장벽은 조직
[의견/화자] 기업 AI 도입을 막는 장벽은 기술보다 조직 쪽에 더 많다. 보안팀은 회사 데이터가 외부망을 타는 걸 꺼리고, IT팀은 AI용 권한 체계를 새로 만드는 걸 부담스러워하며, 재무팀은 자석(월/고정) 단위로 고정 지출이 나가는 요금제를 달가워하지 않는다.

### [07:06–07:21] Bedrock이 세 장벽을 동시에 해소
[의견/화자] Bedrock은 이 셋을 한 번에 건드린다: 회사 내부망에서 프라이빗 연결을 하고, 이미 쓰던 권한 관리 체계를 그대로 적용하며, 실제 쓴 만큼만 비용을 낸다. OpenAI 혼자서는 낮추기 어려웠던 장벽을 AWS가 기업들과 오래 쌓아온 신뢰 위에서 푸는 구조다.

### [07:21–07:34] 편리함의 이면 — 락인
[의견/화자] 그런데 편리함과 의존성은 방향은 다르지만 결과가 같을 수 있는 문제다. OpenAI 모델이 AWS 인프라 안에, 그리고 기업 업무 프로세스 안에 깊숙이 들어갈수록 나중에 다른 모델로 갈아타는 비용은 자연스럽게 올라간다.

### [07:34–07:55] 두 흐름의 충돌 — 오토라우터 vs OpenAI 설계 라우팅
[의견/화자] 질문 중 나온 오토라우터 이야기, 즉 저렴한 모델 여러 개를 엮어 비싼 프론티어 모델을 이긴다는 연구가 가능하다면 기업은 굳이 특정 회사 모델에 매일 이유가 없어진다(복잡도에 따라 그때그때 더 싼 모델로 갈아타면 됨). 그런데 프로토파이 사례에서 본 복잡도 라우팅은 그 갈아타기를 OpenAI가 직접 설계해 주는 방식이었다.

### [07:55–08:18] 자유를 주는 흐름 vs 깊이 심는 흐름
[의견/화자] AWS 파트너십도 OpenAI 모델을 기업 내부망·권한 체계·업무 프로세스에 깊숙이 심는 구조다. 모델을 갈아탈 자유를 주는 흐름과, 그 모델을 더 깊이 심어 갈아타기 어렵게 만드는 흐름이 같은 무대에서 동시에 벌어지고 있는 셈이다. 그래서 이 파트너십을 단순히 친절한 생태계 지원으로만 읽으면 그 이면에서 진행되는 락인(가둬 두기)을 놓치게 된다.

### [08:18–08:41] 세 장면이 가리키는 한 방향
[의견/화자] 오늘 본 세 장면 — 플리토의 글로벌 데이터 루프, 프로토파이의 복잡도 기반 모델 라우팅, AWS를 통한 기업 인프라 진입 — 은 모두 한 방향을 가리키는 것 같다. 토큰을 더 태우게 만드는 게 아니라 모델이 실제 업무 안에서 결과를 내도록 하는 일이다. 그렇다면 마크 챈이 무대에서 인정한 말이 진짜 빈말이 아니라는 신호로 받아들여진다.

### [08:41–09:04] 짚고 넘어갈 두 지점
[의견/화자] 다만 짚고 넘어가야 할 부분이 있다. 아웃컴 맥싱을 실현하는 경로가 결국 OpenAI에게 더 의존하는 구조를 통해서라면, 이는 팔란티어가 지적한 문제가 해결된 건지 아니면 형태가 바뀐 건지 생각해 볼 필요가 있다. 그리고 한국 기업의 비용 부담도 OpenAI가 가격과 크레딧으로 일부 덜어주겠다고 했지만, 달러 기준 비용으로 ROI를 증명해야 하는 구조 자체가 사라지는 건 아니다.

### [09:04–09:37] 데모가 아니라 결과 — 남는 질문
[의견/화자] 그래서 앞으로 봐야 할 건 데모가 아니라 결과다. 프로토파이의 복잡도 라우팅이 실제 고객 이탈을 줄이는지, 플리토의 데이터 루프가 제품 개선 속도를 높이는지, 아웃컴이 중요하다고 인정한 OpenAI가 토큰 기반 수익 구조를 실제로 어떻게 바꿔 나가는지. 모델이 똑똑해지는 속도는 계속 빨라지지만, 기업 AI의 승부는 그 모델이 조직 안에서 실제로 무엇을 바꿨느냐에 따라 갈릴 것이다. 비판하는 쪽도 비판받는 쪽도 이제는 같은 곳을 보고 있고, 다음 질문은 과연 누가 먼저 그걸 증명하느냐가 될 것이라고 마무리한다.

### [09:37–09:39] 클로징
화자가 "오늘은 여기까지, 안될공학 패치였습니다"라고 인사하며 영상을 끝맺는다.
