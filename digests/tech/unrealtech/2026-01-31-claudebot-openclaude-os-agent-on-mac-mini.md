## 요약 (주제별)

### 1. 클로드봇/오픈클로의 정체와 이름 변천
- 전 세계 개발자가 열광 중인 오픈소스 AI 에이전트 프로젝트. "자비스의 첫 프로토타입"이라는 평가를 받는다(화자 인용).
- 앤트로픽의 Claude와 무관하다. 이쪽은 `CLWD`. 상호명 문제로 앤트로픽이 딴지를 걸어 이름을 **클로드봇 → 몰트봇 → 오픈클로(OpenClaw/오픈클러)**로 바꿔 왔다.
- 제작자는 은퇴했다가 AI 때문에 복귀한 개발자 [불명확: 자막 음차 "피터스 타임버그"]. 1월 29일에 관련 글을 올렸고, "두 달 전 주말 동안 뚝딱 만들었다"고 함(화자 인용).
- 공개는 2025년 12월경(자막상 "12월 1일"), 실리콘밸리 중심으로 확산. GitHub에서 두 달 만에 스타 114K, 포크 16.2K, 스타 10만 개 이상(주장/미검증). 취재 시점에 GitHub 접속은 잠깐 안 되는 상태였다.

### 2. 핵심 차별점 — "OS 위에서 메신저로 부리는 에이전트"
- 기존 챗봇은 웹/앱에 입력하면 텍스트·이미지를 출력하고, 실행은 샌드박스 안이나 클릭 정도에 그쳐 나머지는 사람이 한다.
- 클로드봇의 본질은 LLM이 OS 위에서 도구·스킬을 스스로 가져와 호출·조합하고 **행동(결제·파일 작성/삭제·검색·실행)까지 이어 결과물을 내는 것**. 답변이 아니라 업무 대리인(에이전트)이다.
- 화자 비유: 챗봇은 "설명서만 잘 써 주는 친구", 에이전트는 "내 컴퓨터 자체가 알아서 움직이는 것".
- 바이럴 요인은 **메신저 인터페이스**. 사람에게 문자 보내듯 텔레그램·아이메시지·디스코드·왓츠앱·Gmail·트위터 등으로 명령하면 24시간 켜진 내 컴퓨터(서버)가 알아서 처리한다.

### 3. 실제 활용 사례 (화자 인용, 미검증)
- 클로드봇 + make(자동화 툴)로 하루 550개 영상 제작 — 24시간 돌며 숏폼 광고를 자동 생성·확장.
- 암호화폐 뉴스를 텔레그램으로 물으면 실시간 정리해 줌. 가격·유동성 분석을 공유하는 시스템을 만든 사람도 있음.
- 클로드봇으로 주식 트레이딩을 한 사람 — "결국 다 잃었다"고 하지만 다양한 걸 알아서 실행함.
- ElevenLabs(11랩스) 보이스 에이전트를 텔레그램 프롬프트로 시켜 음성 생성·설명, 영상 편집, 인스타그램 업로드, 베스트 뷰 분석까지 자동. 예: "야 헤이 클로드, 11랩스 보이스 에이전트… 테스트 에이전트 하나 더 만들어라" 식으로 지시.

### 4. 왜 하필 맥미니인가 (꼭 맥미니여야 하는 건 아님)
- 24시간 상시 구동 전제 → 맥미니가 전성비 좋고 소음 적고 성능 좋아 서버용으로 적합. 북미에서 M4 품절, 대체로 M1 농장을 꾸린 사례도.
- 한국은 아이폰·맥 유저가 많아 아이메시지 연동 자동화가 강력한 기능이 됨.
- 보안상 메인 PC에 바로 붙이기보다 **깡통 맥미니를 따로 사서 디스플레이 없이 독립 서버로** 돌리는 방식 권장. 화자 본인은 맥미니 M4 Pro 48GB 사용.

### 5. 로컬 vs 클라우드 LLM — 비용·성능·프라이버시 트레이드오프
- 모델은 OpenAI GPT, 앤트로픽 Claude 또는 맥미니 로컬 LLM 중 택. 클라우드를 쓰면 개인정보가 서버로 넘어갈 수 있고, 로컬은 그 위험이 덜하다.
- 에이전트는 브라우저 열기·파일 읽기/쓰기·외부 API 요청·앱 실행 등 실행/입출력 중심이라 토큰을 폭발적으로 쓰지 않을 수도 있음[의견]. 다만 클라우드 API는 호출량이 많아지면 네트워크 지연이 병목이 되고 비용이 누적된다.
- 로컬은 메모리·모델 크기가 성능을 좌우하고 GPU 대신 CPU 부하가 커짐. 에이전트 루프(플랜→실행→체크→픽스 반복)로 사람 개입은 줄지만 API 호출 자체는 늘 수 있다.

### 6. 산업 파장 — SaaS·OS·빅테크 구도 [의견]
- 에이전트가 다 해 주면 사람이 보기 좋은 웹/앱 UIX가 덜 필요해져 **SaaS·UI 가치가 약해지고**, 대신 API·권한·보안(에이전트 대응) 설계가 기업 경쟁력이 된다.
- OS 역할 축소: 과거엔 OS(구글/안드로이드, 애플/맥OS·iOS, 윈도우) 소유자가 권력. 이제 **AI가 OS 위 상위 계층**으로 올라가 OS가 AI의 지시를 수행하는 구도로 전락할 수 있다.
- 포지션별 강점: **애플**=OS 레벨 권한·키체인, 실행에 유리하나 개방성이 닫힘. **구글**=서비스·애플리케이션 표면이 넓고 자체 AI 보유. **OpenAI/앤트로픽**=모델 중심이라 에이전트 루프가 커지면 호출 위험↑, 온디바이스 전환 시 모델이 부품화될 리스크. OpenAI는 IPO 등으로 모델 리더십 확보 노력 중.

### 7. 보안 리스크 — 이미 터지고 있는 사고
- 각자 맥미니로 서버 호스팅하다 보니 **어느 IP·포트에서 돌리는지 등 정보가 그대로 유출**. 1,000개 넘는 인스턴스가 노출됐다는 얘기(주장/미검증), 관리·제어 패널 수백 개가 인증 없이 노출.
- 제어 패널에는 대화 로그, 서비스 토큰, **API 키**가 들어 있음. API 키 유출은 비밀번호·신용카드 분실과 같아, 남이 쓰면 그만큼 과금이 나에게 청구된다.
- 왜 기존 자동화보다 위험한가: 일반 오토메이션은 정해진 규칙만 실행하지만, 이건 **OS보다 높은 실행 권한**으로 알아서 다 시행한다. OS·브라우저·터미널을 다 다뤄 피해 범위가 넓다.
- LLM은 속임수에 취약 — **프롬프트 인젝션**(문서 속 지시문을 진짜 명령으로 착각), 오픈 생태계에서 공유되는 스킬·기능 속 **멀웨어**도 위협. 초기 인터넷처럼 무방비 배포 과정에서 사고가 터지는 중.

### 8. 안전한 사용·설계 원칙
- 제어 포트·관리 화면을 그대로 노출하지 않기.
- 파일 임의 수정 방지를 위해 **읽기 전용** 설정(단, 에이전트가 다시 쓰기로 바꿀 수도 있음).
- 크리티컬한 작업엔 **승인 게이트**를 넣어, 실행 전 메신저(텔레그램/아이메시지)로 "이거 해도 돼요?"를 묻고 승인받게 하는 절차적 모델이 필요.

### 9. 전망 [의견]
- 클로드봇 자체는 새 모델이라기보다 **"AI가 OS 위에서 행동하는 구조"의 프로토타입**을 보여준 사건. 그래서 바이럴.
- 앞으로 경쟁의 핵심은 더 똑똑한 모델이 아니라 **에이전틱 AI로 OS를 얼마나 안전하게 구현하느냐**. 이걸 잘 챙기는 쪽이 AI 경쟁 판도를 바꾼다.
- 생산성 킬러 앱이 등장하면 자비스가 순식간에 확산되고, 이를 서버형으로 잘 돌릴 **새로운 폼팩터의 맥미니 같은 PC**가 나올 수 있다.

## 3줄 오리엔테이션
- "클로드봇"(현 오픈클로)은 앤트로픽 Claude와 무관한 오픈소스 AI 에이전트로, 24시간 켜둔 맥미니 같은 서버에 메신저로 명령하면 결제·영상편집·검색·파일작업까지 알아서 해 주는 "자비스 프로토타입"이다.
- 이게 특별한 이유는 성능이 아니라 **AI가 OS보다 위에서 모든 실행 권한을 갖고 행동하는 구조**이며, 이 때문에 SaaS·OS·빅테크 경쟁 구도가 흔들릴 수 있다는 것.
- 동시에 API 키·제어 패널 유출, 프롬프트 인젝션 등 **보안 사고가 이미 터지고 있어**, 앞으로 승부는 "누가 에이전틱 OS를 안전하게 구현하느냐"로 간다는 게 영상의 결론.

### [01:01–01:45] 실제 예시 — 하루 550개 영상, 크립토 비서, 맥미니 품절
한 사람은 클로드봇과 make("메이크")를 이용해 하루 550개 영상을 만들었다고 한다(주장/미검증). 24시간 돌아가니 AI 엔진이 숏폼 광고를 자동 생성·확장하며 계속 시킬 수 있다는 것. 또 다른 사람은 암호화폐 뉴스를 실시간 확인 — 직접 안 찾고 텔레그램으로 물어보면, 자기가 만든 클로드봇 계정에 연결해 원하는 정보가 쫙 나온다. 나만의 비서를 구축할 수 있어 북미에서 맥미니가 동나고 있다고 한다.

### [01:45–02:20] M1 농장, "OS 위의 AI" — 앱을 만들 필요가 없다
어떤 이는 맥미니 M4가 품절이라 M1으로 API를 호출하며 24시간 모니터링 시스템을 구축했다. 핵심은 맥OS 위에서 이 에이전틱 모델(클로드봇)을 메신저로 관리하다 보니 **애초에 앱을 만들 필요가 없다**는 점 — AI가 OS에서 스스로 필요한 정보를 찾고 결제도 한다. 왜 하필 맥미니인지 짚지만, 사실 맥미니가 아니어도 상관없다고 한다.

### [02:20–02:52] 해킹 경고와 다룰 주제 예고
잘못 이용하면 해킹되거나, 자비스처럼 알아서 결제까지 해 버리는 일이 벌어진다. AI 업계에 "에이전틱 모델은 이렇게 돼야 한다"는 기준을 제시했고 오픈 모델이라 널리 퍼져 있다. 빅테크의 위협인지, 돈을 더 버는 건지 등 산업 영향을 정리하겠다고 예고한다.

### [02:52–03:16] 제작자와 GitHub 지표
현재 오픈클로로 개명된 이 프로젝트는 [불명확: 자막 음차 "피터스 타임버그"]라는 사람이 만들었다. 개발계 은퇴 후 AI 때문에 복귀했고 1월 29일에 글을 올렸다. "두 달 전 주말 동안 뚝딱 만들었다"고 하며, GitHub에서 10만 개 이상 스타, 포크 16.2K, 스타 114K로 두 달 만의 엄청난 열풍이라고 전한다(주장/미검증).

### [03:16–03:58] 정의 — 24시간 서버 + 메신저 지시 = 내 부하 직원
쉽게 말해 맥미니든 윈도우 PC든 24시간 돌아가는 걸 전제로, 스마트폰 메신저로 내 컴퓨터의 클로드봇에게 지시하면 된다. 항상 서버처럼 대기하는 "나만의 에이전트 = 부하 직원/매니저"가 생긴 것. 메신저로 말만 하면 결제·영상 편집·보고서 작성은 물론, 필요하면 더 찾아본다. 이 오픈소스 AI 에이전트 프로젝트를 처음에 클로드봇이라 명명했다.

### [03:58–04:44] 그냥 챗이 아니다 — 로컬 LLM·메신저 명령이 바이럴 요인
공개는 2025년 12월경, 실리콘밸리 중심 확산. 새 모델을 쓰는 게 아니라 로컬 LLM도 쓸 수 있다. 화자가 보는 최고의 바이럴 요인은 **메신저로 명령할 수 있다는 점** — 사람에게 메시지 보내듯 내 서버에 명령하면 24시간 돌아가는 내 컴퓨터가 알아서 처리한다. 맥미니는 전성비·성능이 좋아 확장성이 뛰어나고, 텔레그램 등 뭐든 쓸 수 있다. 화자는 맥미니 M4 Pro 48GB로 직접 설치해 볼 예정이라고.

### [04:44–05:07] 데모가 아니라 커뮤니티 — 광범위한 연동
12월 1일부터 두 달간 급상승 중. 유튜브 데모가 아니라 개발자들이 실제로 붙어 새 커뮤니티 이벤트가 형성되고 있다. 시작 방법뿐 아니라 왓츠앱·텔레그램·디스코드·스포티파이·GPT·아이메시지(애플)·클로드·Gmail·트위터 등 온갖 것을 연동해 명령할 수 있다.

### [05:07–05:20] 스킬 공유 커뮤니티
사람들이 만든 다양한 기능을 GitHub에 올린다(현재 잠깐 접속 불가). 각자 24시간용으로 잘 만든 에이전트·스킬을 서로 공유하는 커뮤니티가 이미 생성되고 있다.

### [05:20–05:40] 진짜 주목점 — OS보다 위에서 모든 권한
주목할 것은 AI 성능 향상이 아니라, **AI가 OS 밑이 아니라 OS보다 훨씬 위에서** 컴퓨터를 스스로 판단해 파일 삭제·작성, 결제 등 모든 권한을 갖고 실행하는 첫 대형 이벤트라는 점이다.

### [05:40–06:11] 크립토·주식 자동매매 사례
텔레그램에서 현재 가격·유동성 등 분석 결과를 공유하는 시스템을 만든 사람도 있다. 클로드봇으로 주식 트레이딩을 한 사람은 "결국 다 잃었다"고 하지만 다양한 걸 알아서 실행했다. 더 발전하면 자동 매매 기법이 에이전틱으로 들어오며 이전과 다른 양상이 될 수 있다고 해석한다[의견].

### [06:11–06:37] 영상 제작 파이프라인 자동화
텔레그램 프롬프트로 ElevenLabs("일랩스")가 음성 목소리를 만들고 설명까지 하며, 영상 편집도 알아서 한다. "어떻게 만들어라"까지 지시할 필요 없이 특정 방향만 주면 만들어 인스타그램에 올리고, 인스타 뷰에 맞춰 베스트 뷰가 뭔지까지 정리 — 24시간 돌아가니 알아서 된다.

### [06:37–06:53] "사람 시키듯" 지시 → 진짜 자비스
지시 예시:
> "야 헤이 클로드, 11랩스의 보이스 에이전트가 네가 갖고 있는 게 뭐냐 하면 테스트 에이전트… 한 개 더 만들어라." [06:41]

사람에게 시키듯 메신저로 명령하며, 이걸 나중에 오디오(음성)로 다 바꾸면 진짜 자비스가 된다고 한다[의견].

### [06:53–07:34] 챗봇 vs 에이전트의 본질
기존 챗봇은 웹/앱에 입력하면 텍스트·이미지를 출력하고, 실행은 샌드박스 안이나 클릭 정도에 그쳐 나머지는 사람 몫이다. 반면 클로드봇 같은 에이전트의 본질은 LLM이 OS 위에서 도구·스킬을 가져와 호출·조합하고 **행동까지 이어 결과물을 내는 것**. 답변이 아니라 업무를 대신하는 대리인(에이전트)이다.
> "챗봇은 설명서만 잘 작성해 주는 친구… 에이전트는 그 내 컴퓨터 자체인 거야. 내 컴퓨터가 알아서 막 움직여." [07:36]

### [07:34–08:04] 동작 흐름 — 메신저 → 맥미니 → LLM
바이럴 구조를 도식화하면, 텔레그램 같은 메신저로 뿌리면 게이트를 통해 맥미니(로컬)가 메시지를 받고, 그걸 LLM으로 처리한다. LLM은 OpenAI GPT, 앤트로픽 Claude, 또는 맥미니 로컬 중 무엇이든 쓸 수 있다.

### [08:04–08:35] 프라이버시 주의 + AI의 자율 실행
GPT나 Claude 등 클라우드를 쓰면 개인정보가 서버로 넘어갈 수 있고 로컬이면 덜하다. 그 다음 AI가 필요한 툴을 스스로 판단 — 파일 복사, CLI 창을 띄워 커맨드 입력, 웹페이지 검색 등을 모두 알아서 판단한 뒤 실제 결과를 낸다.

### [08:35–09:10] 결제·영상편집 실행 방식과 비용 구조
결제가 필요하면 결제 API를 호출하고, 영상 편집이면 파이썬으로 FFmpeg("FF엠팩") 같은 걸 가져와 잘라낸다. 컴퓨터만 켜져 있으면 서버용 맥미니로 24시간 돌아간다. GPU보다 CPU 부하가 커진다는 얘기가 있고, 클라우드 AI(GPT/앤트로픽)를 쓰면 서버 호출이 필요해 API 호출 비용이 사용량만큼 누적된다.

### [09:10–09:42] 토큰 소비는 생각보다 적을 수도 — 병목은 네트워크
에이전트가 엄청난 토큰을 소비하리라 생각하지만 아닐 수도 있다[의견] — 브라우저 열기, 파일 읽기/쓰기, 외부 API 요청, 앱 실행 등 실행/입출력 중심이기 때문. 다만 API 호출 시 인터넷을 왔다 갔다 하니 **네트워크 지연이 병목**이 되고 코스트도 든다. 로컬로 돌리면 메모리·모델 크기가 성능을 좌우한다.

### [09:42–10:16] 에이전트 루프 — 사람 개입↓ API 호출↑
보통 LLM은 말하고 고치고를 왕복하지만, 에이전트는 어느 정도 알아서 24시간 돌아가 "한 번 키면 알아서 끝낸" 느낌이 강해진다. 에이전트는 플랜 → 실행 → 체크 → 픽스 → 재제시를 반복하는데, 사람 인터랙션은 줄지만 API 호출 자체는 많아질 수 있어 토큰 코스트가 커질 수 있다.

### [10:16–10:49] 맥미니가 잘 팔리는 이유
24시간 돌아도 전성비가 좋고, 저전력·저소음인데 성능은 좋다. GPU 서버보다 항상 켜둘 수 있어 북미에서 많이 팔린다. 특히 한국은 아이폰·맥 유저가 많아 아이메시지 문자 자동화를 맥으로 쓰면 컴퓨터·아이폰 모두에서 쓸 수 있어 메신저 자동화가 강력한 기능이 된다.

### [10:49–11:21] 보안상 "깡통 맥미니 따로" 권장
맥은 보안이 강한 편이지만, 에이전트가 파일·앱·기능에 접근하니 내가 쓰던 메인 PC에 바로 쓰기보다 가성비 좋고 성능 좋은 **깡통 맥미니를 따로 사서** 독립 구동하는 게 낫다. 디스플레이는 필요 없다 — 서버로 계속 돌 것이고 나머지는 메신저로 구동하기 때문. 그래서 많이 팔린다.

### [11:21–12:20] SaaS·UI의 약화 [의견]
화자가 크게 본 지점: 사람이 직접 보려면 보기 좋은 웹/앱 디자인(SaaS 클라우드, UIX)이 필요하다. 그런데 에이전트가 알아서 다 해 주면 **사람이 보기 좋게 만드는 과정이 필요 없어진다**. 지금까지 SaaS는 화면을 얼마나 예쁘고 가독성 좋게 구현하느냐가 핵심 기술이었는데, 에이전트가 끼면 화면 클릭 대신 API 호출로 처리하니 **SaaS·UI 가치가 약해지고**, 오히려 API·권한·시큐리티 등 에이전트 대응 설계가 기업 경쟁력이 될 수 있다.

### [12:20–13:26] OS의 지위 하락 — AI가 상위 계층으로 [의견]
OS 역할이 점차 줄어든다. 예전엔 OS(구글/안드로이드, 애플/맥OS·iOS, 윈도우)를 가진 자가 기본 플랫폼을 깔아주는 권력자였다. 이제 OS보다 위에서 AI를 잘 돌리는 기업(구글·OpenAI·앤트로픽)의 모델이 상위에 서고, OS는 가만히 있고 AI가 파일 찾고 로그인·도구 실행·없으면 생성까지 24시간 대신한다. 결국 AI가 OS 지위를 얻으면 OS는 반대로 AI가 시키는 역할을 수행하는 쪽으로 전락하고, "AI 쓰는 OS가 더 좋다"는 식으로 AI가 플랫폼 중심이 된다.

### [13:26–14:04] 권한의 딜레마 — 샌드박스·VM 방어
에이전트가 커질수록 실행과 권한이 중요하다. 결제 권한을 준다는 건 개인정보·신용을 그대로 주는 것이라 현실 피해로 이어질 수 있는 딜레마다. 그래서 빅테크는 별도 샌드박스/VM(가상머신)을 띄워 문제가 생겨도 그 안에서만 발생하게 하고 꺼버려 내 컴퓨터에 영향을 안 주도록 한다. 프롬프트 인젝션 같은 해킹도 방어한다.

### [14:04–14:46] 애플·구글의 강점
애플 관점에선 OS 레벨의 권한·키체인 등이 실행에 유리해 "AI가 OS 위에서 일하게 만들" 자연스러운 플레이어지만, 개방성이 닫혀 있어 해결할 부분이 있다. 구글은 서비스가 엄청 많아 에이전트가 대응할 업무·애플리케이션 표면이 넓고, 자체 AI도 갖고 있어 강점이다.

### [14:46–15:14] OpenAI·앤트로픽의 리스크와 3사 요약 [의견]
OpenAI·앤트로픽은 모델 중심이라 에이전트 루프가 커지면 호출이 굉장히 많아져 위험할 수 있고, 다시 온디바이스 AI 쪽으로 가면 모델 자체가 부품이 될 리스크도 있다. OpenAI는 모델 리더십을 위해 IPO도 한다고 한다. 정리하면 — 애플=시큐리티·강력한 OS, 구글=OS·AI·애플리케이션의 총합적 위치, OpenAI/앤트로픽=모델 리더십.

### [15:14–16:07] 이미 털리고 있다 — 1,000개+ 인스턴스 노출
결국 시큐리티가 핵심이다. 에이전트에 모든 실행 권한을 주니 잘못 실행하면 파일 삭제·유출, 계정 탈취, 함부로 한 결제가 곧바로 현실 피해가 된다. 생산성만큼 리스크도 크다. 빅테크 입장에선 "에이전트 OS 표준을 누가 잡느냐"가 핵심. 그런데 지금 클로드봇은 각자 맥미니로 서버 호스팅하다 보니, 어느 위치·포트·IP에서 돌리는지가 다 유출됐다. **1,000개 넘는 인스턴스가 노출**됐다고 얘기된다(주장/미검증).

### [16:07–16:41] 제어 패널·API 키 유출의 치명성
관리·제어 패널이 인증 없이 노출돼 수백 개 단위로 새어 나갔다. 제어 패널에는 대화 로그, 서비스 토큰, **API 키**가 들어 있어 치명적이다. API 키(예: GPT API, 네이버지도 API)는 쓴 만큼 과금되는데, 노출되면 비밀번호가 털린 것과 같아 남이 쓰면 계속 나에게 과금된다 — "신용카드를 잃어버린 것"과 같다.

### [16:41–17:34] 왜 기존 자동화보다 위험한가
강력한 자동화 도구가 초기 인터넷처럼 무방비로 배포되는 과정에서 사고가 터지고 있다. 일반 오토메이션은 정해진 규칙만 시행하지만, 이건 **스스로 다 시행**하고 OS보다 높은 실행 권한(OS·브라우저·터미널)을 가져 잘못하면 피해가 넓어진다. 게다가 LLM은 속임수에 취약해 — 프롬프트 인젝션처럼 문서를 읽다 지시문을 진짜 명령으로 착각할 수 있어 보안 중요성이 커진다.

### [17:34–18:01] 유출·인젝션·멀웨어 종합
정리하면 컨트롤 패널 노출로 API 키·개인정보가 그대로 나가거나, 프롬프트 인젝션, 오픈 생태계에서 공유되는 스킬·기능 속 멀웨어 등이 문제. 결과적으로 API·신용카드 남용에 따른 막대한 과금, 데이터 유출, 계정의 연쇄 탈취 등 여러 문제가 발생할 수 있다.

### [18:01–18:41] 안전 수칙 — 노출 차단·읽기 전용·승인 게이트
벌써부터 "제어 포트·관리 화면을 그대로 노출하지 말라"는 얘기가 나온다. 파일을 함부로 수정할 수 있으니 읽기 전용으로 설정(단, 에이전트가 다시 쓰기로 바꿀 수도 있음)하고, 크리티컬한 작업엔 **승인 과정**을 넣어야 한다 — 그러면 메신저(텔레그램/아이메시지)로 "이거 해도 돼요?"가 오고 그때 승인한다. 부하 직원이 내 카드를 맘대로 쓰면 스트레스이듯, 절차적으로 통제할 모델이 필요하다.

### [18:41–19:23] 결론 — 프로토타입의 의미와 전망 [의견]
