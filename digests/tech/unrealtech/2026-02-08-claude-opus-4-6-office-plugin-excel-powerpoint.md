## 요약 (주제별)

### 무엇이 새로 나왔나
- Anthropic이 Claude Opus 4.6을 출시하면서 Microsoft Office(PowerPoint, Excel)에 Claude를 직접 연동하는 플러그인이 등장했다.
- PowerPoint 플러그인은 원하는 자료를 넣어 주면 초벌 슬라이드를 자동 생성하고, 편집 가능한 형태로 만들어 준다(영상 도입부 시연 언급).
- Excel 연동 플러그인은 화자 확인으로는 이미 "지난 달"에 나와 있었다고 함(주장/미검증).

### 플랜별 사용 가능 여부 (화자 확인 기준)
- PowerPoint: 현재 Max / Team / Enterprise 플랜 이상에서만 사용 가능한 것으로 보인다(화자 주장/미검증). 화자는 Pro 사용자라 연동 시 "PowerPoint is coming soon"이 뜨며 접근 불가.
- Excel: Pro에서도 지금 사용 가능. 화자는 Excel만으로도 할 게 굉장히 많다고 평가.
- 플러그인 버전 정보상 PowerPoint용 1.0 버전이 2026년 2월 5일, 즉 Opus 4.6 출시 당일에 공개됐다고 함(화자 확인).

### 설치 방법
- PowerPoint에서 "추가 기능(Add-ins)"으로 들어가 Microsoft AppSource에서 해당 플러그인/앱을 검색해 추가한다.
- AppSource에는 Word/Excel/PowerPoint용 플러그인이 매우 많으며, Anthropic 공식 Excel·PowerPoint 플러그인 외에도 서드파티가 만든 것들이 있다.
- 로그인하면 Opus 4.6, Sonnet 4.5/4.6 등 최근 모델을 선택해 사용할 수 있다.

### 서드파티 vs 공식의 차이
- "챗GPT Excel / 챗GPT PowerPoint" 류는 OpenAI가 직접 만든 게 아니라 별도 스타트업이 만든 것으로 보인다(화자 추정).
- 이런 서드파티는 프리 트라이얼 토큰을 주는 방식이라 별도 과금이 필요할 수 있어, 추가 결제가 싫으면 안 써도 된다.
- 반면 Claude는 이미 구독한 계정을 연동만 하면 되므로 화자가 소개하는 것이라고 밝힘.

### Excel 시연 내용
- 동작 모드: 수정 전에 매번 확인받는 모드와 "Accept all edits"(전체 자동 적용) 모드가 있음. 시연에서는 편의상 Accept all edits 사용.
- 요청 예시: "구글(알파벳) 최근 10년 재무 데이터를 엑셀표로 보기 좋게 정리해 줘."
- 처리 과정: SEC의 알파벳 공식 10-K(영상 내 "XK") 보고서를 API로 직접 가져와 그 자료를 토대로 엑셀 표로 정리해 준다(한국의 공시 보고서에 해당).
- 산출: 마진, 그로스, EPS 등을 계산해 "알파벳 재무 요약"을 표로 깔끔하게 생성. 중간 과정도 보여 줌.
- Opus 4.6은 이런 재무 데이터 처리가 매우 강력하다고 Anthropic이 주장했다고 화자가 전함(주장/미검증).

### 활용 관점 [의견]
- 그냥 챗봇에 물어보고 복사해 붙일 수도 있지만, Excel에서 직접 작업하는 사람이 많으므로 구독 계정을 연동해 자기 시트를 바로 수정·정리시키는 게 유용하다는 관점.
- Microsoft Copilot이 Excel/PowerPoint용으로 이미 이런 기능을 지원하지만 점유율이 크게 오르지 않고 있고, Copilot을 위해 구독하는 사람보다 LLM/코딩용으로 돈 쓰는 비중이 훨씬 높다 → 이미 내는 월 구독을 생산성 도구에 연결하면 좋다는 시각(화자 의견/분석).

### 한계·아쉬운 점
- 글자 크기 등 서식 통일이 안 되는 부분이 있고, 글자가 잘 안 들어가는 부분도 일부 있음.
- 에이전틱 동작은 토큰을 많이 소비한다. 화자가 "투자자용 그래프를 만들어 줘"까지 시키려 하자 리밋에 걸림. 토큰↑ = 컴퓨팅 자원↑ 상황이라 아쉬운 부분.
- 그럼에도 이 사용 형태 안에서 쓸 수 있는 부분이 많아 빠르게 정리했다는 마무리.

- 화자가 Pro 계정으로 Excel 연동을 직접 시연: 알파벳 10-K 재무 데이터를 SEC에서 끌어와 엑셀 표로 자동 정리.
- 서식 통일·토큰 리밋 등 한계는 있지만 이미 낸 구독을 생산성 도구에 연결한다는 관점에서 쓸모 있다는 평가.

## 전문 — 시간순 워크스루

### [00:00–00:29] 도입: PowerPoint에 Claude를 붙이는 플러그인
화자(에러)는 Anthropic이 Claude Opus 4.6을 출시하며 코딩 쪽으로 주목받고 있지만, 사람들이 잘 모르는 소식이 있다고 연다 — Microsoft PowerPoint를 플러그인과 연동해 원하는 자료만 넣어 주면 PowerPoint에서 직접 편집 가능한 형태로 초벌 슬라이드를 만들어 주는 플러그인이 생겼다는 것.

### [00:51–01:08] 결론 먼저: 플랜별 사용 가능 범위
결과부터 말하면, PowerPoint는 Max나 Enterprise 이상 / Team 이상 구독자 대상으로만 지금 쓸 수 있는 것으로 보이고(화자 확인/미검증), 화자는 Pro여서 접근이 안 된다. 반면 Excel은 Pro에서 지금 쓸 수 있다. 화자는 Excel만으로도 할 게 굉장히 많다며 빠르게 시연하겠다고 한다.

### [01:08–01:34] 설치 방법: AppSource에서 추가 기능 검색
설치는 PowerPoint에서 "추가 기능"으로 들어가 Microsoft AppSource에서 해당 파일/프로그램을 검색해 추가하면 된다. 처음부터 "오픈 클로드"라고 표기돼 있지는 않다고 덧붙인다. AppSource에는 PowerPoint·Word·Excel에 넣을 수 있는 플러그인이 상당히 많아서, 사용할 만한 것을 골라 추가해 쓰면 된다.

### [01:34–01:57] 서드파티 vs Anthropic 공식, 그리고 출시일
"챗GPT Excel", "챗GPT PowerPoint" 같은 것은 OpenAI가 직접 만든 게 아니라 별도의 스타트업이 만든 것으로 보인다고 화자가 추정한다. 반면 Anthropic은 직접 Excel용 플러그인을 내놓았고, Anthropic PowerPoint 플러그인도 확인할 수 있다. PowerPoint용은 1.0 버전이 2026년 2월 5일, 즉 Opus 4.6이 출시된 그날 공개됐다고 한다(화자 확인).

### [01:57–02:33] PowerPoint는 "coming soon" — Pro 한계
가입하면 플러그인을 받아 쓸 수 있는지 체크할 수 있는데, 실제로 켜서 연동하니 "PowerPoint is coming soon"이 떠서 안 됐다고 한다. 확인해 보니 현재는 Max·Team·Enterprise 플랜 대상으로만 지원한다고. 화자는 Pro 사용자라 이걸 위해 Max로 올릴까 고민했지만, Genspark·Manus·Skywork 같은 통합 플랫폼도 이미 쓰고 있어서 따로 가입하진 않고 말로만 전한다며, 업데이트되면 다시 알려주겠다고 한다.

### [02:33–03:01] Excel 연동 실제 사용 시작
Pro로 쓸 수 있는 것으로 넘어가 Excel을 시연한다. "오픈 클로드"도 추가하면 되고, 로그인하면 Opus 4.6, Sonnet 4.5/4.6 등 최근 모델을 골라 Excel에 액션을 할 수 있다. 선택할 때 "수정하기 전에 먼저 물어봐라/확인해라" 옵션과 "Accept all edits"(전체 자동 적용) 옵션이 있는데, 지금은 귀찮으니 Accept all edits로 진행한다.

### [03:01–03:35] 시연: 알파벳 10년 재무 데이터를 엑셀표로
예시로 "구글이 최근 10년 동안의 재무 데이터를 엑셀표로 보기 좋게 정리해 줘"라고 요청한다. 화자는 이번 Opus 4.6이 이런 재무 데이터에 굉장히 강력하다고 Anthropic이 주장했다고 전한다(주장/미검증). 모델은 SEC에서 알파벳 공식 10-K(영상 내 "XK") 보고서를 가져온다 — 한국의 공시 보고서에 해당하는 자료를 API를 통해 직접 가져온 것으로 보이며, 그 자료를 토대로 바로 엑셀로 표현하는 과정을 볼 수 있다.

### [03:35–03:58] 서드파티 과금 vs Claude 연동의 편의
이전의 "챗GPT Excel" 같은 건 OpenAI가 직접 지원한 게 아니라 다른 스타트업이 프리 트라이얼 토큰을 주는 방식이라, 따로 과금해 쓰기 싫으면 안 써도 된다. 반면 Claude는 이미 가입된 걸 연동만 하면 쓸 수 있어서 소개하는 것이라고 다시 강조한다.

### [03:58–04:31] Copilot 대비 관점 [의견]
이게 중요한 이유로, Microsoft가 Copilot을 통해 이미 이런 걸 지원하고 있지만 Copilot 점유율이 크게 오르지 않고 있다는 점을 든다. Excel·PowerPoint 용도로 Copilot을 구독하는 사용자보다 일반 LLM이나 코딩을 위해 돈 쓰는 비중이 훨씬 높으니, 이미 월 구독을 내고 있다면 거꾸로 그걸 이런 생산성 도구에 연결하면 좋지 않겠냐는 관점을 제시한다(화자 의견/분석). 시연에서는 마진, 그로스, EPS 등이 다 계산돼 나온다.

### [04:31–04:58] 결과물 평가와 실사용 시나리오 [의견]
"알파벳 재무 요약"이 표로 잘 나온다. 그냥 물어보고 붙여도 되지만, Excel에서 직접 작업하는 사람이 많으니 구독 기반으로 이 플러그인 하나 깔아 두고 "내가 쓰고 있는 시트 좀 수정해 줘" 같은 요청을 할 수 있다는 점을 든다. 중간 과정을 다 보여 주고 보기에도 좋게 나온다고 평가. PowerPoint까지 하려면 더 비싼 구독료가 필요해 아쉽지만, Excel만으로도 이 정도면 상당히 쓸모 있다는 생각을 밝힌다.

### [04:58–05:47] 한계: 서식 통일·토큰 리밋
부족한 점도 짚는다. 아직 글자 크기 등 서식 통일이 안 되는 부분이 있고, 글자가 잘 안 들어가는 부분도 일부 있다. 특히 아쉬운 건 이런 작업이 토큰을 굉장히 많이 소비한다는 것 — "투자자들이 보기 좋은 그래프를 만들어 줘"까지 시키려 하자 리밋에 걸렸다고 한다. 결국 에이전틱한 동작은 토큰을 많이 쓰고, 토큰을 많이 쓸수록 컴퓨팅 자원을 많이 쓰는 상황이라 아쉬운 부분이 있다고 본다. 그래도 이 사용 형태 안에서 쓸 수 있는 부분이 많아 빠르게 정리했다고 마무리한다.
