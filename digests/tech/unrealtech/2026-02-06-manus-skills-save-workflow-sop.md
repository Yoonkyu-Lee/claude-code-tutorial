## 요약 (주제별)

### Manus Skills가 해결하려는 문제
- AI 툴(ChatGPT, Claude, Gemini)은 이미 실전에서 쓰이지만, 기업 분석 리포트·경쟁사 조사·발표 자료 같은 일은 여전히 매번 같은 프롬프트를 반복 입력하는 패턴을 벗어나지 못한다는 문제 제기.
- 결과 포맷이 들쭉날쭉해서 또 프롬프트를 치고, 적당히 나온 걸 사용자가 추가로 고쳐야 하는 반복 노동이 계속됨. 프롬프트를 따로 저장했다가 옮겨 쓰는 것 자체가 시간 소모라는 지적.

### 핵심 개념 — "프롬프트가 아니라 일하는 방식(SOP)을 저장"
- 마누스 스킬스의 핵심은 프롬프트 한 줄을 저장하는 게 아니라, 사용자가 실제로 일한 그 과정 = 작업 방식(SOP) 전체를 저장해 계속 재사용하는 것.
- 스킬은 "프롬프트 + 설정 + 과정 + 결과물 구조"까지 다 포함한 패키지라는 점에서, 흔한 프롬프트 템플릿(한 줄 문구)과 다르다는 게 화자 주장.
- [의견] 클로드 코드를 비롯한 에이전틱 스킬은 일반인이 쉽게 쓰기 어려운 반면, 마누스는 엄마·아빠·부장님·사촌 꼬마도 따라만 하면 되는 사용성을 구현한 게 놀랍다고 평가.
- [의견/맥락] 최근 메타가 마누스를 인수한 이유가 이런 대중 사용성 개발 지속성 때문이 아닐까 하는 화자 추측. (인수 언급은 영상 내 서술이며 세부는 [불명확])

### 만드는 법 — Skill Creator
- 별도 설정·옵션·구조 설정이 필요 없고, 코딩·터미널·CLI·환경 세팅 없이 마누스 안에서 그냥 "스킬 크리에이터"라고 치기만 하면 된다.
- 그러면 마누스가 그 세션 전체를 하나의 작업 방식으로 이해하고, 지금까지 쓴 프롬프트·중간 분석 흐름·결과물 구조를 통째로 하나의 스킬로 묶어 제안한다.
- 스킬 가이드에 어떤 스킬을 만들지 지정(예: 기업명을 입력받으면 유튜브 영상 링크·외부 투자은행 보고서 포함, 데이터 검증, 보고서+슬라이드 생성)하면 그 전체가 하나의 스킬이 됨. 스킬 정보는 MD 파일로 저장됨.

### 시연 1 — 기업 분석 보고서 스킬
- 먼저 일반 LLM(ChatGPT)으로 메타 기업 분석 보고서를 만드는 과정을 보여줌: 10년 재무 데이터 추가, 타사 비교, 20년 소송 리스크, 음슴체 포맷, 참고 영상/사진/PDF 추가, 슬라이드 생성 등을 프롬프트로 계속 덧붙여야 함. [의견] 결과 퀄리티가 나쁘다는 게 아니라 LLM이 좋아서 충분히 쓸 수 있지만, 반복 노동이 문제라는 것.
- 이 전체 과정을 스킬로 만들면, 이후엔 "Company Analysis Report" 스킬에 기업명(예: 구글)만 치면 품질 체크리스트·톤·구조가 그대로 적용돼 보고서+슬라이드가 한 방에 생성됨. 구글 결과물에는 PPT, 재무 정보, 이미지, 최근 연도 테이블, 경쟁사 비교가 요청대로 반영됨.

### 시연 2 — 포스터 얼굴 합성(Poster Face Swap) 스킬
- 젠슨 황 얼굴을 오징어 게임 포스터 주인공 자리에 합성하는 작업을 스킬로 저장. 이후엔 첫 번째 이미지 인물을 두 번째 영화 포스터 인물로 대체하는 스킬로 재사용.
- 재사용 예: 마크 저커버그 얼굴 + 테슬라 일론 머스크 포스터. "테슬라"라고 말 안 해도 알아서 두 번째 테슬라 이미지의 인물 얼굴을 교차 인식·처리. 스킬 파일을 자동 확인하고, 샌드박스(우분투 베이스 리눅스) 안에서 명령줄로 합성 수행.

### 시연 3 — 컴퍼니 대시보드 제너레이터
- 메타의 최근 뉴스·실적을 한눈에 보는 대시보드 웹사이트/모바일 페이지를 만드는데, 처음부터 완성이 아니라 여러 번 명령으로 손봄: 상단 주요 기업 상황 텍스트 조정, 메타 KPI(패밀리 앱·쓰레드 DAU 등) 추가, 세로 나열을 가로 레이아웃으로 변경, 버튼 손보기 등 반복 미세 조정.
- 이 미세 조정 전부를 스킬로 저장 → "Company Dashboard Generator"에 애플 입력 시 원하는 형식 그대로 인식해 애플 대시보드를 자동 생성.

### 시연 4 — 시장 분석·경영진 보고 스킬
- Market Analyzer: 라면 시장 등을 분석해 원하는 대로 커스터마이징한 뒤 스킬로 저장.
- Executive Presentation Generator: 임원 보고용 PPT 포맷을 미세 조정(그래프 삽입 등)한 뒤 스킬화 → "AI 반도체 시장"을 넣으면 경영진용 포맷대로 그래프·검증된 최근 데이터 기반 PPT가 자동 생성. [의견] 요즘 PPT 수준을 보여주는 것이며, 매번 달라지지 않고 정의해 둔 방식대로 일관 생산되는 게 핵심.

### 시연 5 — 스킬 조합 + 쇼폼 영상 스킬
- 여러 스킬을 엮어 종합 기업 분석 보고서와 임원 프레젠테이션을 한 번에 둘 다 생성 가능.
- 자기 영상을 넣어 하이라이트를 찾아 1분 이내 쇼폼 영상을 만드는 것도 마누스가 지원. 이를 스킬화하면 9:16 세로 전환, 블러 배경 확대, AI 리프레이밍 등을 적용해 쇼폼을 바로 생성.

### 클로드 코드와의 차이 (화자 주장)
- "클로드 코드나 깃허브 스킬 레포로도 비슷한 거 하지 않냐"는 질문에 대해: 마누스는 웹 기반이고 별도 설정이 필요 없다는 점을 세일즈 포인트로 봄.
- 마누스는 격리된 샌드박스(버추얼 머신) 안에서 스킬을 실행 → 뭔가 잘못되거나 자동화하는 것을 VM 안에서 처리해 위험을 원천 차단. 반면 [의견] 클로드 코드류는 로컬 머신/환경에서 실행되는 성격이 강해 세팅·안전 책임이 사용자에게 더 크게 온다는 것.

- [의견] 스킬을 팀 단위로 그대로 넘겨 표준화 가능 → 기업 분석·웹페이지·영상 편집·리서치·콘텐츠 제작에서 한 번 제대로 만든 걸 계속 같은 퀄리티로 뽑는 색다른 협업 방식이 등장할 것으로 전망.

### [04:40–05:26] Skill Creator — 설정·코딩 없이 한 줄로
그 "한 큐로 만드는 과정"은 클로드 코드처럼 뭔가 설정하고 옵션 고르고 구조 잡는 게 아니라, 마누스 안에서 그냥 "스킬 크리에이터"라고 치면 된다고 시연한다. 그러면 지금까지의 과정이 쫙 만들어지고, 추가로 보정해 주면 뭔가 나온다. 이때 마누스가 하는 생각은 "이 세션 전체가 하나의 작업 방식이구나"라고 이해하면서, 지금까지 쓴 프롬프트·중간 분석 흐름·결과물 구조를 통째로 하나의 스킬로 묶자고 제안하는 것이다. 따로 코딩하거나 터미널 CLI를 켜거나 환경 세팅이 필요 없이 이 안에서 그냥 하면 된다는 점을 강조한다.

### [05:26–06:01] 기업 분석 스킬 생성 & 재사용
스킬 크리에이터를 치고 스킬 가이드에 어떤 스킬을 만들지 지정한다 — 기업을 입력받으면 앞서 얘기한 것(유튜브 영상 링크 포함, 외부 투자은행 보고서 삽입, 깔끔한 출력, 데이터 검증, 보고서+슬라이드 생성)이 다 나오도록. 이 전체가 하나의 스킬("Company Research/Analysis Report")이 된다. 이후엔 기업만 바꿔서 쓰면 되는데, 예를 들어 "구글"이라고만 치면 이 스킬 기준으로 구글에 대한 것들이 쫙 만들어진다. 하나하나 칠 필요 없이 구글 보고서와 슬라이드를 다 생성한다.

### [06:01–06:50] 구글 결과물 & 프롬프트 템플릿과의 차이
구글 결과물에는 앞서 프롬프트로 다 했던 품질 체크리스트(음슴체, 요약 등)가 하나의 스킬로 포함돼 한 방에 생성된다. 원하는 구조·톤·포맷 그대로 기업만 바꿔 유지된다. [의견] 흔한 프롬프트 템플릿과 다른 이유는, 스킬은 한 줄 문구가 아니라 "프롬프트 + 설정 + 과정 + 결과물 구조까지 다 포함한 패키지"이기 때문이며, 마누스가 이걸 플랫폼 안에서 굉장히 쉽게 구현했다는 것이다. 구글 결과물에는 PPT 분석, 보고서, 재무 정보, 이미지, 최근 연도 테이블, 경쟁사 비교가 요청대로 반영돼 있고, PPT도 그래프로 그대로 표현된다.

### [06:50–08:17] 포스터 얼굴 합성 스킬 (Poster Face Swap)
재밌는 예로, 젠슨 황과 오징어 게임을 같이 두고 "두 번째 영화 포스터 가운데 주인공 인물을 첫 번째 젠슨 얼굴로 넣어 달라"고 해 합성이 잘 나온 것을 스킬로 만든다. 스킬 크리에이터로 "포스터의 특정 인물을 대체 — 첫 번째 이미지 인물을 두 번째 영화 포스터로 바꾼다"는 "포스터 페이스 스왑" 스킬을 만든다. 재사용 때는 메타(마크 저커버그) 첫 번째 사진을 올리고 두 번째로 테슬라 일론 머스크 이미지를 넣어 바꿔 달라고 한다. "테슬라"라고 말 안 했는데도 두 번째 테슬라 이미지의 인물 얼굴을 알아서 인식해 교차 작업한다. 저장된 스킬 파일을 확인하고, 샌드박스(우분투 베이스 리눅스) 안에서 명령줄로 합성하는 모습을 보여주며 결과를 바로 넣어 준다. 자주 쓰는 기능이면 스킬로 만들어 쓰면 된다는 것.

### [08:17–10:14] 컴퍼니 대시보드 제너레이터 (메타 → 애플)
더 정교한 작업으로, 메타의 최근 뉴스·실적을 한눈에 보는 대시보드와 소셜 미디어 커뮤니티 대표 반응을 모은 웹사이트를 만든다. 웹페이지/모바일용으로 볼 수 있는데, 처음부터 완성된 게 아니라 계속 명령으로 손봤다고 밝힌다 — 맨 위 주요 기업 상황 텍스트 조정, 메타 KPI(패밀리 앱·쓰레드 DAU 등 사용자 지표) 추가(처음엔 없었음), 핵심 성장 지표를 세로 나열 대신 가로로 한눈에 보이게 변경, 레이아웃 버튼 손보기 등. 이 과정을 반복하기 싫으니 스킬 크리에이터로 특정 기업 이름을 입력하면 다 해 주는 "컴퍼니 대시보드 제너레이터"를 만든다. 이 정보들은 MD 파일로 저장된다. 이후 "컴퍼니 대시보드 제너레이터 + 애플"을 치면, 미세 조정한 원하는 형식 그대로 인식해 애플 대시보드를 메타 때처럼 쫙 만들어 준다.

### [10:14–11:08] 마켓 애널라이저 & 이그제큐티브 프레젠테이션 제너레이터
스킬은 여러 개 만들 수 있다. 라면 시장을 분석해 원하는 대로 조정한 뒤 스킬로 만들면 "마켓 애널라이즈" 스킬이 생긴다. 또 임원 보고용 AI 서비스 시장 트렌드 PPT를 만드는데, 임원 보고서는 포맷이 계속 바뀌면 안 되므로 그래프 삽입 등 미세 조정을 다 해 둔 뒤 "이그제큐티브 프레젠테이션 제너레이터" 스킬로 만든다. 그러면 "AI 반도체 시장"을 돌리면 그 경영진용 형식대로 나온다.
> "매번 뭐 해 가지고 다시 프롬프트 치는게 아니라 이 스킬을 바로 호출해 가지고 내가 일하는 방식 자체를 저장하고 있다가 그걸 그대로 써 먹는 거죠." [11:01]

[의견] 요즘 PPT 수준을 보여주는 것이면서, 매번 달라지지 않고 원하는 방식을 스킬로 정의해 두면 생산성이 극대화된다는 점이 중요하다고 강조한다. 그래프도 알아서 그리고 데이터도 검증된 최근 것 기준이다.

### [11:08–12:10] 스킬 조합 — 보고서 + 임원 프레젠테이션 동시
기업 분석이 자동화될 뿐 아니라 여러 스킬을 엮어 컴퍼니 애널리시스와 PPT를 함께 만들 수도 있다. 스킬 두 개를 써서 기업 분석 보고서와 임원 프레젠테이션을 둘 다 생성 — 해당 PPT를 알아서 만들어 줄 뿐 아니라, 종합 기업 분석 보고서(페이지 요약 내용 등)까지 사용자에 맞춰 해 준다.

### [12:10–12:46] 쇼폼 영상 스킬
자기 영상을 넣고 하이라이트가 되는 부분을 찾아 쇼폼 형태 영상을 만들어 달라고 한 예도 소개한다(마누스가 원래 지원). 내용을 분석해 1분 이내로 쪼개 만들어 주고 파일도 생성한다. 여기서 스킬 크리에이터로 "새로 쇼폼 형태로 자연스럽게 확장하는 방법을 찾아 반영해 달라"고 하면, 9:16 세로로 바꾸고 블러 배경 확대, AI 리프레이밍 등을 적용하는 스킬을 만들 수 있다. 이런 것조차 스킬로 만들어 두고 바로바로 뽑을 수 있다는 것.

### [12:46–13:34] 클로드 코드와의 차이 — 웹 기반 + 샌드박스
많은 이들이 "클로드 코드로도 비슷한 거 하지 않냐, 깃허브 스킬 레포 가져다 쓰면 되지 않냐"고 물을 수 있다고 짚는다. [의견] 마누스가 이 기능을 내면서 세일즈 포인트로 삼은 부분은 웹 기반이고 별도 설정이 필요 없다는 것이다. 마누스가 알아서 안에서 해 주며, 격리된 샌드박스(버추얼 머신) 안에서 스킬을 실행한다. 샌드박스 안에서 AI가 작동하니 뭔가 잘못되거나 자동화하는 것을 이 안에서 다 구동해 위험을 원천 차단한다는 것.
> "마누스가 사실 이 기능을 내면서 포인트로 삼은 부분은 마노스는 이게 웹 기반이고 별도 설정이 이렇게 필요 없다는 거죠." [12:54]

[의견] 클로드 코드류는 다 좋은 것들이지만 로컬 머신/환경에서 실행되는 성격이 강해 세팅·안전 책임이 사용자에게 더 크게 올 수 있는데, 마누스는 웹 기반 VM 안에서 모든 걸 처리하니 잘못될 위험을 원천 차단한다고 본다.

### [13:34–14:15] 결론 — 운영·표준화 시대의 생산성
[의견] 정리하며 "AI가 대답을 잘하는 시대는 이미 왔고, 이제는 운영에서 차이가 난다"고 말한다. 마누스 스킬스는 내가 잘 만든 업무 방식(SOP)을 재사용 가능한 자산으로 만들고 같은 품질로 제안하며, 팀 단위로 그냥 넘겨 "이거 써서 하라"고 표준화할 수 있다는 것이다.
> "AI가 대답을 잘하는 시대는 이미 왔고 이제는 이제 운영에서 차이가 나는 겁니다." [13:34]

[의견] 기업 분석이건 웹페이지 제작이건 영상 편집이건 리서치·콘텐츠 제작이건, 한 번 제대로 만든 걸 계속 똑같이 뽑는 것이 진짜 생산성이라고 본다. 스킬만 잘 짜도 다른 사람이 그대로 똑같은 퀄리티로 일할 수 있는 색다른 협업 방식이 등장할 것 같아 빠르게 정리해 봤다며, "지금까지 에러였습니다"로 마무리한다.
