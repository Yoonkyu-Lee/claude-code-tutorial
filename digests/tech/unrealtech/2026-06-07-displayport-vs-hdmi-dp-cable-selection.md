# HDMI 말고 DisplayPort 쓰는 이유… DP 케이블 고를 때 진짜 봐야 할 것

> unrealtech · 2026-06-07 · [영상 링크](https://youtu.be/36JHK2Vgf_w)
> 정리 방식: 시간순 심층 워크스루 (paraphrase) + 주제별 상세 요약

## 요약 (주제별)

### 1. 규격의 태생 차이 — HDMI vs DisplayPort
- HDMI는 원래 TV·셋톱박스·콘솔·AV 리시버 같은 **거실형 영상 기기** 중심으로, 음성과 영상을 하나로 합치는 것을 시작으로 규모를 키워온 규격이다.
- DisplayPort(DP)는 그래픽카드·PC·노트북·모니터처럼 **컴퓨터 화면 출력 환경**을 강하게 의식하고 발전한 규격이다.
- 화자 요약: "HDMI는 거실에서 강한 규격, DisplayPort는 (PC) 책상 위에서 강한 규격"이다.
- 겉보기에는 둘 다 화면이 나오는 비슷한 단자지만, 태어난 배경·주로 쓰이는 환경·케이블 고르는 기준이 꽤 다르다.

### 2. PC에서 DP가 더 자주 쓰이는 이유
- PC에서는 화면만 뜨면 끝이 아니다. 고주사율(1초에 얼마나 자주 깜빡이는지), 가변 주사율(깜빡임 정도 조절), 고해상도, 멀티모니터, USB-C 연결 등이 모두 중요하다.
- DP는 이런 조건까지 고려된 규격이라 PC·게이밍 모니터 중심 환경에서 더 자주 선택된다.
- HDMI는 TV·AV 기기 연결과 넓은 호환성 유지가 중요했던 반면, DP는 PC가 요구하는 **더 높은 대역폭·더 유연한 연결 구조·확장성**을 중시했다.
- 게이밍 모니터 시장은 오랫동안 144/165/240Hz, 4K 144Hz 조합을 다뤄왔고 이 영역에서 DP가 더 익숙한 규격으로 자리 잡았다. 그래서 [의견/화자] 같은 모니터라도 최대 성능은 DP로 연결했을 때 기준인 경우가 꽤 많다.

### 3. 주사율·해상도와 대역폭의 관계
- 주사율 = 화면이 1초에 몇 번 바뀌는지 (60Hz=60번, 144Hz=144번, 240Hz=240번). 숫자가 클수록 부드럽다.
- 주사율·해상도가 올라가면 케이블이 보내야 하는 데이터 양도 같이 늘어난다. 1080p보다 4K가 훨씬 더 많은 픽셀을 담으므로 더 많은 데이터가 필요하다.
- 해상도와 주사율이 둘 다 올라가면 케이블 부담이 매우 커지며, 바로 이 지점에서 DP가 강하다.

### 4. 가변 주사율(VRR)과 멀티모니터(MST)
- 가변 주사율 = 게임 프레임이 흔들릴 때 화면 찢어짐·버벅임을 줄여주는 기술. AMD FreeSync, NVIDIA G-Sync/G-Sync Compatible 등이 이 범주다. DP는 PC 생태계에서 이 기능들과 함께 강하게 자리 잡았다.
- HDMI도 지금은 VRR 계열 기술을 지원한다(화면이 고정 박자가 아니라 게임이 뽑는 실제 프레임에 맞춰 유연하게 움직이도록). 다만 [의견/화자] PC 모니터 쪽에서는 여전히 DP 조합이 더 익숙하게 받아들여지는 경우가 많다.
- MST(Multi-Stream Transport): 하나의 출력에서 여러 화면 스트림을 다루는 구조. 하나의 DP 출력으로 커브드 화면을 쓰거나 여러 화면을 다루기 유리하다. 개발·트레이딩·편집·멀티모니터 환경처럼 여러 대를 쓰는 경우 DP가 "PC 규격 같다"는 느낌을 준다.

### 5. 반대로 HDMI가 더 맞는 경우 (균형 잡기)
- PS5·Xbox·셋톱박스·사운드바·AV 리시버·TV 쪽은 거의 HDMI 중심이다. 영상만이 아니라 오디오·기기 제어까지 AV 생태계 전체가 함께 움직이기 때문이다.
- 영화 보고 콘솔 연결하고 사운드 시스템까지 묶는 환경이라면 HDMI가 훨씬 잘 맞는다.
- 모니터라고 무조건 DP를 써야 하는 것도 아니다. 사무형 모니터, 간단한 듀얼 모니터, 노트북 발표용 연결처럼 240Hz나 고급 가변 주사율이 필요 없는 환경은 HDMI로 연결해도 문제없다.
- 화자 결론: 핵심 질문은 "HDMI가 더 좋냐 DP가 더 좋냐"가 아니라 "**내 환경에 뭐가 더 잘 맞느냐**"이다.

### 6. DP 케이블 고르는 기준 — 버전명이 아니라 인증 등급
- 지난 HDMI 편에서는 "2.1이라 써 있어도 실제 성능은 제품마다 다를 수 있다"가 핵심이었다. DP 케이블은 조금 다른 방식으로 봐야 한다.
- 많은 사람이 "DP 1.4 케이블", "DP 2.1 케이블"이라 부르지만, 실제로 중요한 건 그 이름보다 **이 케이블이 어느 정도 속도·대역폭을 제대로 통과시키느냐**이다.
- 대역폭 = 한 번에 얼마나 많은 데이터를 보낼 수 있느냐. (비유: 차선이 넓거나 차가 빨리 달릴수록 같은 시간에 더 많은 차가 지나간다.)
- 최신 세대 DP 케이블은 **DP40 / DP54 / DP80** 식으로 나뉘며, 숫자는 그 케이블이 감당하는 대역폭 급 — 각각 40Gbps / 54Gbps / 80Gbps급이다.
- 그래서 "DP 2.1이라 적혀 있나"보다 "실제로 어떤 **인증 등급**을 받았나"를 보는 게 더 중요하다.

### 7. DP의 4레인 구조 · HBR/UHBR 속도 등급
- DP는 기본적으로 4개의 고속 데이터 레인(레인=데이터가 지나가는 차선)을 쓴다. "4차선 도로"로 이해하면 된다.
- 세대가 올라가며 이 4차선 각각의 속도가 빨라졌고, 이를 표현하는 용어가 HBR·UHBR이다. HBR = High Bit Rate(고속 전송 모드), UHBR = Ultra High Bit Rate(더 높은 속도의 최신 고속 전송 모드).
- 초기에는 HBR → HBR2 → HBR3로 올라왔고, 최근 DP 2.x 세대에서는 UHBR10 → UHBR13.5 → UHBR20으로 올라오며 40Gbps → 54Gbps → 80Gbps급까지 커졌다.
- 핵심: 같은 4차선 도로라도 세대가 오르면 차가 더 빨리 달려, 같은 시간에 더 많은 데이터를 보내 더 높은 해상도·더 잦은 주사율·더 빡센 모니터 환경을 감당한다.

### 8. 인코딩 효율 — 8b/10b vs 128b/132b
- 고속 인터페이스는 원본 데이터를 그냥 생으로 보내면 수신 쪽에서 신호 경계가 흐려지거나 타이밍을 헷갈리거나 비트가 흔들렸을 때 복원이 어려워질 수 있다. 그래서 원래 데이터에 **추가 비트**를 붙여 보낸다(비유: 택배에 방향 표시·완충재·라벨을 붙여 보내는 것). 이 추가 비트 덕에 수신 쪽이 신호가 제대로 들어오는지, 어디까지가 한 덩어리인지, 흔들린 부분을 어떻게 재해석할지 판단한다.
- DP 1.x(HBR/HBR2/HBR3) 시절: **8b/10b 인코딩** — 실제 8비트 데이터를 보내려고 10비트를 전송(2비트 추가). 실제 데이터 효율 약 **80%**.
- DP 2.x(UHBR10/13.5/20): **128b/132b 인코딩** — 128비트 데이터에 4비트만 추가해 전송. 효율 약 **97%**까지 상승.
- 결과 예: UHBR20의 총 링크 속도는 80Gbps인데 실제 쓸 수 있는 데이터량은 약 **77Gbps대**로 올라간다. 즉 최신 DP는 숫자만 커진 게 아니라 추가 비트를 훨씬 덜 붙이면서 안정적으로 복원하도록 바뀌었다.

### 9. USB-C 연결 — DP Alt Mode의 이점
- 요즘 노트북은 HDMI 포트 없이 USB-C 포트만 있는 경우가 많아 USB-C로 모니터를 연결하게 된다. 중요한 건 **USB-C는 포트 "모양"일 뿐**이라는 것 — 겉이 USB-C라고 안에서 다 같은 신호가 흐르는 건 아니다.
- **DP Alt Mode**: 원래 USB-C는 USB 데이터용이지만, 필요할 때 그 통로를 활용해 DP 신호를 내보낼 수 있다. 그래서 모니터에 DP 포트가 있으면 USB-C to DP 연결은 매우 자연스러운 선택이다(노트북 USB-C 안에 이미 DP가 들어 있고, 그걸 비교적 곧바로 모니터 DP 입력으로 이어줌).
- 반면 USB-C to HDMI는 기기 조합에 따라 중간에 **프로토콜 변환기**가 들어간다(USB-C에서 나온 DP 계열 신호를 HDMI가 알아듣는 방식으로 변환). USB-C의 DP Alt Mode는 예전 DP 단자의 DP++ 방식을 그대로 지원하지 않으므로, USB-C→HDMI는 대부분 중간에 DP→HDMI 변환칩이 들어간다.
- 그래서 USB-C to DP가 USB-C to HDMI보다 구조가 단순한 경우가 많고, 기기·모니터가 지원한다면 고해상도·고주사율·VRR 같은 기능이 더 자연스럽게 연결될 가능성이 크다.

### 10. DP80 케이블이 짧은 이유 · 흔한 문제 · 실전 선택 가이드
- DP80(80Gbps급) 케이블은 시중 제품 길이가 1m 안팎인 경우가 많다. 이유는 "너무 빨라서". 속도가 올라갈수록 케이블 안에서 신호가 약해지고 타이밍이 흔들리고 옆선과 간섭도 심해져 고속 신호를 멀리 보내기 어렵다. 80Gbps급 패시브 구리 케이블은 길수록 훨씬 빡세다.
- 흔한 문제: **144Hz 모니터인데 60Hz만 뜬다** — 모니터·그래픽카드 탓으로 생각하지만 의외로 케이블이 병목인 경우가 많다. 케이블이 대역폭을 못 버티면 연결은 되더라도 더 낮은 속도로 타협해 잡히므로, 화면이 나온다고 항상 제대로 연결된 건 아니다.
- **래치(걸쇠) 주의**: DP 커넥터는 꽂으면 고정되는 래치 구조가 많다. 모르고 억지로 잡아당기면 포트·케이블이 손상될 수 있으니 주의.
- 실전 선택 가이드:
  - 1080p·1440p 60~144Hz급: 품질이 너무 떨어지는 비인증 제품만 피하면 문제없는 경우가 많다.
  - 1440p 240Hz, 4K 120~144Hz급부터: 케이블 품질 차이가 체감되기 시작 — 아무 DP 케이블이 아니라 실제 인증 등급을 봐야 한다.
  - 4K 240Hz·8K급: 대역을 정말 많이 먹으므로 DP40/DP54/DP80을 구분해서 보고, 압축 없이 갈지 **DSC**를 쓸지도 같이 봐야 한다.
  - **DSC**(Display Stream Compression): 화면 데이터를 아주 효율적으로 압축해 대역폭 부담을 줄이는 기술. 케이블 능력이 빡빡할 때 부담을 줄여주는 보조 수단.
  - USB-C to DP는 깔끔한 경우가 많지만, USB-C 모양만 보지 말고 실제 포트가 USB4인지·썬더볼트 4인지·썬더볼트 5인지도 같이 봐야 한다.
- 최종 정리: "TV·콘솔은 HDMI, 게이밍 모니터는 DP"라는 큰 그림 아래에서, 각자 쓸 해상도·주사율·기능을 고려해 실제 인증 등급과 대역폭으로 케이블을 고르는 것이 좋다.

## 3줄 오리엔테이션
- HDMI는 거실(TV·콘솔·AV) 규격, DisplayPort는 PC·게이밍 모니터 규격 — 태생과 강점 환경이 다르다.
- PC에서 DP가 더 자주 쓰이는 이유(고주사율·VRR·멀티모니터·USB-C DP Alt Mode)를 짚고, 반대로 HDMI가 맞는 경우도 균형 있게 제시한다.
- DP 케이블은 "DP 1.4냐 2.1이냐"가 아니라 **DP40/DP54/DP80 인증 등급과 대역폭**으로 골라야 한다는 게 실전 핵심.

## 전문 — 시간순 워크스루

### [00:00–01:04] 도입 — HDMI 잘 되는데 왜 굳이 DisplayPort?
지난 영상에서 HDMI 케이블 고를 때 뭐가 중요한지, 왜 "2.1"이라 적혀 있어도 실제 성능은 제품마다 다를 수 있는지를 설명했는데, PC·게이밍 모니터 쪽으로 오면 이야기가 조금 달라진다고 운을 뗀다. "잘 되는 HDMI가 있는데 왜 굳이 DP를 쓰지?", "PC나 모니터 바꾸고 나서 왜 DP 규격으로 바뀌지?" 같은 질문이 자주 나온다는 것. 실제로 그래픽카드 뒤에는 HDMI와 DP가 같이 달렸거나 아예 DP만 있는 경우도 많고, 모니터 뒤에도 요즘은 두 포트가 다 있는 경우가 많다. 겉보기에는 비슷해도 두 규격은 태어난 배경·쓰이는 환경·케이블 고르는 기준이 꽤 다르다며, 오늘은 ①왜 PC에서 DP를 더 많이 쓰는지 ②반대로 어떤 상황에서 HDMI가 더 자연스러운지 ③DP 케이블은 어떻게 골라야 하는지를 한 번에 정리하겠다고 예고한다.

### [01:08–01:37] 두 규격의 태생 차이
지난 영상에서 말했듯 HDMI는 원래 TV·셋톱박스·콘솔·AV 리시버 같은 거실형 영상 기기 중심으로, 음성과 영상을 합치는 것을 시작으로 규모를 키워온 규격이다. 반면 DP는 그래픽카드·PC·노트북·모니터처럼 컴퓨터 화면 출력 환경을 더 강하게 의식하고 발전한 규격이다.
> "쉽게 말하면 HDMI는 거실에서 강한 규격이고 DisplayPort는 (PC) 책상 위에서 강한 규격입니다." [01:31]

### [01:37–02:08] PC에서는 "화면만 뜨면 끝"이 아니다
이 차이가 중요한 이유는 PC에서는 단순히 화면만 뜨면 되는 게 아니기 때문이다. 1초에 얼마나 자주 깜빡거리는지를 나타내는 고주사율, 깜빡거림 정도를 조절하는 가변 주사율, 얼마나 촘촘하고 선명한지를 나타내는 고해상도, 멀티모니터나 USB-C 연결 같은 것도 다 중요하다. 이런 조건까지 고려된 규격이 DP이기에 PC·게이밍 모니터 중심 환경에서 더 자주 선택된다.

### [02:08–02:26] HDMI는 호환성, DP는 대역폭·확장성
HDMI는 TV·AV 기기에 잘 연결되고 많은 사람이 쉽게 쓸 수 있으며 호환성이 넓게 유지되는 게 굉장히 중요했다. 반면 DP는 PC 쪽이 요구하는 더 높은 대역폭, 더 유연한 연결 구조, 확장성 같은 부분을 더 중요하게 봤다.

### [02:26–03:09] 주사율과 해상도가 케이블에 주는 부담
"DP는 실제로 뭐가 더 좋을까?"를 주사율부터 본다. 주사율은 화면이 1초에 몇 번 바뀌는지 — 60Hz는 60번, 144Hz는 144번, 240Hz는 240번. 숫자가 올라갈수록 화면이 부드러워진다. 문제는 주사율이 올라가면 케이블이 보내야 할 데이터 양도 같이 늘어난다는 것. 해상도도 마찬가지로, 1080p보다 4K가 훨씬 많은 픽셀을 담아 더 많은 데이터를 보내야 한다. 즉 해상도와 주사율이 둘 다 올라가면 케이블 부담이 매우 커지고, 바로 이 지점에서 DP가 강하다.

### [03:09–03:29] 게이밍 모니터 시장에서 자리 잡은 DP
게이밍 모니터 시장은 오랫동안 144Hz·165Hz·240Hz·4K 144Hz 같은 조합을 많이 다뤄왔고, 이 영역에서 DP가 훨씬 더 익숙한 규격으로 자리 잡았다. [의견/화자] 그래서 같은 모니터라도 최대 성능은 DP로 연결했을 때 기준인 경우가 꽤 많다.

### [03:29–04:08] 가변 주사율(VRR) — DP가 강한 영역
여기서 나오는 말이 가변 주사율이다. 쉽게 말해 게임 프레임이 흔들릴 때 화면 찢어짐이나 버벅임을 줄여주는 기술로, AMD FreeSync, NVIDIA G-Sync, G-Sync Compatible 같은 기능이 이 범주에 든다. DP는 이런 기능들과 함께 PC 생태계에서 강하게 자리 잡았다. 물론 HDMI도 지금은 VRR 계열 기술을 지원한다 — 화면이 고정된 박자로만 움직이지 않고 게임이 실제로 뽑아내는 프레임에 맞춰 모니터가 유연하게 따라가도록 해준다. [의견/화자] 그런데 PC 모니터 쪽에서는 여전히 DP 조합이 더 익숙하고 자연스럽게 받아들여지는 경우가 많다.

### [04:08–04:43] 멀티모니터(MST)와 DP
DP는 멀티모니터 환경에서도 꽤 강하다. 흔히 MST라고 부르는 이 기술은 하나의 출력에서 여러 화면 스트림을 다루는 구조로, 하나의 DP 출력을 기준으로 커브드 화면을 쓰거나 특정 구성에서 여러 화면을 다루기 유리하다는 뜻이다. 딱 봐도 TV보다는 일하면서 게임하고 노래 듣고 주식하는 현대인의 PC 환경에 더 잘 맞는 기능이다. 개발·트레이딩·편집 등 모니터 여러 대를 쓰는 경우 DP가 "더 PC 규격 같다"는 느낌을 주는 이유가 여기에 있다.

### [04:43–05:24] 그렇다고 DP가 무조건 좋은 건 아니다 — HDMI가 맞는 경우
지금까지 들으면 DP가 무조건 더 좋은 것처럼 느껴질 수 있는데 꼭 그렇진 않다. PS5·Xbox·셋톱박스·사운드바·AV 리시버·TV 쪽은 거의 HDMI 중심이라고 보면 된다. 영상만 연결하는 게 아니라 오디오·기기 제어 등 AV 생태계 전체가 같이 움직이기 때문이다. 그래서 영화 보고 콘솔 연결하고 사운드 시스템까지 묶는 환경이라면 HDMI가 훨씬 잘 맞는다. 모니터라고 해서 무조건 DP를 써야 하는 것도 아니다. 사무형 모니터, 간단한 듀얼 모니터, 노트북 발표용 연결처럼 굳이 240Hz나 고급 가변 주사율까지 필요 없는 환경도 많고, 그런 경우 HDMI로 연결해도 아무 문제 없다.

### [05:24–05:52] 결론적 질문 & DP 케이블은 다르게 봐야
결국 중요한 질문은 "HDMI가 더 좋냐, DP가 더 좋냐"가 아니라 "내 환경에는 뭐가 더 잘 맞느냐"이다. 지난 HDMI 편에서는 "2.1이라 써 있어도 실제 성능은 제각각일 수 있다"가 중요했다면, DP 케이블은 조금 다른 방식으로 봐야 한다. 많은 사람이 "DP 1.4 케이블", "DP 2.1 케이블"이라 부르는데, 실제로 중요한 건 그 이름보다 이 케이블이 어느 정도 속도·대역폭을 제대로 통과시키느냐이다.

### [05:52–06:29] 대역폭과 DP40/DP54/DP80
대역폭은 한 번에 얼마나 많은 데이터를 보낼 수 있느냐를 뜻한다(비유: 차선이 넓거나 차가 빨리 달릴수록 같은 시간에 더 많은 차가 지나감). 케이블도 똑같아서, DP 케이블은 버전 번호보다 실제로 어느 정도 대역폭을 버틸 수 있는지가 더 중요하다. DP 케이블은 최신 세대로 갈수록 DP40, DP54, DP80 식으로 나뉘고, 여기서 숫자는 그 케이블이 감당하는 대역폭 급 — 각각 40Gbps, 54Gbps, 80Gbps급이라고 이해하면 된다.

### [06:29–06:40] 버전명보다 인증 등급
그래서 DP 케이블을 볼 때는 "DP 2.1이라고 적혀 있나"를 보는 것보다 실제로 어떤 인증 등급을 받았는지를 보는 게 더 중요하다.

### [06:40–07:11] 4개 레인 구조와 HBR/UHBR
DP는 기본적으로 4개의 고속 데이터 레인을 쓴다. 레인은 쉽게 말해 데이터가 지나가는 차선으로, DP는 4차선 도로를 쓴다고 생각하면 된다. 세대가 올라가면서 이 4개 차선 하나하나의 속도가 빨라졌고, 이를 표현하는 용어로 HBR·UHBR이 등장했다. HBR은 High Bit Rate의 줄임말로 고속 전송 모드이고, UHBR은 Ultra High Bit Rate로 더 높은 속도의 최신 고속 전송 모드다.

### [07:11–07:25] HBR 세대에서 UHBR 세대로
초기에는 HBR, HBR2, HBR3로 올라왔고, 최근 DP 2.x 세대에서는 UHBR10, UHBR13.5, UHBR20으로 올라오면서 40Gbps, 54Gbps, 80Gbps급까지 커졌다.

### [07:25–07:45] 핵심 정리 — 같은 4차선, 더 빠른 차
핵심은 하나다. 세대가 올라갈수록 같은 4차선 도로라도 차가 더 빨리 달리게 됐다. 그래서 같은 시간 내 더 많은 차량(데이터)을 쏘아 보낼 수 있으니, 더 높은 해상도로 더 자주 화면이 바뀌게, 그리고 더 빡센 모니터 환경을 감당할 수 있게 됐다.

### [07:45–08:33] 인코딩이 왜 필요한가 — 추가 비트의 역할
기술적으로 또 하나 중요한 게 인코딩이다. 케이블 안에서는 영상 데이터가 엄청난 속도로 쏟아지는데, 이걸 그냥 생으로 보내면 받는 쪽에서 신호 경계가 흐려지거나 타이밍을 헷갈리거나, 중간에 일부 비트가 흔들렸을 때 원래 데이터를 안정적으로 해석하기 어려워질 수 있다. 그래서 DP 같은 고속 인터페이스는 원래 데이터에 추가 비트를 붙여 보낸다. 이 추가 비트는 낭비가 아니라, 받는 쪽이 지금 신호가 제대로 들어오는지, 어디까지가 한 덩어리인지, 흔들린 부분이 있으면 어떤 규칙으로 다시 해석해야 하는지를 더 안정적으로 판단하게 돕는다.
> "쉽게 말하면 물건만 택배로 보내는 게 아니라 박스에 방향 표시도 붙이고 완충제도 놓고 라벨도 정리해서 보내는 느낌에 가깝습니다." [08:23]

### [08:33–08:46] 100비트가 전부 순수 영상은 아니다
케이블 안에서 100만큼의 비트가 지나가도 그 100이 전부 순수 영상 데이터는 아닐 수 있다는 뜻이다. 일부는 신호를 더 안정적으로 전달하고 받는 쪽이 더 정확하게 복원하도록 돕는 역할을 한다.

### [08:46–09:10] 8b/10b 인코딩 (DP 1.x)
DP 1.x, 즉 HBR·HBR2·HBR3를 쓰던 시절에는 기본적으로 8b/10b 인코딩을 사용했다. 이건 원래 데이터 8비트를 보내기 위해 10비트를 전송하는 방식으로, 8비트 실제 데이터에 추가로 2비트를 더 붙여 보내는 구조다. 그래서 실제 데이터 효율은 약 80% 수준이 된다.

### [09:10–09:53] 128b/132b 인코딩 (DP 2.x)과 효율 97%
DP 2.x 세대로 오면서 큰 변화가 생긴다. UHBR10/13.5/20 같은 최신 DP 2.x 링크에서는 128b/132b 인코딩을 쓴다. 이건 실제 데이터 128비트를 보내기 위해 132비트를 전송한다는 뜻으로, 비율적으로 예전보다 훨씬 적은 추가 비트만 붙이고도 신호를 안정적으로 다룬다. 효율로 보면 약 97% 수준까지 올라간다. 즉 UHBR20의 총 링크 속도는 80Gbps인데 실제로 쓸 수 있는 데이터량은 약 77Gbps대로 올라간다. 최신 DP는 숫자만 커진 게 아니라 추가 비트를 훨씬 덜 붙이면서도 안정적으로 복원할 수 있게 바뀐 것이다.

### [09:53–10:31] USB-C는 "모양"일 뿐 — DP Alt Mode
요즘 노트북은 HDMI 포트 없이 USB-C 타입 포트만 있는 경우가 많아 자연스럽게 USB-C로 모니터를 연결하게 된다. 여기서 중요한 건 USB-C는 포트 모양일 뿐이라는 것 — 겉모양이 USB-C라고 그 안에서 다 똑같은 신호가 흐르는 건 아니다. 여기서 나오는 게 DP Alt Mode(얼터네이트 모드)다. 원래 USB-C는 USB 데이터용 포트인데, 필요할 때 그 통로를 활용해 DP 신호를 내보낼 수도 있다는 것. 그래서 모니터에 DP 포트가 있다면 USB-C to DP 연결은 굉장히 자연스러운 선택이다. 쉽게 말해 노트북 USB-C 포트 안에 이미 DP가 들어 있고, USB-C to DP는 그걸 비교적 곧바로 모니터 쪽 DP 입력으로 이어주는 느낌이다.

### [10:31–11:04] USB-C to HDMI는 변환 과정이 낀다
반면 USB-C to HDMI는 기기 조합에 따라 중간에 프로토콜 변환기가 들어간다. 쉽게 말해 서로 다른 언어를 통역하는 과정이 있다는 것 — 원래 USB-C 쪽에서 나온 DP 계열 신호를 HDMI가 알아들을 수 있는 방식으로 바꿔줘야 한다. 그래서 USB-C to DP가 USB-C to HDMI보다 구조가 단순한 경우가 많다.

### [11:04–11:36] DP++ 미지원과 변환칩, 그리고 이점
USB-C의 DP Alt Mode는 예전 DP 단자의 DP++ 방식을 그대로 지원하지 않는다. 그래서 USB-C에서 HDMI로 연결할 때는 단순히 선만 바뀌는 게 아니라 대부분 중간에 DP 신호를 HDMI 신호로 바꿔주는 변환칩이 들어간다. 반면 USB-C에서 DP로 가는 경우는 같은 DP 계열로 이어지는 경우가 많아 구조가 비교적 단순하다. 그래서 기기와 모니터가 지원한다면 고해상도·고주사율·VRR 같은 기능이 더 자연스럽게 연결될 가능성이 크다.

### [11:36–12:04] DP80 케이블은 왜 이렇게 짧을까
DP80이면 80Gbps급 케이블인데, 막상 시중 제품을 보면 길이가 1m 안팎인 경우가 많다. 이유는 간단하다 — 너무 빨라서다. 속도가 올라갈수록 케이블 안에서 신호가 약해지고 타이밍이 흔들리고 옆선과 간섭도 심해진다. 즉 고속 신호를 멀리 보내기가 점점 어려워진다. 그래서 80Gbps급 패시브 구리 케이블은 길어질수록 훨씬 더 빡세다.

### [12:04–12:31] 흔한 문제 ① 144Hz인데 60Hz만 뜬다
실제 사용자들이 가장 많이 겪는 문제를 살펴본다. "144Hz 모니터인데 60Hz만 뜬다"는 정말 흔하게 나타난다. 사람들은 보통 모니터 문제인가 그래픽카드 문제인가부터 생각하는데, 의외로 케이블이 병목인 경우가 많다. 케이블이 충분한 대역폭을 못 버티면 연결은 되더라도 더 낮은 속도로 타협해서 잡혀버릴 수 있다. 즉 화면이 나온다고 해서 항상 제대로 연결된 건 아니다.

### [12:31–12:52] 흔한 문제 ② 래치(걸쇠) 손상 주의
DP 케이블은 잘 안 빠지는데 함부로 뺐다가 손상되는 경우가 있다. DP 커넥터는 꽂았을 때 고정되는 래치 구조가 달린 경우가 많은데, 래치란 쉽게 말해 걸쇠다. 한 번 꽂으면 덜렁덜렁 빠지지 않도록 잡아주는 구조인데, 이걸 모르고 억지로 잡아당기면 포트나 케이블이 손상될 수 있으니 주의해야 한다.

### [12:52–13:33] DP 케이블 선택 핵심 정리 & DSC
DP 케이블을 어떻게 고르면 되는지 핵심만 정리한다. 1080p·1440p 60~144Hz급은 품질이 너무 떨어지는 비인증 제품만 피하면 문제없는 경우가 많다. 그런데 1440p 240Hz, 4K 120~144Hz급 구간부터는 케이블 품질 차이가 체감되기 시작하며, 이제는 아무 DP 케이블이 아니라 실제 인증 등급을 보는 것이 중요하다. 더군다나 4K 240Hz·8K급은 진짜로 대역을 많이 먹는 영역이라 DP40, DP54, DP80을 구분해서 봐야 하고, 압축 없이 갈지 DSC를 쓸지도 같이 봐야 한다. 여기서 DSC는 Display Stream Compression으로, 화면 데이터를 아주 효율적으로 압축해 대역폭 부담을 줄이는 기술이다. 케이블 능력이 아주 빡빡할 때 그 부담을 줄여주는 보조 수단으로 보면 된다.

### [13:33–13:59] USB-C 연결 시 포트 실체 확인
모니터에 DP 포트가 있다면 USB-C to DP 연결이 상당히 깔끔한 경우가 많다. 다만 USB-C 모양만 보고 판단하면 안 되고, 실제 포트가 USB4인지, 썬더볼트 4인지, 썬더볼트 5인지도 같이 봐야 한다.

### [13:59–14:29] 두 편의 핵심 요약

> HDMI와 DisplayPort는 둘 다 화면을 출력하는 규격이지만, 태어난 배경과 주로 쓰이는 환경이 다릅니다.
> HDMI가 TV, 콘솔, 셋톱박스, AV 기기 중심으로 발전했다면, DisplayPort는 PC, 그래픽카드, 노트북, 게이밍 모니터 환경에 더 맞춰 발전해왔습니다.

> 하지만 HDMI가 무조건 나쁘고 DP가 무조건 좋은 것은 아닙니다.
> 콘솔, TV, 사운드바, AV 리시버처럼 거실형 기기 생태계에서는 여전히 HDMI가 훨씬 자연스러운 선택입니다.
