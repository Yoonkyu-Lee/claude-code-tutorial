## 요약 (주제별)

### 1. 핵심 논지 — "흑백 논리로 볼 문제가 아니다"
- 시장에는 두 극단 주장이 섞여 있다: "TPU가 엔비디아를 다 쓸어버린다" vs "CUDA 해자 때문에 TPU는 절대 침투 못 한다". 화자는 **둘 다 부분적으로 맞고**, 뒤섞여서 사람들이 헷갈린다고 본다.
- 세상은 이분법이 아니라 스펙트럼·다차원이므로, "TPU가 이겼다/졌다"로 단순 해석하면 안 된다. 결론: 엔비디아 독점 시장의 **일부를 TPU가 커버할 수 있게 됐다는 변화 양상**이 핵심이지, 완전 대체나 완전 방어가 아니다.

### 2. 왜 지금 갑자기 화제가 됐나
- Gemini(제미나이) 3.0 Pro가 TPU로 전량 학습해 나왔고 벤치마크 점수·사용자 만족도가 압도적으로 높게 나온 것 + 나노바나나 프로(이미지) 호평이 겹쳐 "구글이 자체 설계 칩으로 좋은 걸 내놨다"는 주목을 받음.
- 단, TPU 전량 학습 자체는 처음이 아님 — **제미나이 2.0부터** 이미 있던 얘기라 이번이 완전한 신규 사건은 아니라는 점을 화자가 명확히 함.
- 또 하나의 변화: 원래 구글은 TPU를 자사 학습·AI 서비스·구글 클라우드에만 쓴다고 했는데, 이번에 아이언우드(Ironwood)를 포함해 **AWS 등 외부 기업에도 엔비디아가 GPU 팔듯 TPU를 공급**하기 시작한 것. 화자는 이걸 "어마어마한 일"로 평가.

### 3. CUDA 해자의 실체 — 커널 엔지니어와 OS 비유
- CUDA는 여전히 강력한 해자가 맞고 "옛날 얘기"가 아니다. 다만 어떤 부분이 해자인지 설명이 잘 안 되는 경우가 많음. 화자는 사카나 AI(Sakana AI)의 "AI CUDA Engineer" 정리를 인용해 설명.
- CUDA를 쓴다고 다 CUDA 엔지니어가 아니다. **하드웨어(CUDA 코어·텐서코어·트랜스포머 엔진)가 실제로 어떻게 동작하고 메모리를 어디서 읽어오는지 세밀하게 이해**하고, PyTorch 일반 연산보다 10~100배 빠른 새 커널을 직접 디자인하는 사람은 CUDA 사용자 중 **상위 1%(커널 엔지니어)**뿐. (주장/미검증)
- 구조: 그 1%가 커널을 만들면 나머지 99%가 그걸 가져다 쓴다. 커널은 쉽게 말해 **OS(윈도우·맥)** 같은 존재 — CPU/GPU/스토리지/메모리를 어떻게 운영할지 아는 시스템. 커널 엔지니어 몸값이 매우 비싸고 AI 세계를 리딩한다.

### 4. PyTorch vs JAX / GPU vs TPU — 언어 자체가 다르다
- 메타·오픈AI·테슬라 등 메이저 AI 기업 대다수(약 80%, 주장/미검증)가 **PyTorch 기반**. 반면 구글은 **JAX(+TensorFlow)** 생태 → 하드웨어를 구동하는 언어가 근본적으로 다름(영어 vs 프랑스어 비유; x86 vs ARM 비유).
- 실행 방식 차이: PyTorch는 파이썬처럼 **한 줄씩 실행(인터프리터)** → 디버깅 쉽지만 느릴 수 있음. TPU는 **그래프 컴파일러**로 전체 연산 그래프를 한 번 그린 뒤 컴파일해 실행 → 잘 컴파일하면 빠르지만, 중간에 변화가 들어와 재컴파일이 필요하면 중단 이슈.
- 수렴 중: PyTorch 2.0부터 그래프 컴파일 도입, JAX도 부분 컴파일 캐싱으로 유연화. x86과 ARM이 서로 닮아가듯 두 진영도 닮아가지만 **태생적으로 타겟이 다르다**는 점은 유지.

### 5. TPU 하드웨어 구조와 전성비
- TPU는 10년 됨. 1세대 V1 때 이미 GPU 대비 **15~30배 전성비 우위**(주장/미검증). V5/V6로 오면서 칩 단위는 블랙웰과 **대등**해졌다고 봄.
- 시스템 레벨 우위: 시스톨릭 어레이(데이터가 칩 내부 계산기 사이로 흐름, "심장이 피를 쏘듯"), **3D 토러스** 인터커넥트(칩이 동서남북·위아래 6방향으로 직접 케이블 연결) → 별도 스위치 장비 불필요, 확장성 내장. 리퀴드 쿨링으로 9000개 넘는 칩 연결.
- 반면 엔비디아는 NVLink·NVSwitch(랙 내부)·Spectrum X·InfiniBand(랙 간)로 연결 → 스위치 장비가 비싸지만 "어디든 빠르게 보낼 수 있는 고속도로". 결론: 구글은 **스위치·전력 값을 아껴 가성비 좋은 공장**, 엔비디아는 비싸도 유연.
- 트레이드오프: TPU는 멀리 있는 칩과 통신 시 중간 다리를 여러 번 건너 지연 발생 가능. GPU는 각 코어가 개별 메모리 접근("오토바이 배달") → 데이터 이동 낭비 가능.

### 6. 엔비디아의 대응 전략 — 루빈 CPX & DGX Spark
- **루빈 CPX(Context Prefill Acceleration)**: HBM 대신 **GDDR7** 사용(대역폭 계산상 약 32GB/s급, 삼성 GDDR7 언급; 하이닉스 48GB 고속품도 있으나 ~30GB대로 충분히 커버 가능 — 화자 계산). 추론을 프리필(문서 이해, self-attention, 연산량 많지만 HBM 덜 필요 → 값싼 GDDR)과 디코드(실제 답변 생성, 메모리 대역폭 필요 → HBM)로 역할 분리. **[의견]** 화자는 루빈 CPX가 나온 이유를 "TPU 때문"이라고 개인적으로 봄.
- **DGX Spark**: AI PC로 불리지만 **소비자용 PC가 절대 아님** — Ubuntu OS 기반, AI 엔지니어용. 젠슨이 **CUDA 생태 유지**를 위해 전략적으로 만든 것. 데이터센터 GPU 할당이 부족하니, 서버와 동일한 시스템을 작게 보급해 로컬에서 개발 후 센터로 포팅 → 호환성으로 CUDA 해자를 더 강화. 원래 이름 "Project Digits", "서버를 그대로 작게 만들었다"고 했지 개인용이라 한 적 없음. 그래서 맥미니·맥스튜디오보다 **로컬 성능은 오히려 떨어짐**(개발용이라서).
- 추론 시장이 학습보다 칩이 더 많이 팔릴 영역(사용자 수 증가). AMD도 MI400/MI450X 랙스케일 헬리오스 시스템으로 추론 대응.

### 7. 구글의 진짜 무서운 점 — 풀스택
- 애플리케이션(응용 스택) + 파운데이션 모델 + 클라우드 + **하드웨어**까지 다 가진 기업은 지금 구글밖에 없음. 애플(OS+하드웨어 수직통합)에 비유 — 자기가 뭘 돌릴지 정확히 아니까 특화 최적화 가능(유튜브 코덱 전용기, 추천 알고리즘 전용 하드 등).
- 엔비디아는 어디까지나 **칩 판매 중심** 회사(고객: 기상청·게임사·오픈AI·구글 등). 파운데이션 모델·클라우드(DGX Cloud)는 있으나 약하고 응용 서비스는 직접 안 함(B2B). 구글은 클라우드가 있어 TPU로 더 싸게 굴려 Gemini 3.0 Pro도 저렴하게 출시.

### 8. 시장 전망 — 마진, 점유율, 추론 시대
- 엔비디아 매출총이익률 70% 중반·영업이익률 60%(주장/미검증) = "엔비디아 텍스(세금)". 빅테크가 칩을 10만 개 단위로 굴리며 70% 마진을 주는 건 거의 **두 배 값을 주는 낭비**.
- 회사 의사결정: CTO(빠른 개발 → CUDA)와 CFO(비용 → TPO)가 충돌 → 장기적으로 엔비디아 탈피 압력. "누가 가장 빠른 칩이냐"는 핵심이 아님 — 구글은 애초에 **속도로 엔비디아를 이기려 하지 않음**, 마진 없이 원가로 먹고 들어감.
- 프리미엄 시장(랩·AI 스타트업·대학 연구·선행 R&D)은 속도·유연성이 중요하고 인건비 > 하드웨어 비용이라 여전히 엔비디아 수요 유지. VC도 "H100 몇 천 개 확보"를 경쟁력으로 봄. 현재도 대부분 논문이 CUDA 기반 → CUDA 해자 여전히 강력, 점유율·가격 결정력 유지, GPU는 여전히 부족.
- 다만 아직 대중화 전 단계(추론 시대 도래). AI 서비스가 일상화·특정화될수록 변화 적고 타겟 고정된 **전용 하드웨어 시장(TPU류)**이 유리 → 엔비디아는 시장이 커질수록 **점유율(%)은 내려가되 매출 규모는 계속 커짐**. 엔비디아도 이를 알기에 GPU만 팔지 않고 서버 전체·NVLink·AI Enterprise·CPX를 따로 판매. 결국 완전 대체가 아니라 **듀얼(dual) 병행 + 레버리징**으로 감.

### 9. 한국 메모리 기업 수혜
- TPU도 GPU도 결국 **HBM이 엄청나게 많이 필요** → 한국 기업(삼성·하이닉스)에는 오히려 호재. 엔비디아가 CPX를 내놔도 **GDDR7을 결국 한국이 만든다** → 메모리 시장 기회는 여전히 많음.

## 3줄 오리엔테이션
- 구글 Gemini 3.0 Pro가 TPU 학습으로 뜨고 TPU를 외부 판매하기 시작하면서 "엔비디아 위기론"이 나왔는데, 화자는 CUDA 해자의 실체와 TPU·GPU의 구조적 차이를 뜯어 "완전 대체도 완전 방어도 아니다"라고 정리한다.
- CUDA 해자는 여전히 강력(커널 엔지니어 상위 1%, PyTorch 생태)하지만 엔비디아 70% 마진이 너무 비싸 빅테크가 자체칩·TPU로 이탈할 유인이 커졌고, 엔비디아는 루빈 CPX·DGX Spark로 각각 추론 시장 방어와 CUDA 생태 강화를 노린다.
- 추론 시대가 오면서 전용 하드웨어(TPU) 시장이 커져 엔비디아 점유율은 %로는 하락하되 시장 파이 자체가 커지고, TPU·GPU 모두 HBM을 대량 소모해 한국 메모리 기업엔 호재라는 결론.

### [00:00–00:12] 훅: 엔비디아의 어마어마한 마진
영상은 엔비디아 매출총이익률을 실시간으로 계산하는 장면으로 연다. 70%가 넘는 고마진이며, 팹리스 수익 구조에서 매출총이익률 70% 중반·영업이익률 60%(주장/미검증)면 "진짜 어마어마하게 돈을 벌고 있다"고 강조한다.

### [00:15–01:24] 오프닝: TPU 화제와 이 영상이 답하려는 질문
호스트(에러)는 구글 Gemini 3.0 Pro가 TPU로 전량 학습해 나왔고 성능이 압도적으로 높아 관심이 크다고 소개한다. 동시에 구글이 아이언우드를 포함한 TPU를 AWS 등 외부에도 엔비디아가 GPU 팔듯 공급한다고 해서 "엔비디아 아성이 무너진다, 위험하다"는 얘기가 나왔고(본인도 그런 언급을 한 적 있음), 여기에 왈가왈부가 많다고 짚는다.
> "그냥 뭐 TPU가 무조건 엔비디아를 다 쓸어버릴 거야, 아니야 엔비디아가 쿠다 때문에 절대로 TPU는 접근할 수 없어, 이렇게 얘기하는 사람들이 많은데 사실은 정확하게 보면 둘 다 맞는 얘기가 있거든요." [01:00]
오늘 다룰 것: 엔비디아가 진짜 위험한지·어떤 면에서 위험한지·그게 거짓인지, TPU가 어디까지 대체 가능한지, 한국 메모리 기업이 얼마나 수혜받을지.

### [01:24–02:04] 왜 지금 갑자기 화제인가
TPU로 전량 학습했다는 얘기는 이번이 처음이 아니라 **제미나이 2.0부터** 있었다. 그런데 지금 놀라는 이유는 두 가지가 섞였다고 본다. (1) 제미나이 3.0 Pro의 벤치마크 점수가 너무 좋고 사용자 만족이 높다는 얘기 + 나노바나나 프로 호평이 겹치며 "구글이 자체 설계 칩으로 좋은 걸 내놨다"는 주목. (2) 원래 자사 학습·AI 서비스·구글 클라우드에만 쓰겠다던 TPU를 **딴 데도 팔기 시작**한 것.

### [02:04–03:06] TPU 전성비·시스템 우위 (원래부터의 팩트)
전성비 관점에서 TPU는 벌써 10년 됐고 1세대 V1 때도 GPU 대비 15~30배 전성비 우위(주장/미검증)를 가졌다. V5·V6로 오며 칩은 블랙웰과 대등해졌고, 구글은 시스템 칩을 9000개 넘게 연결하는 구조에 리퀴드 쿨링을 적용, 인터커넥트도 3D 토러스라 엔비디아의 NVLink·NVSwitch·이더넷·InfiniBand 같은 복잡한 독자 규격과 다르게 간다. 전체 코스트에서 우위를 유지해온 건 기존부터의 팩트다.

### [03:06–05:00] CUDA 해자의 실체 — 커널 엔지니어 상위 1%
"TPU가 GPU를 절대 대체 못 한다"는 말이 왜 나오나 → 이름부터 다르듯 타겟이 다르다. 화자는 CUDA가 AI의 해자라고 늘 말해왔지만 어떤 부분이 해자인지 설명이 어려웠다며, 사카나 AI의 "AI CUDA Engineer" 정리를 인용한다. CUDA를 쓴다고 다 CUDA 엔지니어가 아니고, GPU의 CUDA 코어·트랜스포머 엔진·텐서코어가 각각 어떻게 움직이는지, 메모리를 캐시에서 어떻게 가져와 오프로딩·어텐션하는지 아키텍처 플로우를 정확히 이해하는 개발자만이 새 연산기가 나올 때 직접 커널을 만든다.
> "파이토치에서 일반적인 머신러닝 연산보다 10배에서 100배 빠른 속도를 달성하는 새로운 커널을 디자인한다… 쿠다를 사용하는 사람들 중에서 진짜 상위 1%만이 커널 엔지니어예요." [04:26]
100명 중 1명이 만들고 99명은 그걸 쓴다.

### [05:00–06:00] 커널 = OS 비유
CUDA 커널은 쉽게 말해 OS(윈도우·맥)와 같다. OS가 다른 프로그램과 다른 건 CPU·GPU·스토리지·메모리를 어떻게 운영할지 아는 시스템이라는 점. CUDA도 하드웨어 아키텍처이자 소프트웨어 구조라, 새 연산기가 나오면 그 하드웨어를 어떻게 최적화해 돌릴지 아는 엔지니어들의 몸값이 매우 비싸고 AI 세계를 리딩한다.

### [06:00–07:05] PyTorch = 소프트웨어 스택, 엔비디아 지표 중심
PyTorch는 소프트웨어 스택/플랫폼이라 대부분의 엔지니어는 하드웨어를 신경 안 쓰고 펑션·커맨드만 날린다(더하기·매트릭스 연산 등). 커널을 건드릴 수 있는 소수가 선구적으로 만드는 것이 **엔비디아 GPU 중심**으로 만들어져 온 관습이다. 엔비디아 GPU가 제일 많고 플랫폼이 공통적이며 생태가 이미 갖춰져 있어 빠르게 퍼지기 때문. → CUDA 해자는 여전히 강력하고, TPU가 이걸 뚫을 수 있나가 관건.

### [07:05–09:00] PyTorch vs JAX — 언어 자체가 다르다
GPU를 돌릴 때 메이저 AI 기업 대다수(약 80%, 주장/미검증)인 메타·오픈AI·테슬라 등은 전부 PyTorch 기반. 구글은 JAX(+TensorFlow) 생태라 하드웨어를 굴리는 언어 자체가 다르다("한쪽은 영어, 한쪽은 프랑스어"). x86 CPU와 ARM CPU에서 프로그램이 돌고 안 돌 듯, 하드웨어가 달라지면 베이스 언어도 다를 수밖에 없다. PyTorch는 파이썬처럼 한 줄씩 실행하는 인터프리터식, C/C++ 같은 컴파일러는 전체를 한 번 해석 후 실행. TPU는 **그래프 컴파일러**로 전체 연산 그래프를 그린 뒤 이해하고 나서 TPU에 넣어 돌린다 — 잘 컴파일해두면 빠르지만, 동적으로 변화가 들어와 재컴파일이 필요하면 중단 이슈. 반대로 PyTorch식은 한 줄씩이라 디버깅은 쉽지만 느릴 수 있다. 하드웨어가 다르니 생기는 장단점.

### [09:00–09:47] ARM/윈도우 비유와 두 진영의 수렴
퀄컴 스냅드래곤 X 엘리트로 윈도우 온 ARM을 한 것처럼, 겉은 같은 윈도우여도 ARM 하드웨어로 구동된다. 아이폰·갤럭시의 카카오톡·네이버 앱이 겉은 같아 보여도 다른 하드웨어에서 도는 사실상 다른 프로그램인 것과 같다. x86과 ARM이 닮아가듯 엔비디아 PyTorch 진영도 구글 JAX처럼 그래프 컴파일을 도입하기 시작(PyTorch 2.0부터), 구글 JAX도 부분 컴파일 캐싱으로 유연화 중. 근본적으로 태생적 타겟은 다르다.

### [09:47–11:05] 흑백논리 반박 + 엔비디아 마진과 구글의 위치
구글이 엔비디아를 다 대체하냐 → 이거 아니면 저거로 빠지기 쉬운데 세상은 스펙트럼·다차원이라 단순 판단 불가. 핵심은 다른 차원에 있다: 엔비디아 매출총이익률 70%+·영업이익률 60%(주장/미검증)의 고마진 = "엔비디아 텍스". 빅테크는 성능이 좀 떨어져도 마진 빼고 싸게 가져가고 싶어 하고, 구글은 자기가 TPU를 만들어 학습에 쓰는 TPU만큼은 엔비디아에 돈을 안 줘도 된다. 게다가 구글은 서치·유튜브·Gmail·클라우드까지 가진 초대형 플랫폼이라 수많은 AI 서비스를 돌릴 TPU 수요가 크고, 이번 아이언우드도 추론 중심으로 바뀌었다.

### [11:05–12:04] 자체칩 설계는 바보짓이 아니다
장기적으로 구글은 전성비 좋게 총 코스트를 싸게 만들 수 있으니, CUDA의 유용함을 일부 포기하더라도 그 접근을 할 것. 빅테크들이 자체칩을 설계하는 건 실패할 걸 알면서도 하는 게 아니라, AI 서비스엔 전력 효율성이 중요하고(학습뿐 아니라 사용자들이 다 추론을 쓰며 데이터센터 전기를 소모, 에너지가 부족한 상황) 조금이라도 돈을 아끼면 그게 곧 영업이익이 되기 때문. 구글은 10년간 TPU를 고도화했고 그 결과물인 Gemini 3.0 Pro 성능이 너무 좋아 사람들이 확 와닿는 것. CUDA 해자는 분명 있지만 **너무 비싸서** 다른 이들이 굳이 선택 안 하는 분야가 생길 가능성이 중요.

### [12:04–14:33] 엔비디아 대응 ①: 루빈 CPX (프리필/디코드 분리)
두 달 전 엔비디아가 발표한 **루빈 CPX(Context Prefill Acceleration)**는 HBM을 안 쓰고 GDDR7을 쓴다(대역폭 계산상 약 32GB/s급, 삼성 GDDR7 언급; 하이닉스가 48GB 고속품을 냈으나 계산상 ~30GB대로도 충분히 커버 가능 — 화자 계산). 추론은 (1) 프리필: 입력 문서를 self-attention으로 이해하는 단계, 연산량이 매우 많지만 HBM은 그리 필요 없어 값싸고 빠른 GDDR 사용, (2) 디코드: 실제 답변 생성 단계, 메모리 대역폭이 쭉쭉 나와야 해 HBM 사용. **[의견]** 화자는 루빈 CPX가 나온 이유를 "TPU 때문"이라고 개인적으로 본다. 구글 아이언우드로 추론 시장에서 엔비디아·GPU 없이도 성능이 나오고, AMD도 MI400/MI450X 랙스케일 헬리오스로 추론에 HBM을 많이 박아 대응 중. 추론은 사용자 수가 늘수록 학습보다 칩이 더 많이 팔릴 영역이라, 엔비디아는 호퍼부터 텐서코어·트랜스포머 엔진을 넣은 하이브리드로 트랜스포머 친화적으로 갔지만 그것만으로 안 되니 GPU 안에서 루빈 CPX라는 별도 장치로 추론 시장에 대항한다.

### [14:33–16:17] 엔비디아 대응 ②: DGX Spark (CUDA 생태 강화용)
DGX Spark는 "AI PC"로 불리지만 소비자용 개인 PC가 절대 아니다. Ubuntu OS 기반이며, 일반인이 로컬 LLM을 우분투에서 돌리진 않는다. 이건 AI 엔지니어를 위한 시스템이자 **CUDA 생태 유지를 위해 젠슨이 전략적으로 만든 것**. 데이터센터의 H100·블랙웰 GPU 할당이 워낙 부족해 하고 싶어도 못 하는 상황에서, 서버와 동일한 시스템을 작게 보급하면 여기서 개발해 센터로 그대로 포팅할 수 있다. 원래 "Project Digits"로 "서버를 그대로 작게 만들었다"고 했지 개인용이라 한 적 없다. 그래서 맥미니·맥스튜디오(LPDDR 많이 넣은 PC)보다 **로컬 성능은 오히려 떨어진다** — 로컬 성능이 아니라 개발용이기 때문. 여기서 개발해 서버에 올리면 호환성이 좋아 CUDA 해자를 더 강력하게 한다.

### [16:17–19:20] GPU vs TPU 하드웨어 구조 차이
칩 내부: TPU는 매트릭스 연산 중심으로 간소, GPU는 CUDA 코어 + 텐서코어(매트릭스) + 트랜스포머 엔진 등 별도 연산까지 보유해 변형된 계산이 나오면 커널 엔지니어가 빨리 만들어 뿌린다. 수천 개 칩 연결 철학이 다름: GPU는 수천 개 코어가 각자 메모리에 접근("수천 명이 오토바이 타고 배달") → 데이터 이동 낭비 가능. TPU는 **시스톨릭 어레이**로 데이터가 칩 내부 계산기 사이를 흐르고, **3D 토러스**로 각 칩이 6방향(동서남북·위아래)으로 직접 케이블 연결 → 별도 스위치 장비 없이 확장 내장("심장이 피를 쏘듯 한 방에 내보냄"), 10년 연구 결과물. 다만 멀리 통신하려면 중간 다리를 여러 번 건너 지연 발생 가능. 엔비디아는 하나의 GPU에 HBM, 블랙웰 GPU와 그레이스 CPU가 NVLink·NVSwitch로 연결(랙 내부 예: NVL72 = 72개), Spectrum X·InfiniBand로 랙 간 연결(비싼 네트워크 스위치). 장점은 어떤 GPU든 스위칭으로 거의 동일 시간에 도달, 단점은 스위치 도입으로 비싸게 팔 수 있음. 결론: 구글은 스위치·전력을 아껴 가성비 좋은 공장, 엔비디아는 비싸도 어디든 뚫린 고속도로. 트랜스포머처럼 예측 가능한 구조적 모델엔 TPU가 최적화됨.

### [19:20–21:08] 구글의 무서운 점 — 풀스택 & 애플 비유
응용 스택 + 파운데이션 모델 + 클라우드 + **하드웨어**까지 다 가진 기업은 지금 구글밖에 없다(옛날부터 그랬고 이제 성과가 나오니 주목). 엔비디아도 파운데이션 모델은 있으나 유명하지 않고 클라우드(DGX Cloud)는 약하며 응용 서비스는 B2B라 직접 안 함 — 어디까지나 칩 판매 중심 회사(고객: 기상청·게임사·오픈AI·구글 등)라 고객이 어떤 모델을 쓸지 모른다. 반면 구글은 자기가 뭘 만들지 아니 특화 가능 — **애플**(OS+하드웨어 수직통합)을 연상시킴. 유튜브 코덱 전용 하드웨어, 추천 알고리즘 특화 하드, 제미나이 대소 모델 구분 등 다 알고 만들 수 있고, 클라우드로 TPU를 더 싸게 굴려 유튜브·Gmail·워크스페이스를 싸게 공급. 이번 제미나이 3.0 Pro도 매우 싸게 나옴.

### [21:08–22:43] "가장 빠른 칩"이 핵심이 아니다 + CFO/CTO 의사결정
"누가 가장 빠른 칩을 가졌나"에 관심이 많지만 그게 중요한 게 아니다. 구글은 애초에 속도로 엔비디아를 이기려 하지 않는다(전성비는 매우 좋음에도). 엔비디아는 70% 마진을 떼야 하지만 구글 TPU는 마진 없이 여기서 먹고 들어간다. 기업 의사결정에서 CTO는 "빨리 개발하려면 CUDA 써야 한다"지만 CFO는 "돈 계산하면 싸게 해서 장기적으로 엔비디아에서 벗어나라"고 할 수 있다(예: 마이크로소프트가 "CUDA 그만 쓰고 TPU 사라"). 지력(재무압박)이 커지면 결국 둘 다 하게 된다. 데이터센터는 칩이 아니라 전기 에너지 공급이 병목인 상황이라, 빡센 범용 지표보다 특정 서비스 특화 칩으로 세분화될 것. 자체 설계 능력 있는 구글 TPU가 자기들만 쓴다더니 외부로 팔게 된 건 어마어마한 일 — 화자는 "생각보다 빨리 바뀔 것"이라고 개인적으로 본다.

### [22:43–23:08] AI 버블론과 자체칩 러시의 이유
AI 버블 얘기가 나오는 건 너무 과하게 돈이 들고 그 중심에 엔비디아 지표가 너무 비싸기 때문. 빅테크가 안 하던 칩 설계를 지금 다 하는 이유는 딱 원하는 AI 연산 기능만 남겨 마진을 확보하려는 것. 결국 "구글이냐 엔비디아가 망하냐"의 이분법이 아니라 시장이 성숙하며 각자 특화 영역으로 퍼져나갈 것. 원래 엔비디아가 거의 완전 독점하던 시장을 **TPU가 일부 커버할 수 있게 된 변화 양상**이 중요.

### [23:08–25:00] 프리미엄 시장은 엔비디아 유지 + 대중화 전 단계
"TPU가 이겼다 끝" 또는 "CUDA 해자 절대 못 넘어" 둘 다 단순 해석은 금물. 프리미엄 시장(랩·AI 스타트업·대학 연구·빅테크 선행 R&D)은 속도·플렉서빌리티가 중요하고 인건비 > 하드웨어 비용이라, 비싼 인력이 TPU 호환 맞추는 시간에 그냥 H100 빨리 사서 개발하는 게 나아 엔비디아 수요 유지.
