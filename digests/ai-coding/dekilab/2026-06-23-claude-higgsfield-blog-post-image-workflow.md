## 요약 (주제별)

### 목표 — 키워드 한 줄로 완성 블로그 글
- 키워드 한 줄만 넣으면 경제 블로그·여행 블로그 글을 뽑아내는 방법을 소개한다. 결과물에는 (1) 나의 문체, (2) 이미지, (3) 실제 데이터 기반 위치 정보까지 모두 포함된다.
- 누구나 따라할 수 있게 처음부터 끝까지 보여주며, 사용된 학습 자료(프로젝트 지침·파일)는 영상 설명란에 모두 올려둔다고 안내.

### 도구 선택 — 왜 Claude + 힉스필드(Higgsfield) 조합인가
- 글 작성 도구로 **Claude**를 쓴다. [의견] ChatGPT·Gemini는 요청 내용을 중간에 빠뜨리거나 멋대로 줄이는 일이 많은데, Claude는 시키는 대로 끝까지 빠짐없이 글을 완성해줘서 블로그 글 작성에 적합하다고 본다.
- Claude Code가 아니라 **Claude 채팅**으로 작성하므로 무료로도 사용해볼 수 있다는 장점.
- 아쉬운 점: 이미지 모델을 탑재한 ChatGPT·Gemini와 달리 Claude는 이미지 자체를 만들지 못한다. API로는 가능하지만 꽤 번거롭다.
- 그래서 **힉스필드(Higgsfield) MCP**로 이미지를 생성한다. Higgsfield MCP는 Nano Banana·GPT 이미지 모델뿐 아니라 Veo 3·Kling·Seedance 등 동영상 생성 모델까지 폭넓게 쓸 수 있는 툴. 글은 Claude가, 이미지는 Higgsfield가 맡아 블로그 글을 더 풍성하게 만든다.

### 실전 절차 1 — Higgsfield MCP 연결
- Claude에 로그인 → 좌측 사이드바 열기 → "사용자 지정" 버튼 → **커넥터 탭** 이동.
- 플러스 버튼 → "커스텀 커넥터 추가" → 이름에 "힉스필드" 입력 → 하단 **원격 MCP 서버 URL**에 Higgsfield 사이트의 MCP URL을 그대로 복사·붙여넣기(URL 링크는 영상 설명란에 기재).
- 추가 → 연결 클릭 → 커넥터 연결 확인 → 도구 권한을 **모두 "항상 허용"**으로 변경. 이러면 사용 준비 완료. 한 번만 연결하면 된다.

### 실전 절차 2 — 문체 지정 + 워크플로 요청
- 그냥 "경주 1박 여행 코스 소개글 작성해줘"라고만 해도 Claude는 실제 데이터 기반으로 꽤 잘 쓴다. 다만 조금 밋밋하고 요청할 때마다 문체가 매번 달라진다.
- 이를 막으려고 **원하는 문체의 예시 글(텍스트 파일)**을 미리 준비해 첨부한다(영상 예시: 직접 다녀온 척이 아니라 담백하게 정보를 전달하는 말투).
- 첨부와 함께 요청: "이 문체로 여행지 블로그 글을 쓰고 싶다 → 여행지를 정하면 리서치해 관광지 입장료·운영시간·맛집 등 최신 정보를 확인하고 글 작성 → 장소마다 구글맵 링크 첨부 → 힉스필드 MCP로 어울리는 이미지까지 생성. 추가로 제한할 게 있으면 제안해줘."
- Claude는 내가 제안한 워크플로 외에 이미지 스타일·비율·지도 처리·이미지 저작권 문제까지 되묻고, 어떤 여행지로 쓸지 물어본다.

### 실전 절차 3 — 1차 생성물 확인·검증
- 답변으로 이미지 무드(사람이 직접 찍은 듯한 포토리얼리스틱 + 핸드메이드 느낌)와 글 분량(첨부 글 분량만큼)을 지정해 위임.
- Claude가 알아서 여행지를 선정 → 구글맵 포함 정확한 장소 정보 리서치 → 글 작성 → 이미지 모델로 16:9 비율 이미지 4장 생성 → 첨부 문체에 맞춰 본문 작성.
- 검증: 구글맵 링크를 클릭하니 실제 장소 링크가 정확히 들어가 있었고, 입장 시간·휴관일 정보도 실제와 일치.
- (지명 관련: 링크로 확인한 장소명은 [불명확: 원문 "겟로크엔"].)

### 실전 절차 4 — 티키타카 수정
- 생성 글을 읽고 마음에 안 드는 부분을 수정 요청: 글 양 2배로 늘리기, 이미지 생성 모델 교체, 이미지 가로세로 비율 변경 등.
- 영상에서는 이미지 모델을 **Nano Banana 2**, 비율을 **4:3**으로 지정하고, 결과물을 텍스트 파일 + HTML 파일로 반환 요청. 네이버 블로그 화면 캡처를 첨부해 "HTML은 이 이미지처럼 미리보기용으로 만들어줘"라고 요청.
- 원하는 결과가 나올 때까지 티키타카로 반복 수정.

### 도구·자동화 — 이미지 생성 승인 게이트
- 기본은 허가 없이 Claude에 위임해 이미지가 자동 생성된다.
- 생성 전 승인을 받고 싶다면: "이미지 생성 전에 **생성 프롬프트와 그 한글 번역**을 사용자에게 보여주고 허가받은 다음 생성해줘"라고 요청 → 이후 Claude는 이미지 생성 전 사용자 허가를 받고 Higgsfield MCP로 생성.
- 결과로 붙여넣을 수 있는 텍스트 파일 + 미리보기용 HTML 파일을 이미지 4장과 함께 생성해준다.

### 실전 절차 5 — 프로젝트화(재사용)
- 마음에 드는 글이 나오면 "지금까지 한 내용을 프로젝트로 만들 거야. 지침으로 만들어줘"라고 요청 → Claude가 작업 순서·문체 규칙·이미지 규칙 등 지침을 정리해준다.
- 이 지침을 복사 → Claude **프로젝트 탭**에서 새 프로젝트("블로그 프로젝트") 생성 → 지침 저장 + 방금 생성한 블로그 텍스트 파일 첨부 + 맨 처음의 문체 예시 파일 첨부.
- 이후에는 매번 과정을 반복하지 않고 프로젝트를 켜서 **키워드 하나만으로** 원하는 블로그 글을 생성 가능.

### 실전 절차 6 — 최종 업로드(수동)
- 생성된 글을 다운로드/복사해 블로그에 붙여넣고, 생성 이미지는 다운로드 버튼으로 받아 첨부해 완성.
- 블로그 업로드 자동화는 API가 있으면 가능하지만 꽤 까다로운 것으로 알려져 있어, 텍스트로 옮기는 과정에서 어색한 부분·데이터 누락이 없는지 **휴먼 터치(human touch)**로 점검하며 작성하는 편이 빠르고 정확.

### 리스크·품질에 대한 [의견]
- "AI로 쓴 글은 저품질에 걸리거나 AdSense 승인이 어렵다/알고리즘을 타기 어렵다"는 말이 많다.
- [의견] 화자 생각: AI를 썼다는 이유로 우선순위에서 밀리는 게 아니라, 결과적으로 **업로드된 글의 퀄리티가 나쁠 때** 경쟁에서 밀린다. 오늘 방식대로 나만의 문체·글의 양·SEO를 고려해 사용자가 읽고 싶은 글을 만들면 충분히 "팔리는 글"을 손쉽게 만들 수 있다.

### 확장 — 다른 주제(경제 블로그)
- 여행뿐 아니라 실제 데이터 기반 경제 블로그도 가능. 예: 어젯밤 미국 주식 시황을 분석하는 블로그 글 워크플로.
- Claude와 함께 독자층·들어갈 내용·글 분량을 골라 나만의 블로그 글 구조를 만들고, 리서치 기능으로 최근 데이터를 반영해 시의적절한 글 작성. 이 경제 블로그 프로젝트 파일도 학습 자료에 포함한다고 안내.

### 마무리
- 글 작성은 Claude가 가장 뛰어나고 이미지·영상 생성은 힉스필드가 꽉 잡고 있어 꽤 강력한 조합이라는 결론. Higgsfield MCP를 써보고 싶으면 영상 설명란 링크 참고, 질문은 댓글로.

## 3줄 오리엔테이션
- "키워드 한 줄 → 내 문체의 블로그 글 + 실제 데이터 위치정보 + 이미지"까지 한 번에 뽑는 워크플로를 처음부터 끝까지 시연하는 영상.
- 핵심 조합은 **글=Claude(채팅) / 이미지=힉스필드(Higgsfield) MCP**. Claude에 커스텀 커넥터로 Higgsfield MCP를 붙이고, 문체 예시 파일을 첨부해 여행/경제 블로그 글을 생성한다.
- 마지막엔 완성한 흐름을 Claude **프로젝트 지침**으로 만들어, 이후 키워드 하나로 재생산 가능하게 만드는 게 목표.

## 전문 — 시간순 워크스루

### [00:00–00:27] 오프닝 — 무엇을 만드는가
키워드 한 줄만 넣으면 경제 블로그·여행 블로그 글을 뽑아낼 수 있는 방법을 소개한다. 이 블로그 글에는 나의 문체가 담기는 것은 물론, 사진과 실제 데이터를 기반으로 한 위치 정보까지 모두 포함해 작성한다. 누구나 따라할 수 있게 처음부터 끝까지 보여줄 것이며, 영상에서 사용한 학습 자료는 설명란에서 모두 확인할 수 있다고 안내한다(구독·좋아요 요청).

### [00:27–00:57] 왜 글은 Claude인가
블로그 글 작성에는 Claude를 사용한다. [의견] AI로 글을 써본 사람은 알겠지만 ChatGPT나 Gemini에 비해 Claude가 글을 뽑아내는 능력이 특출나게 좋다는 것이 이유다. [의견] ChatGPT·Gemini는 요청한 내용을 중간에 빠뜨리거나 멋대로 줄여버리는 일이 많은데, Claude는 시키는 대로 끝까지 빠짐없이 글을 완성해줘서 블로그 글 작성에 적합하다고 본다.

### [00:57–01:22] Claude 채팅의 장점과 한 가지 한계
Claude Code가 아니라 **Claude 채팅**으로 작성하기 때문에 무료로도 사용해볼 수 있다는 장점이 있다. 다만 한 가지 아쉬운 점이 있는데, 이미지 모델을 탑재한 ChatGPT·Gemini와 달리 Claude는 이미지 자체를 만들지 못한다. API를 쓰면 가능은 하지만 솔직히 꽤 번거롭다.

### [01:22–01:49] 해결책 — 힉스필드(Higgsfield) MCP
그래서 Claude의 **힉스필드(Higgsfield) MCP**를 활용해 블로그 글에 맞는 이미지를 생성한다. Higgsfield MCP는 Nano Banana·GPT 이미지 모델뿐 아니라 Veo 3·Kling·Seedance 등 동영상을 생성할 수 있는 모델들까지 폭넓게 활용할 수 있는 툴이다. 이렇게 하면 글은 Claude가 쓰고 거기 들어갈 이미지는 Higgsfield가 만들어줘서 블로그 글을 훨씬 더 풍성하게 만들 수 있다.

### [01:49–02:24] 커넥터 연결 방법(1) — 커스텀 커넥터 추가
Claude에서 Higgsfield MCP를 연결하는 방법은 간단하며 한 번만 하면 된다. Claude에 로그인한 뒤 좌측 사이드바를 열고 "사용자 지정" 버튼을 클릭해 **커넥터 탭**으로 간다. 플러스 버튼을 누르면 "커스텀 커넥터 추가" 버튼이 있는데, 이를 클릭해 이름에 "힉스필드"를 입력하고, 하단의 **원격 MCP 서버 URL**에 Higgsfield 사이트의 MCP URL을 그대로 복사해 붙여넣는다(이 URL 링크는 영상 설명란에 적어둔다).

### [02:24–02:46] 커넥터 연결 방법(2) — 권한 허용
추가 버튼을 누르고 연결 버튼을 클릭하면 내 Claude와 Higgsfield가 바로 연결된다. 커넥터가 연결된 것을 확인하고, 도구 권한에서 권한들을 모두 허용한다 — 모두 "항상 허용"으로 바꿔준다. 이러면 Claude에서 Higgsfield MCP를 사용할 준비가 모두 완료된다.

### [02:46–03:01] 기본 요청만으로도 되는가
준비가 끝났으니 본격적으로 블로그 글을 작성한다. Claude에게 그냥 블로그 글을 작성해달라고 요청만 해도 꽤 잘 작성해주는 것을 확인할 수 있다.

### [03:01–03:36] 기본 생성물의 한계 — 밋밋함과 문체 불일치
예를 들어 "경주 1박 여행 코스 소개글을 작성해줘"라고 하면, Claude가 마치 본인이 직접 다녀온 것처럼 1박 여행 코스를 설명하는 글을 써준다. 표현도 괜찮고 실제로 있는 데이터를 바탕으로 작성했기 때문에 나쁘지 않지만, 조금 밋밋하고 글을 작성할 때마다 매번 다른 문체로 결과물이 나온다. 따라서 매번 다른 문체로 완성되는 것을 막고, 내가 쓰고 싶은 블로그 글의 문체 그대로 Claude가 작성하도록 만든다.

### [03:36–03:56] 문체 예시 파일 준비
우선 여행지를 소개하는 블로그 글 양식을 하나 준비했다. 해당 여행지를 직접 다녀온 것처럼 쓰는 게 아니라, 담백한 말투로 정보를 전달하는 글의 문체 형식으로 텍스트 파일을 만들었다. 이 텍스트 파일을 Claude에 첨부하고 요청한다.

### [03:56–04:27] 워크플로 요청 프롬프트
요청 내용:
> "여행지 소개 블로그 글을 작성하고 싶어. 지금 첨부한 글이 내가 원하는 문체야. 여행지를 정하면 해당 여행지를 리서치해서 관광지의 입장료·운영 시간·맛집 같은 최신 정보를 확인하고 글을 작성해줘. 장소마다 구글맵 링크를 붙여주고, 힉스필드 MCP를 활용해 어울리는 이미지까지 만드는 흐름으로 만들고 싶어. 추가로 제한할 게 있으면 더 말해줘." [03:56]

### [04:27–04:55] Claude의 역질문 — 스타일·비율·저작권·여행지
잠시 기다리면, 추가 제안을 요청했기 때문에 Claude가 내가 제안한 워크플로뿐 아니라 이미지 스타일을 어떻게 할지, 이미지 비율은 어떻게 할지 되묻는다. 그 외에 지도 처리나 이미지 저작권 같은 문제도 추가로 제안한다. 그다음 어떤 여행지로 글을 작성할지 물어보는데, 화자는 Claude에게 맡기기로 한다.

### [04:55–05:23] 무드·분량 지정 후 자동 진행
이미지 무드는 사람이 직접 찍은 듯한 포토리얼리스틱한 느낌과 핸드메이드(핸들메이드) 느낌이 나는 이미지로 만들어달라고 했고, 글 분량은 첨부한 글의 분량만큼 만들어달라고 요청했다. 그러자 Claude가 알아서 여행지를 선정하고 최신 정보를 리서치하기 시작한다. 구글맵을 포함한 정확한 장소 정보를 확인하고, 글을 작성한 다음, 이미지 모델을 활용해 16:9 비율로 이미지 4개를 생성해준다고 한다.

### [05:23–05:53] 생성물 검증 — 구글맵·운영정보 일치
Claude가 Higgsfield MCP를 활용해 블로그 글에 필요한 이미지 4가지를 뽑아주었고, 이어서 블로그 글까지 첨부한 문체에 맞춰 작성해주었다. 구글맵 링크까지 자연스럽게 넣어주었는데, 링크를 클릭해 확인해보니 실제 장소 링크(장소명 [불명확: 원문 "겟로크엔"])가 정확히 들어가 있었다. 또 입장 시간과 쉬는 날(휴관일) 같은 정보도 실제 정보와 일치하는 것을 확인할 수 있었다.

### [05:53–06:14] 마음에 맞게 수정하기(개요)
이대로도 나쁘지 않지만 입맛에 맞춰 더 수정한다. 작성된 글을 쭉 읽어보고 마음에 들지 않는 부분을 고치면 된다. 예를 들어 글의 양을 두 배로 늘려달라고 요청하거나, 생성된 이미지의 모델을 바꾸거나, 이미지 가로세로 비율을 바꾸는 식이다.

### [06:14–06:34] 구체적 수정 요청 — 모델·비율·출력형식·미리보기
화자의 수정 사항: 이미지 생성 모델은 **Nano Banana 2**, 가로세로 비율은 **4:3**으로 선택하고, 결과물은 텍스트 파일과 HTML 파일로 반환해달라고 요청한다. 그리고 네이버 블로그 같은 이미지를 하나 캡처해 첨부하며, HTML은 미리보기용으로 이 첨부 이미지처럼 만들어달라고 요청했다.

### [06:34–07:08] 이미지 생성 승인 게이트 넣기
그러면 Claude가 Higgsfield MCP로 Nano Banana 2 모델로 이미지를 생성한다. 현재는 따로 허가를 받지 않고 Claude에 모두 위임해 이미지를 생성하고 있는데, 만약 생성 전에 사용자의 승인을 받게 하고 싶다면 이렇게 요청한다: 이미지 생성 전에 생성할 프롬프트와 그 한글 번역을 사용자에게 보여주고, 허가받은 다음 이미지를 생성해달라. 그러면 Claude는 이미지 생성 전에 사용자의 허가를 받고 Higgsfield MCP로 이미지를 만든다.

### [07:08–07:31] 출력물 — 텍스트 + HTML 미리보기
잠시 기다리면 Higgsfield MCP로 이미지를 총 4장 생성해주고, 바로 붙여넣을 수 있는 텍스트 파일과, 블로그에 글을 쓰기 전 미리보기용으로 HTML 파일을 생성해준 것을 확인할 수 있다. 이런 방식으로 원하는 결과물이 나올 때까지 티키타카를 하며 글을 더 수정한다.

### [07:31–08:16] 프로젝트 지침으로 만들기
마음에 드는 글이 나왔다면 Claude에게 "지금까지 한 내용을 프로젝트로 만들 거야. 지침으로 만들어줘"라고 요청한다. 그러면 Claude가 작업 흐름(작업 순서·문체 규칙·이미지 규칙 등)을 쭉 정리해 지침으로 만들어준다. 이 지침을 그대로 복사해 좌측의 Claude 프로젝트 탭으로 넘어가 새 프로젝트를 만들고, "블로그 프로젝트"라는 이름으로 프로젝트를 만든 다음, 복사한 프로젝트 지침을 저장하고, 방금 생성한 블로그 텍스트 파일을 파일에 추가하고, 맨 처음 첨부했던 나의 문체가 잘 드러난 파일까지 첨부한다. 이러면 매번 Claude와 이 과정을 거치지 않고, 해당 프로젝트를 켜서 키워드 하나만으로 원하는 블로그 글을 생성할 수 있게 된다.

### [08:16–08:47] 최종 업로드는 수동(휴먼 터치)
바로 다운로드해 생성된 글을 그대로 복사·붙여넣고, 생성된 이미지는 다운로드 버튼을 눌러 받아 그대로 첨부해 블로그 글을 완성한다. 블로그 업로드도 API가 있으면 자동화할 수 있겠지만 꽤 까다로운 걸로 알려져 있어서, 텍스트로 옮기는 과정에서 어색한 부분은 없는지·데이터는 잘 들어갔는지 **휴먼 터치**를 하며 작성함으로써 빠르고 정확하게 글을 완성할 수 있다. 오늘 사용한 블로그 프로젝트의 지침과 파일은 모두 학습 자료에 넣어둔다고 안내.

### [08:47–09:28] AI 글의 품질/노출에 대한 [의견]
가끔 "AI를 활용한 글은 저품질에 걸리거나 AdSense 승인을 받기 어렵다, 알고리즘을 타기 어렵다"는 말을 많이 한다. [의견] 화자 개인 생각으로는, 단순히 AI를 썼다는 이유로 블로그 글의 우선순위에서 밀려나는 것이 아니라, 결과적으로 업로드된 글의 퀄리티가 나쁠 때 블로그 글 경쟁에서 밀린다고 본다. 따라서 오늘 알려준 방식대로 나만의 문체와 글의 양, SEO 등을 고려해 사용자가 읽고 싶은 글을 만들면 충분히 팔리는 글을 손쉽게 만들 수 있을 거라 생각한다.

### [09:28–10:12] 확장 — 경제 블로그 워크플로
이런 여행 주제뿐 아니라 다른 주제로도 블로그 글을 작성할 수 있다. 예를 들어 실제 데이터를 바탕으로 경제 블로그 글을 작성할 수도 있는데, 어젯밤 미국 주식 시황을 분석하는 블로그 글을 만드는 워크플로를 만들어볼 수 있다. Claude와 함께 블로그 독자층도 고르고, 들어가야 할 내용도 함께 고른 다음, 글의 분량까지 정하면서 나만의 블로그 글 구조를 만든다. 이렇게 리서치 기능을 활용해 Claude로 글을 작성하면 최근 데이터를 반영해 훨씬 더 시의적절한 글을 완성할 수 있다. 이 경제 블로그를 만든 프로젝트 파일도 학습 자료에 넣어둔다고 안내.

### [10:12–10:43] 마무리
오늘은 Claude와 힉스필드(Higgsfield) MCP를 활용해 각자의 장점을 온전히 살려 나만의 블로그 글을 생성하는 방법을 알아보았다. [의견] 글 작성은 Claude가 가장 뛰어나고 이미지·영상 생성은 Higgsfield가 꽉 잡고 있어 꽤 강력한 조합인 것 같다는 결론. Higgsfield MCP를 활용해보고 싶다면 영상 설명란의 링크를 참고하고, 궁금한 점은 댓글로 남겨달라고 안내하며 마친다.
