# 코딩 몰라도 OK | 클로드로 유튜브 자동 요약 워크플로우 만들기

> dekilab · 2026-06-17 · 2,379회 · 17분38초 · [영상 링크](https://youtu.be/MJiQFNp-k10)
> 정리 방식: 시간순 심층 워크스루 (paraphrase) + 주제별 상세 요약

## 요약 (주제별)

### 1. 워크플로우 전체 목표
- 구독 채널이 10~20개만 돼도 매일 올라오는 영상을 다 챙겨보기 어렵다는 문제에서 출발. 안 보면 놓칠까 불안하고, 다 보자니 영상이 너무 길다.
- 목표: 내가 트래킹하는 유튜브 채널에서 **전날(어제) 올라온 영상을 자동 감지 → 핵심 요약 → 한 곳(노션 페이지)에 날짜별로 정리**. 요약만 보고 파악한 뒤, 관심 영상만 링크 클릭해 필요한 파트만 골라보게 만들어 시간 절약.
- 사용자는 "유튜브 채널 주소만 넣어 두면" 나머지는 클로드가 알아서 처리하도록 만드는 것이 지향점.

### 2. 사용 도구
- **Claude Code**: 워크플로우 제작 주체. 코딩을 몰라도 자연어로 요청해 결과물을 받아낸다는 것이 핵심 컨셉. VS Code 위에서 실행.
- **yt-dlp**: 유튜브에서 데이터를 받아오는 무료 오픈소스 도구. 유튜브 공식 API는 하루 사용량 한도가 있고 초과 시 과금되지만, yt-dlp는 오픈소스라 무료로 활용 가능(화자 설명). 내부 동작을 몰라도 클로드가 대신 다뤄 준다.
- **Claude Desktop**: 노션 커넥터 연결과 스케줄링(루틴) 등록에 사용.

### 3. 프로젝트 구성 요소
- **channels.txt**: 트래킹할 유튜브 채널들의 URL을 줄바꿈으로 나열. 채널 URL은 해당 채널 상단의 유튜브 핸들을 입력.
- **format.md**: 요약 출력 형식을 미리 정의(영상 제목·조회수·핵심 요약·타임라인·한 줄 인사이트 등). 정의해 두면 매번 같은 구조로 결과가 나옴. 취향껏 수정 가능.
- **collect 스크립트**: channels.txt를 읽어 각 채널의 최근 영상을 yt-dlp로 수집(제목·조회수·영상 날짜·자막 등), 결과를 JSON으로 저장.
- **output 폴더**: 요약된 마크다운 결과물을 모아두는 곳(날짜 형태 파일명).
- **CLAUDE.md**: 인수인계서. 세션이 끝나도 프로젝트가 별도 설명 없이 동작하도록 흐름(collect 실행 → 결과 읽기 → format 형식으로 요약 → output 저장)을 기록.

### 4. 실전 제작 절차
1. 새 폴더 생성 후 Claude Code 실행, yt-dlp로 영상 1개의 데이터(제목·조회수·업로드 날짜·URL·자막)를 가져와 동작 확인.
2. 클로드에게 요청해 channels.txt와 output 폴더 생성 → channels.txt에 트래킹 채널 URL 입력.
3. format.md로 요약 형식 정의.
4. collect 스크립트 생성 요청(channels.txt 읽기 → yt-dlp로 영상별 메타·자막 수집 → JSON 저장), JSON 출력 확인.
5. collect로 수집한 데이터를 format에 맞춰 output 폴더에 오늘 날짜 마크다운으로 저장 요청 → 채널당 최근 2개 영상 요약 확인.
6. collect 스크립트를 "어제 업로드된 영상만" 가져오도록 수정.
7. CLAUDE.md 생성으로 전체 흐름 고정.
8. (선택) 노션 저장으로 전환 후 CLAUDE.md 업데이트.
9. JSON·테스트 파일 정리, 스케줄링 등록.

### 5. 신규 영상 감지 로직 (어제 업로드분만)
- **처음에는 "최근 영상"으로 테스트**: 등록한 채널들이 어제 영상을 안 올렸을 수도 있어, 어제분부터 바로 시작하면 결과물이 안 나와 동작 확인이 안 됨. 그래서 먼저 최근 영상으로 요청해 흐름을 검증.
- 흐름 검증 후 collect 스크립트를 **"어제 날짜에 업로드된 영상만 가져오도록"** 수정. **어제 업로드된 영상이 없는 채널은 스킵**. 결과물은 output 폴더에 날짜 형태로 저장.
- CLAUDE.md 덕분에 별도 지시 없이 그냥 실행하거나 "어제 거 요약해 줘"라고만 해도 이 흐름대로 동작.

### 6. 저장 대상 — 로컬 마크다운 vs 노션
- 기본은 output 폴더에 로컬 마크다운으로 저장.
- 노션 저장을 원하면 Claude Web/Desktop에서 노션 **커넥터 연결**(설정 → 커넥터 탭 → 커넥터 둘러보기 → 노션 검색·연결). 노션 연결·스케줄링 상세는 이전 영상에서 다룸.
- 연결 후 클로드에 "작업 결과를 노션에 저장, 이 페이지에 날짜별 페이지 생성, 각 영상 요약을 그곳에 저장"이라 요청하고 **노션 페이지 URL 끝의 페이지 ID**를 복사해 붙여넣음 → 날짜별 페이지가 format 형태로 정리됨.
- 이때 CLAUDE.md도 자동 일부 수정됨. 로컬 마크다운 + 노션 동시 저장 상태에서, 노션 연동 방식으로 바뀌었으니 CLAUDE.md를 업데이트하라고 다시 요청.

### 7. 스케줄링·자동 실행 (특정 시각 자동 감지→요약)
- 매일 직접 실행하지 않도록 **클로드(Claude Desktop) 스케줄링(루틴) 기능**으로 자동화. 화자 본인도 실제로 이 워크플로우를 스케줄링으로 돌리는 중.
- 등록: 좌측 **루틴 버튼** → **로컬 루틴** 선택 → 이름·설명("전날 추가된 유튜브 영상 정리 루틴") 입력 → 하단 **폴더 선택**으로 프로젝트 폴더 지정 → 지침에 "어제 업로드된 영상을 수집하고 노션에 요약 저장해 줘" 입력 → **원하는 시간 지정** → 만들기.
- **로컬 루틴은 컴퓨터가 켜져 있는 상태에서만 작동**(원격 루틴과 대비). 지정 시간에 컴퓨터가 꺼져 있으면 그 시간대 작업은 건너뛰고 **다음 시간대에 동작**.
- 저장하면 매일 아침 지정 시각에 유튜브 영상이 수집·요약되어 노션 페이지에 저장됨. 스케줄링 상세 과정은 이전 영상 참고.

### 8. 철학·팁 [의견]
- [의견] 처음부터 완벽하게 만들려 하지 말고, 써 보면서 "어떻게 하면 더 편리할까" 고민하며 하나씩 고쳐 나가는 습관이 가장 중요. 반복할수록 워크플로우가 점점 내 입맛에 맞게 다듬어짐(로컬 마크다운 → 노션 전환이 그 예).
- [의견] 자동화가 목표라면 매번 클로드에게 수집 방법을 고민시키는 것보다 수집 스크립트를 따로 만들어 두는 편이 훨씬 효율적. 코드를 만들어 실행시키는 편이 토큰 사용량 등 여러모로 경제적.
- 채널 목록·요약 형식·저장 위치 등은 전부 시청자가 원하는 대로 바꿀 수 있는 예시일 뿐.

### 9. 자료 공유 구조
- 영상에서 사용된 프롬프트·프로젝트 파일은 오픈 채팅방에서 공유 중(필요 시 받아갈 수 있음).

## 3줄 오리엔테이션
- Claude Code로 "구독 채널의 어제 올라온 영상만 자동 수집 → 핵심 요약 → 노션에 날짜별 정리"하는 워크플로우를 코딩 없이 자연어 요청만으로 만드는 튜토리얼.
- 핵심 부품은 channels.txt(추적 채널) · format.md(요약 형식) · collect 스크립트(yt-dlp 수집) · output 폴더 · CLAUDE.md(인수인계서)이고, 저장처는 로컬 마크다운 또는 노션.
- 마지막에 Claude Desktop **로컬 루틴(스케줄링)**으로 지정 시각에 "어제 업로드분 자동 감지→요약→노션 저장"까지 무인 실행되게 한다.

## 전문 — 시간순 워크스루

### [00:00–00:33] 훅: 노션에 쌓인 자동 요약 보고서
매일 아침 챙겨보던 유튜브 영상들을 핵심만 뽑아 출근 전에 읽기 좋게 정리해 두면 좋겠다는 상상으로 시작. 화면의 노션 페이지는 클로드가 알아서 구독 채널에 들어가 어제 올라온 영상을 긁어와 분석·요약해 보고서로 만들고, 노션에 날짜별로 차곡차곡 정리해 준 결과물이다.
> "저는 유튜브 채널 주소만 넣어 놨을 뿐인데 말이죠." [00:27]
오늘은 이 방법을 하나부터 열까지 만든다고 예고하며 구독·좋아요를 요청.

### [00:37–01:20] 문제 정의: 다 보기엔 길고 안 보자니 불안
구독 채널이 10~20개만 돼도 매일 올라오는 영상을 다 챙겨보기 어렵다. 안 보자니 놓칠까 불안하고, 다 보자니 너무 길다. 요즘 유튜브도 제미나이로 내용을 분석해 주긴 하지만, 그보다 더 편리하게 일일이 영상을 고르지 않아도 **전날 올라온 영상을 자동 감지해 핵심을 요약하고 한 곳에 모아 보게** 만들겠다는 것. 요약만 보고 파악한 뒤 관심 있으면 링크를 클릭해 필요한 파트만 골라 보거나, 요약만 보고 넘어가며 시간을 아낄 수 있다.

### [01:20–01:41] 도구: Claude Code
이 워크플로우 제작에는 Claude Code를 활용. 많은 이가 "Claude Code"라는 단어나 VS Code 같은 IDE를 꺼리지만 생각보다 어렵지 않으며, 아직 설치를 안 했다면 우측 상단 클로드 코드 입문 영상을 보고 오라고 안내.

### [01:41–02:10] 프로젝트 전체 흐름 개요
전체 흐름: (1) 내가 구독하는 채널에서 전날 올라온 영상 정보를 자동으로 가져오는 **영상 수집**을 제작, (2) 스크립트로 가져온 데이터를 클로드가 읽고 요약해 내 컴퓨터 혹은 노션에 저장, (3) 세션이 종료돼도 잘 작동하도록 **클로드 파일(CLAUDE.md)을 저장**하고 **클로드의 스케줄링 기능으로 자동화**까지.

### [02:10–03:00] 자연어 요청 컨셉 · 커스터마이즈
이 과정에서 유튜브 데이터를 수집하는 오픈소스 yt-dlp 같은 생소한 내용이 나오지만 그게 뭔지 몰라도 상관없다. 처음부터 끝까지 만들고자 하는 걸 **자연어로 요청**해 원하는 동작의 결과물을 받아내기 때문. 따라 만들다 보면 코딩을 몰라도 워크플로우가 컴퓨터에 완성돼 있을 것.
> "코딩을 전혀 모르셔도 관심 있는 유튜브 영상이 자동으로 수집되는 이 워크플로우가 여러분 컴퓨터에 완성되어 있을 거니까" [02:33]
오늘 과정은 하나의 예시이며, 채널 목록·요약 형식·저장 위치는 전부 원하는 대로 바꿀 수 있으니 개인 입맛에 맞게 만들라고 강조.

### [03:00–03:37] yt-dlp란 무엇인가
yt-dlp는 유튜브에서 데이터를 받아올 수 있는 무료 오픈소스 도구. 유튜브 공식 API는 하루 사용량 한도가 있고 초과 시 과금되지만(화자 설명), yt-dlp는 오픈소스라 무료로 활용 가능. 겉보기엔 복잡해 보여도 내부를 하나하나 알 필요 없고, Claude Code와 함께라면 쉽게 쓸 수 있다.

### [03:37–04:09] 실습 준비: VS Code + 새 폴더
Claude Code 활용을 위해 VS Code를 사용. 새 프로젝트용 폴더를 만들고 Claude Code를 켠 뒤, yt-dlp로 어떤 데이터가 수집되는지 직접 확인해 본다.

### [04:09–05:05] yt-dlp로 영상 1개 데이터 뽑기
클로드에게 "yt-dlp를 활용해서 이 영상 하나의 데이터를 가져와서 보여 줘. 제목과 조회수, 업로드 날짜, 영상 URL 등 자막들을 포함해서"라고 요청하고, 예시로 클로드 스케줄링 영상을 첨부. 몇 가지 권한을 승인하고 기다리면 제목·조회수·재생 시간·**자막 전체가 텍스트로** 수집된다. 이 자막 데이터가 있어 영상을 직접 안 봐도 클로드로 전체 내용을 요약·파악 가능. 즉 이 **수집 과정 + 클로드 요약 과정**을 붙이면 자동화 워크플로우가 된다.

### [05:05–05:39] 프로젝트 구조 시작: channels.txt + output 폴더
클로드에게 channels.txt 파일과 output 폴더를 만들어 달라고 요청. channels.txt는 트래킹하고 싶은 채널을 적는 곳, output 폴더는 수집·요약한 데이터를 모아두는 곳. (직접 만들어도 되지만) 클로드에게 요청해도 두 가지가 잘 생성됨.

### [05:39–05:56] channels.txt에 채널 URL 입력
channels.txt를 열어 트래킹할 채널 URL들을 줄바꿈하며 입력. 채널 URL은 해당 유튜브 채널에 들어갔을 때 상단의 **유튜브 핸들**을 입력하면 된다.

### [05:56–06:38] format.md로 요약 형식 정의
그냥 요약해 달라고 하면 매번 클로드 마음대로 요약하므로, 영상의 제목·조회수·핵심 요약·타임라인·한 줄 인사이트 같은 원하는 형식을 파일로 정의해 두면 매번 **같은 구조**로 나온다. 프로젝트 폴더 안에 format.md 파일을 만들어 영상 제목·핵심 요약·파트별 내용으로 구성(형식도 취향껏 수정 가능).

### [06:38–07:47] 왜 "수집 스크립트"를 따로 만드나
채널 목록과 포맷을 정했으니 영상 수집을 담당할 **collect 스크립트**를 만든다. 그냥 "channels.txt의 채널 영상들을 요약해 줘"라고만 해도 클로드가 그때그때 처리해 주긴 하지만, **자동화가 목표라면** 매번 수집 방법을 고민시키는 게 아니라 수집 방법(=스크립트)을 따로 만들어 두는 편이 훨씬 효율적. 클로드가 매번 고민·작업하는 것보다, 유튜브 영상을 수집하는 코드를 만들어 두고 그 코드를 실행해 정보를 가져오는 편이 **토큰 사용량을 비롯해 경제적**이기 때문. '스크립트'란 쉽게 말해 "이 작업을 수행할 때는 이 파일에 나온 대로 실행해라"라는 식으로 만들어 두는 파일이고, 한 번 만들면 매번 클로드가 그 스크립트를 활용해 작업을 수행한다.

### [07:47–08:41] collect 스크립트 생성 요청 + JSON 확인
클로드에게 "channels.txt에 있는 채널 목록을 읽어서 각 채널의 최근 영상을 수집하는 collect 스크립트를 만들어 줘. yt-dlp를 활용하고 각 영상마다 제목·조회수·영상 날짜·자막 등을 가져오고, 수집 결과는 JSON 파일로 저장해 줘"라고 요청. collect 스크립트가 완성되고, 채널 목록의 채널에서 JSON 형태로 영상 정보들이 수집됨. 실제 클릭해 보면 해당 영상 기반으로 데이터가 추출된 것을 확인.

### [08:41–09:38] JSON → format 마크다운 저장 (먼저 "최근 영상"으로 테스트)
이번엔 JSON을 그대로 두지 않고, 받아온 JSON 데이터를 앞서 만든 format 형식에 맞춰 output 폴더에 마크다운으로 저장. 요청: "collect 스크립트로 채널들의 최근 영상 데이터를 추출하고, 수집이 끝나면 format 형식으로 요약해 output 폴더에 오늘 날짜 파일명으로 마크다운을 만들어 줘." 원 목표는 '어제 업로드분'이지만, 등록한 채널들이 **어제 영상을 안 올렸을 수도 있어** 어제분부터 바로 시작하면 결과물이 안 나올 수 있고, 결과물이 없으면 정상 작동 확인이 안 된다. 그래서 일단 **최근 영상**으로 요청.

### [09:38–10:15] 마크다운 결과 확인
기다리면 작성해 둔 마크다운 포맷을 확인해 자막 기반으로 결과물을 수집하고, **채널당 최근 2개 영상**을 요약한 마크다운이 나온다. 이제 이 파일만 봐도 영상 핵심이 파악되고 관심 영상은 링크 클릭으로 바로 확인 가능.

### [10:15–11:00] "어제 업로드분만" 으로 수정
전체 흐름이 확인됐으니 원 목표대로 **어제 날짜에 업로드된 영상만** 가져오도록 수정. 클로드에게 "collect 스크립트를 수정할 건데 이제 어제 날짜에 업로드된 영상만 가져오도록 해 줘. 어제 업로드된 영상이 없는 채널은 스킵하고, 결과물이 output 폴더에 날짜 형태로 들어가도록"이라 요청. 기다리면 '최근 몇 개'에서 '어제 업로드분만'으로 collect 파일이 바뀐 것을 확인.

### [11:00–12:10] CLAUDE.md(인수인계서) 생성
워크플로우가 완성됐으니, 별도 설명 없이도 작업을 수행하도록 CLAUDE.md를 생성. 클로드는 세션(대화창)이 끝나면 이전 내용을 잘 기억하지 못해, 내일 다시 실행하면 처음부터 설명해야 한다. 이를 막기 위한 '인수인계서'가 CLAUDE.md이며, 프로젝트 폴더를 열 때마다 먼저 읽고 작업을 시작하게 된다. 요청: "지금까지 만든 작업 방식을 기반으로 CLAUDE.md를 작성해 줘. 이 프로젝트가 뭘 하는지, 실행하면 collect 스크립트를 먼저 실행하고 결과를 읽은 다음 format 형식에 맞춰 요약해 output 폴더에 결과물을 생성해 줘. 추가로 별도 지시 없이 그냥 실행하거나 '어제 거 요약해 줘'라고만 해도 이 흐름대로 작동하도록."

### [12:10–12:39] CLAUDE.md 완성 · 흐름 재확인
CLAUDE.md가 완성됨. 흐름을 보면 맨 처음 collect 스크립트로 영상을 수집하고 요약 결과물을 output 폴더에 저장. 이제 CLAUDE.md가 있으니 별도 과정 없이 "그냥 어제 거 요약해 줘"라고만 작성해도 유튜브 정보들이 저장된다.

### [12:39–13:44] 노션 저장 옵션: 커넥터 연결 + 페이지 ID
마크다운 대신 처음 보여준 것처럼 **노션에 저장**하고 싶은 경우. 이를 위해 클로드 웹이나 클로드 데스크탑에서 노션 **커넥터를 연결**해야 한다. 클로드 데스크탑에서는 좌측 하단 설정 → 커넥터 탭 → 커넥터 둘러보기에서 노션 검색·연결. (노션 연결 방법과 이후 나올 스케줄링은 저번 영상에서 다뤘으니 참고.) 연결했다면 클로드로 돌아와 "이 작업 결과를 마크다운 파일이 아니라 노션에 저장하고 싶어. 이 페이지에 날짜별 페이지를 만들고 각 영상 요약을 그곳에 저장해 줘"라고 요청하고, **노션 페이지 링크 가장 뒤의 페이지 ID**를 복사해 붙여넣어 요청. 기다리면 노션 페이지에 날짜별 페이지가 생성되고 각 영상이 format 형태로 정리됨.

### [13:44–14:13] CLAUDE.md 자동 수정 · 노션 모드로 업데이트
방금 요청으로 CLAUDE.md도 일부 수정됨. 이제 로컬 마크다운으로도 저장되고 동시에 노션 페이지에도 저장되는 상태. 노션 연동 방식으로 바뀌었으니 CLAUDE.md도 업데이트해 달라고 다시 요청.

### [14:13–14:44] [의견] 점진적 개선 습관
[의견] 처음엔 그냥 로컬 마크다운으로 받다가 더 편한 방식이 없을까 고민한 결과 더 편리한 방식으로 클로드에 요청해 수정하는 이런 과정이 가장 중요. 이 습관("어떻게 하면 더 편리할까")이 반복될수록 워크플로우가 점점 내 입맛에 맞게 다듬어진다. 처음부터 완벽하게 만들려 하지 말고 써 보면서 하나씩 고쳐 나가길 추천.

### [14:44–15:29] 스케줄링 전 파일 정리
이제 매일 직접 실행하지 않고도 자동으로 되도록 클로드의 **스케줄링** 기능으로 한층 더 자동화. 그전에 쌓인 파일들을 정리. JSON 파일이나 테스트용 파일을 정리해 둬야 나중에 테스트가 깔끔. 클로드에게 "collect와 format, CLAUDE, 그리고 channels 파일과 output 폴더는 유지하고 나머지 파일들은 정리해 줘"라고 요청 → 불필요한 파일 삭제됨.

### [15:29–16:09] 스케줄링(로컬 루틴) 등록 시작
클로드 스케줄링으로 오늘 제작한 워크플로우를 자동화. 화자 측도 실제로 이 워크플로우를 스케줄링으로 돌리는 중이며, 업무 시작 전 원래라면 매일 확인했어야 할 AI 유튜브 채널들을 따로 시간 내지 않고 바로바로 받아본다. 등록은 클로드 데스크탑에서. 좌측 **루틴 버튼** 클릭 → **로컬 루틴**으로 유튜브 영상 수집을 루틴화. 이름을 적당히 짓고 설명은 "전날 추가된 유튜브 영상을 정리해 주는 루틴"이라고 입력.

### [16:09–16:36] 폴더 선택 · 지침 · 시간 지정
하단 **폴더 선택** 버튼으로 지금까지 활용한 프로젝트 폴더를 선택. 지침에는 "어제 업로드된 영상을 수집하고 노션에 요약 저장해 줘"라고만 입력. 그다음 **원하는 시간**을 지정하고 **만들기** 버튼을 눌러 루틴 생성.

### [16:36–17:02] 로컬 루틴 주의점 · 매일 아침 자동 저장
원격 루틴이 아니라 **로컬 루틴은 컴퓨터가 켜져 있는 상태에서 작동**한다는 점에 주의. 컴퓨터가 꺼져 있으면 해당 시간대 작업은 이루어지지 않고 **다음 시간대에 동작**한다. 워크스페이스에 저장하면 루틴이 저장되고, 매일 아침 지정한 시각에 유튜브 영상들이 수집되어 내 노션 페이지에 저장되는 것을 확인 가능.

### [17:02–17:35] 마무리 · 자료 공유
오늘은 Claude Code로 유튜브 영상을 자동 수집하는 과정을 함께 봤다. 따라 해 보고 본인이 원하는 대로 변형해 원하는 결과물을 만들라고 권유. 영상에서 사용된 프롬프트나 프로젝트 파일은 오픈 채팅방에서 공유 중이니 필요하면 받아갈 수 있다. 궁금한 주제는 댓글로 남겨 달라며 마무리.
