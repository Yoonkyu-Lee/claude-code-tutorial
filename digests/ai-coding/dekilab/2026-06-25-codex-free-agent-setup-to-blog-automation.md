# 클로드가 먹통일 땐, 코덱스를 써보세요! (필수 설정부터 블로그 자동화까지)

> dekilab · 2026-06-25 · [영상 링크](https://youtu.be/yCtHqxek8Uo)
> 정리 방식: 시간순 심층 워크스루 (paraphrase) + 주제별 상세 요약
> 검증 배너: 사용량·차감 비율 등 수치는 화자 주장(미검증). 모델명 표기는 자동자막 오인식 가능성 있음.

## 요약 (주제별)

### 코덱스란 무엇인가 (포지셔닝)
- 코덱스는 OpenAI(ChatGPT 제작사)가 만든 AI 에이전트로, 클로드의 Cowork/Code와 비슷한 방식으로 쓸 수 있다. 클로드 Cowork/Code는 반드시 유료 구독이 필요하지만, 코덱스는 **무료로도 사용 가능**하다는 점이 핵심 차별점이다 (화자 주장).
- 일반 ChatGPT·Gemini는 질문→답변→그 답을 다른 곳에 옮겨 쓰는 방식이지만, 코덱스는 **작업 폴더 통제권을 맡겨** 폴더 안 파일을 직접 열어 작업하고 결과물까지 내 PC 폴더에 저장해 주는 "옆에 앉은 직원"에 가깝다 [의견].

### 요금제·사용량
- ChatGPT 무료 요금제에서도 코덱스를 쓸 수 있고, Plus 요금제면 ChatGPT 포함해 코덱스를 더 여유롭게 쓸 수 있다.
- 실습(무료 계정)에서 PPT 1건 생성 후 사용량이 약 84% 남음 = 1개월 사용량 기준 약 16% 소진(주장/미검증). 무료라 넉넉하진 않지만 입문 테스트 몇 번은 충분하다는 평가 [의견].

### 설치·로그인
- 검색창에 "코덱스" → OpenAI 코덱스 홈페이지 → 본인 OS(맥/윈도우)에 맞는 앱 다운로드·설치. 화자는 macOS 버전 사용.
- 앱 실행 후 "ChatGPT로 계속하기" 버튼으로 ChatGPT 계정 로그인하면 바로 사용 가능 상태가 된다. (터미널/IDE뿐 아니라 앱 형태로도 쓸 수 있어 클로드 Cowork처럼 입문 장벽이 낮다는 게 화자 강조점.)

### 핵심 설정 4종
- **프로젝트(폴더)**: 프롬프트 입력창 하단 "프로젝트에서 작업하기" → "새 프로젝트 추가". 프로젝트 = 평소 쓰던 컴퓨터 폴더 개념. 처음이라면 작업용 폴더를 하나 만들어 지정하는 걸 추천.
- **권한**: 프로젝트 폴더 위 "승인 요청" 버튼에서 설정. 기본값 "승인 요청"은 대부분 작업에 사용자 승인을 받음. 화자 추천은 **"나 대신 승인"** — 웬만한 작업은 알아서 하고 민감한 것만 물어봐 흐름이 안 끊김. 폴더 내용이 안 중요하면 "전체 권한", 민감하면 기본값 "승인 요청" 사용.
- **모델**: 우측에서 설정. GPT 5.5 / GPT 5.4 미니가 있음(모델명은 자동자막 오인식 가능, [불명확]). Plus면 GPT 5.5 + 추론 단계 "중간"이 무난, 무료 테스트면 5.4 미니. **차감 비율 비유**: GPT 5.5 ≈ 클로드 Opus와 Sonnet 사이, 5.4 미니 ≈ 클로드 Sonnet과 Haiku 사이 (화자 제공 기준).
- **개인 맞춤 설정·메모리**: 좌측 사이드바 하단 설정. 말투 설정, 맞춤형 지침(코덱스 전체 작업 방식 지침 — 예: 용어 풀어 설명, 파일 수정 시 승인, 다단계 작업은 설명 요청) 입력 가능. 메모리 활성화 시 이전 세션 대화를 기억해 이어서 작업 가능. (화자가 쓴 지침 예시는 영상 설명란 오픈채팅방 학습자료로 공유.)

### 실습 1 — PPT 만들기
- 디자인은 직접 맡길 수도 있으나 화자는 기존 디자인 활용. 캠바(Canva) 같은 사이트의 PPT를 첨부할 수도 있고, 화자는 대기업 디자인 문서를 무료로 보여주는 사이트([불명확: 원문 ".md라는 사이트"], 예시로 Airbnb·Apple·Nike·Notion)에서 **Airbnb 디자인 문서를 복사**해 코덱스에 붙여넣음.
- 프롬프트: 첨부한 Airbnb 디자인 문서를 참고해 PPT 형식으로 바꾸고, 그 톤에 맞춰 **부산 여행 일정을 직접 리서치·계획해 하루 1슬라이드로** 생성. 모델은 저렴한 5.4 미니, 무료 계정.
- 결과: 처음엔 영어로 생성 → 한글 수정 재요청하니 한글 PPT로 재생성, 하단에 부산 관광 정보 출처까지 표기. 결과가 PPT 파일이라 코덱스에 추가 요청하거나 PPT 앱에서 직접 수정 가능. 같은 프롬프트를 클로드(Sonnet, 작업량 "낮음")로도 비교 생성. 무료 코덱스에서 이 정도 결과가 나온 게 인상적이라는 평가 [의견].

### 실습 2 — 내 PC 폴더 직접 접근·정리
- 문서·이미지·영상이 뒤섞인 테스트 폴더를 만들어 프로젝트 폴더로 지정 → "폴더 안에 새 폴더들을 생성해서 파일들을 적절하게 정리해 줘"라고 간단 요청.
- 코덱스가 폴더 구조를 먼저 읽고 기준을 세워 비디오/이미지/문서별로 정리. "원래대로 되돌려 줘"라고 하면 원상복구. 프로젝트 폴더에 직접 접근해 읽기·쓰기·저장까지 한다는 걸 실증.

### 실습 3 — 플러그인·스킬(노션 자동화)
- **플러그인** = 코덱스에 새 능력을 붙이는 도구. 기본은 내 폴더 안에서만 일하지만, 플러그인 연결 시 폴더 바깥의 외부 앱([불명확: 원문 "GL"]·노션·구글 캘린더 등)과 연동 가능. 좌측 플러그인 탭에서 검색·연동.
- 노션 플러그인 추가 → 연결 → 노션 계정 연동. 새 채팅에서 **@(골뱅이)** 입력 시 연결된 플러그인 확인, 노션 플러그인 선택 후 노션 페이지 ID(페이지 주소 뒷부분) 붙여넣기.
- "오늘 날짜로 코스피(KOSPI) 시황을 리서치해서 노션 페이지에 정리해 줘" → 코덱스가 리서치 후 지정 노션 페이지에 한줄요약·핵심 포인트·시장 배경·체크포인트·참고 자료 순으로 정리.
- **스킬** = 자주 쓸 작업을 미리 가르쳐 저장한 기능. 작업을 다 한 뒤 "지금까지 한 작업을 스킬로 만들어 줘"라고 하면 "코스피 노션 리서치" 스킬 생성. 새 대화에서 **/(슬래시) + 스킬명(코스피)** 만으로 추가 프롬프트 없이 실행. 정리: **@ = 플러그인, / = 스킬**.

### 실습 4 — 블로그 글 + 이미지 자동화
- AI에게 그냥 쓰라면 "AI 냄새" 나는 글이 나오므로, 내가 기존에 쓴 블로그 글을 첨부해 문체를 참고시킴. 화자는 특정 문체가 드러난 블로그 글 3개를 텍스트 파일로 첨부.
- 프롬프트: "첨부한 3개 글은 내가 쓴 블로그 글이고, 내용·주제는 무시하고 **어체·문체 위주로 분석**해 여행 후기 글을 작성해 줘." → 코덱스가 3개 글을 분석 후 유사 문체의 초안 생성.
- 이어서 "블로그 글에 어울리는 썸네일·본문 이미지를 GPT 이미지 모델로 만들어 줘" 요청. 클로드였다면 별도 API로 이미지를 만들어야 했지만, 코덱스는 GPT 이미지 모델로 자유롭게 생성 가능하다는 게 장점 (화자 주장). 미리보기용 HTML 파일까지 생성해 글+이미지 결과 확인. 코덱스+GPT 이미지 모델로 블로그 글 생성 에이전트를 더 강력하게 만들 수 있겠다는 전망 [의견].

### 영상의 메시지
- 토큰·비용 경쟁이 계속되는 가운데(구글·앤트로픽·오픈AI), **같은 종류의 툴이라면 하나의 원리·사용법만 익히면 다른 도구도 쉽게 쓸 수 있다**는 걸 보여주려는 게 영상 취지. 코덱스 입문을 권장. 사용된 프롬프트는 영상 설명란 오픈채팅방에서 공유.

- 클로드(Cowork/Code)가 유료·크레딧 부족으로 막힐 때 쓸 무료 대안으로 OpenAI 코덱스 앱을 소개하는 입문 튜토리얼.
- 설치·로그인 → 프로젝트/권한/모델/개인설정 4종 세팅 → PPT·파일정리·노션 플러그인/스킬·블로그+이미지 4개 실습 순으로 진행.
- 핵심 주장: 무료로도 쓸 만하고, 클로드 사용 경험이 있으면 개념이 그대로 옮겨져 입문 장벽이 낮다.

## 전문 — 시간순 워크스루

### [00:00–00:32] 훅 — 유료·크레딧 부담
AI 에이전트를 써 보고 싶지만 유료라 망설이거나, 큰맘 먹고 클로드 프로를 결제했는데 크레딧이 어느새 바닥난 경험이 있는지 물으며 연다. 오늘은 **무료로도 바로 써 볼 수 있는 에이전트 코덱스**를 다룬다고 예고하고, 입문 가이드부터 여러 꿀팁, 그리고 PPT·블로그 글 작성 같은 실전 예시로 코덱스를 본격적으로 알아보겠다고 한다. 구독·좋아요 요청.

### [00:32–01:14] 코덱스 소개 + 요금제
코덱스는 OpenAI가 만든 에이전트로, ChatGPT를 만든 그 회사 제품이다. 클로드의 Cowork나 Code와 비슷한 방식이라 클로드를 써 봤다면 더 쉽게 적용할 수 있다. 같은 급인 클로드 Cowork/Code는 반드시 유료 구독이 필요하지만, ChatGPT 요금제를 보면 **무료로도 코덱스를 활용**할 수 있고 Plus 요금제면 ChatGPT 포함 코덱스를 더 여유롭게 쓸 수 있다.
> "일단 한번 테스트해 보고 싶다라고 하시는 분들을 위해 바로 사용해 볼 수 있다는 장점이 있습니다." [01:04]

크레딧이 얼마나 쓰이는지는 뒤 실습에서 하나씩 보여준다고 예고.

### [01:14–02:04] 설치·로그인
설치는 간단하다. 검색창에 "코덱스" 검색 → 상단 OpenAI 코덱스 홈페이지 진입 → 맥/윈도우 중 본인 OS에 맞는 앱 설치. 화자는 macOS라 macOS 버전을 받아 설치 파일을 실행해 앱을 설치하고, 완료 후 더블클릭으로 실행한다. 실행되면 "ChatGPT로 계속하기" 버튼을 눌러 ChatGPT 계정으로 로그인하면 바로 코덱스를 쓸 수 있는 상태가 된다.

### [02:04–02:38] 개념 — "말로 돕는 게 아니라 직접 처리하는 직원"
코덱스는 평소 쓰던 ChatGPT·Gemini와 달리 **내가 요청한 일을 끝까지 수행하는 직원**이라 생각하면 된다 [의견]. ChatGPT는 질문→답변, 또는 파일 첨부→답을 받아 다른 곳에 옮겨 쓰는 식이다. 반면 코덱스는 AI 에이전트로서 작업 폴더 통제권을 맡길 수 있어, 폴더 안 파일을 직접 열어 작업하고 결과물 파일까지 내 컴퓨터 폴더에 알아서 저장한다.
> "옆에서 말로만 도와주는 게 아니라 제 옆에 앉아서 제가 시키는 일을 직접 처리해 주는 직원에 가깝다." [02:30]

### [02:38–03:23] 설정 ① 프로젝트(폴더)
프롬프트 입력창 하단 "프로젝트에서 작업하기" 버튼 → "새 프로젝트 추가" 탭. 여기서 프로젝트는 평소 컴퓨터에서 쓰던 폴더라고 생각하면 이해가 쉽다. 컴퓨터에서 폴더 단위로 일하듯 코덱스도 지정한 폴더(프로젝트) 단위로 작업한다. 처음 쓴다면 작업할 폴더를 하나 만드는 걸 추천. 화자는 "기존 폴더 사용" 버튼으로 새 폴더를 만든 뒤 선택해 프로젝트 폴더로 지정.

### [03:23–04:16] 설정 ② 권한
프로젝트 폴더 위 "승인 요청" 버튼을 누르면, 코덱스가 작업 실행 시 어떤 권한을 갖는지 설정하는 탭이 나온다. 기본값 "승인 요청"은 요청 작업 상당 부분에 사용자 승인을 받고 진행한다. 추천 설정은 **"나 대신 승인"** — 웬만한 작업은 알아서 하고 꽤 민감한 사항만 물어봐 작업 흐름이 안 끊긴다. 일단 이걸로 해 보다가 너무 귀찮고 폴더에 중요한 내용이 없으면 "전체 권한", 반대로 폴더 내용이 중요해 조심스럽다면 기본값 "승인 요청"을 쓰면 된다. 화자는 "나 대신 승인"으로 놓고 실습.

### [04:16–05:02] 설정 ③ 모델 + 차감 비율
모델은 우측에서 설정한다. 추론 단계와 하단을 보면 **GPT 5.5**와 **GPT 5.4 미니**가 있다 [불명확: 자동자막상 모델명이 "GPT 5.5"·"GPT 5.4 4 미니"로 잡혀 정확한 버전 표기 불확실]. Plus를 이미 쓴다면 GPT 5.5 + 추론 단계 "중간"이 무난하고, 무료로 테스트만 하려면 5.4 미니로 시도해도 된다. 클로드 쓰던 사람을 위한 **차감 비율 비유**: GPT 5.5는 클로드의 Opus와 Sonnet 사이 정도, 5.4 미니 모델은 클로드의 Sonnet과 Haiku 사이 정도. 화자는 무료 버전 사용량 테스트를 위해 5.4 미니로 진행.

### [05:02–06:31] 설정 ④ 개인 맞춤 설정 + 메모리
좌측 사이드바 → 하단 설정 버튼에서 코덱스 여러 설정을 볼 수 있다. "개인 맞춤 설정"에서 나만의 커스텀이 가능한데, 최상단에서 코덱스 **말투**를 정하고, **맞춤형 지침**에서 코덱스 전체에 적용될 작업 방식 지침을 입력한다. 예시로, 답변 언어·용어 설명을 풀어서 해 달라, 폴더 내 파일 수정 시 무엇이 어떻게 바뀌는지 승인을 받아 달라, 다단계 작업이면 어떻게 진행할지 충분히 설명해 달라 등을 적을 수 있다. 꼭 지침을 넣을 필요는 없고, 쓰다가 원하는 방식이 생기면 여기에 적으면 된다. 작성한 지침은 학습 자료로 넣어 뒀으니 영상 설명란 오픈채팅방에서 확인하라고 안내(세일즈/유입 장치). 하단 **메모리 활성화**를 켜면 코덱스가 지금까지 한 대화를 기억해 이전 세션 내용을 자연스럽게 이어 작업할 수 있다.

### [06:31–09:07] 실습 1 — PPT 만들기
첫 실습은 PPT 만들기. 디자인 자체를 코덱스에 맡길 수도 있으나 화자는 기존 디자인을 활용한다. 캠바(Canva) 같은 사이트의 PPT를 첨부하는 방법도 있지만, 화자는 예전 클로드로 PPT 만들 때 쓴 사이트에서 디자인을 가져온다 — 대기업 디자인 문서를 무료로 볼 수 있는 사이트로, Airbnb·Apple·Nike·Notion 등 다양한 기업 디자인이 있다 [불명확: 원문 ".md/md라는 사이트" — 사이트명 오인식]. 화자는 **Airbnb 디자인**을 골라 그 디자인 문서를 복사해 코덱스에 붙여넣고 요청한다: 이 Airbnb 디자인 문서를 참고해 PPT 형식으로 바꾼 뒤 그 톤에 맞춰 **부산 여행 일정을 직접 리서치·계획해 하루 1슬라이드로** 생성. 모델은 저렴한 5.4 미니, 무료 계정.

잠시 기다리니 Airbnb 디자인 문서에 맞춰 PPT를 생성했는데, 아쉽게 영어로 나와 한글로 수정 재요청하니 한글 PPT로 다시 만들어 줬고, 하단에 부산 관광 정보를 리서치한 출처까지 표기됐다. 결과가 PPT 파일이라 부족한 부분은 코덱스에 더 요청하거나 PPT 앱에서 직접 수정 가능. 같은 프롬프트를 클로드(Sonnet, 작업량 "낮음")로도 만들어 비교. 둘 다 프롬프트를 더 다듬으면 더 좋은 결과가 나오지만, **무료 코덱스에서 이 정도가 나온 게 인상적**이라 평가 [의견]. 사용량을 보니 약 84% 남음 = 1개월 사용량 기준 약 16% 소진(주장/미검증). 무료라 넉넉하진 않아도 입문 테스트 몇 번은 충분하다는 정리.

### [09:07–10:39] 실습 2 — 내 PC 폴더 접근·파일 정리
다음은 코덱스가 PC에 직접 접근해 작업하는 테스트. 영상 초반 말한 "폴더 접근해 파일 열고 생성·수정"이 실제 되는지 확인하려는 것. 문서·이미지·영상이 섞인 테스트 폴더를 만들고 하단에서 그 폴더를 프로젝트 폴더로 선택. 기준 없이 지저분하게 나열된 파일들을 정리하기 위해 "코덱스 폴더 안에 새 폴더들을 생성해서 파일들을 적절하게 정리해 줘"라고 간단 요청. 코덱스가 먼저 폴더 구조를 읽어 어떤 파일을 어떤 기준으로 나눌지 정하고, 잠시 뒤 비디오는 비디오대로·이미지는 이미지대로·문서는 문서대로 정리해 준다. 여기서 "원래대로 되돌려 줘"라고 하면 원래 형태로 복구한다. 프로젝트 폴더에 직접 접근해 읽고 쓰고 저장까지 함을 실증하며, "내 PC에 접근할 수 있다"는 말이 체감됐길 바란다고 한다.

### [10:39–12:11] 플러그인 — 노션 연결
다음은 플러그인 기능. **플러그인은 코덱스에 새 능력을 하나씩 붙여 주는 도구**다. 코덱스는 기본적으로 내 컴퓨터 폴더에서 일하지만, 플러그인을 연결하면 폴더 바깥의 외부 앱들과 연동할 수 있다 — [불명확: 원문 "GL"]이나 노션, 구글 캘린더 등 코덱스가 못 하던 일을 플러그인으로 진행 가능. 좌측 플러그인 탭 → 플러그인·스킬 검색창에서 원하는 플러그인을 검색해 연동. 다양한 플러그인이 준비돼 있는데 화자는 **노션 플러그인**을 추가한다. 추가→연결 버튼을 누르면 쓰던 노션 아이디와 연결되고, "계속하기"로 노션 플러그인이 연결된다. 테스트를 위해 새 채팅에서 **@(골뱅이)**를 입력하면 현재 연결된 플러그인들이 뜨고, 노션 플러그인을 선택. 쓰는 노션 페이지 주소 맨 뒤가 페이지 ID인데, 이 ID를 복사해 연결을 확인한다.

### [12:11–12:35] 노션에 코스피 시황 정리
"오늘 날짜로 코스피(KOSPI) 시황을 리서치해서 노션 페이지에 정리해 줘"라고 요청하면, 코덱스가 코스피 시황을 리서치하고 입력해 둔 노션 페이지에 정리해 준다. 보면 **한 줄 요약 → 핵심 포인트 → 시장 배경 → 체크포인트 → 참고 자료** 순으로 시황이 정리돼 있다.

### [12:35–13:43] 스킬 만들기
코덱스와 티키타카하며 데이터를 수정할 수도 있고, 한발 더 나아가 "오늘 날짜 코스피 시황 리서치→노션 정리" 작업 자체를 하나의 기능으로 저장할 수도 있다. 이걸 **스킬**이라 부르며, 자주 쓸 작업을 코덱스에 미리 가르쳐 두는 것이다. 만드는 법은 간단 — 하고 싶은 작업을 다 진행한 뒤 "지금까지 한 작업을 바탕으로 스킬을 만들어 줘"라고 요청하면 된다. 그러면 "코스피 노션 리서치" 스킬이 생성되고, 새 대화창에서 **/(슬래시) + 스킬명(코스피)**을 입력하면 추가 프롬프트·설명 없이 그 스킬이 실행된다. 실행하면 앞서처럼 오늘 코스피 시황을 리서치해 문서로 정리해 준다.
> "골뱅이는 플러그인, 슬래시는 스킬이라는 거 기억해 두시면 되겠습니다." [13:38]

### [13:43–15:41] 실습 3 — 블로그 글 + 이미지
마지막 실습은 간단한 블로그 글 쓰기 + 안에 들어갈 이미지까지 만들기. 그냥 AI에 글을 써 달라면 "AI 냄새" 나는 글이 나오므로, 내가 기존에 쓴 블로그 글을 첨부해 문체를 참고시킨다. 화자는 특정 문체가 잘 드러난 블로그 글 예시를 텍스트 파일로 정리해 문체를 분석시킨 뒤 그 말투 그대로 새 글을 쓰게 한다. 문서를 끌어다 첨부하고 요청: "첨부한 세 가지 글은 내가 쓴 블로그 글이고, 그 글에서 내 말투·문체 패턴을 분석해 여행 후기 글을 작성해 줘. 내용·주제는 무시하고 어체·문체 위주로 분석해 줘." 그러면 코덱스가 세 글을 쭉 분석한 뒤 글 초안을 작성하고, 확인해 보면 첨부 글과 유사한 문체의 결과물이 나온다.

이어서 글과 어울리는 이미지도 생성한다. 클로드였다면 별도 API로 이미지를 만들어야 했지만, 코덱스는 GPT 이미지 모델로 자유롭게 생성할 수 있는 장점이 있다 (화자 주장). "방금 쓴 블로그 글에 어울리는 썸네일과 본문 이미지도 GPT 이미지 모델로 만들어 줘"라고 하면 글과 어울리는 이미지들이 바로 생성된다. 미리보기용 HTML 파일도 생성해, 블로그 글과 GPT 모델 이미지가 함께 든 결과물을 확인. 코덱스+GPT 이미지 모델로 블로그 글 생성 에이전트를 더 강력하게 만들 수 있겠다는 전망 [의견].

### [15:41–16:38] 마무리 — 툴 하나 익히면 다 통한다
이렇게 코덱스에 입문해 여러 설정을 거치고 실습으로 친해지는 시간을 가졌다. 생각보다 간단했다며, 코덱스도 IDE·터미널뿐 아니라 앱에서 쓸 수 있어 클로드 Cowork처럼 비교적 쉽고 거부감 없이 입문했을 거라 본다 [의견]. 최근 토큰·비용 경쟁이 계속되는데(구글·앤트로픽·오픈AI), 회사가 달라도 **같은 종류의 툴이라면 하나의 원리·방법만 깨치면 다른 도구에서도 쉽게 쓸 수 있다**는 걸 보여주고 싶었다고 밝힌다. 오늘 영상을 보고 꼭 코덱스에 한 번씩 입문해 보길 권하고, 사용된 프롬프트들은 영상 설명란 오픈채팅방에서 공유 중이니 참고하라고 안내(세일즈/유입 장치). 인사로 마무리.
