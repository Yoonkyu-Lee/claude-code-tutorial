## 요약 (주제별)

### 1. 목표 · 배경 — Fable 5로 게임 개발
- 클로드의 최고급 모델 **Fable 5**로 게임을 만들어 본 실습 영상. 화자에 따르면 Fable 5는 출시 4일 만에 사라졌다가 다시 돌아왔고, 커뮤니티에서 성능에 놀라고 있다(화자 진술/미검증). 제목의 "미국이 금지시켰던"도 미검증 배경 서술.
- Fable이 게임 개발에서도 뛰어난 성능을 보인다고 해서 직접 만들어 봤다는 것이 출발점. 총 3개 게임 데모(3D 축구 / 배틀로얄 / 곰돌이 횡스크롤)로 구성.

### 2. 세팅 · 작업 방식
- 클로드에서 모델을 **Fable 5**로 변경(기간 한정 제공이라 함), 작업량은 '높음' 정도로 둠.
- 핵심 방식: 화자가 직접 리서치해 지시하지 않고 **클로드가 스스로 리서치하고 게임을 고도화**하도록 맡김. 간단한 자연어 프롬프트만 제공.
- 데모 3(곰돌이 게임)은 **클로드 코드**에서 진행, 권한은 **수동 승인**으로 설정.

### 3. 데모 1 — 3D 축구 '월드 스트라이커즈'
- 요청: "국가별 필살기가 있는 아케이드 스타일 3D 축구 게임". 결과물 이름 **월드 스트라이커즈(World Strikers)**. 대한민국 포함 여러 국가 팀 생성.
- 조작: **X=공 뺏기, Z=패스, C=필살기**. 상대 AI가 강해 3대 0까지 벌어지고 상대가 필살기까지 사용.
- 평가([의견]): 패스·슛·태클이 자연스럽고, 요청하지 않았던 것이 아니라 요청한 필살기까지 잘 구현, 눈에 띄는 버그 없이 간단 프롬프트로 뛰어난 결과.

### 4. 데모 2 — 배틀로얄 '아일랜드 로얄'
- 요청: 섬 맵에 낙하해 아이템·무기 파밍하고 싸우는 서바이벌 배틀로얄 + **시간이 지날수록 안전지대가 축소되는 로직** 추가. 결과물 이름 **아일랜드 로얄(Island Royale)**.
- 구현: **WASD 이동, 조준·재장전, 점프, 파밍, 질주**, 낙하 모션, 봇끼리 전투, 안전지대 실시간 축소, **엄폐물 뒤 숨으면 피격 감소**. 화자는 접전 끝에 최후의 1인 달성.
- 화자가 지정한 로직(안전지대 축소)을 그대로 반영해 준 점을 강조.

### 5. 클로드 단독의 한계
- 클로드에는 **자체 이미지 생성 모델이 없어** 게임 오브젝트가 간단한 도형(모형)으로만 생성됨 → 시각적 완성도의 아쉬움. 이 한계를 Higgsfield로 보완하는 것이 후반부 주제.

### 6. Higgsfield MCP — 연결 · 역할
- **Higgsfield MCP**를 붙이면 Higgsfield의 **이미지·영상·오디오**를 활용해 게임을 개발 가능. 사이트에는 이미 Fable로 만든 앱·웹·게임이 있고 구현 수준이 높았으며 멀티플레이도 가능했다고 함([의견]).
- 연결 절차: Higgsfield 사이트 상단 **MCP 탭 → 가운데 URL 복사 → 클로드 데스크탑 '사용자 지정' → '커넥터' → '커스텀 커넥터 추가' → 이름 Higgsfield + 복사한 URL 붙여넣기 → 추가 → 데스크탑 앱 재시작**.

### 7. 데모 3 — 곰돌이 횡스크롤 '베어대시' (Higgsfield 에셋 활용)
- 요청: 고전 게임 스타일의 귀여운 곰돌이 횡스크롤 게임 + **Higgsfield MCP로 필요한 에셋 생성**. 결과물 이름 **베어대시(Bear Dash)**.
- 클로드가 스스로 생성한 것: 몬스터 캐릭터, 게임 에셋(**Nano Banana 모델** 활용), 배경 이미지, **BGM**, **효과음**(동전 먹는 소리·몬스터 죽는 소리 등).
- 디테일: 살짝 누르면 절반·지긋이 누르면 높이 뛰는 **가변 점프**, 특정 점수 도달 시 **무적 모드로 고속 전진**. 특별한 버그 없이 완성([의견]).

### 8. 크레딧 구조 (주의)
- 클로드 안에서 Higgsfield MCP 사용 시: 게임 개발·대화 등 작업은 **클로드 크레딧**에서 소모, 실제 생성되는 **이미지·영상·오디오만 Higgsfield 크레딧** 소모.
- Fable 특성상 방치하면 Higgsfield가 에셋을 끊임없이 생성하므로, **수동 승인**으로 두고 필요한 에셋만 만들도록 통제하는 것을 권장.

### 9. 결론 · 세일즈 장치
- [의견] 원하는 캐릭터를 바로 만들어 나만의 게임을 만들 수 있다는 점이 인상적. **Fable + Higgsfield 조합**이면 비전문가도 캐릭터 디자인·게임 로직을 넣어 부담 없이 게임 개발 가능.
- AI 발전으로 전문 기술이 필요하던 게임 개발이 손쉬워졌다는 마무리. Higgsfield 사용은 **영상 설명란·고정 댓글** 참고 안내(유입/세일즈 장치, 정보로 1회 기록) + 구독·좋아요 CTA.

- 클로드의 최고급 모델 **Fable 5**로 자연어 프롬프트만 써서 3D 축구 게임·배틀로얄 게임을 개발해 본 실습 영상.
- Fable 단독은 자체 이미지 생성이 없어 오브젝트가 단순 도형인데, **Higgsfield MCP**를 클로드 데스크탑에 연결하면 이미지·영상·오디오(BGM·효과음) 에셋까지 클로드가 스스로 생성해 게임에 넣어 준다.
- 결론: Fable + Higgsfield 조합이면 비전문가도 원하는 캐릭터 디자인과 게임 로직을 넣어 부담 없이 게임을 만들 수 있다는 것을 세 개의 데모로 보여줌.

Higgsfield 사이트에 들어가니 이미 Fable을 사용한 앱·웹사이트·게임들이 있었고, 직접 플레이해 보니 구현 수준이 꽤 높았으며 다른 사람들과 함께 플레이도 가능했다([의견]). 많은 사용자가 Higgsfield에서 만든 스크린샷/에셋(자막 "SS")을 바탕으로 나만의 게임을 만들어 둔 것을 확인했고, **클로드 단독으로 못 하는 부분을 Higgsfield로 채운 사례**로 소개한다.

### [04:38–05:10] Higgsfield MCP 연결 방법
화자도 따라 만들기 위해 Higgsfield MCP를 연결한다. 절차는: Higgsfield 사이트 최상단의 **MCP 탭 클릭 → 가운데 URL 버튼을 클릭해 복사 → 클로드 데스크탑 앱에서 좌측 '사용자 지정' → '커넥터' → 우측 상단 '커스텀 커넥터 추가' 클릭 → 이름은 Higgsfield, URL에 복사한 MCP URL을 붙여넣고 추가**. 추가하면 즉시 사용할 수 있다.

Higgsfield 커넥터가 연동된 것을 확인하고 **데스크탑 앱을 재시작**한 뒤 바로 게임 개발을 시작한다.

### [05:20–05:34] 데모 3 요청 — 고전 스타일 곰돌이 횡스크롤 게임 + Higgsfield로 에셋 제작
이번엔 **고전 게임 스타일의 귀여운 곰돌이 캐릭터로 횡스크롤 게임**을 만들어 달라고 요청하고, 단 이때 **Higgsfield MCP를 활용해 게임에 필요한 에셋(자막 "SS")을 만들어 달라**고 함께 요청한다.

### [05:34–05:54] 데모 3 세팅 — 클로드 코드 · 수동 승인
모델은 Fable 5, **클로드 코드**에서 진행했고, 권한 설정은 **수동 승인**으로 뒀다. 그 이유는 Fable 특성상 그냥 맡겨 두면 Higgsfield에서 **에셋을 끊임없이 생성**하기 때문이라, 수동 승인으로 두고 **필요한 에셋만** 만들도록 지정해 개발했다.

### [05:54–06:12] 크레딧 구조 안내
클로드 안에서 Higgsfield MCP를 쓸 때의 **크레딧 소모 구조**를 설명한다. 게임 개발·대화 등에서 쓰이는 크레딧은 **모두 클로드 크레딧**에서 소모되고, **실제 생성되는 이미지·영상·오디오에 대해서만 Higgsfield 크레딧**이 소모된다.

### [06:12–06:37] 데모 3 진행 — 캐릭터·에셋·배경·BGM 자동 생성
요청하면 사고 과정이 시작되며 스스로 게임 구성을 시작한다. **몬스터로 보이는 캐릭터**도 스스로 생성하고, 고전 게임 스타일의 게임 에셋을 **Nano Banana 모델**을 활용해 생성한다. 배경 이미지까지 만들고, 게임에 들어갈 **BGM**도 생성해 준다.

### [06:37–06:55] 효과음까지 자동 생성 → '베어대시' 완성
활기찬 BGM이 만들어졌고, 이외에 **동전 먹는 소리·몬스터 죽는 소리** 등 게임에 필요한 효과음들도 클로드가 스스로 Higgsfield MCP로 생성해 준다. 잠시 기다리니 **베어대시(Bear Dash)**라는 이름으로 게임 개발이 완료됐다.

### [06:55–07:26] 데모 3 결과물 — 이미지·BGM·효과음 반영, 디테일한 조작
방금 클로드와 함께 만든 이미지·배경 음악·효과음을 모두 사용해 게임이 완성됐다. 디테일로 **점프를 살짝 누르면 절반만, 지긋이 누르면 더 높이 점프**하는 가변 점프가 들어갔고, **특정 점수가 되면 무적 모드가 돼 빠른 속도로 전진**하는 설정까지 있었다. 특별한 버그 없이 게임이 잘 개발됐다고 평한다([의견]).

### [07:26–07:42] 종합 평가 — 나만의 캐릭터로 게임 제작
무엇보다 **내가 원하는 캐릭터를 바로 만들어 나만의 게임을 만들 수 있다**는 점이 인상적이라고 한다([의견]). 이런 식이면 누구나 **클로드 Fable + Higgsfield 조합**으로 원하는 캐릭터 디자인과 게임 로직 설정을 넣어 부담 없이 게임 개발을 해 볼 수 있을 것 같다고 정리한다.

### [07:42–08:15] 마무리 — 접근성 강조 + CTA
AI 발전에 따라 **전문 기술이 필요한 게임 개발도 한층 손쉬워졌다**는 점을 확인했다며, 클로드와 Higgsfield를 한 번쯤 써 보길 권한다. Higgsfield를 써 보고 싶다면 **영상 설명란과 고정 댓글**을 확인하라고 안내하고(세일즈/유입 장치, 정보로 1회 기록), 궁금한 점은 댓글로 남기라고 한 뒤 구독·좋아요를 요청하며 마친다.
