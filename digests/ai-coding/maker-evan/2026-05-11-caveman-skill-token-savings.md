# 클로드한테 우가우가 시켰더니 토큰 87% 줄었습니다

> maker-evan · 2026-05-11 · 약 6분 26초 · [영상 링크](https://youtu.be/iEScgjg_eR4)
> 정리 방식: 시간순 심층 워크스루 (paraphrase) + 주제별 상세 요약
> 검증 배너: 토큰 절감률·정확도 향상 수치는 도구 제작자의 벤치마크/인용 논문에 근거한 화자 전달값이며, 화자가 직접 재현 검증한 것은 아니다 (주장/미검증).

## 요약 (주제별)

### 1. caveman(원시인) 스킬이란
- 케이브맨(caveman, 우리말로 "원시인")은 GitHub에서 빠르게 퍼지고 있는 도구로, Claude Code·Cursor·Codex 같은 AI 코딩 도구에 끼우는 **스킬 파일**이다.
- 스킬 = AI에게 "이렇게 행동하라"고 미리 알려주는 매뉴얼. 새 인턴에게 "여기선 이메일 짧게, 인사말 빼고 써"라고 첫날에 알려주는 것과 같은 개념.
- caveman 스킬의 지시 내용: "군더더기 빼, 짧게 말해, 정관사(the/a)도 빼. 단 기술 용어 영어는 정확히." → AI가 답변 모드 자체를 바꿔 원시인처럼 짧게 말하게 된다.

### 2. 왜 필요한가 — 긴 답변의 두 문제
- AI 도구는 답이 길게 나오는 게 기본이다. 화자는 두 가지 문제를 지적한다.
- **비용**: AI는 토큰(출력량) 단위로 과금되므로 답이 길수록 청구서가 두꺼워진다.
- **시간**: 결국 코드 한 줄·답 한 줄을 보고 싶은 건데, "자서전 한 권"을 읽고 그 안에서 답을 찾아야 하는 상황이 된다.
- [의견/비유] 친구에게 "지하철역 어디야" 물었는데 "좋은 질문이에요. 이 동네 교통 인프라는 1980년대에 설계되었는데…" 하고 박찬호처럼 장황하게 말하는 것과 같다. "두 번째 골목 끝" 한 마디면 되는데 AI는 친절이 디폴트라 5분짜리 설명을 반복한다.

### 3. 작동 방식 · 설치
- 설치는 터미널에 **curl 한 줄** 붙여넣으면 끝. macOS/Linux는 터미널 한 줄, Windows는 PowerShell 명령어 한 줄. 조건은 **Node 18 이상**.
- 설치 시 컴퓨터에 깔린 AI 도구들을 자동 인식한다 — Claude Code, Codex, Gemini, Cursor, Windsurf, Cline, Copilot 등 **30개 넘는 에이전트**. 안 깔린 것은 건너뛰므로 안전하다.
- 인식한 각 에이전트의 스킬 폴더에 caveman 스킬을 넣어준다. 이후 도구에서 그냥 `caveman`이라고 치면 모드가 켜지고, `normal`(노말 모드)이라고 하면 평상시 모드로 돌아간다.

### 4. 강도 4단계
- **light(라이트)**: 가벼운 버전. 군더더기 인사말만 제거.
- **full(풀)**: 기본값. "원시인 모드 정통."
- **ultra(울트라)**: 전보처럼 짧아짐. 단어 하나하나가 다이아몬드급으로 압축.
- **wenyan([불명확: 원문 "웨안"], 文言/한문 모드)**: 답이 한자(한문)로 나온다. 영어보다도 글자 수가 짧다고 함. 화자는 "농담 같기도 한데 진짜 있다"고 언급.

### 5. 벤치마크 — 토큰 절감 주장
- 제작자가 Claude API에 실제 개발자 질문 10가지를 던져 측정 (React 리렌더 설명, 인증 미들웨어 수정, Docker 멀티스테이지 빌드 등).
- 평균 결과: 원래 답 하나당 평균 **1214 토큰** → caveman 적용 시 **294 토큰**. 차이 약 920토큰, **평균 65% 절약** (주장/미검증).
- 가장 많이 절약된 경우는 **87%**까지 (주장/미검증).
- 비교 기준의 정직성: 그냥 길게 답하는 모드가 아니라, **이미 "짧게 답해줘"라고 부탁한 모드**와 비교했다. 즉 이미 짧은 모드보다도 caveman이 더 짧다는 것.

### 6. 정확도 논점 — 짧은 답이 더 정확할 수도
- 2026년 3월 논문이 인용됨: LLM에게 짧게 답하라고 강제했더니 어떤 벤치마크에서 정확도가 **26점 올랐다**는 내용 (주장/미검증, 화자 전달).
- [의견/해석] 길게 말한다고 더 정확한 게 아니라, 오히려 길게 말하다 보면 모델이 자기 말에 흔들려 틀린 답으로 가는 경우가 있다. 시험에서 처음 고른 답을 자꾸 의심하며 고치다 틀리는 것과 비슷.
- 결론: caveman은 단순히 돈만 아끼는 게 아니라 답이 더 정확해질 수도 있다는 주장.

### 7. 부가 기능 4가지
- **caveman commit**: 커밋 메시지를 원시인처럼 짧게(50글자 이내), "무엇을 했는지"가 아니라 "왜 했는지" 위주로.
- **caveman review**: PR 리뷰를 한 줄로 끊어줌.
- **caveman stats([불명확: 원문 "스치"])**: 그동안 caveman 덕분에 아낀 토큰·달러를 보여줌.
- **caveman compress**: CLAUDE.md 같은 메모리 파일(AI가 매 세션 읽는 매뉴얼)을 caveman 식으로 다시 써 평균 **46% 축소** (주장/미검증). 매 세션 읽히는 파일이라 한 번 줄여두면 토큰이 영구적·반복적으로 절약됨(일회성 아님). 화자가 개인적으로 가장 마음에 든 기능.

### 8. 상태 표시 · UX
- Claude Code에서는 상태바에 지금까지 절약한 **누적 토큰**이 게임 점수처럼 표시된다.

### 9. 화자의 개인 경험 · [의견]
- 화자는 영상 만들며 직접 켜고 일주일 사용. 처음엔 답이 너무 짧아 "이게 맞나, 더 설명 안 해주나" 싶었지만, 일주일쯤 지나니 반대가 되어 평소 Claude 답이 잔소리처럼 너무 길게 느껴졌다고 함.
- [의견] "토큰도 가벼워지고 답도 빨라지고 핵심만 딱 보여주니 안 쓸 이유가 없다." "이거 진짜 개꿀."
- 다음 영상 예고: caveman 제작자가 함께 공개한 도구 두 개 — caveman(원문 반복)과 **cave kit(케이브 킷)** 소개 예정. [불명확: 앞의 "caveman"과 별개 도구 두 개라는데 하나가 caveman으로 중복 언급됨]

설치는 빠르다. macOS/Linux면 터미널 열고 한 줄, Windows면 PowerShell에서 명령어 한 줄. 조건은 **Node 18 이상**. 자기 컴퓨터에 깔린 에이전트만 자동 인식해 깔아주고, 안 깔린 건 건너뛰니 안전하다. 설치가 끝나면 Claude Code 같은 도구를 켜고 `caveman`을 치면 된다.

### [05:04–05:20] 상태바 표시와 끄기
또 하나 재밌는 건 Claude Code에서는 상태바에 표시가 뜬다는 것 — 지금까지 절약한 누적 토큰이 게임 점수처럼 보인다. 끄고 싶으면 그냥 `normal`(노말 모드)이라고 하면 다시 평상시 Claude로 돌아간다.

### [05:20–05:38] 세 줄 정리
- 첫째, AI에게 원시인처럼 말하라고 시키는 건 한 줄짜리 스킬이다.
- 둘째, 평균 출력 토큰을 65% 줄여준다. 정확도는 그대로거나 오히려 올라갈 수도 있다.
- 셋째, 설치 한 줄·명령어 한 줄이니 Claude Code 쓰는 사람이라면 한 번 켜놓고 며칠 써볼 만하다.

### [05:38–06:06] 화자 개인 경험과 마무리
[의견] 화자가 개인적으로 가장 마음에 든 건 caveman compress였다 — 매번 읽히는 매뉴얼 파일을 줄여 놓으면 토큰이 반복적으로 절약되니까. 이어 시청자에게 "AI 도구 쓸 때 토큰 비용 신경 쓰는지, 아니면 길게 받고 보는 편인지" 물으며 어떤 도구를 한 달에 얼마 정도 쓰는지 댓글 공유를 요청. 좋아요·구독 요청 후 마지막 한 마디: "이거 진짜 개꿀." 직접 켜고 일주일 써봤는데 처음엔 답이 너무 짧아 "이게 맞나, 더 설명 안 해주나" 싶었지만, 일주일쯤 되니 반대가 되어 평소 Claude 답이 잔소리처럼 너무 길게 느껴졌다고 함. 토큰도 가벼워지고 답도 빨라지고 핵심만 딱 보여주니 안 쓸 이유가 없다는 소감.
