# 제가 챗GPT 버리고 클로드로 다시 돌아간 진짜 이유

> 정리 방식: 시간순 심층 워크스루 (paraphrase) + 주제별 상세 요약
> 검증 배너: 성능·순위·모델 버전 관련 수치는 화자 주장(미검증). 유료 플랜 가격은 화자가 소개한 값.
> 자막 정정: 한국어 자동자막 음차를 문맥으로 복원 — `채비티`→ChatGPT, `재미나이`→Gemini, `클로드`/`클러드`→Claude, `오퍼스`/`오포스`/`오프스`→Opus, `소넷`/`소네`→Sonnet, `하이쿠`→Haiku, `코워크`→Cowork, `맥OS`→macOS. 모델 버전 숫자(`4.6`, `4.0`)는 발언 그대로 보존(영상 내 표기 불일치).

## 요약 (주제별)

### 배경 — 클로드로 돌아온 이유
- 화자는 원래 Claude 주력 사용자였고, 대화가 자연스럽고 사용 느낌이 다른 AI와 달랐다고 함 [의견].
- 작년에 Gemini가 무료+성능 향상으로 올라오면서 서브로 쓰다 Claude 사용 빈도가 줄었다고 솔직히 인정.
- Opus `4.6` 업데이트 이후 다시 Claude로 확 돌아왔고, 오히려 격차가 더 벌어진 느낌이라고 주장(미검증).
- 이 영상의 타깃: 코딩·개발을 전혀 몰라도 Claude를 업무에 바로 써먹는 법을 실연으로 보여주는 것.

### `4.6` 업데이트에서 느낀 3가지 변화 (주장)
- 첫째, 글쓰기 퀄리티가 또 올라감. 대본·기획 문서 등 글쓰기가 핵심인 작업에서 현재 AI 중 Claude가 가장 낫다고 봄 [의견]. 다른 AI로 바꿔봤다가 다시 돌아온 이유가 이것.
- 둘째, Skill과 Cowork 기능이 생김. 업무 방식을 저장해 두고 내 PC 파일을 직접 정리해 주는 기능으로 활용도가 한 단계 올라감.
- 셋째, 개발자 사이에서 Claude Code가 인기이며 "코딩 실력 평가에서 현재 세계 1위"라고 함(주장/미검증). Opus 모델 자체의 코딩 능력이 매력적이라고 평가.

### 모델 3종 선택 가이드 (Opus / Sonnet / Haiku)
- Opus: 최고 성능 모델. 계약서 검토, 긴 보고서 분석, 여러 단계를 거치는 기획서 작성 등에 사용. 답변이 느리고 비용도 더 들지만 결과물이 확실히 좋음.
- Sonnet `4.0`: 일상 업무용. 요약, SNS 문구, 이메일, 간단한 분석 등. Opus보다 훨씬 빠르고 비용도 낫다. 웬만한 글쓰기는 Sonnet만으로 충분.
- Haiku: 에이전트 개발자용이라 (비개발자에게는) 쓰는 걸 추천하지 않음.
- 최종 정리에서는 "Opus `4.6`은 복잡한 작업, Sonnet `4.6`은 일상 업무"로 상황에 따라 골라 쓰라고 함 (버전 표기가 앞부분 Sonnet `4.0`과 불일치 — [불명확]).

### Skill — 전문 지식 주입 (영상 핵심)
- Skill = Claude에게 특정 분야 전문 지식을 심어 주는 기능. 카드뉴스 스킬을 만들면 카드뉴스 전문 에디터처럼, 뉴스레터 스킬을 넣으면 뉴스레터 전문가처럼, 영업제안서 스킬을 넣으면 영업 전문가처럼 행동.
- 효과: "카드뉴스 만들어 줘" 한 마디만 해도 대상·문체가 전부 맞게 나옴. AI 잘 쓰는 사람들의 공통점(매번 긴 설명 안 해도 자기 스타일 결과물이 나옴)의 비결이라고 주장.
- 설정 경로: 왼쪽 아래 프로필 → 설정 → 기능 탭 → 스킬 → 추가 버튼. 처음이면 "대화로 만들기" 추천 — Claude가 어떤 스킬을 원하는지 물어보고 초안을 만들어 줌. 과거 대화를 보고 만들 법한 스킬을 추천까지 해 줬다고 함.
- 스킬 이름은 영문으로 작성. 지침란에 뭘 쓸지 모르면 Claude에게 "카드뉴스 제작 좋은 지침 만들어 줘"라고 물어 답을 그대로 복사·붙여넣기 — 이것이 전문 지식 주입 과정.
- 업종별 활용: 카페 운영 → SNS 마케터 지식, 영업직 → 영업제안서 전문가 지식. 업무마다 하나씩 만들어 두면 됨.

### Skill 실연 — 카드뉴스 제작
- 카드뉴스 스킬 저장 후 새 채팅에 "2026년 최신 AI 활용해서 업무시간 줄이는 비즈니스 팁 카드뉴스 만들어 줘"만 입력.
- 스킬을 쓰라고 말하지 않았는데도 알아서 스킬을 확인하고 시작. 표지 제목 + 본문 3장 + 마무리까지 한 번에, 디자인을 하나도 안 했는데 완성 형태로 나옴.
- 수정도 대화로 가능("이 부분 좀 더 간결하게 바꿔줘"). 캡처해 인스타그램·블로그에 바로 업로드 가능.
- 한 번 잘 만들어 두면 이후엔 주제만 바꿔 던지면 됨(신메뉴 소개, 매장 할인 이벤트 홍보 등).

### Claude Code — 비개발자 맛보기
- Claude Code = 코딩을 전혀 몰라도 말로 시키면 Claude가 코드를 직접 짜 줌. 앱·웹사이트도 코딩 모르는 사람이 만들 수 있음.
- 브라우저에서 바로 써 보는 맛보기 버전이 있음("이런 기능 만들어 줘"로 요청).
- 진짜 Claude Code는 터미널 기반. 터미널 = 컴퓨터의 명령어 입력창. 내 컴퓨터 파일 전체를 읽고 여러 파일을 동시에 수정·실행까지 해 줌.
- macOS에 최적화되어 있어 개발자가 맥을 쓰는 이유 중 하나라고 함(주장). 화자도 이 때문에 맥북 추가 구매를 고민 중(현재 윈도우 노트북 보유).

### 단점 — 토큰 소모와 플랜
- Claude는 토큰을 많이 씀. 토큰 = 일을 할수록 소모되는 사용량(데이터 요금제 비유).
- 월 20달러 Pro 사용자가 Cowork로 파일 정리하거나 Opus로 긴 문서 작업하면 생각보다 빠르게 한도가 참.
- 본격 업무 사용자는 월 100달러 / 월 200달러 Max 플랜을 쓰는 경우도 많음(주장).
- 추천: 처음이면 Pro로 시작하고 부족하면 상위 플랜으로 올리기.

### 최종 4줄 정리
- ① 모델: Opus `4.6`은 복잡한 작업, Sonnet `4.6`은 일상 업무로 골라 쓰기.
- ② Projects 기능으로 전용 작업 공간을 목적별로 만들어 두면 매번 설명 안 해도 내 스타일 그대로 나옴.
- ③ Skill로 전문 지식을 심어 두면(카드뉴스·뉴스레터·제안서) 이후 주제만 바꿔 던지면 전문가처럼 뽑아 줌.
- ④ Claude Code는 코딩 몰라도 말로 시키면 되고, 제대로 쓰려면 macOS 터미널이 최적.

- 비개발자가 코딩 없이 Claude를 글쓰기·기획·업무 자동화에 바로 쓰는 입문 가이드.
- 핵심 무기는 Skill(전문 지식 주입)이며 카드뉴스 제작을 실연으로 보여준다.
- Opus/Sonnet/Haiku 모델 선택, Projects·Cowork, Claude Code 맛보기, 토큰·플랜 단점까지 훑는다.

## 전문 — 시간순 워크스루

### [00:00–00:22] Claude 팬이었다가 흔들렸다가 다시 돌아온 배경
화자는 자신이 Claude의 팬이며 ChatGPT·Gemini가 나왔을 때도 Claude를 주력으로 써 왔다고 밝힌다. 대화가 자연스럽고 사용 느낌이 다른 AI와 달랐다는 것 [의견]. 다만 작년에 잠깐 흔들렸다고 솔직히 인정하는데, Gemini가 무료에 성능까지 올라오면서 서브로 쓰다 보니 Claude 사용 빈도가 줄었다고 한다. 그러다 Opus `4.6` 업데이트 이후 다시 확 돌아왔고, 원래도 좋았지만 격차가 더 벌어진 느낌이었다고 주장한다(미검증).

### [00:22–00:43] 영상의 목적과 3가지 변화 예고
화자가 중요하게 쓰는 기능은 글쓰기·기획·업무 자동화이며, 이쪽에서 Claude가 다른 AI와 비교가 안 될 정도라고 말한다 [의견]. 오늘 영상은 코딩·개발을 전혀 몰라도 Claude를 업무에 바로 써먹는 법을 따라할 수 있게 실연으로 보여주겠다고 예고한다. 그리고 Opus `4.6` 업데이트 이후 느낀 변화가 크게 세 가지라고 운을 뗀다.

### [00:43–01:04] 변화 ① 글쓰기 퀄리티
첫 번째는 글쓰기 퀄리티가 또 올라간 것. 원래도 Claude가 글쓰기에 강했는데 이번에 더 좋아진 게 느껴졌다고 한다. 대본 작업이나 기획 문서를 Claude로 많이 쓰는데, 다른 AI로 바꿔 봤다가 다시 돌아온 이유가 바로 이것이라고. 글이 자연스럽고 제목(맥락)을 잘 잡아 준다며, 대본·강의 자료·제안서처럼 글쓰기가 핵심인 자료라면 지금 AI 중 Claude가 가장 낫다고 본다 [의견].

### [01:04–01:26] 변화 ② Skill·Cowork, 변화 ③ Claude Code
두 번째는 Skill과 Cowork 기능이 생긴 것. 업무 방식을 저장해 두고 내 PC 파일을 직접 정리해 주는 기능으로, 이게 생기면서 활용도가 한 단계 올라갔다고 하며 뒤에서 자세히 설명하겠다고 예고한다. 세 번째는 개발자들 사이에서 Claude Code가 크게 인기라는 점.
> "코딩 실력 평가에서 현재 세계 1위라고 하는데" [01:16] (주장/미검증)
이것도 비개발자 시각에서 어떻게 쓰는지 보여주겠다고 하며, Opus 모델 자체가 굉장한 코딩 능력을 지녀 이 부분이 매력적이라고 평가한다.

### [01:26–01:53] 모델 3종 — Opus / Sonnet / Haiku
Claude에는 Opus·Sonnet·Haiku 세 버전이 있다고 소개한다. Opus는 최고 성능 모델로 계약서 검토, 긴 보고서 분석, 여러 단계를 거쳐야 하는 기획서 작성 등에 쓴다. 답변이 느리고 비용도 더 들지만 결과물이 확실히 좋다고. Sonnet `4.0`은 일상 업무용으로 요약·SNS 문구·이메일·간단한 분석에 쓰며, Opus보다 훨씬 빠르고 비용도 낫고 웬만한 글쓰기는 이것만으로 충분하다고 한다. Haiku는 에이전트 개발자용이라 (비개발자에게) 쓰는 걸 추천하지 않는다.

### [01:53–02:25] Skill이란 무엇인가 — 전문 지식 주입
화자는 이 부분이 영상에서 가장 중요하다고 강조한다. AI를 잘 쓰는 사람들의 공통점은 매번 긴 설명을 안 해도 결과물이 딱 자기 스타일로 나온다는 것인데, 그 비결이 Claude Skill이라고 한다. Skill을 한 줄로 설명하면 Claude에게 특정 분야 전문 지식을 심어 주는 기능. 카드뉴스 스킬을 만들면 카드뉴스 전문 에디터처럼, 뉴스레터 스킬을 넣으면 뉴스레터 전문가처럼, 영업제안서 스킬을 넣으면 영업 전문가처럼 행동한다.
> "그래서 카드뉴스 만들어 줘, 한 마디만 해도 그 분야 전문가 수준으로 딱 나오는 거예요" [02:17]
아무 설명 안 했는데 대상·문체가 전부 맞게 나오는 걸 보고 소름 돋았다고 한다 [의견].

### [02:25–02:56] Skill 설정 방법과 업종별 활용
설정 경로를 안내한다. 왼쪽 아래 프로필을 클릭하고 설정 탭 → 기능 탭으로 가면 스킬이 있고, 추가 버튼을 누르면 된다. 처음 하는 사람은 "대화로 만들기"를 추천하는데, Claude가 어떤 스킬을 만들고 싶은지 물어보고 원하는 걸 말하면 알아서 스킬 초안을 만들어 준다고. 심지어 이전에 화자가 대화했던 내용을 보고 만들 법한 스킬을 추천까지 해 줬다고 한다. 만들어진 스킬은 내 스킬에 복사하면 바로 사용 가능. 업종별로는 카페 운영이면 SNS 마케터 지식을, 영업직이면 영업제안서 전문가 지식을 심어 두는 식으로 업무마다 하나씩 만들어 두면 된다.

### [02:56–03:19] 카드뉴스 스킬 만들기 실연
Skill 기능으로 카드뉴스를 만드는 걸 실연한다. 먼저 설정 → 기능 탭 → 스킬 탭에서 플러스 버튼을 누른다. 스킬 이름은 영문으로 써야 하며 예로 "카드 뉴스"(card news)를 든다. 지침란에 뭘 쓸지 모르겠으면 Claude에게 "카드뉴스 제작할 때 좋은 지침 만들어 줘"라고 물어 그 답을 그대로 복사해 붙여넣으면 된다고. 이것이 바로 Claude에게 카드뉴스 전문 지식을 주입하는 과정이라고 설명한다.

### [03:19–03:50] 카드뉴스 결과물과 수정·재사용
스킬을 저장한 뒤 새 채팅에서 "2026년 최신 AI 활용해서 업무시간 줄이는 비즈니스 팁 카드뉴스 만들어 줘"라고만 입력했더니, 스킬을 쓰라는 말도 안 했는데 알아서 스킬을 확인하고 시작했다고 한다. 결과물은 표지 제목, 본문 세 장, 마무리까지 한 번에 나왔고 디자인을 하나도 안 했는데 완성 형태로 나온다. 아쉬운 부분은 "이 부분 좀 더 간결하게 바꿔줘"처럼 말하면 바로 수정된다. 캡처해서 인스타그램이나 블로그에 바로 올릴 수 있으며, 스킬을 한 번 잘 만들어 두면 다음부터는 주제만 바꿔 던지면 된다(신메뉴 소개, 매장 할인 이벤트 홍보 등).

### [03:50–04:24] Claude Code — 비개발자 맛보기
마지막으로 요즘 개발자들 사이에서 가장 핫한 Claude Code를 소개한다. 코딩을 전혀 몰라도 말로 시키면 Claude가 코드를 직접 짜 주고, 앱이나 웹사이트도 코딩 모르는 사람이 만들 수 있다고. 브라우저에서 바로 써 보는 맛보기 버전이 있어 "이런 기능 만들어 줘"라고 요청하면 만들어 준다. 진짜 Claude Code는 터미널 기반인데, 터미널을 몰라도 괜찮고 "컴퓨터에 있는 명령어 입력창"이라고 보면 된다고 설명한다. 여기서는 내 컴퓨터 파일 전체를 읽고 여러 파일을 동시에 수정하고 실행까지 해 준다. macOS에 최적화되어 있어 개발자가 맥을 쓰는 이유 중 하나라고 하며(주장), 화자도 이 때문에 맥북을 하나 더 살까 고민 중이라고(현재 윈도우 노트북도 보유).

### [04:24–04:53] 단점 — 토큰 소모와 플랜 선택
좋은 점만 얘기했으니 단점도 말한다며, Claude는 토큰을 많이 쓴다고 한다. 토큰은 Claude가 일을 할수록 소모되는 사용량으로, 데이터 요금제처럼 쓰면 쓸수록 닳는 것이라고 비유한다. 월 20달러 Pro 버전 사용자가 Cowork로 파일을 정리하거나 Opus로 긴 문서 작업을 하다 보면 생각보다 빠르게 한도가 찬다고. 그래서 본격적으로 업무에 쓰는 사람들은 월 100달러, 월 200달러 Max 플랜을 쓰는 경우도 많다고 한다(주장). 처음 시작한다면 Pro로 먼저 써 보고 부족하면 올리라고 추천한다.

### [04:53–05:27] 4줄 최종 정리와 마무리
오늘 핵심 네 가지를 정리한다. 첫째, Opus `4.6`은 복잡한 작업, Sonnet `4.6`은 일상 업무로 상황에 따라 골라 쓴다(여기서 Sonnet을 `4.6`으로 말해 앞의 `4.0`과 불일치 — [불명확]). 둘째, Projects 기능으로 내 전용 작업 공간을 목적별로 만들어 두면 매번 설명 안 해도 내 스타일 그대로 나온다. 셋째, Skill 기능으로 Claude에게 전문 지식을 심어 두면(카드뉴스·뉴스레터·제안서) 다음부터 주제만 바꿔 던져도 전문가처럼 뽑아 준다. 넷째, Claude Code는 코딩 몰라도 말로 시키면 되고 제대로 쓰려면 macOS 터미널이 최적이다. 마지막으로 스킬을 처음 써 봤다면 어떤 스킬을 만들었는지 댓글로 남겨 달라며 구독·좋아요를 요청하고 마무리한다.
