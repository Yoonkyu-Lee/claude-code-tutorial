# PPT 딸깍 한 번에 끝냈습니다… 진짜로요

> 정리 방식: 시간순 심층 워크스루 (paraphrase) + 주제별 상세 요약
> 검증 배너: 영상 말미에 유료 강의/추가 자료 링크(설명란) 유입 구조가 있음. 성능·품질 관련 서술은 화자 주장이며 미검증.

## 요약 (주제별)

### 핵심 주장 — AI가 못 하는 게 아니라 "기준"을 안 줬을 뿐
- 화자는 "강의자로 PPT 만들어 줘, 디자이너 링크 참고해 줘" 프롬프트 한 줄로 PPT를 완성했다고 주장한다(주장/미검증). 수정 시도·색상 지정·레이아웃 설명을 전혀 하지 않았다고 함.
- 다만 "아무 AI나 되는 건 아니고 세팅이 돼 있어야 된다"가 전제. [의견] AI는 결과물을 못 만드는 게 아니라, 사용자가 기준을 안 줬던 것이다.

### 문제 진단 — 대부분이 AI를 잘못 쓰는 방식
- 흔한 사용법: ChatGPT 열고 "사업 계획서 PPT 만들어 줘" → 흰 배경 검은 글씨, 불릿 나열의 "초등학교 조별 과제 수준" 결과물.
- 그래서 "색상 바꿔 줘" → 파란색, "더 고급스럽게" → 반복. 딸깍 한 번이 아니라 10~20번 왕복하게 되고, [의견] 그럴 바엔 직접 만드는 게 더 빠르다.
- 사람들은 "AI는 아직 멀었어, PPT는 못 맡겨"라고 결론 내리지만, [의견] 그건 AI가 못 하는 게 아니라 잘못 쓰는 것이다.

### 비유 — 매일 기억을 잃는 비서
- AI를 매일 아침 출근하면 기억이 리셋되는 비서에 비유. 사장이 누군지, 회사가 뭘 하는지, 보고서 양식이 뭔지 매일 다시 설명해야 한다.
- ChatGPT든 Claude든 매번 새 대화를 시작하면 AI는 아무것도 모르므로, 매번 프롬프트로 처음부터 설명하고 빠뜨리는 부분 때문에 결과물이 들쭉날쭉하다.
- 해법: "출근하면 자동으로 읽는 업무 매뉴얼"을 만들어 주면 된다. 그게 바로 Skills.

### Skills란 무엇인가 (개념)
- Skills = AI에게 미리 세팅해 두는 "전문가 매뉴얼". PPT는 이 규칙으로, 보고서는 이 형식으로, 이메일은 이 톤으로 — 전문 지식을 파일로 만들어 두면 이후 AI가 알아서 참고한다.
- 매번 긴 프롬프트를 쓸 필요 없이 "PPT 만들어 줘" 딸깍이면 끝.
- 이 Skills가 들어 있는 도구가 Claude Code이며, Anthropic이 만든 도구라고 소개. PPT·워드·엑셀·PDF 전용 스킬이 이미 설치돼 있거나 사용자가 설치할 수 있고, 내장 스킬만으로도 프로 수준이며 나만의 스킬 추가도 가능(주장).

### PPT 스킬 내부 — 색상
- 일반 AI는 "색상 골라 줘"에 파란색, 헬스케어면 초록, 금융이면 네이비를 내놓는데 그것도 잘 고른 케이스이고, 대다수는 그라데이션 떡칠의 "초딩 수준" 퀄리티가 나온다(주장).
- 이 스킬 안에는 큐레이션한 18가지 색상 팔레트가 들어 있음(주장). 다만 팔레트를 전부 쓰는 게 아니라, 화자는 입력 시 무조건 레퍼런스 이미지를 넣는다.
- 화자는 Pinterest에서 프레젠테이션/디자인/템플릿 이미지를 구해 명령 시 꼭 첨부하도록 세팅해 둠. 그러면 글씨·배치·컬러감이 완벽하게 조화된 결과가 나온다고 주장.

### PPT 스킬 내부 — 타이포그래피
- 일반 AI: 제목 24pt, 내용 18pt 식으로 시각적으로 안 좋게 만든다(주장).
- 화자 스킬: 72pt 헤드라인 + 11pt 본문을 꼭 쓰도록 규칙화(주장). 숫자가 나오면 모노스페이스 폰트 자동 적용, 대문자 제목에는 자간을 넓게 벌리는 것까지 규칙화돼 있음.

### PPT 스킬 내부 — 레이아웃
- 일반 AI는 전부 50:50 정중앙 배치. 화자 스킬은 비대칭(3:7, 4:6, 25:75)을 사용.
- 차트가 있으면 텍스트 옆에 나란히 배치, 배경 색상 블록은 슬라이드의 40~60%를 채우도록 처리. 이걸 사용자가 일일이 지시하는 게 아니라 스킬에 다 적혀 있어서 자동 적용.

### PPT 스킬 내부 — 자가 검수(화자가 가장 강조)
- 화자가 가장 세심하게 만든 부분이라고 강조: PPT를 만들고 나면 AI가 스스로 체크하도록 처리. 텍스트 잘림·겹침 여부, 색상 대비 충분 여부, 브랜드 컬러 적용 여부, 이상한 색 유입 여부 등을 검수하는 로직.
- 덕분에 사용자가 크게 검수할 필요가 없다(주장). [의견] 사람한테 PPT 맡겨도 이 정도 검수 안 하는 경우가 많다. 한 번 세팅하면 딸깍 한 번에 제작부터 검수까지 자동.

### 확장 활용과 세일즈 구조
- PPT 스킬은 이미 많은 유저가 만든 케이스가 있어 인터넷에서 다운로드해 쓰면 된다(주장).
- 예: 매주 같은 형식의 주간 보고서를 AI에 시키면 매번 형식이 달라 고쳐야 하지만, 스킬로 만들면 일정한 결과물이 나온다. 파일 저장해 두면 이후 "주간 보고서 작성" 딸깍에 규칙이 전부 자동 적용.
- 코딩 불필요 — 마크다운이라는 간단한 텍스트 파일로 스킬을 작성. 한 번 만들면 계속 쓰고 팀원과 공유도 됨. 같은 프롬프트를 3번 이상 쓰고 있다면 스킬로 만들 타이밍. 적용 대상: PPT, 보고서, 이메일, 제안서, 회의록, SNS 콘텐츠.
- 더 자세한 내용(스킬 세팅법, 나만의 스킬 만드는 법, 업무별 활용 템플릿)은 설명란 링크로 유도(유료/추가 자료 유입 구조).

### 마무리 3줄 정리 및 주의
- 하나: AI가 결과물을 못 만드는 게 아니라 우리가 기준을 안 줬던 것. 스킬즈로 기준을 세팅하면 딸깍 한 번에 전문가급 결과물(주장).
- 둘: 반복 업무는 전부 스킬로. 한 번 만들면 끝, 긴 프롬프트를 매번 복붙하는 시대는 끝났다.
- 셋(주의): 그래도 최종 결과물은 한 번 확인하라. AI가 검수까지 해 주지만 숫자·날짜·고유명사는 사람이 체크하는 게 안전하다.
- [의견] 프롬프트를 잘 쓰는 시대는 끝났고, 이제는 세팅을 잘하는 사람이 이기는 시대.

## 3줄 오리엔테이션
- AI가 PPT를 못 만드는 게 아니라, 매번 새 대화마다 "기준"을 안 줘서 결과물이 들쭉날쭉하다는 진단으로 시작한다.
- 해법은 Skills — AI가 자동으로 읽는 "전문가 매뉴얼"(마크다운 파일)을 미리 세팅해, 색상·타이포·레이아웃·자가검수 규칙을 딸깍 한 번에 자동 적용하는 것.
- Claude Code 내장 PPT 스킬의 내부(18색 팔레트·72pt/11pt·비대칭 배치·자가검수)를 뜯어 보여 주고, 반복 업무는 전부 스킬화하라는 권유와 설명란 링크(유료 자료) 유입으로 마무리한다.

## 전문 — 시간순 워크스루

### [00:00–00:21] 딸깍 한 줄 프롬프트로 PPT 완성 주장
화자는 자신이 PPT를 "강의자로 PPT 만들어 줘, 디자이너 링크 참고해 줘" 프롬프트 한 줄로 만든다고 연다(주장/미검증). 수정 시도도, 색상 지정도, 레이아웃 설명도 안 했고 "딸깍 한 번 끝"이라고 강조한다. "무슨 AI가 그렇게까지 해요"라는 반문에 "진짜 됩니다"라고 답하면서도, 아무 AI나 되는 건 아니고 "세팅"이 돼 있어야 한다고 단서를 단다. 오늘 그 세팅이 뭔지, 어떻게 하는지 전부 보여 주겠다고 예고.

### [00:21–00:57] 흔한 잘못된 사용법 — 10~20번 왕복
사실 대부분은 이렇게 쓸 것이라며 반례를 든다: ChatGPT 열고 "사업 계획서 PPT 만들어 줘" 엔터. 결과물은 흰 배경에 검은 글씨, 불릿 나열의 "초등학교 조별 과제 수준"이다. 그래서 "색상 좀 바꿔 줘" 엔터 → 파란색, "좀 더 고급스럽게" 엔터 → 그래도 만족스럽지 않다. 딸깍 한 번이 아니라 10번, 20번 왔다 갔다 하게 된다.
> "시간 아끼려고 AI 쓰는 건데 이렇게 하면 직접 만드는 게 더 빠릅니다." [00:46]

### [00:50–00:59] 잘못된 결론 vs 화자의 반박
[의견] 사람들은 여기서 "AI는 아직 멀었어, PPT는 AI한테 못 맡겨"라고 결론 내린다고 한다.
> "근데요, AI가 못 하는 게 아닙니다. 여러분이 잘못 쓰고 계신 거예요." [00:55]

### [00:59–01:37] 비유 — 매일 기억을 잃는 비서
회사에 특이한 비서가 하나 들어왔는데, 매일 아침 출근하면 기억이 리셋된다. 사장이 누군지, 회사가 뭘 하는지, 보고서 양식이 뭔지 모른다. 그래서 매일 아침 "너는 우리 회사 비서야, 보고서는 이 형식으로 써, 색상은 이걸 써, 숫자는 콤마 찍어, 모호한 표현 쓰지 마"를 반복해야 한다. [의견] 이게 지금 사용자들이 AI를 쓰는 방식이라고 한다. ChatGPT든 Claude든 매번 대화를 새로 시작하면 AI는 아무것도 모르므로, 매번 프롬프트로 처음부터 설명해야 하고 빠뜨리는 부분 때문에 결과물이 들쭉날쭉하다.

### [01:37–02:12] 해법 = Skills(업무 매뉴얼)
답은 간단하다 — 비서에게 "출근하면 자동으로 읽는 업무 매뉴얼"을 만들어 주면 된다. 매일 아침 말로 설명할 필요 없이 매뉴얼만 펴면 전부 적혀 있는 것. 그게 바로 Skills다. Skills는 AI에게 미리 세팅해 두는 "전문가 매뉴얼"로, "PPT를 만들 때는 이 규칙을 따라, 보고서는 이 형식으로, 이메일은 이 톤을 유지해라" 같은 전문 지식을 파일로 한 번 만들어 두면 이후 AI가 알아서 참고한다. 매번 프롬프트를 쓸 필요 없이 "PPT 만들어 줘" 딸깍이면 끝.

### [02:09–02:36] Skills를 담은 도구 = Claude Code
이 Skills가 들어 있는 도구가 바로 Claude Code이며, Anthropic이 만든 도구라고 소개한다. 여기에 PPT·워드 문서·엑셀·PDF 전용 스킬이 이미 설치돼 있거나 사용자가 설치할 수 있다. "내장 스킬만 돼도 이미 프로 수준"이고 여기에 나만의 스킬을 추가로 만들 수도 있다(주장; 자세한 건 뒤에서). 일단 내장 PPT 스킬 안에 뭐가 들어 있는지 직접 뜯어 보겠다며, 이걸 보면 왜 결과물이 차원이 다른지 바로 이해될 거라고 한다.

### [02:38–03:29] PPT 스킬 내부 ① 색상 — 18색 팔레트 + 레퍼런스 이미지
PPT에서 제일 먼저 눈에 들어오는 게 색상이다. 일반 AI에 "색상 골라 줘" 하면 파란색, 헬스케어면 초록, 금융이면 네이비가 나오는데 그것도 잘 고른 케이스이고, 대다수 AI는 그라데이션으로 떡칠된 "초딩 수준" 퀄리티가 나온다고 주장한다. 하지만 이 스킬 안에는 큐레이션한 18가지 색상 팔레트가 들어 있다(주장). 다만 팔레트를 전부 쓰는 건 아닌데, 화자는 입력 시 무조건 레퍼런스 이미지를 넣기 때문이다. 화자는 Pinterest에서 프레젠테이션·디자인·템플릿 이미지를 구해 명령 시 꼭 첨부하도록 세팅해 뒀고, 그러면 글씨·배치·컬러감까지 완벽하게 조화된 결과가 생성된다고 한다.

### [03:30–03:57] PPT 스킬 내부 ② 타이포그래피
일반 AI는 제목 24pt, 내용 18pt 식으로 시각적으로 안 좋게 만든다(주장). 반면 화자가 만든 스킬은 72pt 헤드라인에 11pt 본문을 꼭 쓰도록 해 놔서 더 이쁘고 퀄리티 있게 PPT를 만들 수 있게 처리했다(주장). 또한 숫자가 나오면 모노스페이스 폰트를 자동 적용하고, 대문자 제목에는 자간을 넓게 벌리는 것까지 전부 규칙화돼 있다.

### [03:57–04:20] PPT 스킬 내부 ③ 레이아웃 — 비대칭
레이아웃도 마찬가지다. 일반 AI는 전부 50:50 정중앙 배치인데, 화자의 스킬은 비대칭(3:7, 4:6, 25:75)을 사용한다. 차트가 있으면 텍스트 옆에 나란히 배치하고, 배경은 색상 블록이 슬라이드의 40~60%를 채우도록 처리한다. 이걸 사용자가 하나하나 지시하는 게 아니라 스킬에 다 적혀 있어서 자동 적용된다.

### [04:20–04:52] PPT 스킬 내부 ④ 자가 검수(가장 강조)
화자가 이 스킬에서 가장 세심하게 만든 부분은 "검수"다. PPT를 만들고 나면 AI가 스스로 체크하도록 처리해 놨다: 텍스트 잘림 없는지, 겹침 없는지, 색상 대비 충분한지, 브랜드 컬러 충분히 적용했는지, 이상한 색이 들어갔는지 등을 검수하는 로직을 넣었다. 덕분에 사용자가 딱히 크게 검수할 필요가 없다(주장).
> "솔직히 사람한테 PPT 맡겨도 이 정도 검수 안 하는 경우 많잖아요. 그게 바로 스킬의 힘입니다." [04:43]
한 번 세팅해 두면 딸깍 한 번에 제작부터 검수까지 자동 생성된다.

### [04:52–05:11] 정리 — 스킬 유무의 차이
정리하면, 스킬 없이 딸깍 열 번 "만들어 줘 → 구림, 색상 구림, 모든 게 다 구린 결과물"을 받을 것인지, 스킬 세팅 후 딸깍 한 번에 모든 결과물을 받을 것인지의 문제다. 같은 AI, 같은 모델, 같은 프롬프트이며 차이는 오직 "스킬의 세팅 여부"뿐이라고 강조한다.

### [05:11–05:43] 다운로드 가능한 스킬 + 반복 보고서 예시
PPT 스킬은 이미 많은 유저가 만들어 놓은 케이스가 있어 인터넷에서 다운로드해 사용하면 된다(주장). 예로 매주 주간 보고서를 든다: 매번 같은 형식으로 쓰는데 AI에 시키면 매번 형식이 달라 고쳐야 한다. 하지만 스킬로 만들면 같은 포맷으로 일정한 결과물이 나온다. 이 파일을 저장해 두면 다음부터 "주간 보고서 작성" 딸깍에 규칙이 전부 자동 적용된다.

### [05:43–06:03] 코딩 불필요 — 마크다운 스킬, 반복 업무 전부 스킬화
코딩은 필요 없고, 마크다운이라는 간단한 텍스트 파일로 스킬을 쓰면 된다. 한 번 만들면 계속 쓸 수 있고 팀원과 공유도 된다. 반복하는 업무가 있으면 전부 스킬로 만들라고 권한다.
> "같은 프롬프트를 세 번 이상 쓰고 있으시다면 그건 스킬로 만들 타이밍입니다." [05:55]
적용 대상으로 PPT, 보고서, 이메일, 제안서, 회의록, SNS 콘텐츠를 든다.

### [06:08–06:50] 마무리 3줄 정리 + 주의 + 세일즈 CTA
마무리로 세 가지를 정리한다. 하나, AI가 결과물을 못 만드는 게 아니라 우리가 기준을 안 줬던 것 — 스킬즈로 기준을 세팅하면 딸깍 한 번에 전문가급 결과물이 나온다(주장). 둘, 반복되는 업무는 전부 스킬로 만들어라, 한 번 만들면 끝이고 매번 긴 프롬프트를 복붙하는 시대는 끝났다. 셋, 그래도 최종 결과물은 한 번 확인하라 — AI가 검수까지 해 주지만 숫자·날짜·고유명사는 사람이 체크하는 게 안전하다. 스킬 세팅법, 나만의 스킬 만드는 법, 업무별 활용 템플릿을 더 알고 싶으면 설명란 링크를 확인하라고 안내(유료/추가 자료 유입 구조).
> "프롬프트를 잘 쓰는 시대는 끝났습니다. 이제는 세팅을 잘하는 사람이 이기는 시대예요." [06:43]
