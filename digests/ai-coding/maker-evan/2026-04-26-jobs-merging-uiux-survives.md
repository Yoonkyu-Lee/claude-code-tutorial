# 직군이 사라집니다... 5년 뒤 진짜 살아남는 단 1가지

> maker-evan · 2026-04-26 · 길이 약 9분 · [영상 링크](https://youtu.be/tvdSwKLqNTo)
> 정리 방식: 시간순 심층 워크스루 (paraphrase) + 주제별 상세 요약
> 검증 배너: 이 영상은 특정 도구(Gstack·Superpowers) 사용을 권하는 홍보성 톤을 포함하며, "일주일 걸리던 일이 하루로 / 시간이 10분의 1로 줄었다" 같은 시간 단축 수치는 모두 화자 개인 경험에 근거한 **주장(미검증)**이다.

## 요약 (주제별)

### 1. 핵심 논지 — 직군은 사라지는 게 아니라 합쳐진다
- 화자는 바이브 코딩이 개발자·디자이너·PM·PO의 경계를 빠르게 무너뜨리고 있다고 본다 [00:04].
- "개발자가 필요 없어지느냐"는 질문에 대한 답은 "필요 없다"가 아니라 **"개발자라는 정의가 바뀐다"**이다 [01:51]. [의견]
- 사라지는 것이 아니라 한 사람이 여러 역할을 흡수하는 방향으로 **통합**된다는 것이 전체 메시지 [02:48].

### 2. 세 가지 구조적 변화
- **① 기술 장벽 소멸**: 예전엔 파이썬·자바스크립트·React 훅을 1년 배워도 게시판 하나 못 만들었지만, 지금은 한국어로 "이런 기능 만들어 줘"라고 말하면 AI가 만든다. 개발 지식이라는 진입 장벽이 사라져 초등학생·마케터·디자이너·사장님도 만들 수 있다 [01:17][01:42]. [의견]
- **② 직군의 무자비한 통합**: 프론트(화면)·백엔드(서버)를 AI가 다 해주니 나눌 이유가 없고, HTML/CSS를 AI가 뽑아주니 디자이너가 따로 시안 그려 넘길 필요가 없다. PM/PO의 요구사항 정리·우선순위·기획·브레인스토밍·로드맵·사용자 분석까지 AI가 더 잘한다고 주장 [02:01][02:22][02:29]. [의견/미검증]
- **③ 결국 남는 것은 UIUX 감각 하나**: 기술이 평준화되면 비슷한 서비스가 쏟아지고, 그중 사람들은 "느낌이 좋은 것"을 쓴다. 색감·여백·폰트·인터랙션·애니메이션을 잘 잡는 사람이 이긴다 [02:56][03:19].

### 3. UIUX가 무기가 되는 이유 (경험 경쟁 논리)
- 기술이 평준화될수록 미적 감각·취향·taste·유니크함이 오히려 **더 비싸진다** [03:26]. [의견]
- 예전엔 "이 기능 만들 수 있나"가 핵심이라 만든 사람이 승자였지만, 지금은 경쟁사가 같은 기능을 한 시간 만에 따라잡는다 → 차이는 **사용자가 그것을 어떻게 느끼느냐**에서 난다 [03:47][03:53].
- 화자의 명제: "기능 100개보다 좋은 인터랙션 한 개가 더 비싸다. 기능 경쟁은 끝났고 경험 경쟁이 시작됐다" [04:04][04:09]. [의견]

### 4. 화자가 쓰는 두 도구 — Gstack · Superpowers
- **Gstack**: PO/PM 역할을 대신. 아이디어를 던지면 경쟁사 분석·핵심 기능 정리·우선순위까지 알아서 해준다. 예전 PM 회의 한 시간이면 나올 결과가 이미 꽤 괜찮은 수준의 기획안으로 나온다고 주장 [04:37][04:52][05:04]. [의견/미검증]
- **Superpowers**: Gstack에서 정리된 기획안을 들고 가 "목업 만들어 줘"라고 하면 깔끔한 디자인 시안이 바로 나온다. 이 시안은 **정적 이미지가 아니라 실제로 동작하는 HTML** — 클릭·스크롤·인터랙션이 다 살아 있어 "느낌"을 시안 단계에서 미리 검증할 수 있다 [05:24][05:40][05:48]. [주장]

### 5. 피그마 무용론
- 화자는 이제 피그마를 거의 안 쓴다. 피그마는 디자이너와 개발자 사이의 **중간 다리**였는데, 시안을 HTML로 바로 뽑으면 그 다리 자체가 사라진다 [05:58][06:37].
- 피그마 시안은 결국 개발자가 코드로 다시 만들어야 하지만, HTML로 뽑으면 그게 곧 결과물이라 중간 변환 과정이 사라져 시간이 "절반이 아니라 10분의 1로" 줄어든다고 주장 [06:41][06:50]. [의견/미검증]

### 6. 화자의 하루 워크플로 (4단계)
- ① Gstack에 "이런 서비스 만들고 싶어, 같이 정리해 줘" → 30분이면 타깃·핵심 기능·차별점이 담긴 기획안 [07:04][07:09].
- ② Superpowers에 기획안을 넘겨 "메인 화면·핵심 페이지 뽑아줘" → 한 시간이면 실제 동작하는 HTML 시안 [07:14][07:19].
- ③ 시안을 보며 색·인터랙션·폰트를 다듬음 — "이게 진짜 디자이너의 시간" [07:23][07:29].
- ④ 마음에 들면 그대로 프론트/백엔드 코드로 변환해 배포. AI와 함께하면 하루면 끝 — 예전엔 이 전체가 한두 달, 회의만 수십 번이었다는 주장 [07:32][07:39][07:44]. [의견/미검증]

### 7. 근거로 든 해커톤 경험
- 최근 48시간 해커톤에서, 디자인·컨셉 정하는 데 업무의 2/3를 쓰고 코딩엔 1/3만 썼다고 함 → 기획·디자인이 그만큼 중요해졌다는 방증으로 제시 [07:48][07:56]. [주장/개인 경험]

### 8. 결론 — 무엇을 해야 하나
- 5가지 정리: ① 코딩 장벽 소멸 ② 직군 통합 ③ 남는 건 UIUX 감각 ④ 도구가 중간 과정을 흡수 ⑤ 피그마 같은 중간 도구는 소멸 [08:07][08:22].
- "기술만 파는 시대는 끝났다. 리액트·스프링 잘한다는 걸로는 차별화 안 된다. 기본기는 필요하되 무기는 아니다" [08:24][08:30]. [의견]
- 해야 할 것: **감각을 길러라** — 좋은 디자인을 많이 보고, 좋은 서비스를 많이 써 보고, "왜 좋은지"를 계속 고민해 자기 것으로 만들기. 그리고 **도구를 영리하게** — Gstack으로 기획 빠르게, Superpowers로 시안 바로, 피그마 같은 중간 도구는 과감히 버리고 빠르게 만들고 검증 [08:33][08:41]. [의견]
- 이렇게 일하는 사람과 아직 직군 나눠 회의하고 시안 넘기는 사람 사이 격차가 어마어마해질 것이라는 경고로 마무리 [08:50].

## 3줄 오리엔테이션
- 바이브 코딩으로 기술 장벽이 사라지면서 프론트·백·디자인·PM/PO 직군의 경계가 무너지고, 한 사람이 전 과정을 처리하는 방향으로 **통합**되고 있다는 것이 핵심.
- 기술이 평준화될수록 유일한 차별화 무기는 **UIUX 감각(경험 설계)**이라는 주장.
- 화자 본인은 Gstack(기획)·Superpowers(HTML 시안)로 하루 만에 기획→배포를 끝내며, 피그마 같은 중간 도구는 버렸다고 소개 — 도구 홍보 성격이 섞인 영상.

## 전문 — 시간순 워크스루

### [00:00–00:16] 인트로 훅 — 하나만 잘해선 안 된다
프론트엔드·백엔드·디자이너·PM의 경계가 지금 진짜 빠르게 무너지고 있으며, 그 와중에 딱 하나 끝까지 살아남는 게 있다고 운을 뗀다. 이걸 모르면 큰일 나고, 영상을 끝까지 보면 앞으로 5년 동안 뭘 준비해야 하는지 명확히 잡을 수 있다고 약속한다.

### [00:18–00:34] 오늘 다룰 두 가지
진지한 얘기를 하겠다며, "바이브 코딩이 우리한테 뭘 남겼나"와 "앞으로 살아남으려면 뭘 해야 하나" 두 가지를 풀겠다고 예고한다.

### [00:26–00:54] 예전의 개발 과정 — 최소 네 명
요즘 "코딩 한 줄 못 하는데 앱 만들었다", "디자이너인데 백엔드 직접 만들었다", "PM인데 프로토타입까지 만들었다"는 얘기가 들린다고 소개한다. 예전엔 백엔드 하나 짜려 해도 서버 띄우고 DB 연결하고 API 설계하고, 프론트는 React·Vue를 고르고, 디자인은 피그마로 시안 뽑아 개발자에게 넘기는 과정에 **최소 네 명**이 필요했고 끝없는 회의와 의견 충돌이 있었다고 회고한다.
> "이 과정에 사람이 최소 네 명은 필요했어요. 회의도 끝없이 했고요." [00:50]

### [00:54–01:13] 지금은 한 사람 손에서 끝난다
바이브 코딩을 해 보면 이 모든 게 한 사람 손에서 끝난다고 말한다. 어려운 언어나 프레임워크를 깊게 몰라도, 하고 싶은 걸 말하면 AI가 다 만들어 준다. 그래서 무서운 건 **직군 자체가 흐려지고 있다**는 점 — "나는 프론트엔드/PM/디자이너야"라는 구분이 점점 의미가 없어지고, 이걸 인정 못 하면 뒤처진다고 경고한다.

### [01:15–01:43] 첫 번째 변화 — 기술 장벽이라는 단어의 소멸
예전엔 파이썬 배우고 자바스크립트 문법 배우고 React 훅 공부하다 짜증나고, 1년을 공부해도 게시판 하나 제대로 못 만들었다고 한다(본인도 헤맸다고 인정). 지금은 머릿속 아이디어를 그냥 한국말로 "이런 기능 만들어 줘, 여기 색깔 좀 바꿔 줘"라고 말하면 된다. 즉 **개발 지식이라는 진입 장벽이 사라졌다**는 뜻이고, 이제 초등학생·마케터·디자이너·사장님 누구나 만들 수 있다. [의견]

### [01:47–01:54] 개발자는 필요 없어지나 → 정의가 바뀐다
자연히 "그럼 개발자는 이제 필요 없나?"라는 질문이 따라오는데, 화자의 답은 "필요 없다"가 아니라 **"개발자라는 정의가 바뀐다"**이다. 이 차이가 진짜 중요하다고 강조한다. [의견]

### [01:56–02:40] 두 번째 포인트 — 직군의 무자비한 통합
기술 장벽이 없어지면 직군이 무자비하게 통합된다고 본다. 프론트(화면)와 백엔드(서버)의 경계는 잘하는 영역이 달라서 있었는데 둘 다 AI가 해주니 나눌 필요가 없다. 디자인도 마찬가지로 HTML/CSS를 AI가 뽑아주니 디자이너가 시안 그려 개발자가 옮기는 과정 자체가 사라진다. PM/PO가 하던 요구사항 정리·우선순위·기획도, 나아가 브레인스토밍·로드맵·사용자 분석까지 AI가 더 잘한다고 주장한다. 결국 한 사람이 기획·디자인·프론트·백엔드·배포까지 다 해버리는 일이 지금 일어나고 있다고 말한다. [의견/미검증]
> "한 사람이 다 해요. 기획하고 디자인하고 프론트 짜고 백엔드 짜고 배포까지 한 사람이 다 해 버려요." [02:34]

### [02:40–02:53] 사라지는 게 아니라 합쳐진다
지금까지 들으며 "내 직군은 사라지는 거냐"는 두려움이 들었을 텐데, **사라지는 게 아니라 합쳐지는 것**이고 그 안에서 완전히 새로운 기회가 열리고 있다고 방향을 튼다.

### [02:55–03:34] 차별화의 답 — UIUX 감각
기술이 다 평준화되면 뭘로 차별화하냐는 질문에, 화자는 답이 하나로 모인다며 **UIUX 감각**을 든다. 백엔드 만들기가 쉬워져 누구나 비슷한 서비스를 쏟아내면, 사람들은 기능이 같을 때 "느낌이 좋은 것", "감각이 살아 있는 것", "딱 봤을 때 다르네 싶은 것"을 쓴다. 색감·여백·폰트·인터랙션·애니메이션을 잘 잡아넣는 사람이 결국 이기고, 기술이 평준화될수록 미적 감각·취향·taste·유니크함이 오히려 더 비싸진다고 주장한다. [의견]
> "나만의 느낌이라는 게 진짜 무기가 되는 순간이 올 겁니다." [03:32]

### [03:36–04:11] 서비스의 본질이 바뀐다 — 경험 경쟁
백엔드 허들이 낮아졌다는 건 서비스의 본질이 바뀐다는 뜻이라고 한다. 옛날엔 "이 기능 만들 수 있나"가 핵심이라 만들 수 있는 사람이 승자였지만, 지금은 누구나 만들 수 있어 경쟁사가 같은 기능을 일주일 아니 한 시간 만에 따라잡는다. 그러면 차이를 만드는 건 **사용자가 그것을 어떻게 느끼느냐** — 버튼을 누를 때의 기분, 화면 전환의 부드러움, 처음 들어왔을 때의 인상이 모두 차별화 포인트가 된다. 그래서 "기능 100개보다 좋은 인터랙션 한 개가 더 비싸다"고 말한다. 기능 경쟁은 끝났고 경험 경쟁이 시작됐다. [의견]

### [04:11–04:22] 중간 정리 3줄
지금까지를 정리: ① 코딩 장벽이 사라졌다 ② 직군이 다 합쳐지고 있다 ③ 결국 남는 건 UIUX 감각 하나.

### [04:22–05:21] 실전 도구 ① Gstack — PO/PM 대체
이제 본인이 실제로 어떻게 일하는지 보여주겠다며, 요즘 많이 쓰는 두 도구로 **Gstack**과 **Superpowers**를 든다. Gstack은 한마디로 PO/PM 역할을 대신해 준다. 아이디어를 던지면 예전처럼 PM 회의·요구사항 정리·우선순위·로드맵을 며칠~몇 주씩 거칠 필요 없이, 함께 브레인스토밍하며 경쟁사 분석부터 핵심 기능 정리, 우선순위 판단("이건 먼저, 이건 나중")까지 해준다. 처음 써보고 충격받았다고 하며, PM 회의 한 시간이면 이미 꽤 괜찮은 수준의 기획안이 나와 있었다고 한다. 즉 PM/PO 직군의 일상 업무가 한 명의 도구로 대체 가능해졌다는 것.
