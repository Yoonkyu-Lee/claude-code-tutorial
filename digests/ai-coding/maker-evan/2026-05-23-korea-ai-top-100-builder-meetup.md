### 행사 개요 — 빌더 모임 브이로그
- 이 영상은 [불명확: 원문 "빌더 올리라는"] 프로그램을 통해 "대한민국의 빌더 100명"을 소개하는 브이로그다. 진행자(Evan)와 친구들이 어떻게 사고하고 AI를 쓰는지를 현장에서 보여 주는 것이 목적.
- 행사명은 [불명확: 원문 "빌더리의 필더레이데이"] — 첫 회차이며, 두 달쯤 전 같은 장소에서 열린 [불명확: 원문 "더 헤퍼톤"] 해커톤 행사에서 파생됐다. 그때 약 100명의 빌더가 모인 열정을 이어가려고 "두 달에 한 번씩" 정기 모임으로 시작했다(주최 측 설명).

### 진행자 Evan의 실시간 "입코딩" 시연
- Evan은 현장에서 자신의 코딩 방식을 시연한다. 대상은 "인스타 카드뉴스"라는 자신의 스킬(skill)을 **음성으로 지시해 업데이트**하는 작업.
- 개선 요청 내용: 인트로가 별로고 동기부여가 안 됨, 배경 이미지가 너무 전문적이라 공감이 안 됨 → "더 자연스러운 이미지를 만들어 넣어 줘"라고 음성 지시.
- 음성을 입력받아 처리한 뒤 엔터만 누르면 결과가 나오는 구조. 결과물(인스타 카드뉴스)은 "짬뽕하게" 나왔다고 스스로 평가 [의견]. 자신의 유튜브 내용을 기반으로 인스타 카드뉴스를 만드는 작업도 진행 중이며 추후 공개 예정.

### 이커머스 빌더 — AI 제품/모델 이미지 제작 노하우
- 한 참가자는 이커머스를 하며, [불명확: 원문 "스필드"]라는 도구로 AI 모델 컷과 제품 이미지를 만든다. 경험이 많이 쌓여 이를 **프롬프트 팩 제품으로 만들어 같은 이커머스 사장들에게 공유·판매**할지 고민 중 (세일즈 요소).
- 두 가지 접근: (1) 레퍼런스 없이 프롬프트만으로 생성, (2) 원하는 느낌을 정확히 구현하려 적절한 레퍼런스를 가져와 구현.
- 핵심 워크플로: 핀터레스트에서 원하는 감성(예: "블루 앤 그레이" 느낌)의 이미지를 찾아 저장 → **표절 금지, 느낌만 참고해 재해석·재창조하라**는 프롬프트를 넣어 제품 이미지 생성. "절대 표절하면 안 된다"는 점을 프롬프트에 명시적으로 강조.

### 이미지 생성 실전 데모 — 귀걸이 제품 컷
- 귀걸이 제품 컨셉 사진을 만드는 실연: 제품 이미지 2장 첨부 + 색감·컨셉 레퍼런스 사진 3·4번 첨부.
- 프롬프트 요지: "1·2번 사진 속 귀걸이(동일 제품)의 제품 컨셉 사진을 만들어 줘. 컨셉은 소녀스럽고 러블리하고 [불명확: 원문 "페리한/모한적인"] 느낌. 3·4번은 색감·컨셉 레퍼런스로만, 절대 표절 말고 느낌만 참고해 새 이미지를 재창조."
- 결과: 첨부한 귀걸이 이미지로 제품 사진을 "공장형으로 마구 뽑아내는" 데 성공했다고 시연 [의견].

### 이미지 모델·도구 선택 기준
- 모델 선택: 여러 이미지 모델 중 **나노바나(Nano Banana)를 가장 많이 사용** — 토큰이 적게 들고 이미지 생성이 빠르다는 이유(화자 주장/미검증).
- 플랫폼 장점: [불명확: 원문 "스필드"]는 이미지를 동시에 약 8개까지 돌려 놓을 수 있어, 속도를 위해 한 번에 4장 정도씩 뽑는다.
- 참가자는 축적한 노하우를 "정리해 둔 프로젝트"가 있다고 언급 (세일즈/자료화 요소).

### 네이버 마케터 빌더 — 영상 자막 자동화 질문
- 또 다른 참가자 김영준은 네이버 마케팅을 하며 Evan의 구독자. Evan처럼 영상을 만들고 싶어 [불명확: 원문 "스케샵"] 스킬로 유튜브 편집을 하려 함.
- 질문: 녹음한 자막을 어떻게 영상에 입히는지, 녹음을 텍스트로 뽑아 다시 넣어야 하는지.
- Evan의 답: **녹음 파일만 넣으면 된다.** 필요한 것은 OpenAI API 키 하나. 녹음 파일을 던지면 (OpenAI 키 기반으로) 음성 → 텍스트 추출 → 자막 생성이 자동으로 이뤄지는 구조.

### [03:25–04:16] 이미지 제작 노하우 — 레퍼런스와 "표절 금지" 프롬프트
Evan이 노하우를 묻자 참가자가 두 가지 방법을 설명한다. 하나는 레퍼런스 없이 프롬프트만으로 만드는 방법, 다른 하나는 원하는 느낌을 정확히 구현하기 위해 적절한 레퍼런스를 가져와 이미지를 구현하는 방법. 주로 쓰는 방식은 핀터레스트 같은 데서 원하는 감성(예: "블루 앤 그레이" 느낌)의 이미지를 찾아 저장한 뒤, **여기서 이미지를 표절하면 절대 안 되고, 이러한 느낌으로 재해석해 새로 만들어 달라**는 프롬프트를 넣어 제품 이미지를 만드는 것이라고 강조한다.
> "절대 중요한 건 여기서 이미지를 표절하면 안 돼요. 표절하지 않고 이러한 느낌으로 네가 재해석해서 새로이 만들어 줘라는 프롬프트를 넣어서." [04:00]

### [07:22–08:09] Evan의 답 — 녹음 파일 + OpenAI API 키로 자막 자동화
