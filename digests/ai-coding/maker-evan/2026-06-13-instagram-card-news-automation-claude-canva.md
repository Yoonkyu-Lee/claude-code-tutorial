# 업무 자동화 3강

> maker-evan · 2026-06-13 · 길이 약 8분 · [영상 링크](https://youtu.be/NJmuCGrQZTc)
> 정리 방식: 시간순 심층 워크스루 (paraphrase) + 주제별 상세 요약

## 요약 (주제별)

### 목표 — 무엇을 만드는가
- 클로드 코드(Claude Code)로 인스타그램 카드뉴스를 자동 생성하고, 그 결과물을 캔바(Canva)에서 다시 편집 가능한 형태로 만드는 워크플로를 처음부터 끝까지 시연한다.
- 핵심 가치: "자연스럽게" 자동으로 카드뉴스를 뽑고, 이미지 감도(퀄리티)까지 확보하면서 캔바에서 수정까지 이어지게 하는 것.

### 사전 세팅 — 계정·CLI·로그인
- **GitHub 회원가입 + 로그인**: 첫 준비물. github.com에서 가입 후 로그인.
- **GitHub CLI 설치**: GitHub CLI 사이트에서 OS에 맞는 버전(Mac이면 Mac용, Windows면 Windows용)을 다운로드해 세팅.
- **GitHub CLI 로그인**: 터미널/파워쉘을 열고 클로드 코드에게 자연어로 요청. 예시 발언: "깃허브 CLI 이용해서 로그인해줘. 웹브라우저 통해서 로그인 할 거야." 클로드가 되묻는 것들에 응하며 로그인 진행.

### Replicate API (선택 사항)
- **Replicate 계정 + API 토큰**: 어카운트 세팅 → API 토큰 발급, 빌링에서 크레딧 충전.
- 충전액: "한 5불 정도만 충전하면 충분히 많이 쓸 수 있다"고 추천 (주장/미검증).
- [의견] Replicate가 **없어도 동작에는 문제 없다.** 다만 고퀄리티 이미지를 원하면 쓰는 걸 추천. 이번 시연은 Replicate 없는 버전으로 진행.

### 프로젝트 초기화 — 폴더 + GitHub 연동
- 워크스페이스에 새 폴더 "인스타 카드뉴스"를 만들고 열기 → 워크스페이스 신뢰(trust) 클릭.
- 앞서 GitHub CLI 로그인을 해뒀으므로 GitHub가 연결된 상태. 클로드 코드에 "이 프로젝트 깃허브 연동해줘. 깃 레포지토리 초기화."를 요청 → 이 초기화가 **맨 처음 필요한 작업**.
- 화자는 권한 확인을 "건너뛰기"로 처리. 클로드가 git status 확인 후 GitHub에 연동, **퍼블릭**으로 레포 생성, 업로드까지 알아서 처리.

### 카드뉴스 생성 — 스킬 트리거
- 준비된 스킬 중 **"인스타 카드뉴스 제너레이터"** 스킬을 트리거.
- 프롬프트 예시: "카드 뉴스 만들어줘. 나는 메이커 에반이고 바이브 코딩 관련된 주제로 카드 뉴스 만들고 싶어. 관련 뉴스로 한번 만들어줘봐. 클로드 코드 스킬이면 더 좋아. 카드 여덟 장 만들어줘."
- 즉 화자 정체성(메이커 에반) + 주제(바이브 코딩/클로드 코드) + 장수(8장)를 자연어로 지정.

### 캔바 MCP 사전 설정 (누락됐던 필수 단계)
- 카드 생성 전에 반드시 해둬야 할 설정: 설정 → 커넥터 → 맞춤 설정 → 커넥터의 MCP에 **캔바 MCP**를 등록.
- 연결 후 **캔바 로그인**까지 완료해야 함. (이 로그인이 있어야 뒤에서 캔바 업로드가 동작)

### 내부 동작 방식 (파이프라인)
- 스킬이 트리거되면 클로드가 순서대로: (1) 웹에서 적절한 내용을 조사, (2) 그 내용에 걸맞는 이미지를 검색해 긁어옴("웹 이미지 소싱" 디스크립션을 읽고 관련 이미지 수집), (3) **PPT 스킬**로 카드를 PPT 포맷으로 합성, (4) 그 결과를 캔바에 업로드.
- 생성 과정에서 클로드가 **밸리데이션(검증)도 알아서** 수행. 결과 이미지는 아웃풋(output) 폴더의 슬라이드로 확인 가능(파인더로 열어봄).

### 결과·비용·마무리
- [의견] 나온 이미지가 "감도 높게 꽤" 나와 화자는 "꽤 만족스럽다"고 평가.
- 캔바 업로드가 완료되면 캔바에서 표지 등 이미지들이 **편집 가능한** 상태로 열림 → 캔바에서 뚝딱 편집.
- 비용 구조: Replicate API를 안 쓰면 **API 비용은 안 들고 구독 비용만** 나간다 (주장/미검증).
- 결론: 이 스킬로 카드뉴스 자동화가 가능하고 캔바 연동까지 가능하다.

## 3줄 오리엔테이션
- 클로드 코드 + "인스타 카드뉴스 제너레이터" 스킬 + 캔바 MCP로 인스타 카드뉴스를 자동 생성·편집하는 실습 강의(3강).
- 준비물은 GitHub(가입·CLI·로그인), 선택적 Replicate API 키, 그리고 캔바 MCP 연결·로그인.
- 파이프라인은 "웹 조사 → 이미지 소싱 → PPT 합성 → 캔바 업로드"이며, Replicate를 빼면 구독 비용만 든다는 게 핵심 세일즈 포인트.

## 전문 — 시간순 워크스루

### [00:00–00:20] 오프닝 · 이번 세션 목표
메이커 에반이 인사하며, 이번 세션 목표는 세 가지라고 밝힌다: (1) 인스타 카드뉴스를 자동으로 만드는 법, (2) 그렇게 클로드 코드로 만든 결과물을 캔바에서 수정 가능하게 만드는 법, (3) 이걸 "자연스럽게" 만드는 법. 천천히 하나하나 따라오면 된다고 안내한다.

### [00:20–00:38] 준비물 1 — GitHub 가입 + GitHub CLI 설치
가장 먼저 할 일은 회원가입이고, 그 대상은 GitHub다. GitHub 사이트에서 회원가입 후 로그인한다. 그다음 GitHub CLI 사이트로 가서, Mac이면 Mac 버전 CLI를, Windows면 Windows 버전 CLI를 다운로드해 세팅을 완료한다.

### [00:38–01:26] 준비물 2 — GitHub CLI 로그인 (클로드에게 요청)
설치 후에는 로그인이 필요하다. 터미널이나 파워쉘을 연 다음, 로그인을 클로드 코드에게 직접 부탁하면 된다. 화자가 제시한 요청 문구는 다음과 같다.
> "깃허브 CLI 이용해서 로그인해줘. 웹브라우저 통해서 로그인 할 거야." [01:11]

그러면 클로드가 이것저것 되묻고, 그 흐름을 따라가며 웹브라우저를 통해 로그인을 마치면 된다.

### [01:29–02:04] 준비물 3 — Replicate API 키 (선택)
다음 세팅은 Replicate다. Replicate에는 API 키가 필요한데, 해당 버튼을 눌러 어카운트 세팅으로 들어가 API 토큰을 발급받고, 빌링에서 크레딧을 충전한다. 화자는 "한 5불 정도만 충전하면 충분히 많이 쓸 수 있다"고 추천한다(주장/미검증). [의견] 이게 없어도 사용에는 문제가 없지만, 고퀄리티 이미지를 만들려면 쓰는 걸 추천한다고 덧붙인다. 이번 시연은 Replicate 없는 버전으로 진행하겠다고 한다.

### [02:04–02:36] 프로젝트 폴더 생성
먼저 폴더를 추가해야 한다. "다른 폴더를 추가"를 누르고 워크스페이스에서 새 폴더를 만들어, "인스타 카드뉴스"라는 폴더를 하나 생성한다. 그다음 그 폴더를 "열기"로 열면 인스타 카드뉴스 폴더가 열린다.

### [02:36–03:33] GitHub 연동 · 레포 초기화
워크스페이스에서 "인스타 카드뉴스"의 워크스페이스 신뢰(trust)를 누르고, 여기서부터 시작한다. 앞서 GitHub CLI 로그인을 해뒀으므로 GitHub가 연결된 상태다. 이제 클로드 코드에 다음을 요청한다.
> "이 프로젝트 깃허브 연동해줘. 깃 레포지토리 초기화." [03:00]

그러면 클로드가 이것저것 물으며 처리한다. 화자는 권한을 "건너뛰기"로 넘긴다. 클로드가 git status를 확인하며 GitHub에 연동하고, 퍼블릭으로 "인스타 카드뉴스" 레포를 만들면 된다. 클로드가 알아서 처리해 GitHub에 업로드까지 해준다. 이 작업이 맨 처음에 필요한 작업이다.

### [03:40–04:26] 카드뉴스 생성 스킬 트리거
다음으로 화자가 제공할 스킬의 트리거를 하면 된다. 스킬 목록에는 여러 스킬이 있는데, 그중 "인스타 카드 뉴스 제너레이터" 스킬이 있다. 이 스킬을 이용해 카드뉴스를 만들라고 요청한다. 화자가 든 프롬프트 예시는 다음과 같다.
> "카드 뉴스 만들어줘. 나는 메이커 에반이고 바이브 코딩 관련된 주제로 카드 뉴스 만들고 싶어. 관련 뉴스로 한번 만들어줘봐. 클로드 코드 스킬이면 더 좋아. 카드 여덟 장 만들어줘." [04:01–04:22]

### [04:26–04:51] 빠뜨렸던 필수 설정 — 캔바 MCP
카드를 만들기 전에 설명하지 않은 설정이 있다고 짚는다. 설정으로 들어가 커넥터의 "맞춤 설정"으로 들어간 다음, 커넥터의 MCP에 **캔바 MCP**를 설정해야 한다. 이걸 연결하고 로그인까지 해야 한다. 즉 캔바 로그인이 필요하다.

### [04:51–05:41] 내부 동작 방식 설명
스킬이 트리거되면 카드를 PPT 포맷으로 만들고, 그 PPT 포맷 카드를 캔바에 업로드하는 과정을 거친다. 기다리면 결과가 나온다. 클로드가 생각을 하며 "웹 이미지 소싱"이라는 디스크립션을 읽고 관련 이미지를 가져온다. 동작 방식을 간단히 정리하면: 먼저 웹에서 적절한 내용을 조사하고, 그 내용에 걸맞는 이미지들을 검색해 긁어온 뒤, 그것을 PPT 스킬로 합성하고, 다시 캔바에 업로드하는 과정을 거쳐 처리한다.

### [05:41–06:16] 생성 결과 확인
카드가 나왔다. 적절한 이미지를 가져오고, 클로드가 밸리데이션(검증)도 알아서 수행한다. 이런 식으로 이미지가 생성된다. 아웃풋(output) 폴더에서 슬라이드를 확인하고, 파인더로 열어보면 이미지들이 나온다. [의견] 이미지들이 "감도 높게 꽤" 나와서 화자는 "꽤 만족스럽다"고 한다.

### [06:20–07:31] 캔바 업로드 · 편집 가능 확인
캔바로 업로드해달라고 하면 클로드가 캔바로 업로드해주고, 캔바에서 편집이 가능해진다. 잠시 기다리면(약 07:18에) 캔바 업로드가 완료된다. 열어서 보면 표지 등 편집 가능한 이미지들이 나오는 것을 확인할 수 있다.

### [07:31–08:04] 비용 정리 · 마무리
이 스킬을 쓰면 캔바에서 뚝딱 인스타 카드뉴스를 편집할 수 있다. 이 모든 걸 하는 데 Replicate API를 안 써서 API 비용은 안 들고, 구독 비용만 나간다고 보면 된다(주장/미검증). 이 스킬을 이용하면 손쉽게 인스타 카드뉴스를 만들고 편집할 수 있다는 게 결론이다. 오늘 클래스는 여기서 마무리하며, 이런 식으로 카드뉴스 자동화가 가능하고 캔바 연동도 가능하다고 정리한다. 들어줘서 감사하다는 인사로 끝맺는다.
