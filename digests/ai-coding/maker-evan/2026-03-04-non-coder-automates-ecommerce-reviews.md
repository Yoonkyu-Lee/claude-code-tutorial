## 요약 (주제별)

### 사례 개요 — 누가, 무엇을 만들었나
- 주인공은 패션 커머스를 하는 사업가 지인. 코딩 경험이 전무하고 터미널 창도 처음 보는 사람이다 (주장).
- 클로드 코드를 **4시간 배우고**, 그날 밤새 혼자 작업해 **일주일 만에** 자동화 프로그램을 완성했다 (주장/미검증).
- 만든 것: 에이블리·지그재그 후기를 카페24(자사몰)에 연동하고, 나아가 쇼피파이·큐텐 등 해외 채널에 각 나라 언어로 번역해 올리는 프로그램.

### 문제 배경 — 왜 필요했나
- 패션 커머스는 에이블리·지그재그 같은 플랫폼에서 장사하는데, 거기 쌓인 고객 후기가 구매 전환율에 큰 영향을 주는 소중한 자산이다.
- 이 후기를 자사몰(카페24)에도 똑같이 올리고 싶지만, 카페24에서 쓰는 CRM(고객 관리 툴)이 **자동 연동 기능을 제공하지 않는다.**
- 결국 손으로 복사·붙여넣기 해야 함: 에이블리에서 후기 복사 → 카페24에 붙여넣기 → 이미지 따로 저장해 업로드. 하루 수십 개면 이것만 하다 하루가 다 간다 (주장).
- 개발자에게 자동화를 맡기면 기능 범위에 따라 견적이 크게 나와서, 그동안 포기하고 있었다.

### 4시간 동안 가르친 것 — 핵심 4가지
1. **어떻게 말하는지(프롬프트 구체성)**: "에이블리 후기 자동화해 줘"처럼 두루뭉술하게 말하면 안 되고, "에이블리에서 후기 목록을 불러와 각 후기의 내용·별점·이미지 URL을 추출한 뒤 카페24 API 형식으로 변환해 업로드해 줘"처럼 구체적으로 지시해야 한다. 화자가 꼽은 가장 중요한 핵심.
2. **문서 찾는 법**: 클로드 코드가 만능이지만 카페24 API 같은 건 공식 문서가 있어야 정확히 만든다. API 문서를 찾아 "이 문서 참고해서 만들어 줘"라고 하는 방법.
3. **오류 대응**: 초보가 가장 많이 막히는 지점. 오류 메시지가 뜨면 포기하지 말고 그 메시지를 그대로 클로드 코드에 복사해 "이 오류 떴는데 어떻게 해?"라고 물으면 된다.
4. **스킬(skill)**: 반복 작업을 안정적으로 자동화하려면 일정한 워크플로우를 돌려야 하고, 그걸 가능케 하는 게 스킬이다. 이 사람은 스킬로 반복 작업을 전부 자동화했다.

### 실전 진행 — 확장 순서
- 시작은 배운 대로 **에이블리 후기 수집**: "에이블리 후기 데이터(내용·별점·이미지 포함) 가져오는 파이썬 스크립트 만들어 줘" → 코드 생성 → 실행 시 오류 → 오류 복사해 재질문, 몇 번 반복하니 작동.
- 스스로 확장: 에이블리뿐 아니라 **지그재그 버전**도 제작.
- **카페24 업로드**: 카페24 API 문서를 찾아 클로드 코드에 던져 후기 업로드 기능 완성.
- **해외 확장(화자가 가장 놀란 부분)**: 국내 후기를 해외 고객에게도 보여주려 한국어 후기를 영어·일본어·중국어로 번역해 올리는 기능 추가. 쇼피파이는 영어, 큐텐 일본 사이트는 일본어로 자동 처리.
- **상세 페이지 번역**: 한국어 상세 페이지를 각 나라 언어로 번역해 여러 사이트에 동시에, 그것도 병렬로 올리게 만듦.

### 비용·규모 대비 (주장/미검증)
- 화자 추산: 에이블리·지그재그·카페24 후기 연동만 해도 개발 견적 수백만~수천만 원. 다국어 자동 번역, 쇼피파이·큐텐 연동, 상세 페이지 다국어 번역은 추가 비용. 유지보수는 별도.
- 전체를 외주 주면 규모에 따라 다르지만 "몇백에서 몇 천짜리"라고 화자는 본다.
- 반면 이 사람은 **클로드 코드 구독료 100달러(주장)**를 내고 일주일 만에 전부 만들었다.
- 모든 금액·시간 수치는 화자 주장이며 미검증. 화자 스스로 "규모에 따라 다르다"는 단서를 반복해서 달았다.

### 핵심 주장 — 비개발자·사업가가 오히려 유리하다 [의견]
- [의견] AI 코딩 툴은 개발자만 쓰는 게 아니라, 오히려 비개발자·특히 사업가에게 파괴력이 더 크다.
- [의견] 개발자는 이미 코딩할 줄 알아서 빨라지긴 해도 그걸 사업화할 줄은 모른다. 반면 비개발자 사업가는 자신이 뭐가 필요한지, 어떤 문제를 풀어야 하는지, 업무 흐름이 어때야 하는지를 누구보다 잘 안다.
- [의견] 지금까지는 "어떻게"를 몰라 못 만들었는데, 클로드 코드가 그 "어떻게"를 대신해 준다. 업무 맥락을 가장 잘 아는 사람이 직접 만드니 외주보다 오히려 자기 업무에 더 딱 맞는 결과가 나온다.

### 현실적 조언 — 시작하는 법
- [의견] 마법처럼 쉽지는 않다. 처음엔 분명히 막히고, 오류 메시지에 당황하고, 제대로 돌아가는 건지 모르겠는 순간이 온다 — 그게 정상이다.
- 중요한 건 막혔다고 포기하지 않는 것. 오류 나면 클로드 코드에 묻고, 또 막히면 또 묻는 반복이 전부다.
- 시작 방법: 지금 업무 중 가장 귀찮고 반복적인 걸 떠올려, "나 이런 업무가 있는데 자동화하려면 어디서부터 시작해야 해?"라고 구체적으로 설명하면 클로드 코드가 방향을 잡아준다.

### 큰 그림 [의견]
- [의견] 비즈니스에서 기술 장벽이 무너지고 있다. 예전엔 아이디어가 있어도 개발자 없이는 못 만든다는 게 상식이었지만, 이제 비즈니스 맥락을 아는 사람이 AI를 도구로 써 자기 문제를 스스로 해결하는 시대로 가고 있다.
- [의견] 이 흐름은 점점 빨라지고, 할 줄 아는 사람과 모르는 사람 사이 격차가 크게 벌어질 것이다. 개발자가 되자거나 코딩을 배우자는 게 아니라, AI를 도구로 써 자기 문제를 해결하는 "AI 네이티브"가 되자는 것.

### 세일즈/CTA 구조
- 직접적인 유료 강의·단톡방 유입 장치는 명시되지 않음. 마무리 CTA는 댓글 유도: "업무에서 제일 자동화하고 싶은 게 뭔지 댓글에 적어달라, 다 읽어보고 다음 영상 소재로 쓸 수도 있다."

## 3줄 오리엔테이션
- 코딩 무경험 패션 커머스 사업가가 클로드 코드를 4시간 배우고 후기 연동+다국어 번역 자동화를 스스로 완성했다는 실제 사례담(수치는 화자 주장/미검증).
- 화자가 4시간에 가르친 건 딱 4가지 — 구체적으로 말하기, API 문서 참고시키기, 오류 복붙해 물어보기, 스킬로 워크플로우 자동화.
- 핵심 메시지: AI 코딩은 개발자보다 업무 맥락을 아는 비개발자 사업가에게 더 파괴력이 크고, 지금이 "AI 네이티브"가 될 때라는 것.

## 전문 — 시간순 워크스루

### [00:00–00:26] 훅 — 코딩 모르는 사업가가 4시간 만에 자동화
매일 반복하는 업무 중 "이거 자동화됐으면" 싶은 게 있지만 "나 개발자 아니잖아"라며 포기했을 거라고 시청자에게 말을 건다. 오늘은 그 생각을 완전히 깨뜨린 실제 사례를 가져왔다며, 코딩 한 줄 모르는 사업가 지인이 클로드 코드를 딱 4시간 배우고 하루 만에 자동화 프로그램을 만들었다고 소개한다. 만든 것은 에이블리·지그재그 후기를 자사몰에 연동하고, 나아가 쇼피파이·큐텐에 각 나라 언어로 번역해 올리는 프로그램. 외주 맡기면 몇백만~몇천만 원짜리 작업(주장/미검증)이라며 어떻게 가능했는지 보자고 연다.

### [00:26–00:54] 주인공 소개 — 코딩 경험 전무, 터미널도 처음
주인공은 패션 커머스를 하는 사업가. "클로드 코드 그게 뭐야? 유용하다던데 나도 쓸 수 있어?"라며 화자에게 연락해 왔다. 화자는 처음엔 걱정했다고 한다 — 이 사람은 코딩 경험이 전무하고 터미널 창도 처음 보는 사람이었기 때문. 하지만 걱정은 완전히 틀렸다. 4시간 세션이 끝나고 일주일 뒤 "나 다 만들었어"라는 카톡이 왔고, 보내준 걸 보고 할 말을 잃었다고 한다.

> "4시간과 끝나고 일주일 뒤에 카톡이 왔는데 나 다 만들었어 이러더라고요." [00:46]

### [00:54–01:39] 문제 배경 — 카페24가 후기 자동 연동을 안 해준다
본격 사례 전에 왜 이걸 만들어야 했는지 배경을 설명한다. 패션 커머스는 에이블리·지그재그 같은 플랫폼에서 장사하는데, 거기 쌓이는 후기("디자인 이뻐요", "배송 빨라요" 같은)가 구매 전환율에 큰 영향을 주는 소중한 자산이다. 이 후기를 자사몰인 카페24에도 똑같이 올리고 싶지만, 카페24에서 쓰는 CRM(고객 관리 툴)이 자동 연동 기능을 주지 않는 게 문제다. 그래서 손으로 복사해야 한다 — 에이블리 들어가 후기 복사, 카페24 들어가 붙여넣기, 이미지도 따로 저장해 업로드. 하루에 후기가 수십 개면 이것만 하다 하루가 간다(주장). 개발자에게 자동화를 맡기면 기능 범위에 따라 견적이 나오는데, 견적 한번 받아보면 안다며 — 그래서 그냥 포기하고 있었다고 한다.

### [01:39–02:03] 4시간에 가르친 것 (1) 구체적으로 말하기
화자는 클로드 코드 사용법을 깊게 가르친 게 아니라 핵심 딱 4가지만 가르쳤다고 한다. 첫째는 "어떻게 말하는지". 클로드 코드에 일을 시킬 때 두루뭉술하게 "에이블리 후기 자동화해 줘"라고 하면 안 된다. 대신 "에이블리에서 후기 목록을 불러와 각 후기의 내용·별점·이미지 URL을 추출한 다음, 카페24 API에 맞는 형식으로 변환해 업로드해 줘"처럼 구체적으로 말해야 한다는 것 — 이게 핵심이라고 강조한다.

### [02:03–02:16] (2) 문서 찾는 법
둘째는 문서 찾는 법. 클로드 코드가 만능이긴 하지만 카페24 API 같은 건 공식 문서가 있어야 정확하게 만들어진다. 그래서 API 문서를 찾아 클로드 코드에게 "이 문서 참고해서 만들어 줘"라고 하는 방법을 알려줬다고 한다.

### [02:16–02:27] (3) 오류 났을 때 당황하지 않기
셋째는 오류가 났을 때 당황하지 말라는 것. 처음 쓰는 사람이 가장 많이 막히는 지점이다. 오류 메시지가 뜨면 "아 망했다" 하고 포기하는데, 그 오류 메시지를 그냥 클로드 코드에 복사해 "이 오류 떴는데 어떻게 해?"라고 물으면 된다.

### [02:27–02:40] (4) 스킬(skill)
넷째는 스킬. 반복 작업을 안정적으로 자동화하려면 일정한 워크플로우를 돌려야 하는데, 그걸 가능케 하는 게 바로 스킬이라고 설명한다. 이 사람은 스킬을 이용해 반복 작업을 전부 자동화했다.

> "반복 작업을 자동화를 성적으로 하기 위해서는 일정한 워크플로우를 돌려야 한다고 제가 말씀드렸잖아요. 그걸 가능케 하는게 바로 스킬입니다." [02:28]

### [02:40–03:06] 다음 날 — 밤새 혼자 만든 과정
그날 밤 이 사람은 집에 가서 혼자 밤새 만들었다. 시작은 배운 대로 에이블리 후기 수집. 클로드 코드에 "에이블리 후기 데이터 가져오는 파이썬 스크립트 만들어 줘, 후기·내용·별점·이미지 다 포함해서"라고 하니 코드를 뚝딱 만들어 줬다. 실행하니 오류가 났고, 오류 메시지를 복사해 다시 물어보기를 몇 번 반복하니 돌아가기 시작했다. 여기서 스스로 생각을 확장해 — 에이블리만이 아니라 지그재그도 해야 한다며 지그재그 버전도 만들었다.

### [03:06–03:19] 카페24 업로드까지
이제 카페24에 올리면 되는 거니까 API 문서를 찾아봐야겠다고 판단, 카페24 API 문서를 찾아 클로드 코드에 던져줬다. "이 문서 참고해서 후기 업로드 기능 만들어 줘" 하니 만들어졌고, 이게 돌아가기 시작하니 여기서 더 나아갔다.

### [03:19–04:01] 해외 확장 — 다국어 번역 병렬 처리
화자가 진짜 놀란 부분. 이 사람은 쇼피파이·큐텐 같은 곳에서 해외 판매도 하는데, 국내 후기가 좋으니 해외 고객에게도 보여주고 싶었다. 문제는 한국어 후기를 영어·일본어·중국어로 번역해 올려야 한다는 것 — 손으로 하면 구글 번역기로 하나하나 번역하거나 채팅형 툴에 복붙해야 한다. 그런데 이 사람은 클로드 코드에 그냥 "이 한국어 후기를 자동으로 번역해 쇼피파이에 올리는 스킬도 추가할 수 있어?"라고 물었고, 그냥 만들어졌다. 쇼피파이는 영어로, 큐텐 일본 사이트는 일본어로 자동 처리. 상세 페이지도 한국어를 각 나라 언어로 번역해 여러 사이트에 동시에, 그것도 전부 병렬로 돌아가게 만들었다.

> "쇼피파이는 영어로, 큐1 일본 사이트는 일본어로, 자동으로요. … 그것도 다 병렬로 돌아가게 동시예요." [03:51]
