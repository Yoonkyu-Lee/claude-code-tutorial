## 요약 (주제별)

### 코워크가 무엇인가 — 개념과 패러다임 전환
- 코워크(Cowork)는 "함께 일하기(협업)"라는 뜻으로, 클로드를 채팅 상대가 아니라 진짜 동료처럼 쓸 수 있게 만든 것이라고 소개한다. 영어로는 **에이전틱 워크스페이스(agentic workspace)**라 부른다.
- 기존 방식은 "묻고 답하는 핑퐁/캐치볼" 채팅이었다면, 코워크는 **결과물을 말해주면 클로드가 알아서 만들어 내는** 구조다. 과정을 일일이 안 봐도 되고, 일을 던져 놓고 다른 일 하다가 돌아오면 결과가 나와 있다.
- 화자는 이를 "우리 역할이 실행자(도구를 직접 다루는 사람)에서 **디렉터(결과를 정의하고 위임하는 사람)**로 바뀌는 것"이라 정리한다. [의견] 코워크는 단순 기능 모음이 아니라 "일하는 방식(패러다임) 자체를 바꾸는 것".

### 코워크가 푸는 기존 AI의 3가지 답답함
- **다단계 작업 불가**: 보고서 하나도 자료 찾기→정리→분석→작성→결론이 한 호흡에 안 끝나서, 매번 다음 단계를 시키며 옆에 붙어 있어야 했다("인턴 옆의 사장님").
- **기억 상실**: AI가 세션마다 이전 대화를 잊어("매일 아침 어제 일을 잊고 출근하는 신입").
- **로컬 파일 접근 불가**: 파일을 열어 복사→채팅에 붙여넣기→답변 다시 파일로 붙여넣기의 반복. 화자는 이 불편으로 AI를 포기하는 사람도 있고 자신도 그랬다고 말한다. Anthropic이 이를 알고 코워크를 만들었다는 설명.

### 핵심 4대 기능
- **① 로컬 파일 관리 (가장 중요)**: 클로드가 내 컴퓨터 파일을 직접 읽고 쓴다. 업로드/다운로드 불필요, 폴더만 알려주면 됨. 화자는 "외주 직원 → 옆자리 직원으로 들어온 것"에 비유. 보안은 폴더 단위 권한 관리로 통제되며, 허용 안 한 폴더는 못 보고 중요 작업 시 매번 확인을 받는다.
- **② 크롬 연동**: 'Claude in Chrome'과 짝을 이뤄 클로드가 웹브라우저를 대신 조작(채용 공고·경쟁사 가격 페이지를 클릭·읽어 표로 정리). 로컬 파일 + 크롬이 만나면 "웹에서 자료 가져와 내 파일로 정리"가 완전 자동화. 다만 잘못 클릭 시 결제 진행 등 위험이 있어 민감 작업은 확인을 받고, **10분 내 미응답이면 실행하지 않고 넘어간다**.
- **③ 병렬 작업**: 복잡한 일을 클로드가 작은 일로 쪼개 동시에 여러 개 처리해 시간 단축("셰프 여러 명이 코스요리를 동시에"). 별도 설정 없이 자동.
- **④ 전문 산출물 생성 (사람들이 제일 좋아함)**: 엑셀·파워포인트·워드를 생성. 엑셀은 표뿐 아니라 SUM·VLOOKUP·피벗 테이블 등 **함수·차트까지 실제 작동**. PPT는 목차·슬라이드 내용·디자인까지 어느 정도 잡아줌. [의견] "글 쓰는 비서 → 업무를 아는 비서로". 화자 주장: 코워크가 80%를 만들면 사람은 20%만 손보면 된다(주장/미검증).

### 디스패치 (원격 조작)
- 핸드폰으로 집 컴퓨터에 일을 시키는 원격 조작 기능. 집 데스크톱에서 클로드 데스크톱 앱을 켜 두면, 외출 중 모바일 앱으로 일을 던지고 실제 작업은 데스크톱에서 돌며 결과를 핸드폰으로 확인.
- 데스크톱에서 돌리는 이유 2가지: (1) 컴퓨터 성능을 그대로 사용(핸드폰보다 빠르고 안정적), (2) 데스크톱에 등록된 폴더·도구·서비스를 그대로 활용. "데스크톱이 일터, 핸드폰은 리모컨".
- 조건: **프로/맥스 구독** + 맥이나 윈도우 최신 클로드 데스크톱 앱. 민감 작업은 알림으로 확인받고 **10분 미응답 시 자동 취소**.

### 프로젝트 (영역별 워크스페이스)
- 특정 영역 작업을 위한 연구 워크스페이스("부서별 사무실"). 한 번 세팅하면 그 프로젝트에 들어갈 때마다 자동 적용되어 컨텍스트 전환이 빨라진다.
- 각 프로젝트에 들어가는 **5가지 구성요소**: (1) **설명** — 무엇을 하는 프로젝트인지. 디스패치가 작업을 어느 프로젝트로 보낼지 판단할 때 참고. (2) **폴더** — 클로드가 읽고 쓸 로컬 디렉토리, 파일당 최대 50MB(주장/미검증), 여러 개 등록 가능. (3) **지시(instructions)** — 매 세션 자동 적용되는 기본 안내(예: 한국어로 답변, 코드는 TypeScript, 보고서 템플릿). "회사 매뉴얼" 역할. (4) **링크** — 참고 URL. (5) **메모리** — 세션이 끝나도 사라지지 않는 프로젝트 단위 기억 저장소. 클로드가 배운 것·발견한 것을 계속 저장.
- 세션 시작 시 프로젝트를 선택하면 폴더 자동 마운트 + 지시 자동 적용 + 산출물이 그 폴더에 저장 + 쌓인 지식이 메모리에 자동 축적.
- **코워크 프로젝트 ≠ 클로드 AI 프로젝트**: 클로드 AI 프로젝트는 계정/클라우드에 있어 어디서든 접속·팀 공유 가능하지만 로컬 폴더를 못 본다. 코워크 프로젝트는 로컬에만 존재해 로컬 폴더를 다 보지만 다른 컴퓨터에선 못 본다. 둘은 병합되지 않으나, 클로드 AI 프로젝트를 코워크에 링크해 함께 쓸 수는 있다.
- 팁: (1) 첫 세팅에 시간 투자(지시를 꼼꼼히). (2) 디스패치와 함께 쓰기. (3) 메모리를 활용·점검(잘못된 정보는 수정). (4) 아카이브 활용 — 아카이브하면 메타데이터만 사라지고 로컬 파일은 유지, 나중에 다시 꺼낼 수 있음.

### 확장 기능 — 커넥터 / 스킬 / 플러그인
- **커넥터**: **MCP(Model Context Protocol)** 표준으로 클로드를 외부 도구·데이터 소스에 연결. 화자는 MCP를 "AI 세계의 콘센트 규격(한국 220V처럼 한 번 정하면 어떤 도구든 꽂힘)"에 비유. 예: 노션(페이지 읽고 쓰기), 슬랙(메시지), GitHub(코드 저장소). "회의록 보고 액션 아이템을 리니어 티켓으로" 같은 자동화 가능.
- **스킬(Skills)**: 특정 작업을 어떻게 처리할지 알려주는 "매뉴얼". `SKILL.md` 파일 + 보조 스크립트·리소스가 든 폴더. 항상 활성화가 아니라 처음엔 메타데이터만 읽고, 해당 작업이 들어오면 그때 전체를 불러오는 **점진적 공개(progressive disclosure)** 방식 — 효율 때문.
- **플러그인**: 커넥터·스킬·에이전트·훅을 묶은 패키지("핸드폰 앱"에 비유). 에이전트는 대신 작업하는 서브 에이전트, 훅은 특정 시점에 자동 실행되는 스크립트. 설치는 사이드바 → 커스터마이즈 → 플러그인 → Anthropic 마켓플레이스. Git 저장소(GitHub, GitLab, Bitbucket)도 마켓플레이스로 사용 가능.
- 제한 사항(주장/미검증): 플러그인당 200MB / 파일 5,000개, 계정당 마켓플레이스 25개, 마켓플레이스당 플러그인 500개.

### 시작 단계 & 주의사항 & 대상
- **시작 6단계**: (1) 맥/윈도우 최신 클로드 데스크톱 앱 설치(코워크는 데스크톱 앱 전용, 웹 불가) + 프로/맥스 구독. (2) 첫 프로젝트를 작게 만들기(예: 주간 보고서 폴더 + 지시 + 메모리). (3) 디스패치로 원격 조정 해보기. (4) 기본 스킬로 엑셀/PPT 만들어보기. (5) 커넥터 추가(구글 드라이브·지메일 등 연결해 흐름 확인). (6) 플러그인 마켓플레이스 둘러보기.
- 추천 자동화 시나리오: 정기 보고서 자동화, 경쟁사 모니터링, 회의 후처리, 코드 리뷰·PR 정리, 데이터 분석.
- **주의 5가지**: (1) 권한 관리 꼼꼼히. (2) 결과를 그대로 신뢰하지 말 것. (3) 처음엔 작게 시작. (4) 비용 고려. (5) 백업 잘 할 것.
- **잘 맞는 사람**: 반복 작업 많은 사람 / 여러 도구를 옮겨다니는 사람 / 백그라운드 작업 많은 사람 / 팀 단위 워크플로우 정리하려는 사람. **안 맞는 경우**: 매번 다른 종류의 일이라 자동화할 반복 작업이 별로 없는 경우 — 굳이 안 쓰길 권함.

### FAQ & 세일즈 장치
- FAQ: (1) 기존 클로드와 차이 = **자동화**(옆에 안 붙어도 됨). (2) 데이터가 웹으로 나가나 — 로컬 파일 처리는 내 컴퓨터 안에서만이지만 AI 처리 자체는 Anthropic 서버를 거치므로 완전한 로컬 처리는 아님. (3) 내가 만든 스킬/플러그인은 Git 저장소로 공유 가능. (4) 맥·윈도우 모두 되지만 [의견] 맥이 좀 더 안정적. (5) 챗지피티 등 다른 AI와 병행 사용 가능("도구는 도구일 뿐").
- **세일즈 장치(정보로 기록)**: 영상 말미에 AI 스터디·업무 자동화·코워크 자동화 테크닉을 알려주는 **유료 멤버십**(설명란 링크) 가입을 유도. 다음 영상은 코워크로 다른 자동화 예시를 준비하겠다고 예고.

## 전문 — 시간순 워크스루

### [00:00–00:26] 도입 — 코워크는 "같이 일하기"
클로드 코워크는 그대로 풀면 "같이 일하기(협업)"로, 클로드를 진짜 동료처럼 쓸 수 있게 만든 것이라 소개한다. 그동안은 채팅창에서 질문을 던지고 답을 받는 핑퐁 방식이었지만, 코워크는 일을 통째로 맡겨 놓고 다른 걸 하다가 돌아오면 결과가 나와 있는 "옆자리 동료에게 일 부탁하는 느낌"이라 말한다. 오늘 영상에서 코워크가 무엇인지, 어떤 기능이 있는지, 어떻게 시작하는지를 차근차근 정리하겠다고 예고한다.

### [00:26–01:33] 기존 AI의 3가지 답답함
먼저 문제부터 짚는다. AI에 일을 시키면 처음엔 신기하지만(코드·글 써줌), 쓰다 보면 답답해진다. 이유 첫째는 **일이 한 단계로 안 끝나기 때문**이다. 보고서 하나도 자료 찾기·정리·분석·작성·결론이 한 호흡에 안 끝나서 매번 다음 단계를 시키며 옆에 붙어 있어야 한다("인턴 옆에서 한 마디 한 마디 가르치는 사장님"). 둘째는 **AI가 매번 기억을 잃는다**는 점("매일 아침 어제 일을 다 잊고 출근하는 신입"). 셋째는 **AI가 내 컴퓨터 파일을 못 본다**는 점 — 파일을 열어 복사→채팅에 붙여넣고→답변을 다시 파일에 붙여넣는 반복에 하루 종일 하면 지친다. 이 불편으로 AI를 포기하는 사람도 있고, 화자 자신도 그랬다고 공감한다.

### [01:33–02:21] 코워크의 정의 — 에이전틱 워크스페이스
Anthropic도 이를 알고 코워크를 만들었다. 한 줄 정의는 "AI에게 한 번 시켜 놓고 다른 걸 할 수 있게 해 주는 공간", 영어로 **에이전틱 워크스페이스(agentic workspace)**다. 기존은 채팅(묻고 답하는 캐치볼)이었다면, 코워크는 **결과물을 말해주면 클로드가 그걸 만들어 내는** 구조로 과정을 일일이 안 봐도 된다.
> "회사에 PM이 있어서 그 사람한테 '다음 주까지 이 보고서 좀 정리해 주세요' 하고 던져 놓으면 그 사람이 알아서 자료 모으고 분석하고 발표자료 만들어서 가져다 주는 거랑 같아요." [02:02]

핵심은 **자동화** — 옆에 붙어 있을 필요 없이 일을 던지고 다른 일을 하다 결과를 받으면 끝이라, 비로소 진짜 동료처럼 느껴진다고 말한다.

### [02:21–03:00] 인테리어 비유
좀 더 와닿는 비유로 인테리어 업체를 든다. 인테리어를 맡기면 도면 그리고 자재 정리하고 일정 잡은 뒤, 업자들이 며칠간 작업하는 동안 우리는 출근·일하고 가끔 확인만 하다 끝나면 들어가 살면 된다. 기존 AI 채팅은 "벽지 색, 거실 조명, 콘센트 위치를 매번 전화로 물어보는" 것처럼 우리가 다 결정·지시해야 해 진이 빠졌다. 코워크는 도면과 시방서를 한 번 주면 알아서 진행하고 막히는 부분만 가끔 물어봐서 우리 시간이 많이 확보된다.

### [03:00–03:32] 비포/애프터 & 시간 절약
비포는 채팅창에 질문→답변 복사→엑셀 붙여넣기→다시 질문의 반복으로 30분짜리 일을 한 시간 붙어서 한다. 애프터는 "이번 분기 매출 데이터 정리해서 엑셀로 만들어 줘" 한 마디 던지고 점심 먹고 오면 함수까지 들어간 엑셀 파일이 폴더에 만들어져 있어 검토만 하면 끝. 하루 한 시간씩 줄면 한 달 20시간, 1년이면 240시간(일주일치 근무)이 절약된다는 계산을 든다(주장/미검증).

### [03:32–04:43] 핵심 기능 ① 로컬 파일 관리
핵심 4가지 중 첫째는 **로컬 파일 관리**로, 화자는 이게 가장 중요하다고 강조한다. 코워크에서 클로드는 내 컴퓨터 안 파일을 직접 읽고 쓴다 — 업로드·다운로드 없이 "어제 작업한 보고서 좀 손봐줘" 하면 그 파일을 찾아 열고 수정해 다시 저장한다. 우리가 하는 일은 폴더만 알려주는 것.
> "그동안 클로드는 회사 밖에서 일하는 외주 직원이었어요. … 근데 코워크는 클로드가 회사 안으로 들어온 거예요. 우리 옆자리 직원처럼요." [03:59]

진짜 일은 다 파일(보고서·데이터·코드·디자인)에 있으므로 이게 중요하다. 파일이 100개면 채팅 붙여넣기는 100번 해야 하지만 코워크는 폴더만 알려주면 된다. 보안 우려에 대해서는 **폴더 단위 권한 관리**로 허용 안 하면 못 보고, 중요 작업 시 "이 파일 지워도 돼요?"처럼 매번 물어봐서 갑자기 뭐가 사라지지 않는다고 설명한다.

### [04:43–05:43] 핵심 기능 ② 크롬 연동
둘째는 **크롬 연동**으로, 'Claude in Chrome'과 짝을 이뤄 클로드가 우리 대신 웹브라우저를 조작한다. "이 회사 채용 공고 다 정리해 줘" 하면 채용 페이지를 열고 공고를 하나씩 클릭·읽어 표로 만들어, 마우스를 한 번도 안 잡아도 된다. 로컬 파일 + 크롬이 결합되면 "웹에서 자료 가져와 내 컴퓨터 파일로 정리"가 완전 자동화되어, 매주 월요일 경쟁사 가격 추적해 엑셀 업데이트 같은 30분짜리 일을 알아서 해준다. 다만 웹브라우저 조작은 위험할 수 있어(잘못 클릭 시 결제 진행 등) 민감 작업 시 "이거 진행할까요?"를 꼭 확인받고, **10분 내 답이 없으면 안 하고 넘어가는** 안전장치가 있다.

### [05:43–06:36] 핵심 기능 ③ 병렬 작업
(중간 [05:43]에서 로컬 파일·크롬 두 기능을 짧게 재정리한 뒤) 셋째는 **병렬 작업**이다. 복잡한 일이 들어오면 클로드가 알아서 작은 일로 쪼개 동시에 여러 개 처리해 결과가 빨리 나온다. 기존엔 A 끝나고 B, B 끝나고 C로 순차 처리라 오래 걸렸지만, 코워크는 A·B·C를 동시에 돌려 시간이 준다.
> "예전 방식은 셰프 한 명이 전체 요리를 만들고 … 이제는 여러 셰프가 한 번에 만들어요." [06:20]

큰 일을 작은 일로 쪼개는 능력과 쪼갠 일을 따로 진행시키는 능력이 갖춰져 있어, 우리가 따로 설정 안 해도 알아서 한다.

### [06:36–07:37] 핵심 기능 ④ 전문 산출물 생성
넷째는 사람들이 제일 좋아하는 기능으로, **엑셀·파워포인트·워드** 같은 실제 업무 문서를 만들어 준다. 클로드가 텍스트만 뱉는 게 아니라, 엑셀은 표만이 아니라 **SUM·VLOOKUP·피벗 테이블 등 함수까지 작동**한다. "이번 분기 매출·비용 분석해서 엑셀로 정리해 줘" 하면 셀에 함수가 들어가고 차트도 그려진 진짜 작동하는 파일이 만들어진다. 파워포인트도 목차·슬라이드 내용·디자인까지 어느 정도 잡아준다. [의견] "글 쓰는 비서에서 진짜 업무를 아는 비서로 바뀌었다." 회사 일은 다 문서(발표자료·보고서·데이터 정리)로 끝나는데 코워크가 80%를 만들어 주면 우리는 20%만 손보면 되어 시간이 크게 절약된다(주장/미검증).

### [07:37–08:52] 디스패치 (원격 조작)
다음은 **디스패치** — 핸드폰으로 집 컴퓨터에 일을 시키는 원격 조작 기능이다. 집 데스크톱에 클로드 데스크톱 앱을 켜 두고 인터넷에 연결한 채 외출하면, 밖에서 모바일 앱으로 일 하나를 던지고 실제 작업은 집 데스크톱에서 돌며 결과는 핸드폰으로 확인한다("집 PC를 원격 조종"). 굳이 데스크톱에서 돌리는 이유는 두 가지 — (1) 컴퓨터 성능을 그대로 써서 핸드폰보다 빠르고 안정적, (2) 데스크톱에 미리 등록한 폴더·도구·서비스를 다 쓸 수 있음(핸드폰엔 없음). "데스크톱이 진짜 일터, 핸드폰은 원격 리모컨." 쓰려면 **프로/맥스 구독** + 맥이나 윈도우 최신 클로드 데스크톱 앱 두 가지가 필요하다. 민감 작업은 알림으로 확인받고 **10분 내 미응답이면 자동으로 안 하고 넘어간다**.

### [08:52–10:32] 프로젝트 & 5가지 구성요소
**프로젝트**는 코워크의 또 다른 핵심 기능으로, 일들을 영역별로 정리해 두는 곳(특정 영역 작업을 위한 연구 워크스페이스)이다. "회사에서 부서별로 사무실 나눠 놓은 것"에 비유 — 영역별로 나누면 효율이 오른다. 미리 한 번 세팅하면 그 프로젝트에 들어갈 때마다 자동 적용되어 매번 폴더·규칙을 설명할 필요가 없다. 각 프로젝트엔 **5가지**가 들어간다: (1) **설명** — 무엇을 하는 프로젝트인지 적어 두면, 디스패치가 작업을 어디로 보낼지("이 작업은 마케팅 프로젝트로") 판단할 때 참고한다. (2) **폴더** — 클로드가 읽고 쓸 로컬 디렉토리, 파일당 최대 50MB(주장/미검증), 여러 개 등록 가능. (3) **지시(instructions)** — 매 세션 자동 적용되는 기본 안내(예: 한국어로 답변, 코드는 TypeScript, 보고서는 이 템플릿). "회사 매뉴얼" 역할이라 한 번 적으면 매번 말 안 해도 된다. (4) **링크** — 참고 URL. (5) **메모리** — 세션이 끝나도 사라지지 않는 프로젝트 단위 기억 저장소로, 클로드가 배운 것·발견한 것을 계속 저장한다.

### [10:32–11:26] 프로젝트 동작 & 클로드 AI 프로젝트와의 차이
세션 시작 시 프로젝트를 선택하면 그 폴더들이 자동 마운트되고 지시가 자동 적용되며, 만든 파일은 그 폴더에 저장되고 쌓인 지식은 메모리에 자동으로 들어간다. 장점은 컨텍스트 전환이 빨라진다는 것("마케팅 프로젝트=마케팅 모드, 개발 프로젝트=개발 모드"). 헷갈리기 쉬운 부분으로 **코워크 프로젝트 ≠ 클로드 AI 프로젝트**를 짚는다. 클로드 AI 프로젝트는 계정/클라우드에 있어 어디서든 접속·유료 플랜에서 팀 공유가 되지만 로컬 폴더를 못 본다. 코워크 프로젝트는 우리 컴퓨터(로컬)에만 존재해 로컬 폴더를 다 보지만 다른 컴퓨터에선 못 본다. 둘은 합쳐지지 않으나, 클로드 AI 프로젝트를 코워크에 **링크**해 같이 쓸 수는 있다.

### [11:26–12:07] 프로젝트 활용 팁 4가지
(1) 처음 한 번 세팅에 시간을 투자 — 지시를 꼼꼼히 적을수록 이후 세션이 편해진다(일하는 스타일·선호 형식·자주 쓰는 도구를 다 기록). (2) 디스패치와 함께 쓰기 — 백그라운드 작업을 특정 프로젝트로 보내 원격 컨트롤. (3) 메모리 활용 — 클로드가 뭘 저장했는지 가끔 확인하면 신뢰가 오르고, 잘못된 정보는 수정. (4) 아카이브 활용 — 프로젝트를 아카이브하면 메타데이터만 사라지고 로컬 파일은 그대로 유지되어, 옛 프로젝트를 부담 없이 아카이브했다 나중에 다시 꺼낼 수 있다.

### [12:07–13:15] 커넥터 (MCP)
**커넥터**는 **MCP(Model Context Protocol, 모델 컨텍스트 프로토콜)** 표준으로 클로드를 외부 도구·데이터 소스에 연결하는 것이다. MCP는 "AI와 외부 서비스를 연결할 때 쓰는 공통 약속" 정도로 이해하면 된다며 콘센트 규격에 비유한다.
> "한국 어디 가도 220V 콘센트가 똑같이 생겼잖아요. … MCP가 AI 세계의 콘센트 규격이에요. 한번 정해 놓으니까 어떤 도구도 다 꽂아서 쓸 수 있어요." [12:26]

예로 노션 연결 시 페이지 읽기·쓰기, 슬랙 연결 시 메시지 주고받기, GitHub 연결 시 코드 저장소 다루기가 가능하다. 매일 쓰는 도구들에 커넥터를 걸어두면 "어제 회의록 보고 액션 아이템을 리니어 티켓으로 만들어 줘"처럼 클로드가 대신 조작해, 우리가 노션 열고 리니어 열어 티켓 만들 필요가 없어진다.

### [13:15–13:57] 스킬 (점진적 공개)
**스킬(Skills)**은 커넥터가 외부와 연결하는 통로라면, 특정 작업을 어떻게 처리할지 알려주는 "매뉴얼"이다. `SKILL.md` 파일과 보조 스크립트·리소스 파일이 든 폴더로, "이 작업이 들어오면 이렇게 처리해 줘"를 적어 둔다. 특이한 점은 스킬이 항상 활성화돼 있지 않다는 것 — 클로드가 처음엔 메타데이터만 읽고, 시킨 일이 어떤 스킬에 해당되면 그때서야 그 스킬 전체를 불러온다. 이를 **점진적 공개(progressive disclosure)**라 부르며, 모든 매뉴얼을 한 번에 다 읽는 것보다 필요한 순간에만 부르는 것이 훨씬 효율적이기 때문이다.

### [13:57–15:02] 플러그인 & 마켓플레이스
마지막으로 **플러그인**은 커넥터·스킬 등을 다 합쳐 패키지로 만든 것이다. 플러그인 하나에 **스킬, 커넥터, 에이전트, 훅** 네 가지가 들어갈 수 있다 — 에이전트는 우리 대신 작업하는 서브 에이전트, 훅은 특정 시점에 자동 실행되는 스크립트. "핸드폰 앱 하나가 여러 기능을 묶어 제공하는 것(카카오톡의 채팅·통화·페이)"에 비유한다. 설치는 사이드바 → 커스터마이즈 → 플러그인 메뉴 → **Anthropic 마켓플레이스**에서 원하는 걸 설치. 마켓플레이스는 공식 Anthropic 카탈로그가 기본이고, 추가로 Git 저장소(GitHub, GitLab, Bitbucket)도 마켓플레이스로 쓸 수 있다. 제한 사항(주장/미검증): 플러그인당 200MB / 파일 5,000개, 계정당 마켓플레이스 25개, 마켓플레이스당 플러그인 500개. 조직 관리도 가능하다고 덧붙인다.

### [15:02–16:20] 실전 시작 6단계
기능을 다 봤으니 실전 시작을 단계별로 정리한다. (1) 맥/윈도우 최신 클로드 데스크톱 앱 설치 — 코워크는 데스크톱 앱에서만 작동하고 웹에선 안 되며, 프로/맥스 구독이 있어야 모든 기능을 쓸 수 있다. (2) 첫 프로젝트를 거창하지 않게 작게 만들기 — 예: 매주 쓰는 주간 보고서 폴더를 만들어 코워크 프로젝트로 등록, 지시엔 "주간 보고서는 이런 형식", 메모리엔 "지난주 핵심 이슈는 이거". (3) 디스패치로 원격 조정 해보기. (4) 기본 스킬 활용 — 엑셀·PPT를 만들어 보기(잘 만들어 준다). (5) 커넥터 추가 — 구글 드라이브·지메일 등 매일 쓰는 도구를 연결해 흐름 확인. (6) 플러그인 마켓플레이스 둘러보기 — 이미 좋은 플러그인이 많으니 내 일에 맞는 걸 설치. 추천 시나리오로 정기 보고서 자동화, 경쟁사 모니터링, 회의 후처리, 코드 리뷰·PR 정리, 데이터 분석을 든다.

### [16:20–16:34] 주의할 점 5가지
좋은 점만 알려줬으니 주의점도 짚는다. 코워크가 만능은 아니다. (1) 권한 관리 꼼꼼하게. (2) 결과를 그대로 신뢰하지 말 것. (3) 처음엔 작게 시작. (4) 비용도 고려. (5) 백업 잘 할 것.

### [16:34–17:15] 무엇을 바꾸는가 & 잘 맞는 사람
