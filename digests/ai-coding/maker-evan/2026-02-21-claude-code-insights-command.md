# 이 명령어 모르면 Claude Code 손해입니다

> maker-evan · 2026-02-21 · [영상 링크](https://youtu.be/aOJI944guKI)
> 정리 방식: 시간순 심층 워크스루 (paraphrase) + 주제별 상세 요약
> 검증 배너: 마찰 건수·세션 수 등 리포트에 나오는 수치는 화자가 소개한 사례/타인 보고 기반이며 미검증. 화자가 직접 지적하듯 리포트 숫자 자체가 AI 할루시네이션일 수 있음.

## 요약 (주제별)

### 기능 정체 — `/insights`
- Claude Code에서 `/insights`를 치면 실행되는 자기 성찰(self-reflection) 분석 기능. 지난 한 달간 사용자가 AI를 어떻게 썼는지 분석해 "이렇게 하면 더 잘 쓸 수 있다"고 코칭해 주는 리포트를 만든다.
- [의견] 화자는 이를 "PT(퍼스널 트레이너)가 동작을 콕 집어 교정해 주는 것"에 비유. AI 코딩 도구 중 이런 자기 성찰 기능을 넣은 건 Claude Code가 처음이라고 주장(주장/미검증).
- 단순 사용 횟수·비용 집계가 아니라, 어디서 막혔는지·개선 방법까지 제안하는 것이 차별점.

### 내부 작동 — 6단계
- 화자의 비유: "일기장을 읽어서 분석 리포트를 써 주는 비서".
- (1) 대화 기록 수집: 컴퓨터에 쌓인 AI 대화 기록을 불러오되, 너무 짧거나 잠깐 나눈 건 제외하고 의미 있는 것만 선별.
- (2) 기본 정보 추출: 대화 길이, 작업 종류, 사용 빈도 정리.
- (3) 긴 대화 축약: 두꺼운 책을 요약본으로 만들듯 긴 대화는 요점만 줄임.
- (4) 만족도 채점(핵심): AI가 각 대화를 읽고 사용자가 만족했나·어디서 답답해했나·결국 원하는 걸 얻었나 평가.
- (5) 패턴 도출: 한 달치 대화를 종합해 패턴을 찾음.
- (6) 리포트 생성: 예쁜 리포트로 만들어 브라우저로 띄움.

### 리포트 구성
- 한눈 요약: 잘 되고 있는 것 / 막히는 지점 / 바로 적용 가능한 개선 방법을 핵심만.
- 통계 대시보드: 주고받은 메시지 수, 바꾼 코드 줄 수, 건드린 파일 수 등 숫자.
- 마찰(friction) 분석: 실사용자 리포트 예시에서 마찰 1순위가 "성급한 행동" 46건이었다고 소개(주장/미검증). = AI가 사용자 말을 다 듣지 않고 바로 뭔가를 하려 덤벼드는 것.
- 만족도 평가: "짜증남"부터 "행복함"까지 5단계로 세션별 만족도를 AI가 판단.
- 설정 규칙 제안(가장 실용적): 리포트 마지막에 바로 붙여 넣기 가능한 설정 규칙을 뽑아 줌 — "이 규칙 추가하면 마찰이 줄어들 것"이라고 구체 제안.

### 리스크·한계
- 숫자 날조: 해외 커뮤니티의 한 사용자가 실제로 10번 정도만 썼는데 리포트에 "300번 이상의 데이터베이스 작업 세션"이라 나왔고, AI에게 물으니 "336이라는 숫자는 제가 만들어 낸 것"이라 답했다고 함(주장/미검증).
- 원인: 분석 비용 절감을 위해 작은 AI 모델을 쓰는데, 데이터가 부족하면 그럴듯한 숫자를 지어내는 할루시네이션이 발생.
- 그 외: 버그가 많고, 최근 사용한 것 위주로 분석이 쏠리는 경향이 있다는 피드백.
- [의견] 그래서 구체적 숫자보다 패턴·경향성 중심으로 보는 게 맞다.

### 가치·의의
- 한 블로거 평가: "잘 훈련된 매니저에게 피드백 받는 것 같다"(주장/미검증). [의견] 화자는 이게 핵심이라고 봄.
- 지금까지는 얼마나 썼는지·돈이 얼마인지만 다른 도구로 볼 수 있었지만, "이렇게 쓰면 더 효율적"이라 코칭해 주는 건 이 기능밖에 없음.
- 특히 맞춤 설정 규칙 자동 생성이 실용적 — 최적화하고 싶어도 어디를 건드릴지 모르는 사람에게 유용.
- [의견] "AI를 더 잘 쓰기 위해 AI를 쓴다"는 컨셉 자체가 앞으로 AI 도구의 표준이 될 수도 있다.

### 사용 조건·방법
- 방법: Claude Code를 켜고 `/insights`를 치면 끝.
- 조건 (1) Claude Pro Plus 또는 Max 구독 필요.
- 조건 (2) 최소 2~4주는 Claude Code를 써서 데이터가 쌓여야 함. 처음 쓰자마자 돌리면 볼 게 없음.
- 조건 (3) 최신 버전이어야 함. 안 되면 업데이트.
- 결과는 컴퓨터 어딘가에 저장되고 자동으로 브라우저에서 열림.
- [의견] 숫자는 맹신하지 말고 패턴·한계 위주로, 맞춤 설정 규칙은 적극 활용 권장. 아직 초기라 완벽하진 않지만 방향은 정말 좋다고 봄.

## 3줄 오리엔테이션
- Claude Code의 `/insights` 명령어는 지난 한 달 사용 기록을 AI가 분석해 "당신은 이렇게 쓰면 더 낫다"고 코칭하는 자기 성찰 리포트를 만들어 준다.
- 리포트는 통계·마찰 분석·만족도 평가·붙여넣기 가능한 설정 규칙으로 구성되며, 특히 설정 규칙 제안이 실용적이다.
- 단, 작은 모델이 데이터 부족 시 숫자를 지어내는(할루시네이션) 한계가 있어 구체 수치보다 패턴 위주로 봐야 한다.

## 전문 — 시간순 워크스루

### [00:00–00:20] PT 비유로 여는 도입
화자는 퍼스널 트레이닝(PT)을 받아본 경험에 빗대 시작한다. 운동을 잘하는 것 같아도 트레이너가 "이 동작이 문제니 이렇게 바꾸라"고 콕 집어 주는데, 이것이 AI 코딩 도구에서 실제로 일어나고 있다는 것이다. Claude Code에 명령어 하나를 치면 AI가 지난 한 달간 내가 어떻게 썼는지 분석하고, 더 잘 쓰는 법까지 코칭해 준다며, 오늘 이 기능을 완전히 파헤쳐 보겠다고 예고한다.

### [00:20–00:52] `/insights`가 무엇인가
Claude Code에서 `/insights`를 치면 실행되는 분석 기능이다. 쉽게 말해 사용자가 한 달 동안 AI에게 어떤 질문을 했는지, 어디서 막혔는지, 대화가 어떻게 흘렀는지를 AI가 훑어보고 읽기 좋은 리포트로 만들어 준다. 중요한 건 이게 "이번 달에 몇 번 썼고 돈이 얼마 나왔나" 같은 집계가 아니라, "당신도 자꾸 막히더라, 이 부분은 이렇게 접근하면 훨씬 부드럽다"는 개선 방법까지 제안한다는 점이다.
> "AI 코딩 도구들 중에서 이런 자기 성찰 기능을 넣은 건 클로드 코드가 처음이에요." [00:48] (주장/미검증)

### [00:52–01:38] 내부 작동 6단계
화자는 내부 작동을 최대한 쉽게 풀겠다며 "일기장을 읽어서 분석 리포트를 써 주는 비서"로 비유하고, 6단계로 설명한다. 첫째, 컴퓨터에 쌓인 AI 대화 기록을 불러오되 너무 짧거나 잠깐 나눈 건 제외하고 의미 있는 것만 골라온다. 둘째, 대화가 얼마나 길었는지·어떤 종류의 작업을 했는지·얼마나 자주 썼는지 기본 정보를 뽑는다. 셋째, 대화가 길면 두꺼운 책을 요약본으로 만들듯 요점만 줄인다. 넷째(핵심), AI가 각 대화를 읽고 이 사람이 만족했나·어디서 답답해했나·결국 원하는 걸 얻었나를 평가하는 일종의 만족도 채점을 한다. 다섯째, 한 달치 대화를 전부 종합해 패턴을 찾아낸다. 마지막으로 이걸 예쁜 리포트로 만들어 브라우저로 띄워 준다.

### [01:39–01:56] 리포트 열면 보이는 것 — 요약·통계
실제로 보고서를 열면 제일 먼저 한눈에 보는 요약이 나온다. 잘 되고 있는 것, 막히는 지점, 바로 적용 가능한 개선 방법을 핵심만 뽑아 보여 준다. 그다음엔 통계 대시보드로, AI와 주고받은 메시지 수, 코드를 몇 줄이나 바꿨는지, 파일을 몇 개나 건드렸는지 같은 숫자들이 쭉 나온다.

### [01:56–02:15] 마찰(friction) 분석 — "성급한 행동" 1순위
화자가 진짜 흥미롭다고 꼽는 건 마찰 분석이다. 실제로 써본 사람들의 보고서를 보면 마찰 1순위가 "성급한 행동"이었고, 46건이나 됐다고 한다(주장/미검증). 쉽게 말해 AI가 내 말을 다 듣지도 않고 바로 뭔가를 하려 덤벼드는 것으로, "잠깐만, 내가 원한 건 이게 아닌데" 하는 순간을 가리킨다.

### [02:15–02:30] 만족도 평가와 설정 규칙 제안
이어 만족도 평가를 해 준다. "짜증남"부터 "행복함"까지 다섯 단계로, 세션별로 사용자가 얼마나 만족했는지 AI가 판단한 것이다. 그리고 가장 실용적인 것으로, 보고서 마지막에 바로 붙여 넣기 가능한 설정 규칙을 뽑아 준다. "이 규칙을 추가하면 마찰이 좀 줄어들 것"이라고 구체적으로 제안해 주는 방식이다.

### [02:30–03:09] 솔직한 문제점 — 숫자 날조(할루시네이션)
화자는 솔직한 리뷰이니 문제도 짚어야 한다며 가장 큰 문제로 "숫자를 지어낼 수 있다"는 점을 든다. 해외 커뮤니티의 한 사용자가 실제로 Claude Code를 10번 정도밖에 안 썼는데 보고서에는 "300번 이상의 데이터베이스 작업 세션"이라고 나왔고(주장/미검증), AI에게 직접 이 숫자가 맞냐고 물으니 다음처럼 답했다고 한다.
> "336이라는 숫자는 제가 만들어 낸 겁니다." [02:52] (주장/미검증)
원인은 이 분석의 비용을 줄이려고 작은 AI 모델을 쓰는데, 데이터가 부족하면 그럴듯한 숫자를 지어내는 현상, 즉 AI의 고질적 문제인 할루시네이션이 생기기 때문이다. 그래서 보고서의 구체적 숫자보다는 패턴이나 경향성 중심으로 보는 게 맞다고 조언한다.

### [03:09–03:15] 추가 한계 — 버그·최근 편향
또 하나, 버그도 좀 많고, 최근에 사용한 것들 위주로 분석이 쏠리는 경향이 있다는 피드백이 있다고 덧붙인다.

### [03:15–03:44] 그래도 가치 있는 이유 — 코칭·설정 자동화
문제점을 얘기했지만 이 기능이 그래도 존재하는 이유가 있다. 한 블로거는 "잘 훈련된 매니저에게 피드백 받는 것 같다"고 평가했는데(주장/미검증), 화자는 이게 진짜 핵심이라고 본다. 지금까지는 AI를 얼마나 썼는지·돈이 얼마인지만 다른 도구로 볼 수 있었지만, "너 이렇게 쓰면 더 효율적"이라고 코칭해 주는 건 이 기능밖에 없다는 것이다. 특히 맞춤 설정 규칙을 자동으로 만들어 주는 게 실용적인데, AI 쓰는 방식을 최적화하고 싶어도 어디를 어떻게 건드려야 할지 모르는 상황에서 AI가 분석해 "이 설정 넣어 봐라"고 해 주니 편하다고 강조한다.

### [03:44–04:07] 사용법과 조건
사용법은 아주 간단하다. Claude Code를 켜고 `/insights`를 치면 끝이다. 다만 조건이 있다. 첫째, Claude Pro Plus나 Max 구독이 있어야 한다. 둘째, 최소 2~4주 정도는 Claude Code를 써서 데이터가 쌓여야 하며, 처음 쓰자마자 돌리면 볼 게 없다. 셋째, 최신 버전이어야 하고 안 되면 업데이트를 해 보라고 안내한다. 결과는 컴퓨터 어딘가에 저장되고 자동으로 브라우저에서 열린다.

### [04:07–04:34] 정리 — "AI를 더 잘 쓰기 위해 AI를 쓴다"
정리하자면 `/insights`는 Claude Code만의 독특한 기능으로, 다른 AI 코딩 도구에는 아직 없다(주장/미검증). 내가 AI를 어떻게 쓰고 있는지 AI가 분석해 코칭해 주는 일종의 "AI 피드백 코치"다. 다만 숫자를 맹신하지 말고 패턴과 한계 위주로 보되, 맞춤 설정 규칙 제안은 진짜 실용적이니 적극 활용하라고 권한다.
> "AI를 더 잘 쓰기 위해 AI를 쓴다. 이 컨셉 자체가 앞으로 AI 도구의 표준이 될 수도 있거든요." [04:29] [의견]

### [04:34–04:42] 마무리 CTA
아직 초기라 완벽하진 않지만 방향 자체는 정말 좋다고 본다며, 시청자에게 직접 돌려 보고 특히 마찰 분석 1위가 뭐로 나오는지 댓글로 공유해 달라고 요청한다. 영상이 도움됐다면 구독을 눌러 달라는 인사로 마친다.
