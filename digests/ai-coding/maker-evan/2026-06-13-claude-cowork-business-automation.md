## 요약 (주제별)

### 클로드 코워크(Claude Cowork)란 / 왜 좋은가
- 클로드 코드(Claude Code)나 챗(Chat)보다 자유도가 높으면서 더 편리한 기능을 가진 것으로 소개된다. [의견]
- 핵심 가치는 "다양한 업무 + 스케줄된(예약된) 루틴 업무를 자연어로 생성"할 수 있다는 점.
- 좌측 상단에 채(Chat)·코워크(Cowork)·코드(Code) 세 모드가 있고, 이 영상은 코워크 모드를 다룬다.

### 사전 세팅(2단계 연동)
- 앱 하단 Max 버튼 → 설정 → 커넥터로 진입, "맞춤 설정"에서 웹으로 커넥터를 추가한다.
- 첫째, **플레이 MCP**를 연결한다. 연결하면 커넥터 URL이 뜨고, 그 URL 웹에 들어가 로그인(카카오톡 로그인)을 한다. 로그인 후 "카카오톡 나챗방(나와의 채팅방)" 친구를 클릭 → "도구함에 추가"를 누른다. [불명확: 자막의 "플레이 MCP"가 정확한 제품명인지, 카카오톡 "나와의 채팅"으로 메시지를 보내는 별도 MCP 서비스로 추정되나 확정 못 함]
- 클로드 앱을 껐다 켜면 플레이 MCP에 "쓰기/삭제 도구(카카오톡 나챗방)"가 뜨는데, 여기서 **항상 허용**으로 설정한다.
- 둘째, 브라우저 확장 **클로드 for 크롬(Claude for Chrome / "클로드인 크롬")**을 설치한다. 맞춤 설정 또는 구글 크롬 익스텐션에서 설치하면, 클로드가 브라우저 창을 컨트롤할 권한을 갖는다.

### 스케줄(예약 작업) 만드는 절차
- 코워크 버튼 → 스케줄 버튼 → "새 작업" → "클로드와 함께 창작하기"를 누른다.
- 클로드가 되묻는 것을 막기 위해 먼저 **"질문 대신 묻지 않고 실행"** 기능을 켠다.
- 그다음 프롬프트를 입력한다. 화자는 프롬프트 전문을 댓글창에 올려주겠다고 안내(세일즈/유입 장치).
- 작업 생성 후 "일정" 버튼을 누르면 예약되고, 스케줄 목록에 코워크 스킬로 등록된다.
- 실행 전 주의: 편집에 들어가 "묻지 않고 실행"을 켠 뒤 저장하는 것을 추천. 그 후 "지금 실행"으로 테스트한다.

### 사례 1 — 인스타그램 신규 게시물 알림
- 가정: 병원 마케터. 매일 아침 특정 SNS 계정(예: "바이미 성형외과" 인스타 계정)에 새 마케팅 포스트가 올라왔는지 체크해 알림받기.
- 프롬프트 요지: "나는 병원 마케터인데, 매일 아침 9시에 이 사이트에 새 게시물이 올라왔는지 알림을 줘. 올라왔으면 해당 게시물 링크와 함께 요약해서, 플레이 MCP로 나와의 채팅방에 게시물 내용과 링크를 보내줘."
- 실행하면 클로드 for 크롬 MCP가 인스타그램을 크롤링하며 글을 하나씩 열어 확인 → 정보 수집. 사용자가 손대지 않고 버튼만 눌러도 자동 진행.
- 결과: 카카오톡에 "바이미 PS 새 게시물" 알림이 오고, 링크를 타면 실제 게시물로 연결된다. 스케줄로 매일 반복 실행된다.

### 사례 2 — 일일 마케팅 트렌드 리서치
- 프롬프트 요지: "오늘 하루 인스타·틱톡 SNS에서 활발했던 마케팅 트렌드를 조사해서, 카카오톡 나와의 채팅방에 플레이 MCP로 트렌드 요약을 보내줘." 매일 오전 9시 트리거.
- 실행하면 웹 검색으로 트렌드를 긁어 요약을 카카오톡으로 전송. 예시 결과 항목: 언필터드(unfiltered) 콘텐츠 부상, [불명확: "랩파이"] , 챌린지 부활, 빅 이벤트(올리비아 로드리고 신보), FIFA 월드컵 관련 화제 등.
- 화자 팁: 프롬프트를 세분화(예 "한국의 인스타그램 마케팅 트렌드")하면 더 정확. [의견]

### 사례 3 — X(트위터) → Threads 자동 번역 포스팅
- 화자가 실제로 쓰레드에서 여러 번 돌려봤다는 사례. X(엑스)의 최신 글을 긁어 번역해 "쓰레드 데일리"에 올리는 자동화.
- 동작: 클로드가 컴퓨터/크롬을 연결 → X 사이트에서 글들을 긁고 적절한 게시물을 선별·번역 → 크롬으로 Threads에 들어가 복사·붙여넣기 → 포스트 버튼 클릭까지 자동. 결과가 프로필에 자기 톤(깔)에 맞게 올라온다.

### 확장 활용 사례(언급만)
- 특정 카페 리뷰 크롤링, 이커머스 리뷰 이관(자사몰 리뷰 → 에이블리 업로드, 또는 반대) 등 반복 작업 전반이 가능하다고 언급.
- 라이브 아티팩트, 디스패치, 커스터마이즈, 스킬, 프로젝트 등 다른 기능은 추후 영상에서 다룰 예정.

### 구독 티어 / 도구 구성 팁
- 소넷(Sonnet) 모델, 즉 프로(Pro) 구독만으로도 코워크 사용에 큰 문제는 없다.
- 다만 작업이 많으면 토큰 부족이 종종 발생 → 무조건 Max 100불 플랜 추천. 간단한 작업엔 프로로 충분. [의견]
- 인플루언서 마케팅을 많이 고민하는 사람에겐 "코워크 2" 활용을 추천. [불명확: "코워크 2"가 무엇을 지칭하는지 자막만으로 불명확]
- 화자가 실제 사용한 도구는 플레이 MCP + 클로드 for 크롬 두 개뿐이며, 코딩 지식 없이 로그인만으로 자연어로 처리했다고 강조.

### 화자의 마무리 주장 / 세일즈 구조
- [의견] "앞으로 코워크를 쓸 수 있는 사람과 없는 사람의 격차가 계속 벌어질 것." → 이 툴로 자신만의 장기를 발전시키라 권유.
- 세일즈/유입: 프롬프트는 댓글창으로 안내(댓글 달면 받는 방법 가이드), 멤버십 영상 구독 권유.

## 3줄 오리엔테이션
- 클로드 코워크(Claude Cowork)의 **스케줄(예약 작업)** 기능에 **플레이 MCP**(카카오톡 나와의 채팅 전송)와 **클로드 for 크롬** 확장(브라우저 컨트롤)만 붙이면, 코딩 없이 자연어 프롬프트로 루틴 업무를 자동화할 수 있다는 튜토리얼.
- 세 가지 실사례(인스타 신규 게시물 알림 / 일일 마케팅 트렌드 요약 / X→Threads 자동 번역 포스팅)를 화면으로 시연한다.
- 마지막에 구독 티어(프로 vs Max 100불) 팁과 "코워크를 쓰는 자·못 쓰는 자의 격차" 주장, 프롬프트·멤버십 유입 안내로 마무리.

## 전문 — 시간순 워크스루

### [00:00–00:36] 오프닝 — 코워크가 뭐고 왜 좋은가
클로드 코워크(Claude Cowork)로 다양한 업무 자동화가 가능해졌다며, 짧고 굵게 알기 쉽게 티칭하겠다고 시작한다. 코워크의 장점은 클로드 코드(Claude Code)나 챗(Chat)보다 자유도가 높으면서도 훨씬 편리한 기능들이 있다는 것 [의견]. 핵심은 "다양한 업무 + 스케줄된(예약된) 루틴 업무를 자연어로 생성"할 수 있다는 점이라고 정리한다.
> "훨씬 다양한 업무와 그리고 스케줄된 업무들, 루틴한 업무를 이제 자연어를 통해서 이제 생성을 할 수 있다" [00:29]

### [00:37–01:09] 사전 세팅 ① — 플레이 MCP 커넥터 추가
워크플로 설명 전에 세팅할 값이 몇 개 있다고 안내. 앱 하단 Max 버튼을 눌러 설정 → 커넥터로 들어가고, "맞춤 설정" 버튼을 눌러 웹에서 몇 가지를 추가한다. 첫 번째로 "커넥터 불러오기"를 누른 뒤 **플레이 MCP**를 눌러 연결한다. 연결하면 커넥터 URL이 뜨는데, 그 URL을 실제 웹에 들어가 사용한다. [불명확: "플레이 MCP" 제품명 — 카카오톡 전송용 MCP로 추정되나 확정 불가]

### [01:15–01:38] 사전 세팅 ② — 카카오톡 로그인 & 나챗방 도구 추가
웹에 로그인하는데, 카카오톡 로그인을 사용한다. 로그인 후 아래의 "카카오톡 나챗방(나와의 채팅방)"이라는 친구를 클릭하고, "도구함에 추가" 버튼을 누른다(화자는 이미 추가된 상태). 이 과정을 그대로 따라오라고 안내한다.

### [01:39–01:56] 사전 세팅 ③ — 앱 재시작 후 "항상 허용"
클로드 앱을 껐다가 다시 켜면, 플레이 MCP에 "쓰기/삭제 도구(카카오톡 나챗방)"가 떠 있다. 이때 **"항상 허용"** 버튼을 눌러 항상 허용으로 세팅한다.

### [01:56–02:27] 사전 세팅 ④ — 클로드 for 크롬 확장 설치
다음으로 브라우저에서 확장 프로그램을 찾는다. **"클로드 for 크롬"(클로드인 크롬 / Claude for Chrome)** 버튼을 눌러 맞춤 설정에서 설치하거나, 구글 크롬의 익스텐션(확장 프로그램)에서 설치한다. 이렇게 하면 클로드 코드가 브라우저 창을 컨트롤할 수 있는 권한이 생긴다. 여기까지가 세팅 완료.

### [02:27–02:57] 스케줄 작업 만들기 — 진입 경로
세팅한 것으로 작업을 하나씩 해본다. 좌측 상단의 채·코워크·코드 세 모드 중 **코워크** 버튼을 누르고, 이어서 **스케줄** 버튼을 누른다. 스케줄은 어떤 루틴한 업무를 매일매일 실행해 주는 기능이다. 위쪽 "새 작업"을 누르고 "클로드와 함께 창작하기" 버튼을 누른다.

### [02:57–03:20] "묻지 않고 실행" + 프롬프트 안내
이때 클로드가 질문을 하게 되는데, 먼저 질문 대신 **"묻지 않고 실행"** 기능을 켠다. 그다음 프롬프트를 하나씩 입력한다. 프롬프트 자체는 댓글창에 올려주겠다고 안내한다(세일즈/유입 장치 — 정보로 1회 기록).

### [03:20–03:55] 사례 1 워크플로 설명 — 병원 마케터 가정
자신을 병원 마케팅 마케터라고 가정한다. 임무는 매일 아침 인스타그램의 특정 SNS 계정(예: "바이미 성형외과" 계정)에 새 마케팅 포스트가 올라왔는지 체크해 알림을 보내는 것. 그 계정 주소를 복사·붙여넣기 한다.

### [03:55–04:44] 사례 1 프롬프트 작성
프롬프트를 다음과 같이 구성한다: "나는 병원 마케터인데, 매일 아침 9시에 이 사이트에 새 게시물이 올라왔는지 알림을 주는 기능을 만들고 싶어. 새 게시물이 올라왔으면 해당 게시물 링크와 함께 요약해서, 플레이 MCP를 사용해 나와의 채팅방에 게시물 내용과 링크를 보내줘."
> "새로운 게시물이 올라왔으면 네가 … 해당 게시물 링크와 함께 요약을 해서 플레이 MCP를 사용해서 나와의 채팅방에 해당 게시물의 내용과 링크를 보내 줘." [04:18]

### [04:46–05:11] 사례 1 작업 생성·예약 완료
프롬프트를 넣으면 클로드가 돌면서 태스크를 만든다. 완료를 확인하고 "일정" 버튼을 누르면 예약된다. 스케줄에 들어가면 "인스타그램 바이미 데일리 체크"라는 코워크 스킬이 완료된 것을 볼 수 있다.

### [05:11–05:33] 사례 1 실행 전 주의 + 테스트 실행
테스트로 돌려본다. 실행 전 주의점: 편집에 들어가 이것을 **"묻지 않고 실행"** 버튼으로 켠 다음 저장하는 것을 추천. 그 후 "지금 실행"을 눌러 돌린다.

### [05:33–06:16] 사례 1 실행 — 크롬 MCP 크롤링
예약된 작업이 돌면서 처리된다. 미리 켜둔 인스타그램 창을 활용해 동작한다. 인스타그램 바이미 계정이 뜨면서, 클로드 for 크롬 MCP가 크롤링을 시작한다. 사용자가 건드리지 않고 버튼만 눌렀는데도 알아서 크롤링한다.
> "제가 건드리지 않았는데도 불구하고 저는 이제 버튼만 눌렀습니다. 이 친구가 알아서 이제 크롤링을 하고 있습니다." [06:09]

### [06:16–06:56] 사례 1 결과 — 카카오톡 알림 수신

### [08:10–08:38] 사례 2 스케줄 생성 — 매일 오전 9시

### [09:00–09:36] 구독 티어 팁(중간 삽입)

### [10:27–11:16] 사례 3 — X → Threads 자동 포스팅 소개·실행
세 번째 케이스. 화자가 실제로 쓰레드(Threads)에서 몇 번 돌려본 것으로, X(엑스)의 최신 글을 긁어 번역한 뒤 "쓰레드 데일리"에 올리는 자동화다. 실행하면 클로드가 컴퓨터를 연결해 태스크를 하나씩 만들며 처리한다. 크롬 브라우저를 컨트롤해 X 사이트에 들어가 여러 글을 긁고, 적절한 게시물을 번역한 뒤 Threads에 자동 포스팅한다.

### [11:16–12:03] 사례 3 진행 — 크롤링·번역·붙여넣기
브라우저가 계속 스냅샷을 찍으며, 생각을 거쳐 적절한 글을 찾아 번역한다. X에서 콘텐츠를 가져온 뒤, 크롬 창을 조절해 Threads에 들어간다. X에서 긁은 글을 한글로 번역해 붙여넣고, 코드가 작동하며 실행된다. 이어서 포스트 버튼을 누른다.
> "X에서 긁었던 글을 한글로 번역을 한 다음에 여기다가 붙여 넣는 거를 확인을 해 볼 수가 있습니다." [11:43]

### [12:03–12:26] 사례 3 결과 — Threads 포스팅 완료
포스팅이 완료되고, 프로필에 들어가면 자신의 톤(깔)에 맞게 글이 올라와 있음을 확인할 수 있다. [의견] 이런 Threads 자동화, X 자동화가 다 가능하며 매우 쉽게 활용할 수 있다고 강조.

### [12:26–13:01] 확장 활용 — 리뷰 크롤링·이커머스 이관
[의견] 코워크는 이제 정말 필수 요소가 됐고, 루틴 업무도 다 자동화 가능하다고 정리. 예로 특정 카페에 들어가 리뷰를 긁거나, 이커머스에서 자사몰(자사 쇼핑몰)의 리뷰를 에이블리(Ably)에 올리거나 반대로 에이블리 리뷰를 자사몰에 올리는 작업까지 전부 가능하다고 언급.

### [13:01–13:32] 사용 도구 요약 + 격차 주장
실제 사용한 도구는 플레이 MCP와 클로드 for 크롬 두 개뿐이며, 코딩 지식 없이 자연어만으로, 단지 로그인 상태에서 처리했다고 강조한다. [의견] 정말 쉬운 수단이며, 앞으로 코워크를 쓸 수 있는 사람과 없는 사람의 격차가 계속 벌어질 것이라고 주장. 시청자에게 이 툴로 자신만의 장기를 발전시켜 나가길 추천한다.
> "앞으로는 클로드 코워크를 쓸 수 있는 사람이랑 없는 사람이랑 어 격차가 계속해 가지고 벌어질 것 같습니다." [13:18]

### [13:32–14:17] 마무리 — 다음 기능 예고 + 세일즈
오늘 코워크 소개와 유즈 케이스를 다 보여줬다며, 그 외 라이브 아티팩트·디스패치·커스터마이즈·스킬·프로젝트 등 많은 기능은 추후 강의에서 다루겠다고 예고. 이런 영상이 멤버십에 계속 올라오니 관심 있으면 멤버십 영상도 구독해 보길 추천(세일즈/유입). 프롬프트를 어떻게 받는지는 댓글로 안내할 테니 댓글을 달면 방법을 가이드하겠다고 안내하며, 짧은 영상 시청에 감사 인사로 마친다.
