## 요약 (주제별)

### 오픈클로(OpenClaw)가 무엇인가
- 오픈소스 **개인 AI 비서 게이트웨이**. 와츠앱·텔레그램·슬랙·디스코드 같은 메신저에 AI를 직접 연결해 준다. 12개 넘는 채널을 지원한다.
- ChatGPT 앱처럼 남의 서버에 의존하는 게 아니라 **내가 원하는 서버에서 직접 구동**한다. 어떤 AI 모델을 쓸지, 데이터를 어디로 보낼지 전부 사용자가 통제하는 것이 핵심이다.
- 만든 사람은 오스트리아 개발자로, 이전에 회사를 크게 매각했고 3년간 43개 프로젝트를 실험한 끝에 그중 하나가 크게 성공했다(주장/미검증). 최근 OpenAI에 합류했다.
- 이름 변천: 원래 `클로드봇`(Claude에서 따옴) → Anthropic의 상표권 요청으로 개명 → `몰트보`(발음 문제로 3일 만에 재변경) → `오픈클로`. 일주일 만에 이름이 세 번 바뀌었다.
- GitHub 스타 14만5,000개 이상, 포크 2만 개 이상(주장/미검증).

### 만드는 3단계 절차
- 크게 세 단계: **① 오픈클로 설치 → ② AI 모델 연결 → ③ 레일웨이(Railway)에 배포**.
- 셋업 자체는 터미널에서 명령 몇 줄이면 끝난다. 화자는 상세 절차를 영상 설명란의 가이드 문서 링크로 대체한다(세일즈/유입 장치).

### AI 모델 연결의 갈림길
- **로컬 전용(내 컴퓨터)**: ChatGPT 구독으로 바로 연결 가능. Plus 월 20달러, Pro 월 200달러이며 추가 비용 없이 GPT-5를 쓸 수 있다(주장/미검증).
  - 단, 이 방식은 **내 컴퓨터에서만** 되고 서버 배포 시엔 못 쓴다. 게다가 OpenAI 인증 방식을 우회하는 것이라 **밴(ban) 위험**이 있다.
- **서버 배포용**: 반드시 **API 키 방식**을 써야 한다. 화자 추천은 **Fireworks AI**[불명확: 자막상 "파워스/파이어스/5월스 AI"로 음차됨]에서 GLM 계열 모델[불명확: 자막 "GLM5"]을 쓰는 것. 가성비가 매우 좋다고 주장한다.
  - 가격은 100만 토큰당 [불명확: 자막에 "0.8달러"와 "8달러"가 혼재]. OpenAI/Anthropic 키도 가능하나 훨씬 비싸므로 예산이 넉넉할 때만 권장.
  - 정리: 내 컴퓨터 전용 → ChatGPT 구독 / 서버 24시간 구동 → Fireworks AI API 키.

### 레일웨이(Railway) 배포와 설정
- 레일웨이를 택한 이유: GitHub에 코드를 올리면 자동 배포, 서버가 죽으면 자동 재기동, 가격이 저렴하다.
- 절차: 레일웨이 가입 → GitHub 프로젝트 업로드 → 대시보드에서 자동 배포.
- 사용자가 직접 해야 할 것: **① 저장 공간(storage) 추가, ② 환경 변수 3개 입력**. 나머지 설정은 프로젝트에 다 포함되어 건드릴 게 없다.
- 배포가 끝나면 주소가 나오고, 그 주소로 들어가면 설정 화면이 떠서 텔레그램 같은 메신저를 연결하면 된다.

### 비용 (주장/미검증)
- 레일웨이 서버비 월 약 5달러.
- Fireworks AI 사용량은 달마다 다르지만 추정 5~25달러.
- 합계 월 약 30~40달러. 화자는 개인 AI 비서치고 나쁘지 않은 금액이라고 평가한다([의견]).

### 보안 리스크
- 오픈클로는 보안 이슈가 매우 많다. 2026년 1월 말 보안 감사에서 총 **512개 취약점**이 발견됐고 그중 **8개가 크리티컬** 등급(주장/미검증).
- 가장 심각한 것: **악의적 링크를 한 번 클릭하면 공격자가 내 AI 비서를 장악**할 수 있다.
- 플러그인 마켓 `클로드허브(ClawHub)`[불명확: 자막 "클루트허브"]에 5,700개 넘는 플러그인 중 **341개가 악성**(주장/미검증). 가짜 암호화폐 지갑·유틸리티로 위장해 악성 코드를 심는 사례.
- **프롬프트 인젝션**: 이메일·웹페이지에 숨겨진 악의적 명령을 AI가 읽고 실행할 수 있음. 화자는 이를 AI 에이전트 전반의 근본적 한계로 본다([의견]).

### 화자가 실제 지키는 보안 수칙
1. **강력한 비밀번호** 설정(글자 하나로도 설정 가능하므로 절대 그러지 말 것).
2. **항상 최신 버전 유지**(2~3일마다 보안 업데이트, 한 번에 40개 넘는 취약점이 패치되기도 함).
3. **플러그인 함부로 설치 금지**, 특히 새로 만들어진 계정이 올린 것은 의심.
4. **중국 AI 모델(GLM·딥시크) 주의**: 반드시 Fireworks나 Together AI를 경유해 미국 서버에서 돌릴 것. 중국 회사에 직접 연결하면 데이터가 전부 중국 서버로 가므로 직접 연결 금지.
5. **민감한 개인정보·금융 데이터는 넣지 말 것**. AI 비서에는 일정 관리·메모 정리 같은 가벼운 작업 위주로 맡기는 게 안전.

### 세일즈 구조 (정보로서 1회 기록)
- 구독자 이벤트: "오픈클로로 나만의 AI 비서 만드는 법" 전자책 배포. 참여 조건은 좋아요·댓글·구독·저장 완료 후 댓글에 "전자책"이라 남기면 링크 전송.
- 상세 셋업 명령은 영상 설명란의 가이드 문서 링크로 유도.

### [04:16–05:07] 화자가 실제 지키는 보안 수칙 5가지
보안 이슈가 있다고 못 쓰는 게 아니라 제대로 알고 대비하면 안전하게 쓸 수 있다며, 화자가 실제로 지키는 것들을 나열한다.
- **첫째, 강력한 비밀번호.** 글자 하나로도 설정되니 절대 그러지 말 것.

- **셋째, 클로드허브 플러그인은 함부로 설치하지 말 것.** 특히 새로 만들어진 계정이 올린 건 의심.
- **넷째, 중국 AI 모델 주의.** GLM·딥시크 같은 중국 모델을 쓰려면 반드시 Fireworks나 Together AI를 경유해야 한다 — 같은 모델이라도 미국 서버에서 돌아가 데이터가 중국으로 가지 않는다. 중국 회사에 직접 연결하면 데이터가 전부 중국 서버로 가므로 절대 직접 연결 금지.
- **다섯째, 민감한 개인정보나 금융 데이터는 넣지 말 것.** AI 비서에는 일정 관리·메모 정리 같은 가벼운 작업 위주로 맡기는 게 안전하다.
