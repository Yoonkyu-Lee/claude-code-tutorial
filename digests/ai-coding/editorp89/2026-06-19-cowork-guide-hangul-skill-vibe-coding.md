## 요약 (주제별)

### 코워크(Cowork)의 정체와 동작 원리
- 코워크는 Claude Desktop 안의 기능으로, **지정한 폴더 기준으로만 동작**한다. 에이전트는 폴더를 기준으로 움직이기 때문에, 데스크톱(바탕화면)처럼 파일이 많은 곳을 활동 영역으로 지정하면 의도치 않은 파일을 보거나 실수로 지울 위험이 있다. 그래서 작업 전용 폴더를 지정해야 한다.
- 코워크에서 쓸 수 있는 것: 스킬(약간의 제약 있음, 전역만), 커넥터, 검색, 플러그인. **쓸 수 없는 것: 리서치**(채팅에서만 가능). 따라서 보고서·논문처럼 리서치가 필요한 작업은 채팅과 구분해서 쓰는 것이 좋다.
- 화자 [의견]: 자신은 클로드 코드를 쓰기 때문에 코워크를 거의 안 쓰며, 코워크 영상은 이번 한 번만 올릴 것. 단 "코워크가 싫어서가 아니라 좋은 도구"라고 반복 강조.

### 코워크 여는 법 · 권한 모드
- Claude Desktop 메인 화면 가운데 탭을 누르면 코워크로 전환된다.
- 권한은 두 모드: **실행 전 확인**(잠깐 멈춰 질문) vs **묻지 않고 실행**(질문 횟수가 현저히 줄어 거의 질문 안 함). 후자는 편하지만 의도치 않은 작업이 그냥 일어날 수 있다.
- 화자 [의견]: 제어하고 싶으면 "실행 전 확인"이 좋지만, 대부분 귀찮아서 결국 "묻지 않고 실행"을 쓰게 된다.

### 예정됨(스케줄) 기능과 5시간 토큰 타이머 전략
- 코워크의 여러 기능 중 화자가 **유일하게 유용하게 쓴다**고 꼽는 기능이 "예정됨"(예약 작업).
- 배경: 클로드 코드/데스크톱의 사용량은 채팅이 시작되면 **5시간 타이머**가 켜지고, 그 시간 한도를 다 쓰면 초기화되는 방식(주장). 예약 작업을 아침 특정 시각에 걸어 두면 그 시각부터 타이머가 돌아가, 하루 토큰을 유용하게 배분할 수 있다.
- 화자 사용 예: 매일 아침 9시에 "다이제스트 작업"(메일 활동 수집·보고)을 자동 실행 → 9시에 타이머 시작, 오후 2시 초기화 → 오후에 넉넉한 토큰 확보. 만드는 법은 코워크 새 작업에서 "스케줄 작업 하나 추가하고 싶은데 어떻게 해야 되냐"고 한국어로 자연스럽게 물어보면 된다(외국인 기준 도구라 기본은 영어지만 한글로 시작해도 됨).
- 예정됨도 결국 코워크에서 돌아가므로, **지정 폴더의 파일을 활용**할 수 있다(예: 폴더 내 엑셀 최신 업데이트만 추려 메일 발송).
- 화자 [의견]: 예약 발송 기능은 자신에겐 유용하지 않으나, 미리 작업을 받아 두면 좋은 사람에겐 유용.

### 코워크의 나머지 기능(설명 생략분)
- 발송: 아직 베타라 설명 생략. 아티팩트·프로젝트: 코어 기능 아니고 화자 [의견]으로 굳이 쓸 필요 없어 생략. 사용자 지정: 스킬·커넥터 관리 화면이며 채팅의 그것과 완전히 동일 → 코워크만의 특별한 기능 아님.
- 화자 [의견]: 나중에 스킬·에이전트가 코워크에 더 자연스럽게 들어오면 아티팩트 등은 굳이 필요 없을 것.

### 코워크 vs 클로드 코드 — 전역 스킬 vs 지역 스킬
- "폴더 중심 동작이면 클로드 코드와 차이 없지 않냐"는 질문에 대해, 핵심 차이는 **스킬**이다. 코워크에서는 스킬을 **전역으로만** 쓸 수 있다.
- 스킬 비유: 에이전트("클사원")는 똑똑하지만 어리버리해서, 디테일한 지시가 필요할 때 스킬(=매뉴얼)을 쓴다. CLAUDE.md는 전사 공통 매뉴얼, 스킬은 부서/업무별 세부 매뉴얼. 화자는 스킬을 "에이전트의 꽃"이라 부른다.
- 문제: 스킬이 좋아서 많이 두면, 에이전트가 어떤 매뉴얼인지 살펴보는 시간이 필요해 비효율. 책상(작업)별로 관련 스킬만 두면 찾기 쉬워 효율적 → 이것이 **지역 스킬**의 이점.
- 관리 기준(화자 [의견], 절대 기준 아님): 20~30개는 전역으로 괜찮음, "100개 미만이면 OK"라는 말도 있지만 100개는 과하다고 봄. 많아지면 클로드 코드로 이동 — 클로드 코드는 폴더에 지역 스킬을 지정하면 전역을 안 보고 지역 스킬로만 작업하는 경향이 생겨 효율적. 이 필요성은 스킬을 많이 써 봐야 체감된다.

### 실습 1 — 한글(HWPX) 스킬로 사업자 등록증 가상 작성
- 목표: 한글 스킬을 써서 조사 → 질의응답 → 템플릿 한글 문서 작성. 예시로 "법인 설립 신고 및 사업자 등록 신청서"를 가상으로 채워 봄. 화자는 실제 사업자 등록 경험이 없어 입력 내용은 부정확할 수 있음을 명시.
- 한글 스킬 정의: 화자가 만든, **템플릿 있는 한글 문서 작성에 특화**된 스킬. 글로벌 에이전트가 한글 양식·표를 잘 못 쓰므로, 양식을 복원·준수하며 작성하도록 절차를 나열한 것.
- 설치: 화자 GitHub 레포지토리 주소를 복사(링크는 더보기에) → 코드 탭에서 아무 폴더에서나 "이 스킬 Claude Desktop 전역 스킬로 저장해 줄래?"라고 요청하면 설치됨. "**전역 스킬로 저장**"이라고 말하는 것이 중요. 화자 [의견]: 정석은 아니지만 편한 방법.
- 설치 후: Claude Desktop을 완전 종료(맥 Cmd+Q, 윈도우는 작업 표시줄 아이콘 우클릭 종료) 후 재시작. 스킬은 전역이라 채팅·코워크·코드에서 모두 공유되어 슬래시 입력 시 `HWPX`가 뜬다. (영상 중 일부 화면에서 스킬이 안 뜨는 것을 발견해 "버그/에러 같다"고 함 — 그래도 에이전트는 스킬 존재를 알고 있음.)
- 작업 절차: ① 작업 폴더 지정(HWPX 전용 폴더 새로 만들어 열기, "항상 허용") → ② 국세청에서 "사업자 등록 HWPX" 검색해 빈 양식지 다운로드 후 그 폴더로 이동 → ③ "현재 폴더에 무슨 파일 들어 있게?"로 인식 확인 → ④ "SaaS 사이트를 만들어 판매하려는데, 문서 작성 전에 뭘 입력·준비해야 하는지 웹 검색으로 알려 줘"로 조사 → ⑤ 이름/주소 등 가상 정보와 함께 "이 스킬을 활용해서 사업자 등록증 파일을 충실하게 작성해 줘"로 문서 작성.
- 조사 결과 예시(에이전트가 알려준 것): 사업장 주소는 집 주소도 가능, 통신 판매 신고 추가 필요, 거래 50회 미만이면 면제 대상 가능 등.
- 화자 [의견]: 스킬 없이 하면 잘 안 되므로 대비를 위해 스킬 없이도 한번 해 보길 권함. HWPX 스킬 분파가 많은 이유는 각자 용도에 맞게 조정하기 때문이며, 화자 것은 "양식 최대한 준수 + 범용성"에 초점. 스킬은 skills.sh 같은 사이트에서 용도별로 찾아 쓸 수 있음(스킬 다수 등록, 화자 코드도 올라가 있음 — 버린 구버전은 설치하지 말라고 언급).

### 실습 2 — 바이브 코딩으로 "스킬 시각화" 웹 프로그램 만들기
- 목표: 내 PC의 전역·지역 스킬 구성을 눈으로 볼 수 있게 하는 웹 프로그램을, **react flow**(노드-엣지 그래프/순서도 시각화 도구)로 만들기.
- 절차: 새 작업 → 새 폴더 생성(예: "스터디 look up") + 다른 폴더 체크 해제(활동 영역 한정) → "나는 클로드/클로드 코드/코워크를 쓰는 사람인데, 전역·지역 스킬이 어떻게 구성돼 있는지 눈으로 확인하는 웹 프로그램을 react flow로 간단히 만들어 보자"로 요청.
- 화자 [의견]: 바이브 코딩은 코워크보다 클로드 코드가 더 자연스럽지만, 코드 작성·파일 생성·구조 생성·실행이 코워크에서도 가능해 크게 다르지 않음. 완벽한 프로그램이 아니라 "이런 것도 만들 수 있다"에 초점(진짜 만들면 1~2시간 걸려 영상으로 부적합). react flow는 화자가 그 도구의 존재를 알아서 지정한 것이며, 그냥 만들라 해도 잘 됨.
- 결과: 에이전트가 PC 전역/지역 스킬을 스캔(전역 15개 등 보고, 지역 스킬은 "현재 선택 폴더 없다"고 함 — 폴더 탐색이 충분치 않아 수정 필요). 아티팩트로 열면 안 보였고, **크롬에서 열면 정상**으로 전역/플러그인 스킬이 이름별 분류되어 그래프로 표시됨.
- 화자 [의견]: 이 프로그램을 예전에 열심히 만들어 봤으나 "스킬 관리는 클로드 코드에게 시키는 게 맞지, 시각화는 의미 없다"고 판단해 지웠음. 다만 구조를 눈으로 보고 싶은 사람에겐 유용할 수 있어 재미 삼아 소개.

### 세일즈·유입 구조
- 한글 스킬 GitHub 레포 링크는 영상 더보기에 제공. 스킬 배포처로 skills.sh(다수 스킬 등록) 소개. 별도 유료자료·단톡방 유입 언급은 없음.

## 3줄 오리엔테이션
- Claude Desktop의 코워크(Cowork)는 "지정 폴더 기준으로 동작하는 에이전트"이며, 채팅/코드와 스킬·설정을 공유한다 — 핵심 차이는 코워크가 스킬을 전역으로만 쓴다는 점.
- 전반부는 코워크의 동작 원리·권한·예정됨(스케줄)·전역 vs 지역 스킬 개념 설명, 후반부는 한글(HWPX) 스킬로 사업자 등록증 가상 작성 + react flow로 스킬 시각화 웹앱을 바이브 코딩하는 실습.
- 화자는 자신이 클로드 코드 사용자라 코워크를 거의 안 쓰지만, 요청이 많아 이번 한 번만 정리한다는 톤.

## 전문 — 시간순 워크스루

### [00:00–00:57] 인트로: 코워크 영상 요청에 답해 한 번만 정리
요청이 있어 코워크 사용의 대부분을 커버하는 영상을 너무 길지 않게 올린다고 시작한다. 오늘 실습은 **일반 업무 + 바이브 코딩**을 한꺼번에 진행. 코워크 사용은 생각보다 어렵지 않고 결국 클로드 코드에서 공유하는 사용법이 많이 들어가 있어 설명할 게 많지 않다고 한다.
- [의견] "코워크 영상은 딱 한 번만 올리고 더 안 올리겠습니다." 이유는 자신이 클로드 코드를 쓰기 때문에 코워크를 굳이 쓸 일이 없어, 거의 안 쓰는 도구로 강의하는 게 웃기다고 느껴서. 단 "코워크가 싫어서가 아니다"라고 반복 강조.
> "코워크 좋은 도구입니다." [00:56]

### [00:57–02:07] 동작 원리: 폴더 기준 동작 + 쓸 수 있는/없는 기능
에이전트는 **폴더를 기준으로 동작**한다. 코워크 화면 아래에서 작업 폴더를 정하는데, 예컨대 데스크톱(바탕화면)을 지정하면 파일이 매우 많은 곳이 활동 영역이 되어, 의도하지 않은 파일을 보거나 실수로 지울 수 있다. 그래서 "코워크는 반드시 지정한 폴더 기준으로 동작한다"는 것을 알아야 한다.
- 쓸 수 있는 것: 스킬(약간 제약, 전역만), 커넥터, 검색, 플러그인. 쓸 수 없는 것: **리서치**.
- [의견] 리서치는 채팅에서만 되는 기능이라, 보고서·논문 작업이 많으면 채팅과 구분해서 쓰는 게 좋다.
> "지정한 폴더 기준으로 동작한다라는 걸 우리가 알고 있어야 됩니다." [01:43]

### [02:08–03:07] 코워크 여는 법과 권한 모드
Claude Desktop 메인 화면 가운데 탭을 누르면 코워크로 전환된다(매우 심플). 권한은 두 모드 — **실행 전 확인**과 **묻지 않고 실행**. "묻지 않고 실행"을 켜면 에이전트가 질문하는 횟수가 현저히 줄어 거의 질문을 안 한다. 편하지만 의도하지 않은 작업이 그냥 일어날 수 있다는 게 문제.
- [의견] 웬만하면 안 열어 주는(=실행 전 확인) 게 좋지만, 결국 귀찮아서 "묻지 않고 실행"을 쓰게 된다. 제어하고 싶으면 손바닥(멈춤) 모드를 쓰는 게 좋다.

### [03:08–06:06] 유용 기능: 예정됨(스케줄)과 5시간 토큰 전략
코워크의 여러 기능 중 화자가 유용하게 쓰는 것은 "예정됨"(예약 작업). 배경 설명: 클로드 코드는 채팅이 시작되면 **5시간 타이머**가 켜지고, 그 한도를 다 쓰면 시간이 지나 초기화되는 방식(주장/미검증). 그래서 하루 시간을 유용하게 배분하는 데 예정 대비가 좋다.
- 화자 예: 매일 아침 9시에 채팅 하나(다이제스트 작업 — 메일 활동 수집·보고)를 자동 실행. 9시에 타이머 시작 → 오후 2시 초기화 → 오후 작업용 토큰 확보. 아침 8시 인사 예약이면 8시~1시, 1시~6시 식으로 5시간 전략을 짤 수 있다.
- 만드는 법: 예정됨에서 새 작업 → "나 스케줄 작업 하나 추가하고 싶은데 어떻게 해야 되냐"고 물으면 에이전트가 질문하며 자연스럽게 만들어 준다. 외국인 기준 도구라 기본 영어지만 한글로 시작해도 됨.
- 예정됨도 코워크에서 돌아가므로, 지정 폴더의 파일을 함께 활용 가능(예: 폴더 내 엑셀 최신 업데이트만 추려 메일 발송).
- [의견] 예약 발송 기능은 자신에겐 유용하지 않지만, 미리 작업을 받아 두면 좋은 사람에겐 유용.
> "결국은 저는 코워크에서는 예정됨 하나만 되게 유용하게 쓰고 있는데" [06:22]

### [06:06–06:43] 사용자 지정 = 채팅과 동일, 나머지 기능 정리
사용자 지정은 스킬·커넥터 관리 화면으로, 채팅에서 보던 것과 완전히 동일 → 코워크만의 특별한 기능이 아니다. 발송은 베타라 설명 생략, 아티팩트·프로젝트는 코어 기능이 아니라 생략.
- [의견] 아티팩트·프로젝트는 굳이 쓸 필요 없다고 봄. 나중에 스킬·에이전트가 코워크에 더 자연스럽게 들어오면 더욱 필요 없어질 것이라, 아예 설명에서 뺐다고 함.

### [06:43–09:35] 코워크 vs 클로드 코드: 전역 스킬 vs 지역 스킬
"폴더 중심이면 클로드 코드와 차이 없지 않냐"는 물음에, 핵심 차이는 코워크가 **스킬을 전역으로만** 쓸 수 있다는 점. 스킬을 매뉴얼에 비유:
- 에이전트("클사원")는 똑똑하지만 어리버리해서, 아주 디테일한 지시가 필요할 때 스킬(매뉴얼)을 쓴다. CLAUDE.md는 전사 공통 매뉴얼, 스킬은 부서/업무별 세부 매뉴얼. 화자는 스킬을 "에이전트의 꽃"이라 부른다.
- 문제: 스킬이 너무 많으면 에이전트가 어떤 매뉴얼인지 살펴보는 시간이 필요해 비효율. 업무용 책상마다 관련 스킬만 두면 찾기 쉬워 효율적 → 지역 스킬의 이점. 그런데 코워크는 아직 지역 스킬을 못 써서, 전역 스킬을 너무 많이 두지 않는 게 좋다.
- 관리 기준([의견], 절대 기준 아님): "100개 미만이면 OK"라는 말이 있지만 100개는 과함. 20~30개면 전역으로 괜찮음. 많아지면 클로드 코드로 이동해야 하는데, 클로드 코드는 폴더에 지역 스킬을 지정하면 전역을 안 보고 지역 스킬로만 작업하는 경향이 생겨 효율적.
> "스킬은 에이전트의 꽃이라고 부르는데" [07:53]

### [09:35–10:52] 실습 개요: 한글 스킬 + 사업자 등록증 가상 작성
바로 실습으로 넘어간다. 오늘은 **한글 스킬**로 조사 → 조사 결과 검토 → 질의응답 → 템플릿 있는 한글 문서 작성. 예시로 사업자 등록증(법인 설립 신고 및 사업자 등록 신청서)을 가상으로 완성해 본다. 화자는 나중에 실제로 사업자 등록을 해 보려 하며, 오늘은 연습 겸 시연.
- [의견] 실제 사업자 등록 경험·사전 조사가 전혀 없어 입력 내용이 부정확할 수 있음을 명시. 중요한 건 "한글 스킬을 쓰면 템플릿 채우는 한글 문서 작업을 정말 잘해 준다"는 점.
- 한글 스킬 정의: 화자가 만든, 템플릿 있는 한글 문서 작업에 특화된 스킬. 글로벌 에이전트가 한글 양식·표를 잘 못 쓰므로, 양식을 복원·준수하며 작성하도록 절차를 나열한 것.

### [10:52–12:09] 한글 스킬 설치
GitHub 레포지토리에 한글 스킬이 있다(링크는 더보기). 방법: 레포 주소를 복사 → 코드 탭으로 이동 → 아무 폴더에서나 "이 스킬 Claude Desktop 전역 스킬로 저장해 줄래?"라고 요청하면 설치가 끝난다.
- "**전역 스킬로 저장**"이라고 말하는 것이 매우 중요.
- [의견] 정석 방법은 아니지만 편하게 설치하라고 이렇게 소개. 이 스킬은 많은 사람이 포크해 발전시켜 널리 퍼졌고, 화자 것이 첫 번째 혹은 두 번째쯤(누구 걸 보고 만든 게 아니라 만들고 보니 수요가 많았다는 사연).

### [12:09–14:11] 완전 종료·재시작, 전역 공유, 일부 버그
설치 후 Claude Desktop을 **완전 종료**해야 반영된다(맥 Cmd+Q, 윈도우는 작업 표시줄 아이콘 우클릭 종료). 재시작하면 스킬은 전역이라 채팅·코워크·코드에서 모두 공유 → 슬래시 입력 시 `HWPX`가 여러 곳에서 뜬다.
- 다만 일부 화면에서 스킬이 안 뜨는 것을 발견하고 "로컬로 돼 있어서 그런가, 버그/에러 같다"고 함. 그래도 에이전트에게 "이 PC에 설치된 HWPX 스킬 쓸 줄 알아?"라고 물으면 알고 있다(슬래시엔 안 떠도 인지함). [불명확: 특정 화면에서 슬래시 목록에 스킬이 안 뜨는 정확한 원인]

### [14:11–16:07] 작업 폴더 지정 + HWPX 양식 준비
작업은 폴더 지정부터. HWPX 전용 작업 폴더를 새로 만들어 열고("스터디 HWPX 스킬" 등), "항상 허용" → "묻지 않고 실행"으로 편의성 확보. 그다음 양식이 필요한데, "사업자 등록 HWPX"로 검색하면 국세청에서 빈 양식지를 받을 수 있다. 다운로드한 양식 파일을 작업 폴더로 이동시킨다. "현재 폴더에 무슨 파일 들어 있게?"라고 물으면 폴더를 보고 있어 그 파일을 인식한다.

### [16:07–17:04] 웹 검색으로 준비물 조사
작업 시작 질문: "나 사업자 등록을 하려는데 SaaS 사이트를 하나 만들어 판매해 볼까 해. 문서 작성 전에 내가 뭘 입력해야 하는지, 뭘 준비해야 하는지 웹 검색을 통해 알려 줄래?" 그러면 에이전트가 웹 검색으로 조사해 준다(SaaS=월 구독 사이트). 예전엔 한글 문서와 검색창을 나란히 열고 수기 입력했는데, 이제 그걸 대신 시킨다.
- 조사 결과 예: 사업장 주소는 집 주소도 가능, 통신 판매 신고 추가 필요, 거래 50회 미만이면 면제 대상 가능 등 화자가 몰랐던 것까지 정리됨.

### [17:04–19:03] 스킬로 문서 작성 + 스킬 생태계
이름(편집자 피고)·주소(강남 등) 가상 정보를 주고 "이 스킬을 활용해서 사업자 등록증 파일을 충실하게 작성해 줘"라고 하면, 조사 내용과 함께 스킬을 열어 문서 구조를 확인한 뒤 셀 입력(복잡하지만 잘 처리)을 수행한다.
- [의견] 스킬 없이 하면 잘 안 되므로, 대비 차원에서 스킬 없이도 한번 해 보길 권함. HWPX 스킬 분파가 많은 이유는 각자 용도에 맞게 조정하기 때문. 화자 것은 "양식 최대한 준수 + 범용성"에 초점.
- 스킬 배포처: skills.sh 사이트에 스킬이 매우 많이 등록돼 있음("78만 개"라는 화면 언급 — 수치는 주장/미검증). 화자 코드도 올라가 있으며, 버린 구버전은 설치하지 말라고 안내.
> "스킬이 있어야 되는 이유는 스킬 없이도 한번 해 보시는 과정이 꼭 필요해요." [17:46]

### [19:03–20:11] 결과 비교 및 1차 실습 마무리
미리 작성해 둔 결과를 좌우로 비교해 보여 준다 — 양식에 필요한 내용이 잘 들어가 있음. 실제 신고 시 1차로 이렇게 문서를 작성하고 디테일한 수정만 조금 하면 편하게 작성 가능. "어차피 검색하며 입력할 거면 이렇게 하는 게 요즘 일하는 방식"이라며, 에이전트+스킬+코워크의 기초 사용을 마쳤다고 정리.

### [20:11–22:27] 바이브 코딩 실습: 스킬 시각화 웹앱 (react flow)
남은 실습은 바이브 코딩. 내 PC의 전역·지역 스킬 구성을 눈으로 볼 수 있는 웹 프로그램을 만든다. 절차: 새 작업 → 새 폴더 생성("스터디 look up" 류) + 다른 폴더 체크 반드시 해제(활동 영역 한정) → "나는 클로드/클로드 코드/코워크를 쓰는 사람인데, 전역·지역 스킬이 어떻게 구성돼 있는지 눈으로 확인하는 웹 프로그램을 react flow로 간단히 만들어 보자."
- react flow는 네모 박스와 점선(엣지)으로 순서도/그래프를 그리는 도구. 스킬이 어디 있는지 보여주기에 적합해 화자가 지정(그냥 만들라 해도 잘 됨).
- [의견] 바이브 코딩은 코워크보다 클로드 코드가 더 자연스럽지만, 코드·파일·구조 작성·실행이 코워크에서도 가능해 크게 다르지 않음. 완벽한 프로그램이 아니라 "이런 것도 만들 수 있다"에 초점(진짜 만들면 1~2시간 걸려 영상 부적합).

### [22:27–24:15] 생성 과정과 한계, 아티팩트 실행
에이전트가 PC의 스킬을 전반적으로 스캔 → 전역 스킬 15개, 플러그인 스킬 등을 보고. 다만 지역 스킬은 "현재 선택한 폴더는 없다"고 하며 폴더 탐색이 충분치 않아 보여, 이 부분은 수정이 필요하다고 함. 그럼에도 "스킬 시각화 프로그램을 만들 수 있다"는 가능성으로 진행.
- [의견] 이 프로그램을 예전에 열심히 만들어 봤으나 "스킬 관리는 클로드 코드에게 시키는 게 맞지, 눈으로 보는 건 의미 없다"고 판단해 지웠음. 다만 구조를 눈으로 보고 싶은 사람에겐 유용할 수 있음.
- 완성 후 "다 만든 프로그램을 좀 실행해 볼래?"라고 하면 에이전트가 아티팩트로 열어 보여 준다(그래서 아티팩트를 따로 안 가르침 — 필요하면 알아서 연다). 그런데 아티팩트로는 아무것도 안 보임.

### [24:15–25:07] 크롬에서 열기 → 시각화 결과, 마무리
아티팩트에서 안 보이던 것을 **크롬에서 열면** 정상 표시된다. 전역 스킬·플러그인 스킬이 이름별로 분류되어 그래프로 보인다. 이후 코워크에게 "지역 스킬 스터디 look up 폴더엔 스킬 있어?"라고 물으면 없다고 답하는 식으로 활용 가능. 이렇게 스킬 구성을 눈으로 확인하는 프로그램이 완성됨을 보여 주고, 코워크+한글 스킬+바이브 코딩 실습을 모두 마치며 영상을 마무리한다.
> "더 유용한 영상으로 여러분들에게 찾아오도록 하겠습니다." [25:03]
