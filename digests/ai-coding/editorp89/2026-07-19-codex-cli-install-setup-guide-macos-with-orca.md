## 요약 (주제별)

### 영상 목적과 대상
- macOS(Apple Silicon) 유저를 위한 Codex CLI 설치·실행·세팅 가이드. 그동안 macOS 편을 안 올려("자꾸 까먹어서") 자사 도서 "바이브 코딩 코덱스 완벽 가이드" 구매자에게 사과하며 시작.
- 설치를 오르카(Orca) 에이전트 안에서 진행하되, 오르카 자체는 오늘 주제가 아닌 맛보기. 오르카 상세 사용법·"오르카를 쓰기로 한 이유"는 별도 고급 영상으로 예고.
- 도서 언급은 세일즈 장치: "코덱스 완벽 가이드"로 검색, 판매지수 7,200(주장).

### 핵심 주장 — 앱 코덱스와 CLI는 "동일", 단 멀티세션은 터미널이 유리
- ChatGPT 앱의 코덱스와 Codex CLI는 "완전히 똑같다". CLI에서 남긴 채팅 기록이 앱(코덱스) 프로젝트에 그대로 뜨는 등 완벽 공유되는 동일한 도구(주장).
- 그럼에도 CLI를 쓰는 진짜 이유는 **멀티세션(여러 작업 병렬) 운영에는 터미널이 유리**하기 때문. 에이전트 작업은 4~7분, 때론 1시간+ 걸려 그동안 대기하게 되는데, 채팅창 여러 개를 오가며 병렬 작업하려면 터미널 환경이 강점.
- 현재 ChatGPT 앱 코덱스에 종료(X) 버튼이 안 눌리는 버그가 있다고 지적.
- [의견] 이런 프로그램을 만드는 개발자들이 터미널을 좋아해 터미널 기능을 먼저 만들고 GUI에 나중 반영하므로, 검은 화면이 낯설어도 터미널을 배워두면 반드시 도움이 된다는 논리.

### 오르카(Orca) 설치와 프로젝트(=폴더) 개념
- "오르카 에이전트" 검색 → orca.dev에서 OS별 다운로드. 화자는 macOS Apple Silicon 버전(Homebrew는 예외로 둠). macOS는 드래그 앤 드롭 설치가 정석.
- 첫 화면 환경설정 안내는 "프로젝트 설정으로 건너뛰기"로 무시 가능(지장 없음).
- 오르카에서 "프로젝트"는 곧 **폴더**를 뜻함 — 에이전트는 폴더를 추가해 부리는 구조. 빈 폴더로 시작하는 게 기본. 새 프로젝트 만들기 시 기본 경로가 따라가기 어려우니 "펼치기→위치 변경"으로 지정(화자는 Documents 하위에 "테스트 > 테스트 오르카" 폴더 생성).
- **에이전트는 언제나 폴더 내에서 실행하는 것이 정석**.

### Codex CLI 설치·실행 절차
- 설치: **"코덱스 CLI install"로 구글에서 검색**(구글에서 해야 함) → 맨 위 "install codex" 페이지 → OS를 macOS 선택 → 나오는 설치 명령어 복사 → 오르카 터미널에 붙여넣어 실행.
- 설치 후 "코덱스 시작할 거냐"에 N을 눌러 종료 → 실행법을 먼저 배움. 이후 `codex` 입력→엔터→"Yes, continue"로 실행.
- 로그인: 필요 시 `/logout`으로 로그아웃 시연. `codex` 실행 시 "Welcome to Codex" → "Sign in with ChatGPT"로 브라우저(오르카 통합 브라우저에서 열기 가능) → 구글 계정 로그인으로 설정 완료. (브라우저 안 뜨면 Command+클릭)
- 실행된 코덱스는 그 폴더("테스트 오르카") 기반으로 동작 — 폴더 내용(`.git` 등)을 인식.

### 세팅 — 모델과 리즈닝
- `/model`로 모델 선택. GPT-5.6에 등급 선택지가 있고 성능↓·속도↑ 순서. 화자는 "솔"을 사용, "루나"는 거의 쓸 일 없음, "테라"도 가능(선택지 명칭이 자막상 혼재, `[불명확]`).
- 리즈닝(추론 깊이) 선택: extra high / medium / high / 울트라 등. [의견] **하이(high) 추천**. extra high·울트라("모 리즈닝")는 CLI에서 에이전트를 4개로 쪼개 병렬 작업시키는 것에 불과해 회의적이며, OpenAI 공식 문서도 울트라를 "자주 쓸 일 없는 것"으로 안내한다고 함. 최종 세팅 "GPT-5.6 솔 하이".

### 세팅 — 상태 표시줄(status line)
- `/` → "status line" 검색으로 진입. 스페이스바로 항목 토글. [의견] 추천 6개: **Use team colors / Model with reasoning / Context remaining / 파이브(5시간) 리밋 / Weekly 리밋 / 패스트 모드**.
- 켜면 화면 아래에 모델·리즈닝, 패스트 모드 여부, 남은 컨텍스트 윈도우, 주간 한도(예: 위클리 99% 남음) 등이 표시돼 사용 상태를 확인 가능.

### 입력·단축키 꿀팁
- `/fast`로 패스트 모드 토글.
- 이전 입력을 **키보드 위/아래 키**로 히스토리 탐색·재사용.
- 줄바꿈 **Shift+Enter**, 한 줄 삭제 **Command+Delete**, 전체 삭제 **ESC** 또는 **Control+C / Command+C**. 단, 단축키는 업데이트·환경마다 조금씩 바뀌어 셋 중 하나씩 눌러보라고 안내.
- 나머지 슬래시 메뉴는 영어 설명을 읽으면 감이 오지만 거의 쓸 일 없음.

### `/clear`와 컨텍스트 관리
- `/clear`는 컨텍스트 윈도우에 쌓인 대화 맥락을 비우는 것 — 지우는 게 나쁜 게 아님.
- 비유: "강아지 초콜릿" 대화 뒤 컨텍스트를 안 비우고 "화로회 맛집"을 물으면 에이전트가 "강아지에게 화로회를?"처럼 이상하게 동작할 수 있음. **한 단위 작업이 끝나면 `/clear`로 비우고** 새 주제로 넘어가라고 추천.

### 멀티세션·다중 에이전트 운용(터미널의 핵심 가치)
- 대화를 보관하고 싶으면 멀티세션 유지가 유리. "+"로 새 터미널 탭을 열면 같은 폴더를 공유하되 이전 대화는 남고 새 탭은 새 채팅창.
- 탭을 좌우/상하 분할하면 같은 폴더에서 두 에이전트 동시 운용. Codex CLI 종료는 `/exit` 또는 Control+C 2회 연타.
- 예: 왼쪽 Codex CLI + 오른쪽 Claude Code(`claude --dangerously-skip-permissions`)를 한 폴더에서 동시 사용. [의견] 앱 코덱스 같은 GUI로는 불가(앱은 코덱스만 실행 가능) — 그래서 다중 작업엔 터미널이 유리.
- 오르카는 왼쪽에 폴더별 라인업을 보여줘 여러 폴더를 오가며 작업 가능해 좋은 도구라 평가.

### 대안 환경 — VS Code에서도 동일
- VS Code에서 하려면 오르카 관련은 다 빼고, 상단 터미널 아이콘으로 하단 터미널을 열어 `codex` 실행하면 CLI가 똑같이 실행됨.
- [의견] "완전히 동일한 거예요" — 오르카든 VS Code든 그냥 터미널이든 보여지는 창만 다를 뿐 같은 코덱스이니 오해 말라고 재차 강조.

## 3줄 오리엔테이션
- macOS 유저를 위한 Codex CLI 설치·실행·세팅 가이드. 그동안 macOS 편을 안 올려 도서 구매자에게 사과하며 시작한다.
- 설치 도구로 Orca(오르카) 에이전트를 함께 소개하되 오르카 자체는 오늘 주제가 아니라 맛보기. 코덱스 CLI를 오르카 안에서 설치·실행·세팅하는 흐름으로 진행.
- 핵심 메시지: ChatGPT 앱의 코덱스와 CLI는 "완전히 동일"하지만, 멀티세션(여러 작업 병렬) 운영에는 터미널 기반 도구가 유리하다 — 그래서 터미널을 배워두라는 것.

## 전문 — 시간순 워크스루

### [00:00–00:34] 오프닝 — macOS 편 지각 사과와 도서 언급
macOS(메고 S) 유저를 위한 Codex CLI 설치·실행·세팅 가이드라고 소개하며 시작한다. 앞서 윈도우 편은 올렸는데 macOS 편을 지금까지 안 올렸고, 그 이유는 "별게 없고 자꾸 까먹어서"라고 솔직하게 밝힌다. 자사 출판사 도서 "바이브 코딩 코덱스 완벽 가이드"의 구매자들에게 죄송하다는 말과 함께 영상을 연다. [의견] 설치는 찾아보면 금방 나오는 내용이지만 자사 출판 책이라 직접 찍어 올렸어야 했는데 안 올린 건 잘못한 것 같다고 함.

### [00:34–00:53] 도서 소개(광고)와 "코덱스"라는 단어의 정체
"코덱스 완벽 가이드"로 검색하면 나오는 책이라며 "깜짝 광고"를 한다. 판매지수 7,200으로 꽤 잘 팔린 도서라고 소개(주장). "코덱스"라는 단어는 지금 ChatGPT 앱 안에 들어가 있는 그 녀석이며, 그와 함께 오늘 다룰 것은 터미널에서 실행하는 CLI를 설치·실행·세팅하는 방법이라고 정리한다. 더불어 최근 시청자 요청으로 오르카(Orca)에 세팅한 것을 보여주려 한다고 예고.

### [00:53–01:33] ChatGPT 앱 코덱스 vs CLI — "완전히 똑같다"
미리 나올 질문에 답한다. "ChatGPT 앱의 코덱스랑 뭐가 다른가요?" → "완전히 똑같습니다." 심지어 Codex CLI에서 남긴 채팅 기록이 코덱스(앱) 프로젝트에 다 뜨기 때문에 완벽하게 공유되는 동일한 녀석으로 보면 된다고 함(이따 시연 예고). 다만 현재 ChatGPT 앱 코덱스에 종료(X) 버튼이 안 눌리는 버그가 있다며 빨리 고쳐달라고 언급.

### [01:33–02:20] 왜 CLI를 쓰나 — 멀티세션 운영은 터미널이 유리
"똑같으면 왜 Codex CLI를 쓰나요? 간지나서?"라는 질문에, 간지도 있지만 진짜 이유는 **멀티세션 운영은 터미널이 더 유리하기 때문**이라고 답한다. 에이전트 작업 시간은 짧게는 4분, 길게는 7분, 때론 한 시간 이상 걸리는데 그동안 사용자는 "손가락 빨고" 있게 된다. 그 시간이 비효율적이라 다른 작업도 병렬로 처리하고 싶어지고, 채팅창 여러 개를 왔다 갔다 하며 작업하려면 터미널 환경이 굉장히 유리하다는 게 화자가 보는 터미널의 가장 큰 강점.

### [02:20–03:07] 터미널을 배워두라 — 개발자 우선순위 논리
[의견] 터미널(검은 화면)이 디스거스팅해 보여도 이제는 넘어와야 한다고 권한다. 이유는 "개발자들이 그걸 좋아해서". 이런 프로그램을 만드는 것도 개발자이고, 개발자들은 자기가 좋아하는 터미널 실행 기능을 먼저 만들고 GUI(앱)에는 나중에 반영한다는 것. 실제로 지금 ChatGPT 앱 코덱스의 X 버튼도 안 눌린다는 점을 근거로 든다. 그러니 터미널을 배우면 반드시 도움이 되니, 오늘 영상을 기점으로 Codex CLI를 해보고 "나도 이제 터미널 쓴다"는 자신감을 가져가라고 격려.

### [03:07–03:39] 오르카(Orca)를 함께 다루는 이유
요청이 하도 많아 오르카를 오늘 맛보기로 잠깐 소개한다고 예고. 지금 디테일하게 알아보는 중이고, 원래는 안 쓰려 했는데 써야 할 의미가 하나 생겨버렸다고 함(구체 이유는 뒤에서 "고급 영상에서 다루겠다"고 미룸). 오르카 사용법 자체는 따로 영상을 준비해 올릴 것이고, 오늘은 오르카를 실행해 그 안에서 Codex CLI 설치·실행·세팅까지 한꺼번에 간다고 안내.

### [03:39–04:15] 오르카 설치 — 다운로드
오르카 설치는 "엄청 쉽다". "오르카 에이전트"로 검색하면 맨 위에 orca.dev 사이트가 나오고, 그 사이트에서 데스크톱 OS별 다운로드 버튼을 제공한다. 윈도우 유저는 윈도우용을 쓰면 되고, 화자는 macOS Apple Silicon이라 그 버전을 다운로드했다. Homebrew를 쓰는 사람은 그 방식을 써도 되지만 이 영상에서는 예외로 두고, 다운로드 후 실행하는 것을 전제로 설명한다고 함.

### [04:15–04:44] 클린 삭제 후 클린 설치 시연 준비
설치 직후 첫 화면을 시청자와 함께 보기 위해 화자는 자기 오르카를 클린 삭제한다. 겨우 세팅해 놓은 걸 지웠지만 클린 설치를 보여주기 위함이고, 모든 설정이 날아간 상태이며 어차피 다시 세팅하면 되니 어렵지 않다고 함. 오늘은 오르카에 집중하는 내용이 아니니 오해 말라고 다시 강조하며, 간단히 설치해 Codex CLI 설치·실행 흐름을 보겠다고 안내.

### [04:44–05:01] 오르카 설치 파일 실행(드래그 앤 드롭)
macOS Apple Silicon용을 다운로드하면 설치 파일이 받아지고, macOS에서는 드래그 앤 드롭으로 설치하는 것이 정석이라고 설명. 그 정석대로 실행한다.

### [05:01–05:37] 오르카 초기 화면 — 프로젝트(=폴더) 추가
오르카는 설치 후 첫 화면에서 환경 설정 안내(설치된 에이전트 목록 등)를 보여준다. 화자는 에이전트를 많이 설치해둬서 그게 뜨는데, 이건 무시해도 쓰는 데 지장 없다며 "프로젝트 설정으로 건너뛰기"를 누른다("자유가 필요하니까"). 여기서 말하는 **"프로젝트"는 곧 폴더**를 뜻한다 — 에이전트를 부릴 때는 폴더를 추가해서 부리는 구조이기 때문. 그래서 에이전트를 부리려면 프로젝트 추가로 작업을 시작해야 한다고 설명.

### [05:37–06:16] 빈 폴더로 새 프로젝트 만들기 — 경로 지정
빈 폴더에서 시작하는 게 기본이라며, 프로젝트 폴더가 아직 없다는 가정으로 "새 프로젝트 만들기"를 누른다. 오르카가 기본 경로에 폴더를 만들어주겠다고 하지만 그 기본 경로는 따라가기 어렵다며, "펼치기 → 위치 변경"으로 경로를 바꾼다. 화자는 보통 Documents 폴더 하위에 폴더를 만들어 관리하는 걸 좋아해서, 자기 "테스트" 폴더 하위에 다시 폴더를 추가해 진행하겠다고 함.

### [06:16–06:58] 테스트 오르카 폴더 생성 — 에이전트는 폴더 내 실행이 정석
"테스트" 폴더 하위에 "테스트 오르카"라는 폴더를 하나 만들어 시작한다. **에이전트는 언제나 폴더 내에서 실행하는 것이 정석**이라고 강조. 폴더 선택→열기→프로젝트 생성으로 진행하되, 프로젝트 이름은 폴더명과 동일하게 "테스트 오르카"로 지어준다. 결과적으로 Documents > 테스트 > 테스트 오르카 폴더가 생성되고 거기서 기본 작업이 진행됨을 확인.

### [06:58–07:37] Codex CLI 설치 검색 — install codex 명령어 복사
이 상태를 그대로 두고 Codex CLI 설치로 넘어간다. 설치 문서는 developers.openai.com docs 등으로 길어 찾기 어려우니, **"코덱스 CLI install"로 구글에서 검색**하라고 안내(구글에서 검색해야 한다는 점을 강조 — 가끔 구글로 안 하는 사람이 있음). 검색 결과 맨 위 "install codex" 페이지에서 OS를 macOS로 선택하면 설치 명령어가 나오고, 그 명령어를 그대로 복사한다.

### [07:37–08:03] 오르카 터미널에 붙여넣어 설치 → 실행 방법
복사한 설치 명령어를 오르카에 떠 있는 터미널에 붙여넣고 실행하면 Codex CLI가 설치된다("설치는 어렵지 않습니다"). 설치 후 "코덱스를 시작할 거냐"고 묻는데, 실행하는 법부터 배워야 하므로 N을 눌러 일단 종료한다. 그다음 `codex` 키워드를 입력하고 엔터→"Yes, continue"로 코덱스를 실행할 수 있다고 설명.

### [08:03–08:32] 로그인 — 오르카 통합 브라우저로 ChatGPT 로그인
로그인 안내가 뜰 수 있다며, 화자는 `/logout`(슬래시 로그아웃)으로 로그아웃해 시연한다. 코덱스를 실행하면 첫 실행 시 동글동글 도는 아이콘과 "Welcome to Codex" 인사가 뜨고, "Sign in with ChatGPT"를 골라 웹브라우저를 띄운다. 브라우저가 안 뜨면 링크를 커맨드 키 누른 채 클릭해야 브라우저에서 열린다고 안내. 오르카는 웹브라우저를 자체 제공하므로 "오르카에서 열기"를 하면 오르카 안에서 브라우저가 열리고, 구글 계정으로 계속하기→로그인 진행으로 코덱스 설정이 끝난다.

### [08:32–08:56] 오르카의 브라우저 통합 장점(예고)
[의견] 오르카는 웹브라우저가 통합돼 있다는 게 큰 장점이라고 함(자세한 건 나중 영상 예고). 개발 시 자기가 만든 바이브 코딩 애플리케이션을 웹브라우저 화면으로 확인해야 할 때, 창을 왔다 갔다 할 필요 없이 오르카 안에서 바로 확인할 수 있어 편하다는 것. 추가된 기능이 많지만 오늘 주제가 아니라 넘어가고, 엔터를 누르면 코덱스 진입이 완료된다.

### [08:56–09:27] 코덱스가 폴더 기반으로 동작함 확인
코덱스 설치·실행이 끝났고, 지금 이 코덱스는 "테스트 > 테스트 오르카" 폴더 안에서 실행 중이다. "이 폴더에 뭐가 있지?"라고 물으면 코덱스도 그 폴더 위치를 알고 있고, 폴더 안에 `.git` 폴더가 있어 이미 세팅돼 있다는 식으로 답해준다. 즉 코덱스는 **폴더 기반으로 동작**함을 확인. 여기까지가 아주 기본적인 실행.

### [09:27–10:07] 세팅 1 — 모델 선택(GPT-5.6 솔라/루나)
이제 세팅으로 넘어간다. 첫 번째는 모델 세팅. `/model`(슬래시 모델) → 엔터로 진입하면 GPT-5.6에 솔라/루나 순서로 선택지가 있고, **솔라→루나 순서로 성능이 낮아지고 속도는 빨라진다**. 화자는 지금 "솔"을 쓰고 있고, 원하면 "테라"를 써도 된다고 함(자막상 명칭 혼재). "루나"는 거의 쓸 일 없다고 봄. 모델을 선택하면 리즈닝(추론) 수준을 정할 수 있다. [불명확: 자막에 "테라"·"솔라 루나" 등 모델 등급 명칭이 혼재해 정확한 등급명은 불확실]

### [10:07–10:32] 세팅 1 — 리즈닝 수준은 "하이" 추천
리즈닝은 에이전트가 스스로 "혼잣말(추론)"하는 깊이를 정하는 것으로, extra high / medium / high / (모) 리즈닝 등 선택지가 있다. [의견] 화자는 **하이(high)를 추천**한다. extra high나 "모 리즈닝"(울트라)은 특히 Codex CLI에서 그냥 에이전트를 4개로 쪼개 병렬 작업시키는 것에 불과해 좋은지 잘 모르겠다고 함. 실제로 OpenAI 공식 문서에서도 울트라는 "권하지 않는다"기보다 "자주 쓸 일 없는 녀석"으로 안내돼 있다며, 하이로도 충분히 다 해결할 수 있으니 하이를 추천하고 엔터를 누르면 된다고 정리. 설정 결과 "GPT-5.6 솔 하이"로 세팅됨.

### [10:32–11:02] 세팅 2 — 스테이터스 라인(상태 표시줄) 켜기
화면 아래 상태 표시가 없는 사람을 위해 세팅 하나를 더 알려준다. `/` → "status line" 검색 → 엔터로 들어가면 메뉴들이 나오고, X(체크) 모양으로 표시돼 있다. 스페이스바로 각 항목을 켰다 껐다 할 수 있고, 켤 때마다 아래에 모델·리즈닝 정보, 컨텍스트 리메이닝(남은 컨텍스트), 파이브(5시간) 리밋, 위클리(주간) 리밋 등이 나타났다 사라진다. 예로 위클리(일주일치 토큰)가 99% 남았다는 정보를 보여준다고 설명.

### [11:02–11:45] 세팅 2 — 추천 항목 6개 토글
[의견] 화자 추천 항목은 맨 위 **Use team colors**, 그다음 **Model with reasoning**, **Context remaining**, **파이브 리밋(5시간 한도)**, **Weekly 리밋**, 그리고 아래로 내려가 **패스트 모드(Fast mode)** — 총 6개를 스페이스바로 토글한 뒤 엔터. 그러면 화면 아래에 현재 모델·리즈닝, 패스트 모드 여부, 남은 컨텍스트 윈도우, 주간 코인(한도) 잔량이 표시되어, 자기가 제대로 쓰고 있는지 확인할 수 있게 된다.

### [11:45–12:12] 세팅 3 — 패스트 모드 토글과 채팅 히스토리 위/아래키
`/fast`(슬래시 패스트)를 누르면 패스트가 켜졌다 꺼진다(한 번 더 누르면 다시 켜짐). 이어서 입력 꿀팁. 명령을 빨리 입력하는 방법은 매크로가 아니라, Codex CLI에서 입력했던 채팅 내용을 **키보드 위/아래 키로 히스토리 탐색**할 수 있다는 것. 방금 보낸 내용을 다시 보내려면 위 키를 눌러 이전 입력을 불러오면 된다("깨알 꿀팁").

### [12:12–12:48] 세팅 3 — 줄바꿈·줄 삭제·전체 삭제 단축키
줄바꿈은 **Shift+Enter**로 간단히 할 수 있다. 한 줄을 다 지우려면 **Command+Delete(백스페이스)**로 한 줄씩 지운다. 한꺼번에 다 지우려면 **ESC**, 또는 **Command+C / Control+C**를 누르면 된다. 단, 이 단축키들은 업데이트나 실행 환경마다 조금씩 바뀌는 것 같아 ESC / Control+C / Command+C를 한 번씩 눌러보면 전체 입력이 잘 지워지는 걸 볼 수 있다고 안내.

### [12:48–13:00] 나머지 슬래시 메뉴는 영어 설명으로 감 잡기
기본적으로 써야 할 것은 다 설명했고, 나머지는 슬래시 메뉴의 영어 설명을 읽어보면 대략 감이 온다고 함. 다만 거의 쓸 일은 없을 거라고 덧붙임.

### [13:00–14:07] 세팅 4 — `/clear`의 의미(컨텍스트 윈도우 비우기)
`/clear`(슬래시 클리어)는 지금까지의 채팅 내용, 즉 컨텍스트 윈도우에 쌓인 맥락을 깨끗이 지우는 것. "지우면 안 좋은 거 아닌가?" → 안 좋은 게 아니라고 강조(에이전트 강의에서 여러 번 말한 내용). 에이전트는 컨텍스트 윈도우 안의 내용을 바탕으로 동작하는데, 예를 들어 "강아지가 초콜릿 먹으면 죽는 이유"를 얘기하다가 갑자기 "화로회 맛집"을 물으면서 컨텍스트를 안 비우면, 에이전트가 "강아지한테 화로회를 먹이려나?" 식으로 이상하게 동작할 수 있다는 것. 그래서 **한 단위의 작업이 끝나면 `/clear`로 맥락을 비우고** 새 주제를 물어보라고 추천.

### [14:07–14:54] 대화 보관 욕구 → 멀티세션(탭 분리)로 해결
[의견] 어떤 사람은 에이전트와 나눈 대화가 소중해 보관하고 싶어 하는데, 바로 그럴 때 멀티세션 유지 방식이 유리하다고 함. 같은 "테스트 오르카" 폴더에서도 여러 대화를 동시에 하고 싶을 수 있는데, 그럴 땐 탭을 분리하면 된다. "+"를 눌러 새 터미널을 열면 같은 테스트 오르카 폴더에서 다시 터미널을 실행할 수 있고, 거기서 `codex`를 누르면 같은 폴더를 공유하되 이전 대화는 그대로 남아 있고 새 탭은 새 채팅창이 된다.

### [14:54–15:31] 같은 폴더에서 코덱스 + 클로드 코드 동시 운용
탭을 좌우/상하로 분할(오른쪽 분할·아래로 분할)하면 같은 폴더에서 두 개의 에이전트를 운용할 수 있다. Codex CLI를 끄는 법은 `/exit`(슬래시 엑시트) 또는 Control+C 두 번 연타. 슬래시 exit로 끈 뒤 그 자리에서 "claude --dangerously-skip-permissions"(클로드 데인저러스 퍼미션스)를 실행하면, **왼쪽에서는 Codex CLI, 오른쪽에서는 Claude Code**를 쓸 수 있다.

### [15:31–16:04] 터미널 기반 도구의 우위 — 앱 코덱스로는 불가능
[의견] 한 폴더를 두고 여러 에이전트를 운용하고 싶을 때 터미널 기반 도구가 굉장히 유용하다는 것을 강조. ChatGPT 앱의 코덱스 같은 GUI 도구로는 이런 작업을 할 수 없다 — 앱은 무조건 코덱스만 써야 하고 거기서 Claude Code를 실행할 방법이 없기 때문. 다중 작업을 하고 싶을 때 터미널 환경이 유리하다는 결론.

### [16:04–16:47] 오르카의 폴더별 라인업 장점
오르카는 왼쪽에 폴더별 라인업을 볼 수 있어, 다른 폴더를 추가해 작업을 계속 이어갈 수 있다. 폴더 찾아보기로 즐겨 쓰는 환경 세팅된 폴더를 열면, 위에서는 "테스트 오르카", 여기서는 "워크 GWS"(작업 폴더) 식으로 왔다 갔다 하며 작업할 수 있다. 지금 화면에는 탭 두 개가 열려 있고 한쪽은 코덱스, 한쪽은 클로드 코드이며, 터미널만 열린 탭은 에이전트 세팅이 보이지 않는다고 시연. 이런 식으로 왔다 갔다 작업할 수 있어 오르카도 좋은 도구라고 평가.

### [16:47–17:04] 오르카를 쓰기로 한 이유는 "고급 영상"에서
[의견] 화자가 오르카를 써야겠다고 마음먹은 이유가 딱 하나 있는데, 그건 고급 영상에서 다루겠다며 미룬다. 오늘은 Codex CLI 설치와 함께 오르카를 같이 살펴봤고, 오르카도 결국 터미널 도구가 있으니 "나는 오르카 안 쓸 거다" 하는 사람은 오르카 부분을 완전히 제외하고 터미널에서 그냥 진행해도 무방하다고 안내.

### [17:04–17:15] VS Code에서도 동일 — 결국 같은 코덱스
비주얼 스튜디오 코드(VS Code)에서 하고 싶은 사람은 VS Code를 실행하고 오르카 관련 내용은 다 빼면 된다. 위쪽 텀(터미널) 모양 아이콘을 눌렀다 떼면 아래쪽에 터미널이 나타나는데, 그 상태에서 `codex`를 실행하면 Codex CLI가 똑같이 실행된다. [의견] "완전히 동일한 거예요" — VS Code 터미널로 열어 실행하든 오르카로 열든 터미널로 열든, 보여지는 창(오르카냐 VS Code냐 그냥 터미널이냐)만 다를 뿐 같은 녀석이니 오해 말라고 재차 강조. 설명해도 오해할 거라 알지만 굳이 한 번 더 짚었다고 함. 오늘 설치 가이드는 이 정도면 충분하다며, 에이전트가 혼란스러운 요즘 더 유용한 영상을 준비해 다음에 찾아뵙겠다고 마무리.
