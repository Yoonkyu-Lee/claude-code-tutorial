# [시즌 3] 크아 섭종 기념, 픽셀 아트로 크레이지아케이드 게임을 만들고 싶다면?

> editorp89 · 2026-06-15 · [영상 링크](https://youtu.be/_Ly6NuLcLBU)
> 정리 방식: 시간순 심층 워크스루 (paraphrase) + 주제별 상세 요약
> 검증 배너: 등장하는 API 비용 수치는 화자 본인 사용 경험에 근거한 주장(미검증)이며, 사용량·모델·환율에 따라 달라질 수 있음.

## 요약 (주제별)

### 무엇을 만드는가 / 영상의 핵심
- 크레이지 아케이드(크아) 서비스 종료를 기념해 유사 게임을 만드는 과정을 보여주지만, **핵심은 게임 자체가 아니라 픽셀 아트 스프라이트를 만드는 방법**이다.
- 도구는 `Perfect Pixel`이라는 오픈소스. 이미지를 넣으면 게임에 필요한 스프라이트(방향별·동작별 애니메이션 프레임)를 만들어 준다.
- 과거에는 못생긴 도형 위주로만 게임을 만들어야 했는데, 이 방식으로 자연스러운 애니메이션 캐릭터를 쓸 수 있게 된다는 것이 요지.

### Perfect Pixel 도구의 정체와 역할
- Perfect Pixel은 "픽셀 아트를 잘 그리는" 도구가 **아니다**. 이미지를 그리는 것은 `GPT Image 2`가 하고, Perfect Pixel의 역할은 그 이미지를 기반으로 **게임용 스프라이트를 만드는 것**이다.
- 스프라이트 = 여러 프레임(예: idle 상태 6프레임)을 애니메이션화한 것. 게임 제작을 스프라이트 단위로 처리해 주기 때문에 매끄럽게 연결해 작업할 수 있다.
- idle / walk / run / jump / firecast 등 동작(front·back 등 방향 포함)을 정의하면 도구가 각 동작 스프라이트를 만들어 준다. 애니메이션은 6프레임이 기본.
- 도구가 대신 처리해 주는 것: 배경 투명화, 동작 애니메이션의 위치 조정, 컷 자르기, 몇 장으로 구성할지 등. [의견] 이런 세부를 대신 다 해 주는 점이 매우 편하다.
- 8방향(8-way) 옵션: 한 방향만이 아니라 여덟 방향 이미지를 한꺼번에 생성. 좌우만/상하좌우/8방향 등 게임 종류에 맞춰 쓸 수 있다.

### GPT Image 2 API와 비용 (주장/미검증)
- 이미지 생성에 관여하는 것은 `GPT Image 2`(GPT 이미지 2)이며 `OpenAI API`로 호출된다. 비용을 충전해 쓰는 구조.
- 원래 이 도구는 GPT Image 2를 지원하지 않았으나, 화자가 **코드를 직접 수정해** OpenAI API로 GPT Image 2를 쓸 수 있게 만들어 놨다. 공식 지원은 Gemini API / OpenRouter / Fal / Fal+ 계열이라고 언급.
- 비용(주장/미검증): "연두의 모험"이라는 게임 하나의 이미지를 전부 만드는 데 총 1달러가 채 안 들었다(인풋+아웃풋 합산). walk 8방향 = 6×8 = 48장 생성 시 스펜드가 1달러를 넘겼고, 앞서 불 쏘는 것(약 0.5달러)까지 합쳐 총합이 2달러 정도까지 갔다고 함.
- [의견] 비용이 좀 센 편이지만, 게임 하나 이미지에 1달러 정도라면 들일 만하다.

### 실전 절차 — 베이스 이미지부터 스프라이트까지
1. **베이스 이미지 준비**: 실제 아기 사진이나 본인 사진을 올리고, "메이플스토리 스타일" 프롬프트로 캐릭터 이미지 한 장을 생성(요즘 유행하는 메이플 프롬프트). 프롬프트는 유튜브 더보기(설명란)에 공유.
2. 생성된 이미지를 다운로드해 Perfect Pixel에 업로드.
3. New Project → Clear & Start → Upload Image. 이름(예: `editor P`)은 아웃풋 파일 이름용이라 형식상 지정. 옵션 칸은 안 써도 됨.
4. 아트 스타일은 `chibi`(취비)로 지정하니 제일 잘 나왔다고 함(다른 스타일도 테스트 권장).
5. **이미지 업로드 후 동작만 바꾸는 방식 추천**(처음부터 도구에 그리게 하는 것보다). Add Animation으로 원하는 동작(walk 등)을 하나씩 추가.
6. 프레임 수(기본 6, 뚝뚝 끊기면 올림 — 6이어도 충분), Repeat/Loop(걷기처럼 반복 확인용), FPS(더 부드럽게 원하면 올림) 설정.
7. `8-way`(에웨이) 눌러 8방향 × 6프레임 = 48장 생성. Spend 카테고리에서 GPT Image 2 인풋/아웃풋/비용을 확인 가능(cached input은 없음).
8. **Export** → 게임용 새 폴더 생성(예: `editor P` 폴더)로 스프라이트 내보내기. 이때 이미지뿐 아니라 이미지에 대한 메타데이터까지 함께 나와서, 에이전트가 폴더 구조를 읽고 어떤 이미지들인지 알아서 이해한다.

### 설치 방법
- Perfect Pixel은 화자의 GitHub 레포에 올려 둠(원 제작자 레포를 포크해 자신의 브랜치를 둠). 링크는 설명란 제공.
- 빈 폴더(예: `퍼펙트 픽셀 2`)를 만들어 열고, `Codex`를 켠다(Claude Code를 써도 됨). GitHub 주소를 주며 "이것을 사용할 수 있도록 프로젝트 폴더 세팅을 해라"라고 지시하면, 레포의 사용법 문서를 보고 에이전트가 프로젝트 세팅을 해 준다.
- **API 키 설정**: 도구를 열면 Environment 항목에 이미지 생성용 API 키 칸이 있음. "키는 내가 직접 입력할 테니 `.env` 파일만 만들어 줘"라고 하면 `.env`에 OpenAI API 키 입력 줄이 생성됨. OpenAI 대시보드에서 Create new secret key로 키를 발급(이름 예: `픽셀 퍼펙트 2`)해 붙여넣고 저장하면 GPT Image 2를 쓸 수 있는 상태가 됨.

### 게임 제작 — 스프라이트로 실제 게임 만들기
- Export된 폴더를 게임 제작 에이전트에게 주고 만들게 한다. 화자는 `Cursor`의 `Composer 2.5`를 사용(Codex 등 무엇이든 가능). [의견] Composer 2.5는 매우 값싼 모델인데 이런 픽셀 아트류 게임은 이 정도로도 충분히 만든다.
- **뱀파이어 서바이버류 게임**: "이미지 에셋들을 모두 파악해서 8방향 이동을 테스트할 수 있게, 적은 단순 도형으로, 바닥이 움직이는 뱀서류 게임을 웹 기술로만 만들어 줘"라고 지시. 걷기 애니메이션은 매우 잘 표현됐고 총알 표현·통화(경험치?) 표현은 다소 아쉬웠다.
- **크레이지 아케이드류(플랫포머/봄버맨) 게임**: 스프라이트로 상하좌우만 고려해 크아 유사 게임 제작. 봄버맨이 유명해 아이템까지 만들어 줌. 폭탄에 맞으면 죽는 룰, 폭탄 연쇄 폭발, 불 길어지는 아이템·폭탄 2개 아이템·스피드 아이템, 적을 다 없애면 출구가 열리는 로직까지 구현.
- [의견] 이런 프로토타입은 금방 만든다. 다만 상용화 게임 수준이라고는 생각하지 않는다 — "이 정도면 굉장히 좋다" 수준.

### 리스크 · 알려진 버그
- 프로그램 실행 시 **빈 화면**이 뜨는 문제 발생("resource loading" 관련 메시지, 크래시는 아님). 화자도 원인을 확실히 모르며, 자기 쪽에서 수정해 GitHub 레포를 다시 올릴 예정이라고 함.
- 게임 실행 시 **에셋 로딩 실패** → 에이전트에게 "로컬 서버로 실행하라"는 안내가 나오면 그대로 수정 지시.
- 크아 게임에서 **상하좌우 이동이 안 되는** 이동 로직 버그 → 에이전트에게 수정 지시로 해결.

### 세일즈 / 유입 구조
- 유료 자료·단톡방 유입 장치는 없음. 프롬프트와 GitHub 링크를 **유튜브 설명란(더보기)에 무료 공유**하는 구조.

### 활용 아이디어 · 마무리
- [의견] 모두가 게임을 만드는 것은 아니라 모두에게 유용하진 않겠지만, 게임 외에도 다양하게 쓸 수 있다.
- 예: 아기들과 할 학습 게임을 만들고 싶은데 에셋이 못생겨 고민이던 사람은, 이 방식으로 예쁜 학습 게임을 만들어 학습을 유도하는 데 쓰기 좋다.

## 3줄 오리엔테이션
- 이 영상은 "게임 만들기"가 아니라 **AI로 게임용 픽셀 아트 스프라이트를 만드는 법**이 핵심이다.
- 흐름: GPT Image 2로 캐릭터 이미지 생성 → `Perfect Pixel`(오픈소스, 화자가 OpenAI API 지원하도록 포크·수정)로 방향·동작별 스프라이트 생성·Export → `Cursor Composer 2.5`(또는 Codex/Claude Code)로 실제 게임 제작.
- 실습으로 뱀파이어 서바이버류와 크레이지 아케이드(봄버맨)류 두 게임을 만들며, 중간중간 빈 화면·에셋 로딩·이동 로직 버그를 에이전트로 고쳐 나간다.

## 전문 — 시간순 워크스루

### [00:00–00:57] 오프닝 — 크아 섭종 기념 게임, 진짜 주제는 스프라이트
흥미로운 오픈소스를 소개하려고 영상을 올렸다며 시작한다. 크레이지 아케이드가 서비스 종료를 하기에 유사한 게임을 간단히 만들어 봤다고 화면을 보여주지만, **핵심은 게임 제작이 아니라 캐릭터 스프라이트 이미지를 만드는 것**이라고 못 박는다. 과거에는 못생긴 도형 위주로만 게임을 만들어야 해서 불만족스러웠을 것이라며, 이 예쁜 이미지를 어떻게 만들었는지가 요점이라고 한다. 돈 주고 산 게 아니라 이런 걸 만들 수 있는 오픈소스를 찾았기 때문에 공개하는 것이며, 자신이 불편했던 부분을 수정했다고 밝힌다.
> "제가여신냥 이제 기터에다 올린 건 아니고 이제 이 원 제작자의 레퍼를 제가 포크를 떠 가지고 저만의 브랜치를 일단은 해 둔 게 있는데 이분한테 제가 제안을 하려고 하고는 있습니다." [00:48]

### [00:57–01:49] 오늘의 주제 정리 — Perfect Pixel 오픈소스
발표 자료 두세 장으로 간단히 설명하겠다고 한다. 오늘 주제는 "픽셀 아트 애니메이션을 만들어 게임까지 만들고 싶다면"이며, 그럴 때 `Perfect Pixel`(퍼펙트 픽셀)이라는 오픈소스를 써 보길 권한다. 게임을 만들 때 가장 큰 고민은 앞서 보여준 대로 스프라이트(SS)를 만드는 것인데, 벽 같은 정적 요소는 요즘 금방 만들지만 자연스러운 캐릭터 애니메이션은 다르다고 한다. 좌우 이동 시 방향에 맞게 움직이는 것이 매우 자연스럽고, AI 티가 살짝 나긴 하지만 과거에 억지로 AI로 만들던 스프라이트와 비교하면 훨씬 잘 만들어졌다고 평가한다.

### [01:49–02:38] Perfect Pixel의 정체 — 그리는 건 GPT Image 2, 스프라이트화가 역할
이 도구의 특징을 먼저 이해하고 실습을 시작하라고 한다. Perfect Pixel은 "픽셀 아트를 잘 그리는 데" 초점이 있는 게 아니다. 이미지를 실제로 그린 것은 `GPT 이미지 2`이고, Perfect Pixel의 역할은 그 이미지를 기반으로 **게임에 필요한 스프라이트를 만드는 것**이다. 스프라이트란 idle 상태의 여섯 장 프레임처럼 여러 이미지를 애니메이션화한 것을 말한다.
> "이 퍼펙트 픽셀의 역할은 무엇이냐면은 이 이미지를 기반으로 해서 게임에 필요한 스프라이트, 이걸 스프라이트라고 하거든요 (…) 여섯 개의 프레임이거든요." [02:07]

게임 제작을 스프라이트 단위로 처리해 주기 때문에 매끄럽게 연결해 작업할 수 있는 장점이 있고, 애니메이션은 6프레임이 기본이라고 한다.

### [02:38–03:34] 동작(action) 생성 — idle/walk/run/jump front·back
도구 화면을 보여준다. 이미지 하나를 주고 그에 대해 설명한 뒤 스타일을 chibi로 지정, 프레임 셀 사이즈 등을 정한다(보여지는 것에 대한 설정). 그다음 "아이들 프론트를 만들어라"처럼 정면을 보며 눈을 깜빡이는 상태를 만들라고 지시하면, 도구가 idle front / walk front / run front / jump front 등을 만들어 준다. GPT 이미지 2를 썼기 때문에 이미지 재현이 매우 좋고, 도구는 자연스러운 움직임을 위해 배경 투명화, 동작 애니메이션의 위치 조정, 컷 자르기, 몇 장으로 구성할지 등을 대신 다 처리해 준다.

### [03:34–04:04] 이미지 업로드 vs 처음부터 생성 — 업로드 방식 추천
이미지를 업로드해 그 이미지를 행동으로 변형시키는 방법과, 처음부터 도구에 이미지를 만들게 하는 방법 두 가지가 있다. 화자는 **이미지를 업로드해 행동만 바꾸게 하는 방식을 추천**한다. walk도 run도 다 잘해 준다고 강조한다. 요즘 유행하는 "메이플스토리 프롬프트"를 언급하며, 아기 사진이나 본인 사진을 올리고 메이플스토리 스타일을 요청하면 그렇게 만들어 준다고 한다.

### [04:04–05:02] 실습 1단계 — 베이스 이미지 만들고 업로드
먼저 베이스 이미지가 필요하다. 실제 아기 사진이나 본인 사진을 올리고 프롬프트(유튜브 더보기에 공유 예정)를 밀어 넣으면 메이플스토리 스타일 이미지 한 장이 나온다. 화자의 캐릭터는 복장을 참고해 생성됐다. 그 이미지를 다운로드해 Perfect Pixel에 업로드한다. New Project → 저장/내보내기 → New → Clear & Start → Upload Image 순으로 진행. 업로드 후 이름 칸에 `editor P`를 넣는데 이는 아웃풋 파일 이름용이고, 다음 옵션 칸은 써도 안 써도 된다. 아트 스타일은 chibi(취비)로 하니 제일 잘 나왔다고 한다(차이는 잘 모르겠다고 함).

### [05:02–05:56] Add Animation과 8-way, 그리고 GPT Image 2 API
Add Animation을 누르면 게임에서 쓰이는 여러 동작을 정의한다. idle은 가만히 서 있는 모습, walk는 걷는 모습. 원하는 동작을 하나씩 추가하는데, 예로 firecast(불 쏘기)를 선택해 본다. Generate를 하면 한 방향만이 아니라 `8-way`(에웨이), 즉 여덟 방향으로 이미지를 다 만들어 준다. 좌우만/상하좌우/8방향 등 게임 종류에 맞춰 한꺼번에 8방향을 생성하는 것. firecast front/back 등으로 다 만들어 준다. 이 이미지 생성에 관여하는 것은 `GPT API`이므로, API를 쓴다는 것(비용 충전)을 이해해야 한다고 짚는다.

### [05:56–06:26] 비용 언급 (주장/미검증)
> "연두의 모험이라는 게임을 만들기 위해 이미지를 다 만들었던 비용이 (…) 인풋 아웃풋 해 가지고 1달러가 좀 안 됐습니다." [05:59]

비용이 좀 센 편이라면서도, 게임 하나를 완전히 만드는 이미지에 1달러 정도라면 들일 만하다는 의견을 낸다. [의견] 이런 식으로 GPT 이미지 2를 꽂아 자기만의 게임 스프라이트를 만들 수 있다는 것이 요지.

### [06:26–07:13] 코드 수정으로 GPT Image 2 지원, firecast 결과 확인
이 도구는 원래 GPT 이미지 2를 지원하지 않지만, 화자가 코드를 살짝 수정해 쓸 수 있게 해 놨다고 다시 밝힌다. firecast 하나만 줬는데도 도구에 내장된 특정 프롬프트·이미지 자르는 규칙이 적용돼 자연스러운 애니메이션을 만들어 준다. 불덩이가 뒤에 살짝 남는 점은 아쉽지만 매우 자연스럽게 수정해 준다고 평한다. 이미지를 만든 뒤 Export를 간단히 하면 된다.

### [07:13–08:03] Cursor에서 게임 지시, 플랫포머로 방향 전환
Export한 스프라이트를 그대로 써서 화자는 `Cursor`에서 작업했다고 한다. "상하좌우만 고려해서 크레이지 아케이드 유사 게임을 만들어 봐"라고 지시했더니 나온 게임을 보여준다. 폭탄에 맞으면 죽는 룰로 진행된다. 크아는 서비스 종료(받기?) 중이니 이번엔 대신 **플랫포머 게임**을 만들겠다며, 처음부터 다시 시작해 과정을 보여주겠다고 한다.

### [08:03–08:59] Perfect Pixel 설치 — GitHub 레포 + Codex/Claude Code
Perfect Pixel은 화자의 GitHub에 올려 뒀고 링크를 줄 예정이다. 레포에서 Code를 눌러 주소를 복사한다. 빈 폴더를 하나 만들어(예: 바탕화면에 `퍼펙트 픽셀 2`) 열고, 아무것도 없는 상태에서 `Codex`를 켠다(Claude Code를 써도 된다). 복사한 주소를 넣으며 "이것을 사용할 수 있도록 프로젝트 폴더 세팅을 해라"라고 지시하면, 레포의 사용법 문서를 보고 에이전트가 프로젝트 세팅을 해 준다.

### [08:59–10:08] API 키 설정 — .env에 OpenAI 키
세팅 후 사용자가 할 일은 프로그램을 켜는 것뿐인데, 중요한 문제가 하나 있다. Perfect Pixel을 열면 Environment 항목에 이미지 생성용 API 키 칸이 비어 있다. 공식적으로는 Gemini API 키 / OpenRouter / Fal / Fal+만 지원하지만, 화자는 OpenAI API를 쓸 수 있게 세팅해 놨다. 에이전트에게 "키는 내가 알아서 입력할 테니 `.env`(인바이먼트) 파일만 만들어 줘"라고 하면, `.env`가 만들어지고 OpenAI API 키를 입력하는 빈 줄이 생성된다.
> "내가 키는 알아서 입력할 테니까 인바먼트 파일만 만들어 줘라고 요렇게 얘기를 하시면은 (…) 점 인바먼트가 만들어지고 오픈 AI의 API 키를 입력하는 코드 빈 줄 요렇게 해 가지고 만들어 줄 겁니다." [09:44]

### [10:08–10:52] OpenAI 키 발급 후 붙여넣기, 프로그램 실행
OpenAI 대시보드에서 API keys → Create new secret key로 키를 새로 만들고(이름 예: `픽셀 퍼펙트 2`), 노출한 키는 나중에 지우겠다고 하며 복사해 `.env`에 붙여넣고 저장한다. 이제 이 프로젝트 폴더는 자신이 발급한 OpenAI API 키로 GPT 이미지 2를 쓸 수 있는 상태가 됐다. 그다음 에이전트에게 "프로그램을 실행해 봐라"라고 하면 앞서 본 Perfect Pixel 프로그램이 열린다(빌드된 것을 열어도 됨).

### [10:52–11:43] 빈 화면 버그 — 수정 후 재배포 예정
프로그램이 열렸는데 화면에 아무것도 안 보이는 문제가 발생한다. 약간 수정을 해 줘야 한다며, "resource loading"(웨일즈 리소스 로딩) 관련 메시지가 뜨지만 여전히 빈 화면이고 크래시는 아니라고 한다. 원인을 확실히 모르겠고 "안 될 리가 없는데 이상하다"면서, 자기 쪽에서 한 번에 열리게 수정해 GitHub 레포를 다시 올리겠다고 한다.
> "제가 요거는 한 번에 열릴 수 있게 제 거에서 좀 수정을 해 가지고 기터블 레폰은 다시 이제 보내야겠네요." [11:23]

시청자에게는 "이 프로그램 실행해 봐라"라고 에이전트에 지시하면 열린다고 안내한다.

### [11:43–12:34] 실습 재개 — 이미지 업로드와 프롬프트 수정 위치
빈 화면 상태에서 Upload Image에 앞서 만든 메이플스토리 이미지를 넣는다. 더보기의 프롬프트를 입력해 원하는 사진을 넣은 뒤, 프롬프트에서 중간과 마지막의 특정 부분만 수정하면 된다고 안내한다(프롬프트가 너무 길어 전부는 못 보여줌). 그 부분만 고치면 알아서 이미지를 이용해 프롬프트가 입력되고 결과물이 나온다. 결과 이미지를 다운로드해 Perfect Pixel에 업로드한다.

### [12:34–13:14] 이름·스타일·walk 애니메이션 설정
업로드 후 이름은 형식상 `editor P`로 짓고, 다음 칸은 옵셔널이라 안 해도 된다. 아트 스타일은 하나씩 다 테스트해 보라면서 자신은 chibi만 해 봤다고 한다. Add Animation을 눌러, 크레이지 아케이드를 만들 것이므로 `walk`(걷기)만 만든다. 프레임 수를 늘리고 싶으면 올리면 되지만 6프레임이어도 충분하다고 한다. Repeat/Loop는 걷기 애니메이션이 계속 반복되는지 확인하기 위한 것이고, FPS는 더 부드러운 결과를 원하면 올리는 값인데 화자는 굳이 원하지 않는다(6장이면 충분).

### [13:14–14:24] 8-way 생성과 비용 확인, Export 준비
`8-way`(에웨이)를 눌러 8방향 걷기 이미지를 총 **6×8 = 48장** 만든다. 48개를 만드는 만큼 사용량(달러)이 표시되며, Spend 카테고리를 보면 벌써 비용이 1달러를 넘어갔다. 앞서 불 쏘는 것에 약 0.5달러를 더 쓴 셈이고, 지금 것까지 하면 총 스펜드가 2달러 정도까지 갈 것이라고 한다. GPT 이미지 2 부분의 인풋/아웃풋/비용 세 부분을 확인하면 비용이 잘 들어가는지 체크할 수 있다(cached input은 없음 — 매번 다른 이미지라 다시 생성). 걷는 애니메이션이 지금은 어색해 보여도 실제 게임에서 조작해 보면 매우 잘 된다고 한다. 8방향이 다 그려질 때까지 기다렸다가 Export를 누른다.

### [14:24–15:16] Export와 메타데이터, 게임용 새 폴더
게임을 새로 만들어야 하므로 프로젝트 폴더를 새로 만든다. `퍼펙트 픽셀` 폴더는 이 툴을 쓰기 위한 폴더이고, 게임을 만들려면 게임용 폴더를 따로 만들어야 한다. `editor P` 폴더(빈 폴더)를 만들어 열고 스프라이트를 Export한다. 그러면 이미지뿐 아니라 이미지에 대한 메타데이터까지 함께 들어온다. 시청자가 이 데이터를 이해할 필요는 없고, 에이전트가 폴더 구조와 이미지를 읽고 "이런 형태의 이미지들이 있구나" 알아서 이해하도록 도구가 세팅해 준 것이다.

### [15:16–16:35] 뱀파이어 서바이버류 게임 만들기
이제 툴로 자유롭게 게임을 만들면 된다. 화자는 `Cursor`의 `Composer 2.5`를 활용하지만 Codex 등 무엇이든 된다고 한다. Export된 폴더에 대고 지시한다.
> "여기 있는 이미지 에세트들을 모두 파악해서 (…) 여덟 개의 방향 이동하는 걸 또 테스트를 해 볼 거기 때문에 뱀파이어 서바이버류 게임을 웹 기술로만 만들어 봐. 적은 그냥 단순하게 도형으로 처리해 주고 바닥이 움직임." [15:19]

에이전트가 폴더 안 구조와 메타데이터를 읽고 어떤 이미지들인지 이해한 뒤 바로 게임 제작에 들어가므로 별다른 설명이 필요 없어 편하다고 한다. [의견] 픽셀 아트류 게임은 이렇게만 해도 값싼 모델 Composer 2.5로 충분히 만들 수 있다.

### [16:35–17:13] 뱀서류 게임 실행 확인
서버가 뜨고(200 응답) 들어가 확인한다. 적은 원·사각형·삼각형 도형으로 처리됐다. 캐릭터가 움직이고 총알이 나오는데 총알 표현은 잘 안 됐고, 통화(경험치?)가 없어지지 않는 점이 아쉽다. 다만 **걸어가는 애니메이션은 매우 잘 표현됐다**는 게 핵심이라고 강조한다. 완전하진 않지만 스프라이트 표현에만 집중해서 보라고 한다.

### [17:13–18:11] 크레이지 아케이드(봄버맨)류 게임 만들기
다음으로 크레이지 아케이드 게임을 만든다. "크레이지 아케이드 게임을 만들어 줘"라고 하면 스프라이트를 이용해 상하좌우만 고려한 크아 유사 게임을 제작한다. 봄버맨이 워낙 유명해 아이템도 만들어 주는 것 같고, 작업은 1~2분이면 끝난다. [의견] 옛날엔 스프라이트가 못생겨 게임을 만들어도 밤티(허접한) 느낌이 심했는데 이제는 이런 게임을 금방 만든다. 다만 상용화 게임 수준이라고는 생각하지 않지만 이 정도면 매우 좋다. (앞서 뱀서 게임에는 디버그 빔/총알, 경험치 먹는 범위 확대 등도 추가했다.)

### [18:11–19:07] 에셋 로딩 실패 버그 → 로컬 서버, 활용 아이디어
크아 게임 실행 시 "asset 로딩 실패"가 뜨고 "로컬 서버로 실행하라"는 안내가 나온다. 그대로 수정을 지시한다. [의견] 매우 쉽다며, 오늘 정보가 게임 만들고 싶던 분들에게 유용하길 바란다고 한다. 모두가 게임을 만드는 건 아니라 모두에게 유용하진 않겠지만, 게임 외에도 다양하게 쓸 수 있다고 한다. 예로 아기들과 할 학습 게임을 만들고 싶은데 에셋이 못생겨 고민이던 사람은 이 방식으로 예쁜 학습 게임을 만들어 학습을 유도하는 데 쓰면 좋다고 제안한다.

### [19:07–20:07] 이동 로직 버그 수정 → 게임 완성
여전히 에셋 로딩 실패가 뜨는데 "여기서부터는 아시죠? 그냥 이렇게 고치시면 됩니다"라며 넘어간다. 게임이 만들어졌으나 캐릭터가 잘 안 움직이고 시작을 안 하는 듯하다. "폭탄은 놓는데 상하좌우 이동이 안 된다"고 지시하니 이동 로직을 수정한다. 깔짝 움직였다 멈추는 로직 오류를 에이전트가 코드 수정으로 고치자 게임이 잘 완성된다. 아이템(불 길어지는 것, 폭탄 2개)도 구현됐다.

### [20:07–20:28] 봄버맨 로직 확인과 마무리
폭탄이 연쇄로 터지는 것, 스피드 아이템 등 웬만한 로직이 다 구현됐다. 적을 다 없애야 문(출구)이 열리는, 옛날 봄버맨을 그대로 구현한 듯한 모습이다. 클리어하면 출구가 열리고 맵이 더 넓어지며 게임이 끝난다. 게임 만들 때 이런 고민을 했던 사람은 오늘 영상을 참고하면 큰 도움이 될 것이라며 영상을 마친다.
