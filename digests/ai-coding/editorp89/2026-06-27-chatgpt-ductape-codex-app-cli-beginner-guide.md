## 요약 (주제별)

### 1. 세 도구의 포지셔닝 (ChatGPT / Codex 앱 / Codex CLI)
- **ChatGPT**: 이미지 생성(편의 좋음)과 심층 리서치 위주. 엑셀·구글 스프레드시트 플러그인 지원.
- **Codex 앱**: "클로드 데스크톱의 포지션". 내 PC의 특정 폴더 파일을 직접 보고 작업하는 인공지능. 파일 업로드 없이 폴더를 지정하면 그 안을 스스로 탐색·작업. 판단 기준 = "인공지능이 내 PC를 볼 수 있어야 하나, 없어도 되나".
- **Codex CLI**: 터미널(텍스트) 기반. 결국 코딩 작업을 하려는 사람이 넘어오는 단계. 실행한 폴더 중심으로 동작, 자율성이 가장 높음.

### 2. 요금제와 토큰 분리 (가장 강조한 핵심)
- ChatGPT와 Codex는 **토큰을 공유하지 않는다** (같은 계정이어도 별도 사용량). 이게 강의 최대 포인트.
- ChatGPT는 토큰 사용량을 표시하지 않아서, 갑자기 "다 썼습니다"가 뜨면 소진된 것. Codex 앱/CLI는 "남은 사용량"을 표시해 준다.
- [의견] 결론: 똑같은 돈을 내는 만큼, ChatGPT에서 할 수 있는 작업을 굳이 Codex에서 하지 말 것. 가벼운·겹치는 작업은 ChatGPT에서 먼저.
- 요금제: 20 / 100 / 200달러 등 (본문에 "8달러"도 언급되나 [불명확]). 20달러는 초기화 쿠폰(사용량 리셋)을 안 주는 것 같다고 하나 화자도 "정확하진 않다"고 함 (주장/미검증).

### 3. 토큰 타이머 전략 (화자 꿀팁)
- 남은 사용량은 **5시간 한도**와 **일주일 한도**가 공존.
- 5시간 타이머는 "채팅을 처음 넘기는 순간"부터 돈다. 하루 종일 채팅을 안 하면 타이머가 시작조차 안 됨.
- [의견] 매일 아침(예: 9시)에 예약됨으로 작업을 걸어 타이머를 일찍 시작시키면, 오후 2시에 리셋돼 2시부터 다시 넉넉히 쓸 수 있다 → 토큰을 전략적으로 더 효율적으로 사용.

### 4. ChatGPT — 이미지 생성 & 엑셀/스프레드시트
- 이미지 생성 모델은 **덕테이프**(화자가 부르는 명칭). Codex 앱에서는 이미지 생성 스킬(이미지 제너레이션 스킬)을 써야 하고, ChatGPT는 "이미지 만들기" 버튼 + 수정 UI/UX가 더 편하다.
- [의견] 화자는 유튜브 썸네일 에셋 등 이미지 작업을 주로 ChatGPT에서 함. 최근 이미지 수정 기능이 사라진 것 같다는 제보를 받았고 실제로 없어져 있었다 ([불명확]: 변경 여부 화자도 추정).
- 이미지 대량 생성은 Codex 앱이 유리(한 번에 50개까지 성공), ChatGPT는 한 번에 최대 10개 정도(주장/미검증) — 집중·소량 작업에 적합.
- 엑셀/구글 시트: 인터넷에 "엑셀"·"스프레드시트"로 검색 → "엑셀 및 구글 시트용 ChatGPT 베타" → 플러그인 설치. 엑셀은 Office 365 구독 시 완전 동작(주장). 클로드도 엑셀은 지원하나 **스프레드시트(구글 시트)는 접근 불가** → 구글 웍스 사용자는 ChatGPT가 유리.
- [의견] 셀 선택까지 실시간 연동되고, 리서치→시트 값 입력을 한 번에 처리. 특히 **수식·함수 걸기**를 매우 잘하고 사람이 놓칠 데이터 누락도 잘 찾아줌. 추론 강도는 패스트/스탠더드/헤비 선택 가능(화자는 스탠더드 사용).

### 5. Codex 앱 — 한글(HWPX) 스킬 자동화
- 화자가 배포한 **HWPX 스킬**(K9/골든래빗 배포처, 또는 해외 skills.sh 계열 사이트에서 "HWPX" 검색). 설치 명령어를 복사 → Codex 앱 새 채팅에 "이 명령어를 이용해서 한글 스킬을 설치해 줘" → 에이전트가 알아서 설치.
- 스킬 = 인공지능에게 주는 "디테일한 상세 매뉴얼". 한글은 국내 전용이라 AI가 잘 못 다루므로 이 스킬로 보완. 이미 많이 퍼져 사용자가 많다고 함(주장).
- 설치 후 **반드시 재시작**(Windows: 우측 하단 트레이 아이콘 우클릭 종료 / macOS: Cmd+Q). `$`(달러) 입력 시 스킬 목록이 뜨고 "HWPX"가 검색되면 설치 성공. 안 되면 "설치했다더니 다시 켰는데 HWPX 스킬이 안 보여"라고 에이전트에 다시 시켜 마무리(라이브에서 실제로 몇 차례 재시도함).
- [의견] 이 스킬은 **기존 한글 양식을 깨지 않고** 문서를 작성하는 데 집중해 만들었다. 실습에서 "비영리법인 현황 제출 양식" 1쪽을 가상 정보(레빗 테크놀로지 등)로 채우게 하자 양식을 깨지 않고 성실히 입력(전화번호 등 일부 미완). 화자가 계속 업데이트 중이라, 기존 설치자는 "스킬이 있는데, 설치할 때 바뀐 부분이 있으면 설치해"라고 시키면 됨.

### 6. Codex 앱 — 컴퓨터 유즈 & 카카오톡 자동화
- **컴퓨터 유즈**(플러그인): 설치 후 트레인 체크로 바뀜. 내 PC 애플리케이션을 실제로 조작하고 화면을 확인. 가상 마우스로 창을 올려 조작.
- 실습: "내 카카오톡 오픈 채팅방 중 '내 코드를 부탁해' 방의 채팅 100건을 조사해 보고하라" → 실제로 스크롤하며 100건을 읽고(10시 22분~15시 30분 전후) 매콤·램 이야기, 홍보성 글 관리 이슈, 자동 감시/운영 자동화 언급 등을 요약 보고. 화자의 방은 1300명 규모(주장).
- **예약됨(스케줄)**: 채팅으로 한 작업을 시간 걸어 반복 실행. "채팅으로 만들기"로도 잘 됨. 예: 매일 오후 4시에 '내 코드를 부탁해' 최신 채팅 500건을 살펴 부적절한 유저 닉네임을 보고. 만든 뒤 플레이 버튼으로 즉시 테스트(컴퓨터 유즈 실제 사용 여부 확인 권장).
- [의견] 코드로 카톡 자동화할 자신이 없고 Codex 100달러를 결제한 상태라면, 컴퓨터 유즈 + 예약됨 조합으로 카톡 관리 자동화를 하는 것도 상당히 괜찮다(토큰은 많이 씀).

### 7. Codex CLI — 개념·설치·환경
- CLI = Command Line Interface = "텍스트로만 조작". 터미널이 그 대표 도구.
- [의견] "안티그래비티에서 코덱스 CLI를 쓰면 짬뽕이라 정지당한다"는 오해 해소: 안티그래비티가 보여주는 터미널은 그 IDE 고유 기능이 아니라 **OS의 그냥 터미널**을 한 화면에 제공하는 것뿐. 여러 에이전트가 "협업"하는 게 아니라 같은 폴더를 함께 바라볼 뿐이라 정지 사유 아님.
- 설치: 구글에 "Codex CLI 인(install)" 검색 → 공식 CLI 셋업(macOS/리눅스/Windows, npm/홈브루). Windows는 PowerShell 또는 명령 프롬프트 명령어. **터미널에 명령어를 붙여넣기만 하면 설치 끝**.
- 환경: 초보자는 그냥 터미널에서 실행하면 `~`(물결, macOS 루트/홈)에서 시작해 시스템 파일이 다 보여 **위험**. 대신 VS Code(무료, 안티그래비티의 부모 격)로 작업 폴더를 열고 하단 터미널을 토글해 그 폴더 위치에서 `codex --dashdash로`(권한 전부 부여) 실행 권장. 실행 위치(폴더)가 곧 작업 범위.
- 실습: 데스크톱 폴더에서 "이미지 파일 모두 정리해서 폴더에 넣고, 파일 종류별로, 숨긴 파일 제외, 알아서 폴더 이름 짓고 정리해" → 순식간에 데스크톱이 실제로 정리됨.
- [의견] CLI로 넘어가는 이유는 자율성이 높아서. 화자는 시(Sea/시where?)를 쓰다가 요즘 [불명확: 음차 "시스"] 사용. 시where 계열은 Windows 미지원, Warp도 좋은 도구.

### 8. Q&A / 세일즈 구조
- **헤르메스 vs 코덱스** 토큰 효율: [의견] 헤르메스는 세팅된 스킬·AGENTS.md를 사용할수록 유저 패턴에 맞춰 스스로 진화하는 컨셉. 이상적으로 쓰면 스킬화로 토큰 절약 가능. 단 잘못된 지시로 잘못된 스킬이 주입되면 오히려 낭비(명암이 뚜렷). 화자는 코덱스 모델을 만든 회사의 앱이 더 잘해줄 거란 생각에 코덱스 선호.
- "요약 고정/환경(브랜치·변경 사항)이 프로젝트마다 다르게 나온다"는 질문: 화자 화면엔 '요약 고정'이 없어 확인 못 함([불명확]). 다만 폴더가 **git 세팅**되어 있으면 브랜치·변경 사항·진행 중 스킬 정보가 우측에 나오고, git 없으면 안 나온다고 설명.
- 사용량 확인: 어디서나 `/`(슬래시) → "상태"를 켜면 한도·사용량을 볼 수 있음(유용).
- 스킬/에이전트(AGENTS.md) 상세, "하네스 엔지니어링 입문 가이드", 마크다운·게임 만들기(뱀파이어 서바이버즈 등)·연애 프로필 사진·CS 자동화 등은 **책에 다 있다**며 도서 구매를 반복 요청(화자는 출판사 직원). 7월 중 책 영상 강의 업로드 예정(주장).

### [05:03–06:04] 커리큘럼 브리핑
오늘 순서: (1) 요금제 설명(ChatGPT·코덱스 요금제가 독특해서), (2) 각 도구 특징(ChatGPT/코덱스 앱/코덱스 CLI), (3) ChatGPT에서 이미지 생성 + 엑셀·스프레드시트 조작, (4) 코덱스 앱으로 한글 스킬 자동화 + 컴퓨터 유즈로 카카오톡 채팅 살펴보기, (5) 코덱스 CLI 환경 세팅(설치 문의가 많았다)과 간단 바이브 코딩. 한 시간 안에 다 하긴 힘들 것 같다고 예고.

### [06:04–08:10] 핵심: 토큰 미공유
요금제를 설명하는 이유가 있다며, **코덱스는 (ChatGPT와) 토큰을 공유하지 않는다**는 사실을 강조.
> "코덱스는 토큰을 공유하지 않습니다. 요거요 사실을 아셔야 됩니다." [09:20]
ChatGPT는 토큰 사용량을 보여주지 않아, 남은 양·기한 표시 없이 어느 순간 "다 썼습니다"가 뜨면 소진된 것. 반면 코덱스 앱/CLI는 같은 계정(HP 골든래빗 계정)이어도 "남은 사용량"이 표시된다. 표시를 안 해 줘서 모르는 사람이 많지만 서로 공유하지 않으므로, ChatGPT에서 할 수 있는 작업은 ChatGPT에서 최대한 하고 코덱스 전용 작업만 코덱스에서 하라는 게 결론.
> "채지에서 할 수 있는 작업을 굳이 굳이 코덱스에서 하지 말자가 이제 결론입니다." [13:47 인근/07:47]

### [08:10–08:54] 요금제 세부와 초기화 쿠폰
20달러·8달러·100달러·200달러 결제가 있고([불명확]: "8달러"는 음차/오인식 가능), 특이점으로 **20달러는 초기화 쿠폰(남은 사용량 리셋)을 안 준다**고 언급. 화자는 200달러를 써서 초기화가 가능하지만, 20달러 사용자는 초기화권이 없는 경우가 훨씬 많더라며 "정확한지는 모르겠다"고 단서를 단다(주장/미검증).

### [08:54–11:24] 도구 특징 ① ChatGPT — 이미지 생성
ChatGPT에서 할 수 있는 대표 작업은 이미지 생성(편하게)과 리서치. 코덱스 앱/CLI도 이미지 생성이 되지만 **이미지 제너레이션 스킬**을 써야 하며, 그 모델을 화자는 **덕테이프**라 부른다. 예시로 "주술회전과 기아자동차 콜라보 포스터"를 시연. ChatGPT는 "이미지 만들기" 버튼만 누르면 되고 생성 후 수정 옵션 등 UI/UX를 더 많이 제공.
> [의견] "저 같은 경우도 실제로는 이 이미지 생성하는 거는 채치T에서 좀 더 많이 하는 편입니다." [10:48]
유튜브 썸네일 에셋도 여기서 만든다고. 기능은 같지만 ChatGPT UI가 더 편한 차이가 있다.

### [11:27–13:35] 도구 특징 ② ChatGPT 심층 리서치
`+`(플러스) 메뉴의 **심층 리서치**: 내부적으로 검색 결과를 논문 형태로, 공신력 있는 자료 위주로 오래(한 시간 이상인 경우도) 서치해 신뢰도 높은 문서를 만들어 줌. 이건 ChatGPT에서만 제공하고 코덱스 앱엔 없다(플러스에도 리서치 항목 없음, 스킬·앱 메뉴만). [의견] 그래서 리서치 하나 때문에 ChatGPT를 쓴다고 봐도 됨. 기아 포스터는 차종 모델명을 제대로 주면 매우 잘 만들어 주고, 완성도는 양쪽 큰 차이 없음.

### [13:35–14:12] 겹치면 ChatGPT 먼저
[의견] 겹치는 가벼운 작업이 많은데, 공유를 안 하니 코덱스 앱에서 할 필요 없이 ChatGPT에서 먼저 하는 게 좋다. ChatGPT는 이미지·리서치 위주로 많이 쓰길 권함.

### [14:12–18:53] 도구 특징 ③ Codex 앱 = 클로드 데스크톱 포지션
코덱스 앱은 **클로드 데스크톱의 포지션**으로 생각하라 — 내 PC의 파일과 ChatGPT·코덱스 기능을 섞어 쓰고 싶을 때. 기존 ChatGPT는 파일을 업로드해야 했지만, 코덱스 앱은 "엑세스(권한)"에서 특정 폴더(예: 데스크톱)를 지정하면 그 안 파일을 스스로 보고 판단해 작업한다.
> "내 PC에 상주하는 인공지능을 쓰는 느낌으로 이제 코덱스 애플리케이션을 쓸 수 있습니다." [15:38]
실습: "데스크톱에 팰리세이드 관련 PDF가 있으면 찾아서 요약해 알려 줘" → 업로드 없이 바탕화면을 뒤져 카탈로그를 찾음(파일명 때문에 팰리세이드임을 잘 못 알아채는 등 다소 헤맴 — [의견] "살짝 멍청한 거 같긴 합니다"). 구분 기준: **"인공지능이 내 PC를 볼 수 있어야겠다 vs 없어도 되겠다."** 요약 후 메일 발송까지 이어지는 작업이면 코덱스 앱이 편하다.

### [18:53–19:30] 도구 특징 ④ Codex CLI 예고 + 3구분
코덱스 CLI로 결국 넘어오는 사람은 코딩 작업을 하려는 사람. 오늘 온 사람은 관심이 적을 수 있다고. 세 도구를 이 기준으로 구분해 쓰다 보면 나중엔 자유롭게 오간다고 정리.

### [19:30–21:44] ChatGPT 엑셀/스프레드시트 플러그인 설치
발표자료 없이 바로 실습. 인터넷에 "엑셀" 검색 → "엑셀 및 구글 시트용 ChatGPT 베타" → 엑셀용/구글시트용 설치 버튼. 엑셀은 Office 365 구독 상태에서 완전 동작(주장). "Get it now" 버튼으로 엑셀·파워포인트에 ChatGPT 플러그인 설치. 입력 절차가 조금 귀찮지만 추가 비용은 없음. 설치하면 엑셀 안으로 ChatGPT가 들어와, 엑셀 상황을 다 알고 조작. 예전엔 MCP 개념으로 설명하던 것을, 이제 가이드대로만 하면 계정 토큰으로 엑셀을 조작. 클로드도 엑셀은 지원하나 **스프레드시트(구글 시트)엔 접근 불가**라 구글 웍스 사용자는 ChatGPT가 유리.

### [21:44–26:16] 엑셀/스프레드시트 실전
셀을 클릭해 바꿀 때마다 "시트1 B1 선택됨"을 인지 — 잘 연결된 경험. 조사 작업 후 그 결과로 셀에 값 입력 가능. 추론 강도 패스트/스탠더드/헤비(화자는 스탠더드). 실습: "2026년 상반기 화장품 판매(연령·제품) 인기 조사, 데이터 기준 선정 후 시트1에 상품 50개 나열" → 리서치와 시트 입력을 한꺼번에 처리. 스프레드시트도 "스프레드시트" 검색 → 구글 시트용 설치 → 확장 프로그램에 ChatGPT 추가 → "오픈 사이드바" → 로그인 후 동일 사용. [의견] 스프레드시트엔 제미나이가 있지만 요즘 상태가 좋지 않아 차라리 ChatGPT가 나을 수 있음. 최대 장점은 **수식·함수 걸기**를 매우 잘하고, 복잡한 요청도 데이터 구분·누락 탐지를 잘함. 화자는 엑셀 전문가는 아니지만 기계적 처리를 매끄럽게 하고 싶을 때 아주 좋다고 봄.

### [26:16–27:24] 이미지 대량 vs 소량
이미지 생성 옵션이 계속 조금씩 바뀌었다며, ChatGPT는 한 번에 최대 10개 정도, 코덱스 앱(포스 앱)은 50개까지 한꺼번에 성공(테스트, 주장/미검증). [의견] ChatGPT는 대량 생성엔 부적합하고 집중된 한두 개 작업에 좋으며, 코덱스 앱이 대량 작업에 좋다. 이미지 만들기 실습은 소중한 라이브 시간을 낭비할 만큼 어렵지 않으니 생략.

### [27:24–31:27] 한글 HWPX 스킬 설치
한글 작업이 많으니 화자가 만든 HWPX 스킬로 실습. 주소는 K9(골든래빗) 배포처, 또는 해외 "skills.sh"(음차) 사이트에서 "HWPX" 검색해도 화자가 올린 스킬이 나옴. 설치 명령어를 복사 → 코덱스 앱 새 채팅에 폴더 지정 없이 "이 명령어를 이용해서 한글 스킬을 설치해 줘" → 에이전트가 명령 실행부터 마무리까지 알아서. 코덱스 앱엔 "스킬을 설치하는 스킬"도 있음. **스킬 = 상세 매뉴얼**: AI가 잘 모르거나 절차가 정해진 일을 아주 디테일하게 적어 "이대로만 해라"라고 주는 것. 한글은 국내 전용 프로그램이라 AI가 잘 못 다뤄 이 스킬을 배포했고 많이 퍼졌다(주장). 설치 후 **반드시 재시작**(Windows: 트레이 아이콘 우클릭 종료 / macOS: Cmd+Q). `$` 입력 시 지우개 모양 아이콘들이 스킬 목록이며 "HWPX"가 뜨면 성공. 안 뜨면 "설치했다더니 다시 켰는데 HWPX 스킬이 안 보여"라고 다시 시켜 에이전트가 청소·재설치(라이브에서 실제 몇 차례 반복).

### [31:27–36:48] 한글 스킬 실전 + 업데이트
`$` → "한글 문서 생성/읽기/편집 스킬"(HWPX) 클릭 시 파란색으로 활성화. 실습 파일 네 개(양식류) 준비. "비영리법인 현황 제출 양식" 1쪽을 채우게 함. 넘기는 방법 두 가지: (1) New Project로 폴더 자체를 코덱스 앱이 볼 폴더로 지정, (2) 스킬을 얹은 상태에서 작업 파일을 함께 첨부. "법인명은 레빗 테크놀로지로 치고 가상으로 최대한 성실하게 입력해 봐라" → [의견] 화자가 이 스킬을 만들 때 집중한 건 **기존 한글 양식을 깨지 않고** 작성하는 것. 결과: 데스크톱에 파일을 만들어 넘겨주고, 양식을 깨지 않고 성실히 입력(전화번호 등 일부 미완 — 프롬프트에 정보를 더 주면 개선). 화자가 계속 수정 중이라 완성도를 높이는 중이고, 기존 설치자는 "한글 스킬이 있는데, 설치할 때 바뀐 부분이 있으면 설치해"라고 시키면 업데이트 확인 후 진행.

### [36:48–40:40] 컴퓨터 유즈 — 카카오톡 채팅 조사
카톡방 질문("이 카톡방 내용을 에이전트가 볼 수 있냐")에서 출발. 코덱스 앱 플러그인의 **컴퓨터 유즈**를 설치하면 가이드 후 트레인 체크로 바뀜. 기능은 PC 애플리케이션을 실제 조작하고 화면을 확인하는 것. 카톡 단체방(화자 방 1300명, 주장) 운영의 골칫거리(주기적 알림, 부적절 닉네임 차단·경고 등)를 자동화하기 좋다. 실습: "'내 코드를 부탁해' 방의 채팅 100건을 조사해 보고하라" → 컴퓨터 유즈가 카톡방 존재를 확인하고 없으면 오픈채팅 목록에서 방을 찾아 창을 올려 **가상 마우스로 스크롤**하며 실제로 읽음. 토큰을 많이 쓰지만 [의견] 중요한 일이면 화자는 100달러 결제해 자동화하겠다고.

### [40:40–43:12] 예약됨(스케줄) 자동화
좌측의 예약됨 기능: 채팅으로 하던 작업을 시간 걸어 반복 실행. "채팅으로 만들기"·"수동으로 만들기"(책은 수동 설명)가 있고 채팅으로도 잘 됨. 실습: "컴퓨터 유즈로 매일 오후 4시쯤 오픈채팅방 '내 코드를 부탁해'의 최신 채팅 500건을 살펴 부적절한 유저 닉네임을 보고하라" → 예약 작업 등록(아시아/서울 오후 4시 매일). 100건 조사 결과도 도착: 10시 22분~15시 30분 전후, 매콤 가격·램 이야기, 홍보 광고성 글 관리 이슈, 자동 감시·운영 자동화 얘기 등 실제 내용 요약. 만든 예약은 목록에서 **플레이 버튼**으로 즉시 테스트(컴퓨터 유즈 실제 사용 여부 확인). [의견] 코드로 할 자신 없고 코덱스 100달러 결제 상태면 이 조합으로 카톡 자동화가 괜찮다.

### [43:12–46:16] 토큰 타이머 꿀팁

### [46:16–50:18] Q&A ① 헤르메스 vs 코덱스
너무 빨라 정신없을까 봐 잠깐 질문받음. 화자 저녁은 낙지(낙세) 언급 등 잡담 후, "헤르메스보다 코덱스에서 바로 쓰면 토큰 효율이 좋냐" 질문. [의견] 헤르메스는 세팅된 스킬·AGENTS.md(에이전트 MD)가 있고, 쓸수록 그것들을 스스로 수정해 유저가 자주 쓰는 방향으로 진화하는 컨셉의 툴로 앎. 이상적으로 쓰면 스킬화로 토큰 절약이 있을 것. 단 항상 옳은 지시만 하지 않으므로 **잘못된 스킬이 주입될 수 있고** 그러면 오히려 낭비 — 명암이 뚜렷한 툴. 헤르메스를 쓰는 이유는 토큰 효율보다 "나에게 맞춰지는 에이전트"라서. 화자는 코덱스 모델을 만든 회사의 앱이 더 잘해줄 거란 생각에 코덱스를 선호.

### [50:18–52:26] Q&A ② 요약 고정 / git / 사용량
"코덱스 우측 상단 요약 고정" 질문 → 화자 화면엔 없어 확인 못 함([불명확], "10분밖에 안 남았는데"). 대신 폴더가 **git 세팅**되어 있으면 우측에 브랜치·변경 사항·진행 중 스킬 등 코워크 정보가 나오고, git 없으면 안 나옴 → 프로젝트마다 다르게 보이는 이유. 사용량은 어디서나 `/`(슬래시) → "상태"로 한도·사용량 확인 가능(유용).

스킬·에이전트 MD는 설명이 길어 책의 "하네스 엔지니어링 입문 가이드" 장에 원리·동작·실습까지 아주 구체적으로 담았다며, 질문을 책으로 대체(웃으며 "책을 봐 주세요"). 곧 책 영상 강의도 올릴 예정(주장).

### [53:24–57:15] CLI란? 안티그래비티 "짬뽕" 오해
"안티그래비티에서 코덱스 CLI를 조작하면 정지 안 나?" 오해 해소. CLI = Command Line Interface = 텍스트로만 조작. 터미널이 그 도구로, `cd Desktop`처럼 텍스트 명령으로 이동·조작. [의견] 안티그래비티가 보여주는 하단 터미널은 그 IDE 고유 기능이 아니라 **OS의 그냥 터미널**을 한 화면에 제공하는 것뿐. 여러 에이전트가 협업하는 게 아니라 **같은 폴더를 함께 바라볼 뿐**이라 벤 다이어그램식 병합·정지 사유가 아님.

### [57:15–59:47] Codex CLI 설치
구글에 "Codex CLI 인(install)" 검색 → (맨 위 아닌) 공식 Codex CLI 셋업 항목. macOS/리눅스/Windows, npm/홈브루 안내. Windows는 PowerShell 또는 명령 프롬프트 명령어 둘 중 하나. 방법: 명령어 복사 → 터미널에 붙여넣기 → 설치 끝(화자는 기존 설치가 있어 "기존 것 지울까요?"가 뜸). [의견] "너무 편하죠?"

### [59:47–01:03:04] VS Code 환경 & 실행 위치의 중요성
> 아래 타임스탬프는 실제로 1시간대(≈1:00:01~)다.
코덱스를 쓰기 편한 환경으로 시where 또는 VS Code 추천(VS Code는 무료, 안티그래비티의 부모 격). 초보자가 그냥 터미널에서 실행하면 `~`(물결 = macOS 홈/루트)에서 시작해 모든 시스템 파일이 보여 **위험**(실수로 삭제 시 문제). 대신 VS Code에서 원하는 작업 폴더(예: 데스크톱)를 열고 하단 터미널 토글을 켠 뒤, 그 폴더 위치에서 코덱스를 `--로`(대시대시로 = 권한 전부 부여)로 실행하면 눈에 보이는 화면과 완전히 동일한 위치에서 동작. "지금 폴더에 뭐가 보이니?"로 확인. 실습: "이미지 파일 모두 정리해서 폴더에 넣어 주고, 파일 종류를 따져서, 숨긴 파일 제외하고, 알아서 폴더 이름 짓고 정리해" → 좌측 폴더 구조가 확 바뀌며 데스크톱이 순식간에 실제 정리됨.

### [01:03:04–01:04:00] CLI로 넘어가는 이유 & 도구 선택
[의견] 코덱스 앱은 앱 설치가 필요하지만 CLI는 터미널만 있으면 어디서든 쓸 수 있고, 사람들이 CLI 환경으로 넘어가는 이유는 **자율성이 높아서**. 화자는 시where(Windows 미지원)를 언급하고, Warp도 좋은 도구이며 처음엔 Warp를 많이 썼지만 요즘은 [불명확: 음차 "시스/시where"]를 쓴다고. 초보자는 VS Code 하단 터미널 + 폴더 지정 실행을 추천(작업 결과가 눈에 바로 보임).
