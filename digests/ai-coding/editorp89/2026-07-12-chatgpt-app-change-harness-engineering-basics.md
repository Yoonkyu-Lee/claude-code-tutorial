## 요약 (주제별)

### 챗GPT 앱의 변화점 — 웹 브라우저 → PC 접근 애플리케이션
- Codex가 애플리케이션 이름으로 바뀌어 둘로 분할됐다가 또 이름이 바뀜(영상 시점 이틀 전). 일상 업무는 "챗GPT 워크", 코드 작업은 "코덱스"로 전환해 진행. [의견] 이름·모드가 바뀌어도 동작 원리는 완벽히 동일하고, 일하는 경향성만 조절된 것 — 껍데기만 바뀌고 본질은 같다.
- 핵심 변화: 웹 브라우저 챗GPT는 PC 폴더에 접근·상호작용할 수 없지만, **애플리케이션은 PC의 파일·폴더에 접근할 수 있다**. 이것이 사용자에게 아주 큰 변화라고 강조.

### 인공지능 vs 에이전트 — 자율성과 멀티턴
- 화자는 엄밀한 범주보다 "느낌적 구분"을 권함. 에이전트는 인공지능 범주 안이지만 더 **자율적인 실행·판단**을 하는 존재.
- 과거 챗GPT = 웹에서 대화/이미지 생성/파일 분석, 한 번 대화 가면 한 번 결과가 오는 "일대일 교환"(심심이 비유). [의견] 지금은 한 번 입력에 여러 턴이 돌며 스스로 판단·결과 생성·자기 평가 후 답을 돌려줘 만족도가 올라감.
- 에이전트 정의: 답을 줄 때까지 턴이 길고, 스스로 파일·결과를 만들어 평가하고 사용자 목표 부합을 스스로 점검 후 답한다.
- 현재 한계(명시): "8월 여행 계획 짜줘"에 여행사 등록까지 해주는 수준이 진짜 에이전트지만 **아직 그 정도는 아니다**.

### "컴퓨터 자원을 맡긴다" + Computer Use
- 챗GPT 앱이 PC에 설치돼 파일 생성·삭제·정리, 폴더 생성, 코드 실행(메모리·CPU 사용)을 할 수 있음 = "컴퓨터 자원을 맡긴다".
- 대표 예: **컴퓨터 유즈(Computer Use)**. "오픈 채팅방에서 최근 대화 찾아줘"라 하면 실제로 카카오톡을 실행해 방 내용을 찾아 확인해줌(실제 PC 자원 사용). 나중에 실습 예정.
- 맡길 자원: 키보드·마우스·메모리·운영체제 권한 등. [의견] 이를 감안해 일을 시켜야 하며, 웹 브라우저에서 시키는 대로만 시키면 20달러를 내고도 일을 제대로 안 시키는 셈. 이 개념 유무의 차이가 크다.

### 마크다운 — 인공지능이 선호하는 문서 양식
- 사람이 워드·한글·엑셀을 쓰듯 인공지능은 **마크다운**을 선호. 같은 문서라도 마크다운으로 주면 훨씬 잘 파악함. AI 결과물에 `#`·`*` 기호가 섞이는 것이 마크다운.
- 유래: 개발자용 문서 양식 → 개발자가 많이 만들어 AI가 대량 학습 → AI가 뱉어냄 → 뱉어내는 것을 받아들이는(먹는) 것도 선호. `.md`(AGENTS.md·스킬.md·디자인.md 등)가 다 마크다운.
- 의미 부여: `#`(H1)은 HTML의 볼드·큰 포인트와 같은 의미, `*`로 감싸는 볼드는 사람이 "중요"로 인식하듯 에이전트도 중요로 파악(두께 판단을 못 하니 별표로 대체). 문서에 의미가 남아 전달이 좋아짐.
- 실용 팁: 프롬프트에서 강조하고 싶은 부분을 `*`로 감싸면 AI가 더 중요하게 판단해 거기에 맞춰 답한다.

### AGENTS.md — 채팅에 항상 올라가는 방향성 문서
- 에이전트가 채팅에 **항상 올려두는** 마크다운 문서. 존재하기만 하면 새 채팅 시작 시 시키지 않아도 무조건 먼저 읽은 뒤 답한다(채팅창 안에 들어간 상태로 시작).
- 사전에 나눠야 할 대화를 넣어두면 처음부터 말하지 않아도 기본으로 깔고 대화 → **인공지능의 방향성을 정해줄 수 있다**.
- 챗GPT 앱 접근 경로: 왼쪽 아래 이름 → 설정 → 개인 맞춤 설정 탭 → **맞춤형 지침**. 이것이 곧 AGENTS.md.
- 실습: "앞으로 중국어로만 대답해라" 입력·저장 → "에이전트 MD 저장" 메시지 → 새 채팅에서 "안녕"이라 하면 중국어("니하오")로 답. 코덱스에서도 동일하게 동작.
- 채팅창 관리 대체: 많은 이가 소중한 대화를 잃지 않으려 "채팅창 관리"를 하지만, 컨텍스트 윈도우 때문에 결국 내용을 까먹음. 대신 "지금 대화를 바탕으로 AGENTS.md 요약해봐"로 만들어두면 관리가 불필요. [의견] 이걸 모르면 돈을 하늘에 버리는 것과 같다.

### AGENTS.md 작성 규칙 + 메타프롬프팅
- (공식 문서에 일부 유출된) 규칙: 너무 길게 X(요약식으로) / 모든 채팅에 영향을 주므로 너무 세부적 내용 X(끼어들어 다른 일 방해) / **범용적 규칙**을 적기 / 애매하게 X / **충돌하는 규칙** X.
- 규칙을 외워 직접 타이핑할 필요 없음. "이 규칙에 맞게 네가 AGENTS.md 적어봐"처럼 **에이전트에게 프롬프팅을 시키는 = 메타프롬프팅**을 "숨쉬듯" 쓰는 것이 요즘 대세. 규칙은 기억만 하고 AI에게 작성·관리를 맡긴다.

### 스킬 — 뾰족한 작업용, 필요 시 로드
- 등장 이유: AGENTS.md는 길게 못 쓰고 상황마다 필요한 내용이 다름. 문서 작업/메일 작성/개인 공부/코드 등을 공용 AGENTS.md에 다 넣으면 섞여서 서로 영향 → 각각 **스킬**로 구분하면 잘 동작.
- 발전 순서: 처음엔 AGENTS.md만 → 이후 스킬 등장. 스킬은 구성 방식이 AGENTS.md와 유사하되 **아주 뾰족한 작업에 특화된 방법**을 담음.
- 동작: 에이전트가 항상 읽지 않음. 게임 스킬처럼 채팅 바깥에 대기하다가 필요할 것 같으면 그때 들고 들어옴. **유일한 차이** — AGENTS.md는 항상 채팅창 안, 스킬은 필요 시 로드.
- 장점: 여러 개 있어도 서로 섞이지 않고 AGENTS.md가 알아서 골라옴(일부러 넣을 필요 없음, 넣어도 됨). 잘 만들어두면 알아서 씀. 스킬 크기는 제각각.

### 디자인.md 등 파생 문서
- 이름처럼 디자인을 가이드하는 문서. AI 코딩 시 이상하게 나오는 디자인을 바로잡음(안에 디자인 가이드만 → 디자인 관련 뾰족한 작업만).
- 이런 파생 문서는 생겼다 필요 없어지면 사라지기도 함. 다만 지금은 **AGENTS.md + 스킬 두 축**이 기본 개념.

### 하네스 엔지니어링
- 정의: 넓은 방향성은 AGENTS.md로 정하고, 세부 작업마다 스킬을 가져와 뾰족한 방향성을 조절해주는 것. [의견] 거창하게 볼 것 없이 AGENTS.md·스킬 두 개로 시작해 방향성이 잡히는지 관찰하며 쓰다가 필요한 것을 조금씩 공부하면 됨.
- 비유: 고삐·안장·박차 없이 말 타는 것보다 갖추고 타면 목표 방향으로 더 잘 감. 에이전트를 잘 갖춰진 가이드로 둘러싸 원하는 방향으로 동작시키는 것 = 에이전트를 가이드하는 여러 방법들.

## 전문 — 시간순 워크스루

### [00:00–00:49] 앱 이름 변경 안내, 껍데기만 바뀜
3장 시작. 얼마 전 "Codex"가 애플리케이션 이름으로 바뀌어 둘로 분할됐다가, 또 이름이 바뀌었다고 설명한다. 이제 챗GPT 애플리케이션을 다운로드하면 일상 업무는 "챗GPT 워크"에서 하고, 코드 관련 작업은 "코덱스"를 눌러 전환해 진행하는 구조다. 화자는 미리 못 박는다.

> "두 모드를 바꾼다고 해서 근본적인 동작 원리가 바뀌는 건 없습니다. 그냥 동작 원리는 완벽하게 동일하지만 이제 얘가 일하는 경향성을 조금 이제 조절을 해 놓은" [00:22]

자기 도서(교재)에서는 이를 "코덱스 앱"이라 명명했는데 이름이 바뀐 지 이틀밖에 안 됐다고 한다. 업데이트에 맞춰 바뀐 부분은 설명하겠지만 껍데기만 바뀌고 본질은 똑같으니 혼란 없기를 바란다고 정리한다.

### [00:49–01:56] 웹 브라우저 vs 애플리케이션 — PC 접근이라는 변화점
오늘은 중요한 얘기라며, 챗GPT 애플리케이션(코덱스 앱)으로 넘어올 때 무엇이 달라지는지를 설명한다. 보통 사람들은 챗GPT를 온라인 웹 브라우저에서 쓰는 것으로 생각하는데, 애플리케이션 단계로 내려오면 개념이 달라진다. 비유로, 컴퓨터 안에 챗GPT 애플리케이션이 설치되어 있고, 그 PC 안에는 바탕화면·다운로드 등 여러 폴더가 있는데, 이제부터는 이 챗GPT 애플리케이션이 **여러분 PC(폴더)에 접근할 수 있다**는 것이 핵심 변화라고 강조한다.

> "웹브라우저는 여러분들 PC 안에서 이 폴더와 함께 상호 작용을 할 수가 없기 때문에 애플리케이션 자체가 이 폴더에는 접근을 할 수가 없어요. [근데] 이제는 요렇게 접근을 할 수 있다는 개념이 여러분들한테는 아주 큰 변화가 된다" [01:47]

### [01:56–02:07] 지금부터의 내용은 앱 변경과 무관한 원론
이어지는 내용은 애플리케이션 이름이 바뀌고 말고와 상관없는 "아주 원론적인 얘기"라고 예고한다. 그러니 편하게 듣고 넘어가면 된다고 한다.

### [02:07–03:00] 인공지능 vs 에이전트 — 자율 판단과 멀티턴
"인공지능"과 "에이전트"를 사람들이 섞어 쓰는데, 엄밀한 범주 구분보다 느낌적 구분을 하라고 안내한다. 에이전트는 인공지능 범주 안에 있지만 더 **자율적인 실행·판단**을 할 수 있는 녀석이다. 과거 챗GPT는 웹 브라우저에서 대화·이미지 생성·파일 분석 정도를 했고, 한 번 대화가 가면 한 번 결과가 오는 "심심이처럼 일대일 교환"을 정직하게 하는 존재였다. [의견] 지금은 한 번 대화가 가면 여러 턴이 돌면서 스스로 판단하고 결과를 내리는 과정이 중간에 많이 포함된 뒤 답변을 돌려주며, 그래서 만족도가 올라간다고 설명한다.

### [03:00–03:39] 에이전트의 정의와 현재의 한계
에이전트는 답변을 줄 때까지 턴이 길고, 스스로 판단해 변화를 주거나 파일·결과를 만들어 평가하고, 사용자 목표에 맞는지 스스로 생각한 뒤 답을 준다. 예로 "8월 여행 계획 짜줘"라고 하면, 진짜 에이전트라면 여행 계획을 다 짜고 여행사 등록까지 해주는 수준이어야 하지만 **지금은 아직 그 정도는 아니다**라고 현재 한계를 명시한다.

### [03:39–05:00] "컴퓨터 자원을 맡긴다" + 컴퓨터 유즈 예시
사용자가 일대일 교환이 아니라 "이 정도는 알아서 하겠지"라는 기대를 갖게 된 것은 인공지능이 에이전트 방향으로 체중을 이동했기 때문이다. 이제 인공지능에게 컴퓨터 자원을 맡길 수 있게 됐다는 것 — 챗GPT 애플리케이션이 PC에 설치되어 파일을 생성·삭제·정리하거나 폴더를 만들 수 있고, 코드를 실행하면 메모리·CPU를 쓴다. 이를 통틀어 "컴퓨터 자원을 맡긴다"고 표현한다. 가장 쉬운 예가 **컴퓨터 유즈(Computer Use)** 기능이다.

> "오픈 채팅방에서 최근 대화 찾아줘. 이렇게 얘기를 하면은 진짜로 카카오톡을 연 다음에 실제로 카카오톡에서 방의 내용을 찾아가지고 얘가 실제로 카톡을 실행을 해 가지고 이제 채팅을 확인해 주는 모습을 볼 수가 있습니다." [04:35]

이는 나중에 실습할 예정이라고 한다.

### [05:00–05:25] 자원을 감안해 일을 시켜야 한다
키보드·마우스·메모리·운영체제 권한 등을 에이전트(챗GPT 애플리케이션)가 쓸 수 있게 됐으므로 이를 감안해 작업을 시켜야 한다. 과거처럼 웹 브라우저에서 시키는 대로만 시키면 돈을 주고도 일을 제대로 안 시키는 결과가 된다. [의견] 이 개념을 머릿속에 넣고 쓰느냐 아니냐의 차이가 크며, 의외로 아직도 웹 브라우저 채팅이 전부라 생각하는 사람이 많다. 20달러를 냈으면 이런 것(PC 작업)을 시켜보라고 권한다.

### [05:25–06:35] 인공지능이 좋아하는 문서 양식 — 마크다운
에이전트를 잘 쓰려면 이들이 좋아하는 문서 양식을 알아야 한다. 사람은 워드·한글·구글 독스·엑셀 같은 걸 좋아하듯, 인공지능은 **마크다운**을 매우 선호한다. 선호한다는 것은 같은 문서라도 마크다운으로 주면 훨씬 더 잘 파악한다는 뜻이다. 인공지능 결과물에 `#`(샵)이나 `*`(별) 같은 기호가 섞여 나오는 것이 바로 마크다운이다. 원래 개발자들이 쓰던 문서 양식인데, 개발자들이 이 문서를 워낙 많이 만들어 인공지능이 학습을 많이 했고, 그래서 뱉어내기 시작한 것 — 뱉어낸다는 건 그런 걸 받아들이는(먹는) 것도 좋아한다는 의미다. 공부하다 보면 AGENTS.md·스킬.md·디자인.md처럼 `.md` 파일을 많이 보게 되는데 이게 다 마크다운이다.

### [06:35–07:26] 마크다운의 의미 부여 — H1, 볼드, 프롬프트 팁
마크다운은 어렵지 않다. 큰 제목을 `#`(H1)로 표시하는 것은, HTML에서 볼드로 40포인트 크게 만드는 것과 인공지능에게 동일한 의미다. 텍스트를 `*`로 감싸면 볼드(두껍게)가 되고, 사람이 두꺼운 글씨를 "중요한 내용"으로 인식하듯 에이전트도 똑같이 인식한다 — 에이전트는 두꺼운지 아닌지 판단을 못 하니 별로 감싸는 것을 볼드로 파악한다. 이렇게 문서를 적으면 **의미가 남고**, 그 의미를 전달하기 좋다.

> "프롬프트를 입력을 할 때 별별로 감싸서 이렇게 주시면은 얘는 그걸 더 중요하다라고 판단을 하고 거기에 조금 더 맞춰 가지고 답변을 해 주게 돼 있습니다." [07:10]

지금은 사소하지만 나중에 해보라고 권한다. 마크다운은 그냥 "AI가 좋아하는 형식의 문서"라고 이해하면 된다.

### [07:26–08:22] AGENTS.md — 채팅에 항상 올라가는 문서
에이전트가 항상 채팅에 올려두는 문서가 있는데 그것이 **AGENTS.md**다. 앞서 설명한 마크다운 형식의 문서인데, 이 문서는 존재하기만 하면 새 채팅을 시작할 때 인공지능이 **시키지 않아도 무조건 먼저 읽은 뒤** 답을 하게 되어 있다. 즉 채팅창 안에 이 문서가 들어간 상태로 채팅이 시작된다. 따라서 인공지능과 사전에 나눠야 할 대화 같은 것을 AGENTS.md에 넣어두면, 처음부터 말하지 않아도 그것을 기본으로 깔고 대화하게 된다. 챗GPT 앱에는 이 AGENTS.md를 작성할 수 있는 "맞춤형 지침"이라는 메뉴가 있고, 이것을 쓰면 AGENTS.md를 경험할 수 있다(실제로 이게 AGENTS.md이기도 하다).

### [08:22–09:21] 실습 — 맞춤형 지침(AGENTS.md) 저장, "중국어로만 답해라"
실습. 챗GPT 워크 상태에서(코덱스 상태에서 해도 무방) 왼쪽 아래 이름 → 설정 → **개인 맞춤 설정** 탭 → **맞춤형 지침**으로 들어간다. 여기가 AGENTS.md다. 예시로 "앞으로 중국어로만 대답해라"를 입력하고 저장하면 위쪽에 "에이전트 MD를 저장했다"는 메시지가 뜬다. 즉 이 문서를 항상 올려둔 뒤 읽고 답하겠다는 의미다. 앱으로 돌아가 새 작업(새 채팅)을 열고 "안녕"이라고 하면, AGENTS.md가 올라가 있어 한국어로 물어도 중국어로("니하오") 답한다.

### [09:21–09:56] 코덱스에서도 동일 + 방향성을 정해준다는 의미
이 동작은 코덱스에서도 마찬가지다 — AGENTS.md가 미리 채팅에 올라가 있으므로 "안녕"이라 하면 그것을 읽고 중국어로 답한다. AGENTS.md를 안다는 것은 곧 **인공지능의 방향성을 정해줄 수 있다**는 뜻이다. 화자는 예시 지침을 지운다.

### [09:56–10:47] 채팅창 관리는 필요 없다 — AGENTS.md로 대체
AGENTS.md를 알고 모르고의 생산성 차이가 크다. 에이전트를 쓰는 방향성·목적은 사람마다 다른데, 그때마다 새 채팅을 열 때 알려주고 방향성을 만들어주는 건 불편하다. 그래서 많은 사람이 "채팅창 관리"를 하는데, 이는 안타까운 일 — AGENTS.md가 있으면 그 내용을 거기에 적으면 되므로 채팅창 관리가 필요 없다.

> "이거 가지고 가다가 보면은 얘가 내용을 까먹습니다. 왜냐면 그 나중에 얘기할 우리가 컨텍스트 윈도우나 때문인데" [10:18]

소중한 대화 내용이 있으면, "지금 우리 대화를 바탕으로 AGENTS.md 요약을 한번 해봐라"라고 하면 지금까지의 대화 방식으로 동작할 수 있는 AGENTS.md를 만들어준다. 그러면 주로 하는 채팅 관련 내용이 거기에 들어가, 그다음부터는 가르치지 않아도 챗GPT 에이전트와 함께 일할 수 있다. [의견] 이걸 모르면 돈을 하늘에 버리는 것과 같다.

### [10:47–11:34] AGENTS.md 작성 규칙 (유출된 공식 규칙)
AGENTS.md를 알았다고 모든 것을 거기에만 적으려 하기 쉽지만, 작성에는 (공식 문서에 몇몇 유출된) 규칙이 있다.
- 너무 길게 쓰면 안 되고 요약식으로 써야 한다.
- AGENTS.md는 모든 채팅창에 영향을 준다. 너무 세부적인 내용을 적으면 다른 일을 할 때 그것이 끼어들어 일이 잘 안 될 수 있다.
- 웬만하면 **범용적인 규칙**을 적는다.
- 애매하게 적지 않는다.
- **충돌하는 규칙**(다른 데서 해야 하는 작업과 충돌되는 규칙)을 넣지 않는다.

### [11:34–12:08] 메타프롬프팅 — 규칙에 맞게 AI가 작성하게 하라
이 규칙들을 직접 다 입력할 필요는 없다. "이 규칙에 맞게 네가 AGENTS.md를 적어봐라"라고 하면 AGENTS.md가 작성된다. 이런 방식을 **메타프롬프팅**이라 한다 — 내가 직접 프롬프팅하는 게 아니라 에이전트에게 프롬프팅을 시키는 것.

> "이 메타프롬프팅을 그냥 숨쉬듯이 쓰셔야 돼요. 이거를 [규칙을] 외워 가지고 직접 이제 타이핑하시려는 분들이 있는데 그냥 이런 규칙을 던져 주고 만들어 보라고 하면 되겠죠." [11:53]

요즘은 그렇게 쓰는 게 대세다. 규칙은 기억만 했다가 "이런 방식으로 써봐라"라고 시켜 AGENTS.md를 관리하면 된다.

### [12:08–13:11] 스킬의 등장 이유 — 뾰족한 작업 특화
궁금증이 생긴다. 길게 쓰지 말라는데 하고 싶은 말은 많고, 이 일 저 일 상황마다 필요한 AGENTS.md가 다르다면 어떻게 하나? 이를 **스킬**로 해결한다. 발전 과정상 처음엔 AGENTS.md밖에 없었고 그다음 스킬이 나왔다. 스킬은 AGENTS.md와 유사한 방식으로 구성되지만, AGENTS.md가 전반적인 채팅 방향성을 정한다면 스킬은 **아주 뾰족한 작업에 특화된 방법**을 적어놓는 것이다. 예: 회사 문서 작업만, 회사 메일 작성만, 개인 프로젝트 공부, 코드 관련 내용 등 — 이들을 공용 AGENTS.md에 다 넣으면 섞여서 서로 영향을 주니 좋지 않다. 각각 스킬로 구분해두면 잘 동작한다.

### [13:11–14:14] 스킬의 동작 방식 — 채팅 바깥에 대기, 필요 시 로드
스킬은 에이전트가 항상 읽지 않는다. 게임 스킬처럼 에이전트가 채팅 바깥에 두고 있다가, 채팅을 하다 필요할 것 같으면 그때 들고 들어온다. AGENTS.md와 스킬의 유일한 차이가 이것 — AGENTS.md는 **항상 채팅창 안에** 있고, 스킬은 항상 있는 게 아니라 필요할 때 들어온다. 그래서 스킬은 여러 개가 있어도 서로 섞이지 않고, AGENTS.md가 알아서 골라오므로 일부러 넣어줄 필요가 없다(넣어줘도 되지만 알아서 가져온다). 정리하면: 채팅창을 만들면 AGENTS.md가 무조건 채팅창 안에 올라가고, 바깥에는 세부 작업용 스킬들(크기는 제각각)이 대기하다가, 이 스킬로 일해야겠다는 판단이 되면 스킬을 들고 와서 일한다.

### [14:14–14:53] 디자인.md 등 파생 문서
비슷하게 **디자인.md**도 나왔다. 이름처럼 디자인을 가이드하는 문서로, AI 코딩을 하다 보면 디자인이 이상하게 나오는데 그것을 바로잡아주는 녀석이다. 안에는 디자인 관련 가이드만 들어 있어 디자인 관련 뾰족한 작업만 한다. 이런 문서들은 생겨났다가 필요 없어지면 사라지기도 할 것이다. 하지만 지금은 **AGENTS.md와 스킬 두 축**이 거의 기본 개념이라고 가져가면 된다.

### [14:53–15:49] 하네스 엔지니어링의 정의
넓은 방향성은 AGENTS.md로 정하고, 세부 작업마다 스킬을 가져와 뾰족한 작업의 방향성을 조절해주는 것이 **하네스 엔지니어링**이다. 거창하게 생각하는 사람이 많지만 별게 없다 — AGENTS.md와 스킬 두 개로 시작해 방향성이 어떻게 잡히는지 관찰하며 쓰다가, 이후 필요한 것을 조금씩 공부하면 된다.

> "이렇게 곱비를 걸어 줬잖아요. … 세팅을 잘 해 주고 그다음에 스킬들 주변에 내가 해야 되는 일들 위주로만 잘 주변에 이렇게 배치를 해 주면은 이 에이전트가 어떻게 동작하겠어요? 완전 뜬금 없는 데서 시작하는 것보다는 주변에 이렇게 잘 갖춰진 가이드들이 있으면은 훨씬 더 내가 원하는 방향으로 잘 동작을 하겠죠." [15:20]

즉 에이전트를 가이드하는 여러 방법들이 하네스 엔지니어링이다.

### [15:49–16:56] 오늘 실습 정리 + 다음 예고
이것들은 도서(교재) 실습을 진행하며 하나씩 같이 해볼 예정이고, 오늘은 하나만 실습했다 — **자기 맞춤형 지침(AGENTS.md) 하나**. 개념을 꼭 알아두고, 나중에 실습할 때 "그때 했던 얘기가 이거구나" 하고 떠올리면 좋다. 오늘은 03-1 챕터를 진행했다. 앞으로 챗GPT 애플리케이션에서 에이전트와 다양한 작업을 하며 AGENTS.md·스킬·앱 기능을 하나씩 알아볼 것이다. 오늘은 다소 어렵게 느껴질 수 있으나 돌아보면 별 얘기 없고, 요지는 "에이전트에게 특수한 방향성을 정해줄 수 있고 그걸 쓸 줄 알면 에이전트를 더 잘 부린다"는 것.

> "고삐 없는 채로 말 타는 것보다는 고삐도 있고 안장도 있고 박차도 있으면은 조금 더 잘 타겠죠. 목표하는 방향대로 조금 더 잘 가기 위해서 이 하네스 엔지니어링을 해야 된다" [16:38]

다음 영상에서는 실습을 진행하며 이것저것 확인하는 시간이 될 것이라고 예고하며 마친다.
