## 요약 (주제별)

### Insane-search가 뭔가 (도구 정체성)
- Claude Code용 **플러그인**. 제작자는 지피타쿠(GP타쿠, GP님)이며, 화자는 이분과 전화 통화까지 해 본 **한국에 사는 한국인 제작자**라고 밝힘 (주장).
- 핵심 컨셉: "사람이 웹브라우저로 접속해 볼 수 있는 공개 페이지라면, 클로드 코드(에이전트)도 볼 수 있어야 한다."
- 소개 페이지의 카피 "포기는 배추 셀 때나 쓰는 말이다" — 공개된 웹페이지라면 끝까지 가져온다는 강한 메시지.

### 동작 원리 (봇 차단 우회가 아니라 '단계 상승')
- 많은 사이트가 에이전트를 차단하는 장치를 둠. 예로 쿠팡에 클로드 코드/코덱스로 몇 번 조회하면 갑자기 막고 **403**을 반환 (화자 경험담).
- Insane-search는 합법적·공식적인 접근 방법들을 **순차적으로 단계를 올려 가며** 다 시도해서, 결국 브라우저에서 보이는 데이터를 다 볼 수 있게 해 준다.
- 특히 **내부 API 호출**을 찾아 활용: 상품 목록 같은 데이터는 페이지가 내부적으로 API 콜로 프리패치한 뒤 봇탐지 보안으로 감싸 화면에 그린다. 단순 페이지 크롤링은 보안이 먼저 동작해 막히지만, 그 원천 API를 찾아 직접 호출하면 가져올 수 있다는 설명.
- 안 되면 Playwright로 실제 웹브라우저를 띄워 작업하는 것까지 알아서 시도. "이것도 해 봐, 저것도 해 봐"를 사람이 일일이 안 시켜도 자동 진행.

### 불법 아님 — 선 긋기
- 로그인해야만 볼 수 있는 것, 결제해야만 볼 수 있는 유료 블로그 등은 **가져오지 않겠다**는 것이 제작자의 명시적 경계. 불법 로그인·유료 우회는 안 함.
- [의견] 화자는 "눈으로 볼 수 있으면 에이전트로도 볼 수 있으면 좋지 않나"라는 입장이며, 우회가 아니므로 문제없을 거라 생각한다고 밝힘 (개인 의견).

### 설치 절차 (라이브 시연)
- 작업용 디렉토리를 하나 만들고(예: `랩 인세인 search`) `cd`로 이동한 뒤 `claude`를 skip-permissions 옵션으로 실행.
- 소개 페이지의 install 명령을 복사. 처음엔 셸에 붙여넣어 안 됨 → 슬래시 명령(내장) 형태라 **클로드 코드 안에서 한 줄씩** 실행해야 함을 확인.
- 흐름: marketplace 추가(`successfully added marketplace`) → `plugin install insane-search` → `reload` 하면 그 채팅 세션에서 사용 가능. 이후 사용자가 더 할 일은 없음 — 평소 안 가져와지던 데이터를 그냥 시켜 보면 됨.

### 시연 케이스와 결과 (모두 화자 라이브 주장)
- **유튜브 자막**: "이 영상의 자막을 가져와 봐"라고만 해도 인세인 서치가 유튜브 메타데이터 + 한국어 자동생성 자막까지 텍스트로 깔끔히 추출. 유튜브로 학습하는 사람에게 유용(뽑은 텍스트를 학습용 웹페이지로 정리시키는 활용도 언급).
- **네이버 쇼핑 핫딜(키즈/라이프/푸드)**: 상품을 싹 가져와 **CSV로 저장**. 클로드 코드에서 "**308개가 중복 제거까지** 완벽하게" 됐다고 확인(주장). "약 3분 만에" 핫딜 데이터를 다 가져왔다며 냅다세일/파리바게트 망고밤케이크 등 실제 항목 확인. 과거 브라이트 데이터에는 네이버가 없어 못 했던 걸 이걸로 해결.
- **쿠팡 / 해외 사이트(알리익스프레스)**: 알리익스프레스 식료품 100개를 CSV로 요청. 알리는 봇탐지가 강해 화자가 예전에 포기했던 대상. 내부 상품 피드 API를 찾아 시도했고 화자는 "무조건 된다"고 기대(주장). 다만 영상 내 라이브에서 알리는 "제일 안 되네요"라며 API 오버로드 에러가 반복 발생 → 결국 Playwright로 브라우저를 띄워 작업하는 단계까지 진행. 화자는 결국 될 것이라며 끝까지 기다리진 않고 넘어감.

### 마케팅/세일즈 구조 (정보로 1회 기록)
- 깃허브 별(star)을 눌러 달라는 요청 반복 — "개발자끼리 인기 투표일 뿐, 돈이 오가지 않는다"는 설명. 별을 누르면 제작자 인기도가 올라가 더 좋은 걸 만들어 커뮤니티에 공유하게 된다는 논리.
- 화자·제작자 모두 **금전 이득 없음**을 반복 강조. 화자는 "최근 유료 광고를 두 개 올려서 돈에 미쳤나 오해받는데, 이건 광고가 아니고 내 수익 없다"고 해명 (주장/미검증).
- 도구 사용 비용: 별도 결제 없음. 클로드 코드 구독 사용량(토큰)에서만 소모.

### 화자의 관점·활용 제안 [의견]
- 상품 사업/가격 비교/마케터(가격 경쟁력, 이벤트가 산정)에게 특히 유용하다는 추천.
- [의견] "크롤링 강의 몇 개는 없어지겠다" — 자신이 올린 무료 크롤링 강의조차 이제 필요 없을 것 같다고 표현.
- [의견] 바이브 코딩에만 집중하지 말고, 에이전트로 일상 업무를 돕는 방향도 생각해 보라는 권유. 유용한 도구가 보이면 갑자기 찾아와 소개하고 사라지겠다며 마무리.

## 3줄 오리엔테이션
- 지피타쿠가 만든 Claude Code 플러그인 **Insane-search**를 소개·라이브 시연하는 무료 도구 리뷰 영상.
- 핵심 주장: 봇 차단(403)으로 막히던 공개 페이지를, 단계를 올리고 내부 API를 찾아 결국 다 가져온다(로그인·결제 벽은 넘지 않음).
- 시연: 유튜브 자막 / 네이버 핫딜(308개 CSV) 성공, 알리익스프레스는 라이브에선 에러가 이어지며 브라우저 자동화까지 진행.

## 전문 — 시간순 워크스루

### [00:00–00:24] 인사와 오늘의 주제 소개
화자는 새 영상으로 "인센치"(인세인 서치, Insane-search)를 들고 왔다고 하며 제작자 "지피타쿠 님"을 부른다. 자신은 아직 한 번도 안 써 봤지만 커뮤니티에서 언급이 많았고, 실제 기능을 보니 "이건 좋겠다" 싶어 소개하러 왔다고 밝힌다.

### [00:24–01:00] 본질 — 공개 페이지라면 에이전트도 봐야 한다
Insane-search의 본질은 "사람이 웹브라우저로 접속해 볼 수 있는 것이라면 클로드 코드도 볼 수 있어야 한다"는 발상에서 출발한 Claude Code 플러그인이라고 설명한다. 한국어 소개 페이지의 강한 카피("포기는 배추 셀 때나 쓰는 말이다")를 인용하며, 공개된 웹페이지라면 결국 그것을 가져와 본다는 뜻이라고 한다.
> "여러분들이 직접 웹브라우저에 접속을 해서 볼 수 있는 것이라면은 우리 클로드 코드도 볼 수 있어야 되는 거 아니냐" [00:27]

강한 카피 탓에 "불법으로 우회해 데이터를 가져오는 거냐"는 오해가 생길 수 있지만 그건 아니라고 선을 긋는다.

### [01:00–01:44] 봇 차단과 403, 그리고 '단계 상승' 원리
에이전트를 차단하는 장치를 둔 사이트가 많다며 X, 레딧, 유튜브, 해커뉴스, 네이버, 쿠팡, 링크드인, 미디엄, 서브스택 등을 예로 든다. 화자 본인 경험으로, 쿠팡에 클로드 코드나 코덱스로 데이터를 몇 번 조회하다 보면 갑자기 막고 **403**을 던진다고 한다(주장). Insane-search는 공식적·합법적 방법을 다 써 보며 순차적으로 단계를 올려, 결국 웹브라우저에서 보이는 데이터를 다 볼 수 있게 해 준다는 원리다. "수단과 방법을 가리지 않되 불법적인 방법만 뺀다"고 정리한다.

### [01:44–02:09] 넘지 않는 선 — 로그인·결제 벽
불법의 예로 "로그인해야만 볼 수 있는 것을 불법 로그인으로 뚫는" 경우, "결제해야만 볼 수 있는(돈 내야 보이는 블로그 등)" 경우를 든다. 이런 것은 당연히 안 가져오며 거기까지는 안 간다고 명시한다. 클로드 코드 단독이면 다 막히지만, Insane-search를 쓰면 단계를 상승시키며 될 때까지 해서 "하여간 가져와 주겠다"는 비교가 소개 페이지에 명확히 돼 있다고 한다.

### [02:09–02:34] 오늘의 테스트 케이스 예고
오늘 시연할 케이스로 (1) 유튜브 영상의 메타데이터·자막 가져오기, (2) 네이버·쿠팡 검색을 진행하겠다고 예고한다. 잘 되는지 테스트하면서 "인세인 서치가 조금 더 흥했으면 하는 마음으로" 찍는다고 밝힌다. 뒤이어 나오는 원리 설명은 어려우니 안 봐도 된다고 덧붙인다.

### [02:34–02:49] 다시 강조 — 공개된 것만
소개 페이지가 마지막에 재차 강조하는 바를 옮긴다: 이건 공개된 것을 그냥 가져오는 것이지, 비밀스럽게 감춰진 것·로그인해야 볼 수 있는 것·돈 내야 볼 수 있는 것을 우회해 불법으로 가져오는 도구가 절대 아니다.

### [02:49–03:34] 제작자와 깃허브 별 응원 (돈 안 됨)
영어 페이지라 외국인이 만든 줄 오해할 수 있지만, 제작자는 한국에 사는 한국인이며 화자가 전화도 해 봤다고 한다(주장). 에이전트 시대에 이런 걸 만들어 그냥 배포하는 사람은 정말 좋은 존재라며 많이 써 주고 응원해 달라고 한다. 단, 응원한다고 돈 내는 것도 없고 화자에게 들어오는 돈도 없음을 강조. 깃허브의 별(star)은 개발자끼리 하는 인기 투표일 뿐 금전 보상이 없으니, 한 번씩 눌러 주면 제작자에게 큰 도움이 될 거라고 한다.

### [03:34–04:16] 클로드 코드로 이동, 작업 디렉토리 생성
클로드 코드로 이동해 시연을 시작한다. 선호 환경(에디터)에서 작업하며, 테스트만을 위한 디렉토리를 하나 만든다. 명령어는 `MKD`(make directory) 설명, 혹은 원하는 위치에서 GUI로 "새 폴더" 생성해도 된다고 안내. 자신은 `랩 인세인 search` 폴더를 만들어 그리로 이동하겠다고 한다.

### [04:16–05:10] 폴더 이동·claude 실행
폴더 이동은 `CD`(change directory) 뒤 한 칸 띄우고 폴더를 드래그 앤 드롭하면 쉽다고 반복 팁을 준다. 왼쪽 이름이 바뀐 걸 확인한 뒤 `claude`를 skip-permissions 옵션(발음 헷갈려 함)으로 실행한다. full-screen 렌더러 관련 문구는 중요치 않다며 넘어가고 설치를 시작한다.

### [05:10–06:18] 슬래시 명령으로 설치 (marketplace → install → reload)
소개 페이지의 install 명령을 복사해 폴더 위치에서 붙여넣어 엔터 → 설치가 안 됨. 알고 보니 **슬래시 명령**이라 셸이 아니라 **클로드 코드 안에서** 실행해야 함을 깨닫는다. 큰따옴표를 넣어 "이거 이용해서 이 플러그인 설치해라"라고 지시하는 방법도 보여 준다. 내장 슬래시 명령이므로 한 줄씩 클로드 코드 안에 넣어 진행: `successfully added marketplace ... plugin` → `plugin install insane-search` → 설치 후 `reload plugin` 하면 그 채팅 세션에서 인세인 서치를 쓸 수 있다고 정리한다.

### [06:18–07:01] 유튜브 자막 케이스 시작
설치 후에는 사용자가 더 할 게 없고, 평소 잘 안 가져와지던 데이터를 시도해 보면 된다고 한다. 예로 방금 자신이 업로드한(자막 작업이 끝난) 영상을 대상으로, "영상을 다 보기 귀찮을 때 자막을 가져오면 공부에 큰 도움이 된다"며 "이 영상의 자막을 가져와 봐"라고 지시한다.

### [07:01–08:08] 유튜브 자막 추출 성공 + 활용
클로드가 고민하다 예상대로 Insane-search를 사용해 유튜브 자막을 뽑겠다고 하고, 관련 스킬을 불러온다. 화자는 플러그인이 스킬 등을 묶어 놓은 것이라 설명한다(도구가 별을 달라고 요청하자 눌러 준다). 결과로 유튜브 메타데이터와 **한국어 자동 생성 자막까지** 쭉 가져오는 모습을 보여 준다. 유튜브로 학습하는 사람은 이것만 설치해도 자막 텍스트를 깔끔히 얻어 학습에 큰 도움이 되고, 뽑은 텍스트를 보기 좋은 학습용 웹페이지로 정리시키는 것도 가능하다고 한다.

### [08:08–09:20] 네이버 핫딜 → CSV 저장 시도
다음은 네이버. 최적가 핫딜의 키즈/라이프/푸드 카테고리 상품들을 "싹 다 가져와서 정보 정리해 CSV 파일로 저장하라"고 지시한다(핫딜이라 가격이 싼 데이터). 예전 브라이트 데이터 영상에서는 네이버가 따로 없어 못 했는데, 지피타쿠의 인세인 서치로는 될 수 있다고 기대한다. 클로드가 "공개 페이지 수집 작업이니 내부 API를 찾아야 한다"며 내부 API를 탐색하기 시작하고, GP타쿠 플러그인을 bash 명령으로 불러 동작시킨다.

### [09:20–10:16] 여러 방법 자동 시도, 마케터 활용
클로드가 Playwright도 써 보고, 본문도 받아오고, 상품 데이터 임베딩·체크 등 별의별 방법을 자동으로 시도한다. 화자는 생짜 클로드 코드로는 "이것도 해 봐 저것도 해 봐"를 다 시켜야 하는데 이걸 알아서 해 준다는 점을 강조. 오픈된(공개) 데이터라 보여야 한다고 보며, 안 되면 제작자에게 리포트를 보내 수정 요청하겠다고 한다. 네이버 쇼핑 데이터를 얻으면 가격 비교·상품 구성·이벤트 가격 산정이 필요한 마케터에게 유용하다고 활용처를 짚는다.

### [10:16–11:41] 308개 중복제거 완료, 클로드(코워크)에서는 안 됨
플러그인이라 Claude(코워크/데스크톱 계열) 환경에서도 쓸 수 있는지 확인하려 하지만, 그쪽에서는 인세인 서치가 안 보여 껐다 켜 보는 등 헤맨다("코워크를 잘 안 써서"라 함, 다음에 별도 영상 예고). 반면 클로드 코드에서는 "**308개가 중복 제거까지 해서 완벽하게** 되는" 결과를 확인한다(주장). 웬만하면 클로드 코드에서 하는 게 좋겠다고 정리.

### [11:41–12:00] CSV 저장 확인 — 3분 만에 핫딜 데이터
한때 API 오버로드로 CSV 저장이 중간에 멈춘 듯 보였으나(왜 저장하다 말았냐고 물어봄), 결국 폴더에 CSV 파일이 들어와 데이터가 다 채워진 걸 확인한다. "약 3분 만에" 핫딜 데이터를 싹 가져올 수 있다며, 냅다세일·파리바게트·망고밤 케이크 등 실제 항목을 눈으로 확인해 보여 준다.

### [12:00–12:34] 쿠팡·해외 사이트(알리익스프레스)로 확장
사이트 앤 퍼블릭 페이지는 다 된다는 얘기라며, 해외 사이트도 되는지 쿠팡에 이어 알리익스프레스를 시도한다. 알리익스프레스는 한때 유행했으나 봇탐지 때문에 잘 안 된다며(본인 경험) 식료품 카테고리로 들어가 "여기 있는 것 100개만 가져와서 CSV로 저장하라"고 지시한다.

### [12:34–13:07] "크롤링 강의 필요 없어질 것"
화자는 이건 무조건 된다고 기대하며, [의견] "크롤링 강의 몇 개는 없어지겠다, 내가 올린 무료 크롤링 강의도 이제 필요 없을 것 같다"고 말한다. 강한 봇 차단이 있어 공식적인 방법(엔진)으로 페이지를 받아오려 하는 중이고, 중간 출력에서 막히면 다른 방법을 시도하며, 클로드가 GP타쿠 플러그인을 계속 사용하는 모습을 짚는다.

### [13:07–14:07] 내부 API 원리 상세 설명
"핵심 발견: 페이지는 내부 API로 상품을 프리패치한다"는 클로드의 중간 출력을 근거로 원리를 풀어 설명한다. 우리가 화면에서 상품을 보는 순간, 내부적으로는 그 상품 블록을 채우는 API 콜이 존재하고, 사이트는 그 결과를 봇탐지 보안으로 감싼 뒤 화면에 그린다. 단순 페이지 크롤링은 보안이 먼저 동작해 막히지만, 원천 API 콜을 찾아 직접 호출하면 데이터를 가져올 수 있다 — 그래서 절대 불법이 아니라는 것. 함수명 불일치 등 시행착오를 거쳐 공식 제공 데이터 API를 찾아 상품 정보를 가져오려 시도한다.

### [14:07–15:00] 알리 100개 API 시도 + 크롤링의 실제 난이도
상품 피드 API를 찾은 것 같다고 하며, 100개 정도는 API로 불러오는 게 일도 아니라고 한다. 첫 값으로 "당일 도정 2025년산 햇살당진 나루살" 같은 실제 상품명이 오면 나이스하다고 기대. [의견] 상품 사업·가격 비교를 하는 사람에게 인세인 서치가 엄청 좋을 것이며, 자신은 켜 놓고 매일 쓸 것 같다고 한다. 다만 "직접 해 보면 (생짜로는) 완전 속 터진다, 안 된다"며 이 도구의 가치를 강조. 이 시점에도 알리에서는 에러가 계속 남.

### [15:00–16:00] 광고 아님 해명 + 관점
고생해서 만든 걸 공짜로 풀어도 되냐며 감탄하고, 업데이트도 열심히 하고 다른 사람들도 좋아한다고 전한다. 자신이 소개하는 이유는 웹브라우저로는 보이는데 클로드 코드로는 못 가져오던(봇탐지에 걸리던) 데이터를 서치하고 싶은 사람들을 위해서라고 설명. [의견] "눈으로 볼 수 있으면 에이전트로도 볼 수 있으면 좋지 않나"는 생각을 밝히고, 우회가 아니니 문제없을 거라고 본다.

### [16:00–17:00] 알리는 제일 안 됨 → 브라우저 자동화까지
화자는 알리익스프레스가 제일 안 된다고 인정한다. 하다하다 안 되니 클로드가 Playwright로 아예 웹브라우저를 띄워 작업하려는 단계까지 왔다며, 되게 만들려고 애쓰는 중이라고 한다. [의견] 바이브 코딩으로 직접 했어야 할 일을 알아서 다 진행해 주니 괜찮다고 평가. 자꾸 칭찬해 돈 받은 것 아니냐는 오해에 대해, 최근 유료 광고를 두 개 올려 오해가 몰렸지만 "이건 절대 광고가 아니고 내 수익 없다, 그냥 공짜니 써 보라"고 재차 해명한다(주장/미검증). 도구를 많이 써도 클로드 코드 구독 사용량(토큰)에서 나갈 뿐 별도 결제는 없다고 한다. 결국 해낼 것 같지만 다 기다릴 필요는 없다며 넘어간다.

### [17:00–17:42] 마무리
알리 데이터도 결국 다 받은 것 같다고 확인하고 넘어간다. "포기는 배추 (셀) 때나 쓰는 말", 공개된 페이지는 다 가져오는 인세인 서치를 꼭 써 보라고 권한다. 유용한 도구가 보이면 갑자기 찾아와 소개하고 사라지겠다며, [의견] 바이브 코딩에만 집중하지 말고 에이전트로 일상 업무를 돕는 방향도 생각해 보라고 조언한다. 궁금한 점은 댓글로 남기면 GP님에게 전달하겠다며 영상을 마친다.
