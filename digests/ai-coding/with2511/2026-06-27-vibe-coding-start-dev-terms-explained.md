## 요약 (주제별)

### 바이브 코딩의 정의와 진입 문턱
- 화자는 이 영상 하나로 바이브 코딩을 다 설명하면 "더 얘기할 게 없을 수도 있다"고 할 만큼 압축된 실습이라고 소개한다.
- 방식은 단순하다: Claude Code(또는 Curs)에서 ChatGPT 쓰듯 자연어로 요청하면 코드가 만들어진다. "ChatGPT 한 번이라도 써 본 사람이면 무조건 할 수 있다"는게 화자 주장(미검증).
- 다만 영상 중반에 화자 스스로 "무작정 쉽다고 한 건 정정하겠다"며, DB·백엔드·서버 같은 개념 이해가 필요한 지점에서 난도가 올라간다고 인정한다. [의견]

### 실습 환경 셋업 (도구·에디터)
- 편집기: Visual Studio Code. 빈 폴더 하나(화자는 "테크노트_당"이라는 이름)를 만들어 연다.
- Claude Code 설치: VS Code 익스텐션 검색에서 "클로드"로 찾아 "클로드 코드"를 설치. 이미 설치돼 있으면 인스톨 표시 대신 다른 상태가 뜬다.
- Cursor 사용자는 별도 설치 없이 에디터에서 바로 시작하면 된다.
- 설치 후 VS Code 안에 Claude Code 패널이 뜨고, 거기에 ChatGPT처럼 요청을 입력한다.

### 실전 절차 — 화면부터 만들기 (당근마켓 클론)
- "가장 쉽고 무식한 방법": 만들고 싶은 실제 사이트(당근마켓)에 들어가 화면을 캡처하고, 그 이미지를 Claude Code에 붙여넣은 뒤 "이대로 만들어 줘"라고 요청.
- 실제 프롬프트: "지금부터 당근마켓 같은 중고거래 사이트를 만들려고 해. 캡처 화면대로 물품 상세 화면을 만들어 줘."
- 결과물이 근사하게 나오며, "이걸 그대로 쓰겠다가 아니라 이런 식으로 한다"는 감을 잡는게 목적. [의견]
- 필요한 화면(상세 화면, 전체보기 화면, 물품 등록 화면 등)을 하나씩 추가로 요청해 서비스를 점진적으로 키워 간다.
- 파일은 VS Code 익스플로러에서 확인 가능(assets 이미지, index.html 등). 결과 확인은 우클릭 → "Open with Live Server" 또는 폴더에서 HTML을 브라우저로 직접 열기.
- 도구 승인: Claude Code가 작업 시 매번 "허용(어로우/allow)"을 눌러야 하는데, 매번 누르기 불편하면 "예스, 올(Yes to all)" 옵션으로 넘긴다.

### DB·백엔드로 확장 (기능 있는 서비스로)
- 정적 화면만으로는 데이터 입력·표시가 안 되므로, 데이터를 저장할 데이터베이스와 입력 화면이 필요하다.
- 요청: "물품 등록 화면을 만들어 주고, 물품 데이터를 저장할 DB도 SQLite로 설치해서 만들어 줘."
- DB 선택: 실무 대부분은 MySQL을 쓰지만, 화자는 가벼운 테스트용이라 SQLite를 골랐다. (선택은 상황에 따라 다르다는 점을 명시)
- Claude가 자동으로 진행한 것: 프로젝트 구조 설정, Express(노드 모듈) 기반 백엔드, SQLite DB 스키마, REST API 서버 작성, 상세 페이지를 DB 연동 동적 페이지로 전환, register.html 등록 화면 작성, 서버 실행 및 브라우저 테스트까지.
- 화자는 "테스트까지 다 해 주네요", "엄청나게 좋아졌다"고 반응. 매너온도 같은 세부 기능까지 반영된다는데 이건 직접 확인해 봐야 한다고 유보. [의견]

### 개발 용어 해설 (영상의 핵심 교육 파트)
- **프론트엔드 화면**: 아까 만든 HTML/CSS 화면은 "그냥 보여 준 것"이고, 실제 데이터를 가져와 보여 주는 게 동적 페이지.
- **백엔드**: 서버에 있는 서버·데이터베이스 등을 통칭. DB에 데이터가 곧바로 저장되는게 아니라, 보통 백엔드(API 서버)를 앞에 두고 그 뒤에서 DB와 데이터를 주고받는다.
- **Node.js**: 백엔드 서버를 만드는 소프트웨어 기술 중 하나. 같은 백엔드를 Spring·Java 등으로도 만들 수 있다.
- **Express**: HTTP 요청을 처리할 수 있는 노드 모듈(일종의 프레임워크).
- **스키마**: 테이블 간 관계를 포함한 전체 설계도 같은 것.
- **REST(API)**: 서버끼리(클라이언트-서버가) 데이터를 서로 주고받는 규약.
- **npm start**: 노드로 만들어진 서버를 구동(실행)하는 명령.
- **포트**: 프로그램들이 맞물려 뜨는 번호. 노드 서버를 3000번 포트로 띄우면 URL 뒤에 `:3000`을 붙여 접속.
- **localhost**: 지금 실행되고 있는 이 컴퓨터 자신을 가리키는 표현. `localhost:3000` = 이 컴퓨터의 3000번 포트로 접속.

### 서버 실행과 확인
- VS Code에서 Ctrl + 물결표(`~`, 숫자 1 왼쪽)를 누르면 터미널(도스창)이 열린다. 거기에 `npm start` 입력.
- 실행되면 "당근마켓 클론 서버 실행 중" 표시가 뜨고 서버가 구동 상태가 된다.
- 웹 브라우저에서 `localhost:3000`으로 접속하면 만든 사이트가 뜬다. 단, 이건 내 컴퓨터에 떠 있는 소프트웨어적 서버라는 점을 강조.
- 이후 화면에서 실제로 물품(가격/나눔 등)을 등록하면 조회 화면에 반영된다. 화자는 결과에 "올리고도 깜짝 놀랐다"고 반응. [의견]

### 반복 개선과 배포
- 여기서부터는 원하는 것을 계속 요청해 붙여 나가는 방식: "옵션 정보는 제외해 줘", "이 선택지는 추가해 줘", "리스트를 더 보여 줬으면 좋겠어" 등 자연어로 요구하면 착착 만들어진다.
- 배포: 완성된 걸 실제 서버에 올리고 DNS와 연결하면 된다. 배포·DNS 연결은 별도 영상을 참고하라고 안내(그 작업은 나중 단계).

### 학습 순서에 대한 조언 [의견]
- DB·SQLite·네트워크가 뭔지 다 알고 시작하려면 시간도 오래 걸리고 뭘 공부할지 감도 안 온다.
- 그래서 오히려 이렇게 무작정 만들어 보고, "DB가 뭐지? SQLite가 뭐지? 네트워크가 뭐지?" 하고 거꾸로 각 개념 영상을 찾아보는게 낫다는게 화자 입장.

### 세일즈 구조 (정보로서 1회 기록)
- 화자는 "기술루트"라는 이북(도서)에 상세 내용이 있다고 언급하고, 각 개념(DB, 백엔드, 프론트-백엔드-DB, 네트워크, 배포·DNS)마다 별도 영상을 올려 뒀으니 궁금하면 보라고 반복 유도한다.
- 마무리에 좋아요·댓글을 요청하며, 반응이 좋으면 이어지는 영상을 계속 올리겠다고 한다.

### [01:20–01:43] Claude Code / Cursor 설치

### [02:10–02:49] 캡처 이미지 붙여넣고 프롬프트 입력
당근마켓 화면을 캡처해 Claude Code에 붙여넣는다(이미지가 그대로 들어간다). 실제로 입력한 프롬프트는 "지금부터 당근마켓 같은 중고거래 사이트를 만들려고 해. 캡처 화면대로 물품 상세 화면을 만들어 줘"이다.

### [02:50–02:56] 첫 결과 — "시작이 반"
"시작이 반"이라며, 이렇게 요청하면 웹사이트가 근사하게 나온다고 말한다.

### [02:57–03:29] 기술 스택 선택 (Next.js/React vs 순수 HTML)
어떤 기술 스택으로 시작할지 물으면 고민이 생길 수 있다고 한다. Next.js는 프레임워크, React는 페이스북이 만든 라이브러리라고 짧게 소개하면서, 바이브 코딩 입장에서는 큰 차이가 없다고 말한다. 이번엔 간단한 테스트니까 순수 HTML/CSS/JavaScript로 하겠다고 선택한다.

> "바이브 코딩하는 입장에서 큰 차이는 없어요." [03:20]

### [03:39–03:55] 도구 승인 (allow) 처리
Claude Code가 작업하려면 "어로우(allow)"를 눌러 줘야 한다. 어로우를 안 하는 경우가 거의 없으니, 매번 누르기 불편하면 그냥 "예스, 올(Yes to all)"로 처리하면 된다고 안내한다.

### [03:56–04:10] 상세 페이지 완성 — 점진적 제작 개념
상세 페이지 만들기가 완료됐다며, 필요한 화면들을 이런 식으로 계속 만들어 나가는 것이라고 강조한다.

### [04:11–05:06] 만들어진 파일 확인과 브라우저 열기
VS Code 익스플로러에서 결과를 확인한다. assets 이미지, index.html(HTML 코드) 등이 생성돼 있다. 모르면 우클릭 후 "Open with Live Server"를 쓰거나, 익스플로러 폴더로 가서 HTML을 브라우저로 직접 열어도 된다. 원본 사이트와 비교하며 "그대로 쓰겠다가 아니라 이런 식으로 한다는 것"이고 상당히 그럴듯하게 만들어 준다고 정리한다. [의견]

### [05:07–05:56] 필요한 화면 추가와 DB 필요성 설명
각 필요한 화면이 있으면 그 화면을 그대로 만들어 달라고 하면 만들어진다. 여기서 DB까지 이야기하려는데, 화면에서 내용을 입력할 수 있어야 하고(입력 화면), 입력한게 보여져야 하므로(표시) 데이터베이스가 필요하다고 설명한다. 실제로 입력해서 보여 주는 것까지 되면 "서비스의 기능들이 하나씩 만들어져 가는 것"이라고 표현. 현재는 화면만 만든 상태라고 명확히 한다.

### [05:57–06:28] 등록 화면 + SQLite DB 요청, DB 선택 설명
"해당 내용들을 입력할 수 있도록 물품 등록 화면도 만들어 달라"고 하고, "물품 데이터를 저장할 수 있도록 DB도 SQLite로 설치해서 만들어 줘"라고 요청한다. 이 서비스엔 DB가 어쨌든 필요한데, MySQL을 쓸 수도 있고 다른 DB를 쓸 수도 있으며 대부분은 MySQL을 쓴다고 본다. 화자는 가벼운 테스트용이라 SQLite 정도로 이야기하는 상황이라고 설명한다.

DB 설치, 서버, 네트워크 같은 것들이 조금 어렵다고 인정한다. 다 해 달라고 하면 해 주지만, 이게 어떤 건지 알고 골라 써야 한다는 점이 진입 장벽이라고 말한다. 이에 대한 디테일한 내용은 "기술루트"라는 이북(도서)에도 있고, 각 영상들에 자세하게 올려 뒀다고 안내한다. (세일즈 장치 — 1회 기록)

### [08:00–08:58] 백엔드·Node.js 개념 해설
DB에 데이터가 곧바로 저장되는게 아니라, 보통 백엔드(API 서버 역할)를 앞에 두고 그 뒤에서 DB와 데이터를 주고받는다고 설명한다. 그 기술 중 하나가 Node.js이고, Node.js로 백엔드 서버를 만들 수 있으며 Spring이나 Java로도 만들 수 있다. 이런 걸 통틀어 백엔드라고 한다며, 자신이 올린 "프론트엔드-백엔드-DB" 영상을 보면 더 쉽고 자세히 알 수 있다고 안내한다. 백엔드는 "서버에 있는 서버들 또는 데이터베이스"를 말한다고 정리.

### [08:58–09:11] 지금 상황 정리
데이터를 저장하기 위해 중간에 이게 필요해서 지금 설치·추가하는 상황이라고 요약한다.

### [09:12–10:03] 진행 상황 해설 — Express·스키마·REST API
Claude의 진행 로그를 보며 용어를 푼다. Express는 HTTP를 처리할 수 있는 일종의 모듈. SQLite DB 스키마의 "스키마"는 테이블 간 관계를 포함한 전체 설계도. REST API 서버를 작성해 데이터를 등록한다. API 서버는 지금 Express, 즉 Node.js로 만들어지고 있다. Node.js는 백엔드 서버(소프트웨어적인 것)를 뜻하고, 그것들이 서로 주고받는 규약이 REST라고 이해하면 된다고 설명한다. 진행 로그를 보며 궁금할 만한 것들을 하나씩 다시 설명해 주는 상황이라고 밝힌다.

### [10:03–10:41] 동적 페이지 전환과 등록 화면 예고
상세 페이지를 DB 연동 동적 페이지로 전환한다고 로그를 읽는다. 아까는 HTML 태그와 CSS로 화면을 그냥 보여 준 것이고, 이제 실제 데이터를 가져와 보여 주는 작업이라고 설명. register.html(물품 등록 화면) 작성이 이제 진행되는데, 지금까지 물품 상세 화면만 있었고 등록 화면은 없었으니 새로 만들 예정이라고 한다. 서버 실행 후 등록·상세 플로우를 브라우저 테스트까지 한다고 로그에 적혀 있다.

### [10:53–11:09] 자동 테스트까지 수행
"엄청나게 좋아졌네요, 테스트까지 다 해 주네요"라고 반응. 등록 → SQLite 저장 → 상세 페이지 동적 렌더링까지 보고, 매너온도까지 반영된다는데 이건 직접 해 봐야 한다고 유보하며 일단 해 보겠다고 한다. [의견]

### [11:19–11:41] 완료 — 구성 요소 재정리
작업이 완료됐다며 구성 요소를 정리한다. Node.js 백엔드 서버로 띄우고 SQLite에 저장하는 데이터베이스가 있고, DB도 생성됐다. API 서버(데이터를 주고받는 서버), 상세 페이지, 물품 등록 페이지가 만들어졌다고 정리한다.

### [11:41–12:33] 용어 해설 — npm start·포트·localhost
몇 가지 용어를 더 푼다. npm start는 노드로 만들어진 서버를 구동(실행)하는 것. 포트는 프로그램들이 맞물려 뜨는 번호로, 이 노드 서버를 띄우기 위한 포트다. 3000번으로 접속하면 이 노드 서버로 접속하는 것이고, 포트는 URL 뒤에 붙인다. localhost는 지금 실행되고 있는 이 컴퓨터를 가리키는 표현이며, `localhost:3000`은 이 컴퓨터의 3000번 포트(현재 실행 중인 서버의 포트)로 접속하라는 URL이 된다고 설명한다.

### [12:34–13:03] 터미널 열고 npm start
편하게 하려면 VS Code에서 Ctrl + 물결표(숫자 1 왼쪽의 `~`)를 누르면 터미널(도스창)이 나온다. 거기에 `npm start`를 입력하라는 의미라고 안내. 실행하면 "당근마켓 클론 서버 실행 중"이 뜨고 서버가 구동 상태가 된다.

### [13:03–13:47] 브라우저 접속 — localhost:3000
접속하려면 웹 브라우저를 켜고 `localhost:3000`으로 들어가면 된다. 접속에 성공하고, 이건 이 컴퓨터에 떠 있는 소프트웨어적 서버라는 점을 강조한다. 실제로는 나중에 어딘가에 있는 실제 서버로 올리는 작업을 별도로 하게 된다고 언급. 화면에 물품 등록 버튼도 생겼다.
