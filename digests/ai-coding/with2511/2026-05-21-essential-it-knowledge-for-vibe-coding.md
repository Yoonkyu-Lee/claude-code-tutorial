## 요약 (주제별)

### 1. 이 영상의 목적
- 화자([불명확: 원문 "기슬로트 알렉"])가 직접 바이브 코딩으로 개발·외주 개발을 해보면서, AI가 대부분 해주더라도 "그럼에도 알아야 했던" 필수 IT 지식을 되짚어 정리한 강의형 영상. [의견] AI가 많이 해주지만 그래도 알아야 되는 것들이 있었다는 것이 전제.

### 2. AI 개발 도구와 개념
- LLM(Large Language Model) = 언어가 학습된 인공지능. 대표적으로 ChatGPT는 질문-답변형 대화 AI.
- 개발할 때는 ChatGPT보다 Gemini CLI, Cursor, Claude가 많이 언급됨. 안티그라비티(Antigravity) 등 다른 도구도 존재.
- CLI(Command Line Interface) = 커맨드 라인. 윈도우에서 CMD 치면 뜨는 명령어 프롬프트처럼 한 줄씩 명령어를 입력하는 형태.
- IDE(Integrated Development Environment, 통합 개발 환경) = 소스 편집 + 컴파일/빌드/실행 + 소스 관리를 한 곳에서. Cursor는 IDE 형태라 편하고, Claude는 더 전문적인 개발 기능 제공. [의견] 화자는 Cursor가 IDE 형태여서 쉽게 접했다고 함.
- VS Code(Visual Studio Code)가 대표 IDE. Cursor는 자체에 IDE 내장, Claude는 VS Code에 확장 모듈(extension) 설치해서 사용. 설치하면 아이콘이 뜨고 연동됨.
- 도구 우측의 채팅창에 원하는 것을 입력하면 바로 만들어 주는 방식.

### 3. AI 개발의 핵심 개념 진화
- 개념 발전 순서: 프롬프트(prompt, 질문을 체계화) → 컨텍스트(context, 전후 상황을 다 전달해 AI가 스스로 판단하게 함) → 하네스(harness)·에이전트(agent). Claude 이후 여러 에이전트를 써서 기존 코드의 테스트·안전성 확보 등으로 체계화되는 흐름. [의견]

### 4. 소스·언어·빌드
- AI는 주로 기존 개발자의 코딩 역할을 하고, 코딩 전 설계·계획도 잡음. 개발 결과물 = 소스.
- 대표 언어: Java, Python, JavaScript, HTML. 언어에 따라 소스 형태가 다름.
- 소스는 컴파일되어 실행되거나, 실행 환경·라이브러리까지 포함해 한 덩어리로 빌드(build)됨.

### 5. 소스 관리 (Git/GitHub)
- 소스는 주로 Git(GitHub)에서 프로젝트별로 관리. 회원가입·로그인 후 프로젝트별 소스 생성·수정.
- 도구와 연결해 소스를 올리고 내려받는 git 명령어를 이해하는 것이 좋음.

### 6. 프론트엔드 / 백엔드 구분
- 화면(프론트엔드): 웹은 HTML/CSS/JavaScript(웹서버에서 구동), 그 외 React, Flutter, React Native(Android/iOS) 등 프레임워크·라이브러리.
- 프론트엔드 결과물은 앱스토어·구글 플레이에 올라가거나, 웹서버에 올라가 응답으로 화면이 보임.
- 백엔드(서버): Spring Boot, Node.js, FastAPI(Python) 등으로 서버 프로그램을 만듦.

### 7. 서버·포트·배포
- 서버 프로그램을 "띄운다" = 실행. 실행 시 포트(번호)와 함께 뜸. 포트 예: 22(SSH), 80(웹), 443, 3306(DB). 서버 도착 시 포트로 해당 프로그램을 찾아감.
- 로컬호스트(localhost) = 내 컴퓨터 자체를 가리키는 지칭.
- 서버 프로그램은 실제 서버(주로 데이터센터·클라우드)로 올라가야 데이터 저장 및 수많은 앱 접속을 처리 가능.
- CI/CD = 서버에 올리는 배포를 자동화하는 것. 팀·회사 개발 시 자동 배포를 미리 설정. [불명확: 자막에서 "CCD"·"CICD"로 혼용됨 → 통상 CI/CD]
- 간편 배포 서비스: Cloudflare(자막의 "클라우드 플레이어"), Vercel(버셀). 소스만 전달하면 서버로 올려지고 접속 URL 제공.
- 제대로 된 서버가 필요하면 클라우드(AWS, Azure). AWS(Amazon Web Services)를 가장 많이 씀. 서버 OS는 Windows 또는 Linux, 주로 Linux.

### 8. Cloudflare Pages 데모
- 화자는 자신의 마인드맵(테크맵)·위키·네트워크 구성도·클라우드 구성도 등을 Cloudflare에 올려둠.
- 절차: Workers & Pages → Create Application → Git 연결 또는 직접 시작 → 이름 생성 → Create Project → 폴더를 드래그 앤 드롭 → Deploy site. 아주 간단하게 페이지를 올려주는 서비스라고 설명.

### 9. HTTP·API·DB
- HTTP 프로토콜 = HTML을 주고받는 약속. 웹서버는 80포트로 뜨는 예. 페이지를 받거나 API로 통신.
- API(Application Program Interface) = GET/POST/PUT/DELETE 메소드 + URL. GET은 데이터를 가져오는 명령. 응답은 주로 JSON 포맷(예: 위치·날씨 정보). 클라이언트는 응답을 받아 화면 표시 등 처리.
- 데이터베이스: MySQL, MongoDB 등. 회원 관리·게시판 등 보여주는 정보(데이터)를 저장. 서버가 DB와 주고받는 명령어가 SQL(데이터 가져오기·저장).

### 10. 세일즈/CTA 구조
- 참고용으로 자신의 AWS 관련 영상을 안내. 마무리에 "도움이 됐으면 좋아요를 눌러 달라, 계속 관련 영상을 올리겠다"는 좋아요·구독 유도 CTA.

도구를 선택해 AI 개발을 하게 되면, AI는 주로 기존 개발자가 하던 코딩 역할을 하고 코딩 전 설계·계획 과정도 잡는다. 개발 결과로 나오는 것은 "소스"라는 형태다.

### [06:11–06:44] 언어와 컴파일·빌드
소스는 언어가 어떤 것이냐에 따라 형태가 조금씩 다르다. 대표 언어로 Java, Python, JavaScript, HTML 등이 있다. AI가 개발해도 이런 언어를 통해 소스가 만들어지고, 그 소스가 컴파일되어 실행되거나, 실행 환경과 라이브러리까지 포함해 한 덩어리로 빌드된다.

### [06:44–07:37] Git/GitHub로 소스 관리
소스는 주로 Git(GitHub)에서 관리한다. GitHub는 회원가입·로그인 후 프로젝트별로 소스를 관리하고 생성·수정할 수 있으며, 개발 도구와 연결해 소스를 올릴 수도 있다. 이때 Git 사용법(소스 올리기·내려받기 명령어)을 이해하는 것이 좋다고 한다. 빌드까지 하면 소스가 올라갈 수 있는 형태가 된다.

### [07:37–08:23] 프론트엔드(화면) 개발 개념
소스는 서버로 올릴 수도, 앱 형태로 만들 수도 있다. 화면을 만드는 개념으로는 웹의 HTML/CSS/JavaScript가 대표적이며 웹서버에 올라가 구동돼야 한다. 그 외 React, Flutter 같은 프레임워크, React Native(Android/iOS)가 각각 프레임워크이자 라이브러리로, 화면·앱을 만드는 데 쓰인다.

### [08:23–08:57] 프론트엔드 배포처와 백엔드 도입
프론트엔드 결과물은 앱스토어나 구글 플레이에 올라가거나, 웹서버에 올라가 웹서버가 응답함으로써 화면이 보이는 형태가 된다. 그다음 백엔드 서버 쪽에는 서버가 올라간다.

### [08:57–09:15] 백엔드 프레임워크
백엔드 서버의 대표 프레임워크로 Spring Boot, 그리고 많이 쓰는 Node.js, Python 언어로 만드는 FastAPI 등을 든다. 이런 것들로 백엔드(서버 프로그램)를 만든다.

### [09:15–10:30] 서버 프로그램 "띄우기"와 포트
서버 프로그램은 서버에 저장되고 실행("띄운다")될 때 포트와 함께 뜬다. 포트는 번호로, 22·443·80·3306 같은 숫자다. 서버에 도착했을 때 포트로 자기 프로그램을 찾아가는데, 80은 웹서버, 22는 SSH, DB 등 각각의 포트가 어떤 프로그램인지를 가리킨다([불명확: 자막의 "쉐이고 HP"는 SSH 등 음차 오인식으로 보임]).

### [10:30–10:48] 로컬호스트
내 컴퓨터에서 프로그램을 띄운다는 개념이 로컬호스트(localhost)이며, 로컬호스트는 다른 말이 아니라 내 컴퓨터 자체를 가리키는 지정이라고 설명한다.

### [10:48–11:45] 서버 배포의 이유와 CI/CD
서버 프로그램은 실제 서버로 올라가야 한다. 클라이언트(앱)가 내 단말에 설치되듯이, 서버 프로그램은 서버로 올라가야 한다. 이유는 서버 뒤에 데이터가 저장돼 있고, 이 서버를 통해 수많은 앱이 접속해 데이터를 제공받기 위함이다. 서버는 보통 내 컴퓨터가 아니라 특정 데이터센터(예: 클라우드)에 있다. 이렇게 서버를 올리는 것을 자동화하는 것을 CI/CD라고 하며([불명확: 자막에 "CCD"·"CICD" 혼용]), 팀이나 회사에서 개발할 때 자동 배포를 미리 설정해 둔다.

### [11:45–12:22] 간편 배포 서비스 (Cloudflare·Vercel)
클라우드에 배포하는 것을 더 편하게 하는 방법으로 배포 서비스를 이용할 수 있다. 대표적으로 Cloudflare(자막의 "클라우드 플레이어"로 오인식됨)나 Vercel(버셀) 등이 있다. 소스만 전달하면 서버로 올려지고, 서버에 접속할 수 있는 URL을 제공해 준다. 그 URL로 접속하면 바로 자신이 올린 소스에 접근 가능하다.

### [12:22–13:55] Cloudflare Pages 데모
화자는 자신의 마인드맵(테크맵)·위키·네트워크 구성도·클라우드 구성도 등을 Cloudflare에 올려뒀다고 하며, 유튜브 연결도 돼 있다고 한다. 올릴 때 Workers & Pages로 가서 Create Application으로 만들 수 있고, Git과 연결하거나 그냥 시작(직접 페이지 보유 시)하는 방식이 있다. 이름을 하나 생성하고 Create Project를 하면 생성되며, 폴더째로 드래그 앤 드롭한 뒤 Deploy site를 하면 페이지를 올려주는, 아주 간단하게 이용할 수 있는 서비스라고 설명한다.

### [13:55–14:27] 제대로 된 서버는 클라우드(AWS·Azure)
Cloudflare·Vercel을 이용할 수도 있지만, 서버를 제대로 갖춰야 한다면 클라우드를 이용하는 것이 좋다. AWS와 Azure가 있고, 클라우드는 AWS를 가장 많이 쓴다. AWS는 Amazon Web Services이며, 관련 내용은 자신의 AWS 영상에서 많이 다루니 그 영상을 참고하라고 안내한다.

### [14:27–15:06] AWS 서버와 OS
AWS를 이용한다는 것은 서버를 이용한다는 것인데, 서버의 OS는 Windows나 Linux이고 주로 Linux를 많이 쓴다. 서버 쪽에 서버 프로그램이 올라갔을 때, 그 프로그램은 실제 서버에 저장되고 포트와 함께 떠 있다고 이해하면 된다.

### [15:06–16:03] HTTP 프로토콜과 API 도입
간단한 예로 웹서버 80포트가 서버에서 구동되고 있을 때, 앱이 설치돼 있거나 URL로 접속할 때의 약속(프로토콜)을 HTTP 프로토콜이라 한다. HTTP는 HTML을 주고받을 수 있는 약속이다. 서버가 제공하는 웹서버(80포트)에서 페이지들을 받을 수도 있고, 또 하나는 API 형태로 단말과 통신할 수도 있다.

### [16:03–16:46] API의 구조와 JSON 응답
API는 Application Program Interface로, 애플리케이션이 제공해 데이터를 주고받을 수 있는 호출 형태다. API는 GET·POST·PUT·DELETE라는 메소드와 함께 URL을 갖는다. GET은 데이터를 가져오는 명령이며, 응답은 주로 JSON 포맷을 가장 많이 사용한다. 예를 들어 JSON으로 "위치는 어떻고 날씨는 어때요"라고 응답해 주면, 클라이언트는 응답을 받아 화면에 보여주거나 어떤 처리를 하게 된다.

### [16:46–17:29] 데이터베이스와 SQL
API가 개발에서 상당히 중요하며, 또 하나 중요한 것이 데이터베이스다. 데이터가 실제 저장되는 것이 데이터베이스로, 대표적으로 MySQL·MongoDB 등을 많이 쓴다. 회원 관리·게시판 등 보여주는 정보가 다 데이터이며, 이를 저장하려면 데이터베이스를 이용한다. 서버가 DB와 데이터를 주고받을 때의 명령어를 SQL이라 하며, 데이터를 가져오거나 저장하는 명령어다.

전반적으로 AI로 개발하더라도 곳곳에 어느 정도 알아야 할 부분들이 있어서 이것들 위주로 다시 정리해 봤다고 마무리한다. 도움이 되고 좋았다면 좋아요를 눌러 달라, 그러면 계속해서 관련 영상을 올리겠다는 CTA로 끝낸다.
