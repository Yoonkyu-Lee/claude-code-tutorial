## 핵심 개념

### 1. 활용 메타의 3단계 진화
화자는 AI 활용법의 메타가 빠르게 바뀌고 있다고 본다.

> "메타가 좀 빨리 바뀌죠?" [00:06]

| 단계 | 정의 (화자 표현) | 핵심 질문 |
|---|---|---|
| 프롬프트 엔지니어링 | 질문만 잘하면 된다 / 역할을 줘라 / 단계별로 시켜라 / 예시를 넣어라 | "뭘 물어볼까" [03:32] |
| 컨텍스트 엔지니어링 | AI한테 뭘 보여줄지 설계 | "뭘 보여줄까" [03:35] |
| 하니스 엔지니어링 | 컨텍스트 + MCP + Skill + Agent를 다 합친 전체 환경 설계 | "AI를 감싸는 전체 환경을 어떻게 설계할까" [03:37] |

ChatGPT 초기에는 프롬프트 엔지니어링이 전부였다.

> "채 GPT 처음 났을 때 기억하시죠? 질문만 잘하면 된다. 역할을 줘라. 단계별로 시켜라. 예시를 넣어라. 이런게 프롬프트 엔지니어링이었어요." [00:13]

### 2. 왜 메타가 바뀌었나 — AI가 멀티스텝 에이전트로
질문 한 번/대답 한 번 수준을 넘어, AI가 혼자 수십~수백 단계를 실행하는 시대가 됐다는 것이 메타 변화의 근거.

> "AI가 이제 질문 한 번, 대답 한 번 하는 수준이 아닌 거예요. 혼자서 수십계, 수백 단계를 알아서 실행하는 시대가 된 거거든요." [00:36]

근거로 든 사례:
- OpenAI는 Codex라는 AI 에이전트만으로 내부 제품을 만들고 있다 [00:25]
- Manus라는 서비스는 AI 하나가 한 작업에서 평균 50번이나 도구를 쓴다 [00:30]

### 3. 컨텍스트 엔지니어링의 등장 (25년)
> "25년에 컨텍스트 엔지니어링이라는 말이 나왔어요." [00:43]

화자가 인용한 외부 근거 두 가지:
- Andrej Karpathy(테슬라 AI를 이끌던 인물)가 "프롬프트보다 컨텍스트 엔지니어링이 핵심"이라고 했다 [00:49]<sup>주1</sup>
- Gartner가 "프롬프트 엔지니어링 시대는 끝났다"고 공식 선언했다 [00:54]

> "질문을 잘 쓰는게 아니라 AI한테 뭘 보여 줄지 설계하는게 더 중요해진 거예요." [00:59]

### 4. 하니스 엔지니어링의 정의
> "하니스 엔지니어링은요. 컨텍스트 엔지니어링을 포함해서 AI 에이전트를 감싸[는] 모든 설정 파일, 그러니까 프로젝트 자체를 말하는 걸 의미합니다." [01:09]

하니스를 구성하는 요소 (화자 열거):
- **CLAUDE.md** 같은 프로젝트 지시 사항 [01:20]
- **MCP 서버 설정** — AI가 쓸 수 있는 외부 도구 연결 [01:22]
- **Skill 파일** — 전문 기술을 가르치는 설정 [01:26]
- **Agent 설정** — 유저가 커스텀 전문가를 주는 설정 [01:30]

> "이런 것들이 전부 합쳐진게 하니스라고 보시면 됩니다. 컨텍스트 엔지니어링이 하니스의 일부인 거죠." [01:35]

### 5. 주방 비유
화자가 하니스의 각 구성 요소를 주방에 빗댄다.

> "컨텍스트 엔지니어링이 요리사[에게] 좋은 재료를 골라주는 거라면 MCP 설정은 외부 업체랑 연결하는 어댑터 같은 거예요. 수산시장 정육점 농장이랑 직통으로 연결해 주는 거죠." [01:43]

| 하니스 요소 | 주방 비유 |
|---|---|
| 컨텍스트 엔지니어링 | 좋은 재료를 골라줌 |
| MCP 설정 | 외부 업체와 연결하는 어댑터 (수산시장·정육점·농장 직통) |
| Skill 파일 | 레시피를 정리해 둠 |
| Agent | 셰프를 만듦 |
| 하니스 엔지니어링 (전체) | 주방 전체를 설계 |

> "아무리 좋은 재료를 줘도 주방이 엉망이면 좋은 요리가 나올 수 없잖아요. 반대로 주방이 잘 세팅되어 있으면 평범한 재료로도 훌륭한 요리가 나오는 거죠." [02:00]

### 6. 핵심 명제 — 모델이 아니라 하니스가 병목
> "모델이 병목이 아니라 하니스가 병목이다." [02:12]

같은 모델이라도 환경 세팅에 따라 결과가 10배 차이 날 수 있다는 주장.

> "실제로는 같은 모델이라도 환경을 어떻게 세팅하느냐에 따라 결과가 열배 차이가 날 수가 있어요." [02:22]

축구 비유:
> "똑같은 축구 선수인데 좋은 감독 밑에서는 발롱도르를 타고 엉망인 팀에서는 벤치만 달구는 거랑 같은 거예요." [02:29]<sup>주2</sup>

## 카탈로그 — 인상적인 사례: OpenAI Codex 내부 실험

화자가 "진짜 인상적인 사례"로 강조한 부분. OpenAI가 25년 8월부터 시작한 실험이다 [02:41].

자기들이 만든 Codex 에이전트만으로 내부 제품을 만들기 시작한 결과 [02:45]:
- 수동으로 작성된 코드가 0 (제로) [02:49]
- 5개월 만에 100만 줄의 코드 생성 [02:52]
- 1500개의 Pull Request 머지 [02:54]
- 엔지니어 한 명이 하루 평균 3.5개의 작업 수행 [02:57]

**화자가 강조한 교훈 ("진짜 교[훈]이 뭐냐면")**:
> "처음에는 느렸대요. 생각보다 훨씬. 왜 [느]렸냐면요. 코덱스가 무능한게 아닙니다. 하니스 세팅이 제대로 안 됐기 때문입니다. 환경 설정이 부족하고 도구 연결이 어설프고 에러 복구 로직이 없었던 거예요." [03:01]

이걸 하나하나 잡아 나가니 그때부터 폭발적으로 성과가 나왔다 [03:14].

OpenAI 내부 발언으로 인용된 것:
> "엔지니어링 팀의 주된 역할이 에이전트가 유용한 일을 할 수 있게 만드는 것이 됐다." [03:19]

> "코드를 짜는게 아니라 에이전트가 코드를 잘 짤 수 있는 환경을 만드는게 일이 된 거예요." [03:25]

## 팁 — "그래서 나는 뭘 하면 되는데"

화자가 "오늘 영상에서 가장 중요한 부분"이라고 명시한 실행 가이드 [03:45]. 세 가지로 시작하라.

### 1. CLAUDE.md 같은 컨텍스트 파일을 제대로 써라
> "클로드.md 같은 컨텍스트 파일을 제발 제대로 쓰세요. 이게 하니스의 기본 중에 기본입니다." [03:52]

내용: "이 프로젝트는 이런 구조고 이런 규칙을 따르고 이런 패턴을 써라"라고 알려주는 것 [03:58].

### 2. MCP 서버를 세팅하라
> "MCP가 뭐냐면요. AI 에이전트가 외부 도구를 쓸 수 있게 연결해 주는 거예요. 브라우저 자동화, 문서 검색, 디자인 도구 연결 이런 것들이요." [04:04]

**주의 (화자가 강조한 대조)**:
> "다만 이때 MCP를 많이 주면 줄수록 좋지 않습니다. 또 MCP를 사용할 때 토큰이 터지지 않도록 여러 가지 테크닉들을 써야 돼요." [04:12]

이 토큰 관련 테크닉은 후속 영상에서 다루겠다고 예고 [04:21].

### 3. Skill 파일을 만들어라
> "스킬이 뭐냐면요. AI한테 특정 작업의 전문 지식을 파일로 정리해 주는 거예요. 영상 편집할 때 이 워크플로우로 해. 코드 리뷰는 이 기준으로 해. 이런 걸 파일로 만들어 두면 AI가 그 분야의 전문가처럼 일하는 거예요." [04:25]

## 핵심 주장 — 하니스가 진짜 경쟁 우위

화자가 "마지막으로 하나만 더"라며 강조한 대조 구문 [04:38].

> "모델은 몇 주 만에 따라잡을 수 있어요. GPT가 좋으면 클로드가 쫓아오고 [Gemini]도 올라오잖아요. 근데 하니스는요. 잘 만들려면 시간이 좀 걸립니다." [04:42]<sup>주3</sup>

하니스를 잘 만들려면 익혀야 할 것: 도구 연결, 에러 처리, 승인 흐름, 그리고 개발자가 어떻게 쓰느냐 방법론 [04:48].

> "그래서 지금 하니스에 투자하는 회사가 나중에 진짜 경쟁 [우]위를 가지게 될 거예요. 모델은 다 비슷해질 수 있어도 하니스는 쉽게 복[제]가 안 되거든요." [04:55]

개인 차원의 결론:
> "AI 도구를 쓸 줄 아는 건 이제 기본입니다. 차이를 만드는 건 그 AI를 얼마나 잘 세팅하느냐 마느냐입니다." [05:08]

## 최종 정리 (화자의 4줄 요약)

화자가 영상 끝에 직접 정리한 네 가지 [05:15]:
1. 하니스 엔지니어링 = 컨텍스트 엔지니어링 + MCP + Skill + Agent 설정까지 합친 전체 환경 설계
2. 같은 AI 모델이라도 하니스에 따라 10배 차이 날 수 있다
3. OpenAI도 코드를 짜는 게 아니라 에이전트 환경을 만드는 게 1순위 일이 됐다
4. 컨텍스트 파일 + MCP 연결 + Skill 정리, 이 세 가지부터 시작하면 된다

> "25년이 컨텍스트 엔지니어링의 해였다면 26년은 하니스의 해입니다. AI가 아무리 똑똑해져도 결국 환경을 어떻게 만들어 주느냐가 결과를 결정합니다." [05:36]
